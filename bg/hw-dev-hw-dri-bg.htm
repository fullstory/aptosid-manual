Инструкции за хардуер който изисква несвободен софтуер

Поради сложностите на закона, sidux live-cd-то предоставя само dfsg-свободен софтуер.

Свободни, с отворен код драйвери за nVidia, ATI, Radeon, Intel

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel are pre-installed with sidux.

Intel drivers should work perfectly for 2D and 3D video acceleration as the drivers are included in the Intel free series.

If you have been running proprietary drivers and wish to revert back to opensource drivers you need to edit /etc/X11/xorg.conf, HOWEVER YOU MUST MAKE A BACKUP COPY FIRST, as editing /etc/X11/xorg.conf incorrectly will result in many tears by you.

За да промените /etc/X11/xorg.conf цъкнете с десен бутон на мишката, и го отворете с права на root. Намерете секцията SECTION DEVICE, и променете в реда за видео драйвера името му, със nv ИЛИ ati OR intel (to name but a few)..

Editing xorg.conf is entirely at your own risk!!

"Sources List" файлове

Тъй като sidux iso-образа съдържа само dfsg-свободен софтуер, сигурно ще искате да добавите contrib/ и non-free към редовете във вашите файлове /etc/apt/sources.list.d/debian.list и /etc/apt/sources.list.d/sources.list и /etc/apt/sources.list.d/sidux.list или /etc/apt/sources.list и да се уверите че имате интернет:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Binary, closed source drivers for: nVidia with dmakms

NOTE: Use apt-cache search nvidia and apt-cache show <package> to acertain the correct driver for you. Currently Debian sid only supports 3d nvidia drivers ≥ GeForce 6xxx. For cards older than the 6xxx series see nvnews

Prepare the module:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Then activate Dynamic Module-Assistant Kernel Module Support (dmakms) for nvidia, so that the next time the Linux kernel is updated a nvidia module is prepared for it too, without manual intervention. To do that, add nvidia-kernel-source to the /etc/default/dmakms configuration file:

echo nvidia-kernel-source >> /etc/default/dmakms

Then:

m-a a-i nvidia-kernel-source

Followed by:

apt-get install nvidia-glx

Next alter xorg.conf:

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Reboot PC for installation of the module to take effect.

When xorg updates you only need to reinstall nvidia-glx:

apt-get install --reinstall nvidia-glx

When a new Debian sid nvidia driver becomes available thru apt:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Reboot PC for installation of the module to take effect.

Module load failure

Should nvidia fail to load, for whatever reason:

modprobe nvidia

Then reboot the computer.

Should the module still fail to load:

 m-a a-i -f nvidia-kernel-source

This rebuilds the module then reboot.

Read:

$ /usr/share/doc/dmakms

ATI, Radeon and other video drivers

2D Drivers

Свободните драйверите за X сървъра предоставяни от X.Org (вижте xserver-xorg за подробности), обезпечават поддръжка за 2d графика за NVIDIA Riva, TNT, GeForce, и Quadro видеокартите; ATI Mach, Rage, Radeon, и FireGL видеокартите, както и atimisc, r128 and radeon под-драйверите. Radeon and Intel both support 2d acceleration (textured xv) for video playback.

3D Drivers

Some ATI cards also support 3D, so far up to the r5xx chipsets.

To automatically inherit newly packaged non-free firmware when they get updated for 2D and 3D video cards:

apt-get install firmware-linux

Then reboot the computer.

Firmware drivers

Необходимия firmware може да бъде определен използвайки следната номерация на устройство/firmware , или чрез използване на новия скрипт на sidux, fw-detect (намира се в пакета sidux-scripts).

$ fw-detect

Добра практика е да извършите modprobe -r <modulename> преди modprobe

Записване върху NTFS-раздели с помощта на ntfs-3g

ПРЕДУПРЕЖДЕНИЕ: Макар че ntfs-3g е деклареран като "стабилен", никога не го използвайте без да сте направили резервни копия на външен носител, и разбира се не на продукционни системи! Ако го правите - това е на ваша отговорност, не е изключена възможността да изгубите всички данни от съответния раздел!

В терминала въведете следните команди: Вижте също Разделяте на диска с cfdisk - именоване на разделите

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
За да излезете от терминала въведете: exit

Сега вашият NTFS-раздел би трябвало да е монтиран rw (с права и за четене, и за запис), и би трябвало да можете да записвате на него. Но отново, внимавайте! Използвайте само в крайни ситуации, не се препоръчва за ежедневна употреба.
