Rady pro hardware s nesvobodnými požadavky

aptosid live-ISO bude dodáváno pouze s dfsk-svobodným softwarem.

Seznam zdrojů

Jelikož aptosid obsahuje pouze dfsk svobodný software, je možné, že budete chtít přidat contrib non-free do vašeho /etc/apt/sources.list.d/debian.list a /etc/apt/sources.list.d/aptosid.list. Vaše řádky by nyní měly vypadat takto, v závislosti na volbě vašeho zrcadla:

# Debian
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# aptosid
deb http://aptosid.com/debian/ sid main fix.main
#deb-src http://aptosid.com/debian/ sid main fix.main

Po změně vašich zdrojů budete muset zadat apt-get update před vyhledáváním nebo instalováním nového balíčku (apt-get update je dobrý zvyk před každou insalací balíčku).

Open Source Xorg ovladače pro nVidii, ATI/AMD, & Intel

Open Source Xorg ovladače pro nVidii, ATI/AMD, Radeon, Intel jsou předinstalvány v aptosid.

Intel ovladače by měly perfektně pracovat pro 2D a 3D video akceleraci, jelikož jsou zahrnuty ve svobodné řadě Intelu.

Poznámka: xorg.conf není obecně potřeba pro open source ovladače

Pokud jste používali proprietární ovladače a přáli byste se vrátit zpět k open-source ovladačmů, upravte /etc/X11/xorg.conf.d/xx-xxxx.conf editorem s root oprávněními. Najděte sekci pro SECTION DEVICE a změňtě ovladač zobrazení, aby obsahoval nv NEBO radeon NEBO intel (vyjmenovány jen některé).

Úprava xorg.conf je zcela na vaše vlastní riziko.

Více informací o Intelu   ATI/AMD   nv   X.Org

Binární, uzavřené ovladače pro: nVidii s dmakms & xorg.conf.d

Pro úplný a nejpřesnější sezna podporováných grafických čipů se prosím podívejte ne Seznam podporovaných produktů dostupný na NVIDIA Linux Graphics Driver download page.

Také si můžete přečíst nvnews pro další možnosti.

Nové a staré instalace budou muste zaručit, že nastavení uplatňující se na celý systém /etc/X11/xorg.conf.d existuje a přidává soubor do adresáře nazvaném 20-nvidia.conf : 

mkdir /etc/X11/xorg.conf.d
touch /etc/X11/xorg.conf.d/20-nvidia.conf

S vačím oblíbeným textovým editorem, (například kwrite, kate, mousepad,mcedit, vi, vim), otevřete:

<editor> /etc/X11/xorg.conf.d/20-nvidia.conf

a přidejte následující pole kódu do celého obsahu 20-nvidia.conf:

#
Section "Device"
         Identifier      "Device 0"
         Driver          "nvidia"
EndSection
# Toto je vodici radka, slouzi k tomu, aby End Section nebyla na posledni radce

Jestli máte více než jednu grafickou kartu, budete potřebovat zjisti PCI a zahrnout ho do 20-nvidia.conf:

lspci | grep -i vga

To bylo mělo vrátit syntaxi podobnou:

01:00.0 VGA compatible controller:

Přidejte 01:00.0 busid jako extra řádku pod 'Driver' řádku, ale všimněte si, že syntaxe je PCI:x:y:z: s vynechanými nulami a přidanými uvozovkami, takže:

BusID		 "PCI:1:0:0"

Instalace nvidia ovladačů

POZNÁMKA: Použijte apt-cache search nvidia a apt-cache show <balíček> pro ujištění se o správnosti ovladače. V podstatě existují 2 typy ovladačů nvidie, aktuální Debian sid 3D ovladač a legacy Debian sid 3D ovladač.

Pro současné 3d nvidia ovladače ≥ GeForce 6xxx :

Připravte modul:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Poté aktivujte Dynamic Module-Assistant Kernel Module Support (dmakms) pro nvidii, aby při příští aktualizaci jádra Linuxu byl také připraven modul nvidie bez ručního zásahu. Pro provedené přidejte nvidia-kernel-source do konfiguračního souboru /etc/default/dmakms :

echo nvidia-kernel-source >> /etc/default/dmakms

Potom:

m-a a-i nvidia-kernel-source

Následovaný:

apt-get install nvidia-glx

Restartuje počítač, aby instalace modulu nabyla efektu.

Když se aktualizuje xorg, budete muset přeinstalovat jen nvidia-glx:

apt-get install --reinstall nvidia-glx

Když se aktualizuje nvidia-kernel-source:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Restarujte počítač, aby instalace modulu nabyla efektu.

Schéma pojmenovávání pro legacy nvidia ovladače v Debianu

  • nvidia-kernel-legacy-71xx pro GeForce 2
  • nvidia-kernel-legacy-96xx pro GeForce 4
  • nvidia-kernel-legacy-173xx pro GeForce 5
Příklad poro legace 3D nvidia ovladač za použití ≤ GeForce 5xxx :

Pro ostatní legacy ovladače jen zaměňte 173xx číslo za číslo vašho ovladače.

m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Pozměňte dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

						      
Když se aktualizuje xorg, budete muset přeinstalovat jen nvidia-glx-legacy:

apt-get install --reinstall nvidia-glx-legacy-173xx

Chyba načtení modulu

Jestli z jakékoliv příčiny dojde k chybě načtení nvidie:

modprobe nvidia

Poté restartuje počítač.

Jestlí stále bude načítání chybovat:

 m-a a-i -f nvidia-kernel-source

nebo

 m-a a-i -f nvidia-kernel-legacy-173xx-source

To přestaví modul. Poté restartuje.

Čtěte:

$ /usr/share/doc/dmakms

ATI/AMD, Radeon a ostatní ovladače

2D ovladače

Ovladače pro X.Org server (podívejte se do xserver-xorg pro další popis) umožňují podporu 2D pro karty NVIDIA Riva, TNT, GeForce a Quadro a karty ATI Mach, Rage, Radeon a FireGL spolu s atimisc, r128, r6xx/r7xx a radeon sub-ovladači. Radeon i Intel podporují 2D akceleraci (xv texturovanou) pro přehrávání videa.

3D ovladače

Některé karty ATI/AMD také podporují 3D, (a KDE animace), s xserver-xorg-video-radeon. K dnešku jsou podporovány až do r700.

Pro automatickou aplikace nových nesvobodných balíčků pro 2D a 3D grafické karty, když jsou aktualizovány:

apt-get install firmware-linux

Poté restartujte počítač.

Nesvobodné firmware ovladače

Potřebné firmware ovladače mohou být určeny zapomocí následující výčtu zařízení/firmware nebo pomocí nově vyvinutého skriptu fw-detect (zabalíčkovaného v aptosid-scripts).

$ fw-detect

Výstup z fw-detect popisuje příkazy nutné k instalaci a aktivace firmware ovladačů

Příklad:

#Detected driver that requires firmware to operate
#Follow these instructions to obtain the correct firmware
# and activate the zd1211rw driver:
apt-get update
apt-get install zd1211-firmware
modprobe -r zd1211rw
modprobe zd1211rw

Pro instalaci firmware ovladačů z git repozitářů, potřebujete:

apt-get install git-core
non-free firmware debs on a stick

Jestli potřebujete získat .deb balíčky za učelem je umístit na usb-klíč a pak na jiný počítač, můžete je stáhnout buť jako zip nebo jako tar.gz z http://cdimage.debian.org/cdimage/unofficial/non-free/firmware/sid/current/ a rozbalit je do adresáře firmware. Měli byste vidět počet .deb souborů. Poté stáhněte firmware-linux-free*.deb a přidejte je ke všem ostatním .deb souborům do adresáře, kam jste nesvobodný firmware rozbalili. Poté je přemístěte s usb-klíčem na počítač, který nesvobodný firmware potřebuje:

dpkg -i firmware-linux-nonfree*.deb

Snažíme se dodat balíčky pro všechen legálně šiřitelný firmware z našich nesvobodných repozitářů, ale ne všichni výrobci to dovolují.
