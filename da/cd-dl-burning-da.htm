Definitions of files on each mirror

Each sidux mirror has following files:

MD5SUM
MD5SUM.asc
SHA256SUM
SHA256SUM.asc
SOURCES
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

The .manifest file lists all the packages contained in the specific iso. The .iso is the image to download. The .md5 and .sha256 files are to check the integrity of the iso.

The .asc files are signature files, so you can verify that the checksums (.md5 .sha256) have not been changed and that they can be used to verify the integrity of the iso.

The sources tar is only for those who require a static copy of the source code for redistribution purposes to comply with the licensing requirements. Read the SOURCES file for more information.

Vi takker alle værterne for disse fil-spejle for deres indsats.

md5sum Validering

MEGET VIGTIGT: sidux, som en Linux LIVE-CD, baseres på højkompressions teknologi, og det er derfor meget vigtigt brænde at CD'en efter følgende retningslinier: Brug kun kvalitets CD-medier [eller DVD+RW] og brænd i DAO-mode (disk-at-once) og aldrig hurtigere end x8.

En md5sum er checksummen af en datafil. Den bruges til at validere integriteten af filen. Den kalkulerede checksum sammenlignes med den for filen kendte checksum, og på denne måde kan det afsløres om filen er beskadiget eller ændret; det kan anbefales altid at gøre dette med filer hentet ned fra internettet.

Hvis den kalkulerede md5-sum af den hentede fil korresponderer med den sum der er i md5-filen, så kan du være sikker på at din download forløb korrekt. I Linux finder du en fils checksum med følgende kommando (det taget et øjeblik at kalkulere summen):

$ md5sum fil_der_skal_valideres

Summen vil blive skrevet i konsollen.
Med programmet md5sum (486 kb) kan md5-summen også valideres i windows.

ISO-image filerne af sidux har altid en tilhørende md5sum fil og bør altid valideres inden du brænder dem. Hvis der skulle opstå problemer så garanterer denne fremgangsmåde at problemet ikke i sidste ende skyldes en fejlbehæftet download, og på denne måde holdes forummet også fri for problemer, der aldrig kan løses fordi deres årsag ultimativt er at ISO-filen var beskadiget. Du validerer filen i et konsol vindue. Gå til den mappe hvor ISO- og md5-sum-filerne er, og skriv følgende kommando for at validere md5-checksummen:

Download the following files from the mirror:

MD5SUM
MD5SUM.asc
SHA256SUM
SHA256SUM.asc

In a terminal:

$ md5sum sidux.iso

Hvis ikke der er overensstemmelse imellem de to summer så får du en fejlmeddelse:

"sidux.iso: Error
md5sum: Warning: calculated checksum does not match!"

Hvis alt er i orden afsluttes programmet uden nogen meddelelser.

If you wish to be able to verify the validity of the .asc files do the following as $USER:
Example using:
sidux-2009-02-aether-kde-lite-i386-200907141544.iso

1) Import "Stefan Lippers-Hollmann's" (slh's) public key from a key-server:

$ gpg --keyserver wwwkeys.eu.pgp.net --recv-keys F80994F6
gpg --edit-key F80994F6
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Now you can verify the integrity of the checksum with:

$ gpg --verify MD5SUM.asc

If you have classified the key as trustworthy using the necessary gpg commands, then this is the output you should get with "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

You will get 'Good signature' with a WARNING if you did not choose to classify the key:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"
gpg: WARNING: This key is not certified with a trusted signature!

Hvordan brænder du CD'en i windows?

MEGET VIGTIGT: sidux, som en Linux LIVE-CD, baseres på højkompressions teknologi, og det er derfor meget vigtigt brænde at CD'en efter følgende retningslinier: Brug kun kvalitets CD-medier [eller DVD+RW] og brænd i DAO-mode (disk-at-once) og aldrig hurtigere end x8.

Du kan naturligvis også brænde CD'en i windows. Den hentede fil skal brændes som ISO-image (cd-billede). Måske ligner filen et arkiv fordi Winrar eller et andet kompressionsprogram har overtaget filassociationen med ISO-filer.

Der er adskillige programmer til at brænde CD'en i windows.

Open Source brændere til Windows

  • Nero (ikke Nero express!)
  • En anden god freware brænder er CD/DVD Burner XP pro
  • Burncdcc fra terabyteunlimited -den kan kun brænde ISO'er..

Brænd CD'en i Linux

MEGET VIGTIGT: sidux, som en Linux LIVE-CD, baseres på højkompressions teknologi, og det er derfor meget vigtigt brænde at CD'en efter følgende retningslinier: Brug kun kvalitets CD-medier [eller DVD+RW] og brænd i DAO-mode (disk-at-once) og aldrig hurtigere end x8.

Hvis du allerede kører Linux kan du brænde CD'en enten i kommandovinduet, eller med hvilket som helst af dine brænderprogrammer.

I sidux og andre Linux distributioner kommer k3b med som standard brænder-program. Start programmet, -vælg "Ekstra" -> "brænd CD-Image..." og vælg så den ISO-fil der skal brændes (e.g. sidux.iso).

K3B starter med at kalkulere MD5-summen for ISO-filen (det tager et øjeblik). Hvis den fundne sum stemmer overens med den som står i md5-filen, så var din download uden fejl og du kan brænde filen ved at trykke "Start".

Problems with burning mostly happen with a frontend. You can burn directly from the shell with burniso.
