Binary, non-free drivers for: Nvidia / ATI

Der er 3 måder at installere 3d drivere på dit system

  1. sgfxi / !h2 smxi

    To install smxi (if its not presently in sidux-scripts)

    wget -qO /usr/local/bin/smxi http://techpatterns.com/smxi
make it executable:
chmod +x /usr/local/bin/smxi
To start, type, as root: 
smxi
and follow the prompts :: script homepage - http://techpatterns.com/h2

    or to just install the 3d driver script

    wget -qO /usr/local/bin/sgfxi http://techpatterns.com/downloads/distro/sgfxi
make it executable:
chmod +x /usr/local/bin/sgfxi
sgfxi
To see options:
sgfxi -h

    sgfxi understøtter i øjeblikket ATI fglrx og Nvidia drivers. Det understøtter også at konvertere fra eller til xorg free drivers som ati, intel, eller nv. !h2 smxi bruger sgfxi som "motor" til sin grafik driver installation, men tilføjer derudover yderligere valgfri muligheder for brugerne.

    Standard driveren er den driver der installeres hvis ikke nogle argumenter bruges til at ændre dette. Medmindre dit kort er for gammelt og kræver en legacy driver, -i hvilket tilfælde scriptet vil afgøre dette for dig.

    Når du kører scriptet, stopper det og fortæller hvilken driver det vil installere. Du kan så acceptere dette, eller afslutte og køre scriptet igen med andre argumenter hvis du vil installere noget andet.

    Du kan også køre sgfxi instalationen med !h2 smxi, som bruger sgfxi til at installere dit grafik driver. Først, læs !h2 smxi homepage for yderligere information, kør så denne kommando:

    smxi -kiwdt

    sgfxi-h: Når du bruger -h flaget vil du også se en liste over alle understøttede drivere, samt noget information om dit systems specifikationer, kernel og xorg version. Her er nogle nyttige sgfxi flag:

    -c Anvend composite mode (kun nVidia kort).
-d Fjern IgnoreDisplayDevices, tillad TV for eksempel (kun nVidia kort).
-h Se denne hjælp menu.
-n Installerer automatisk den korrekte native xorg nv, intel, eller ati driver til dit system.
   Udrenser gammel binary driver stuff, opdaterer xorg.conf med den nye driver information,
   og renser nvidia og fgrlx stuff ud af xorg.conf
-N Tving installation af specifik xorg driver. 
Syntax: sgfxi -N <en af driverne fra listen, som -N radeon>
   Understøttede Xorg drivers: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfigurerer også xorg.conf hvis nødvendigt. Fuld radeon xorg 3d konfiguration automatisk.
-o Override default: installer en specifik driver, dette er en absolut override - syntax:
   -o 9746 (for nVidia, brug ikke 1.0- delen) eller: -o 8.32.5 (for fglrx)
-r Fjern mode lines fra xorg.conf
-t Tillad dynamic twinview (kun nVidia)

  2. install-binary-gfx

    Det ny script til installation af grafik drivere kaldes: install-binary-gfx. For at hente scriptet skal du som root-bruger skrive get-sidux-binary-gfx i en konsol.

    get-sidux-binary-gfx
install-binary-gfx -a (to build sid source)

    Den korrekte driver til dit kort bliver valgt, en .deb oprettes, installeres og din xorg.conf bliver automatisk tilpassset. For at den nyinstallerede driver kan anvendes skal du genstarte X (Ctrl+Alt+Backspace, men log ud af KDE først).

    Options

    Der er valgmuligheder som du måske manuelt vil indstille:

     Usage: ./install-binary-gfx [options]
-a | --auto            Automatic mode
-b | --build           MOD Build only (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          Detect Card only
-u | --update          Update Script
-v | --verbose         Verbose

    Eksempler

    install-binary-gfx -a (Detektere dit kort og installerer sid driveren automatisk.)
install-binary-gfx -b nvidia (Opretter kun nvidia-module....deb i /usr/src. Virker også med fglrx og nvidia-legacy.)
install-binary-gfx -b ati (Opretter kun ati-module....deb i /usr/src. Virker også med fglrx og ati-legacy.)
install-binary-gfx -d (Detekterer dit kort og skriver resultat til dig.)
install-binary-gfx -u (opdaterer dette script. )

    Problemløsning

    Hvis scriptet ikke virker for dit kort bedes du køre scriptet i debug mode og poste outputtet såvel som logfilen /var/log/sidux-CARD-installer.log i sidux forummet, eller bruge http://paste.debian.net og forbind til #sidux og giv dem dit paste link for at få hjælp via IRC

  3. For live-cd Binære, non-free driver installation:

    sudo sgfxi

    dette henter det seneste script, så:

    ctrl+alt+f1

    og kør

    sgfxi

    sgfxi -h,
for at se valgmulighederne

    X genstartes efter driver installationen

  4. Anvend Nvidia/ATI binær installer
    Download det binære driver installationsprogram direkte fra Nvidia
Download det binære driver installationsprogram direkte fra ATI

Free xorg drivere til nVidia, ATI, Radeon, Intel

  1. sgfxi

    Du kan også bruger sgfxi til at skifte imellem free og non-free drivere, eller til at genoprette native free drivers: De vigtigste kommandoer er:

    -n Installerer automatisk den korrekte native xorg nv, intel, eller ati driver til dit system.
   Udrenser gammel binary driver stuff, opdaterer xorg.conf med den nye driver information,
   og renser nvidia og fgrlx stuff ud af xorg.conf
-N Tving installation af specifik xorg driver. 
Syntax: sgfxi -N <en af driverne fra listen, som -N radeon>
   Understøttede Xorg drivers: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfigurerer også xorg.conf hvis nødvendigt. Fuld radeon xorg 3d konfiguration automatisk.

    Først

    apt-get update
apt-get install sidux-scripts
sgfxi

    For at installere eller genoprette native drivere

    sgfxi -n
ELLER
sgfxi -N <xorg driver>

  2. sgfxi / !h2 smxi

    Du kan også anvende sgfxi via !h2 smxi. Start scriptet med følgende parameter og du kommer øjeblikkeligt til grafik driver installations-delen. Hvis du bruger smxi skal du ikke installere sgfxi, eftersom smxi installere dette automatisk.

    !h2 smxi hjemmeside for mere information, kør så denne kommando:

    smxi -kiwdt

  3. Den svære metode

    Driverne til X.Org X server (se xserver-xorg for yderligere beskrivelse) understøtter 2d for NVIDIA Riva, TNT, GeForce, og Quadro kort og ATI Mach, Rage, Radeon, og FireGLcards sammen med atimisc', 'r128' og 'radeon' sub-drivere. Understøtter også Intel free series i 3d

    For at installere nv-driveren:

    Log ud af KDE og gå til text-mode tilstand med Ctrl+Alt+F1, log på som root og skriv så

    init 3
(det skulle gerne generere en meddelelse om at X er stoppet)
apt-get update

    og så (for nvidia)

    apt-get install xserver-xorg-video-nv
    ELLER (for ATI) 
    apt-get install xserver-xorg-video-ati
init 5 && exit

    Bemærk at dette ikke er den samme driver som den binære-kun 'fglrx' driver der leveres fra nVidia/ATI, og som har yderligere 3D funktionalitet for nogle nyere Radeon kort, men som ikke er understøttet.

    Som det næste skal du redigere /etc/X11/xorg.conf, MEN HUSK AT TAGE EN SIKKERHEDSKOPI AF FILEN FØRST, eftersom at fejl i redigeringen vil medføre en meget tidskrævende fejlsøgning.

    For at redigere /etc/X11/xorg.conf kan du højreklikke den og vælge at åbne den med administrator-rettigheder. Find sektionen der hedder SECTION DEVICE, her skal du ændre 'display driver' så værdien bliver nv ELLER ati

    ADVARSEL!! ÆNDRINGER I XORG.CONF ER PÅ EGEN ANSVAR!!

    Mere information om X.Org

Tips til hardware med non-free behov

Af juridiske årsager er der kun dfsg-free software på sidux-CD'en.

Sources List

Eftersom sidux kun indeholder dfsg-free software, kan det være at du vil tilføje contrib/ non-free til din /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list og sikre dig du har internet adgang:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in seperate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Video Cards
  • ATi Radeon graphics: 3d acceleration for ældre kort op til r35x burde fungere, nyere Radeon X1xxx kort skal anvende non-free drivers, for accelereret ydelse, og disse er endnu ikke kompatible med X.org 7.2. Afhængig af det specifikke chipset kan cheatcodes "xmodule=vesa" eller "xmodule=fbdev" være nyttige for nyere Radeon X1xxx modeller.
Firmware
  • To help detect what firmware you need to install, run, as user:
$ fw-detect

This will give you a list of recommendations and what commands to run. It will return 'null' if you do not need anything, or firmware is already installed.

  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • AVM ISDN/ ADSL PCI/ USB cards: AVM's closed source drivere er endnu ikke kompatible med kernel 2.6.21.
  • Broadcom, bcm43xx/ b43/ b43 legacy wlan: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • Broadcom Blutonium Bluetooth chipset (BCM2033): 
    apt-get install bluez-firmware
  • Eagle USB ADSL modem: hent firmware fra http://eagle-usb.org/ueagle-atm/non-free/ og placer den under /lib/firmware/.
  • DVB firmwares for flere forskellige full featured DVB TV kort (de fleste lavpriskort kræver ikke dette): hent firmeware (check dmesg) fra http://www.linuxtv.org/downloads/firmware/ og placer den under /lib/firmware/.
  • IVTV firmware:fetch the firmware from http://dl.ivtvdriver.org/ivtv/firmware/ and place it under/lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan Fjern venligst iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 and firmware-iwl4965 hvis allerede installeret.: 
    apt-get install firmware-iwlwifi
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54 MBit/s wlan: hent firmware fra http://prism54.org/firmware/ og placer den under /lib/firmware/.
  • nVidia graphics: 3d acceleration er endnu ikke muligt med de frie drivere, anvend get-sidux-binary-gfx or sgfxi for at hente installations script for disse kort
  • RaLink rt61 and rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    Fjern venligst rt61-fwcutter, rt73-fwcutter, firmware-rt61 og firmware-rt73, hvis idsse allerede skulle være installeret.

  • Ricoh R5U870 OEM webcams:fetch the firmware from http://mediati.org/r5u870/r5u870-0.11.0.tar.gz and copy the r5u870_*.fw files to /lib/firmware/.
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan: 
    m-a a-i acx100
    hent firmware fra http://www.kazer.org/acx-firmware-20060207.tar.bz2 og placer den under /lib/firmware/.
  • ZyDAS zd1201 11 MBit/s wlan: Fetch the firmware from http://surfnet.dl.sourceforge.net/sourceforge/linux-lc100020/zd1201-0.14-fw.tar.gz and place it under /lib/firmware/.
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Vi undersøger om vi kan tilbyde pakker for i det mindste nogle af disse enheder, men den juridiske situation komplicerer dette.

Skrive til NFTS-partitioner med ntfs-3g

ADVARSEL: ntfs-3g er en tidlig beta driver, så brug den aldrig uden ekstern backup, og aldrig på arbejdsstationer! Anvendelse er for egen risiko og dette inkluderer eventuelle tab af data!

Åbn en konsol og skriv følgende kommandoer: Se Partitionering af harddiske - UUID, Partitions betegnelser og fstab

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit
exit

Nu skulle dit NTFS-drev være mounted rw og du skulle være i stand til at gemme data til drevet. Men igen en advarsel! Det er experimentiel software!
