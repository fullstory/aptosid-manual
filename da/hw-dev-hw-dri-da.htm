Tips til hardware med non-free behov

Af juridiske årsager er der kun dfsg-free software på sidux-CD'en.

Sources List

Eftersom sidux kun indeholder dfsg-free software, kan det være at du vil tilføje contrib/ non-free til din /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list or /etc/apt/sources.list og sikre dig du har internet adgang:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in seperate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Binary, closed source drivers for: nVidia with dmakms

Prepare the module:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Then activate Dynamic Module-Assistant Kernel Module Support (dmakms) for nvidia, so that the next time the Linux kernel is updated a nvidia module is prepared for it too, without manual intervention. To do that, add nvidia-kernel-source to the /etc/default/dmakms configuration file.

Then:

m-a a-i nvidia-kernel-source

Followed by:

apt-get install nvidia-glx

Next alter xorg.conf:

sed -i s/nv\"/nvidia\"/ /etc/X11/xorg.conf

Reboot PC for installation of the module to take effect.

Module load failure

Should nvidia fail to load, for whatever reason [new xorg component, a fs problem or if X doesnt start after reboot] :

modprobe nvidia

Then reboot the computer.

Should the module still fail to load:

 m-a a-i -f nvidia-kernel-source

This rebuilds the module then reboot.

Read:

$ /usr/share/doc/dmakms

Open Source Xorg drivere til nVidia, ATI, Radeon, Intel

Open Source Xorg drivere til nVidia, ATI, Radeon, Intel kommer pre-installeret med sidux.

Driverne til X.Org X server (se xserver-xorg for yderligere beskrivelse) understøtter 2d for NVIDIA Riva, TNT, GeForce, og Quadro kort og ATI Mach, Rage, Radeon, og FireGLcards sammen med atimisc', 'r128' og 'radeon' sub-drivere. Understøtter også Intel free series i 3d

For at installere nv-driveren:

Log ud af KDE og gå til text-mode tilstand med Ctrl+Alt+F1, log på som root og skriv så

init 3
(det skulle gerne generere en meddelelse om at X er stoppet)
apt-get update
og så (for nvidia)
apt-get install xserver-xorg-video-nv
ELLER (for ATI)
apt-get install xserver-xorg-video-ati
ELLER
xserver-xorg-video-intel
init 5 && exit

For nogle kort kan dette understøtte 3D, men det er som regel kun 2D der understøttes.

If you have been running proprietary drivers and wish to revert back to opensource drivers you need to edit /etc/X11/xorg.conf, HOWEVER YOU MUST MAKE A BACKUP COPY FIRST, as editing /etc/X11/xorg.conf incorrectly will result in many tears by you.

For at redigere /etc/X11/xorg.conf kan du højreklikke den og vælge at åbne den med administrator-rettigheder. Find sektionen der hedder SECTION DEVICE, her skal du ændre 'display driver' så værdien bliver nv ELLER atiOR intel.

Editing xorg.conf is entirely at your own risk!!

Firmware drivers

The required firmware can be determined with the following device/ firmware enumeration or by using the newly developed fw-detect script (packaged in sidux-scripts).

$ fw-detect

It is good practice to do a modprobe -r <modulename> before modprobe <modulename>, especially for intel.

Skrive til NFTS-partitioner med ntfs-3g

ADVARSEL: ntfs-3g er en tidlig beta driver, så brug den aldrig uden ekstern backup, og aldrig på arbejdsstationer! Anvendelse er for egen risiko og dette inkluderer eventuelle tab af data!

Åbn en konsol og skriv følgende kommandoer: Se Partitionering af harddiske - UUID, Partitions betegnelser og fstab

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit
exit

Nu skulle dit NTFS-drev være mounted rw og du skulle være i stand til at gemme data til drevet. Men igen en advarsel! Det er experimentiel software!
