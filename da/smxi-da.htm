smxi kommando syntaksen

For at anvende (sidux community contributed multi-function administration tool) smxi, forkortelse for sidux-maintenance, skal du installere skriptet:

For at installere smxi:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip && smxi
og følg instruktionerne

eller for bare at installere 3d driver skriptet

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
For at se valg-mulighederne:
sgfxi -h

Basale smxi script features og funktioner

Dette script har mange features, de fleste vil brugerne kun se en enkelt gang, andre er der hver gang. Efter det initiale system check, og et par spørgsmål om konfiguration er dette de centrale mulighederne du vil se:

  • Script information. Denne sektion viser dig nuværende kernel, sidst du anvendte dette script til at opdatere dit system, hvornår du sidst kørte dette script, og noget mere informatoin. Kan være nyttigt, specielt på systemer du ikke opdaterer så ofte (anbefaling: dist-upgrade mindst 1 gang om måneden. Mindre end dette er risikabelt.)
  • Kernel install. Tilbyder den nuværende stable kernel som standard, og tilbyder avancerede muligheder der tillader dig at installere andre, ældre versioner. Hvis du starter smxi med -K parametren vil den første kernel i listen over stable være den nuværende testing kernel, om nogen findes.
  • dist-upgrade warning sektion: Tilbyder live advarsler om nuværende og forehenværende sid problemer. Efter at du har accepteret at fortsætte (tip: sid er næsten aldrig i den grønne problemløse tilstand, som regel er tilstanden gul, sikker men med fixes),
  • pre dist-upgrade fixes, om nogen, kører dernæst.
  • config file alerts: du ser en 'update these configuration files' information. Tag et hurtigt kig på denne liste, og hvis opgraderingen spørger dig om du vil beholde (keep) eller erstatte (replace) dine konfigurations filer for dette emne, da skal du svare y (ja) i stedet for det normale n (nej).
  • dist-upgrade sektion, inkluderer error handling sådan at du får at vide om din dist-upgrade gik godt eller der var fejl.
  • post dist-upgrade fixes kører dernæst, om nogen.
  • miscellaneous options inklusiv:
    • fixfonts - giver dig forskellige muligheder for at fixe fonte. Godt ved første gangs installationer, men sjældent nyttigt herefter.
    • package-install - installations muligheder for grupper af almindeligt anvendte pakker, inklusiv non-free stuff såsom flash og googleearth installer, samt office installers. Bemærk: OpenOffice.org installer'en tilbyder sprogpakke valg og andre goodies du muligvis ikke kendte til. Også mulighed for at installere diverse værtøjer samt andre pakker.
    • package-removal - oplister forskellige pakker du muligvis gerne vil fjerne, for eksempel wifi eller de tyske sprogpakker, etc.
    • clean-up-stuff - forskellige oprydnings muligheder, såsom kernel fjerner (bemærk: behold altid mindst en gammel kernel du kan boot ind i ved problemer med din nuværende)
    • miscellaneous-tweaks - diverse såsom den populære mozilla settings and file hander tweak, andre ting der varierer med tiden, check it out.
    • continue - går vidre til grafikkort driver installation
    • quit - afslutter smxi på dette tidspunkt, hvis du ikke behøves gøre mere. Bemærk at grafikkort driver installations sektionen har en start KDE/ init 5 mulighed som du altid kan bruge til at komme tilbage til dit skrivebord.
  • graphics installer - anvender sgfxi graphics installer fra sidux til at installere nvdia, og sædvanligvis ati fglrx proprietære drivere (ati har været meget ustabile det sidste år, men som regel fungerer det). Tilbyder også xorg driver installation, samt opstart af din x/desktop

Hvordan du anvender smxi script parametre

parametre: i grupper: 

-deghiIklmrstuwx

eller separate: 

-giktw -f -x

Du kan køre dette script med alle kombinationer af disse parametre.

Du kan sætte dem sammen, som dette: -ik, eller separat, som dette: -w -x

Hver parameter, eller gruppe af parametre, skal begynde med en:

 -

Eksempel:

 smxi -ikw

overspringer system info, kernel install, og warning.

smxi script parametre: overspring dele af script'et

Følgende tillader dig at overspringe dele af du script'et:

  • -d Overspringer dist-upgrade hvis du også sætter w parametren.
  • -e Overspringer nogle af 'Please hit enter to continue' spørgsmålene.
  • -g Overspringer grafik driver installation sektionen.
  • -i Overspringer system information sektionen.
  • -k Overspringer kernel installation seksionen.
  • -r Overspringer kontrol for seneste script version, genstart og download af seneste script version. Anbefales ikke for andre end dem der installerer ny kernel og så genstarter maskinen og kører script'et igen.
  • -s Overspringer nogle sektioner af script'et der kræver en internetforbindelse. Unlad at bruge dette eftersom det reducerer funtionaliteten af script'et drastiskt.
  • -S Overspring internet forbindelses testen. Bruges hvis script'et fejler i at detektere din forbindelse men du ved at den er fin (anbefales ikke at anvende, og virker ikke altid).
  • -t Overspring Post dist-upgrade sektionen.
  • -w Overspring du warnings sektionen.

For eksempel:

smxi -kwid

går direkte til misc options sektion efter dist-upgrade. smxi -ki ville overspringe kernel install spørgsmålet og informations sektionen.

smxi advanced options

Følgende er avancerede script muligheder, og er til at udføre usædvanlige opgaver, som dog kan være nyttige for dig:

  • -I Indstiller forskellig Init level for X restart. Bruger denne syntaks: -I 3
    BEMÆRK: Nummeret er den init level du vil have script'et til at starte x i.
  • -K Advanced Kernel Install. Hvis nyere test-kernels er tilgængelige vil de ses under kernel install options. Installer avancerede kernels for egen risiko!
  • -l Opretter tidsstemplede log filer af alle update og dist-upgrade data. Logfiler gemmes i /var/log/smxi/
  • -m Giver dig muligheden for at skifte mirrors i /etc/apt/sources.list
  • -M Runs the option to change default kernel mirrors for smxi
  • -p Kører apt-get update uden af anvende pdiffs. Godt til hurtige forbindelser.
  • -P Kør script med Proxy indstillinger for forbindelse, bruger denne syntaks:
    -P http://[<username><:password>@]<server><:port>
  • -U Tvinger script til at opdatere sig selv før det kører igen.
  • -x Kører ekstra script features som ikke er del af den centrale funktionalitet.
  • -h Printer smxi script Help menu.
  • -v Prints version and system information, including distro version, current kernel, apt update and dist-upgrade last used, and last use of smxi.

Bruger-indstillinger af smxi script mulighederne

Du kan også få script'et til at udføre nogle andre tin ved at ændre i konfigurationsfilen /etc/smxi.conf

Bemærk at denne fil ikke eksisterer før script'et er blevet kørt en gang, så hvis du vil tilføje nogle af de følgende muligheder men ikke har filen kan du bare oprette den.

  • sgfxi=<nogle normale sgfxi parametre som ikke er standard i graphics installer>, for eksempel, hvis du tilføjer følgende linie:
    sgfxi=b
to 
/etc/smxi.conf,

    vil din nvidia installation automatisk bruge parametren t såvel som standarden -c
    Bemærk at disse muligheder er understøttet af sgfxi, ikke af smxi, smxi vil bare tilføje dem til de parametre den sender til sgfxi når du kører grafikkort driver installationen. Dette understøtter ikke nogle parametre som kræver yderligere information, som for eksempel: -o 8.33.6, udelukkende enkelte bogstavs parametre til sgfxi graphics install.
  • hold-install=<package list separated by ^> - Dette giver dig mulighed for at tilføje pakker som script'et så automatisk sætter på hold. Det kan være nyttigt hvis du du har forbigående problemer med ødelagte pakker, eller hvis du har pakker du ikke vil opgradere af andre grunde, uden at du glemmer dem. Eksempel:
    hold-install=nano^libgtk2.0
    Bemærk: Vigtigt: ingen mellemrum, brug kun ^ som pakkenavn separator!

sgfxi

For at installere 3d drivere med lukket kildekode:

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
For at se valgmulighederne:
sgfxi -h

sgfxi understøtter i øjeblikket ATI fglrx og Nvidia drivers. Det understøtter også at konvertere fra eller til xorg free drivers som ati, intel, eller nv. smxi bruger sgfxi som "motor" til sin grafik driver installation, men tilføjer derudover yderligere valgfri muligheder for brugerne.

Standard driveren er den driver der installeres hvis ikke nogle argumenter bruges til at ændre dette. Medmindre dit kort er for gammelt og kræver en legacy driver, -i hvilket tilfælde scriptet vil afgøre dette for dig.

Når du kører scriptet, stopper det og fortæller hvilken driver det vil installere. Du kan så acceptere dette, eller afslutte og køre scriptet igen med andre argumenter hvis du vil installere noget andet.

Du kan også køre sgfxi instalationen med smxi, som bruger sgfxi til at installere dit grafik driver. Først, læs smxi homepage for yderligere information, kør så denne kommando:

smxi -kiwdt

sgfxi-h: Når du bruger -h flaget vil du også se en liste over alle understøttede drivere, samt noget information om dit systems specifikationer, kernel og xorg version. Her er nogle nyttige sgfxi flag:

-c Anvend composite mode (kun nVidia kort).
-d Fjern IgnoreDisplayDevices, tillad TV for eksempel (kun nVidia kort).
-h Se denne hjælp menu.
-n Installerer automatisk den korrekte native xorg nv, intel, eller ati driver til dit system.
   Udrenser gammel binary driver stuff, opdaterer xorg.conf med den nye driver information,
   og renser nvidia og fgrlx stuff ud af xorg.conf
-N Tving installation af specifik xorg driver. 
Syntax: sgfxi -N <en af driverne fra listen, som -N radeon>
   Understøttede Xorg drivers: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfigurerer også xorg.conf hvis nødvendigt. Fuld radeon xorg 3d konfiguration automatisk.
-o Override default: installer en specifik driver, dette er en absolut override - syntax:
   -o 9746 (for nVidia, brug ikke 1.0- delen) eller: -o 8.32.5 (for fglrx)
-r Fjern mode lines fra xorg.conf
-t Tillad dynamic twinview (kun nVidia)

Rulle tilbage til åben kildekode drivere

Disse åben kildekode drivere til nVidia, ATI, Radeon, Intel findes alle præinstalleret med sidux. Hvis du vil vende tilbage til dem efter at du har overskrevet dem med drivere der har lukket kildekode, kan du gøre dette med sgfxi, først:

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
For at se valgmulighederne:
sgfxi -h

For nu at vende tilbage til de oprindelige drivere (åben kildekode)

sgfxi -n
ELLER
sgfxi -N <Xorg driver>

De almindligste kommandoer er:

-n Installerer automatisk den korrekte native xorg nv, intel, eller ati driver til dit system.
   Udrenser gammel binary driver stuff, opdaterer xorg.conf med den nye driver information,
   og renser nvidia og fgrlx stuff ud af xorg.conf
-N Tving installation af specifik xorg driver. 
Syntax: sgfxi -N <en af driverne fra listen, som -N radeon>
   Understøttede Xorg drivers: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfigurerer også xorg.conf hvis nødvendigt. Fuld radeon xorg 3d konfiguration automatisk.
