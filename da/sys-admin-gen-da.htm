Opsætning af tjenester i sidux

insserv (Pontos 2008-04 and forward ): To start/stop already installed services:

Read /usr/share/doc/insserv/README.Debian, the release notes and the man pages carefully:

 $ man insserv
 $ man invoke-rc.d
 $ man update-rc.d
 google LSB headers

For 'start':

/etc/init.d/<service> start

For 'stop':

/etc/init.d/<service> stop

For 'restart:

/etc/init.d/<service> restart

To prevent services running at-startup

update-rc.d <service> remove
[will remove all startup links]

To make sure a service runs at boot [not always required]

update-rc.d <service> defaults
[will make startup links]

To read the list of what services are running:

ls /etc/rc5.d

S means the service will start.
K means service will not start.

Older Installs (before Pontos 2008-04)

insserv on installed systems before Pontos 2008-04

Warning: Older installs will not get insserv automatically, due to users own initscripts or symlinks. It is NOT recommended to install insserv on installations prior to Pontos. This package should be used with care, as incorrect or missing LSB dependencies of existing initscripts can lead to an unbootable system. Installing insserv on an existing install may require some manual changes.

Read read the Debian documentation notes, and release notes before installing and enabling insserv.

Boot-Up Manager (bum) - Graphical services configuration tool

If the logic of a debian system boot up sequence is not very clear and familiar to you, you should not play with symlinks, permissions and so forth. In order to avoid messing up your system. Boot-Up Manager will help you to automate your configuration.

Boot-Up Manager is a GUI runlevel configuration editor which allows you to configure what init services are invoked when the system boots up or reboots. It displays a list of every service which could be started at boot. You can toggle individual services on and off..

apt-get install bum

To use the Boot-Up Manager GUI:

$ sux
password:
bum

Detailed documentation on Boot-Up Manager (bum).

Afslutte tjenester eller processer

pkill er meget nyttigt, eftersom det er til at danne sig et overblik over, og det fungerer både i bruger- og root-mode, i enten terminalen eller tty

pkill -n service

Hvis du ikke er sikker på hvordan tjenesten eller processen som du vil dræbe staves kan du bruge pkill <tab> <tab> will provide a list

htop er også et godt alternativ. (killall -9 er dit sidste alternativ).

sidux runlevels - init

Dette er listen over sidux operativ systemets runlevels, bemærk venligst at de adskiller sig fra debain stable runlevels:

  • runlevel 0 powers off the PC.
  • runlevel 1 single user (safety mode). Dæmons like apache and sshd are stopped. Do not go to this level via remote access.
  • runlevel 2 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows
  • runlevel 3 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows (this is where dist-upgrades are actioned).
  • runlevel 4 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows
  • runlevel 5 Multi-User mode with the network running the X-windows system, and/or to start X-windows.
  • runlevel 6 reboot.
  • runlevel S where early boot time services are executed on a 'once only basis'. You cannot switch to it after it has been run.

Mistet root passwords

Du kan ikke genfinde et mistet root password, men du kan oprette et nyt.

Til at starte med booter du Live-CD'en.

Som root monterer du din root partition (f.eks. /dev/sdb2)

mount /dev/sdb2 /media/sdb2

Nu kan du chroot ind i din oprindelige root-partition med og oprette et nyt root password:

chroot /media/sdb2 passwd

Fonte i sidux

Korrekte dpi indstillinger - Grundlæggende Filosofi

DPI indstillingerne er problematiske at gætte, men indstilles faktisk helt nøjagtigt af X, forudsat at DisplaySize indeholder de korrekte fysiske dimensioner af skærmen i /etc/X11/xorg.conf "Monitor Section".

Korrekt skærm-opløsning og opdaterings frekvens

Hver skærm har sine egne perfekte indstillinger, men desværre er det ikke alle der korrekt rapporterer DCC værdierne, og nogle gange er det nødvendigt med manuelle indstillinger.

Korrekt grafik-kort driver

Nogle nyere ATI og Nvidia kort vil bare ikke rigtigt med free Xorg drivere, og den eneste løsning i sådan et tilfælde er de kommercielle closed source drivere. sidux vil ikke præ-installere disse af juridiske årsager, løsningen finder du her.

Standard font valg, rendering og størrelse

sidux bruger (Debian) præ-valgte frie fonte der har vist sig at dække et bredt udvalg, dine egne fonte kan vise sig at nedsætte rendering kvaliteten. Der er dog et par valgmuligheder i debian (apart from KDE>systemsettings ) der kan hjælpe med at give dig perfekt rendering af andre fonte. Men vær venligst opmærksom på at hver font har kun et par virkelig perfekte størrelser, andre størrelser end disse kan give et ringere resultat.

At ændre dpi størrelsen med fgl. kommando kan også være værd at prøve:

fix-dpi-kdm

Du burde så se din spi for din skærmstørrelse, men du kan også ændre på dette. Du skal gå til init 3 og tilbage for at ændringerne træder i kraft, eller genstarte.

Efter at du har ændret font typen eller DPI (i X eller Firefox/Iceweasel), kan det være at du skal genjustere for at få indstillingerne til at passe til dine behov, specielt efter ændring fra Bitmap Fonts til True Type Fonts eller den anden vej rundt:

dpkg-reconfigure fontconfig-config

Vælg native og sæt autohinter til automatic. Ellers prøv at lege lidt rundt med indstillingerne.

Hvis dette ikke virker kan det være at du skal genopbygge din fonts cache med:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
GTK baserede programmer såsom Firefox/Icewasel

GTK baserede programmer er -generelt set- problematiske under KDE's standard opsætning. Dette kan du løse med følgende:

apt-get install gtk2-engines system-config-gtk-kde gtk-qt-engine gtk2-engines-qtcurve

In System Settings >Appearance you will have a new menu item called GTK Styles and Fonts. Set 'GTK Styles' to use 'Clearlooks' and have the 'GTK Fonts' set to use 'KDE fonts' or experiment with the various options to suit your preferences.

CUPS

KDE har en stor sektion inkluderet i KDE-hjælp, dist-upgrade's kan dog nogle gange forårsage at CUPS bliver adfærdsvanskelig. Følgende er én kendt løsning på dette:

modprobe lp
echo lp >> /etc/modules
apt-get remove --purge cupsys cups
apt-get install cups
        ELLER
apt-get install cups cups-driver-gutenprint hplip

Og så i en web browser:

http://localhost:631

For at være sikker på at CUPS kører, genstart den med følgende kommando:

/etc/init.d/cups restart

En anden faldgrube findes i, at hvis du sætter CUPS op ved hjælp af den grafiske brugerflade, så bringer den en dialog boks frem hvor du skal taste brugernavn og password. Boksen er auto-udfyldt med dit navn, men skriver du din kode ind får du en fejl når du trykker OK. Hvad du skal gøre er at slette dit eget navn og skrive root og så skrive dit root password.

Sound in sidux

Sound is muted by default in sidux.

The KDE version uses Kmix and XFCE uses Mixer.

Often it is only a matter of clicking on the sound icon in the taskbar and unchecking the 'Mute box'.

Kmix

In Kmix you will need to activate the options you prefer for channel options, Kmix>Setting>Configure Channels. Or in a terminal:

$ kmix
XFCE

In XFCE run the mixer application and add some controls via the Multimedia>Mixer and click the Select Controls box. Or in a terminal:

$ xfce4-mixer

Alsamixer

If you prefer to use Alsamixer, it is in the alsa-utils package:

apt-get update
apt-get install alsa-utils
exit

Set your preferred sound settings as $user from the terminal:

$ alsamixer

Then as root:

alsactl store

See also the sidux wiki.
