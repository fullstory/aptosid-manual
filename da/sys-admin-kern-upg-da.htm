Installering af ny kernel

sidux kernels findes i apt. Vælg venligst det nærmeste filspejl.:

Kernels findes i følgende varianter:

  • sidux-686 sidux-686 - kernel til i686 Processor Family med single/dual core eller flere CPUer
  • sidux-amd64 kernel til 64 bit sidux
4 trin for manuel udførsel i en konsol:

1.Bliv root i en konsol:

apt-get update

2.For at installere den seneste kernel:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3.Kernel opdateringen er nu færdig. Hvis du har brug for dem kan du nu installere de sidste grafikdrivere, ATI, NVIDIA eller INTEL kort. Først skal du opdatere skriptet:

sgfxi -U

4.Genstart nu din PC. Den nye kernel vil blive anvendt som standard.

For at opdatere dine 3D drivere kører du nu:

sgfxi

Alternativt kan du gå direkte til runlevel 3 ved den næste opstart (dette opnås ved at tilføje et 3 til Grub-Boot-Line) og så køre sgfxi.

Mere detajleret sgfxi information findes her.

Hvis den nye kernel giver dig problemer kan du vælge at genstarte og vælge en ældre kernel.

Moduler

For at finde ud af hvilke moduler du anvender giver følgende kommandoer dig lister over de i øjeblikket tilgængelige moduler. Kopier denne linie til din konsol/ terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

For at få den fulde beskrivelse af modulerne, kopier denne linie ind i din konsol/ terminal:

apt-cache search 2.6.*.slh.*-sidux

For at installere kernel, headers og de nødvendige moduler (for eksempel virtual-ose, og qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)

For at se hvilke moduler der er indlæst med din kernel:

ls /sys/module/
eller
cat /proc/modules

VIGTIGT: hvis du skal anvende specifikke kernel modules, for eksempel til wlan, skal du installere de rigtige .deb pakker for dem. For eksempel hvis du anvender Agere et131x ethernet adapter:

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb anvendes kun hvis du har debs lokalt allerede, men selv i det tilfælde, hvis de findes i apt-cache, så er apt-get den bedste løsning.

Andre metoder til kernel installation

smxi

smxi skriptet tilbyder en integreret metode til kernel installation, driver installation samt andre sys-admin skripts:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Du kan også installere kernels med GUI kontrol centret siduxcc: som du finder i KDE-Start-Menu>System>sidux Kontrol Center. siduxcc tilbyder også andre system administrations GUI programmer.

Afinstallation af gamle kernels

Efter vellykket installation af en ny kernel kan de/ den gamle slettes, det anbefales dog at beholde dem i mindst et par dage hvis der skulle dukke problemer op med den nye kernel. Så kan du altid boot den gamle kernel fra grub menuen

Gamle kernels kan fjernes fra systemet. For at gøre dette anvender du skriptet kernel-remover der findes i pakken sidux-kernelhacking. Dette skript skal være installeret og tillader dig at vælge hvilke kernels der skal fjernes.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Alternativt kan du anvende siduxcc, der beror på kernel-remover skriptet.
