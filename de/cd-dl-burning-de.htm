Die sidux-CD herunterladen

WICHTIGE INFORMATION: sidux, als Linux-LIVE-CD, basiert auf einer ursprünglich von Knoppix entworfenen und nun weiterentwickelten Hochkomprimierungs-Technologie. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbildes geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus (disk-at-once) und nicht schneller als achtfach (8x).

Mit Bittorrent herunterladen

Bittorrent ist ein Filesharing-Protokoll, das auf der sidux-CD enthalten ist. Es ermöglicht, Dateien gleichzeitig herunter- und für andere hochzuladen. Die Adressen der aktuellen Torrents sind auf sidux.com zu finden.

Den Download eines Torrents kann man in einem Terminal wie der Konsole zum Beispiel so durchführen:

# btdownloadcurses --max_upload_rate 12 ftp://dummy/sidux/sidux-200xxxxx/sidux-200xxxxx.iso.torrent

Das Herunterladen sollte bald beginnen.

Achtung: In dem Verzeichnis, aus dem der Befehl ausgeführt wird, muss genug Platz frei sein, also ca. 700 MB.

Nach dem Herunterladen bitte das Programm weiterlaufen lassen, um auch anderen die Möglichkeit eines Downloads zu bieten. Das entlastet unsere Server und beschleunigt das Herunterladen für andere. Eine gute Einleitung zum Thema gibt es hier: Brian's BitTorrent FAQ and Guide.

Bitte nicht vergessen, die md5sum des heruntergeladenen ISOs zu überprüfen.

Wenn jemand einen FTP-Server mit entsprechendem Traffic zur Verfügung stellen kann, sind wir jederzeit im Forum oder im IRC-Channel erreichbar.

md5sum und Integritätsprüfung von heruntergeladenen Dateien

WICHTIGE INFORMATION: sidux, als Linux-LIVE-CD, basiert auf einer ursprünglich von Knoppix entworfenen und nun weiterentwickelten Hochkomprimierungs-Technologie. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbildes geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus (disk-at-once) und nicht schneller als achtfach (8x).

Eine md5sum ist die Check-Summe einer Datei. Diese Check-Summe wird zur Integritätsprüfung der zugehörigen Datei benutzt. Dabei wird die momentane md5sum der Datei mit einer bekannten früheren Summe verglichen. So kann festgestellt werden, ob die Datei verändert oder beschädigt wurde, was bei heruntergeladenen Dateien ratsam ist.

Die Datei ist unbeschädigt heruntergeladen worden, wenn die md5sum der heruntergeladenen Datei mit der Summe in der MD5-Datei übereinstimmt. Unter Linux erhält man die md5sum einer Datei mit:

$ md5sum zu_prüfende_datei

Das dauert ein wenig und die Summe wird dann in der Konsole ausgegeben. Mit dem md5summer (486 KB) kann die md5sum auch in Windows geprüft werden.

Die ISO-Images von sidux werden immer mit der entsprechenden md5sum zum Download angeboten und sollten stets vor dem Brennen geprüft werden. Das kann zweckmäßiger Weise in einem Terminal durchgeführt werden. Dazu wechselt man in das Verzeichnis, in dem sich sowohl die ISO-Datei als auch die MD5-Summe befinden, und kann mit dem Befehl:

$ md5sum -c sidux-Name.iso.md5

die md5sum überprüfen lassen. Wenn die beiden Prüfsummen nicht übereinstimmen, erhält man vom Programm die Meldung:

sidux-Name.iso: Fehlschlag md5sum: Warnung: 1 von 1 berechneten Prüfsumme stimmen nicht überein.

Ist aber die heruntergeladene Datei korrekt, dann endet das Programm ohne jegliche Meldung.

Eine Live-CD mit Windows brennen

WICHTIGE INFORMATION: sidux, als Linux-LIVE-CD, basiert auf einer ursprünglich von Knoppix entworfenen und nun weiterentwickelten Hochkomprimierungs-Technologie. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbildes geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus(disk-at-once) und nicht schneller als achtfach (8x).

Selbstverständlich kann die CD auch in Windows gebrannt werden. Die heruntergeladene Datei muss als ISO-Abbilddatei gebrannt werden. Falls Winrar (oder ein anderes Archivierungsprogramm) mit einer ISO-Datei verknüpft ist, könnte dieses Programm die ISO-Datei als eine Archivdatei ansehen. Aus der ISO-Datei muss eine CD gebrannt werden.

Es gibt verschiedene gute Optionen, ISO-Dateien in Windows zu brennen.

Open-Source-Brennsoftware für Windows

cdrtfe: kompatibel mit Windows 9x/ME/2000/XP (getestet mit Win95, Win98SE, Win2000, WinXP). Nur für Win9x/ME: funktionierende ASPI-Layer (z. B. Adaptec ASPI 4.60)

Closed-Source- und proprietäre Brennsoftware für Windows
  • Nero (nicht nero express!)
  • CDR-WIN Eine Testversion kann aus dem Internet geladen werden, die aber mit reduzierter Geschwindigkeit brennt, was jedoch für diese Aufgabe ausreichend sein sollte. Im englischsprachigen Menu muss folgender Punkt gewählt werden: "Burn a ISO 9660 Image File to CD".
  • Eine weitere gute Freeware-Brennsoftware ist der CD/DVD Burner XP pro
  • Burncdcc von terabyteunlimited kann nur ISO-Abbilddateien brennen.

Die CD mit Linux brennen

WICHTIGE INFORMATION: sidux, als Linux-LIVE-CD, basiert auf einer ursprünglich von Knoppix entworfenen und nun weiterentwickelten Hochkomprimierungs-Technologie. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbildes geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus(disk-at-once) und nicht schneller als achtfach (8x).

Wer bereits Linux auf dem Rechner hat, kann die CD mit jedem installierten Brennprogramm erstellen. Bei sidux ist K3b das Standard-Brennprogramm. Dort muss man im Menüpunkt "Extras" -> "CD-Abbilddatei brennen..." anklicken, das zu brennende ISO-File (z.B. sidux-Name.iso) auswählen und den Brennmodus DAO (Disk At Once) einstellen.

K3b berechnet zuerst die MD5-Summe des ISO-Files (dauert einen Moment). Stimmt die angezeigte Prüfsumme mit der angegebenen Zeichenfolge der sich im selben Ordner befindlichen MD5-Datei (z.B. sidux-Name.iso.md5) überein, war der Download erfolgreich und die Datei kann mit einem Klick auf "Start" gebrannt werden.

Klickt man auf die berechnete Prüfsumme, erscheint daneben ein Symbol. Klickt man wiederum darauf und fügt in das Feld die Prüfsumme aus der MD5-Datei ein, so werden die beiden Prüfsummen verglichen.

Man kann auch direkt auf der Shell brennen, hier findet man mehr Informationen.

Eine aktuelle sidux-CD unter Verwendung eines XDELTA-Patch herstellen

WICHTIGE INFORMATION: sidux, als Linux-LIVE-CD, basiert auf einer ursprünglich von Knoppix entworfenen und nun weiterentwickelten Hochkomprimierungs-Technologie. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbildes geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus(disk-at-once) und nicht schneller als achtfach (8x).

xdelta ist ein Programm, das unter anderem aus einem ISO-Image und einer Differenzdatei ein neues ISO-Image erstellt. Es ist also wie geschaffen, kleinere Fehlerkorrekturen und Fixes ohne großen Downloadaufwand anzubieten.

Als root in einem Terminal bzw. einer Konsole:

apt-get update
apt-get install xdelta

Die Änderungen befinden sich in der .xdelta-Differenzdatei (zum Beispiel patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta), welche kleiner als die ISO-Abbilddatei selbst ist. Als Beispiel: .xdelta: 127 MB, .iso: 553 MB.

Voraussetzung:

  • ein ISO-Abbild auf der Festplatte, z.B. sidux.iso
  • Eine Differenzdatei, z. B. patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta

Anwenden eines Patches auf ein ISO-Abbild als User in der Konsole:

$ cd /Pfad/zu/iso/und/xdelta-patch
$ xdelta patch patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta

Das Programm extrahiert die notwendigen Dateinamen aus dem PATCH. Es führt auch eine md5sum-Überprüfung durch. Achtung! Es prüft nicht, ob genug Speicherplatz vorhanden ist.

Ein Programm, um Xdeltas in Windows zu benutzen und eine ISO-Datei zu patchen, ist xdelta.exe. Auf derselben Seite gibt es auch Informationen, allerdings nur auf Englisch.
