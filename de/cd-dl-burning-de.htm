Die aptosid ISO herunterladen

WICHTIGE INFORMATION: aptosid, als Linux-LIVE-DVD/CD, ist sehr stark komprimiert. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbilds geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus (disk-at-once) und nicht schneller als achtfach (8x).
Bitte verwenden Sie den nächstgelegenen Spiegelserver. Spiegelserver, die unterhalb des Links mit Angaben für den Eintrag in /etc/apt/sources.list.d/aptosid.list gelistet sind, werden zeitnah aktualisiert.

Dateidefinitionen auf den aptosid-Spiegelservern

Jeder Spiegelserver umfasst folgende Dateien:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

Die .manifest-Datei listet alle Pakete der jeweiligen ISO.

.iso ist die für den Download angebotene Abbilddatei.

Die Dateien .md5 und .sha256 dienen der Überprüfung der Integrität der ISO.

Die .gpg-Dateien sind die Signaturdateien, mit denen Checksummen-Dateien (.md5 .sha256) überprüft werden, ob sie verändert wurden. Letztere werden zur Integritätsüberprüfung der ISO verwendet.

Das Tar-Archiv mit den Quellen aptosid*-source.tar ist nur für diejenigen gedacht, welche eine statische Kopie des Quellcodes für die Weitergabe benötigen, um die Lizenzbestimmungen erfüllen zu können. Lesen Sie die Datei SOURCES für weitere Informationen.

Wir bedanken uns recht herzlich bei webtropia.com, dem Sponsor von aptosid, und allen Betreibern von Spiegelservern für ihre Unterstützung.

Download-Links und Spiegelserver findet man auf aptosid.com

Falls Sie einen FTP-Server mit entsprechendem Traffic zur Verfügung stellen können, sind wir jederzeit in den aptosid-Foren oder im IRC-Channel erreichbar.

md5sum und Integritätsprüfung von heruntergeladenen Dateien

WICHTIGE INFORMATION: aptosid, als Linux-LIVE-DVD/CD, ist sehr stark komprimiert. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbilds geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus (disk-at-once) und nicht schneller als achtfach (8x).

Eine md5sum ist die Prüfsumme einer Datei. Diese Prüfsumme wird zur Integritätsprüfung der zugehörigen Datei benutzt. Dabei wird die aktuelle md5sum der Datei mit einer bekannten früheren Summe verglichen. So kann festgestellt werden, ob die Datei verändert oder beschädigt wurde, was bei heruntergeladenen Dateien ratsam ist.

Die Datei ist unbeschädigt heruntergeladen worden, wenn die md5sum der heruntergeladenen Datei mit der Summe in der MD5-Datei übereinstimmt. Unter Linux erhält man die md5sum einer Datei mit:

$ md5sum zu_prüfende_datei

Das dauert ein wenig und die Summe wird dann in der Konsole ausgegeben. Mit dem md5summer (486 KB) kann die md5sum auch in Windows geprüft werden.

Die ISO-Abbilddateien von aptosid werden immer mit der entsprechenden md5sum zum Download angeboten und sollten stets vor dem Brennen geprüft werden. Dies lässt sich am einfachsten in einem Terminal durchführen. Dazu wechselt man in das Verzeichnis, in dem sich sowohl die ISO-Datei als auch die MD5-Summe befinden. Das folgende Beispiel erläutert die Vorgehensweise.

Diese Dateien werden vom Spiegelserver heruntergeladen:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

In einem Terminal:

$ md5sum aptosid.iso

Wenn die beiden Prüfsummen nicht übereinstimmen, erhält man vom Programm die Meldung:

aptosid-Name.iso: Fehlschlag md5sum: Warnung: 1 von 1 berechneten Prüfsumme stimmen nicht überein.

Ist die heruntergeladene Datei korrekt, dann endet das Programm ohne jegliche Meldung.

Falls die Gültigkeit der .gpg-Dateien überprüft werden soll, geschieht dies als $USER.

In diesem Beispiel nutzen wir
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) Importieren des öffentlichen Schlüssels von "Stefan Lippers-Hollmann" (slh) von einem Schlüsselserver. (Ein alternativer Schlüsselserver ist subkeys.pgp.net):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Jetzt kann die Integrität der Checksummen damit bestätigt werden:

$ gpg --verify SHA256SUM.gpg SHA256SUM

Falls dieser Schlüssel mittels der üblichen GPG-Befehle als vertrauenswürdig gesetzt wurde, sollte diese Rückmeldung erfolgen: "5 trusted ultimately".

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

Die Rückmeldung 'Good signature' mit einer WARNUNG erhält man, wenn der Schlüssel nicht bewertet wurde:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: 64C3 6120 DA8D 91E7 378B  E79F 3916 C431 F809 94F6

Eine Live-CD mit Windows brennen

WICHTIGE INFORMATION: aptosid, als Linux-LIVE-DVD/CD, ist sehr stark komprimiert. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbilds geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus (disk-at-once) und nicht schneller als achtfach (8x).

Selbstverständlich kann die CD auch in Windows gebrannt werden. Die heruntergeladene Datei muss als ISO-Abbilddatei gebrannt werden. Falls Winrar (oder ein anderes Archivierungsprogramm) mit einer ISO-Datei verknüpft ist, könnte dieses Programm die ISO-Datei als eine Archivdatei ansehen. Aus der ISO-Datei muss eine CD gebrannt werden.

Es gibt verschiedene gute Optionen, ISO-Dateien in Windows zu brennen.

Open-Source-Brennsoftware für Windows

cdrtfe: kompatibel mit Windows 9x/ME/2000/XP (getestet mit Win95, Win98SE, Win2000, WinXP). Nur für Win9x/ME: funktionierende ASPI-Layer (z. B. Adaptec ASPI 4.60)

LinuxLive USB Creator, ein Open-Source-Projekt, bietet eine GUI-Applikation für MS Windows™, welche es ermöglicht, eine aptosid-i386.iso (32 bit) auf einen USB-Stick zu installieren. Dafür muss fromhd als Option in die Bootzeile geschrieben werden.

Closed-Source- und proprietäre Brennsoftware für Windows
  • Nero (nicht nero express!)
  • Eine weitere gute Freeware-Brennsoftware ist der CD/DVD Burner XP pro
  • Burncdcc von terabyteunlimited kann nur ISO-Abbilddateien brennen.

Die CD mit Linux brennen

WICHTIGE INFORMATION: aptosid, als Linux-LIVE-DVD/CD, ist sehr stark komprimiert. Aus diesem Grund muss besonders auf die Brennmethode des ISO-Abbilds geachtet werden. Wir empfehlen hochwertige CD-Medien (oder DVD+R), das Brennen im DAO-Modus (disk-at-once) und nicht schneller als achtfach (8x).

Wer bereits Linux auf dem Rechner hat, kann die CD mit jedem installierten Brennprogramm erstellen. Bei aptosid ist K3b das Standard-Brennprogramm. Dort muss man den Menüpunkt "Extras" -> "ISO-Abbild brennen..." anklicken, die zu brennende ISO-Datei (z.B. aptosid-Name.iso) auswählen und den Brennmodus DAO (Disk At Once) einstellen.

K3b berechnet zuerst die MD5-Summe des ISO-Files (dauert etwas). Stimmt die angezeigte Prüfsumme mit der angegebenen Zeichenfolge der sich im selben Ordner befindlichen MD5-Datei (z.B. aptosid-Name.iso.md5) überein, war der Download erfolgreich und die Datei kann mit einem Klick auf "Start" gebrannt werden.

Klickt man auf die berechnete Prüfsumme, erscheint daneben ein Symbol. Klickt man wiederum darauf und fügt in das Feld die Prüfsumme aus der MD5-Datei ein, so werden die beiden Prüfsummen verglichen.

Die Ursache von Problemen beim Brennen findet sich zumeist in den Frontend-Applikationen. Zum unmittelbaren Brennen von der Konsole kann das Skript burniso verwendet werden.

Siehe auch Installation auf USB-Stick/SSD von einem anderen System (Linux. MS Windows, Mac OS X).
