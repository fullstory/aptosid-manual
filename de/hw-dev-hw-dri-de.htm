Binäre, non-free Treiber für: Nvidia / ATI

Es gibt drei Methoden, 3D-nVidia-Treiber zu installieren

  1. install-binary-gfx

    Das neue Skript zur Installation von Grafikkartentreibern nennt sich: install-binary-gfx. Um das Skript zu installieren, muss einmal der Befehl get-sidux-binary-gfx als Root in einer Konsole ausgeführt werden.

    get-sidux-binary-gfx
install-binary-gfx -a (Verwendung des Quellcodes aus dem sid-Repository)

    Damit wird der richtige Treiber für die Grafikkarte ausgewählt, eine Debian-Installationsdatei (.deb) gebaut und installiert. Auch wird automatisch die Konfigurationsdatei /etc/X11/xorg.conf angepasst. Um den neuen Treiber zu aktivieren, muss der Grafikserver X neu gestartet werden (Strg+Alt+Rücktaste).

    Optionen

    Folgende Optionen können gesetzt werden:

     Usage: ./install-binary-gfx [options]
-a | --auto            automatischer Modus
-b | --build           nur Bau eines MODs (Moduls) (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          nur Grafikkartenerkennung
-u | --update          Update des Skripts
-v | --verbose         mit Rückmeldung

    Beispiele

    install-binary-gfx -a (Dies ermittelt die Grafikkarte und installiert die automatisch die benötigten Treiber aus dem sid-Repository)
install-binary-gfx -b nvidia (Damit wird nur das Modul für nvidia gebaut, ein deb-File in /usr/src. Gleiches gilt für fglrx (ATI) und nvidia-legacy)
install-binary-gfx -b ati (Damit wird nur das Modul für ati gebaut, ein deb-File in /usr/src. Gleiches gilt für fglrx (ATI) und nvidia-legacy)
install-binary-gfx -d (Dies ermittelt die Grafikkarte und gibt Informationen über diese aus)
install-binary-gfx -u (Damit wird das Skript aktualisiert. )

    Hilfe bei Problemen

    Falls das Skript die Grafiktreiber nicht installiert, kann der debug-Modus verwendet werden. Dieser erzeugt eine Protokolldatei /var/log/sidux-CARD-installer.log. Um Hilfe bekommen zu können, kann im sidux-Forum der Inhalt dieser Datei gepostet werden, oder man speichert ihn auf http://paste.debian.net und wendet sich an den IRC #sidux, wo der erzeugte Link angegeben werden kann.

  2. sgfxi / du-fixes-h2.sh

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxi unterstützt im Augenblick ATI/fglrx- und Nvidia-Treiber. Auch wird die Konvertierung von oder zu freien xorg-Treibern wie ati, intel oder nv unterstützt. du-fixes-h2.sh nutzt sgfxi als Installationsbackend für Grafiktreiber und fügt einige Optionen zur einfacheren Benutzung hinzu.

    Der Default-Treiber wird installiert, wenn keine weiteren Optionen für die Installation angegeben werden. Auch erkennt das Skript automatisch ältere Grafikkarten und installiert Legacy-Treiber ohne weiteres Zutun.

    Nach Start des Skriptes und Erkennung der Grafikkarte wird ausgegeben, welcher Treiber installiert werden soll. Man hat die Möglichkeit, diese Auswahl zu bestätigen und den Treiber zu installieren oder den Installationsvorgang abzubrechen und mit einer selbstgewählten Option das Skript neu zu starten.

    Sie können auch den sgfxi-Installer mit du-fixes-h2.sh ausführen, welches auch sgfxi benutzt um die Grafik-Treiber zu installieren. Zuerst, besuchen sie die du-fixes-h2.sh-Homepage für mehr Informationen, und dann führen sie dieses Kommando aus:

    du-fixes-h2.sh -kiwdt

    sgfxi -h listet alle Optionen. Die Optionen werden im Laufe der Zeit ergänzt. Auch werden mit sgfxi -h die augenblicklich unterstützten Treiber und Spezifikationen des benutzten Systems wie Kernel und xorg-Version angezeigt. Nachfolgend einige nützliche sgfxi-Optionen:

    -c Benutze den Composite Modus (nur für nVidia Karten).
-d Entferne IgnoreDisplayDevices, lasse zum Beispiel TV zu (nur für nVidia Karten).
-h Öffne dieses Hilfe Menü.
-n Installiert automatisch den korrekten nativen xorg nv, intel, oder ati Treiber für ihr System.
   Säubert alte Binär-Treiberreste, erneuert die xorg.conf mit neuen Treiberinformationen,
   und entfernt Reste von nvidia und fglrx aus der xorg.conf
-N Erzwinge Installation eines spezifischen xorg Treibers. 
Syntax: sgfxi -N <einer der Treiber aus der Liste, wie z.B. -N radeon>
   Unterstützte Xorg-Treiber: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfiguriert auch die xorg.conf sofern notwendig. Vollautomatische radeon xorg 3d Konfiguration.
-o Überschreibt Grundeinstellung: Installiert einen spezifischen Treiber. Dies ist die Syntax eines völligen Overrides (alles wird überschrieben):
   -o 9746 (für nVidia, benutzen Sie nicht den Teil 1.0-) oder: -o 8.32.5 (für fglrx)
-r Entfernt "modelines" aus der xorg.conf
-t Aktiviert dynamische Verwaltung von Doppelmonitoren (dynamic twinview) (nur nVidia)

  3. Für die Live-CD Installation von binären, nicht-freien Treibern:

    sudo sgfxi

    dies lädt das neueste Skript herunter, dann:

    ctrl+alt+f1

    und starte

    sgfxi

    sgfxi -h,
um die Optionen zu sehen

    X wird nach der Treiber-Installation neugestartet.

    Laden Sie den binären Treiber direkt von Nvidia,
oder ATI herunter.

Freie xorg-Treiber für nVidia, ATI, Radeon, Intel

  1. sgfxi

    Sie können sgfxi auch nutzen, um zwischen freien und nicht-freien Treibern zu wechseln, oder auf native, freie Treiber zurückzusetzen: Die Schlüsselkommandos sind:

    -n Installiert automatisch den korrekten nativen xorg nv, intel, oder ati Treiber für ihr System.
   Säubert alte Binär-Treiberreste, erneuert die xorg.conf mit neuen Treiberinformationen,
   und entfernt Reste von nvidia und fglrx aus der xorg.conf
-N Erzwinge Installation eines spezifischen xorg Treibers. 
Syntax: sgfxi -N <einer der Treiber aus der Liste, wie z.B. -N radeon>
   Unterstützte Xorg-Treiber: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfiguriert auch die xorg.conf sofern notwendig. Vollautomatische radeon xorg 3d Konfiguration.

    Zuerst

    apt-get update
apt-get install sidux-scripts
sgfxi

    Dann um einen nativen Treiber zu installieren, bzw. diesen zurückzuspielen

    sgfxi -n
ODER
sgfxi -N <xorg driver>

  2. sgfxi / du-fixes-h2.sh

    Sie können sgfxi auch via du-fixes-h2.sh benutzen. Starten sie einfach das Skript mit den folgenden Optionen und es wird sie direkt zur Grafiktreiber-Sektion führen. Wenn sie du-fixes nutzen brauchen sie sgfxi nicht zu installieren, da du-fixes es automatisch installiert.

    Besuchen Sie diedu-fixes-h2.sh-Homepage für mehr Informationen, und führen sie dann dieses Kommando aus:

    du-fixes-h2.sh -kiwdt

  3. Der harte Weg

    Dieser X-Server-Treiber von X.Org (siehe xserver-xorg für eine eingehendere Beschreibung) bietet Unterstützung für die NVIDIA Riva-, TNT-, GeForce- und Quadro-Karten und die Mach-, Rage-, Radeon- und FireGL-Karten von ATI mittels der Subtreiber 'atimisc', 'r128' und 'radeon'. Es wird auch Ünterstützung für die freien 3D-Treiber von Intel angeboten.

    Um ihn zu installieren:

    Aus KDE abmelden und mit Ctrl+Alt+F1 in den Textmodus wechseln
Als root anmelden
und dann "init 3" eintippen
(Eine Nachricht sollte anzeigen, dass der X-Server angehalten wurde.)
apt-get update
apt-get install xserver-xorg-video-nv
ODER
apt-get install xserver-xorg-video-ati
init 5 && exit

    Anmerkung: Dies ist nicht der binäre nvidia/ati-Treiber, welcher 3D-Unterstützung bietet. Der binäre nvidia/ati-Treiber ist aber ausschließlich binär zu erhalten und im Grunde nicht von Linux unterstützt.

    Nun muss die /etc/X11/xorg.conf editiert werden. Auch sollte zuvor eine Sicherungskopie erstellt werden, fälschliches Editieren der /etc/X11/xorg.conf kann sonst viele Tränen und eine zeitaufwendige Fehlersuche mit sich führen.

    Um /etc/X11/xorg.conf zu editieren, muss sie mit Rootrechten geöffnet werden. Im Abschnitt SECTION DEVICE den Display-Treiber in nv ODER ati ändern.

    Manuelles Editieren der xorg.conf geschieht auf eigenes Risiko!

    Mehr Informationen über X.Org

Bildschirmauflösungen, Monitore und Schriftarten

Änderung der Bildschirmauflösung

Als Root wird in der Konsole folgendes durchgeführt:

change-res 1024x768
oder  1600x1200 1400x1050 1280x1024 1152x864 800x600

fix-res 1024x768
oder 1600x1200 1400x1050 1280x1024 1152x864 800x600

Neustart von X mittels 

ctrl+alt+backspace

Dual-Head (zwei) Monitore

Da nun die Konfigurationsdateien von xorg geändert werden, bitten wir, die Datei xorg.conf zu sichern, bevor der nächste Schritt vollzogen wird!

Diese Richtlinien können sich mit der Version xorg 7.3 noch im Jahre 2007 ändern.

Nativer ATI-Treiber - Methode 1

Im Forums-Thread http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=19794#19794 gibt es einige funktionierende Beispiele für die xorg.conf mit dem freien Radeon-Treiber.

ZUR BEACHTEN: Um die Konfigurationsinformationen des zweiten Monitors zu erhalten, muss der erste abgehängt werden und mit der Live-CD gebootet werden, um eine xorg.conf generieren zu lassen. Von dieser xorg.conf muss eine Kopie angelegt werden. Um die Konfigurationsinformationen des ersten Monitors zu erhalten, wird das gleiche Verfahren angewendet.

Nativer ATI-Treiber - Methode 2

Zuerst muss die Auflösung des ersten (primären) Monitors festgelegt werden. Der zweite Monitor nutzt die Auflösungseinstellungen des ersten (primären) Monitors. Beachtet muss werden, dass der zweite Monitor die Auflösungseinstellungen des ersten technisch umsetzen kann.

In der Sektion:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

muss folgendes eingefügt werden:

Option "MergedFB"
Option "CRT2Position" "RightOf"

Danach sieht die Sektion wie folgt aus:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Umfassende Informationen zur Konfigurationseinstellung gibt es hier: http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html

ANMERKUNG: Genannter Link verwendet den Radeon-Treiber, aber die Einstellungen sind auch auf ATI-Treibern getestet worden, und sie funktionieren.

Nicht freier ati-Treiber (fglrx)

Zunächst muss der nicht freie Treiber (fglrx) installiert werden.

get-sidux-gfx
oder
sgfxi / du-fixes-h2.sh

Bitte die Anleitung im sidux-Handbuch beachten.

Im Anschluss daran muss der Grafikserver X verlassen werden.

ctrl+alt+F1
root
[Passworteingabe] # wird nicht ausgegeben
init 3

Dann:

aticonfig --dtop=horizontal
init 5 && exit

Falls die Monitoranordnung vertauscht ist:

aticonfig --dtop=horizontal,reverse

Zur weiteren Information kann die Hilfe-Anleitung gelesen werden, welche sehr ausführlich ist und umfassende Optionen sowie Beispiele bereit stellt:

aticonfig --help

Zur Beachtung: Beide Monitore müssen dieselbe Auflösung besitzen. Im Regelfall stellt man die höchste Auflösung des leistungsschwächeren Monitors ein. Die Spezifikationen der Monitore bitte in den Handbüchern der Monitore nachschlagen, falls die maximale Auflösung unbekannt ist.

In dem Forumsthread http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=20231#20231 ist weitere Information zu finden.

Um von einer Dual-Monitor-Konfiguration zu einer Konfiguration mit einem einzelnen Monitor zurück zu gehen, muss folgender Befehl eingegeben werden:

aticonfig --initial --input=/etc/X11/xorg.conf
nvidia

Für den nicht-freien nvidia-Treiber kann das Konfigurationsprogramm für xorg von nvidia verwendet werden ( http://www.sorgonet.com/linux/nv-online/) und die Einstellungen können in diesem vorgenommen werden. Auch hier ist zu beachten: die maximale Auflösung des schwächeren Monitors soll gewählt werden bzw. beide Monitore müssen auf die gleiche Auflösung eingestellt sein.

Tipps zu Hardware, die unfreie Treiber benötigt

Auf Grund der schwierigen Rechtslage liefert sidux auf der Live-CD nur DFSG-freie Software aus.

Daher könnte es vom Anwender gewünscht sein, contrib/non-free zur /etc/apt/sources.list hinzuzufügen:

# See sources.list(5) for more information, especialy
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • ATI Radeon: 3D-Beschleunigung für ältere Karten bis r35x sollte funktionieren. Neuere Grafikkarten (Radeon X1xxx) benötigen unfreie Treiber für eine beschleunigte Leistung, die jedoch noch nicht mit X.org 7.2 kompatibel sind.
  • Atheros/ "madwifi" wlan: In der Konsole: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: In der Konsole: 
    apt-get install atmel-firmware
  • AVM-Karten (ISDN/ ADSL PCI/ USB): Die nicht offenen Treiber von AVM sind noch nicht kompatibel mit Kernel 2.6.21.

  • Broadcom/bcm43xx wlan: In der Konsole: 
    apt-get install bcm43xx-fwcutter
  • Eagle USB ADSL modem: Die Firmware gibt es bei http://eagle-usb.org/ueagle-atm/non-free/ und wird gespeichert unter: /lib/firmware/

  • DVB Firmware für diverse DVB-TV-Karten (die meisten einfacheren Karten benötigen das nicht). Die benötigte Firmware (dmesg ansehen) bei http://www.linuxtv.org/downloads/firmware/ herunterladen und unter /lib/firmware/ speichern.

  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945/ iwlwifi, 54 Mbit/s wlan: 
    apt-get install iwlwifi-ucode
    Die alten unfreien Treiber können aktiviert werden. Falls iwlwifi Probleme bereitet: 
    apt-get update
apt-get install ipw3945d firmware-ipw3945
update-rc.d ipw3945d start 39 S . start 36 0 6 .
LANG= m-a a-i --text-mode ipw3945-source
echo "blacklist iwlwifi" > /etc/modprobe.d/mac80211
  • Intersil prism54, 54-MBit/s-WLAN: Firmware herunterladen von http://prism54.org/firmware/ und unter /lib/firmware/ speichern.

  • nVidia Grafikkarten: 3D-Beschleunigung ist bisher mit den freien Treibern nicht möglich. Unter Verwendung von get-sidux-binary-gfx lassen sich die unfreien Treiber jedoch nachträglich installieren.

  • RaLink rt61 54-MBit/s-WLAN: 
    apt-get install rt61-fwcutter
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) WLAN: Firmware bei http://www.kazer.org/acx-firmware-20060207.tar.bz2 herunterladen und unter /lib/firmware/ speichern.

  • ZyDAS zd1201 11 MBit/s wlan: 
    apt-get install zd1201-firmware
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Wir untersuchen noch die Möglichkeit, Treiber mancher Geräte mitzuliefern. Der rechtliche Status dieser Treiber ist leider häufig kompliziert.

HD NTFS-Partitionen mit ntfs-3g beschreibbar machen

Achtung: Obwohl der Treiber ntfs-3g nun als stabil erklärt ist, empfehlen wir keine Benutzung ohne vorher die Daten zu sichern, vor allem nicht auf Arbeitsplattformen! Die Benutzung erfolgt auf eigene Gefahr, da ein Datenverlust nicht gänzlich ausgeschlossen werden kann!

In eine Konsole das folgende eingeben: Siehe Partitionieren der Festplatte, um Benennungskonventionen der Festplatte in Linux nachzulesen.

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Zum Verlassen der Konsole: exit

Das NTFS-Volume sollte jetzt mit Schreibzugriff eingehängt sein und man sollte Daten darauf speichern können. Und noch einmal unsere Warnung: Dieses Vorgehen ist für Situationen gedacht, für die es keine anderen Lösungsmöglichkeiten gibt. Für die tägliche Routinenutzung es nicht empfohlen!
