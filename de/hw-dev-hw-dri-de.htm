Binäre, non-free Treiber für: Nvidia / ATI

Es gibt 3 Methoden, 3D-Treiber zu installieren

  1. sgfxi / smxi

    Verwendung des von der sidux-Community bereit gestellten Multifunktions-Tools smxi/sgfxi. Mehr dazu unter smxi Befehls-Syntax.

  2. Benutzung des Nvidia/ATI Binär-Installers
    Die binären Treiber müssen direkt von Nvidia
oder ATI heruntergeladen werden.

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel are pre-installed with sidux. If you have overwitten them with closed source drivers you can revert or roll back to them with sgfxi.

Dieser X-Server-Treiber von X.Org (siehe xserver-xorg für eine eingehendere Beschreibung) bietet Unterstützung für die NVIDIA Riva-, TNT-, GeForce- und Quadro-Karten und die Mach-, Rage-, Radeon- und FireGL-Karten von ATI mittels der Subtreiber 'atimisc', 'r128' und 'radeon'. Es wird auch Unterstützung für die freien 3D-Treiber von Intel angeboten.

Um ihn zu installieren:

Aus KDE abmelden und mit Ctrl+Alt+F1 in den Textmodus wechseln
Als root anmelden
und dann "init 3" eintippen
(Eine Nachricht sollte anzeigen, dass der X-Server angehalten wurde.)
apt-get update
apt-get install xserver-xorg-video-nv
ODER
apt-get install xserver-xorg-video-ati
ODER
xserver-xorg-video-intel
init 5 && exit

For some cards it may add 3D support, however it is usually 2D.

Nun muss die /etc/X11/xorg.conf editiert werden. Auch sollte zuvor eine Sicherungskopie erstellt werden, fälschliches Editieren der /etc/X11/xorg.conf kann sonst viele Tränen und eine zeitaufwendige Fehlersuche mit sich führen.

Um /etc/X11/xorg.conf zu editieren, muss sie mit Rootrechten geöffnet werden. Im Abschnitt SECTION DEVICE den Display-Treiber in nv ODER ati ändern.

Manuelles Editieren der xorg.conf geschieht auf eigenes Risiko!

Tipps zu Hardware, die unfreie Treiber benötigt

Auf Grund der schwierigen Rechtslage liefert sidux auf der Live-CD nur DFSG-freie Software aus.

Sources List

Daher könnte es vom Anwender gewünscht sein, contrib/non-free zur /etc/apt/sources.list.d/debian.list oder /etc/apt/sources.list hinzuzufügen:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Video Cards
  • ATI Radeon: 3D-Beschleunigung für ältere Karten bis r35x sollte funktionieren. Neuere Grafikkarten (Radeon X1xxx) benötigen unfreie Treiber für eine beschleunigte Leistung, die jedoch noch nicht mit X.org 7.2 kompatibel sind. Abhängig vom jeweiligen Chipset können die Cheatcodes "xmodule=vesa" oder "xmodule=fbdev" für neuere Modelle Radeon X1xxx funktionieren.
Firmware
  • Um zu ermitteln, welche Firmware benötigt wird, verwendet man als User diesen Befehl:
$ fw-detect

Damit erhält man eine Empfehlungsliste mit den benötigten Befehlen. Falls nichts zu tun ist oder die Firmware bereits installiert ist, bekommt man die Antwort "null".

  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • Broadcom bcm43xx/b43/b43legacy wlan: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • Broadcom Blutonium Bluetooth Chipset (BCM2033): 
    apt-get install bluez-firmware
  • Eagle USB ADSL modem: Die Firmware gibt es bei http://eagle-usb.org/ueagle-atm/non-free/ und wird gespeichert unter: /lib/firmware/

  • DVB Firmware für diverse DVB-TV-Karten (die meisten einfacheren Karten benötigen das nicht). Die benötigte Firmware (dmesg ansehen) bei http://www.linuxtv.org/downloads/firmware/ herunterladen und unter /lib/firmware/ speichern.

  • IVTV-Firmware: diese muss von hier runtergeladen http://dl.ivtvdriver.org/ivtv/firmware/ und im Ordner /lib/firmware/ gespeichert werden.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan) bzw. ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan Bitte iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 und firmware-iwl4965 entfernen, falls bereits installiert.: 
    apt-get install firmware-iwlwifi
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54-MBit/s-WLAN: Firmware herunterladen von http://prism54.org/firmware/ und unter /lib/firmware/ speichern.

  • nVidia Grafikkarten: 3D-Beschleunigung ist bisher mit den freien Treibern nicht möglich. Unter Verwendung von get-sidux-binary-gfx oder sgfxi lassen sich die unfreien Treiber jedoch nachträglich installieren.

  • RaLink rt61 und rt73 54 MBit/s wlan: 
    apt-get install firmware-ralink

    rt61-fwcutter, rt73-fwcutter, firmware-rt61 und firmware-rt73 müssen entfernt werden, so sie installiert sind.

  • Ricoh R5U870 OEM Webcams: Download der Firmware http://mediati.org/r5u870/r5u870-0.11.0.tar.gz und die Dateien r5u870_*.fw nach /lib/firmware/ kopieren.
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) WLAN: 
    m-a a-i acx100
    danach Download der Firmware von http://www.kazer.org/acx-firmware-20060207.tar.bz2 und speichern unter /lib/firmware/
  • ZyDAS zd1201 11 MBit/s wlan: Download der Firmware http://surfnet.dl.sourceforge.net/sourceforge/linux-lc100020/zd1201-0.14-fw.tar.gz und unter /lib/firmware/ speichern.
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Wir versuchen, Pakete für legal vertreibbare Firmware in unseren non-free-Repositorys zur Verfügung zu stellen, aber nicht alle Anbieter erlauben dies.

HD NTFS-Partitionen mit ntfs-3g beschreibbar machen

Achtung: Obwohl der Treiber ntfs-3g nun als stabil erklärt ist, empfehlen wir keine Benutzung ohne vorher die Daten zu sichern, vor allem nicht auf Arbeitsplattformen! Die Benutzung erfolgt auf eigene Gefahr, da ein Datenverlust nicht gänzlich ausgeschlossen werden kann!

In eine Konsole das folgende eingeben: Siehe Partitionieren der Festplatte, um Benennungskonventionen der Festplatte in Linux nachzulesen.

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Zum Verlassen der Konsole: exit

Das NTFS-Volume sollte jetzt mit Schreibzugriff eingehängt sein und man sollte Daten darauf speichern können. Und noch einmal unsere Warnung: Dieses Vorgehen ist für Situationen gedacht, für die es keine anderen Lösungsmöglichkeiten gibt. Für die tägliche Routinenutzung es nicht empfohlen!
