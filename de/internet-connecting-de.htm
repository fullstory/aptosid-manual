Mit dem Internet verbinden, konfigurieren der Netzwerkkarte

Wenn im LAN, dem lokalen Netzwerk, ein DHCP-Server läuft, sollte sich die Live-CD beim Booten automatisch damit verbinden. Ist das nicht geschehen, muss man sidux' Skript zur Konfiguration der Netzwerkkarte starten, zu finden unter: KMenü > sidux > Netzwerk/Internet > Netzwerkkarte konfigurieren.

Wem der Weg über die grafische Oberfläche nicht zusagt, steht es frei, die Konsole zu benutzen:

sudo
netcardconfig

Nun muss man nur noch den Anweisungen folgen.

Ndiswrapper aktivieren

sidux hat schon alle benötigten Pakete (kernel-module und ndiswrapper-utils) auf der HD installiert.

Du musst nur noch die .inf-Datei für Windows (den Treiber) finden und ihn laden:

ndiswrapper -i /pfad/zu/driver.inf

Lade das Ndiswrapper-Modul:

modprobe ndiswrapper

und überprüfe, ob es korrekt installiert ist:

iwconfig
iwconfig wlan0 essid ESSIDNAME

Überprüfe, ob der Access-Point gefunden wurde:

iwlist wlan0 scan

Dann kannst du das Gerät (Device) konfigurieren mit:

netcardconfig
see also more on the wiki

PC mit zwei Netzwerkkarten, Internetsharing

Wie konfiguriere ich den PC, um mit dem Notebook (oder einem anderen PC) ins Internet zu kommen (Internetsharing). Wir zeigen hier eine einfache minimale Lösung:

  • Öffne eine Konsole und werde mit "sux" root, um die folgenden Schritte auszuführen:

  • Konfigurieren der ersten Karte mit:

    netcardconfig

  • Für die 2. Karte wählen wir eine feste IP-Adresse in einer privaten Domain, z. B. 192.168.1.1, und konfigurieren auch diese:

    netcardconfig

    Achtung: Deine IP muss im gleichen Subnetz sein wie die IPs, die per DHCP zugeteilt werden. Wenn fälschlicherweise ein Gateway während der Konfiguration festgelegt wurde, muss folgendes Script ausgeführt werden:

    remove-gateway.sh

  • Einrichten von DHCP/DNS:

    apt-get install dnsmasq

    Editiere die Datei /etc/dnsmasq.conf durch Hinzufügen einer Zeile wie

    dhcp-range=192.168.1.50,192.168.1.150

    Der Rest der Datei bleibt so, wie er ist.

  • Der DHCP-Server des Desktop-PCs kann jetzt gestartet werden. Wenn die Clients jetzt auf DHCP eingestellt sind, wird automatisch eine IP-Adresse zugeteilt.

    /etc/init.d/dnsmasq start

  • Um die Anfrage des Clients ins Netz zu routen, musst du folgendes Script ausführen:

    /usr/sbin/masquerade.sh

    Danach sollte jeder Client ins Internet kommen.

  • Um diese Konfiguration beim Booten des Desktop-PCs zu laden, aktiviere dnsmasq im Bootprozess mit:

    update-rc.d -f dnsmasq defaults

  • Dann das Masquerade-Script ausführen:

    echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Internetverbindung mit einem 56k Dial-up Modem

KDE hat ein Frontend für Dial-up Modems: KPPP (Internet Dial-up Tool). Es befindet sich in K-Menü > Internet.

Die Anwendung hat eine interne Hilfe und bietet eine gut verständliche Anleitung zum Aufsetzen der Verbindung.

Eine Mini-Firewall

Falls Sie ein kleines Firewall-Skript benötigen, weil Sie eine direkte Verbindung zum Internet aufbauen müssen, probieren Sie folgendes (nicht nötig, wenn Sie sich hinter einem Router befinden):

wget http://rebelhomicide.demon.nl/scripts/mini-miyagi.sh
chmod 755 mini-miyagi.sh 
apt-get remove --purge {firestarter,guarddog}

Starten Sie es mit

./mini-miyagi.sh

Um mini-miyagi zu stoppen

/etc/init.d/mini_miyagi_fw.sh stop
