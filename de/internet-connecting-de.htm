Mit dem Internet verbinden, konfigurieren der Netzwerkkarte

Wenn im LAN, dem lokalen Netzwerk, ein DHCP-Server läuft, sollte sich die Live-CD beim Booten automatisch damit verbinden. Ist das nicht geschehen, muss man sidux' Skript zur Konfiguration der Netzwerkkarte starten, zu finden unter: KMenü > sidux > Netzwerk/Internet > Netzwerkkarte konfigurieren.

Wem der Weg über die grafische Oberfläche nicht zusagt, steht es frei, die Konsole zu benutzen:

netcardconfig

Nun muss man nur noch den Anweisungen folgen.

Für eine Festplatten-Installation kann man 'Kmenu>sidux>Net>Netzwerkkarte konfigurieren' benutzen und braucht lediglich den Anweisungen zu folgen, oder man benutzt eine Konsole (als $user), verwendet:

/usr/sbin/netcardconfig

und folgt dann den Anweisungen.

PC mit zwei Netzwerkkarten, Internetsharing

Wie konfiguriere ich den PC, um mit dem Notebook (oder einem anderen PC) ins Internet zu kommen (Internetsharing)? Wir zeigen hier eine einfache minimale Lösung:

  • Man öffnet eine Konsole und wird mit "sux" root, um die folgenden Schritte auszuführen:

  • Die erste Karte wird konfiguriert mit:

    /usr/sbin/netcardconfig

  • Für die zweite Karte wählen wir eine feste IP-Adresse in einer privaten Domain, z. B. 192.168.1.1, und konfigurieren auch diese:

    /usr/sbin/netcardconfig

    Achtung: Die IP muss im gleichen Subnetz sein wie die IPs, die per DHCP zugeteilt werden. Wenn fälschlicherweise ein Gateway während der Konfiguration festgelegt wurde, muss folgendes Script ausgeführt werden:

    remove-gateway

  • Einrichten von DHCP/DNS:

    apt-get install dnsmasq

    Danach editiert man die Datei /etc/dnsmasq.conf durch Hinzufügen einer Zeile wie

    dhcp-range=192.168.1.50,192.168.1.150

    Der Rest der Datei bleibt so, wie er ist.

  • Der DHCP-Server des Desktop-PCs kann jetzt gestartet werden. Wenn die Clients jetzt auf DHCP eingestellt sind, wird automatisch eine IP-Adresse zugeteilt.

    /etc/init.d/dnsmasq start

  • Um die Anfrage des Clients ins Netz zu routen, muss folgendes Script ausgeführt werden:

    /usr/sbin/masquerade

    Danach sollte jeder Client ins Internet kommen.

  • Um diese Konfiguration beim Booten des Desktop-PCs zu laden, soll dnsmasq im Bootprozess aktiviert werden mit:

    update-rc.d -f dnsmasq defaults

  • Dann das Masquerade-Script ausführen:

    echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Internetverbindung mit einem 56k Einwahlmodem

KDE hat ein Frontend für Einwahlmodems: KPPP (Einwahl ins Internet). Es befindet sich in K-Menü > Internet.

Die Anwendung hat eine interne Hilfe und bietet eine gut verständliche Anleitung zum Aufsetzen der Verbindung.

Firewalls

Firewalls werden normalerweise nicht benötigt, wenn man sich hinter einem ordentlich konfigurierten Router befindet. Dennoch spielen sie eine sehr wichtige Rolle bei der Sicherheit, wenn man sich direkt über ein DSL-(USB-)Modem oder ein Einwahlmodem ins Internet verbinden muss:

apt-get install guarddog
ODER
apt-get install firestarter

Guarddog ist ein benutzerfreundliches Firewall-Werkzeug für ein auf Linux laufendes KDE. Der beste Weg, es zum Laufen zu bringen, ist, die Kurzanleitungen zu lesen.

Firestarter - Eine Open-Source-Firewall mit grafischer Benutzeroberfläche für Linux. [Hinweis: Wenn man die grafische Oberfläche verlässt, bleibt die Firewall im Hintergrund weiterhin aktiv.]
