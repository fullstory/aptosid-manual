Das root-Passwort auf der Live-CD

Achtung: Wann immer man mit root-Rechten arbeitet, sollte man wissen, was man macht. Für das Surfen im Internet sind keine root-Rechte nötig.

Die Eingabe von sux ermöglicht root mit Xapp-Privilegien (Start von Programmen mit graphischer Oberfläche).

Auf der Live-CD ist kein root-Passwort gesetzt. Es gibt mehrere Möglichkeiten, ein Programm mit root-Rechten auszuführen:

Ein (temporäres) root-Passwort setzen:

Eine Konsole öffnen:

sidux@0[sidux]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

Dieses Passwort kann den Rest der Live-Sitzung verwendet werden.

Ein Programm kann als root mit einem der folgenden Befehle gestartet werden

sudo

sux

kdesu

Man drückt Alt+ F2 und gibt 

sudo

vor dem gewünschten Programmnamen ein, z. B.:

sudo gparted
Das Programm aus einem root-Terminal starten:

Man öffnet ein Terminal/eine Konsole und gibt z. B. ein:

sidux@0[sidux]$ sux
root@0[sidux]# gparted (oder das gewünschte Programm)

Um den root-Modus zu verlassen, tippt man in die Konsole:

exit

oder man klickt das "x" in der rechten oberen Ecke zum Schließen des Konsolenfensters.

Die Installation von Software während einer Live-CD-Sitzung

apt-get update
apt-get install ihr-gewünschtes-paket

Achtung: Wenn Sie die Live-CD herunterfahren, werden keine Änderungen behalten.

NTFS-Partitionen mit ntfs-3g beschreibbar machen

Achtung: Obwohl der Treiber ntfs-3g nun stabil erklärt ist, empfehlen wir keine Benutzung ohne vorher Daten zu sichern, vor allem nicht auf Arbeitsplattformen! Die Benutzung erfolgt auf eigene Gefahr, da ein Datenverlust nicht gänzlich ausgeschlossen werden kann!

Folgende Konsolenbefehle werden benutzt (siehe Partitionieren der Festplatte, um Benennungskonventionen der Festplatte in Linux nachzulesen):

sidux@9[sidux]$ sux
root@0[sidux]# apt-get update && apt-get install ntfs-3g
root@0[sidux]# umount /media/xdxx
root@0[sidux]# mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Zum Verlassen der Konsole gibt man nur ein: exit

Das NTFS-Volume sollte jetzt mit Schreibzugriff eingehängt sein und man sollte Daten darauf speichern können. Und noch einmal unsere Warnung: Dieses Vorgehen ist für Situationen gedacht, für die es keine anderen Lösungsmöglichkeiten gibt. Für die tägliche Routinenutzung es nicht empfohlen!
