Die Befehls-Syntax von smxi/sgfxi

Das von der Community entwickelte Skriptpaket zur Systemadministration smxi, kurz für sidux-maintenance ("sidux-Wartung"), kann man auf den Computer laden und installieren.

Download und Installation des Skriptpakets und Start von smxi:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip && smxi
Danach folgt man den Anweisungen

Download, Installation und Start nur des 3D-Grafiktreiber-Skriptes sgfxi:

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
Liste der Optionen:
sgfxi -h

Die grundlegenden Funktionen des smxi-Skripts

Dieses Skript beinhaltet viele Funktionen, von denen viele nur einmal dem Benutzer zur Auswahl angeboten werden, andere wiederum immer. Nach einer Systemüberprüfung und einigen Abfragen zu Konfigurationseinstellungen werden folgende Hauptoptionen angeboten (das Skript hat eine englischsprachige Benutzerführung, daher ist nicht alles übersetzt):

  • Script information. Dieser Abschnitt zeigt den aktuellen Kernel, wann zuletzt ein Upgrade mit dem Skript durchgeführt wurde, wann das Skript zuletzt benutzt wurde und weitere Informationen. Dies ist besonders nützlich auf Systemen, die nicht sehr oft aktualisiert werden (Empfehlung: eine Aktualisierung mindestens einmal im Monat. Weniger oft kann Risiken mit sich bringen).
  • Kernel install. Als Grundeinstellung wird der aktuelle stabile Kernel angeboten, in den erweiterten Optionen ("advanced options") können weitere, auch ältere Kernelversionen installiert werden. Wenn smxi mit der Option -K gestartet wird, wird der aktuelle Test-Kernel als erster in der Liste als "stable" angeboten, so einer vorhanden ist.
  • dist-upgrade warning section: Diese Sektion bietet Hinweise zu gegenwärtigen und älteren Problemen in Debian Sid. Dieser Teil sollte genau beachtet werden, da Debian Sid häufig Anpassungen vor einem dist-upgrade benötigt. Die Fortsetzung dieses Menüpunktes (continue) führt zu
  • pre dist-upgrade fixes: Hier werden Anpassungen vorgenommen, die vor einem dist-upgrade notwendig sind.
  • config file alerts: Hier sind Konfigurationsdateien gelistet, welche während des Upgrade-Vorgangs erneuert werden müssen. apt-get bietet als Standardeinstellung für Konfigurationsdateien die Option "N" an, um vom Nutzer angepasste Konfigurationsdateien nicht anzutasten. Die hier gelisteten Konfigurationsdateien müssen jedoch aktualisiert werden, benötigen daher die Antwort "Y", falls während eines dist-upgrades von apt-get eine Anfrage gestellt wird.
  • dist-upgrade section: Dieser Abschnitt beinhaltet Möglichkeiten zur Fehlerbehebung und informiert, ob der Upgrade fehlerfrei oder mit Fehlern durchgelaufen ist.
  • post dist-upgrade fixes. Automatisierte Fehlerbehebung nach einem Upgrade, falls benötigt.
  • miscellaneous options. Verschiedene Optionen werden angeboten:
    • fix-fonts - bietet eine Auswahl zur Behebung von Fehlern bei der Schriften-Registrierung. Dies ist wichtig nach einer Erstinstallation, wird danach jedoch selten benötigt.
    • package-install - allgemein gewünschte Programmpakete werden zur Installation angeboten, darunter nicht freie Software wie flash oder eine Installationsroutine für Google Earth, aber auch Büroprogramme. Zur Beachtung: die Installationsroutine für Openoffice.org bietet auch eine Auswahl zur Installation von Sprachpaketen und weiteren Zusatzprogrammen. Bitte schauen Sie die Liste durch.
    • package-removal - eine Liste von Paketgruppen, die eventuell deinstalliert werden sollen: wifi, Sprachkomponenten u.a.
    • clean-up-stuff - hier werden Säuberungsoptionen angeboten wie zum Beispiel eine Routine zur Entfernung von Kerneln (zur Beachtung: um Problemen vorzubeugen, empfehlen wir, immer einen alten Kernel zusätzlich installiert zu haben).
    • miscellaneous-tweaks - hier werden Konfigurationsoptimierungen angeboten wie zum Beispiel Einstellungen in Mozilla-Produkten (der File-Handler zum Verwalten von Dateien). Sehen Sie sich die Liste dieser nützlichen Skripte durch.
    • continue - dies führt zum nächsten Schritt, der Installation von Grafiktreibern.
    • quit - beendet smxi an dieser Stelle, falls keine weiteren Optionen benötigt sind. Beachtet werden soll, dass die Installationsroutine für Grafiktreiber eine Option kde/init 5 hat, womit jederzeit zum Desktop zurückgekehrt werden kann.
  • graphics installer - dieses Skript benutzt den sgfxi-Grafiktreiber-Installer von sidux, um proprietäre Treiber von nVidia und in der Regel ATI-fglrx zu installieren (ati war im letzten Jahr immer wieder fehlerhaft, aber die meiste Zeit funktioniert der Treiber). Weiters wird die Installation freier xorg-Treiber angeboten wie eine Rückkehroption zum X-Desktop.

Wie man die Skript-Optionen von smxi nutzt:

Alle Optionen zusammen: 

-deghiIklmrstuwx

oder getrennt: 

-giktw -f -x

Das Skript kann mit Kombinationen jeder dieser Optionen gestartet werden.

Die Optionen können zusammen angegeben sein (wie hier: -ik) oder getrennt (wie hier: -w -x).

Jede Option bzw. Gruppe von Optionen muss beginnen mit:

 -

Beispiel:

 smxi -ikw

Obiges Beispiel überspringt die System-Information, die Kernel-Installation und die Warnungen.

smxi: Skript-Optionen und Überspringen von Teilen des Skripts

Folgende Optionen erlauben das Überspringen von Teilen des Skripts:

  • -d überspringt den dist-upgrade, wenn auch die Option w gewählt ist.
  • -e überspringt einige Fragen zur Bestätigung.
  • -g überspringt die Frage nach der Installation von Grafiktreibern.
  • -i überspringt den Abschnitt zur Systeminformation.
  • -k überspringt die Frage nach der Kernelinstallation.
  • -r überspringt die Überprüfung der letzten Version des Skripts und dessen Installation wie Neustart. Dies ist nur empfohlen bei einem Neustart nach einer Kernel-Installation.
  • -s überspringt einige Teile des Skripts, die eine Internetanbindung benötigen. Dies sollte vermieden werden, da die meisten Funktionen des Skripts damit außer Kraft gesetzt werden.
  • -S überspringt den Test der Internetverbindung. Dies sollte dann benutzt werden, wenn der Test nicht gelingt, Sie aber sicher sind, eine funktionierende Internetanbindung zu haben (diese Option ist nicht empfohlen und ist auch nicht immer aktiviert).
  • -t überspringt die Optionen nach einem dist-upgrade.
  • -w überspringt die Warnungen vor einem dist-upgrade.

Nun ein Beispiel:

smxi -kwid

geht direkt zu den weiteren (misc) Optionen nach dem dist-upgrade. smxi -ki überspringt die Frage nach der Kernel-Installation und die Informationen.

Erweiterte Optionen von smxi

Die erweiterten Optionen werden im Allgemeinen nicht benötigt, können aber für spezielle Zwecke sehr nützlich sein:

  • -I setzt andere Runlevels (Init-Levels) für den Neustart von X. Dies benötigt die Synatx -I 3.
    Zur Beachtung, die Ziffer ist der Runlevel (Init-Level), in den das Skript X starten soll.
  • -K Option zur erweiterten Kernelinstallation. Falls neuere Test-Kernel vorhanden sind, werden diese in den Installationsoptionen angezeigt. Die Installation erfolgt auf eigenes Risiko!
  • -l erstellt Protokolldateien von allen Updates und dist-upgrades mit einem Zeitstempel. Die Protokolldateien finden sich in /var/log/smxi/
  • -m Mit dieser Option können die Debian-Spiegelserver in /etc/apt/sources.list geändert werden.
  • -M Mit dieser Option können Sie einen Spiegelserver für den Download der Kernel auswählen
  • -p Mit dieser Option erfolgt apt-get update, ohne dass pdiffs benutzt werden. Empfehlenswert für sehr schnelle Verbindungen.
  • -P Mit dieser Option können die Proxy-Einstellungen für die Internetverbindung gesetzt werden. Die Syntax ist:
    -P http://[<username><:password>@]<server><:port>
  • -U erzwingt das Update des Skripts.
  • -x exekutiert die erweiterten Skript-Optionen, die nicht Teil der zentralen Funktionalität sind.
  • -h gibt das Hilfe-Menü des Skripts aus.
  • -v Ausgabe von Informationen zu Version, System, Kernel, letzter Verwendung von 'apt-get update' und 'apt-get dist-upgrade' sowie der letzten Verwendung des Skriptes 'smxi'.

Benutzeroptionen

Das Skript kann weitere Funktionen ausführen, wenn die Konfigurationsdatei /etc/smxi.conf bearbeitet wird.

Beachtet werden muss, dass die Konfigurationsdatei erst nach dem ersten Durchlauf des Skripts vorhanden ist. Vor einem ersten Durchlauf muss diese Datei selbst erstellt werden, falls dies gewünscht ist.

  • sgfxi=<einige Standard-Optionen von sgfxi> - ein Beispiel:
    sgfxi=b
nach
/etc/smxi.conf,

    Die Installation des nVidia-Grafiktreibers nutzt automatisch das Argument -t wie auch das Standardargument -c.
    Beachtet werden muss, dass dies eine Standardoption von sgfxi ist und nicht von smxi. smxi hängt diese Option nur an sgfxi an, wenn der Grafiktreiber installiert wird. Erweiterte Optionen wie -o 8.33.6 werden nicht unterstützt, sondern nur Optionen mit einem einzelnen Buchstaben.
  • hold-install=<Paketliste getrennt mit ^> - Diese Option ermöglicht, Pakete in die automatisierte hold/release-Funktion des Skripts einzubinden. Dies hilft, temporäre Probleme mit Programmpaketen zu vermeiden oder Programmpakete zu behalten, die nicht aktualisiert werden sollen, ohne dies zu vergessen. Zum Beispiel:
    hold-install=nano^libgtk2.0
    Wichtig: nur ^ darf als Trennzeichen zwischen Paketnamen verwendet werden, kein Leerzeichen!

sgfxi

Download, Installation und Start des 3D-Grafiktreiber-Skriptes sgfxi:

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
Liste der Optionen:
sgfxi -h

sgfxi unterstützt im Augenblick ATI/fglrx- und Nvidia-Treiber. Auch wird die Konvertierung von oder zu freien xorg-Treibern wie ati, intel oder nv unterstützt. smxi nutzt sgfxi als Installationsbackend für Grafiktreiber und fügt einige Optionen zur einfacheren Benutzung hinzu.

Der Default-Treiber wird installiert, wenn keine weiteren Optionen für die Installation angegeben werden. Auch erkennt das Skript automatisch ältere Grafikkarten und installiert Legacy-Treiber ohne weiteres Zutun.

Nach Start des Skriptes und Erkennung der Grafikkarte wird ausgegeben, welcher Treiber installiert werden soll. Man hat die Möglichkeit, diese Auswahl zu bestätigen und den Treiber zu installieren oder den Installationsvorgang abzubrechen und mit einer selbstgewählten Option das Skript neu zu starten.

Der sgfxi-Installer kann auch mit smxi ausgeführt werden. Dieses Skript benutzt auch sgfxi, um die Grafik-Treiber zu installieren. Mehr Informationen auf der smxi-Homepage. Mit diesem Befehl wird die Grafikkarteninstallation gestartet:

smxi -kiwdt

sgfxi -h listet alle Optionen. Die Optionen werden im Laufe der Zeit ergänzt. Auch werden mit sgfxi -h die augenblicklich unterstützten Treiber und Spezifikationen des benutzten Systems wie Kernel und xorg-Version angezeigt. Nachfolgend einige nützliche sgfxi-Optionen:

-c Benutzung des Composite-Modus (nur für nVidia Karten).
-d Entfernt IgnoreDisplayDevices, um andere Ausgabegeräte wie einen TV-Monitor nutzen zu können (nur für nVidia Karten).
-h Öffnet dieses Hilfe-Menü.
-n Installiert automatisch den korrekten nativen xorg nv, intel, oder ati Treiber für ihr System.
   Säubert alte Binär-Treiberreste, erneuert die xorg.conf mit neuen Treiberinformationen,
   und entfernt Reste von nvidia und fglrx aus der xorg.conf
-N Erzwingt die Installation eines spezifischen xorg Treibers. 
Syntax: sgfxi -N <einer der Treiber aus der Liste, wie z.B. -N radeon>
   Unterstützte Xorg-Treiber: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Konfiguriert auch die xorg.conf sofern notwendig. Vollautomatische radeon xorg 3d Konfiguration.
-o Überschreibt Grundeinstellung: Installiert einen spezifischen Treiber. Dies ist die Syntax eines völligen Overrides (alles wird überschrieben):
   -o 9746 (für nVidia, benutzen Sie nicht den Teil 1.0-) oder: -o 8.32.5 (für fglrx)
-r Entfernt "modelines" aus der xorg.conf
-t Aktiviert dynamische Verwaltung von Doppelmonitoren (dynamic twinview) (nur nVidia)

Rückgriff auf quelloffene Treiber

Quelloffene Xorg-Treiber für nVidia, ATI, Radeon und Intel werden von sidux vorinstalliert. Falls diese von unfreien Treibern überschrieben wurden, kann mittels sgfxi auf die quelloffenen Treiber zurückgegriffen werden. Als Erstes:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip; smxi
Als root gibt man ein: 
smxi
und folgt den Anweisungen :: Skript-Homepage - http://techpatterns.com/h2

(Neu)Installation der quelloffenen Treiber:

sgfxi -n
ODER
sgfxi -N <Xorg driver>

Die wichtigsten Befehlsoptionen sind:

-n Installiert automatisch den richtigen nativen Xorg-Treiber (nv, intel oder ati).
   Entfernt Reste der binären Treiber und schreibt die neuen Treiberinformationen nach xorg.conf.
-N Erzwingt Installation eines spezifischen Xorg-Treibers. 
Syntax: sgfxi -N <einer der Treiber aus der Liste, zum Beispiel: -N radeon>
 Unterstützte Xorg-Treiber sind: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
 Auch wird xorg.conf neu konfiguriert. Eine vollständige radeon-Xorg-3D-Konfiguration wird automatisch erstellt.
