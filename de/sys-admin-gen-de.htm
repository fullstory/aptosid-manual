Dienste starten in sidux

Folgende Dienste sind bei sidux in der Grundeinstellung gestartet: cpid, cron, dbus, hotkey-setup, kdm, klogd, sysklogd und sie werden aktualisiert, wann immer ein dist-upgrade durchgeführt wird.

Zu beachten ist, dass Dienste, die während des dist-upgrade durch neue Versionen aktualisiert werden, danach aktiviert sind!

Um einen Dienst während einer Sitzung zu starten, wird folgender Befehl ausgeführt:

/etc/init.d/service start

oder

rcconf

um einen Dienst während des Bootvorgangs zu aktivieren.

Anmerkung zu rcconf: man verwendet Pfeile zum Scrollen (auf/ab), Leertaste zur Auswahl, Tabulator zum Markieren von "Ok" oder "Abbrechen", Enter-Taste um "Ok" oder "Abbrechen" auszuführen.

Eine Alternative zu rcconf ist:

update-rc.d service defaults

Vergessenes Rootpasswort

Ein vergessenes Rootpasswort kann nicht wiederhergestellt werden, aber ein neues kann gesetzt werden.

Dazu muss zuerst die Live-CD gebootet werden.

Als Root muss die Rootpartition eingebunden werden (z. B. als /dev/hdb2)

# mount /dev/hdb2
Nun ein chroot in die Rootpartition
(chroot=changed root)
# chroot /media/hdb2
und der Befehl passwd wird zum Erstellen eines neuen Passworts verwendet:
# passwd

Schriftarten in sidux

Korrekte DPI-Einstellung und Grundlegendes

DPI-Einstellungen sind schwierig zu ermitteln, aber letztlich durch xorg perfekt umgesetzt, wenn DisplaySize die korrekten physischen Bildschirmdimensionen in der Monitor-Sektion ("Monitor Section") von /etc/X11/xorg.conf angibt.

Korrekte Bildschirmauflösungen und Bildwiederholungsraten

Jeder Monitor hat seine eigene perfekte Einstellungskombination, aber leider übergeben nicht alle Monitore die perfekten DCC-Werte, so muss manchmal manuell eingegriffen werden, um die Grundeinstellungen zu überschreiben.

Korrekte Grafiktreiber

Einige neuere Grafikkarten von ATI und Nvidia harmonieren nicht besonders mit den freien Xorg-Treibern. Einzig vernünftige Lösung ist in diesen Fällen die Installation von kommerziellen closed-source Treibern. Aus rechtlichen Gründen kann sidux diese nicht vorinstallieren. Eine Anleitung zur Installation dieser Treiber finden Sie auf folgender Seite des Handbuchs.

Basisauswahl der Schriftarten, ihre Darstellungsart und ihre Größe

sidux nutzt freie Fonts, die sich in Debian als ausgewogen bewährt haben. Wenn eigene Schriftarten gewählt werden, müssen eventuell neue Konfigurationsanpassungen als Root vorgenommen werden, um das gewünschte Schriftbild zu erhalten. Neben den Einstellungsmöglichkeiten im KDE-Kontrollzentrum (kcontrol) gibt es bei Debian einige mächtige Einstellungsoptionen zum Erzielen einer sauberen Schriftartendarstellung. Beachtet werden muss jedoch, dass jede Schriftart ein ideales Größenspektrum besitzt, sodass andere Größen mit der gleichen Schriftart nicht zum gleich optimalen Ergebnis führen müssen.

Eine Änderung der DPI-Größe kann auch zu einer besseren Schriftartendarstellung beitragen. Dazu wird dieser Befehl verwendet:

fix-dpi-kdm

Dies sollte die passende DPI-Größe für den verwendeten Monitor anzeigen, aber sie kann geändert werden. Um die Änderungen wirksam werden zu lassen, muss der Grafikserver X neu gestartet (init 3 - init 5) oder ein Reboot durchgeführt werden.

Nach Änderung einer Schriftart oder der DPI-Größe (in X oder Firefox/Iceweasel), müssen eventuell noch einige Anpassungen vorgenommen werden, besonders wenn von einem Bitmap Font zu einem True Type Font gewechselt wurde (oder umgekehrt):

dpkg-reconfigure fontconfig-config

Die ersten beiden Einstellungen sind am besten (1) "native" und (2) "autohinter" als "automatic". Andere Einstellungsmöglichkeiten können ausprobiert werden. Der dritte Menupunkt lässt für Bitmap-Schriften ("yes") oder True-Type-Schriften ("no") optimieren.

Falls dieser Befehl Änderungen nicht gestattet, kann der Neuaufbau des Font-Caches eine Lösung bedeuten (der zweite Befehl ist ohne Zeilenumbruch, d. h. in einer Zeile einzugeben):

rm -rf /etc/fonts
apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config

Zu beachten ist auch, dass nicht jede Schriftart in allen Größen die gleiche Schriftqualität liefert.

Voreinstellungen in Firefox/Iceweasel

Programme, die auf GTK2 basieren, sind bezüglich Schriftbild in KDE-Umgebungen problematisch. Eine der Lösungsmöglichkeiten ist die Installation von:

apt-get install gtk2-engines gtk-qt-engine

Im KDE-Kontrollzentrum unter Punkt "Erscheinungsbild" wird es nach Installation einen neuen Menüpunkt geben: "GTK-Stile und Schriftarten". Dort kann die Nutzung der KDE-Fonts in GTK2-Applikationen ausgewählt werden, und als GTK-Stil hat sich "Clearlooks" bewährt.

Bei Stabilitätsproblemen der gtk-qt-engine empfehlen wir eine Deinstallation, obwohl seit 2006 diese engine stabil lief (Stand: Ende März 2007). Die Einstellungen in den $HOMES bleiben erhalten.

apt-get remove gtk-qt-engine
oder
apt-get remove --purge gtk-qt-engine

Dieses Vorgehen kann helfen, Probleme der Schriftendarstellung in GTK-Programmen zu lösen.
