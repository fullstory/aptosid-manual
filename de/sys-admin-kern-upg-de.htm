Neue Kernel installieren

Die sidux-Kernel sind via apt beziehbar. Man wählt den nächstgelegenen sidux-Spiegelserver und adaptiert seine $sources.list:

Folgende Kernel sind verfügbar:

  • sidux-686 - Kernel für die i686-Prozessorfamilie mit Single/Dual/Multi Core Prozessoren
  • sidux-amd64 - Kernel für 64 bit sidux
Vier Schritte in einer Root-Konsole:

1. Aktualisierung der Paketdatenbank:

apt-get update

2. Installation des aktuellen Kernels:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. Die Aktualisierung des Kernels ist nun abgeschlossen. Falls benötigt, können nun die neuesten Grafikkartentreiber von ATI, NVIDIA oder INTEL installiert werden. Dazu aktualisiert man als Erstes das Skript:

sgfxi -U

4. Nun wird der Computer mit dem aktualisierten Kernel neu gestartet (dieser sollte jetzt Standardkernel sein).

Die 3D-Treiber werden mit diesem Befehl aktualisiert:

sgfxi

Alternativ kann beim ersten Neustart mit dem neuen Kernel direkt in den Runlevel 3 gegangen werden (man fügt eine 3 der Grub-Bootzeile hinzu) und startet als Root sgfxi.

Hier finden sich mehr Informationen zu sgfxi

Falls sich mit dem neuen Kernel Probleme zeigen, kann man nach einem Neustart einen älteren Kernel wählen.

Module

Mit diesem Befehl können die benötigten Module ermittelt werden (bitte kopieren und in eine Konsole einfügen):

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Damit erhält man eine ausführliche Beschreibung eines jeden Moduls (bitte kopieren und in eine Konsole einfügen):

apt-cache search 2.6.*.slh.*-sidux

So installiert man benötigte Module (zum Beispiel virtual-ose und qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (BEISPIEL)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (BEISPIEL)

So überprüft man, welche Module in den Kernel geladen sind:

ls /sys/module/
oder
cat /proc/modules

WICHTIG: falls bestimmte Treibermodule benötigt werden, zum Beispiel für WLAN, müssen die diesbezüglichen DEB-Pakete installiert werden. Hier als Beispiel der Agere et131x Ethernet-Adapter: 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb kann nur benutzt werden, wenn das DEB-Paket lokal gespeichert ist. Aber selbst dann ist, wenn das Paket sich im apt-Cache befindet, apt-get eine bessere Option.

Siehe auch Internet und Netzwerk bzw. WIFI - WLAN

Andere Methoden der Kernelinstallation

smxi

Das Skript smxi ermöglicht eine Kernel- und Treiberinstallation und bietet auch eine Reihe weiterer integrierter Skripte zur Systemadministration.

So wird smxi installiert:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip && smxi
## danach folgt man den Anweisungen

So wird das Skript sgfxi zur Installation von 3D-Grafikkartentreibern installiert:

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
## Die Optionen werden durch diesen Befehl gelistet:
sgfxi -h
siduxcc

Kernel können auch mit dem graphischen Kontrollzentrum siduxcc installiert werden, welches sich hier befindet: KDE-Start-Menü > System > sidux Kontrollzentrum. siduxcc bietet auch eine Reihe weiterer Anwendungen zur Systemadministration.

Entfernen alter Kernel

Nach erfolgreicher Installation eines neuen Kernels können alte Kernel entfernt werden. Es ist jedoch empfohlen, alte Kernel einige Tage zu behalten. Falls mit dem neuen Kernel Probleme auftauchen, kann in einen der alten Kernel gebootet werden, welche im Grub-Startbildschirm gelistet sind.

Zur Entfernung alter Kernel kann das Skript kernel-remover verwendet werden, welches sich im Paket sidux-kernelhacking befindet. Dieses Skript muss installiert werden und ermöglicht eine genaue Auswahl, welche Kernel entfernt werden sollen.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Als Alternative kann siduxcc verwendet werden, welches das Skript kernel-remover nutzt.
