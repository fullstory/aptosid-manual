Neue Kernel installieren

Die sidux-Kernel sind hier zu finden und in folgenden Formaten kompiliert:
  • smp - Kernel für i686-Prozessoren mit einem, zwei oder mehreren CPU-Kernen
  • 64-smp - Kernel für AMD64-Prozessoren, wenn sidux 64 bit verwendet wird

Es ist hilfreich, sich vor Installation die Kernelversion, die benötigt wird, zu notieren (im folgenden nennen wir das Archiv "kernel.zip", der wirkliche Name sieht aber zum Beispiel so aus: kernel-2.6.21.3-slh-smp-4.zip ). Der Kernel kann mit Hilfe der Konsole als normaler User werden:

Beispiel einer Installation als User (in der Konsole)
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.21.3-slh-smp-4.zip
unzip kernel-2.6.21.3-slh-smp-4.zip -d kernel-2.6.21.3-slh-smp-4
cd kernel-2.6.21.3-slh-smp-4
./install-kernel-sidux.sh       # fragt nach dem Root-Passwort

Der Computer muss jetzt neu gestartet werden, und der neue Kernel wird automatisch ausgewählt. Falls der neue Kernel Probleme bereitet, muss der Computer neu gestartet werden, und im Boot-Menu kann der alte funktionierende Kernel gewählt werden.

WICHTIG: wenn bestimmte Treiber-Module, zum Beispiel für WLAN, benötigt werden, müssen diese via .deb-Pakete installiert werden, die sich im Ordner des entpackten kernel.zip befinden. Die madwifi Treiber-Module werden als Root folgendermaßen installiert:

dpkg -i madwifi-modules*.deb

Weitere Informationen hier zu Internetverbindung und ndiswrapper sowie hier für WIFI

Das Kernel-Update ist nun abgeschlossen. Falls 3D-Grafiktreiber für NVIDIA oder ATI Grafikkarten installiert werden sollen, muss dies vor dem Aufruf des X-Grafikservers geschehen. Am einfachsten bootet man in den Runlevel 3 ("init 3"), indem man in die Befehlszeile des Bootmanagers Grub einfach eine 3 eintippt. Mehr Informationen dazu finden sich hier.

Eine halbautomatisierte Methode einen neuen Kernel zu installieren ist das Skript du-fixes-h2.sh.now called smxi, which is short for sidux-maintenance:
apt-get update
apt-get install sidux-scripts
smxi

Entfernen alter Kernel

Nach der erfolgreichen Installation eines neuen Kernel können kernel.zip und das Kernel-Archiv gelöscht werden, obwohl es empfehlenswert ist, diese und die älteren Kernel einige Tage zu behalten, falls es zu unvorhergesehenen Problemen kommt und mit einem älteren Kernel gebootet werden muss. Ältere Kernel sind im Menu des Bootmanagers Grub gelistet.

Alte Kernel können mit dem Skript kernel-remover entfernt werden, welches sich im Paket sidux-kernelhacking befindet. Dieses Skript listet die Kernel, welche entfernt werden können.
