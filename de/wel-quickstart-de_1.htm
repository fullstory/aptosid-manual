sidux Kurzanleitung

sidux ist voll kompatibel zu Debian Sid. sidux-Pakete, welche aktuelle Pakete aus Debian Sid überschreiben, sind ausschließlich von temporärer Natur und nur so lange in den Repositorien von sidux, bis das Problem upstream von Debian gelöst ist. Die eigenen sidux-Pakete enthalten Skripte und Hilfsprogramme.

Zur Stabilität von Debian Sid

'Sid' ist der Name der "unstable" Repositorien von Debian. Debian Sid wird regelmäßig mit neuen Softwarepaketen beschickt, wodurch diese Debian-Distribution sehr zeitnahe die neuesten Versionen der der jeweiligen Programme enthält. Dies bedeutet aber auch, dass zwischen einem Release im Upstream (vom Softwareentwickler) und der Verteilung in Debian Sid weniger Zeit ist, um die Pakete zu testen.

Der sidux-Kernel

Der Linux-Kernel von sidux ist optimiert, um folgende Ziele zu erreichen: Problembehebung, erweiterte und aktualisierte Funktionen, bessere Performance, höhere Stabilität. Basis ist immer der aktuelle Kernel von http://www.kernel.org/.

ZUR BEACHTUNG: Kernel werden nicht mit apt-get installiert. Diese Vorgehensweise installiert die Kernel von Debian, welche nicht die sidux eigenen Patches und Module zur Funktionserweiterung und Fehlerbehebung beinhalten, die mit dem vorkompilierten Kernel von sidux mitgeliefert werden.

Das ZIP-Archiv des sidux-Kernel enthält das Skript install-kernel-sidux.sh , welches die passenden Pakete installiert und den neu installierten Kernel als Standardkernel setzt.

Eine Liste der Spiegelserver mit dem sidux-Kernel findet sich hier: Neuen Kernel installieren

Die Verwaltung von Softwarepaketen

sidux is fully compliant with vanilla Debian sid. Debian uses apt-get and dpkg for software package management.

apt-get is used for management of packages in conjunction with debian and other repositories identified by the files at /etc/sources.list.d/*

Debian sid's repositories are updated once or twice a day on average so the list of available packages and their versions get out of date fast. Running apt-get update is essential before installing any new packages or before running apt-get dist-upgrade to keep up-to-date with the server's list of packages.

apt-get dist-upgrade is the supported way of upgrading sidux. Using any graphical front end to update sidux is not supported. Upgrade of an Installed System - dist-upgrade

A dist-upgrade is supported only outside of X. Running init 3 from your window manager (KDE, XFCE, etc) or in a virtual terminal (ctrl+alt+f1, ctrl+alt+f2, etc) will stop X from running and allows you to upgrade safely.

smxi

smxi is included in the latest sidux live-cd's, and is a great alternative for running a dist-upgrade. It handles broken packages and is monitored by sidux members to prevent upgrades to problem packages and includes hotfixes to major issues if they arise. Peruse the sidux forum if you are interested on its full functionality.

smxi, besides assisting users with a dist-upgrade, also allows quick and easy installation of new updated sidux kernels. smxi also includes some level of automation for installing virtualbox-ose and qemu.

The smxi Command Syntax sets

smxi

Das Skript smxi ist in den aktuellen sidux Live-CDs enthalten und bietet eine ausgezeichnete Alternative, um eine Systemaktualisierung durchzuführen. Temporäre Probleme einer Aktualisierung von Debian Sid werden von der sidux-Gemeinde beobachtet und Lösungen sind zeitnah in dieses Skript inkludiert, um eine sichere Aktualisierung zu gewährleisten. Für Interessierte: der Funktionsumfang und die aktuellen Problemlösungen werden im Forum von sidux angekündigt und diskutiert.

smxi ermöglicht auch die einfache und schnelle Installation neuer sidux-Kernel sowie die Installation virtueller Umgebungen wie virtualbox-ose und qemu.

Hier findet man die Syntax der smxi-Befehle.

sgfxi - Graphics Driver Installer (proprietary nVidia / ATi and Open Xorg drivers)

Dieses Skript automatisiert die Installation der proprietären Grafiktreiber nvidia und fglrx, die man für den vollen Funktionsumfang von Grafikkarten der Hersteller nVidia und ATi benötigt.

NVIDIA - sgfxi ist sehr empfohlen zur Installation der nvidia-Pakete Debians. Diese sind meist am Tag der Veröffentlichung verfügbar. Das Skript aktiviert die Optionen in xorg.conf, um volle 3D-Funktionalitäŧ zu erhalten.

ATI - sgfxi kann nur dann erfolgreich sein, wenn die Pakete von flrx in Debian Sid mit der aktuellen Version des Grafikservers X.Org und dem benutzten Kernel zusammenwirken können. Diese Eigenschaft kann häufig zu Kompatibilitätsproblemen führen. Bei Problemen wird gebeten, das Forum oder den IRC-Chat-Kanal von #sidux zu Rate zu ziehen.

sgfxi kann jeder Zeit dazu benutzt werden, um die freien Treiber zu installieren. Die Befehle sind -n <treibername> oder -N <treibername> command

Hier findet man ausführlichere Informationen zur Installation binärer Treiber von nVidia bzw. ATi.

Konfiguration von Netzwerken

Ceni is a network configuration tool to quickly configure your network or wireless card with little fuss. The wireless function can scan for networks, use wep and wpa for encryption, and use wireless-tools or wpa_supplicant for wireless configuration. Ethernet is straight forward if using dhcp (automatic ip address assigning) or you can manually set it from netmasks to nameservers.

Ceni is run with the command Ceni or ceni . If not installed, you can install it with the command apt-get install ceni.

Mehr Informationen unter Internet und Netzwerk - Ceni

Weitere Desktopumgebungen

KDE and fluxbox werden von sidux ausgeliefert. Gnome ist von sidux nicht unterstützt. Gnome in Debian Sid ist als sehr instabil bekannt und kann zu Problemen führen. Es gibt aber sidux-Nutzer, die Gnome vewerden und bei Problemen im sidux-Wiki, sidux-Forum und IRC helfen können. Darüber hinaus müssen Probleme mit einem installierten Gnome selbst gelöst werden. Obwohl XFCE nicht unterstützt ist, finden sich wegen seiner Popularität auf älteren Computern höchstwahrscheinlich ebenso sidux-Nutzer, die Informationen via Wiki/Forum/IRC zur Verfügung stellen können.
