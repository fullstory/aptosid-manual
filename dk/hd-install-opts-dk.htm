Oprette et permanent hjemme mappe (en /home mappe) med Live-CD'en

Til almindelig brug anbefaler vi ext3 fil systemet. Det er standard filsystemet i sidux, og det bliver godt vedligeholdt.

Til brug sammen med sidux Live-CD'en kan du oprette en permanent $HOME mappe, hvor du kan gemme dine konfigurationsfiler, bruger data og yderligere installeret software.

Den mest anvendte måde at boot Live-CD'en (men ikke den eneste):
1) Fra CD/DVD-drevet, eller
2) Opstart ISO'en fra harddisken. "fromiso"

At vælge at opstarte fra harddisken (metode 2) er klart den hurtigste metode

Den permanente mappe ($HOME) kan placeres på enten din harddisk, eller på en usb nøgle.

Først skal vi have aktiveret unionfs, dette gør du ved at skrive koden ind i GRUB bootmenuens kommandolinie:

unionfs

Eller hvis systemet allerede er startet op, via konsollen/ terminalvinduet med:

$su
#fix-unionfs

Nu opstarter vi fra Live-CD'en eller fra fromiso ISO, og starter så scriptet der opretter konfigurationsfilen (config.tbz). Denne gemmes på enten harddisken eller usb nøglen

sidux-Menu - Configuration - save configuration

Så opretter vi image-filen til /home mappen:

sidux-Menu - Configuration - install permanent sidux home directory

Husk at den skal være stor nok til at indeholde alt det software du planlægger at installere i fremtiden. Opret nu en tom mappe der hedder unionfs i /home under brugeren sidux (/home/sidux), dvs. /home/sidux/unionfs. Du kan anvende konqueror, konsollen eller hvad du ellers vil bruge til at oprette mappen med.

Nu skal du genstarte med Live-CD'en eller fromiso-ISO'en (afhængigt af hvad du bruger), og anvend følgende boot-parametrer i Grub-Bootscreen:

myconfig=scan home=scan unionrw=/home/sidux/unionfs

Nu er du parat til at installere nyt software til din nye /home mappe på din harddisk/ usb nøgle.

Det gør du på den sædvanlige måde:

#sux
#apt-get update
#apt-get install pakkenavn

Efter næste Opstart, hvor du anvender cheatcodes myconfig=scan home=scan unionrw=/home/sidux/unionfs kan du så bruge dine nyinstallerede programmer.

Opstart med "fromiso"

Med denne cheatcode kan du vælge at starte systemet op fra en ISO-fil placeret på en af dine harddisk partitioner, det er meget hurtigere end at starte systemet fra CD (en harddisk installation med "fromiso" tager f. eks. kun et øjeblik).

'fromiso' er selvfølgelig hurtigere end at anvende CD/DVD-drevet, men derudover gør denne metode samtidigt CD/DVD-drevet ledigt. Som et alternativ kan du anvende QEMU.

Forudsætninger:

* en fungerende GRUB installation (på floppy, Harddisk eller Live-CD'en)
* et sidux ISO-Image e.g.: sidux.iso

Først skal vi vælge et sted at placere ISO-filen. For at få så korte filnavne som muligt vælger vi at placere denne 'basis-mappe' i for eksempel /dev/hda5. opret mappen "sidux" her og kopier ISO-filen dertil:

# mkdir /media/hda5/sidux
# mv sidux.ISO /media/hda5/sidux

Montér ISO-filen:

# mkdir -p /mnt/test
# mount -t iso9660 -o loop,ro /media/hda5/sidux/sidux.ISO /mnt/test

Nu kopierer vi filerne miniroot.gz og vmlinuz fra den monterede ISO-fil til vores mappe:

# cp /mnt/test/boot/vmlinuz /media/hda5/sidux/
# cp /mnt/test/boot/miniroot.gz /media/hda5/sidux/

Nu skal GRUB modificeres. Derfor redigerer vi filen /boot/grub/menu.lst og tilføjer følgende linier:

### ISO boot
title sidux 32bit from ISO
kernel (hd0,4)/sidux/vmlinuz ramdisk_size=100000 init=/etc/init fromiso=/sidux/*.ISO noprompt noeject lang=en apm=power-off nomce quiet vga=791
initrd (hd0,4)/sidux/miniroot.gz

Ved den næste opstart er der et nyt menupunkt i GRUB til at starte fra ISO-Image: "Sidux 32bit from ISO".

Installer sidux på en usb-nøgle/ usb-harddisk

At installere sidux på usb-nøgle eller usb-harddisk er lige så nemt som at installere til en normal harddisk. Følg denne guide:

Den anbefalede minimunsstørrelse for en usb-nøgle:
sidux "LITE" kræver ca. 1.2gig PLUS plads til ekstra data,
sidux "FULL" kræver ca. 2.1gig PLUS plads til ekstra data

USB-installationen (fix-usb-install.sh) er smeltet sammen med sidux-installer, så alt kan nu administreres fra denne.

Start som med en normal installation og vælg den partition på usb-enheden hvortil sidux skal installeres - vi kalder den "sda1" her, men det gælder for enhver anden partition på en usb-disk eller usb-enhed.

Grub skal installeres til partitionen (sda1), ikke MBR !

Partitionen skal formatteres med et linux-filsystem. Vi anbefaler ext3-filesystemet. En eksisterende partition vil blive reformatteret af installationsprogrammet!

Al data på denne partition vil blive tabt under reformatteringen!

du kan nu boote fra din usb-enhed. 'Boot fra USB' skal være aktiveret i din BIOS.

Installing sidux with QEMU

  • 1. create a harddiskimage for qemu
  • 2. boot the iso with qemu
  • 3. install on the image

Creating the hard disk image

To run qemu you will probably need a hard disk image. This is a file which stores the contents of the emulated hard disk.

Use the command:

qemu-img create -f qcow sidux.qcow 3G

To create the image file named "sidux.qcow". The "3G" parameter specifies the size of the disk - in this case 3 GB. You can use suffix M for megabytes (for example "256M"). You shouldn't worry too much about the size of the disk - the qcow format compresses the image so that the empty space doesn't add up to the size of the file.

Installing the operating system

This is the first time you will need to start the emulator. One thing to keep in mind: when you click inside qemu window, the mouse pointer is grabbed. To release it press Ctrl+Alt.

If you need to use a bootable floppy, run Qemu with:

qemu -floppy sidux.iso -net nic -net user -m 512 -boot d sidux.qcow

If your CD-ROM is bootable, run Qemu with:

qemu -cdrom sidux.iso -net nic -net user -m 512 -boot d sidux.qcow

Now install sidux as if you were going to install it on a real HD

Running the system

To run the system simply type:

qemu [hd_image]

A good idea is to use overlay images. This way you can create hard disk image once and tell Qemu to store changes in external file. You get rid of all the instability, because it is so easy to revert to previous system state.

To create an overlay image, type:

qemu-img create -b [[base''image]] -f qcow [[overlay''image]]

Substitute the hard disk image for base_image (in our case win.qcow). After that you can run qemu with:

qemu [overlay_image]

The original image will be left untouched. One hitch, the base image cannot be renamed or moved, the overlay remembers the base's full path.

Using any real partition as the single primary partition of a hard disk image

Sometimes, you may wish to use one of your system partition from within qemu (for instance, if you wish booting both your real machine or qemu using a given partition as root). You can do this using software RAID in linear mode (you need the linear.ko kernel driver) and a loopback device: the trick is to dynamically prepend a master boot record (MBR) to the real partition you wish to embed in a qemu raw disk image.

Suppose you have a plain, unmounted /dev/hdaN partition with some filesystem on it you wish to make part of a qemu disk image. First, you create some small file to hold the MBR:

dd if=/dev/zero of=/path/to/mbr count=32

Here, a 16 KB (32 * 512 bytes) file is created. It is important not to make it too small (even if the MBR only needs a single 512 bytes block), since the smaller it will be, the smaller the chunk size of the software RAID device will have to be, which could have an impact on performance. Then, you setup a loopback device to the MBR file:

losetup -f /path/to/mbr

Let's assume the resulting device is /dev/loop0, because we wouldn't already have been using other loopbacks. Next step is to create the "merged" MBR + /dev/hdaN disk image using software RAID:

 modprobe linear
 mdadm --build --verbose /dev/md0 --chunk=16 --level=linear --raid-devices=2 /dev/loop0 /dev/hdaN

The resulting /dev/md0 is what you will use as a qemu raw disk image (don't forget to set the permissions so that the emulator can access it). The last (and somewhat tricky) step is to set the disk configuration (disk geometry and partitions table) so that the primary partition start point in the MBR matches the one of /dev/hdaN inside /dev/md0 (an offset of exactly 16 * 512 = 16384 bytes in this example). Do this using fdisk on the host machine, not in the emulator: the default raw disc detection routine from qemu often results in non kilobyte-roundable offsets (such as 31.5 KB, as in the previous section) that cannot be managed by the software RAID code. Hence, from the the host:

 fdisk /dev/md0

There, create a single primary partition corresponding to /dev/hdaN, and play with the 's'ector command from the 'x'pert menu until the first cylinder (where the first partition starts), matches to the size of the MBR. Finally, 'w'rite the result to the file: you are done. You know have a partition you can mount directly from your host, as well as part of a qemu disk image:

 qemu -hdc /dev/md0 [...]

You can of course safely set any bootloader on this disk image using qemu, provided the original /boot/hdaN partition contains the necessary tools.

Using the QEMU Accelerator Module

The developers of qemu have created an optional kernel module to accelerate qemu to sometimes near native levels. This should be loaded with the option

major=0

to automate the creation of the required /dev/kqemu device. The following command

echo "options kqemu major=0" >> /etc/modprobe.conf

This will amend modprobe.conf to ensure that the module option is added every time the module is loaded.

qemu [...] -kernel-kqemu

This enables full virtualization and thus improves speed considerably.

To activate qemu:

qemu -cdrom /tmp/pkg/sidux-debug.iso -net nic -net user -m 512
