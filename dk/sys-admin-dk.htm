Opsætning af tjenester i Sidux

Sidux starter som standard følgende tjenester: cpid, cron, dbus, hotkey-setup, kdm, klogd, sysklogd og de bliver opdaterede hver gang en dist-upgrade udføres. Værsærligt opmærksom på at alle installerede men ikke aktiverede tjenester vil blive sat som aktive efter en dist-upgrade.

For at starte en tjeneste for denne session kun, brug:

/etc/init.d/service start

eller

rcconf

for at starte tjenesten op ved hver opstart

Anvendelse af rcconf: Brug pile til at navigere op/ ned, mellemrum til at vælge/ afvælge eselect, tabulatoren til at markere, ok eller cancel, `enter` for at bekræfte og udføre ændringer

Mistet root passwords

Du kan ikke genfinde et mistet root password, men du kan oprette et nyt.

Til at starte med booter du Live-CD'en.

Som root monterer du din root partition (f.eks. /dev/hdb2)

# mount /dev/hdb2
Now chroot into your old root-Partition with
# chroot /media/hdb2
and use the passwd command to set a new password:
# passwd

Screen Resolutions, Monitors and Fonts

Changing the Screen Resolution

In a konsole as root

change-res 1024x768
or  1600x1200 1400x1050 1280x1024 1152x864 800x600

fix-res 1024x768
or 1600x1200 1400x1050 1280x1024 1152x864 800x600

Restart X by doing 

ctrl+alt+backspace

Monitors

You are about to dive into xorg config files, please, please, please make a back-up of xorg.conf, before going any further!

All this may change when xorg 7.3 appears in late 2007.

ATI Single monitor changeover

Single head :    aticonfig --initial --input=/etc/X11/xorg.conf

Dual monitors

native ati

http://sidux.com/index.php?name=PNphpBB ... 9794#19794 has some working xorg.confs with the free radeon driver.

non-free ati

First install the non-free driver (fglrx)

get-sidux-gfx
or
sgfxi / du-fixes-h2.sh

Please refer to Installing 3D drivers for full instructions

Next: You must get out of X

ctrl+alt+F1
type: root
enter your password:
init 3

Then:

aticonfig --dtop=horizontal
init 5 && exit

If your monitors end up in the wrong place, you can do

aticonfig --dtop=horizontal,reverse

Please read the help which is very comprehensive with options and examples:

aticonfig --help

Note: Both monitors have to have the same resolution. So check in the manual of the monitors and MATCH the highest resolution of both.

http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=20231#20231 may be able to provide additional information.

nvidia

For nvidia non-free simply use the nvidia xorg configurator http://www.sorgonet.com/linux/nv-online/ and alter your xorg files accordingly.

Fonts in sidux

Correct dpi settings - Basic Philosophy

DPI settings are problematic to guess, but are actually perfectly done by X, if DisplaySize provides physical screen dimensions in /etc/X11/xorg.conf "Monitor Section".

Correct resolutions and refresh rates

Every monitor has it's own perfect settings combination, but unfortunately not all of them report the right DCC values, and sometimes it is in need of a manual overwride.

Correct graphics adapter drivers

Some newer ATI and Nvidia cards simply don't play well with the free Xorg drivers, and the only reasonable solution in such case are the commercial closed source drivers. sidux will not pre-install those for legal reasons, The solutions can be found here.

Default Font selections, rendering and sizes

sidux uses (Debian) pre-selected free fonts which have proved to be very balanced. If you fiddle with conflicting dpi-settings in X and Firefox/Iceweasel, and then wonder why it looks bad, well....... enough said.

Please be aware that every font has just a few real perfect sizes, other sizes may play well.

Firefox/Iceweasel pre-settings

GTK-based apps in general are problematic with kde defaults. this can be solved by this,

apt-get install gtk2-engines gtk2-engines-gtk-qt

In the control center you will have a new gtk menu item, set it to use clearlooks and have the fonts set to use kde fonts.

Then remove gtk2-engines-gtk-qt

apt-get remove --purge gtk2-engines-gtk-qt

This MAY fix your font rendering in gtk apps
