Εκκίνηση με "fromiso" - Γενικά

Για την κανονική χρήση συστήνουμε το ext4 σύστημα αρχείων. Είναι το σύστημα αρχείων προεπιλογής για το sidux.

Με αυτό το cheatcode μπορείτε να αρχίσετε από το ISO το οποίο βρίσκεται πάνω σε ένα σκληρό δίσκο (ext2/3/4), το οποίο είναι πολύ γρηγορότερο από το CD (οι εγκαταστάσεις στον σκληρό δίσκο με το πρόγραμμα "fromiso" χρειάζονται ένα κλάσμα του χρόνου που χρειάζεται η εγκατάσταση με CD).

Το "fromiso" είναι φυσικά πολύ γρηγορότερο από το CD/dvd-drive και κρατά το drive διαθέσιμο. Σαν εναλλακτική λύση θα μπορούσατε να χρησιμοποιήσετε QEMU.

Απαιτήσεις:

* ένα grub που να λειτουργεί(σε μια δισκέτα, σε ένα HD-Installation ή σε Live-CD) * μια ISO εικόνα (image) sidux π.χ.: sidux.iso και ένα σύστημα αρχείων linux όπως ext2/3/4
*Εάν μετονομάσετε την εικόνα σας σε sidux.ISO θα πρέπει να αλλάξετε τις εντολές ανάλογα, δεδομένου ότι το Unix διακρίνει μικρά και κεφαλαία γράμματα.

fromiso με Grub2

Το sidux παρέχει ένα grub2 αρχείο με όνομα 60_fll-fromiso για την δημιουργία μιας fromiso καταχώρησης στο grub2 μενού. Το αρχείο για ρύθμιση του fromiso ονομάζεται grub2-fll-fromiso και βρίσκεται στο /etc/default/grub2-fll-fromiso..

Πρώτα ανοίξτε ένα τερματικό και γίνετε root με:

sux
apt-get update
apt-get install grub2-fll-fromiso

Μετά ανοίξτε ένα επεξεργαστή κειμένου, ο οποίος μπορεί να είναι το kwrite, mcedit, vim ή κάποιο άλλο που προτιμάτε:

mcedit /etc/default/grub2-fll-fromiso

Στη συνέχεια αφαιρέστε το # από τις γραμμές που χρειάζεστε και αντικαταστήστε τις προεπιλεγμένες οδηγίες μέσα στα "εισαγωγικά" με τις προτιμήσεις σας.

Για παράδειγμα συγκρίνετε αυτό το τροποπιημένο grub2-fll-fromiso με το προεπιλεγμένο, (οι μαυρισμένες γραμμές είναι οι αλλαγμένες γραμμές):

# Defaults for grub2-fll-fromiso update-grub helper
# sourced by grub2's update-grub
# installed at /etc/default/grub2-fll-fromiso by the maintainer scripts

#
# This is a POSIX shell fragment
#

# specify where to look for the ISO
# default: /srv/ISO ### Note: This is the path to the directory that contains the ISO,
 it is not to include the actual sidux-*.iso file.###
FLL_GRUB2_ISO_LOCATION="/media/disk1part4/sidux-iso"

# array for defining ISO prefices --> sidux-*.iso
# default: "sidux- fullstory-"
FLL_GRUB2_ISO_PREFIX="sidux-"

# set default language
# default: en_US
FLL_GRUB2_LANG="en_AU"

# override the default timezone.
# default: UTC
FLL_GRUB2_TZ="Australia/Melbourne"

# kernel framebuffer resolution, see
# http://manual.sidux.com/de/cheatcodes-vga-de.htm#vga
# default: 791
#FLL_GRUB2_VGA="791"

# additional cheatcodes
# default: noeject
FLL_GRUB2_CHEATCODE="noeject nointro"

Φυλάξτε το αρχείο και κλείστε τον επεξεργαστή κειμένου και τρέξτε στο τερματικό

update-grub

Το grub2 grub.cfg θα ενημερωθεί για να δείχνει τα διάφορα ISOs που τοποθετήσατε μέσα στον κατάλογο που έχετε επιλέξει και θα είναι εκεί την επόμενη φορά που θα εκκινήσετε τον υπολογιστή.

fromiso με grub-legacy

Διαδικασία:

Διαλέξτε ένα χώρο για το iso και 2 αρχεία που χρειάζονται ούτως ώστε να έχετε μικρά ονόματα (μικρό μονοπάτι). Δημιουργήστε έναν κατάλογο-βάση πχ "/media/sdaX", (sda1 ή 2 ή 5 ή 6 ή οτιδήποτε), δημιουργήστε τον κατάλογο "sidux" και βάλτε το iso μέσα:

$ sux
mkdir -p /media/sdaX/sidux
mv sidux.iso /media/sdaX/sidux

Τώρα προσαρτήστε (mount) το ISO (Εναλλακτικά μπορείτε να χρησιμοποιήσετε το mc [midnight commander] μιας και κάνει auto-mount .iso αρχεία και σας αφήνει να αντιγράψετε τα αναγκαία αρχεία που βρίσκονται μέσα στο iso):

mount -t iso9660 -o loop,ro /media/sdaX/sidux/sidux.iso /mnt

Τώρα αντιγράψτε το vmlinuz και initrd.img αρχεία από το προσαρτημένο iso στον κατάλογο. Ο σκληρός δίσκος είναι δική σας μεταβλητή::Αυτό είναι μόνο ένα παράδειγμα:

cp /mnt/boot/vmlinuz0.686 /media/sda5/sidux/vmlinuz0.686
cp /mnt/boot/initrd0.686 /media/sda5/sidux/initrd0.686
umount /mnt

Τώρα πρέπει να ρυθμίσετε τον grub. Επεξεργαστείτε το αρχείο /boot/grub/menu.lst και προσθέστε τις παρακάτω γραμμές ΜΕΤΑ το ### END DEBIAN AUTOMAGIC KERNELS LIST: ΣΗΜΕΙΩΣΤΕ ότι στον grub δεν επιτρέπετε η χρήση μπαλαντέρ (wildcard) για ονόματα αρχείων! Ο σκληρός δίσκος είναι δική σας μεταβλητή::Αυτό είναι μόνο ένα παράδειγμα: 

### ISO boot
title sidux 32bit from ISO
kernel (hd0,4)/sidux/vmlinuz0.686 boot=fll quiet vga=791 fromiso=/sidux/sidux.iso
initrd (hd0,4)/sidux/initrd0.686

Με την επόμενη εκκίνηση έχετε ένα νέο αντικείμενο στο μενού του grub για να εκκινήσετε το iso. Σε περίπτωση που αναβαθμίσετε το sidux.iso, vmlinuz και intird.img αρχεία σας με μια μετέπειτα έκδοση, το grub menu.lst δεν χρειάζεται να αλλαχθεί.

Γενικές πληροφορίες για το fromiso (από το ISO) με persist

fromiso και persist σε ένα σκληρό δίσκο

Μπορείτε να έχετε ένα μόνιμο live σύστημα σε ένα εγγράψιμο δίσκο με το να συνδιάσετε ένα fromiso setup μαζί με το persist cheatcode. Όταν το fromiso χρησιμοποιεί ext2/ext3/ext4 σύστημα, η προεπιλογή είναι απλά:

persist

Όταν το fromiso χρησιμοποιεί ένα vfat σύστημα αρχείων, τότε πρέπει να χρησιμοποιήσετε ένα αρχείο το οποίο περιέχει ένα linux σύστημα αρχείων και το cheatcode θα έχει αυτή την μορφή:

persist=/sidux/sidux-rw

Το sidux χρησιμοποιεί το aufs για να ενεργοποιήσει αυτό που είναι γνωστό σαν "copy on write" πάνω στο ISO σας για να σας επιτρέψει να γράψετε νέα αρχεία και φακέλους και να ενημερώσετε υπάρχοντες φακέλους/αρχεία, κρατώντας τα νέα αρχεία στην μνήμη. Το persist cheatcode, θα φυλάξει τα νέα σας αρχεία πάνω στην ίδια κατάτμηση του σκληρού δίσκου στην οποία είναι φυλαγμένο το fromiso ISO image σας.

Το fromiso σας δίνει ένα ζωντανό (live) σύστημα το οποίο εκτελεί όλες τις αυτόματες λειτουργίες του sidux live ISO. Αυτό έχει σαν θετικό την αυτόματη ρύθμιση του υλικού (hardware) και σημαίνει επίσης ότι κάθε φορά που θα ξεκινάτε το σύστημα θα δημιουργούνται τα ίδια αρχεία εκτός και αν χρησιμοποιείτε επιπρόσθετα cheatcodes.

Η χρήση του persist μαζί με άλλα sidux-συγκεκριμένα cheatcodes όπως το noxorgconf, nonetwork, σημαίνει ότι δεν θα ξανά δημιουργούνται τα ίδια αρχεία κάθε φορά που θα ξεκινάτε.

Η χρήση του persist σημαίνει επίσης ότι μπορείτε να εγκαταστήσετε πακέτα με το apt και να έχετε την επόμενη φορά που θα εκκινήσετε πρόσβαση στην εφαρμογή και οποιαδήποτε αρχεία που σώσατε.

fromiso και persist σε ένα εκκινήσιμο USB-stick/SSD-κάρτα

Ίσως ο ιδανικός τρόπος χρήσης της μονιμότητας (persistence) είναι σε συνδυασμό με το εργαλείο install-usb-gui για να δημιουργήσετε το δικό σας εκκινήσιμο flash drive με τα αρχεία και τα προγράμματα που χρειάζεστε. Τα αρχεία σας θα φυλαχτούν σε ένα υπό-φάκελο πάνω στον οδηγό σας.

Για persist πάνω σε ένα FAT σύστημα αρχείων, όπως χρησιμοποιείται συνήθως για dos/Windows 9x εγκαταστάσεις και βρίσκεται συνήθως σαν προεπιλογή πάνω σε flash συσκευές, θα πρέπει να δημιουργήσετε ένα μεγάλο αρχείο για να το χρησιμοποιήσετε σαν loop device και να διαμορφώσετε (format) το αρχείο αυτό.

Για USB-sticks/SSD-κάρτες, ext2 και vfat είναι τα συνιστώμενα συστήματα αρχείων και είναι πιθανών να παρέχουν καλύτερη cross platform (διάφορες πλατφόρμες) συμβατότητα σε περίπτωση που χρειαστεί να σώσετε κάποια δεδομένα μιας και υπάρχει ένας MS Windows™ οδηγός για data-swapping (ανταλλαγή δεδομένων). Ο αριθμός επανεγγραφών σε flash συσκευές εξαρτάται από τις προδιαγραφές του USB-stick/SSD-κάρτας

ext2 συστήματα αρχείων

Με ext2 θα χρησιμοποιηθεί ολόκληρη η κατάτμηση, θα χρησιμοποιηθεί ο τρέχων root και θα δημιουργηθεί ένας κατάλογος /fll ο οποίος θα χρησιμοποιηθεί για το persist επιτρέποντας την χρήση όλου του ελεύθερου χώρου της συσκευής για το μόνιμο (persist) σύστημα.

vfat σύστημα αρχείων

Όταν χρησιμοποιείται το vfat, η μονιμότητα επιτυγχάνεται μέσω ενός αρχείου το οποίο μπορεί να έχει μέγιστο μέγεθος 2GB αλλά να μην είναι μικρότερο από 100MB (θα ήταν άχρηστο). Το αρχείο αυτό θα πρέπει να ονομαστεί sidux-rw

Example of creating persist after initial installation

If you are not sure of the mount point, mount the stick and run ls -lh /media to provide a list of all your system mount points. Look for something like drwxr-xr-x 6 username root 4.0K Jan 1 1970 disk. If your output states differently then replace "/media/disk" in line with your requirement, (for example "/media/sdc1"):

disk="/media/disk"

Set the size of the persist partition:

size=1024

Make a directory on the stick:

mkdir $disk/sidux

Run the code to make the persist partition:

dd if=/dev/zero of=$disk/sidux/sidux-rw bs=1M count=$size &&  echo 'y' | LANG=C /sbin/mkfs.ext2 $disk/sidux/sidux-rw &&  tune2fs -c 0 "$disk/sidux/sidux-rw"

NTFS κατατμήσεις, χρησιμοποιούνται συνήθως για Windows NT/2000/XP (TM) εγκαταστάσεις, ΔΕΝ ΜΠΟΡΟΥΝ να χρησιμοποιηθούν για μονιμότητα.

Εγκατάσταση του sidux σε μια USB/SSD συσκευή

Η εγκατάσταση του sidux πάνω σε ένα USB-stick/SSD-κάρτα είναι το ίδιο εύκολη με την εγκατάσταση σε ένα κανονικό σκληρό δίσκο. Απλά ακολουθήστε αυτές τις απλές οδηγίες.

3 τρόποι εγκατάστασης σε USB/SSD
  • 1) fromiso ; μόνο για sidux: sidux-on-a-stick
  • 2) Πλήρης (η πλήρης εγκατάσταση σε USB/SSD συμπεριφέρεται σαν κανονική εγκατάσταση στον σκληρό δίσκο και γίνεται μέσω του συνηθισμένου εγκαταστάτη).
  • 3) Γράψιμο στην συσκευή . Αυτή είναι η ιδανική λύση όταν έχετε οποιοδήποτε Linux, MS Windows ή Mac OS X λειτουργικό σύστημα και θέλετε να εγκαταστήσετε operating system sidux-on-a-stick.

USB fromiso εγκατάσταση, sidux-on-a-stick

Διαμορφώστε την συσκευή σας με ext2 ή fat32 πριν προχωρήσετε.

If using a GUI formatting application like gparted, please ensure that you first delete the existing partition, then recreate the partition before formatting.

Εάν το 'install-sidux-usb-gui' δεν είναι ήδη εγκατεστημένο στο σύστημα σας, εγκαταστήστε το με 'apt-get install fll-iso2usb'.

USB fromiso from a HD sidux installed system:

Η εγκατάσταση fromiso USB γίνεται, σε ένα εγκατεστημένο σύστημα, μέσω του Kmenu>System>install-sidux-usb-gui

USB fromiso from a sidux-*.iso:

ή με το να κάνετε κλικ πάνω στο sidux Installer Icon (εικονίδιο του sidux εγκαταστάτη) πάνω στο ISO και να επιλέξετε Install to USB (εγκατάσταση σε USB). Η 'fromiso' εγκατάσταση γίνεται στην πρώτη κατάτμηση του USB stick.

Σας δίνεται η ευκαιρία να επιλέξετε cheatcodes για την γλώσσα, ώρα κ.α και μέσω ενός checkbox (κουτάκι να βάλετε μέσα ένα x) να ενεργοποιήσετε το persist.

Τώρα έχετε ένα εκκινήσιμο USB/SSD. Εάν δεν έχετε ενεργοποιήσει το persist, μπορείτε να το ενεργοποιήσετε με το να προσθέσετε persist στην γραμμή του grub στην οθόνη του grub. (Εάν είναι vfat, είναι μάλλον καλύτερο να αρχίσετε από την αρχή).

Παράδειγμα στο τερματικό:
 fll-iso2usb -D /dev/sdb -f none --iso /home/sidux/sidux.iso -p -- lang=no tz=Pacific/Auckland

Αυτό εγκαθιστά την iso εικόνα (image) πάνω στην USB συσκευή sdb με persist, Νορβηγική γλώσσα και Pacific/Auckland (NZL) ώρα στην προεπιλεγμένη γραμμή του grub.

Οι X (κάρτα γραφικών, πληκτρολόγιο, ποντίκι) ρυθμίσεις σας ή το αρχείο των δικτυακών interfaces (network interfaces file) δεν έχουν σωθεί, πράγμα που το κάνει ιδανικό για χρήση σε άλλους υπολογιστές.

Για περισσότερη τεκμηρίωση συμπεριλαμβανομένων και επιπλέον επιλογών δείτε:

$ man fll-iso2usb

Πλήρης εγκατάσταση σε ένα USB/SSD (Συμπεριφέρεται σαν κανονική εγκατάσταση στον σκληρό)

Το μέγεθος που συνιστάτε ως μικρότερο μέγεθος χωρητικότητας ενός USB-stick/SSD-κάρτας είναι:
για το sidux "LITE" χρειάζονται 2.5gig ΣΥΝ ο αποθηκευτικός χώρος για τις ανάγκες σας
για το sidux "FULL" χρειάζονται 4gig ΣΥΝ ο αποθηκευτικός χώρος για τις ανάγκες σας

Διαμορφώστε την συσκευή σας με ext2 και δημιουργήστε κατατμήσεις στο USB-stick/SSD-κάρτα όπως θα κάνατε και με ένα συνηθισμένο PC.

Ξεκινήστε την εγκατάσταση από το Live-ISO και επιλέξτε την κατάτμηση πάνω στην USB/SSD συσκευή στην οποία θέλετε να εγκαταστήσετε το sidux, για παράδειγμα sdbX και ακολουθήστε τους διαλόγους του εγκαταστάτη του sidux. Διαβάστε το Εγκατάσταση στον σκληρό δίσκο

To boot from your USB/SSD 'Boot from USB' must be enabled in your BIOS.

Άλλα σημεία τα οποία αξίζει να προσέξετε είναι:

  • Μια εγκατάσταση σε ένα USB-stick/SSD-κάρτα συνήθως θα εξαρτάται από τον υπολογιστή στον οποίο έγινε η αρχική εγκατάσταση. Αν ο σκοπός σας είναι η χρήση του USB/SSD stick και σε άλλους υπολογιστές, δεν θα πρέπει να έχει προ-ρυθμισμένα μη-ελεύθερους οδηγούς κάρτας γραφικών και cheatcodes με εξαίρεση ίσως το vesa cheatcode μέσα στο menu.lst, (αυτά θα τα τακτοποιήσετε μετά από μια επιτυχημένη εγκατάσταση).
  • Αφού εκκινήσετε το USB-stick/SSD-κάρτα σε ένα άλλο υπολογιστή, θα πρέπει να επεξεργαστείτε το fstab για να έχετε πρόσβαση στους σκληρούς δίσκους του υπολογιστή.
  • "fromiso" με "persist" είναι μια καλύτερη επιλογή σε περίπτωση που έχετε σκοπό να χρησιμοποιήσετε την συσκευή και σε άλλους υπολογιστές.

Πλήρης εγκατάσταση πάνω σε ένα USB Σκληρό Δισκό όπως μια εγκατάσταση σε μια κατάτμηση

Ένας USB Σκληρός Δίσκος είναι χρήσιμος (κυρίως σε νέους χρήστες που έρχονται από την MS ή μια άλλη διανομή) γιατί μπορείτε να εγκαταστήσετε το sidux στον δίσκο και να τον συνδέσετε χωρίς να χρειαστεί να ρυθμίσετε τον υπολογιστή για dual boot (δημιουργία κατατμήσεων, αλλαγές στον grub κτλ.).

Ξεκινήστε την εγκατάσταση από το Live-ISO, (ή από ένα USB-stick/SSD-κάρτα), σαν μια κανονική εγκατάσταση, όχι εγκατάσταση σε USB και επιλέξτε την κατάτμηση στην οποία θέλετε να εγκαταστήσετε το sidux, για παράδειγμα sdbX και ακολουθείστε τους διαλόγους του εγκαταστάτη του sidux. Ο Grub πρέπει να γραφτεί στην κατάτμηση του USB δίσκου.
