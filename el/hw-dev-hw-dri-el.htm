Συμβουλές για υλικό με μη-ελεύθερες (non-free) ανάγκες

A aptosid live-ISO will only provide dfsg-free software.

Λίστα πηγών (sources list)

Επειδή το aptosid iso περιέχει μόνο dfsg ελεύθερο λογισμικό, εσείς πιθανών να θέλετε να προσθέσετε contrib non-free (μή ελεύθερο λογισμικό) στο /etc/apt/sources.list.d/debian.list και /etc/apt/sources.list.d/aptosid.list . Σιγουρευτείτε ότι είστε συνδεδεμένοι με το διαδίκτυο. Οι γραμμές σας θα πρέπει τώρα να είναι κάπως έτσι, αναλόγως του διακομιστή που επιλέξατε:

# Debian
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# aptosid
deb http://aptosid.com/debian/ sid main fix.main
#deb-src http://aptosid.com/debian/ sid main fix.main

Αφού αλλάξετε τις πηγές σας θα πρέπει να κάνετε apt-get update πριν αναζητήσετε ή εγκαταστήσετε νέα πακέτα, ( είναι καλό να κάνετε apt-get update πριν εγκαταστήσετε οποιοδήποτε πακέτο).

Λίστα με τους διακομιστές του Debian και την κατάσταση τους.

Ανοικτού κώδικα Xorg οδηγοί για nVidia, ATI/AMD, & Intel

Οι ανοικτού κώδικα Xorg οδηγοί για nVidia, ATI, Radeon, Intel, είναι προ-εγκατεστημένοι στο aptosid.

Οι Intel οδηγοί θα πρέπει να δουλεύουν τέλεια για 2D και 3D video επιτάχυνση μιας και οι οδηγοί συμπεριλαμβάνονται μέσα στην Intel free series.

Σημείωση: xorg.conf is not generally needed for open source drivers.

If you have been running proprietary drivers and wish to revert back to opensource drivers edit /etc/X11/xorg.conf.d/xx-xxxx.conf with an editor with root permissions. Find the section that reads SECTION DEVICE and change the display driver to read nv OR radeon OR intel (to name but a few).

Επεξεργαστείτε το an xorg.conf file με δική σας ευθύνη.

Περισσότερες πληροφορίες για την Intel   ATI/AMD   nv  X.Org

Binary, κλειστού κώδικα οδηγοί για : nVidia με dmakms & xorg.conf.d

For the most complete and accurate listing of supported GPUs please see the Supported Products List available from the NVIDIA Linux Graphics Driver download page.

Μπορείτε επίσης να διαβάσετε nvnews για άλλες επιλογές.

New and old installs will need to ensure that the systemwide config /etc/X11/xorg.conf.d exists and add a file to the directory called 20-nvidia.conf : 

mkdir /etc/X11/xorg.conf.d
touch /etc/X11/xorg.conf.d/20-nvidia.conf

With your favourite text editor, (e.g. kwrite, kate, mousepad, mcedit,vi, vim) open the file:

<editor> /etc/X11/xorg.conf.d/20-nvidia.conf

and add the following codebox in its entirety to 20-nvidia.conf:

#
Section "Device"
         Identifier      "Device 0"
         Driver          "nvidia"
EndSection
# This is a trailing line, it is needed so that End Section is not the last line

Εάν έχετε περισσότερες από μια κάρτες, θα πρέπει να βρείτε το PCI και να το συμπεριλάβεται στο 20-nvidia.conf:

lspci | grep -i vga

Η εντολή θα πρέπει να επιστρέψει κάτι παρόμιο με το πιο κάτω:

01:00.0 VGA compatible controller:

Προσθέστε το 01:00.0 busid σε έξτρα γραμμή κάτω από την γραμμή 'Driver', προσέξτε όμως ότι η σύνταξη είναι PCI:x:y:z: με τα μηδενικά να αφαιρούνται και με ανώ και κάτω τελείες να προσθέτονται, επομένως:

BusID		 "PCI:1:0:0"

Installing the nvidia drivers

ΣΗΜΕΙΩΣΗ: Χρησιμοποιείστε apt-cache search nvidia και apt-cache show <πακέτο> για να βρείτε τον σωστό οδηγό. There are basically 2 types of nvidia drivers, the current Debian Sid 3D drivers and the legacy Debian Sid 3D drivers.

Για τωρινούς 3d nvidia οδηγούς ≥ GeForce 6xxx :

Προετοιμασία του module:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Μετά ενεργοποιήστε το Dynamic Module-Assistant Kernel Module Support (dmakms) για nvidia, έτσι ώστε την επόμενη φορά που θα ενημερωθεί ο πυρήνας του Linux, θα ετοιμαστεί ένα module για την nvidia χωρίς να χρειαστεί να κάνετε κάτι εσείς. Για να γίνει αυτό, προσθέστε nvidia-kernel-source στο /etc/default/dmakms αρχείο ρύθμισης:

echo nvidia-kernel-source >> /etc/default/dmakms

Μετά:

m-a a-i nvidia-kernel-source

Ακολουθούμενο από:

apt-get install nvidia-glx

Επανεκκινήστε τον υπολογιστή για να τελειώσει η εγκατάσταση του module.

Σε περίπτωση αναβάθμισης μόνο του xorg, χρειάζεται μόνο να εγκαταστήσετε το nvidia-glx:

apt-get install --reinstall nvidia-glx

Όταν διατείθετε ένας νέος Debian sid nvidia οδηγός μέσω του apt:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Επανεκκινήστε το PC για να τελειώσει η εγκατάσταση.

Naming scheme for legacy nvidia drivers in Debian

  • nvidia-kernel-legacy-71xx is for GeForce 2
  • nvidia-kernel-legacy-96xx is for GeForce 4
  • nvidia-kernel-legacy-173xx is for GeForce 5
Example for legacy 3d nvidia drivers using ≤ GeForce 5xxx :

For other legacy drivers just replace 173xx number with your driver number.

m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Αλλάξτε το dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Όταν ενημερωθεί ο xorg χρειάζετε μόνο να εγκαταστήσετε ξανά nvidia-glx-legacy:

apt-get install --reinstall nvidia-glx-legacy-173xx

Αποτυχία φόρτωσης του module

Αν για οποιοδήποτε λόγο αποτύχει η φόρτωση του nvidia module:

modprobe nvidia

Μετά επανεκκινήστε τον υπολογιστή.

Εάν η φόρτωση του module εξακολουθεί να αποτυχαίνει:

 m-a a-i -f nvidia-kernel-source

ή

 m-a a-i -f nvidia-kernel-legacy-173xx-source

Αυτό ξανά κτίζει το module, μετά επανεκκινήστε.

Διαβάστε:

$ /usr/share/doc/dmakms

ATI/AMD, Radeon και άλλοι οδηγοί video

2D Οδηγοί

Οι οδηγοί για τον X. Org X διακομιστή (server) (δείτε xserver-xorg για περαιτέρω περιγραφή) εξασφαλίζουν υποστήριξη σε 2D για NVIDIA Riva, TNT, GeForce και Quadro κάρτες και των ATI Mach, Rage, Radeon και FireGL cards παράλληλα με atimisc r128, r6xx/r7xx και radeon υπό-οδηγούς. Οι Radeon και οι Intel υποστηρίζουν και οι δυο 2d επιτάχυνση (textured xv) για αναπαραγωγή video.

3D Οδηγοί

Μερικές ATI/AMD κάρτες υποστηρίζουν 3D, (και KDE animations), με xserver-xorg-video-radeon. Αυτή τη στιγμή υποστηρίζονται chipsets μέχρι το r700.

Για αυτόματη εγκατάσταση μη-ελεύθερων firmware όταν ενημερώνονται. Για 2D και 3D κάρτες γραφικών:

apt-get install firmware-linux

Μετά επανεκκινήστε τον υπολογιστή.

Firmware οδηγοί

Το firmware που χρειάζεται μπορεί να καθοριστεί από την πιο κάτω λίστα ή με την χρήση του fw-detect script (πακεταρισμένο μέσα στο aptosid-scripts).

$ fw-detect

Η έξοδος του fw-detect περιγράφη τις εντολές που χρειάζονται για εγκατάσταση και ενεργοποίηση του firmware:

Παράδειγμα:

#Detected driver that requires firmware to operate
#Follow these instructions to obtain the correct firmware
# and activate the zd1211rw driver:
apt-get update
apt-get install zd1211-firmware
modprobe -r zd1211rw
modprobe zd1211rw

Για να εγκαταστήσετε firmware από git repos, θα πρέπει να:

apt-get install git-core
μη-ελεύθερα firmware debs πάνω σε ένα stick

Σε περίπτωση που χρειάζεστε να κατεβάσετε firmware .debs, για να τα βάλετε πχ σε ένα usb-key για να τα μεταφέρετε σε ένα άλλο υπολογιστή, μπορείτε να τα κατεβάσετε σε zip ή tar.gz από http://cdimage.debian.org/cdimage/unofficial/non-free/firmware/sid/current/ και να τα αποσυμπιέσετε σε ένα φάκελο με όνομα firmware, θα πρέπει να δείτε ένα αριθμό .deb αρχείων. Μετά κατεβάστε το firmware-linux-free*.deb και προσθέστε το στο φάκελο με τα υπόλοιπα πακέτα που κατεβάσατε. Έπειτα μεταφέρτε τα με το stick στο υπολογιστή που χρειάζετε μη-ελεύθερο firmware και:

dpkg -i firmware-linux-nonfree*.deb

Δοκιμάζουμε να εξασφαλίσουμε πακέτα για νομική επαναδιανομή του firmware από δικές μας μη-ελεύθερες πηγές, αλλά δεν το επιτρέπουν όλοι οι προμηθευτές αυτό.

Γράφοντας στην NTFS-κατάτμηση με ntfs-3g

Προειδοποίηση: Αν και το ntfs-3g έχει δηλωθεί να είναι ‘σταθερό’ ποτέ μην το χρησιμοποιήσετε χωρίς να έχετε σώσει τα αρχεία σας σε εξωτερικό μέσο, και φυσικά ποτέ συστήματα παραγωγής! Αν το κάνετε, θα είναι δικό σας σφάλμα αν χάσετε τα δεδομένα σας, για αυτό χρησιμοποιήστε το με δικιά σας ευθύνη

Ανοίξτε ένα τερματικό και γράψτε τις ακόλουθες εντολές: Βλέπετε Κάνοντας κατατμήσης στον σκληρό σας δίσκο – Ονομασίες δίσκων

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Για να βγείτε έξω από την κονσόλα γράψτε: exit

Τώρα η NTFS-κατάτμηση θα πρέπει να είναι προσαρτημένη (mounted) rw και θα πρέπει να μπορείτε να αποθηκεύσετε τα δεδομένα σας σε αυτή. Αλλά ξανά προ ειδοποιήστε Χρησιμοποιήστε το σε επείγουσες καταστάσεις, δεν συνιστάται για χρήση επί καθημερινής βάσεως.
