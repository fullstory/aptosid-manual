Ενεργοποίηση του Ndiswrapper

Το sidux έχει όλα τα πακέτα που χρειάζονται ήδη εγκατεστημένα: Το kernel module και τα ndiswrapper-utils.

apt-get update
apt-get install ndiswrapper-modules-2.6-sidux-i686 (ή ndiswrapper-modules-2.6-sidux-amd64)
Κάντε το μόνοι σας

Βρέστε ένα κατάλληλο οδηγό (driver) για την κάρτα σας και εγκαταστήστε τον με την εντολή:

ndiswrapper -i driver.inf

Μπορείτε να ελέγξετε αν η εγκατάσταση ήταν επιτυχής με το -l:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Είναι σημαντικό να δείτε την ένδειξη "driver installed" και "device present" στο αποτέλεσμα του -l. Αν δεν δείτε και τα δύο τότε κάτι έγινε λάθος (ελέγξτε το μήνυμα λάθους από το -i και/ή dmesg), γιατί αν το αποτέλεσμα είναι λάθος δεν θα μπορείτε να έχετε μία σύνδεση wlan.

Φορτώστε το ndiswrapper-module:

modprobe ndiswrapper

Για να ελέγξετε αν δημιουργήθηκε μία συσκευή wifi πληκτρολογήστε::

iwconfig

Για να ελέγξετε αν βρέθηκε το access-point πληκτρολογήστε:

iwlist wlan0 scan

Ύστερα μπορείτε να ρυθμίσετε την κάρτα πληκτρολογώντας:

ceni

Μετά φορτώστε το με το modprobe:

modprobe ndiswrapper

Δεν χρειάζεται να εγκαταστήσετε ξάνα τους windows οδηγούς μιας και το μέρος αυτό των ρυθμίσεων είναι ξεχωριστό από το kernel module.

Γράφοντας ndiswrapper στο /etc/modules το ndiswrapper θα φορτώνει κατά την εκκίνηση

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

Αυτό μπορείτε να το κάνετε και γραφικά με το κέντρο ελέγχου (control centre) siduxcc μέσα στο KDE-Start-Menu

Ndiswrapper για προχωριμένους

Το ndiswrapper είναι ένα module το οποίο επιτρέπει στο Linux να χρησιμοποιεί Windows WLAN οδηγούς. Αυτό είναι απαραίτητο γιατί μερικοί κατασκευαστές καρτών wlan δεν δίνουν ανοιχτού κώδικα ή δωρεαν οδηγούς, κάτι το οποίο καθιστά δύσκολη την χρήση της κάρτας στο Linux. Το ndiswrapper λύνει αυτό το πρόβλημα "περιτυλίγοντας" τον Windows οδηγό και μεταφράζοντας την έξοδο και την είσοδο από την κάρτα στο Linux.

Αναγνώριση του απαιτούμενου οδηγού

Πρώτα απ' όλα χρειάζεστε ένα κατάλληλο windows οδηγό για την WLAN κάρτα σας. Οι WLAN κάρτες έχουν ένα μεγάλο αριθμό διαφορετικών chip sets για ένα μοντέλο και ο μόνος τρόπος να αναγνωρίσετε την κάρτα σας με ακρίβεια είναι με την ταυτότητα (id) συσκευής (USBID/PCIID).

Για να πάρετε αυτή την 'ταυτότητα' ανοίξτε ένα τερματικό:

Για μια USB WLAN κάρτα
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

Όπως βλέπετε στο πιο πάνω παράδειγμα, το USBID για την D-Link Corp. [hex] DWL-122 κάρτα είναι 2001:3700.

Για μια (mini) PCI / PCMCIA WLAN κάρτα
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

Και αυτή η 'ταυτότητα' έχει τον ίδιο τρόπο γραφής κατασκευαστής:συσκευή, εδώ 168c:0024 για την Atheros κάρτα.

Τώρα αναζητήστε στο: ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list για μια καταχώρηση με αυτή την 'ταυτότητα'. Εάν είστε τυχερός θα βρείτε την καταχώρηση που ψάχνετε, μαζί με τον κατάλληλο οδηγό.

Χρειάζεστε τα αρχεία INF και SYS από αυτόν τον οδηγό. Στην καλύτερη περίπτωση έχετε ένα ZIP αρχείο το οποίο μπορείτε να αποσυμπιέσετε με το unzip. Εάν έχετε ένα αρχείο EXE τότε είναι αρχείο που αποσυμπιέζεται μόνο του (self-extracting) (χρησιμοποιήστε το wine για να το αποσυμπιέσετε) ή ένα CAB (αποσυμπιέστε με το cabextract). Ο ευκολότερος τρόπος τώρα είναι να φυλάξετε όλα τα αρχεία σε ένα νέο φάκελο μέσα στο home σας.

Ενεργοποίηση του Ndiswrapper και εγκατάσταση του οδηγού

Τώρα μπείτε μέσα στον φάκελο με τα αρχεία του windows οδηγού:

bla@blubb:~$cd /φάκελος/με/οδηγό
bla@blubb:~$su
root@blubb:/φάκελος/με/οδηγό# ndiswrapper -i <driver.inf>
installing net5416

Για να ελέγξετε εάν η εγκατάσταση ήταν επιτυχής, ελέγξτε το με -l:

root@blubb:/φάκελος/με/οδηγό# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Είναι σημαντικό να δείτε "driver installed" και "device present" στην έξοδο (output) του -l. Εάν δεν δείτε και τα δύο, αυτό σημαίνει ότι κάτι πήγε λάθος (ελέγξτε για μήνυμα λάθους από το -i ή/και dmesg) γιατί αν η έξοδος είναι λάθος εδώ, τα βήματα δεν θα δουλέψουν.

Τώρα φορτώστε το kernel module: 

root@blubb:/κάποιος/φάκελος# modprobe ndiswrapper

Εάν δεν σας δοθεί έξοδος (output) εδώ, τότε όλα πήγαν μια χαρά. Αν σας δοθεί έξοδος ελέγξτε το dmesg για περισσότερες πληροφορίες.

Τώρα το "iwconfig" θα πρέπει να δείχνει μια ασύρματη συσκευή:

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

Εάν θέλετε να φορτώνει το ndiswrappper στη εκκίνηση απλά γράψτε ndiswrapper στο /etc/modules, με το χέρι ή χρησιμοποιήστε: 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
Επανεγκατάσταση του Ndiswrapper μετά από ενημέρωση (update) του πυρήνα

Μετά από μια ενημέρωση του πυρήνα, πρέπει να επανεγκαταστήσετε το ndiswrapper kernel module. 

apt-get update
apt-get install ndiswrapper-modules-2.6-sidux-i686 (or ndiswrapper-modules-2.6-sidux-amd64)

Μετά την εγκατάσταση φορτώστε το με:

root@blubb:/some/folder# modprobe ndiswrapper

Δεν χρειάζεται να επανεγκαταστήσετε τους windows οδηγούς καθώς ο οδηγός είναι ξεχωριστός από το kernel module.
