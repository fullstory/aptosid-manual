wpa Roaming GUI

Το wpa-roaming είναι μια μέθοδος με την οποία μπορείτε να "δείτε" ασύρματα δίκτυα και να συνδεθείτε με αυτά χωρίς να χρειάζεστε γραφικό περιβάλλον. Αυτό είναι πολύ βολικό για χρήστες φορητών υπολογιστών.

Το wpasupplicant είναι ένα πρόγραμμα το οποίο δεν απευθύνετε μόνο σε ασύρματα δίκτυα με WPA κλειδιά αλλά σε όλα τα ασύρματα δίκτυα. Σε συνδυασμό με το "κλασικό" /etc/network/interfaces , είναι μια πολύ ισχυρή μέθοδος σύνδεσης σε δίκτυα, χωρίς καν να κάνετε κάτι. Μπορείτε να συνδεθείτε αυτόματα σε οποιοδήποτε ανοικτό δίκτυο ή οποιοδήποτε δίκτυο προστεθεί στο αρχείο ρύθμισης. Ακόμη πιο βολικό: εάν επιτρέψετε το hotplugging, η σωστή σύνδεση ρυθμίζεται αυτόματα μετά την σύνδεση (σε ένα περιβάλλον επιφάνειας εργασίας (desktop environment) ή σε ένα session χωρίς X).

Βασικά

Για να μπορέσετε να χρησιμοποιήσετε το wpa-roaming με το wpasupplicant σε ένα γραφικό περιβάλλον, πρέπει να έχετε 2 πακέτα εγκατεστημένα:
* wpasupplicant
* wpagui

apt-get update && apt-get install wpasupplicant wpagui

Το wpasupplicant είναι αναγκαίο, το wpagui είναι βολικό αν θέλουμε να έχουμε μια γραφική επιφάνεια εργασίας με την οποία μπορούμε να αλλάξουμε access point και να περιηγηθούμε στο τοπικό δίκτυο.

Χρησιμοποίηση του wpa-gui χωρίς ρυθμίσεις δικτύου

Σαν διαχειριστής (root) επεξεργαστείτε το /etc/network/interfaces και προσθέστε τις παρακάτω γραμμές. Το όνομα της συσκευής μπορεί να είναι διαφορετικό:

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf

Ξανά σαν διαχειριστής επεξεργαστείτε ένα "ωμό" (raw) /etc/wpa_supplicant/wpa_supplicant.conf:

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        key_mgmt=NONE
}

Το τελευταίο βήμα είναι να κάνετε το wpa-roaming να δουλεύει για όλους τους χρήστες που είναι μέλη της ομάδας netdev άλλα unreadable (αδιάβαστο) και unwritable (άγραφο) για όλους τους χρήστες εκτός του διαχειριστή (συνιστάται για λόγους ασφαλείας), αλλά μέλη της ομάδας netdev μπορούν ακόμη να προσθέσουν δίκτυα μέσω του wpa_gui:

chown root:netdev /etc/wpa_supplicant/wpa_supplicant.conf
chmod 600 /etc/wpa_supplicant/wpa_supplicant.conf

Τρέχοντας το

Προαπαιτούμενα:

* Σιγουρευτείτε ότι είστε μέλος της ομάδας netdev
*Ξανά ξεκινήστε το δίκτυο ή επανεκκινήστε τον υπολογιστή

Ο χρήστης που δημιουργήθηκε κατά την διάρκεια της εγκατάστασης του sidux, είναι ήδη μέλος της ομάδας netdev. Ένας νέος χρήστης μπορεί να προστεθεί στην ομάδα από τον διαχειριστή (root)
Έπειτα σαν διαχειριστής ξανά φορτώστε το wpasupplicant με:

wpa_action <interface> reload

Εάν είστε ευχαριστημένος και όλα δουλεύουν, μπορείτε να προσθέσετε τις ρυθμίσεις στο /etc/wpa_supplicant/wpa_supplicant.conf με το να επιλέξετε το File > Save Configuration.

Χρησιμοποίηση του wpa-gui με τις ρυθμίσεις δικτύου

Με τη βοήθεια του IDString και του Priority , μπορείτε να ρυθμίσετε σε πιο δίκτυο συνδέετε ο υπολογιστής κατά την διάρκεια της εκκίνησης. Η μεγαλύτερη προτεραιότητα (priority) είναι το 1000, μικρότερη το 0. Επίσης πρέπει να προσθέσετε το IDString στο /etc/network/interfaces .

Η σύνταξη για το /etc/network/interfaces.

Το πρώτο είναι για σύνδεση σε DHCP διακομιστές, το δεύτερο είναι αν έχετε σταθερή διεύθυνση IP. Για να προσαρμόσετε τις ρυθμίσεις σας:

# id_str="home_dhcp"
iface home_dhcp inet dhcp

# id_str="home_static"
iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0
        network 192.168.0.0

        broadcast 192.168.0.255
        gateway 192.168.0.1
Πρακτικά Παραδείγματα

Εάν θέλετε να συνδέεστε αυτόματα στο home (σπίτι) WLAN όταν είστε στο σπίτι, δώστε στο IDString "home" και προτεραιότητα (priority) "15". Εάν ταξιδεύετε και θέλετε ο φορητός υπολογιστής σας να συνδέετε σε οποιοδήποτε ελεύθερο, χωρίς password δίκτυο που είναι διαθέσιμο, δώστε του το IDString "stalk" και προτεραιότητα "1" (πολύ χαμηλή). Αλλά παρακαλώ ελέγχετε πάντα αν η σύνδεση σας είναι νόμιμη και αποσυνδεθείτε εάν είναι φανερό ότι η σύνδεση δεν προοριζόταν να είναι ελεύθερη.

Έτσι φαίνονται οι παράγραφοι μέσα στο /etc/network/interfaces για το από πάνω παράδειγμα:

# /etc/network/interfaces -- configuration file for ifup(8), ifdown(8)

# The loopback interface
# automatically added when upgrading
auto lo
iface lo inet loopback

allow-hotplug eth0
iface eth0 inet dhcp

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf

iface home inet dhcp
iface stalk inet dhcp

και έτσι μέσα στο /etc/wpa_supplicant/wpa_supplicant.conf (SSID και passwords είναι αλλαγμένα ή απλά εξηγούνται):

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        ssid="my_ssid"
        scan_ssid=1
        psk=123ABC  ##το password σε hexadecimal κώδικα!!
#       psk="password_in_ascii"   ##Προαιρετικά το password σε ascii
        key_mgmt=WPA-PSK
        pairwise=TKIP
        group=TKIP
        auth_alg=OPEN
        priority=15
        id_str="home"
}

network={
        ssid=""
        scan_ssid=1
        key_mgmt=NONE
        auth_alg=OPEN
        priority=1
        disabled=1   ## η επιλογή αυτή δεν είναι υποχρεωτική, δέστε την τεκμηρίωση /usr/share/doc/wpasupplicant
        id_str="stalk"
}

Αν θέλετε να συνδεθείτε σε WLANs με κωδικό π.χ. σε καφετέριες ή μπαρ, hot spots, χρησιμοποιήστε το wpa_gui και αποφασίστε αργότερα αν θέλετε να προσθέσετε το δίκτυο ή όχι.

WEP κρυπτογράφηση

Εάν θέλετε να προσθέσετε WEP κρυπτογραφημένα δίκτυα στο wpa_supplicant.conf μόνιμα, η σύνταξη είναι:

network={
	ssid="παράδειγμα ενός wep δικτύου"
	key_mgmt=NONE
	wep_key0="abcde"
	wep_key1=0102030405
	wep_tx_keyidx=0
Σημειώσεις
1. Εύκολο να ξανά χρησιμοποιηθεί

Μόλις ρυθμιστεί, μπορείτε εύκολα να χρησιμοποιήσετε τις ρυθμίσεις σας σε άλλους φορητούς ή σταθερούς (desktop) υπολογιστές με κάρτες WLAN. Απλά αντιγράψτε το /etc/network/interfaces (αλλάξτε τα ονόματα τον συσκευών αν χρειάζεται) και το /etc/wpa_supplicant/wpa_supplicant.conf στον νέο υπολογιστή. Δεν χρειάζεται να "εγκαταστήσετε" τίποτα μετά από αυτό. Είναι "wash & go" (TM).

2. Backup (αντίγραφο ασφαλείας)

Είναι καλό να κάνετε backup το /etc/network/interfaces και το /etc/wpa_supplicant/wpa_supplicant.conf , αλλά κρυπτογραφήστε το backup because it contains sensitive information. An easy way is a password protected archive like 7zip, zip, rar or others.

3. Εύκολη πρόσβαση στο wpa_gui

Είναι βαρετό να γράφετε "/usr/sbin/wpa_gui" μέσα στην κονσόλα. Μπορείτε να προσθέσετε ένα κουμπί εκκίνησης στην επιφάνεια εργασίας (desktop), στο kicker ή στο kmenu. Για παράδειγμα στην επιφάνεια εργασίας, μπορεί να έχει σαν όνομα $HOME/Desktop/wpagui.desktop και σαν δεδομένα να έχει:

[Desktop Entry]
Comment=
Comment[de]=
Encoding=UTF-8
Exec[$e]=/usr/sbin/wpa_gui
GenericName=wpagui
GenericName[de]=wpagui
Icon=kwifimanager
MimeType=
Name=wpagui
Name[de]=wpagui
Path[$e]=
StartupNotify=true
Terminal=false
TerminalOptions=
Type=Application
X-DCOP-ServiceType=
X-KDE-SubstituteUID=false
X-KDE-Username=

Μπορείτε εύκολα να δημιουργήσετε ένα τέτοιο κουμπί κάνοντας δεξί κλικ στην επιφάνεια εργασίας (desktop) και επιλέγοντας "new > link to program".
