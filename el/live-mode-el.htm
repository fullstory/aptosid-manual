Κωδικός υπερχρήστη (root) σε Live Mode

Παρακαλώ σημειώστε: Όποτε εκτελείτε κάτι με τις άδειες υπερχρήστη (root), πρέπει να ξέρετε τι κάνετε! Για διαδικτυακό σερφάρισμα μέσω LAN δικτύου καμία πρόσβαση υπερχρήστη (root) δεν απαιτείται.

Στο sidux LiveCD κανένας κωδικός πρόσβασης υπερχρήστη (root) δεν τίθεται. Εάν θέλετε να τρέξετε ένα πρόγραμμα που απαιτεί τα προνόμια υπερχρήστη (root) έχετε διάφορες επιλογές:

Typing sux θα σας πάρει ως υπερχρήστη (root) με τα προνόμια Xapp.

Θέστε έναν (προσωρινό) root-κωδικό πρόσβασης:

Ανοίξτε μια κονσόλα εντολών 

sidux@0[sidux]$ sudo passwd
Βάλτε νέο UNIX κωδικό:
ξαναγράψτε τον νέο UNIX κωδικό:
passwd: password updated successfully => Ο κωδικός πρόσβασης ενημερώθηκε επιτυχώς
sidux@0[sidux]$

τώρα μπορείτε να χρησιμοποιήσετε αυτόν τον κωδικό πρόσβασης για το υπόλοιπο του sidux-live-session.

τρέξτε το πρόγραμμα με

sudo
ή
sux
ή
kdesu

Πατήστε Alt+ F2, και enter "

sudo

μπροστά από το επιθυμητό πρόγραμμά σας, π.χ.:

sudo gparted

Τρέξτε το πρόγραμμα από την γραμμή εντολών root-shell:

ανοίξετε μια κονσόλα / κέλυφος (shell)

sidux@0[sidux]$ sux
root@0[sidux]# gparted (ή ότι άλλο επιθυμείτε ...)

για να βγείτε από τα δικαιώματα του υπερχρήστη γράψτε:

exit

ή επιλέξτε ακριβώς την κορυφαία δεξιά γωνία για να κλείσετε την κονσόλα.

Εγκατάσταση λογισμικού σε κατάσταση Live-CD

apt-get update
apt-get install το-πακέτο-που-θέλετε

Σημείωση: όταν βγείτε από την κατάσταση live-cd δεν κρατάει καμία αλλαγή

Πως να γράψετε σε NTFS-κατάτμηση με το ntfs-3g

Προειδοποίηση: Ενώ το ntfs-3g δηλώνεται για να είναι 'stable' , μην το χρησιμοποιήστε ποτέ χωρίς να έχετε κάνει backup, και φυσικά όχι στα συστήματα παραγωγής! Εάν τα δεδομένα σας χαθούν να γνωρίζεται ότι το κάνετε με δική σας ευθύνη.

Ανοίξτε ένα τερματικό και εισάγετε τις ακόλουθες εντολές: Δείτε Χωρίζονταν τον σκληρό σας δίσκο - Ονομάζοντας τους δίσκους σας

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Για να βγείτε έξω από την κονσόλα γράψτε : exit

Τώρα ο NTFS-δίσκος σας θα πρέπει να έχει τοποθετηθεί (mount) rw και πρέπει να είστε σε θέση να αποθηκεύσετε αρχεία πάνω του. Αλλά και πάλι, προειδοποιείστε! Χρησιμοποιήστε το στις επείγουσες καταστάσεις, δεν συστήνεται για καθημερινή χρήση.
