Κωδικός υπερχρήστη (root) σε Live Mode

Παρακαλώ σημειώστε: Όποτε εκτελείτε κάτι με δικαιώματα υπερχρήστη (root), θα πρέπει να ξέρετε τι κάνετε! Για διαδικτυακό σερφάρισμα μέσω LAN δικτύου καμία πρόσβαση υπερχρήστη (root) δεν απαιτείται.

Στο sidux LiveCD κανένας κωδικός πρόσβασης υπερχρήστη (root) δεν τίθεται. Εάν θέλετε να τρέξετε ένα πρόγραμμα που απαιτεί προνόμια υπερχρήστη (root) έχετε διάφορες επιλογές:

Θέστε έναν (προσωρινό) root-κωδικό πρόσβασης

Ανοίξτε μια κονσόλα εντολών 

sidux@0[sidux]$ sudo passwd
Βάλτε νέο UNIX κωδικό:
ξαναγράψτε τον νέο UNIX κωδικό:
passwd: password updated successfully => Ο κωδικός πρόσβασης ενημερώθηκε επιτυχώς
sidux@0[sidux]$

τώρα μπορείτε να χρησιμοποιήσετε αυτόν τον κωδικό πρόσβασης για το υπόλοιπο του sidux-live-session.

Τρέχοντας ένα πρόγραμμα από ένα root-shell

Πληκτρολογώντας sux θα γίνετε υπέρ-χρήστη (root) με προνόμια Xapp (θα μπορείτε να τρέξετε γραφικές εφαρμογές σαν root).

ανοίξετε μια κονσόλα / κέλυφος (shell)

sidux@0[sidux]$ sux
root@0[sidux]# gparted (ή ότι άλλο επιθυμείτε ...)

Ορισμένες KDE εφαρμογές χρειάζονται την λέξη dbus-launch μπροστά από το όνομα της εφαρμογής:

dbus-launch <Application>

Μια άλλη επιλογή η οποία δουλεύει και σε άλλους διαχειριστές επιφάνειας εργασίας είναι:

Alt+F2 su-to-root -X -c <Application>

Για παράδειγμα:

Alt+F2 su-to-root -X -c gparted

για να βγείτε από τα δικαιώματα του υπερχρήστη γράψτε:

exit

ή επιλέξτε ακριβώς την κορυφαία δεξιά γωνία για να κλείσετε την κονσόλα.

Εγκατάσταση λογισμικού σε κατάσταση Live-CD

apt-get update
apt-get install το-πακέτο-που-θέλετε

Σημείωση: όταν κλείσετε τον υπολογιστή, το live-cd δεν κρατάει καμία αλλαγή, εκτός και αν ενεργοποιήσετε το fromiso και persist.

Πώς να γράψετε σε NTFS-κατάτμηση με το ntfs-3g

Προειδοποίηση: Ενώ το ntfs-3g δηλώνεται να είναι 'stable' (σταθερό) , μην το χρησιμοποιείτε ποτέ χωρίς να έχετε και ένα backup, και φυσικά όχι στα συστήματα παραγωγής! Εάν τα δεδομένα σας χαθούν να γνωρίζεται ότι το κάνετε με δική σας ευθύνη.

Ανοίξτε ένα τερματικό και εισάγετε τις ακόλουθες εντολές: Δείτε Χωρίζονταν τον σκληρό σας δίσκο - Ονομάζοντας τους δίσκους σας

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Για να βγείτε έξω από την κονσόλα γράψτε : exit

Τώρα ο NTFS-δίσκος σας θα πρέπει να έχει προσαρτηθεί (mount) rw και πρέπει να είστε σε θέση να αποθηκεύσετε αρχεία πάνω του. Αλλά και πάλι σας προειδοποιούμε! Χρησιμοποιήστε το στις επείγουσες καταστάσεις, δεν συστήνεται για καθημερινή χρήση.
