Εκκίνηση του aptosid μέσω δικτύου (network block device - nbd)

Προειδοποίηση: το dnsmasq περιλαμβάνει ένα dhcp διακομιστή ο οποίος μπορεί να "συγκρουστή" με έναν ήδη υπαρκτό dhcp διακομιστή στο δίκτυο σας (ο δρομολογιτής σας μπορεί να περιλαμβάνει έναν). Η ασφαλέστερη επιλογή είναι η χρήση μόνο ενός dhcp διακομιστή σε ένα δίκτυο. Αυτό σημαίνει ότι θα πρέπει να απενεργοποιήσετε οποιοδήποτε άλλο dhcp διακομιστή που βρίσκεται στο ίδιο δίκτυο. Η dnsmasq proxy επιλογές που περιγράφονται πιο κάτω θα πρέπει να είναι σε θέση να συνυπάρξουν με άλλο dhcp διακομιστή στο ίδιο δίκτυο, όμως μην το δοκιμάσετε αυτό εκτός και αν είστε ο διαχειριστής του δικτύου και είστε έτοιμος να αντιμετωπίσετε τυχών προβλήματα που μπορεί να προκύψουν.

Τα βασικά

Για την εκκίνηση μέσω δικτύου χρειάζεστε ένα υπολογιστή ο οποίος υποστηρίζει εκκίνηση μέσω δικτύου, ο οποίος μπορεί να συνδεθεί με το δίκτυο και τον οποίο μπορείτε να ρυθμίσετε να παρέχει υπηρεσίες εκκίνησης μέσω δικτύου.

Δεν θέλετε να το κάνετε αυτό στην δουλειά ή σε κάποιο άλλο δίκτυο το οποίο δεν ελέγχετε εκτός και αν εσείς διαχειρίζεστε το δίκτυο ή έχετε πάρει την άδεια και οδηγίες από τον διαχειριστή. Εάν βρίσκεστε σε ένα μεγαλύτερο δίκτυο, τότε μπορείτε να ερευνήσετε όλες τις επιλογές του dnsmasq, όπως τον περιορισμό των διεπαφών στις οποίες "ακούει" ή τους πελάτες στους οποίους θα απαντήσει για να μειώσετε έτσι τον φόρτο του δικτύου.

Οι προαπαιτήσεις

Ένα 2009-04 (ή νεότερο) aptosid iso εκκινημένο για να χρησιμοποιηθεί σαν network boot server. Οι οδηγίες θα πρέπει να είναι οι ίδιες σε ένα οποιοδήποτε ενημερωμένο aptosid ή debian sid σύστημα και θα πρέπει να παρέχει όλα τα στοιχεία που χρειάζονται για να το χρησιμοποιήσετε και σε άλλα συστήματα. Το Linux είναι υποχρεομένο να παρέχει nbd συσκευές.

Το dnsmasq θα χρησιμοποιηθεί για να παρέχει τα πάντα για την αρχική φάση της εκκίνησης. Και πάλι δεν είναι πολύ δύσκολη η μεταφορά των οδηγιών και σε άλλες εφαρμογές.

Εγκατάσταση
apt-get install nbd-server dnsmasq

Ρύθμιση του nbd-server

Αν υποθέσουμε ότι το ISO μπορεί να βρεθεί στο /dev/scd0, (που μάλλον θα μπορεί αν εκκινήσατε από cd, σε διαφορετική περίπτωση χρησιμοποιείστε οποιοδήποτε άλλο κατάλληλο αρχείο ή συσκευή), τότε μπορείτε να ρυθμίσετε ένα nbd-server αρχείο ρυθμίσεων με όνομα nbd-aptosid.conf με ένα κομμάτι με όνομα aptosid-iso για να διαμοιραστείτε το cd στο port 9040 με το να τρέξετε την παρακάτω εντολή:

echo '[generic]' > nbd-aptosid.conf
nbd-server 0.0.0.0:10809 /dev/scd0 -o aptosid-iso >> nbd-aptosid.conf

Η κεφαλίδα 'generic' χρειάζεται πάντα. Αν θέλετε να ρυθμίσετε τον nbd-server να δουλεύει αυτόματα πάνω σε ένα πραγματικό σύστημα, θα θέλετε μάλλον να ρυθμίσετε το /etc/nbd-server.conf . Ο nbd-server έχει πολλές περισσότερες ρυθμίσεις απ' ότι περιγράφουμε εδώ, δείτε man nbd-server.

Για να ξεκινήσετε τον διακομιστή, σαν απλός χρήστης χωρίς να χρειάζετε να ρυθμίσετε ή να αντιγράψετε το αρχείο στο /etc/nbd-server.conf , μπορείτε απλά να τρέξετε:

nbd-server -C nbd-aptosid.conf

Ο στόχος (target) του nbd-server δεν χρειάζετε να είναι κάποιο ISO ή ένα cd/dvd/usb stick, απλά πρέπει να περιέχει ένα κατάλληλο σύστημα αρχείων.

dnsmasq

Στο παράδειγμα που ακολουθεί, υποθέτουμε ότι έχετε ένα απλό δίκτυο στο οποίο το μηχάνημα σας έχει μια σύνδεση ethernet, η οποία ρυθμίζεται μέσω dhcp από ένα άλλο μηχάνημα και το οποίο οι πελάτες δικτυακής εκκίνησης μπορούν να χρησιμοποιήσουν για να ρυθμίσουν τις διεπαφές τους μέσω dhcp.

Οι επιλογές του dnsmasq που μας ενδιαφέρει να ρυθμίσουμε για να μπορέσουμε να εκκινήσουμε το aptosid μέσω δικτύου, είναι το μονοπάτι για τα αρχεία του tftp διακομιστή και ένα αρχείο για να μπορέσει να εκκινήση από αυτό.

Δημιουργήστε ένα κατάλαγο tftp για την εκκίνηση μέσα στο /home (μπορείτε να τον δημιουργίσετε και αλλού αν προτιμάτε). Το μονοπάτι θα είναι /home/tftp.

Μετά δημιουργήστε ένα αρχείο με όνομα pxe-aptosid.conf και βάλτε μέσα τα ακόλουθα:

dhcp-range=0.0.0.0,proxy
pxe-service=x86PC, "boot linux", pxelinux
enable-tftp
tftp-root=/home/tftp
tftp-secure

Όταν χρησιμοποιείτε τον dhcp proxy, πρέπει να παρέχετε ένα pxe μενού με το pxelinux σαν μόνη επιλογή το οποίο και θα ξεκινάει αυτόματα. Αυτό είναι το μοναδικό pxe-service που αναφέρεται πιο πάνω.

Σαν root, μεταφέρετε το αρχείο pxe-aptosid.conf στο /etc/dnsmasq.d/:

sux
mv pxe-aptosid.conf /etc/dnsmasq.d/

Σημείωση: Για ένα δίκτυο (π.χ. 192.168.0.*) χωρίς άλλο διακομιστή dhcp, μπορείτε να γράψετε τα παρακάτω στην θέση των δυο πρώτων γραμμών:

dhcp-range=192.168.0.100,192.168.0.199,1h
dhcp-boot=pxelinux.0

Για να παρέχετε ip διευθύνσεις ξεκινώντας με 192.168.0.100 και τελειώνοντας με 192.168.0.199 και χρόνο ενικοίασης μια ώρα και για να παρέχετε το όνομα αρχείου για να τρέξει μόνο το pxelinux.0 σαν μέρος του dhcp request (όταν χρησιμοποιήτε το proxy, παρέχετε ένα pxe μενού με μόνη επιλογή το pxelinux το οποίο θα ξεκινάει αυτόματα). Αυτό μάλλον δεν θα ρυθμίσει το δίκτυο σας όπωςε επιθυμείτε εκτός και αν ο dnsmasq διακομιστής θα πρέπει να είναι και ο διακομιστής dns σας και gateway για τους πελάτες.

Για να ενεργοποιήσετε το νέο αρχείο, θα πρέπει να φύγετε το # μπροστά από την γραμμή conf-dir=/etc/dnsmasq.d η οποία βρίσκεται στο τέλος του αρχείου /etc/dnsmasq.conf και να επανεκκινήσετε το dnsmasq.

Το dnsmasq έχει πολλές επιλογές και μπορεί να χρησιμοποιηθεί σαν διακομιστής dns, dhcp, pxe και tftp. Τα πιο πάνω είναι τα βασικά κομμάτια που χρειάζονται για το pxelinux.

tftp

Το tftp είναι το αντίστοιχο του καταλόγου boot για το δίκτυο. Συνεχίζοντας το πιο πάνω παράδειγμα με το /home/tftp, θα πρέπει τώρα να βάλετε κάποια αρχεία στον κατάλογο. Αν το cdrom είναι προσαρτισμένο στο /fll/scd0:

cp /fll/scd0/boot/isolinux/* /home/tftp
mkdir /home/tftp/pxelinux.cfg
mv /home/tftp/isolinux.cfg /home/tftp/pxelinux.cfg/default
mkdir /home/tftp/boot
cp /fll/scd0/boot/vmlin* /fll/scd0/boot/initr* /fll/scd0/boot/memtest* /home/tftp/boot/
cp /usr/lib/syslinux/pxelinux.0 /home/tftp/
# required for the tftp-secure option to dnsmasq
chown -R dnsmasq.dnsmasq /home/tftp/*

Now you can edit boot options to your satisfaction in /home/tftp on both the pxelinux.cfg/default and the gfxboot.cfg file.

In particular it is suggested that under the [install] section you set the install= to install=nbd, install.nbd.server to the server's IP on the network and install.nbd.port to the name of the nbd export section, for example. aptosid-iso (as nbd exports are named now rather then simply using port numbers).

Αντί αυτού θα μπορούσατε να απανεργοποιήσετε το F3 μενού εντελός και να αλλάξετε την γραμμή του πυρήνα να μοιάζει κάπως έτσι:

fromhd=/dev/nbd0 root=/dev/nbd0 nbdroot=192.168.1.23,aptosid-iso nonetwork
toram κωδικός εκκίνησης

Εάν προσθέσετε το toram στις επιλογές εκκίνησεις, υπολογιστές με αρκετή ram θα αποσυνδεθούν από τον διακομιστή μόλις αντιγράψουν τα απαραίτητα αρχεία και υπολογιστές με ανεπαρκή ram θα συνεχίσουν κανονικά την εκκίνηση. Χρειάζεται τουλάχιστον 1 gig ram για το toram (2 gig ή περισσότερα είναι το ιδανικό).

Εκκίνηση μέσω δικτύου

Σιγουρευτείτε ότι το BIOS του υπολογιστή-πελάτη είναι ρυθμισμένο για Boot from Network.

Εάν το BIOS του υπολογιστή υποστηρίζει εκκίνηση μέσω δικτύου, ο υπολογιστής είναι συνδεδεμένος με τον διακομιστή και ο πυρίνας και το initrd.img του aptosid υποστηρίζουν την κάρτα δικτύου σας, θα πρέπει να μπορείτε να εκκινήσετε το aptosid μέσω του δικτύου.

Μερικές κάρτες δικτύου μπορεί να χρειάζονται μη-ελεύθερο firmware πράγμα που σημαίνει ότι θα πρέπει να το προσθέσετε στο initrd.img .
