Eγκατάσταση Timeserver

Αρχικά σε μια γραμμή εντολών, σαν υπερχρήστης root, εισάγουμε

apt-cache search ntp
apt-get update && apt-get install  ntp ntp-doc
update-rc.d -f ntp defaults
Εκτελέστε το update-rc.d αργότερα,αφού έχετε κάνει κάποιες ρυθμίσεις

Βρείτε το έγγραφο οδηγειών στο σύστημα σας στη διεύθυνση

/usr/share/doc/ntp-doc/html/index.html 
και βάλτε έναν σελιδοδείκτη (bookmark)!

Είναι ένα μεγάλο έγγραφο, αλλά δεν εφαρμόζεται όλο, αφού είναι περιεκτικό.

Το ntp δεν θα ενεργοποιηθεί μέχρι να επανεκκίνησετε, αλλά θα πρέπει να ρυθμίσετε την ώρα σας όσο πιο έγκυρα γίνεται πριν την επανεκκίνηση.

Επειδή το ntp τρέχει σαν υπηρεσία, το ntpdate υπάρχει ήδη στο sidux. Για να δείτε αν το ntp είναι ενεργοποιημένο στο σύστημα σας:

sux
rcconf
(χρησιμοποιήστε τα βελάκια στο πληκτρολόγιο για να μετακινηθείτε μέσα στη λίστα,
και το πλήκτρο space για να βάλετε ένα * ώστε να διαλέξετε την υπηρεσία)

Το ntp θα ρυθμίσει την ώρα του από την λίστα των servers στο αρχείο /etc/ntp.conf, το οποίο είναι το κύριο αρχείο ρυθμίσεων.

Και το ntpdate και ο ntpd deamon [που ονομάζεται ntp] χρησιμοποιούν την λίστα των timeservers κοντά στην αρχή του αρχείου /etc/ntp.conf. Εδώ είναι μια λίστα σαν παράδειγμα:

 pool.ntp.org maps to more than 100 low-stratum NTP servers.
# Your server will pick a different set every time it starts up.
#  *** Please consider joining the pool! ***
#  ***  http://www.pool.ntp.org/#join  ***
server 192.168.3.24
server ntp.blueyonder.co.uk
server uk.pool.ntp.org
server 1.uk.pool.ntp.org
server 2.uk.pool.ntp.org
server 0.europe.pool.ntp.org
server 1.europe.pool.ntp.org
server 2.europe.pool.ntp.org

Η πρώτη καταχώρηση είναι ένα άλλο μηχάνημα στο ίδιο δίκτυο, το οποίο τρέχει επίσης το ntp [εκεί είναι αντίστοιχα 'server 192.168.3.1']

Η δεύτερη είναι ο timeserver του ISP που είστε συνδεδεμένοι.

Οι επόμενες είναι μερικές απο το uk.pool.ntp.org, μετά κάποιες ευρωπαικές για καλή τύχη.Παρεμπτιπτώντος,οι isp-nameservers σας είναι συχνά πιθανόν να είναι και αυτοί timeservers, μπορείτε να το δείτε τρέχοντας:

ntpdate -v

Με αυτή την εντολή δεν θα αλλάξει κάτι, αλλά θα επιστρέψει ένα χρονικό αποτέλεσμα, περίπου σαν:

# ntpdate -v 192.168.3.24
19 Sep 19:09:27 ntpdate[13329]: ntpdate 4.2.2@1.1532-o Wed Aug  9 12:08:54 UTC 2006 (1)

Μια πλήρης λίστα των ntp timeservers βρίσκεται εδώ http://www.pool.ntp.org/

Μετά θα θέλετε να επιτρέψετε την πρόσβαση στα τοπικά σας σύστηματα

#Local users may interrogate the ntp server more closely.
restrict 127.0.0.1 nomodify
restrict 192.168.3.0/24

Τώρα θα θέλετε να εκπέμψετε:

# If you want to provide time to your local subnet, change the next line.
# (Again, the address is an example only.)
broadcast 192.168.3.255

Το αρχείο ntp.conf είναι λίγο περίεργο, θεωρείται ως diff αρχείο αν απλά κάνετε κλικ για να το ανοίξετε. Πριν ξεκινήσετε το ntp, πρέπει να ρυθμίσετε την ώρα, πχ

# ntpdate -u -b uk.pool.ntp.org
19 Sep 19:19:33 ntpdate[15641]: step time server 62.3.200.116 offset 0.001523 sec

Μετά ξεκινήστε το ntp, σαν υπηρεσία, για να ξεκινά σε κάθε άνοιγμα [πχ, επανεκκίνηση]. Αφού το ntp έχει τρέξει για λίγο, δώστε:

ntpq -pn

Αν όλα έχουν πάει καλά, θα πρέπει να δείτε κάτι σαν:

Ο αστερίσκος στην τρίτη γραμμή σε αυτό το παράδειγμα: *82.219.3.1, υποδηλώνει τον ενεργό timeserver, αυτός που θεωρείται ο πιο άξιος. Σημαίνει πως πλέον είσαστε σε καλό συγχρονισμό της ώρας, και χρησιμοποιεί την πόρτα 123. Ένα παράδειγμα ενός iptables κανόνα είναι:

# Network Time Protocol (NTP) Server
$IPT -A udp_inbound -p UDP -s 0/0 --destination-port 123 -j ACCEPT
$IPT -A INPUT -j ACCEPT -p tcp --dport 123
