Βασικά

Μια κατάτμηση πρέπει να έχει σύστημα αρχείων. Το Linux υποστηρίζει διάφορα συστήματα αρχείων για χρήση. Το ext4 συνηστάται για το aptosid. Το ext2 είναι εύχρηστο σαν σύστημα για αποθήκευση δεδομένων καθώς υπάρχει διαθέσιμος MS Windows™ οδηγός για data-swapping (ανταλλαγή δεδομένων). Ext2 Installable File System For MS Windows.

Για φυσιολογική χρήση συνιστούμε το ext4 σύστημα αρχείων, αυτό είναι το προκαθορισμένο σύστημα αρχείων για το aptosid και είναι πολύ καλά υποστηριζόμενο.

Using KDE Partition Manager & Gparted

Creating and managing partitions is not something that is typically done every day. Therefore, a good idea is to read this guide once, to get comfortable with the concepts and some of the panels that will appear.

KDE Partition Manager - run in a terminal:

sux
partitionmanager

Gparted - run in a terminal:

sux
gparted

  • When GParted or KDE Partition Manager runs , a window is opened, and drives are scanned.

    In KDE Partition Manager the drives are shown in a list on the left.

    In Gparted, if you click the menu (at the top left), a pop down is presented. You can select to refresh the display of the drives on your system.

The following screenshots are of Gparted. KDE Partition Manager behaves in much the same manner.

  • Επεξεργασία (Edit)

    Το μενού Επεξεργασία έχει τρεις απενεργοποιημένες λειτουργίες οι οποίες μπορούν να σας φανούν πολύ χρήσιμες:
    Undo last operations
    Apply
    Clear all operations.

  • Προβολή (View)

    • Device Information

      Το πεδίο Harddisk Information εμφανίζει λεπτομέρειες για τον σκληρό δίσκο, όπως το μοντέλο, το μέγεθος κλπ. Αυτό είναι και το πιο χρήσιμο πεδίο για ένα σύστημα με πολλούς σκληρούς δίσκους, όπου οι πληροφορίες χρησιμοποιούνται για να επιβεβαιώσουν ότι ο σκληρός δίσκος που εξετάζεται είναι και αυτός ο οποίος θέλουμε.

    • Pending Operations

      Στο παράθυρο που βρίσκεται στη βάση βρίσκεται μια λίστα με λειτουργίες που εκκρεμούν. Οι πληροφορίες αυτές είναι χρήσιμες καθώς παρέχουν μια ένδειξη με τον αριθμό των λειτουργιών εκκρεμούν.

  • Συσκευή (Device)

    Το Device σας επιτρέπει να δώσετε μια ετικέτα σκληρού δίσκου (disk label) εάν η παρούσα ετικέτα είναι ακατάλληλη, μπορείτε να το αλλάξετε χρησιμοποιώντας αυτή την επιλογή.

  • Διαμέρισμα (Partition Menu)

    Αυτό το μενού είναι από τα πιο σημαντικά. Σας επιτρέπει να εκτελέσετε πολλές λειτουργίες, μερικές από τις οποίες είναι επικίνδυνες.

    Το Delete (Διαγραφή) επιλέγεται όταν θέλετε να διαγράψετε μια κατάτμηση. Για να γίνει η διαγραφή, πρέπει πρώτα να επιλέξετε την κατάτμηση.

    Το Resize/Move σας επιτρέπει να αλλάξετε το μέγεθος μιας κατάτμησης.

  • Δημιουργώντας μια νέα κατάτμηση

    Στην μπάρα εργαλείων, το κουμπί New (Νέο) σας επιτρέπει να δημιουργήσετε μια νέα κατάτμηση, αν έχετε ήδη επιλέξει μια μη προσδιορισμένη περιοχή. Ένα νέο παράθυρο εμφανίζεται και σας αφήνει να επιλέξετε το μέγεθος που θέλετε, αν θα είναι πρωτεύoν(Primary), Extended ή λογικό(Logical) τμήμα καθώς και το σύστημα αρχείων.

  • Αν κάνετε λάθος

    Αν έχετε κάνει λάθος μπορείτε να χρησιμοποιήσετε το κουμπί Delete (Διαγραφή) για να διαγράψετε την επιλεγμένη κατάτμηση ή αν δέν έχετε πατήσει apply για να εφαρμοστεί η επιλογή σας, μπορείτε να πατήσετε το κουμπί undo.

  • Αλλαγή μεγέθους/Μετακίνηση (resize/move)

    Όταν θέλετε να αλλάξετε το μέγεθος μιας κατάτμησης που έχετε επιλέξει, πατήστε το κουμπί Resize/Move : ένα νέο παράθυρο εμφανίζεται. Χρησιμοποιήστε το ποντίκι για να μειώσετε ή να αυξήσετε το μέγεθος του τμήματος ή αν προτιμάτε, χρησιμοποιήστε τα βέλη.

    Αφού η εντολή για αλλαγή μεγέθους έχει δοθεί, πατήστε το Apply καθώς μέχρι να το πατήσετε δεν έχει γίνει καμιά αλλαγή στο δίσκο.

    Η διάρκεια των λειτουργιών εξαρτάται από το νέο μέγεθος της κατάτμησης.

    Αφού τελειώσετε με την επεξεργασία του πίνακα κατατμήσεων, παρακαλώ αποσυνδεθείτε και επανεκκινήστε το σύστημα σας για να μπορέσει έτσι το aptosid να διαβάσει τον νέο πίνακα κατατμήσεων.

Αλλαγή μεγέθους σε τμήμα NTFS

Η αλλαγή μεγέθους σε τμήμα NTFS απαιτεί επανεκκίνηση του συστήματος! ΜΗΝ ΠΡΑΓΜΑΤΟΠΟΙΗΣΕΤΕ καμιά άλλη λειτουργία στο τμήμα πρίν την επανεκκίνηση, αλλιώς θα υπάρξουν σφάλματα.

  • Μετά την εκκίνηση σε M$ Windows, το σύστημα θα εμφανίσει μια ειδική οθόνη, και ένα μήνυμα που θα ρωτά για drive consistency : Checking file system on c :
  • Αφήστε το AUTOCHK να εκτελεστεί : Τα NT χρειάζονται να ελέγξουν το σύστημα αρχείων μετά την αλλαγή μεγέθους στο τμήμα.
  • Στο τέλος της διαδικασίας, ο υπολογιστής θα επανεκκινήσει αυτόματα για δεύτερη φορά. Αυτό σιγουρεύει ότι όλα πήγαν τέλεια.
  • Μετά την επανεκκίνηση, τα Windows XP θα είναι καλά, αλλά θα πρέπει πρώτα να αφήσετε το σύστημα να εκκινήσει και να περιμένετε για την οθόνη login!
