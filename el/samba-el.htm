Διαμορφώστε το sidux να χρησιμοποιεί το SAMBA για να μοιράζεστε αρχεία από απομακρυσμένες μηχανές με Windows.

Γράψτε τις εντολές ως root (σε τερματικό ή κονσόλα). Τοποθέτησε την διεύθυνση (URL) στον Konqueror (Τρέξτε τον Konqueror ως κανονικός χρήστης).

εξυπηρετητής (server) =όνομα εξυπηρετητή ή IP από την μηχανή των Windows
μοιρασιά (share) =το όνομα της μοιρασιάς

Στον KDE - Konqueror βάλετε την διεύθυνση (URL) smb://server ή την ολοκληρωμένη διεύθυνση(URL) smb://server/share

Σε μια κονσόλα μπορείτε να δείτε τα αρχεία που διαμοιράζονται και βρίσκονται σε ένα διακομιστή (server) με:

smbclient -L server

Για να τοποθετηθεί (mount) ένας διαμοιρασμός σε έναν κατάλογο - (με πλήρη πρόσβαση για ΟΛΟΥΣ τους χρήστες) θυμηθείτε αυτό: Το σημείο τοποθέτησης πρέπει να υπάρχει. Εάν όχι, πρέπει πρώτα να δημιουργήσετε ένα κατάλογο με (το όνομα είναι αυθαίρετο):

mkdir -p /mnt/server_share

Κατόπιν τοποθετήστε τον διαμοιρασμό - μεμακρυσμένο σύστημα αρχείων VFAT:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777 //server/share /mnt/server_share

ή μεμακρυσμένο σύστημα αρχείων NTFS:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777,lfs //server/share /mnt/server_share

για τερματισμό σύνδεσης χρησιμοποιείστε:

umount /mnt/server_share

εάν θέλετε να υποβάλετε μια καταχώρηση στο /etc/fstab για να είναι η τοποθέτηση ευκολότερη, βάλτε την ακόλουθη γραμμή σε εκείνο το αρχείο:

//server/share  /mnt/server_share       smbfs  defaults,username=your_username,password=**********,fmask=777,dmask=777   0   0

Πως ρυθμίζεις το sidux ως Samba-Server

Δεδομένου ότι το samba δεν είναι στο live-Cd sidux, θα πρέπει να κάνετε τα εξής για να έχετε πρόσβαση στο samba:

su
apt-get update
apt-get install samba
HD (σκληρός δίσκος) εγκατάσταση

Δεδομένου ότι το samba δεν είναι προ-εγκατεστημένο θα πρέπει να κάνετε τα εξής για να έχετε πρόσβαση στον samba:

sux
apt-get update
apt-get install samba

Σε μια HD-εγκατάσταση είναι απαραίτητο να ρυθμίσετε τη διαμόρφωση Samba. Εδώ είναι ένα απλό παράδειγμα. Εάν θέλετε να ξέρετε περισσότερα για τη χρήση Samba και την οργάνωση ενός κεντρικού υπολογιστή Linux Samba ενδείκνυται να διαβάσετε την τεκμηρίωση Samba..

για να ρυθμιστεί η διαμόρφωση samba, κάντε το εξής:

άνοιξε το αρχείο /etc/samba/smb.conf σε έναν συντάκτη (e.g. kedit or kwrite) Και βάλτε αυτό:

# Globale Changes - Proposal everything simple as
#possible - no passwords, perform like Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Share without write-permission -important if NTFS Filesystems are to be shared!

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/hda1 # <-- adjust to your partition
public = yes

# Sharing a partition with permission to write- the partition has to be mounted
# writable - makes sense with e.g. FAT32.

[DATA]
comment = Data Partition (first extended Partition)
browseable = yes
writable = yes
path = /media/hda5
public = yes
Έλεγχος των συμμετοχών (shares) στο samba

Για να θέσει τις συμμετοχές αδιαφορώντας για την ασφάλεια, στο samba βάλτε τις ακόλουθες εντολές (δηλ. για μια οργάνωση του τοπικού LAN):

Καθορίστε τουλάχιστον τον φάκελο και το περιεχόμενο του -rwxr-xr-x:

ls -la μονοπάτι/φάκελος_αρχεία_που_μοιράζονται/*

Εάν όχι, κάντε:

chmod -R 755 μονοπάτι/φάκελος_αρχεία_που_μοιράζονται

Εάν θέλετε να είναι εγγράψιμος (writable)

chmod -R 777 φάκελος_αρχεία_που_μοιράζονται

Ένας τρόπος για να είστε σίγουρος ότι ο διαμοιρασμός σας δουλεύει: (μην ξεχάσετε να εκκινήσετε τον κεντρικό σας υπολογιστή (server) ):

smbclient -L localhost

θα πρέπει να δείτε κάτι σαν αυτό:

 smbclient -L localhost
Password:
Domain=[HOME] OS=[Unix] Server=[Samba 3.0.26a]

        Sharename       Type      Comment
        ---------       ----      -------
        IPC$            IPC       IPC Service (3.0.26a)
        MaShare         Disk      comment
        print$          Disk      Printer Drivers
Domain=[MSHOME] OS=[Unix] Server=[Samba 3.0.26a]

αν δεν έχετε βάλει κάποιο κωδικό απλά πατήστε ΕΝΤΕR

Μη ξεχάσετε να κάνετε save. Τώρα μπορείτε να εκκινήσετε ή να σταματήσετε τον samba με:

/etc/init.d/samba start

και

/etc/init.d/samba stop

Μπορείτε επίσης να ξεκινάτε και να σταματάτε τον samba αυτόματα κάθε φορά που ανοίγεται τον υπολογιστή σας. Δώστε αυτή την εντολή.

update-rc.d samba defaults

Τώρα ο samba ξεκινάει όταν συνδέεστε στον υπολογιστή σας και σταματάει όταν τερματίζεται την λειτουργία του.
