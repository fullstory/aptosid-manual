Το smxi /sgfxi Σετ σύνταξης εντολών

Για να χρησιμοποιήσετε το πολύ-εργαλείο διαχειριστή smxi, το οποίο είναι συντομογραφία του sidux-maintenance (sidux-συντήρηση) και είναι μια συνεισφορά της κοινότητας του sidux, πρέπει να εγκαταστήσετε το script:

Για να εγκαταστήσετε το smxi:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip && smxi
και ακολουθήστε την διαδικασία

ή για να εγκαταστήσετε μόνο το script των 3D οδηγών 

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
Για να δείτε τις επιλογές:
sgfxi -h

Βασικά smxi script χαρακτηριστικά και λειτουργίες

Το script έχει πολλά χαρακτηριστικά, πολλά από τα οποία εμφανίζονται μόνο μια φορά στους χρήστες ενώ άλλα είναι πάντα εκεί. Μετά που κάνετε τον αρχικό έλεγχο του συστήματος και απαντήσετε σε λίγες ερωτήσεις ρυθμίσεων, αυτές είναι οι κύριες επιλογές που θα δείτε:

  • Script πληροφορίες. Αυτό το τμήμα δείχνει τον τρέχον πυρήνα, πότε χρησιμοποιήθηκε τελευταία φορά το script για ενημέρωση (update) του συστήματος , πότε εκτελέστηκε τελευταία φορά το script και κάποιες άλλες πληροφορίες. Αυτό μπορεί να σας φανεί ιδιαίτερα χρήσιμο σε συστήματα που δεν ενημερώνετε συχνά (συνιστάμε: πάντα να ενημερώνετε το σύστημα τουλάχιστον μια φορά τον μήνα. Λιγότερο συχνά είναι επικίνδυνο.)
  • Εγκατάσταση πυρήνα (kernel). Προσφέρει τον τρέχων σταθερό (stable) πυρήνα ως προ επιλεγμένο και προσφέρει ρυθμίσεις για προχωρημένους (advanced options) που σας αφήνουν να εγκαταστήσετε άλλη, παλιότερη έκδοση. Αν εκκινήσετε το smxi με την -Κ επιλογή, ο τρέχων δοκιμαστικός πυρήνας θα προσφερθεί μέσα στην λίστα σαν "σταθερός", αν υπάρχει.
  • dist-upgrade προειδοποιητικό τμήμα (dist-upgrade warning section) : Δίνονται ζωντανές προειδοποιήσεις για τρέχοντα και περασμένα προβλήματα του sid. Αφού συμφωνήσετε να συνεχίσετε (υπόδειξη: το sid είναι σχεδόν ποτέ σε πράσινο, σε κατάσταση χωρίς προβλήματα, είναι συνήθως κίτρινο, ασφαλές αλλά με διορθώσεις).
  • διορθώσεις πριν το dist-upgrade (pre dist-upgrade fixes): Εδώ γίνονται διορθώσεις και προσαρμογές, αν χρειάζονται.
  • προειδοποιήσεις για αρχεία ρυθμίσεων (config file alerts): Θα δείτε ένα 'ενημερώστε αυτά τα αρχεία ρυθμίσεων (update these configuration files)' κομμάτι. Δείτε με μια γρήγορη ματιά αυτά και αν η αναβάθμιση ζητήσει αν θέλετε να κρατήσετε ή να αντικαταστήσετε το αρχείο ρυθμίσεων με εκείνο το αντικείμενο, πείτε y αντί για τα συνηθισμένα n.
  • τμήμα dist-upgrade (dist-upgrade section): συμπεριλαμβάνει χειρισμό σφαλμάτων (error handling) για να σας ειδοποιήσει αν η ενημέρωση έγινε χωρίς προβλήματα ή αν έγινε κάποιο σφάλμα.
  • διορθώσεις μετά το dist-upgrade (post dist-upgrade fixes): εκτελείτε, αν υπάρχει κάποια.
  • Ποικίλες ρυθμίσεις (miscellaneous options) Εμπεριλαμβάνονται:
    • φτιάχνοντας τα fonts (fixfonts) - σας δίνει ένα σύνολο από επιλογές επιδιορθώσεις γραμματοσειρών. Είναι καλό για την πρώτη εγκατάσταση, σπάνια χρειάζεται μετά από αυτό.
    • Εγκατάσταση πακέτων (package-install) - ομάδες από κοινές επιλογές εγκαταστάσεις πακέτων, συμπεριλαμβάνεται και μη ελεύθερο λογισμικό όπως τα πακέτα εγκατάστασης flash και googleearth και εγκαταστάτες (installer) για προγράμματα γραφείου. Σημείωση: ο OpenOffice.org εγκαταστάτης προσφέρει πακέτα διαφόρων γλωσσών να επιλέξετε και κάποια άλλα καλά τα οποία μπορεί να μην ξέρετε ότι υπάρχουν. Επίσης εργαλεία και άλλες επιλογές εγκατάστασης πακέτων.
    • Απεγκατάσταση πακέτων (package-removal) - Μια λίστα από διάφορα ομαδοποιημένα πακέτα που σας δίνετε η δυνατότητα να απεγκαταστήσετε, όπως wifi, κομμάτια γερμανικών κτλ.
    • πράγματα καθαρισμού (clean-up-stuff) - διάφορες επιλογές καθαρισμού, όπως απεγκαταστάτης (remover) πυρήνα (σημείωση: πάντα αφήνετε τουλάχιστον ένα παλαιότερο πυρήνα εγκατεστημένο όταν τρέχετε sidux για να αποφύγετε πιθανά προβλήματα στο μέλλον)
    • διάφορα πειράγματα (miscellaneous-tweaks) - πράγματα όπως τα δημοφιλή πειράγματα των ρυθμίσεων του mozilla και του file hander και άλλα πράγματα που αλλάζουν με τον καιρό, δείτε το.
    • συνέχεια (continue) - συνεχίζει με ερωτήσεις του εγκαταστάτη γραφικών
    • έξοδος (quit) - βγένει από το smxi σε αυτό το σημείο, αν δεν χρειάζεστε να κάνετε κάτι άλλο. Σημείωση, ο εγκαταστάτης γραφικών έχει μια επιλογή start kde/init 5 , έτσι μπορείτε πάντα να το χρησιμοποιείστε αυτό για να επιστρέψετε στην επιφάνεια εργασίας σας
  • εγκαταστάτης γραφικών (graphics installer) - αυτό χρησιμοποιεί τον sgfxi εγκαταστάτη γραφικών από το sidux για να εγκαταστήσει nvdia και συνήθως ati fglrx proprietary (κλειστού κώδικα) οδηγούς (οι ati οδηγοί ήταν λίγο ανώμαλοι τον τελευταίο χρόνο αλλά συνήθως δουλεύουν). Επίσης προσφέρει εγκατάσταση οδηγών xorg καθώς και εκκίνηση του x/desktop σας

smxi script επιλογές: παραλείποντας κομμάτια του script

Τα παρακάτω σας επιτρέπουν να παραλείψετε κομμάτια του du script:

  • -d Παραλείπει το dist-upgrade αν επιλέξετε και την w επιλογή.
  • -e Παραλείπει μερικές από τις 'Please hit enter to continue' (Παρακαλώ πατήστε enter για να συνεχίσετε) ερωτήσεις.
  • -g Παραλείπει την ερώτηση για εγκατάσταση γραφικών.
  • -i Παραλείπει το κομμάτι με τις πληροφορίες συστήματος.
  • -k Παραλείπει την ερώτηση εγκατάστασης πυρήνα.
  • -r Παραλείπει τον έλεγχο για το τελευταίο, επανεκκίνηση και κατέβασμα της τελευταίας έκδοσης του script. Αυτό παραλείπει την επανεκκίνηση του script. Δεν συνιστάται για τους περισσότερους χρήστες εκτός και αν επανεκκινούν από την εγκατάσταση ενός νέου πυρήνα και τρέχουν ξανά το script.
  • -s Παραλείπει μερικά κομμάτια του script που χρειάζονται σύνδεση στο διαδίκτυο. Αποφύγετε να το χρησιμοποιείτε αυτό μιας και απενεργοποιεί σχεδόν όλη την λειτουργικότητα του script.
  • -S Παραλείπει τον έλεγχο για σύνδεση στο διαδίκτυο. Χρησιμοποιήστε αυτό αν το script δεν περνάει τους ελέγχους σύνδεσης αλλά ξέρετε ότι η σύνδεση σας είναι μια χαρά. (αυτή η επιλογή δεν συνιστάται και μπορεί να μην δουλέψει πάντα)
  • -t Παραλείπει το κομμάτι με τις Post dist-upgrade επιλογές.
  • -w Παραλείπει το κομμάτι με τις προειδοποιήσεις ενημέρωσης.

Για παράδειγμα:

smxi -kwid

θα πάει άμεσα στο κομμάτι με τις διάφορες επιλογές, μετά το dist-upgrade. smxi -ki θα παρέλειπε την ερώτηση εγκατάστασης πυρήνα και το κομμάτι με τις πληροφορίες.

smxi επιλογές για προχωρημένους

Τα παρακάτω είναι επιλογές του script για προχωρημένους και είναι για να κάνετε πράγματα τα οποία δεν είναι συνηθισμένα αλλά είναι χρήσιμα σε εσάς:

  • -I Αλλάζει το Init επίπεδο της επανεκκίνησης του X. Χρειάζεται αυτή την σύνταξη: -I 3
    Παρακαλώ σημειώστε, ο αριθμός είναι το init επίπεδο στο οποίο θέλετε το script να ξεκινήσει τον X.
  • -K Εγκατάσταση Πυρήνα για Προχωρημένους. Εάν υπάρχουν νεότεροι δοκιμαστικοί πυρήνες, θα φανούν στις επιλογές εγκατάσταση πυρήνα. Εγκαταστήστε προχωρημένους πυρήνες με δική σας ευθύνη!
  • -l Δημιουργεί time stamped logs (ημερολόγια) όλων των δεδομένων ενημέρωσης και dist-upgrade. Τα αρχεία ημερολογίου (Logfiles) βρίσκονται στο /var/log/smxi/
  • -m Τρέχει την επιλογή για να αλλάξει debian mirrors μέσα στο /etc/apt/sources.list
  • -M Τρέχει την επιλογή για να αλλάξει τα mirrors προεπιλογής για τον πυρήνα για το smxi
  • -p Τρέχει το apt-get update χωρίς την χρήση pdiffs. Καλό για γρήγορες συνδέσεις.
  • -P Τρέχει το script με ρυθμίσεις Proxy για την σύνδεση, απαιτεί αυτή την σύνταξη:
    -P http://[<όνομαχρήστη><:password>@]<διακομιστής><:πόρτα>
  • -U Επιβάλει μια ενημέρωση του script πριν ξανά τρέξει.
  • -x Τρέχει έξτρα λειτουργίες του script οι οποίες δεν είναι μέρος της κεντρικής λειτουργικότητας.
  • -h Τυπώνει το μενού βοηθείας του smxi script.
  • -v Τυπώνει την έκδοση και πληροφορίες συστήματος συμπεριλαμβανομένου της έκδοσης διανομής, τρέχων πυρήνα, πότε χρησιμοποιήθηκαν τελευταία φορά τα apt update και dist-upgrade και τελευταία χρήση του smxi.

Επιλογές του smxi script που ρυθμίζονται από τον χρήστη

Μπορείτε να έχετε το script να κάνει μερικά ακόμη πράγματα με το να επεξεργαστείτε το αρχείο ρύθμισης του /etc/smxi.conf

Παρακαλώ σημειώστε ότι το αρχείο δεν υπάρχει πριν εκτελεστεί το script για πρώτη φορά, γιαυτό αν θέλετε να προσθέσετε οποιαδήποτε από αυτά και δεν έχετε το αρχείο, απλά δημιουργήστε το.

  • sgfxi=<μερικές κανονικές sgfxi επιλογές, όχι συνηθισμένες στον εγκαταστάτη γραφικών>, για παράδειγμα, αν προσθέσετε αυτή την γραμμή:
    sgfxi=b
στο
/etc/smxi.conf,

    η nvidia εγκατάσταση σας θα χρησιμοποιήσει αυτόματα την επιλογή t καθώς και την συνηθισμένη -c
    Σημειώστε ότι αυτές οι επιλογές υποστηρίζονται από το sgfxi, όχι από το smxi, το smxi απλά προσθέτει αυτές στις επιλογές που στέλνει στο sgfxi όταν κάνετε εγκατάσταση των οδηγών γραφικών. Αυτό δεν υποστηρίζει επιλογές που χρειάζονται επιπλέον πληροφορίες, όπως: -o 8.33.6 για παράδειγμα, μόνο επιλογές ενός γράμματος για τον sgfxi εγκαταστάτη γραφικών.
  • hold-install=<λίστα πακέτων διαχωρισμένη με ^> - Αυτή η επιλογή σας επιτρέπει να προσθέσετε πακέτα στην συνάρτηση αυτόματης κράτησης/άφεσης (hold/release) του script η οποία σας βοηθάει να αποφύγετε προσωρινά προβλήματα με σπασμένα (broken) πακέτα ή για να κρατήσετε πακέτα τα οποία δεν θέλετε να αναβαθμίσετε, χωρίς όμως να τα ξεχάσετε. Παράδειγμα:
    hold-install=nano^libgtk2.0
    Σημείωση: Σημαντικό: όχι κενά, μόνο το ^ να χρησιμοποιείται ανάμεσα στα ονόματα των πακέτων!

sgfxi

script για να εγκαταστήσετε μόνο τους 3D οδηγούς

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
Για να δείτε τις επιλογές:
sgfxi -h

Το sgfxi , για τώρα, υποστηρίζει τους ATI fglrx και Nvidia οδηγούς. Επίσης υποστηρίζει αλλαγές από ή στο xorg ελεύθερους οδηγούς όπως ati, intel ή nv. Το smxi χρησιμοποιεί το sgfxi σαν την μηχανή εγκατάστασης οδηγών γραφικών και απλός προσθέτει μερικές επιλογές υποστήριξης για χρήστες.

Εγκαθίσταται ο προ επιλεγμένος οδηγός εάν δεν υπάρχουν άλλες επιλογές για την εγκατάσταση. Ακόμη το script αναγνωρίζει αυτόματα παλιές κάρτες γραφικών και εγκαθιστά legacy-οδηγό, χωρίς να χρειαστεί να κάνετε τίποτα άλλο.

Όταν τρέχετε το script, θα σταματήσει και θα σας πει τι οδηγούς πρόκειται να εγκαταστήσεις. Μπορείτε να το αποδεχτείτε αυτό (accept) ή να βγείτε (exit) και να το ξανά εκτελέσετε, εισάγοντας νέες επιλογές, αν θέλετε κάτι διαφορετικό.

Μπορείτε επίσης να εκτελέσετε τον sgfxi εγκαταστάτη, χρησιμοποιώντας το smxi το οποίο χρησιμοποιεί το sgfxi για να εγκαταστήσει τους οδηγούς γραφικών σας. Πρώτα διαβάστε την smxi κεντρική σελίδα για περισσότερες πληροφορίες και μετά εκτελέστε αυτήν την εντολή:

smxi -kiwdt

sgfxi -h: Όταν χρησιμοποιείτε την επιλογή -h θα δείτε επίσης μία λίστα όλων των για τώρα υποστηριζόμενων οδηγών, καθώς επίσης μια σύντομη λίστα με τις ρυθμίσεις του τρέχων συστήματος σας, έκδοση πυρήνα και xorg. Εδώ είναι κάποιες χρήσιμες sgfxi επιλογές:

-c Χρησιμοποιήστε σύνθετο (composite) τρόπο (nVidia κάρτες μόνο).
-d αφαιρεί IgnoreDisplayDevices, για να σας επιτρέψει να χρησιμοποιήσετε την τηλεόραση σας για παράδειγμα. (nVidia κάρτες μόνο).
-h Δείτε αυτό το μενού βοήθειας.
-n Εγκαθιστά αυτόματα τους σωστούς native οδηγούς xorg nv, intel ή ati για το σύστημά σας.
    Καθαρίζει παλιά binary πράγματα οδηγών, ενημερώνει το xorg.conf με νέες πληροφορίες οδηγών
    και καθαρίζει nvidia και fgrlx πράγματα έξω από το xorg.conf
-N Επιβάλει την εγκατάσταση του συγκεκριμένου xorg οδηγού.
Σύνταξη: sgfxi -N < ένας από τους οδηγούς της λίστας, όπως -N radeon> 
    Υποστηριζόμενοι οδηγοί Xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
    Επίσης ρυθμίζει το xorg.conf κατ’ απαίτησή. Πλήρης ρύθμιση της radeon xorg 3d αυτόματα.
-o Αγνόησή προεπιλογής: εγκατάσταση συγκεκριμένου οδηγού, αυτό είναι απόλυτα αγνοημένο – συντακτικό: -o 9746 (για nVidia, μην χρησιμοποιείτε το 1.0 μέρος)ή -o 8.32.5 (για fglrx)
-r Αφαίρεση mode γραμμών από το xorg.conf
-t Ενεργοποίηση δυναμικού twinview (nVidia μόνο)

Επιστροφή σε οδηγούς ανοιχτού κώδικα

Οι ανοικτού κώδικα (Open Source) Xorg οδηγοί για nVidia, ATI, Radeon, Intel είναι προ-εγκατεστημένοι με το sidux. Εάν τους έχετε αντικαταστήσει με κλειστού κώδικα οδηγούς, μπορείτε να επιστρέψετε πίσω σε αυτούς με το sgfxi. Πρώτα:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip; smxi
Για να ξεκινήσετε, γράψτε σαν υπέρ-χρήστης (root): 
smxi
και ακολουθήστε τα μηνύματα :: κεντρική σελίδα του script - http://techpatterns.com/h2

Μετά για να εγκαταστήσετε ή για να επιστρέψετε σε ένα native οδηγό

sgfxi -n
Ή
sgfxi -N <Xorg driver>

Οι εντολές κλειδί είναι:

-n Εγκαθιστά αυτόματα τους σωστούς native οδηγούς xorg nv, intel ή ati για το σύστημά σας.
    Καθαρίζει παλιά binary πράγματα οδηγών, ενημερώνει το xorg.conf με νέες πληροφορίες οδηγών
    και καθαρίζει nvidia και fgrlx πράγματα έξω από το xorg.conf
-N Force install of specific Xorg driver. 
Σύνταξη: sgfxi -N < ένας από τους οδηγούς της λίστας, όπως -N radeon> 
 Υποστηριζόμενοι οδηγοί Xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
 Επίσης ρυθμίζει το xorg.conf κατ’ απαίτησή. Πλήρης ρύθμιση της radeon xorg 3d αυτόματα.
