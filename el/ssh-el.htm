SSH

Στους ηλεκτρονικούς υπολογιστές, το Secure Shell ή SSH είναι ένα σετ από προδιαγραφές και ένα συνεργαζόμενο δικτυακό πρωτόκολλο το οποίο επιτρέπει την ίδρυση ενός ασφαλούς καναλιού μεταξύ ενός τοπικού υπολογιστή και ενός απομακρυσμένου υπολογιστή. Χρησιμοποιεί κρυπτογραφία δημόσιου κλειδιού για την εξακρίβωση της ταυτότητας του απομακρυσμένου υπολογιστή και (προαιρετικά) για να επιτρέψει στον απομακρυσμένο υπολογιστή να εξακριβώσει την ταυτότητα του χρήστη. Το SSHο παρέχει εμπιστευτικότητα και ακεραιότητα των δεδομένων που ανταλλάσσονται μεταξύ των δύο υπολογιστών, χρησιμοποιώντας κρυπτογράφηση και κωδικών εξακρίβωσης μηνυμάτων (message authentication codes ή MACs). Το SSH χρησιμοποιείται συνήθως για να συνδεθούμε σε ένα απομακρυσμένο υπολογιστή και να εκτελέσουμε εντολές. Επίσης υποστηρίζει tunneling, διαβίβαση αυθαίρετων πορτών TCP και X11 συνδέσεων. Μπορεί να μεταφέρει αρχεία χρησιμοποιώντας συνεργαζόμενα SFTP ή SCP πρωτόκολλα. Ένας διακομιστής SSH , από προεπιλογή, ακούει στο TCP port 22. Αναφερόμενο από wikipedia(Αγγλικά)

Ενεργοποίηση καλών πρωτοκόλλων ασφαλείας για SSH

Να επιτρέπεται το root login, μέσω ssh, δεν είναι ασφαλές. Δεν θέλουμε οι χρήστες να συνδέονται σαν root από προεπιλογή, το debian θα πρέπει να είναι ασφαλής και όχι ανασφαλής και ούτε θέλουμε να δώσουμε στους χρήστες 10 λεπτά να κάνουν μια γρήγορη επίθεση, πάνω στην ssh σύνδεση μας, με την χρήση λεξικού για να βρουν την κωδική λέξη επομένως, εξαρτάτε από εσάς να ορίσετε τον χρόνο και τις προσπάθειες.

Για να κάνετε το ssh σας πιο ασφαλής, απλά πάρτε τον αγαπημένο σας διαχειρηστή κειμένου, ανοίξτε τον με δικαιώματα χρηστή root και μετά ανοίξτε αυτό το αρχείο:

/etc/ssh/sshd_config

Ακολούθως βρίσκουμε τα εμπλεκόμενa αντικείμενα και τα τροποποιούμε

Τα εμπλεκόμενα αντικείμενα που πρέπει να εντοποίσετε είναι τα εξής:

Port <επιθυμητό port>: Αυτό πρέπει να οριστεί στη σωστή πόρτα που διαβιβάζετε από το δρομολογητή (router) σας. Ο δρομολογητής σας πρέπει να είναι ρυθμισμένος να διαβιβάζει ports. Αν δεν ξέρετε πως να το κάνετε αυτό, τότε ίσος να μην πρέπει να χρησιμοποιήσετε το ssh απομακρυσμένα. Το debian χρησιμοποιεί ως port προεπιλογής το port 22 συνιστάται όμως να χρησιμοποιήσετε port το οποίο να βρίσκεται εκτός του συνηθισμένου εύρους σάρωσης. Ας πούμε ότι χρησιμοποιούμε το port 5874 τότε αυτό θα γίνει:

Port 5874

ListenAddress <ip του υπολογιστή ή δικτυακού interface>: Τώρα που διαβιβάζετε ένα port από τον δρομολογητή σας χρειάζεται το μηχάνημα σας να έχει ένα στατικό ip στο δίκτυο, εκτός και αν χρησιμοποιείτε ένα τοπικό διακομιστή dns, αλλά αν κάνετε κάτι το τόσο περίπλοκο και χρειάζεστε αυτές τις οδηγίες τότε μάλλον κάνετε ένα τεράστιο λάθος. Ας πούμε ότι το ListenAddress είναι αυτό: 

ListenAddress 192.168.2.134

Μετά, το Protocol 2 είναι προεπιλεγμένο στο debian, αλλά ελέγξτε το για να είστε σίγουρη

LoginGraceTime <επιτρεπόμενος χρόνος για να συνδεθείτε σε δευτερόλεπτα>: Αυτό έχει για προεπιλογή τα 600 δευτερόλεπτα, πράγμα παράλογο. Δεν χρειάζεστε 10 λεπτά για να γράψετε το όνομα χρήστη σας και το password, γι' αυτό ας κάνουμε την τιμή αυτή λογική:

LoginGraceTime 45

Τώρα έχετε 45 δευτερόλεπτα για να συνδεθείτε και οι χακερς δεν έχουν σε κάθε προσπάθεια τους 600 δευτερόλεπτα για να σπάσουν το password σας

PermitRootLogin <yes>: Ο λόγος που το debian έχει σαν προεπιλογή στο PermitRootLogin το 'yes' είναι ακατανόητος οπόταν το κάνουμε 'no'

PermitRootLogin no

StrictModes yes

MaxAuthTries <xxx>: Αριθμός των προσπαθειών για σύνδεση, εδώ μπορείτε να βάλετε 3 ή 4 προσπάθειες αλλά όχι περισσότερες

MaxAuthTries 2

Μπορεί να χρειαστεί να προσθέσετε οποιαδήποτε από τα παρακάτω αντικείμενα αν δεν υπάρχουν:

AllowUsers <Ονόματα χρηστών, στους οποίους επιτρέπετε η πρόσβαση μέσω ssh. Τα ονόματα είναι χωρισμένα με κενά >

AllowUsers <xxx>:Μόνο χρήστες που είναι γραμμένοι εδώ μπορούν να χρησιμοποιήσουν την πρόσβαση ssh και αυτοί μόνο με δικαιώματα απλού χρήστη. Με το adduser θα πρέπει να προσθέσουμε ένα χρήστη τον οποίο θα χρησιμοποιούμε μόνο για το ssh:

AllowUsers όποιοςείναιοχρήστης

PermitEmptyPasswords <xxx>: του πιο πάνω χρήστη θα πρέπει να του δοθεί ένα τόσο μακρύ password το οποίο θα είναι αδύνατον να βρεθεί ακόμη και σε ένα εκατομμύριο χρόνια. Μόλις συνδεθεί ο χρήστης, μπορεί να χρησιμοποιήσει την εντολή su για να γίνει root:

PermitEmptyPasswords no

PasswordAuthentication <xxx>: φυσικά και πρέπει η τιμή εδώ να είναι yes εκτός και αν χρησιμοποιείτε κλειδί για να συνδεθείτε

PasswordAuthentication yes [εκτός και αν χρησιμοποιείτε κλειδί]

Τέλος:

/etc/init.d/ssh restart

Τώρα έχετε ένα κάπως πιο ασφαλή ssh. Όχι εντελώς ασφαλή, απλά καλύτερο, ιδίως αν δημιουργήσατε ένα χρήστη τον οποίο θα χρησιμοποιείτε αποκλειστικά για την σύνδεση με ssh.

Σημείωση: Αν σας βγάλει μήνυμα λάθους και το ssh αρνείται να σας συνδέσει, πηγαίνετε στο $HOME ψάξτε για ένα κρυμμένο φάκελο με όνομα .ssh και σβήστε από μέσα το αρχείο με όνομα known_hosts και δοκιμάστε ξανά. Αυτό το μήνυμα λάθους το βγάζει κυρίως αν σας δίνετε δυναμικά η διευθύνση IP (DCHP)

Χρησιμοποιώντας Χ εφαρμογές μέσω δικτύου μέσω SSH

Το ssh -X σας επιτρέπει να συνδεθείτε με ένα απομακρυσμένο υπολογιστή και να έχετε το Χ γραφικό του περιβάλλον να φαίνεται στον τοπικό σας υπολογιστή. Σαν $χρήστης (προσέξτε ότι το Χ πρέπει να είναι κεφαλαίο):

$ ssh -X username@xxx.xxx.xxx.xxx (ή IP)

Δώστε το password για το όνομα χρήστη (username) στον απομακρυσμένο υπολογιστή και τρέξτε την X-εφαρμογή μέσα σε ένα κέλυφος (shell):

$ iceweasel Ή oocalc Ή oowriter Ή kspread

Μερικές πολύ αργές δικτυακές συνδέσεις μπορεί να επωφεληθούν εάν υπάρχει κάποιος βαθμός συμπίεσης για να βοηθήσει στην αύξηση της ταχύτητας μεταφοράς. Για αυτό προσθέστε μια έξτρα επιλογή. Σε γρήγορα δίκτυα η επιλογή αυτή θα έχει το αντίθετο αποτέλεσμα:

$  ssh -C -X username@xxx.xxx.xxx.xxx (ή IP)

Διαβάστε:

$man ssh

Σημείωση: Αν σας βγάλει μήνυμα λάθους και το ssh αρνείται να σας συνδέσει, πηγαίνετε στο $HOME ψάξτε για ένα κρυμμένο φάκελο με όνομα .ssh και σβήστε από μέσα το αρχείο με όνομα known_hosts και δοκιμάστε ξανά. Αυτό το μήνυμα λάθους το βγάζει κυρίως αν σας δίνετε δυναμικά η διευθύνση IP (DCHP)

Απομακρυσμένη αντιγραφή αρχείων και καταλόγων μέσω ssh με scp

Το scp χρησιμοποιεί την γραμμή εντολών (τερματικό/cli) για την αντιγραφή αρχείων μεταξύ υπολογιστών σε ένα δίκτυο. Χρησιμοποιεί τις μεθόδους ταυτοποιήσεις και την ασφαλή μεταφορά πληροφοριών του ssh και γιαυτό θα ζητήση password ή passphrase τα οποία απαιτούνται για την ταυτοποίηση.

Αν υποθέσουμε ότι έχετε ssh δικαιώματα σε ένα απομακρυσμένο υπολογιστή ή διακομιστή, το scp σας επιτρέπει να αντιγράψεται κατατμήσεις, καταλόγους ή αρχεία από και προς τον απομακρυσμένο υπολογιστή, σε μια συγκεκριμένη τοποθεσία ή προορισμό της επιλογής σας στον οποίο έχετε δικαιώματα. Για παράδειγμα, αυτό μπορεί να συμπεριλαμβάνει έναν υπολογηστή ή διακομιστή στο LAN σας (ή οπουδήποτε αλλού στον κόσμο) στον οποίο έχετε πρόσβαση, για να σας επιτρέψει την μεταβίβαση δεδομένων σε ένα USB δίσκο συνδεδεμένο στον υπολογιστή σας.

Μπορείτε να αντιγράψετε αναδρομικά ολόκληρες κατατμήσεις και καταλόγους με την επιλογή scp -r. Προσέξτε ότι το scp -r ακολουθεί συμβολικούς συνδέσμους οι οποίοι βρίσκονται στους κατάλογους που αντιγράφει.

Παραδείγματα:

Παράδειγμα 1: Αντιγραφή μιας κατάτμησης:

scp -r <user>@xxx.xxx.x.xxx:/media/disk1part6/ /media/diskXpartX/

Παράδειγμα 2: Αντιγραφή ενός καταλόγου, στην περίπτωση μας τον κατάλογο με όνομα photos μέσα στο $HOME:

scp -r <user>@xxx.xxx.x.xxx:~/photos/ /media/diskXpartX/xx

Παράδειγμα 3: Αντιγραφή ενός αρχείου, στην περίπτωση μας ενός αρχείου που βρίσκεται μέσα στο $HOME:

scp <user>@xxx.xxx.x.xxx:~/filename.txt /media/diskXpartX/xx

Παράδειγμα 4: Αντιγραφή ενός αρχείου:

scp <user>@xxx.xxx.x.xxx:/media/disk1part6/filename.txt /media/diskXpartX/xx

Παράδειγμα 5: Εάν βρίσκεστε ήδη μέσα στον κατάλογο στον οποίο θέλετε να αντιγράψετε το αρχείο ή τον κατάλογο, χρησημοποιήστε μια ' . ' (τελεία) :

scp -r <user>@xxx.xxx.x.xxx:/media/disk1part6/filename.txt .

Παράδειγμα 6: Για να αντιγράψετε αρχεία από τον υπολογιστή σας σε ένα άλλο, (χρησιμοποιείστε scp -r εάν αντιγράφετε ένα κατάλογο ή μια κατάτμηση):

scp /media/disk1part6/filename.txt <user>@xxx.xxx.x.xxx:/media/diskXpartX/xx

Διαβάστε:

man scp

Απομακρυσμένη πρόσβαση ssh με διαβίβαση X (X-Forwarding) από υπολογιστή με Windows:

* Κατεβάστε και κάψετε τοCygwin XLiveCD
* Βάλτε το CD μέσα στο CD-ROM του υπολογιστή με τα Windows και περιμένετε την αυτόματη έναρξη (autorun).
Κάνετε κλικ στο "continue" μέχρι να ανοίξει ένα παράθυρο κέλυφος (shell window) και μέσα στο παράθυρο πληκτρολογείστε τα πιο κάτω:

ssh -X όνομαχρήστη@xxx.xxx.xxx.xxx

Σημείωση: xxx.xxx.xxx.xxx είναι το IP του απομακρυσμένου υπολογιστή με linux ή το URL του (για παράδειγμα ένας λογαριασμός dyndns.org) και το όνομα χρήστη είναι φυσικά το όνομα του λογαριασμού κάποιου χρήστη που υπάρχει στον απομακρυσμένο υπολογιστή. Αφού συνδεθείτε μπορείτε για παράδειγμα να τρέξετε το "kmail" για να δείτε τα e-mail σας.

Σημαντικό: σιγουρευτείτε ότι το hosts.allow έχει μια καταχώριση που να επιτρέπει την πρόσβαση σε υπολογιστές από άλλα δίκτυα. Αν είστε πίσω από ένα NAT-Firewall ή ένα δρομολογητή (router) σιγουρευτείτε ότι το port 22 διαβιβάζεται στον υπολογιστή σας που έχει linux

SSH με το Konqueror

Και το Konqueror και το Krusader μπορούν να έχουν πρόσβαση σε απομακρυσμένα δεδομένα με την χρήση sftp:// το οποίο χρησιμοποιεί ssh.

Πώς λειτουργεί:
1) Ανοίξετε ένα νέο παράθυρο του Konqueror
2) Στην μπάρα διεύθυνσης (address bar) πληκτρολογείστε: sftp://όνομαχρήστη@διακομιστής-shh.com

Παράδειγμα 1: 

sftp://aptosid1@remote_hostname_ή_ip
(Σημείωση: θα ανοίξει ένα popup το οποίο θα σας ζητήσει το ssh password σας, δώστε το και πατήστε OK)

Παράδειγμα 2: 

sftp://όνομαχρήστη:password@remote_hostname_ή_ip
(Αν το γράψετε με αυτή την μορφή ΔΕΝ θα βγει popup που να σας ζητάει password, θα συνδεθείτε απευθείας.)

Για περιβάλλον LAN

sftp://όνομαχρήστη@10.x.x.x ή 198.x.x.x.x
(Σημείωση: θα ανοίξει ένα popup το οποίο θα σας ζητήσει το ssh password σας, δώστε το και πατήστε OK)

Η SSH GUI σύνδεση του Konqueror έχει αναληφθεί. Με αυτό το παράθυρο του Konqueror μπορείτε να δουλέψετε με τα αρχεία (αντιγραφή/επισκόπηση) που βρίσκονται στον διακομιστή SSH ακριβώς σαν τα αρχεία να βρίσκονταν σε ένα φάκελο στον τοπικό υπολογιστή σας.

ΣΗΜΕΙΩΣΗ: Αν έχετε ορίσει το ssh να χρησιμοποιεί άλλο port έκτος του προεπιλεγμένου port 22, τότε θα πρέπει να διευκρινίσετε στο sftp πιο port να χρησιμοποιήσει:

sftp://χρήστης@ip:port

'χρήστης@ip:port' είναι το πρότυπο σύνταξης για πολλά προγράμματα όπως το sftp and smb.

SSHFS - Mounting Απομακρυσμένα

Το SSFS είναι μια εύκολη, γρήγορη και ασφαλής μέθοδος η οποία χρησιμοποιεί το FUSE για να κάνει mount απομακρυσμένα ένα σύστημα αρχείων (filesystem). Η μόνη απαίτηση που έχουμε από τον διακομιστή είναι να τρέχει ένα ssh deamon.

Στον πέλατη (client) πιθανών να χρειαστεί να εγκαταστήσετε πρώτα το sshfs:

apt-get update && apt-get install sshfs

Η εγκατάσταση του fuse και των ομάδων (groups) δεν είναι απαραίτητη με την έκδοση aptosid έρος και τις νεότερες εκδόσεις μιας και είναι προεγκατεστημένα

Τώρα θα πρέπει να αποσυνδεθείτε και να συνδεθείτε ξανά

Να κάνετε mount ένα απομακρυσμένο σύστημα αρχείων είναι πολύ εύκολο:

sshfs όνομαχρήστη@απομακρυσμένο_hostname:φάκελος τοπικό_mount_point

όπου όνομαχρήστη είναι το όνομα του λογαριασμού που βρίσκεται πάνω στον απομακρυσμένο υπολογιστή:

Αν δεν δοθεί φάκελος τότε θα γίνει mount ο home φάκελος που βρίσκετε στον απομακρυσμένο υπολογιστή. Προσοχή: Η άνω και κάτω τελεία : είναι απαραίτητη ακόμη και αν δεν δοθεί φάκελος!

Όταν ο απομακρυσμένος φάκελος γίνει mount, θα συμπεριφέρεται όπως ένα τοπικό σύστημα αρχείων. Θα μπορείτε να περιηγηθείτε μέσα στα αρχεία, να τα επεξεργαστείτε και να τρέξετε scripts όπως ακριβώς θα κάνατε και με ένα τοπικό σύστημα αρχείων.

Αν θέλετε να κάνετε unmount τον απομακρυσμένο υπολογιστή χρησιμοποιείστε την παρακάτω εντολή:

fusermount -u local_mount_point

Αν χρησιμοποιείτε το sshfs συχνά θα ήταν καλή ιδέα να προσθέσετε μια καταχώριση στο fstab:

sshfs#@remote_hostname:/remote_directory /local_mount_point fuse user,allow_other,uid=1000,gid=1000,noauto 0 0

Αυτό θα επιτρέψει σε κάθε χρήστη που είναι μέλος της ομάδας fuse να κάνει mount το σύστημα αρχείων χρησιμοποιώντας την γνωστή εντολή mount:

mount /μονοπάτι/προς/mount/point

Με αυτή την γραμμή στο fstab σας μπορείτε φυσικά να χρησιμοποιήσετε και την εντολή umount:

umount /μονοπάτι/προς/mount/point

Για να ελέγξετε αν είστε μέλος της ομάδας fuse χρησιμοποιείστε την παρακάτω εντολή:

cat /etc/group | grep fuse

Θα πρέπει να δείτε κάτι τέτοιο:

fuse:x:117: <όνομαχρήστη>

Αν δεν δείτε το όνομα χρήστη σας με την πιο πάνω εντολή χρησιμοποιήστε την εντολή adduser σαν root:

adduser <όνομαχρήστη> fuse

Τώρα το όνομα χρήστη σας θα πρέπει να είναι μέλος της ομάδας fuse και θα πρέπει να μπορείτε να τρέξετε τις εντολές:

Σημείωση: Η "ταυτότητα" δεν θα φαίνεται μέσα στην ομάδα "fuse" μέχρι να αποσυνδεθείτε και να συνδεθείτε ξανά

mount τοπικό_mount_point

και

umount τοπικό_mount_point
