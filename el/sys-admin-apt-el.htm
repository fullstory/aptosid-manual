'Eνας μικρός οδηγός για το ΑPT

Tί σημαίνει APT;

Το APT είναι συντομογραφία του Advanced Packaging Tool (Προχωρημένο Εργαλείο Πακέτων) και πρόκειται για μια συλλογή προγραμμάτων και scripts τα οποία βοηθούν τον διαχειριστή συστήματος (στην περίπτωση μας root) τόσο στην εγκατάσταση αλλά και τη διαχείριση αρχείων deb. Αντίστοιχα το σύστημα είναι ενήμερο για το τι είναι εγκατεστημένο.

Η λίστα πηγών

Το σύστημα "APT" χρειάζεται ένα αρχείο ρυθμίσεων το οποίο περιέχει πληροφορίες για τις τοποθεσίες πακέτων προς εγκατάσταση και αναβάθμιση (upgrade). Συνήθως όταν μιλάμε για το αρχείο αυτό, χρησιμοποιούμε το όνομα sources.list.

Οι πηγές περιέχονται σε ένα φάκελο με όνομα:
/etc/apt/sources.list.d/

Μέσα στον πιο πάνω φάκελο είναι 2 αρχεία με όνομα:
/etc/apt/sources.list.d/debian.list και
/etc/apt/sources.list.d/aptosid.list

Αυτό έχει ως πλεονέκτημα την ευκολότερη (αυτοματοποιημένη) αλλαγή mirror και κάνει την αντικατάσταση μιας λίστας πιο ασφαλής.

Μπορείτε επίσης να προσθέσετε τα δικά σας /etc/apt/sources.list.d/*.list αρχεία

Όλα τα aptosid ISO έχουν ως προεπιλεγμένες τις πιο κάτω πηγές :

#aptosid
# Free University Berlin/ spline (Student Project LInux NEtwork), Germany
deb ftp://ftp.spline.de/pub/aptosid/debian/ sid main fix.main
#deb-src ftp://ftp.spline.de/pub/aptosid/debian/ sid main fix.main

Περισσότερες πηγές για non-free ανάγκες (πακέτα κλειστού κώδικα) μπορούν να βρεθούν στο πάντοτε ανανεωμένο aptosid.list και debian.list :

#Debian
# Unstable
deb http://ftp.us.debian.org/debian/ unstable main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ unstable main contrib non-free

# Testing
#deb http://ftp.us.debian.org/debian/ testing main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ testing main contrib non-free

# Experimental
#deb http://ftp.us.debian.org/debian/ experimental main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ experimental main contrib non-free

ΣΗΜΕΙΩΣΗ: σε αυτό το παράδειγμα το ftp.de καθορίζει την χρησιμοποίηση γερμανικής αποθήκης πακέτων(repository).Mπορείτε να τροποποιήσετε αυτό το αρχείο ως root, και να χρησιμοποιήσετε τα κοντινότερα σε εσάς repositories, απλά αλλάζοντας την χώρα, πχ το ftp.us σε ftp.de ή σε ftp.uk. Οι περισσότερες χώρες έχουν τοπικούς mirrors. Εξοικονομείται bandwidth και οι ταχύτητες αυξάνονται.

Λίστα με τους διακομιστές του Debian και την κατάσταση τους.

Το ΑΡΤ για να διοχετεύεται με ενημερωμένες πληροφορίες για τα πακέτα, χρησιμοποεί μια βάση δεδομένων. Αυτή η βάση περιλαμβάνει τα πακέτα αλλά επίσης και άλλα πακέτα που χρειάζονται για να δουλέψει ένα πακέτο (ονομάζονται εξαρτήσεις(dependencies)). To πρόγραμμα apt-get χρησιμοποιεί αυτή τη βάση δεδομένων όταν εγκαθιστά τα πακέτα που επιλέξατε για να ικανοποιήσει όλες τις εξαρτήσεις και να εγγυηθεί ότι τα πακέτα που επιλέξατε να εγκαταστήσετε θα λειτουργούν. H αναβάθμιση αυτής της βάσης δεδομένων πραγματοποιείται με την εντολή apt-get update.

# apt-get update
Get:1 http://aptosid.com sid Release.gpg [189B]
Get:2 http://aptosid.com sid Release.gpg [189B]
Get:3 http://aptosid.com sid Release.gpg [189B]
Get:4 http://ftp.de.debian.org unstable Release.gpg [189B]
Get:5 http://aptosid.com sid Release [34.1kB]
Get:6 http://ftp.de.debian.org unstable Release [79.6kB]

Εγκατάσταση ενός νέου πακέτου

Η αναβάθμιση πακέτων και η εγκατάσταση νέων πακέτων χωρίς το σταμάτημα του X μπορεί να δημιουργήσει προβλήματα. Όλες οι μεθόδοι εγκατάστασης πακέτων μέσα στον X έχουν αυτό το πρόβλημα.

Αν το πακέτο που θέλετε να εγκαταστήσεται δεν αναβαθμίζει άλλα πακέτα, τότε είναι ασφαλής να το εγκαταστήσετε χωρίς να σταματήσετε τον X. Αν όμως η εγκατάσταση του πακέτου θα αναβαθμίσει και άλλα πακέτα, θα πρέπει να είστε πολύ προσεκτικός. Εκτός και να είστε σίγουρος πως τα πακέτα προς αναβάθμιση δεν χρησιμοποιούνται, καλύτερο είναν να σταματήσετε τον X πριν την εγκατάσταση.

Αν δεν είστε σίγουρος, σταματήστε τον X όπως περιγράφεται στις οδηγίες για το dist-upgrade πριν εγκαταστήσετε οποιαδήποτε πακέτα. Αναβάθμιση ενός εγκατεστημένου συστήματος - dist-upgrade - Τα βήματα .

Υποθέτοντας ότι η βάση δεδομένων του Apt είναι αναβαθμισμένη και το όνομα του πακέτου γνωστό, τότε η παρακάτω εντολή θα εγκαταστήσει το πακέτο qemu για παράδειγμα και όλες τις εξαρτήσεις του: (στη συνέχεια θα μάθετε πως να αναζητάτε και να βρίσκετε πακέτα.)

# apt-get install qemu

The following extra packages will be installed:
  bochsbios openhackware proll vgabios
Recommended packages:
  debootstrap vde
The following NEW packages will be installed:
  bochsbios openhackware proll qemu vgabios
0 upgraded, 5 newly installed, 0 to remove and 20 not upgraded.
Need to get 4152kB of archives.
After unpacking 11.9MB of additional disk space will be used.

Διαγράφοντας ένα πακέτο

Παρομοίως δεν θα πρέπει να σας εκπλήξει που το πιο κάτω θα αφαιρέσει (deinstall) ένα πακέτο, αλλά δεν θα αφαιρέσει τις εξαρτήσεις (dependencies): 

apt-get remove gaim

Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim gaim-irchelper
0 upgraded, 0 newly installed, 2 to remove and 310 not upgraded.
Need to get 0B of archives.
After unpacking 4743kB disk space will be freed.
Do you want to continue [Y/n]?
(Reading database ... 174409 files and directories currently installed.)
Removing gaim-irchelper ...
Removing gaim ...

Στο τελευταίο παράδειγμα, τα αρχεία ρυθμίσεων του πακέτου 'gaim' δεν διαγράφηκαν από το σύστημα. Μπορούν να ξαναχρησιμοποιηθούν σε πιθανή επανεγκατάσταση του πακέτου, το οποίο θα λειτουργήσει με τις ίδιες ρυθμίσεις

Αν επιθυμείτε παράλληλα και την διαγραφή των αρχείων ρυθμίσεων,τότε χρησιμοποιήστε την εντολή:

apt-get --purge remove gaim

Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim*
0 upgraded, 0 newly installed, 1 to remove and 309 not upgraded.
Need to get 0B of archives.
After unpacking 4678kB disk space will be freed.
Do you want to continue [Y/n]? Y
(Reading database ... 174405 files and directories currently installed.)
Removing gaim ...
Purging configuration files for gaim ...

Προσέξτε το * μετά το όνομα του πακέτου στην έξοδο (output) του apt. Ο αστερίσκος στην έξοδο του apt δείχνει ότι θα αφαιρεθούν και τα αρχεία ρύθμισης των πακέτων αυτών.

Υποβάθμιση (downgrade) ή/και κράτηση (hold) ένος πακέτου

Κάποτε μπορεί να είναι απαραίτητο να επιστρέψετε σε μια παλαιότερη έκδοση ενός πακέτου ή να μην το εγκαταστήσετε το πακέτο, γιατί η τελευταία έκδοση του πακέτου έχει μέσα ένα σοβαρό λάθος (bug).

Κράτηση:

echo package hold|dpkg --set-selections

Για να φύγετε ένα πακέτο από την κράτηση:

echo package install|dpkg --set-selections

Για να δείτε μια λίστα όλων των κρατούμενων πακέτων:

dpkg --get-selections | grep hold

Υποβάθμιση

Χρησιμοποιώντας το kmahjongg σαν παράδειγμα, τα βήματα είναι ως εξής:

  1. ενεργοποιήστε/uncomment τις testing πηγές μέσα στο /etc/apt/sources.list.d/debian.list 
    (σβήστε το #) για να λέει 
    deb http://ftp.nl.debian.org/debian/ testing main contrib non-free
  2. apt-get update
  3. apt-get install kmahjongg/testing
  4. κάντε κράτηση του νέου πακέτου με την εντολή:
    echo kmahjongg hold|dpkg --set-selections
  5. απενεργοποιήστε (βάλτε # μπροστά) τις testing πηγές μέσα στο /etc/apt/sources.list.d/debian.list 
  6. apt-get update

Όταν θα θέλετε να χρησιμοποιήσετε την τελευταία έκδοση του πακέτου απλά κάντε αυτό:

echo kmahjongg install|dpkg --set-selections
apt-get update
apt-get install kmahjongg

Αναβάθμιση ενός εγκατεστημένου συστήματος - dist-upgrade - Overview

Αν θα αναβαθμιστεί όλο το σύστημα αυτό επιτυγχάνεται μέσω ενός dist-upgrade. Θα πρέπει πάντοτε να κοιτάζετε τις Τρέχουσες Προειδοποιήσεις που βρίσκονται στην επίσημη σελίδα του aptosid και θα πρέπει να ελέγξετε τις προειδοποιήσεις σε σχέση με τα πακέτα που θέλει να κάνει dist-upgrade το σύστημα σας. Αν πρέπει να κάνετε κράτηση ενός πακέτου κοιτάξτε εδώ Υποβάθμιση και Κράτηση ενός πακέτου.

Μια 'αναβάθμιση' μόνο του 'debian sid' δεν συνιστάται.

Συχνότητα ενός dist-upgrade

Κάνετε dist-upgrade όσο πιο συχνά μπορείτε, ιδανικό είναι μια με δύο φορές την εβδομάδα και τουλάχιστον μια φορά τον μήνα για να είστε ασφαλής. Ένα dist-upgrade που γίνεται κάθε 2 με 3 μήνες θα πρέπει να θεωρηθεί ως ασφαλής εξωτερικό όριο. Με μη τυποποιημένα πακέτα ή πακέτα που κάνατε μόνοι σας compile θα πρέπει να είστε πολύ πιο προσεχτικοί με την αναβάθμιση μιας και αυτά μπορούν να καταστρέψουν το σύστημα σας ανά πάσα στιγμή λόγω ασυμβατότητας.

Μετά την αναβάθμιση της βάσης δεδομένων, μπορείτε να ενημερωθείτε για το ποια πακέτα έχουν νέες εκδόσεις με την παρακάτω εντολή (Πρώτα εγκαταστήστε το apt-show-versions):

apt-show-versions -u
libpam-runtime/unstable upgradeable from 0.79-1 to 0.79-3
passwd/unstable upgradeable from 1:4.0.12-5 to 1:4.0.12-6
teclasat/unstable upgradeable from 0.7m02-1 to 0.7n01-1
libpam-modules/unstable upgradeable from 0.79-1 to 0.79-3.........

H αναβάθμιση ενός πακέτου, συμπεριλαμβανομένων και των εξαρτήσεων του, για παράδειγμα το gcc-4.0 μπορεί να γίνει ως εξής:

apt-get install gcc-4.0
Reading package lists... Done
Building dependency tree... Done
gcc-4.0 is already the newest version.
0 upgraded, 0 newly installed, 0 to remove and xxx not upgraded

Μόνο κατέβασμα (downloading)

Μια όχι τόσο γνωστή επιλογή για να ελέγξετε πια πακέτα είναι μέσα σε ένα dist-upgrade είναι η -d επιλογή (-d flag):

apt-get update && apt-get dist-upgrade -d

Η επιλογή -d σας επιτρέπει να κατεβάσετε τα πακέτα του dist-upgrade χωρίς να τα εγκαταστήσετε από την κονσόλα μέσα στον X και σας επιτρέπει να κάνετε αργότερα dist-upgrade μέσα στο init 3. Αυτό σας δίνει επίσης την ευκαιρία να ελέγξετε τα πακέτα και τις προειδοποιήσεις μιας και πολύ σοφά σας δίνει την δυνατότητα να συνεχίσετε ή να ακυρώσετε την εγκατάσταση.

apt-get dist-upgrade -d
Reading package lists... Done
Building dependency tree
Reading state information... Done
Calculating upgrade... Done
The following NEW packages will be installed:
  elinks-data
The following packages have been kept back:
  git-core git-gui git-svn gitk icedove libmpich1.0ldbl
The following packages will be upgraded:
  alsa-base bsdutils ceni configure-ndiswrapper debhelper
  discover1-data elinks file fuse-utils gnucash.........
35 upgraded, 1 newly installed, 0 to remove and 6 not upgraded.
Need to get 23.4MB of archives.
After this operation, 594kB of additional disk space will be used.
Do you want to continue [Y/n]?Y

Η επιλογή Y θα κατεβάσει τα πακέτα χωρίς να πειράξει το σύστημα.

Διαχειριστές πακέτων όπως το adept, το aptitude, το synaptic και το kpackage δεν είναι πάντα ικανά να προσμετρήσουν τις πολυπληθείς αλλαγές που συμβαίνουν στην Sid (αλλαγές στις εξαρτήσεις, αλλαγές ονομάτων, αλλαγή στα scripts των συντηρητών πακέτων, ...). Αυτό δεν είναι λάθος των προγραμματιστών (developers) ή των εργαλείων τους. Αναπτύσσουν μια σειρά από εξαιρετικά εργαλεία για την σταθερή έκδοση του debian, απλά δεν καλύπτουν τις πολύ ειδικές ανάγκες της Debian Sid.

Χρησιμοποιήστε όποια εφαρμογή θέλετε για αναζήτηση πακέτων, απλα χρησιμοποιείτε το apt-get για εγκατάσταση/αφαίρεση/και αναβάθμιση πακέτων.

Διαχειριστές πακέτων όπως το adept, το aptitude, το synaptic και το kpackage είναι μη-ντετερμενιστικά (για σύνθετη επιλογή πακέτων). Συνδυάζοντας αυτό με μια συνεχώς αναβαθμιζόμενη διανομή όπως η sid και ακόμα χειρότερα κάποιο εξωτερικό repository αγνώστου ποιότητας (αν και δεν το προτείνουμε, αποτελεί μια πραγματικότητα στα συστήματα των χρηστών) θα έρθετε αντιμέτωποι με την καταστροφή.

Το apt-get από την άλλη πλευρά, εφαρμόζει αυστηρά ότι του ζητήθηκε. Αν υπάρξει πρόβλημα(breakage) μπορείτε να το εντοπίσετε και να το διορθώσετε ή να εξουδετερώσετε τα σφάλματα(debug). Αν το apt-get επιθυμεί να διαγράψει το μισό σύστημα (εξαιτίας μιας βιβλιοθήκης) αφήνεται στην κρίση του διαχειριστή (στην περίπτωση μας, εσάς) να ρίξει τουλάχιστον μια σοβαρή ματιά.

Αυτός είναι ο λόγος που τα builds του debian χρησιμοποιούν το apt-get, και όχι άλλα εργαλεία διαχείρησης πακέτων.

Aναζήτηση πακέτου με το apt-cache

Μια άλλη πολύ χρήσιμη εντολή του συστήματος APT είναι η εντολή apt-cache. Η εντολή αυτή ψάχνει μέσα στην βάση δεδομένων APT και δίνει πληροφορίες για τα πακέτα π.χ. μια λίστα όλων των πακέτων που περιέχουν ή που αναφέρονται στο "aptosid" και στο "manual" λαμβάνεται με την παρακάτω εντολή:

$ apt-cache search aptosid manual

GUI εφαρμογή αναζήτησης πακέτων στο Debian

apt-get update
apt-get install packagesearch

Όταν τρέξετε για πρώτη φορά το Debian Package Search, θα πρέπει να ρυθμίσετε το Packagesearch > Preferences να χρησιμοποιεί το apt-get.

Παρακαλώ μην χρησιμοποιείτε το Packagesearch για εγκατάσταση πακέτων, παρά μόνο σαν μια GUI μηχανή αναζήτησης. Η αναβάθμιση πακέτων ή η εγκατάσταση νέων πακέτων χωρίς το σταμάτημα του X μπορεί να δημιουργήσει προβλήματα. Παρακαλώ διαβάστε Εγκατάσταση ενός νέου πακέτου.

Την πρώτη φορά χρήσης πιθανών να σας ζητηθεί να εγκαταστήσετε το deborphan. Παρακαλώ χρησιμοποιήστε το με προσοχή.

Μπορείτε να αναζητήσετε με βάση

  • ένα pattern
  • tags (βασισμένο στο σύστημα debtags, ένας νέος τρόπος κατηγοριοποίησης Debian πακέτων)
  • αρχεία
  • κατάσταση εγκατάστασης
  • ορφανά πακέτα

Επιπλέον παρουσιάζονται πολλές πληροφορίες για τα πακέτα, συμπεριλαμβανομένων και των αρχείων που ανήκουν σε αυτά. Παρακαλώ διαβάστε το Help > Contents για μια πλήρη επεξήγηση για το πως να χρησιμοποιήσετε την GUI εφαρμογή αναζήτησης πακέτων στο Debian. Προς το παρών του GUI είναι μόνο στα Αγγλικά.
