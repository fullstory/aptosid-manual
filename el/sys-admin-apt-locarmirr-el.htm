dist-upgrade υπολογιστών όπου το bandwidth/ταχύτητα είναι πρόβλημα

Για χρήστες που έχουν περισσότερους από 1 υπολογιστές ή αυτοί που έχουν περισσότερο από 1 υπολογιστές και έχουν περιορισμούς στο bandwidth ή αυτοί που πρέπει να αναβαθμίσουν ένα υπολογιστή ενώ υπάρχουν περιορισμοί στην ταχύτητα ή/και στο bandwidth από τον πάρoχο υπηρεσιών διαδικτύου (ISP), υπάρχουν λύσεις οι οποίες σας βοηθούν να διατηρήσετε τους υπολογιστές σας αναβαθμισμένους ασχέτως αν βρίσκονται σε ένα μόνιμο ή προσωρινό τοπικό δίκτυο.

Η λύση είναι η χρήση ενός τοπικού αποθετηρίου σε έναν από τους υπολογιστές τον οποίο θα μπορούν να χρησιμοποιήσουν οι υπόλοιποι υπολογιστές του δικτύου για να κάνουν dist-upgrade εξοικονομώντας έτσι bandwidth για να μπορέσει να χρησιμοποιηθεί για πιο σημαντικές καθημερινές ασχολίες.

Προ-απαιτήσεις

Σιγουρευτείτε ότι έχετε 6 gig ελεύθερου δίσκου για το cache.

Χρησιμοποιώντας το approx σαν τοπικό αποθετήριο

Όταν ο υπολογιστής-πελάτης ζητήσει ένα πακέτο, θα πάρει ένα τοπικό πακέτο, εάν έχετε ήδη τρέξει apt-get update, dist-upgrade -d ή dist-upgrade στον υπολογιστή στον οποίο τρέχει ο διακομιστής approx.

Βήμα 1: Ρύθμιση του διακομιστή approx

apt-get install approx

mcedit /etc/approx/approx.conf

Ρυθμίστε το approx.conf αρχείο για να χρησιμοποιεί τα online αποθετήρια:

# Here are some examples of remote repository mappings.
# See http://www.debian.org/mirror/list for mirror sites.

debian http://ftp.iinet.net.au/debian/ << βάλτε το αποθετήριο του debian που χρησιμοποιείτε
sidux http://sidux.net/debian/

Κάντε το ίδιο και για οποιαδήποτε άλλα αποθετήρια θέλετε να χρησιμοποιηθούν σαν τοπικά.

Ξεκινήστε τον διακομιστή approx με:

update-inetd --enable approx

Εάν δεν δουλέψει, επανεκκινήστε τον υπολογιστή στον οποίο έχετε εγκαταστήσει το approx.

Μετά την επανεκκίνηση, τρέξτε apt-get update και dist-upgrade ή dist-upgrade -d. Έτσι θα είστε σίγουροι ότι το approx έχει πρόσβαση στα νεότερα πακέτα για του υπολογιστές-πελάτες, εκτός και αν υπάρχουν πακέτα τα οποία είναι εγκατεστημένα στους υπολογιστές-πελάτες αλλά όχι στον διακομιστή. Σε αυτή την περίπτωση το approx θα κατεβάσει το κατάλληλο πακέτο.

Τα πακέτα μαζεύονται μέσα στο /var/cache/approx το οποίο αρχίζει να γεμίζει αφού τρέξουν για πρώτη φορά οι πελάτες.

Βήμα 2: Ρύθμιση των πελατών για να χρησιμοποιούν τον διακομιστή approx

Πρώτα: αλλάξτε τα /etc/apt/sources.list.d/*.list αρχεία για να χρησιμοποιούν το approx ως debian και sidux αποθετήρια.

Με το mcedit, "σχολιάστε" τις γραμμές που αναφέρονται στα αποθετήρια (βάλτε ένα # μπροστά τους) και προσθέστε τις ακόλουθες γραμμές και φυλάξτε τις αλλαγές, για παράδειγμα:

Λίστα πηγών Debian
mcedit /etc/apt/sources.list.d/debian.list

#deb τρέχων debian mirror

deb http://approx:9999/debian/ sid main contrib non-free
Λίστα πηγών sidux
mcedit /etc/apt/sources.list.d/sidux.list

#deb το αποθετήριο που έχετε τώρα

deb http://approx:9999/sidux/ sid main fix.main
Άλλες λίστες πηγών

Ακολουθείστε τα πιο πάνω βήματα και για οποιαδήποτε άλλα αρχεία πηγών έχετε.

Proxy Hosts

Έπειτα επεξεργαστείτε το /etc/hosts για να προσθέσετε το τοπικό proxy για πρόσβαση στην IP διεύθυνση του διακομιστή σας:

mcedit /etc/hosts

10.1.1.X approx

Τώρα τρέξτε apt-get update και dist-upgrade ή dist-upgrade -d. Η πρώτη φορά σε κάθε υπολογιστή-πελάτη θα είναι αργή και μπορεί ακόμη να κάνει time out, γιαυτό ξανά προσπαθείστε. Οι επόμενες φορές θα πρέπει να σας παρέχουν τα ωφελήματα που ψάχνετε.
