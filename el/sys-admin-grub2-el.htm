GRUB 2

Βασικές διαφορές μεταξύ του GRUB 1 (γνωστό και σαν grub-legacy) και του GRUB 2:
  • Το menu.lst δεν υπάρχει πλέον.
  • Ένα αρχείο με όνομα grub.cfg είναι τώρα υπεύθυνο για την Οθόνη του Grub.
  • Το grub.cfg δημιουργείται αυτόματα από τα scripts μέσα στο /etc/grub.d.
  • Διαφορετική είναι και η αρίθμηση των κατατμήσεων. Η αρίθμηση του ξεκινάει από το 1 και όχι το 0 (η αρίθμηση των οδηγών/συσκευών εξακολουθεί να αρχίζει από το 0): 
    Linux      grub1     grub2
/dev/sda1  (hd0,0)   (hd0,1)
/dev/sda2  (hd0,1)   (hd0,2)
/dev/sda3  (hd0,2)   (hd0,3)

/dev/sdb1  (hd1,0)   (hd1,1)
/dev/sdb2  (hd1,1)   (hd1,2)
/dev/sdb3  (hd1,2)   (hd1,3)
  • Οι γραμμές στο grub.cfg είναι γραμμένες λίγο διαφορετικά σε σύγκριση με αυτές στο menu.lst και δεν μπορούν να αντιγραφούν απευθείας από το Grub 1 menu.lst στο Grub 2 grub.cfg γιατί το grub.cfg αρχείο είναι το 'αποτέλεσμα'. Δεν πρέπει να γίνονται ποτέ χειροκίνητες αλλαγές στο αρχείο grub.cfg.

Το προεπιλεγμένο αρχείο ρύθμισης του Grub 2

Το /etc/default/grub αρχείο περιέχει τις ρυθμίσεις των μεταβλητών του grub2. Για παράδειγμα, timeout στο μενού, προεπιλεγμένες καταχωρίσεις στο μενού, παραμέτρους πυρήνα, χρήση γραφικών στον grub κτλ.

Τα scripts του Grub 2

Τα /etc/grub.d αρχεία ελέγχουν το αρχείο με όνομα grub.cfg, και το οποίο βρίσκεται στο /boot/grub/.

Δεν πρέπει να γίνονται ποτέ χειροκίνητες αλλαγές στο αρχείο grub.cfg. Όλες οι αλλαγές θα πρέπει να γίνονται με ένα ή όλα τα scripts που ακολουθούν και που βρίσκονται στο /etc/grub.d. Το os-prober θα πρέπει να είναι σε θέση να χειριστή το 90% των περιπτώσεων:

00_header:
05_debian_theme: Sets background, text colors, themes
10_hurd: Locates Hurd kernels
10_linux: Locates Linux kernels based on results of the lsb_release command.
20_memtest86+: If the file /boot/memtest86+.bin exists, it is included in the boot menu.
30_os-prober: Searches for Linux and other OS's on all partitions; includes them in the boot menu.
40_custom: A template for adding custom boot menu entries for other Operating Systems.
60_fll-fromiso: A template for adding custom boot menu entries for fromiso to a USB-stick/SSD-card.
60_fll-fromiso is not to be altered use /etc/default/grub2-fll-fromiso
Read  Booting 'fromiso' with Grub 2

Μετά από μια αλλαγή το grub.cfg θα πρέπει να ενημερωθεί για την αλλαγή. Κατά την διάρκεια της αναβάθμισης του πυρήνα του aptosid, το αρχείο ενημερώνεται αυτόματα. Σε περίπτωση που κάνετε αλλαγές εσείς, σαν διαχειριστής του συστήματος, σε ένα από τα πιο πάνω αρχεία, θα πρέπει να τρέξετε: 

update-grub

Το Debian Grub2 deb πακέτο είναι σχεδιασμένο έτσι ώστε μια χειροκίνητη αλλαγή να χρειάζεται σπανίως.

Αναβάθμιση του Grub Legacy σε Grub 2

Συνηστούμε την "καθαρή" αναβάθμιση σε Grub 2 και την ολοκληρωτική διαγραφή του Grub 1. Θα πρέπει να ξέρετε ότι μπορεί να "σπάσετε" τα πάντα γιαυτό θα πρέπει να είστε προσεκτικοί.

Βήμα 1:

Σιγουρευτείτε ότι το σύστημα σας είναι πλήρως ενημερωμένο, dist-upgrad σε init 3. 

apt-get update

apt-get dist-upgrade
init 5 && exit

rm -rf /boot/grub
apt-get purge grub-gfxboot

Σαν αποτέλεσμα θα έχετε την αφαίρεση των fll-iso2usb* grub-gfxboot* install-usb-gui* . Y για επιβεβαίωση.

apt-get install grub2 os-prober

Dual και multi booting με Grub 2

Ο Grub2 παρέχει μια απλή εντολή η οποία ψάχνει για νέο-εγκατεστημένα συστήματα και αν βρει κάποιο το καταχωρεί στο menu.cfg. Η απλή εντολή είναι:

update-grub

Αν για κάποιο λόγο χρειάζεστε να προσθέσετε εσείς μια καταχώρηση στο menu.cfg ή αν το 30_os-prober αποτύχει να γράψει στο grub.cfg με chainloader menus, χρησιμοποιείστε τον αγαπημένο σας επεξεργαστή κειμένου για να κάνετε αλλαγές στο /etc/grub.d/40_custom.

Παράδειγμα ενός 40_custom αρχείου:

menuentry "second mbr"{
set root=(hd1)
chainloader +1
}

menuentry "second partition"{
set root=(hd0,2)
chainloader +1
}

Αφού κάνετε τις αλλαγές τρέξτε:

update-grub

Σε περίπτωση που σας πει ότι δεν ξέρει την grub συσκευή ενός δίσκου, αυτό σημαίνει ότι το devicemap πρέπει να ξανά-δημιουργηθεί.

Σιγουρευτείτε ότι επιλέξατε την κατάτμηση και όχι το MBR κατά την εγκατάσταση του άλλου λειτουργικού συστήματος:

grub-mkdevicemap --no-floppy
update-grub

Τις προειδοποιήσεις μπορείτε να τις αγνοήσετε.

Αν κάνατε κάποιο λάθος, η ενημέρωση θα διαγράψει το MBR και θα πρέπει να το φτιάξετε με Grub2 - MBR Overwritten.

Για να γράψετε το grub2 στο MBR από τον σκληρό δίσκο:

/usr/sbin/grub-install --recheck --no-floppy /dev/sda

Την εντολή μπορεί να χρειαστεί να την τρέξετε περισσότερο από μια φορές μέχρι να πιστεί ότι το εννοείτε.

MBR Bootsector overwritten από Windows, ένα καταστρεμένο ή/και επαναφορά του Grub2

Σημείωση: για επαναφορά του Grub2 bootloader χρειάζεστε ένα 2009-04 (μόρος) aptosid.iso ή νεότερο. Εναλλακτικά μπορείτε να χρησιμοποιήσετε chroot με οποιοδήποτε live.iso.

Για να ξανά-γράψετε τον grub2 στο MBR ή/και για να επαναφέρετε τον grub 2 γενικότεερα θα χρειαστεί να εκκινήσετε ένα aptosid.iso:

  1. Για να αναγνωρίσετε τις κατατμήσεις σας (π.χ. [h,s]d[a..]X) όλες οι ακόλουθες εντολές χρειάζονται προνόμια root. Για να γίνετε root (#): 
    $ sux
  2. Σαν root: 
    fdisk -l
cat /etc/fstab
    Αυτό είναι για να πάρετε τα σωστά ονόματα.
  3. Όταν αναγνωρίσατε την σωστή κατάτμηση, δημιουργείστε το σημείο προσάρτησης: 
    mkdir -p /media/[hdxx,sdxx,diskx]
  4. Προσαρτείστε το: 
    mount /dev/xdxx /media/xdxx
  5. Τώρα ξανά-γράψτε τον Grub στο MBR του πρώτου δίσκου (γενικό): 
    /usr/sbin/grub-install --recheck --no-floppy --root-directory=/media/xdxx /dev/sda

Χρήση chroot για επαναφορά ενός overwritten ή κατεστραμένου Grub στο MBR

Για επαναφορά του Grub , εάν έχει διαγραφή ή καταστραφή στο MBR, ετοιμάστε ένα chroot περιβάλλον. Any live.iso will suffice as the chroot takes you to your hard disk installed system so you can restore the appropriate grub version, grub1 (grub-legacy) or grub2.

Εκκινήστε ένα live aptosid.iso το οποίο είναι κατάλληλο για το σύστημα σας (32 ή 64 bit CD, DVD, USB stick ή μια SSD κάρτα) και ανοίξτε μια κονσόλα. Γράψτε sux και πατήστε enter για να αποκτήσετε δικαιώματα root.

Με την χρήση του fdisk -l ή blkid, βρείτε πια κατάτμηση είναι η boot κατάτμηση και πάρτε τα σωστά ονόματα. Αν προτιμάτε γραφικό περιβάλλον χρησιμοποιήστε Gparted:

blkid

και για να ελέγξετε ότι το fstab ταιριάζει με την έξοδο του blkid:

cat /etc/fstab

Ας υποθέσουμε ότι το root σας βρίσκεται στο /dev/sda2

mkdir /mnt/aptosid-chroot
mount /dev/sda2 /mnt/aptosid-chroot

Έπειτα θα πρέπει να προσαρτήσετε το /proc και το /dev όπως παρακάτω:

mount -t proc proc /mnt/aptosid-chroot/proc
mount -t devpts devpts /mnt/aptosid-chroot/dev/pts
mount -o bind /dev /mnt/aptosid-chroot/dev

Should internet access be a requirement, copy resolv.conf to the chroot environment before entering chroot:

cp /etc/resolv.conf /mnt/aptosid-chroot/etc/resolv.conf

Το chroot περιβάλλον σας είναι τώρα έτοιμο. Μπείτε μέσα με:

chroot /mnt/aptosid-chroot /bin/bash

Τώρα μπορείτε να χρησιμοποιήσετε το τοπικό αποθετήριο του apt ή να τροποποιήσετε τα αρχεία που χρειάζεται να φτιάξετε όπως θα κάνατε να βρισκόσασταν κανονικά μέσα στο λειτουργικό σύστημα, σε αυτή την περίπτωση να βάλετε το Grub πίσω στο MBR:.

Για να επαναφέρετε Grub 2

apt-get install --reinstall grub-pc

Για να επαναφέρετε Grub 1 (grub-legacy)

apt-get install --reinstall grub-legacy

Ακολουθήστε τις οδηγίες του εγκαταστάτη.

Για να βγείτε από το chroot:

Ctrl+d

Επανεκκινήστε τον υπολογιστή.
