Εγκατάσταση νέων πυρήνων

Οι πυρήνες του aptosid βρίσκονται στο αποθετήριο του aptosid σαν .deb και συμπεριλαμβάνονται αυτόματα στο dist-upgrade.

Οι πυρήνες είναι διαθέσιμοι στις ακόλουθες μορφές:

  • aptosid-686 - πυρήνας για τη i686 οικογένεια επεξεργαστών με μονό/διπλό πυρήνα ή περισσότερες CPU
  • aptosid-amd64 - πυρήνας για το 64bit aptosid
Βήματα για να το κάνετε χειροκίνητα χωρίς dist-upgrade:

1. Στην κονσόλα μπείτε ως υπερχρήστης (root)

apt-get update

2. Για να εγκαταστήσετε την πιο πρόσφατη έκδοση του πυρήνα:

apt-get install linux-image-2.6-aptosid-686 linux-headers-2.6-aptosid-686

4. Τώρα επανεκκινήστε τον υπολογιστή σας και ξεκινήστε με τον νέο πυρήνα (πρέπει να αρχίζει ως πρώτος στην λίστα εξ ορισμού).

Εάν ο νέος πυρήνας σας βγάλει προβλήματα, μπορείτε να επανεκκινήσετε και να επιλέξετε έναν παλαιότερο πυρήνα.

Ενότητες (modules)

Για να βρείτε ποιές ενότητες χρειάζεστε, η ακόλουθη εντολή σας παραθέτει τον κατάλογο τρεχουσών διαθέσιμων ενοτήτων, αντιγράψτε αυτήν την γραμμή στο τερματικό σας:

apt-cache search 2.6.*.slh.*-aptosid| awk '/modules/{print $1}'

Για να πάρετε μια πλήρη περιγραφή για κάθε ενότητα, αντιγράψτε αυτήν την γραμμή στην κονσόλα σας

apt-cache search 2.6.*.slh.*-aptosid

Για να εγκαταστήσετε τις απαραίτητες ενότητες (για παράδειγμα virtualbox-ose, και qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-aptosid-686 (ΠΑΡΑΔΕΙΓΜΑ)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-aptosid-686 (ΠΑΡΑΔΕΙΓΜΑ)

Για να ελέγξετε τις ενότητες που φορτώνονται στον πυρήνα:

ls /sys/module/
ή
cat /proc/modules

Εγκατάσταση modules με το Dynamic Module-Assistant Kernel Module Support (dmakms)

Το dmakms είναι χρήσιμο για εγκατάσταση modules του πυρήνα τα οποία δεν είναι ήδη έτοιμα για τον πυρήνα του aptosid και έχει σχεδιαστή για να αυτοματοποιεί την εγκατάσταση module του πυρήνα με το module-assistant (m-a) όταν αναβαθμίζετε ή εγκαθιστάτε νέο πυρήνα Linux.

apt-get install dmakms module-assistant

Πριν ενεργοποιήσετε το Dynamic Module-Assistant Kernel Module Support, τρέξτε το module-assistant για να εγκαταστήσετε το module πυρήνα που θέλετε για τον τρέχων πυρήνα Linux. Για περισσότερες πληροφορίες για το module-assistant παρακαλώ διαβάστε το εγχειρίδιο του:

man m-a

Το όνομα του πακέτου που είναι συμβατό με το module-assistant θα πρέπει να προστεθεί στο /etc/default/dmakms, για να αυτοματοποιηθεί έτσι η διαδικασία ετοιμασίας και εγκατάστασης του module για κάθε νέο πυρήνα Linux.

Παράδειγμα: Εγκατάσταση του speakup module με το module-assistant

Σιγουρευτείτε ότι η λίστα με τις πηγές έχει μέσα τα contrib non-free μέσα στο: /etc/apt/sources.list.d/debian.list 

apt-cache search speakup-s
speakup-source - Source of the speakup kernel modules

Μετά ετοιμάστε το module:

m-a prepare
m-a a-i speakup-source

Μετά ενεργοποιήστε το Dynamic Module-Assistant Kernel Module Support για το speakup, έτσι ώστε την επόμενη φορά που θα αναβαθμιστεί ο πυρήνας Linux, να ετοιμαστεί και ένα module για τον πυρήνα χωρίς να χρειαστεί να κάνετε κάτι εσείς. Για να το κάνετε αυτό προσθέστε speakup-source στο /etc/default/dmakms αρχείο ρύθμισης. 

mcedit /etc/default/dmakms
speakup-source

Επαναλάβετε την ίδια διαδικασία για οποιοδήποτε άλλο module που είναι συμβατό με το module-assistant.

Ένα linux-headers πακέτο πρέπει να είναι εγκατεστημένο για κάθε linux-image για να μπορέσει το module-assistant να κάνει compile τα modules του πυρήνα.

Αποτυχία φόρτωσης του module

Αν για οποιοδήποτε λόγο [νέο κομμάτι του xorg, πρόβλημα με το σύστημα αρχείων ή αν ο X δεν ξεκινά μετά από το reboot] αποτύχει η φόρτωση του module:

modprobe <module>

Μετά επανεκκινήστε τον υπολογιστή.

Εάν η φόρτωση του module εξακολουθεί να αποτυχαίνει:

 m-a a-i -f module-source

Αυτό ξανά κτίζει το module, μετά επανεκκινήστε.

Πώς λειτουργεί

Το Dynamic Module-Assistant Kernel Module Support αποτελείτε από ένα initscript (/etc/init.d/dmakms) το οποίο εκτελείτε κατά την διάρκεια της εκκίνησης ή από ένα script το οποίο τρέχει μετά από την εγκατάσταση ενός νέου πακέτου linux-image Debian.

Σε κάθε εκκίνηση το /etc/init.d/dmakms τρέχει για να δει αν το κάθε module που βρίσκεται μέσα στο /etc/default/dmakms έχει και ένα module για τον τρέχων πυρήνα Linux και καλεί το module-assistant για να κτίσει και να εγκαταστήσει τα modules όποτε χρειάζεται.

Όταν εγκαθίσταται νέο linux-image Debian πακέτο, τότε το /etc/init.d/dmakms εκτελείτε μέσω ενός postinst hook script με δυο παραμέτρους, 'start' και το 'version string' του Linux πυρήνα για τον οποίο ετοιμάζει το νέο module. Τώρα τα modules που αναφέρονται μέσα στο /etc/default/dmakms αρχείο ρύθμισης προετοιμάζονται με το module-assitant, και τα πακέτα των modules προγραμματίζονται για εγκατάσταση κατά την διάρκεια του shutdown. Ο λόγος που η εγκατάσταση γίνεται κατά την διάρκεια του shutdown είναι γιατί πρέπει το πρόγραμμα να περιμένει μέχρι να έρθει μια στιγμή που τα apt και dpkg να μην είναι κλειδωμένα από κάποια άλλη διεργασία .

$ /usr/share/doc/dmakms

Αφαίρεση παλιών πυρήνων (kernel remover)

Μετά από την επιτυχή εγκατάσταση του νέου πυρήνα, οι παλαιοί πυρήνες μπορούν να διαγραφούν, εντούτοις όμως αυτό που συστήνεται είναι να τους κρατάτε για μερικές ημέρες σε περίπτωση που υπάρχουν προβλήματα και επομένως επανεκκινήστε τον υπολογιστή σας από έναν παλαιότερο πυρήνα όπως απαριθμείται στην grub screen

Old kernels can be removed from the system. To do so, install kernel-remover:

apt-get update
apt-get install kernel-remover
