Ρέουσες (rolling) Σημειώσεις

Το sidux προσπαθεί να παρέχει ένα απλό μονοπάτι αναβάθμισης του debian και των πακέτων του sidux, όμως μένουν μερικά πράγματα τα οποία θα μπορούσαν να βελτιωθούν κάνοντας ορισμένες αλλαγές χειροκίνητα.

Η σελίδα αυτή θα ενημερώνεται μόνο μετά από κάθε έκδοση (release). Τρέχουσα έκδοση: Πόντος.

sidux προειδοποίηση ασφαλείας / κλείδωμα του $HOME

Κατά την διάρκεια "καθαρισμού" του εγκαταστάτη του sidux, ανακαλύφθηκε ότι ο εγκαταστάτης αφήνει ένα αντίγραφο του αρχείου ρύθμισης του πάνω στο σύστημα. Το αρχείο αυτό περιέχει MD5 password hashes για τον root και τον πρώτο χρήστη (UID 1000) και μπορεί να διαβαστεί από τον χρήστη. Αν και αυτό δεν αποκαλύπτει τα passwords απευθείας, μπορεί να χρησιμοποιηθεί για μια brute force επίθεση εναντίων τους.

Εκδόσεις που επηρεάζονται:

  • 2007-01 Χάος (Chaos)
  • 2007-02 Τάρταρος (Tartaros)
  • 2007-03/ 2007-03.1 Γαία (Gaia)
  • 2007-04/ 2007-04.5 Έρως (Eros)
  • 2008-01 Νυξ (Nyx)
  • 2008-02 Έρεβος (Erebos)
  • 2008-03 Ουρέα (Ourea)

Λύση:

find /root /home -maxdepth 2 -type f -name '.sidconf*' -delete

Ειδικά σε συστήματα πολλών χρηστών, συνιστάται η αλλαγή του password του root και του πρώτου χρήστη (UID 1000). Αυτό μπορεί να γίνει με την χρήση της εντολής "passwd" σαν root/χρήστης. Ο λόγος που έγινε αυτό έχει ανακαλυφθεί και η sidux 2008-04 "Πόντος" δεν επηρεάζεται.

Το Debian δημιουργεί καταλόγους home με δικαιώματα πρόσβασης 0751 , το sidux αλλάζει αυτή την προεπιλογή και επιβάλλει 0700 για νέες εγκαταστάσεις. 

sed -i 's/^DIR_MODE=.*/DIR_MODE=0700/' /etc/adduser.conf
dpkg-reconfigure adduser
(απαγόρευση πρόσβασεις στους home φακέλους άλλων χρηστών)

ALSA OSS compatibility layer

Ξεκινώντας με τον alsa-driver 1.0.17.dfsg-3, τα modules συμβατότητας με το OSS δεν φορτώνονται πλέον εκ προεπιλογής. Αν και αυτό δεν θα πρέπει να δημιουργήσει οποιαδήποτε προβλήματα σε σύγχρονα linux συστήματα, μερικές εμπορικές ή legacy εφαρμογές, μπορεί να χρειάζονται ακόμη υποστήριξη OSS. Με δεδομένο ότι το OSS emulation του ALSA μέσα στον πυρήνα είναι αρκετά περιορισμένο (δεν υπάρχει mixing βασισμένο στο λογισμικό και έτσι μόνο μια εφαρμογή μπορεί να έχει ανά πάσα στιγμή πρόσβαση στο /dev/dsp), είναι προτιμότερη η χρήση εφαρμογών που είναι συμβατές με το ALSA αλλά αν χρειάζεστε οπωσδήποτε συμβατότητα με το OSS, υπάρχουν δυο εναλλακτικές:

  • aoss wrapper
    Το /usr/bin/aoss είναι μέρος του alsa-oss πακέτου και μπορεί να "τυλιχτεί" (wrap) γύρω από legacy εφαρμογές για να "παγιδεύσει" (trap) τις κλίσεις προς το OSS και να "συνδέσει" το ABI με το ALSA. Αυτό συνήθως σας επιτρέπει να εκμεταλλευτείτε τα περισσότερα χαρακτηριστικά του ALSA.
  • in-kernel ALSA emulation
    καλύτερη binary συμβατότητα (δημιουργούνται emulated /dev/dsp, /dev/audio και /dev/mixer συσκευές), αλλά αυτό δεν επιτρέπει mixing βασισμένο στο λογισμικό και έτσι μόνο μια εφαρμογή ήχου μπορεί να λειτουργεί ανά πάσα στιγμή.

Για τις περισσότερες εφαρμογές είναι προτιμότερο να εκκινείτε την επηρεαζόμενη εφαρμογή με "aoss <εφαρμογή>" για να κρατήσετε τα θετικά του mixing βασισμένου στο λογισμικό και τις συγχρόνως προσβάσεις στην κάρτα ήχου, αλλά μπορεί να υπάρξουν σπάνιες περιπτώσεις οι οποίες χρειάζονται το in-kernel OSS compatibility layer. Αυτό μπορεί να ενεργοποιηθεί, αν χρειαστεί, φορτώνοντας τα snd-pcm-oss, snd-mixer-oss και snd-seq-midi modules. 

for i in snd-pcm-oss snd-mixer-oss snd-seq-midi; do
	grep -q "^${i}" /etc/modules  || echo "${i}" >> /etc/modules
	grep -q "^${i}" /proc/modules || modprobe "${i}"
done

Broadcom b43 wlan

Πυρήνες >2.6.25 χρειάζονται αναθεωρήσεις του firmware ≥4.150.10.5 (εσωτερική αναθεώρηση "410.2160 (2007-05-26 15:32:10)") του Broadcom firmware, ενώ παλαιότεροι πυρήνες (μέχρι τον πυρήνα 2.6.24) χρειάζονται 4.80.53.0 (εσωτερική αναθεώρηση "351.126 (2006-07-29 05:54:02)"), το οποίο εγκαταστάθηκε από παλαιότερες αναθεωρήσεις του b43-fwcutter. Το b43-fwcutter δεν μπορεί να αναβαθμίσει υπάρχουσες εικόνες firmware από μόνο του και γιαυτό θα πρέπει να το κάνετε αυτό χειροκίνητα. 

dpkg --purge b43-fwcutter
rm -rf /lib/firmware/b43 /lib/firmware/b43legacy
apt-get update
apt-get install b43-fwcutter

console-setup

Το "console-setup" διαχειρίζεται τώρα τα keymaps της κονσόλας. 

apt-get install console-setup
dpkg-reconfigure console-setup

gspca2

Ο πυρήνας 2.6.27 έρχεται με τον νέο gspca2 οδηγό βασισμένο στο v4l2, ο οποίος αντικαθιστά τον παλιό εξωτερικό gspca οδηγό βασισμένο στο v4l1, ο οποίος χρειάζεται για ένα μεγάλο αριθμό διαφορετικών webcam. Δυστυχώς τα περισσότερα προγράμματα που χρησιμοποιεί ο χρήστης δεν αναγνωρίζουν το νέο v4l2 API για συσκευές που προηγουμένως δούλευαν με τον gspca. Προς το παρόν τα προγράμματα αυτά πρέπει να εκκινούνται με την χρήση "LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <εφαρμογή>". 

apt-get update
apt-get install libv4l-0

AS USER
$ LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <εφαρμογή>

insserv

insserv σε συστήματα που εγκαταστάθηκαν πριν την Πόντος 2008-04

Προειδοποίηση: Παλαιότερες εγκαταστάσεις δεν θα πάρουν το insserv αυτόματα, λόγω των initscripts ή symlinks του χρήστη. ΔΕΝ συνιστάται η εγκατάσταση του insserv σε εγκαταστάσεις παλαιότερες της Πόντος. Το πακέτο αυτό θα πρέπει να χρησιμοποιείται με προσοχή γιατί λανθασμένες ή ανύπαρκτες LSB εξαρτήσεις υπαρχόντων initscripts μπορούν να οδηγήσουν σε σύστημα που να μην εκκινεί. Η εγκατάσταση του insserv σε ένα υπάρχον σύστημα μπορεί να χρειαστεί μερικές χειροκίνητες αλλαγές.

Διαβάστε τις σημειώσεις τεκμηρίωσης του Debian, και τις σημειώσεις έκδοσης πριν εγκαταστήσετε και ενεργοποιήσετε το insserv.

Απαρχαιωμένα πακέτα

Σαν μέρος τις συνεχούς ανάπτυξης του sidux, ένας αριθμός πακέτων έχει αντικατασταθεί από καλύτερα εναλλακτικά πακέτα ή είναι πλέον περιττά.

  • capi-isdnconf, απαρχαιωμένο.
  • configure-xawtv, απαρχαιωμένο.
  • gprsconnect, αντικαταστάθηκε από umtsmon.
  • ipw2100-firmware, αντικαταστάθηκε από firmware-ipw2x00.
  • ipw2200-firmware, αντικαταστάθηκε από firmware-ipw2x00.
  • modemlink, αντικαταστάθηκε από umtsmon και wvdial.
  • ppp-scripts, αντικαταστάθηκε από umtsmon.

Η πιο κάτω εντολή μπορεί να βρει πακέτα τα οποία δεν είναι διαθέσιμα σε κανένα ενεργό apt αποθετήριο και μπορεί να σας υποδείξει απαρχαιωμένα πακέτα. Προσοχή όμως στα πακέτα που εγκαταστάθηκαν χειροκίνητα. Το apt-show-versions είναι ξεχωριστό πακέτο και δεν είναι κατ' ανάγκη προ εγκατεστημένο. 

$ apt-show-versions | awk '/No available version in archive/{print $1}'
