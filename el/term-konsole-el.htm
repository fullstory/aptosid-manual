Oρισμός του τερματικού/κονσόλα

Ένα τερματικό, επίσης αποκαλούμενο κονσόλα και στο KDE konsole, είναι ένα πρόγραμμα το οποίο καθιστά δυνατή την απευθείας αλληλεπίδραση με το λειτουργικό σύστημα Linux με την έκδοση διαφόρων εντολών οι οποίες στη συνέχεια εκτελούνται άμεσα. Επίσης συχνά γίνεται αναφορά σε αυτό ως 'κέλυφος' ή 'γραμμή εντολών'. Το τερματικό είναι ένα πολύ δυνατό εργαλείο και η απόκτηση βασικής κατανόησης της χρήσης του αξίζει την προσπάθεια.

Στο sidux μπορείτε να βρείτε το τερματικό/konsole κοντά στο K-menu με σύμβολο μια οθόνη ηλεκτρονικού υπολογιστή. Εξαρτώμενο από το θέμα (theme) σας ενδέχεται ή όχι να περιλαμβάνει την φωτογραφία ενός κελύφους. Θα βρείτε επίσης το ίδιο εικονίδιο στο K-menu και στην κατηγορία "System".

Όταν ανοίγετε ένα παράθυρο τερματικού θα παρουσιαστεί μπροστά σας το terminal prompt το οποίο θα έχει την παρακάτω μορφή:

όνομαχρήστη@όνομαοικοδεσπότη(hostname):~$

Το όνομα χρήστη θα πρέπει να είναι το ίδιο με το όνομα που χρησιμοποιείτε όταν κάνετε login. Το ~ (tilde) δείχνει ότι είστε μέσα στον home φάκελο σας και το $ δείχνει ότι είσαστε συνδεδεμένος σαν απλός χρήστης. Στο τέλος θα έχετε τον κέρσορα σας. Αυτή είναι η γραμμή εντολών σας όπου θα δώσετε τις εντολές που θέλετε να εκτελέσετε.

Πολλές εντολές πρέπει να εκτελεστούν με δικαιώματα διαχειριστή (root). Για να το επιτύχετε αυτό, γράφετε στο prompt:sux και πατάτε enter. Έδω θα σας ζητηθεί το password διαχειριστή. Πληκτρολογείστε το password και πατήστε enter (Σημειώστε ότι όταν γράφετε το password δεν θα φαίνεται τίποτα στην οθόνη).

Αν το password είναι σωστό το prompt θα αλλάξει σε:

διαχειριστής@όνομαοικοδεσπότη:/home/όνομαχρήστη#

ΠΡΟΣΟΧΗ: Όσο είστε συνδεδεμένος σαν διαχειριστής, το σύστημα δεν θα σας αποτρέψει από το να κάνετε πράγματα που μπορεί να προκαλέσουν ζημία όπως να σβήσετε σημαντικά αρχεία κτλ., πρέπει να είστε απόλυτα σίγουρος για το τί κάνετε γιατί είναι πολύ πιθανών να προκαλέσετε σοβαρή ζημία στο σύστημα σας.

Σημειώστε ότι το $ έγινε # (hash). Σε ένα τερματικό/κονσόλα το # δείχνει πάντα ότι είστε συνδεδεμένος με δικαιώματα διαχειριστή. Σε όλο το εγχειρίδιο θα παραλείψουμε τα πάντα μπροστά από το $ ή το #. Οπότε μια εντολή όπως την παρακάτω:

# apt-get install something

Σημαίνει: ανοίξτε το τερματικό, γίνετε διαχειριστής (sux) και δώστε την εντολή στο # prompt. (Μην γράψετε το #)

Κάποτε μια κονσόλα ή/και ένα τερματικό μπορεί να καταστραφούν, γράψετε:

reset

και πατήστε το enter.

Αν η έξοδος (output) μιας κονσόλας ή ενός τερματικού φαίνεται διαστρεβλωμένη (distorted), μπορείτε (συνήθως) να το φτιάξετε το πρόβλημα πατώντας ctrl+l, το οποίο αποκαθιστά το παράθυρο του τερματικού. Αυτή η διαστρέβλωση συμβαίνει κυρίως όταν δουλεύετε με προγράμματα που χρησιμοποιούν το ncurses interface, όπως το irssi

Κάποτε μια κονσόλα ή/και ένα τερματικό μπορεί να φαίνονται παγωμένα, δεν είναι όμως και οτιδήποτε γράψετε θα εκτελεστεί. Αυτό μπορεί να συμβεί αν πατήσετε κατά λάθος ctrl+s . Σε αυτή την περίπτωση δοκιμάστε να πατήσετε ctrl+q για να ξεκλειδώσετε το τερματικό.

Σχετικά με το sux

Πολλές εντολές πρέπει να εκτελεστούν με δικαιώματα διαχειριστή (root). Για να το επιτύχετε αυτό γράφετε:

sux

Ενώ η κοινή εντολή για να γίνετε διαχειριστής είναι η 'su' , η χρήση της sux θα σας επιτρέψει να τρέξετε και GUI / X11 εφαρμογές από την γραμμή εντολών και επιτρέπει στον διαχειριστή να τρέξει γραφικές εφαρμογές, μιας και η sux είναι τυλιγμένη (wrapped) γύρω από την κοινή εντολή su η οποία θα μεταφέρει τις X πιστοποιήσεις σας στον χρήστη.

Παραδείγματα εκτέλεσης μιας X11 εφαρμογής μέσω της sux είναι η χρήση ενός επεξεργαστή κειμένου όπως το kwrite ή το kate για την επεξεργασία ενός αρχείου διαχειριστή, να κάνετε κατατμήσεις με το gparted ή η εκτέλεση ενός Xapp διαχειριστή αρχείων όπως το dolphin or thunar.

Ορισμένες KDE εφαρμογές χρειάζονται την λέξη dbus-launch μπροστά από το όνομα της εφαρμογής:

dbus-launch <Εφαρμογή>
KDE επιλογές πληκτρολογίου

Για να τρέξετε το krunner στο KDE:

Alt+F2

ή δεξί κλικ στην επιφάνεια εργασίας και επιλογή του:

Xfce επιλογές πληκτρολογίου

Για να τρέξετε το Run Command στο Xfce:

Επιλογές για άλλους διαχειριστές επιφάνειας εργασίας

Μια keyboard άλλη επιλογή η οποία δουλεύει και σε άλλους διαχειριστές επιφάνειας εργασίας είναι:

Alt+F2

μετά:

su-to-root -X -c <Εφαρμογή>

Τις πιο πάνω επιλογές μπορείτε να τις τρέξετε μέσα σε ένα τερματικό.

Το sudo δεν υποστηρίζεται

Το sudo δεν είναι ενεργοποιημένο εκ προεπιλογής σε μια εγκατάσταση στον σκληρό δίσκο. Μπορείτε να το χρησιμοποιήσετε όσο είστε μέσα σε live περιβάλλον (Live-ISO) γιατί εκεί δεν έχει οριστεί κάποιο root password. Ο λόγος γιαυτό είναι ο εξής: αν κάποιος αποκτήσει το password του χρήστη, δεν θα έχει αμέσως πρόσβαση με δικαιώματα υπέρ-χρήστη και δεν θα μπορέσει έτσι να κάνει ζημιές στο σύστημα σας.

Ένα άλλο πρόβλημα με το sudo είναι ότι τρέχει μια root εφαρμογή με τις ρυθμίσεις του χρήστη πράγμα που μπορεί να αλλάξει είναι να παρακάμψει κάποια δικαιώματα. Σε μερικές περιπτώσεις αυτό μπορεί να έχει σαν αποτέλεσμα την αδυναμία εκτέλεσης της εφαρμογής από τον απλό χρήστη. Συστήνουμε την χρήση τουsux, kdesu, gksu or su-to-root -X -c !

Μέσα σαν root

ΠΡΟΣΟΧΗ: Όσο είστε συνδεδεμένος σαν διαχειριστής, το σύστημα δεν θα σας αποτρέψει από το να κάνετε πράγματα που μπορεί να προκαλέσουν ζημία όπως να σβήσετε σημαντικά αρχεία κτλ., πρέπει να είστε απόλυτα σίγουρος για το τί κάνετε γιατί είναι πολύ πιθανών να προκαλέσετε σοβαρή ζημία στο σύστημα σας.

Σε καμία περίπτωση δεν πρέπει να είστε συνδεδεμένος σαν διαχειριστής μέσα στο τερμάτικο/κονσόλα και να τρέχετε εφαρμογές που χρησιμοποιεί ένας απλός χρήστης για να κάνει τις καθημερινές του εργασίες όπως το να στέλνει email, να δημιουργεί λογιστικά φύλλα, να σερφάρει στο διαδίκτυο και ου το καθ' εξής.

Βοήθεια στην γραμμή εντολών

Ναι υπάρχει. Οι περισσότερες εντολές/προγράμματα Linux έρχονται με το δικό τους εγχειρίδιο που ονομάζεται "man page" ή "manual page", προσβάσιμο από την γραμμή εντολών. Η σύνταξη είνα:

$ man "έντολη"

ή

$ man -k <λέξη_κλειδί>

Αυτό θα ανοίξει την σελίδα του εγχειριδίου για την συγκεκριμένη εντολή. Περιηγηθείτε πάνω και κάτω με τα βελάκια του πληκτρολογίου. Σαν παράδειγμα δοκιμάστε:

$ man apt-get

Για να βγείτε από το εγχειρίδιο πατήστε το q

Μια άλλη χρήσιμη εντολή είναι η "apropos". Η Apropos βασικά σας επιτρέπει να αναζητήσετε μέσα στο εγχειρίδιο για μια εντολή αν π.χ. δεν θυμάστε ακριβώς πώς γράφετε. Για παράδειγμα μπορείτε να δοκιμάσετε:

$ apropos apt-

Αυτό θα σας δώσει μια λίστα με όλες τις εντολές για τον διαχειριστή πακέτων 'apt'. Η εντολή 'apropos' είναι ένα αρκετά ισχυρό εργαλείο, άλλα η λεπτομερής περιγραφή του είναι εκτός των σκοπών αυτού του εγχειριδίου. Για λεπτομέρειες για το πως μπορείτε να την χρησιμοποιήσετε, κοιτάξτε το man page της.

Scripts και πώς να τα χρησιμοποιήσετε

Ένα shell script είναι ένας βολικός τρόπος για να ομαδοποίηση πολλών εντολών μέσα σε ένα αρχείο. Πληκτρολογώντας το όνομα του script και πατώντας enter, εκτελούνται μια μια η εντολές με την σειρά. Το sidux έρχεται με πολλά χρήσιμα scripts για να κάνει την ζωή του χρήστη ευκολότερη.

Εάν το shell-script βρίσκεται στον φάκελο στον οποίο δουλεύετε τώρα (current working directory):

./όναμα_του_shell_script

Κάποια scripts χρειάζονται δικαιώματα διαχειριστή (sux) μέσα σε ένα τερματικό για να τρέξουν και άλλα δεν χρειάζονται. Εξαρτάται εντελώς από τον σκοπό του script.

Εγκατάσταση ενός script και διαδικασία εκτέλεσης

Χρησιμοποιείστε το wget για να κατεβάσετε το αρχείο script, βάλτε το οπουδήποτε σας συνέστησαν να το βάλετε (για παράδειγμα μπορεί να σας ζητήσει να το βάλετε στο /usr/local/bin) . Μπορείτε να χρησιμοποιήσετε το ποντίκι σας για να κάνετε αντιγραφή (copy) και επικόλληση (paste) το όνομα του αρχείου μέσα στην κονσόλα σας αφού προηγουμένως έχετε συνδεθεί σαν sux

Παράδειγμα χρήσης του wget που χρειάζεται δικαιώματα διαχειριστή (sux) 
sux
cd /usr/local/bin
wget όνομα_script

Έπειτα θα πρέπει να κάνετε το αρχείο εκτελέσιμο (executable)

chmod +x  όνομα-script

Μπορείτε επίσης να χρησιμοποιήσετε έναν browser για να κατεβάσετε ένα αρχείο script και μετά να το τοποθετήσετε εκεί που σας συνέστησαν. Ωστόσο θα πρέπει και πάλι να το κάνετε εκτελέσιμο όπως πιο πάνω.

Παράδειγμα χρήσης του wget σαν απλός χρήστης

Για να τοποθετήσετε ένα αρχείο στον $HOME φάκελο σας σαν χρήστης '$':

$wget http://bluewater.sidux.com/shell-script-test/test-script.sh

$chmod +x test-script.sh

Για να τρέξετε ένα script, ανοίξτε ένα τερματικό/κονσόλα και τρέξτε το όνομα του script:

$./test-script.sh

Θα πρέπει μετά να δείτε αυτό:

Congratulations user
You successfully downloaded and executed a bash script!
