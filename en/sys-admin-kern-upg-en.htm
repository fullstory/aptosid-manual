Installing new kernels

The sidux kernels reside here. Please choose your closest mirror:

http://debian.tu-bs.de/project/sidux//kernel Germany
http://sidux.com/files//kernel Germany
http://sidux.net/files//kernel Germany
http://www.mirrorservice.org/sites/sidux.com/sidux/kernel United Kingdom
http://ftp.heanet.ie/pub/sidux//kernel Ireland
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernel  TN, USA
ftp://ftp.spline.de/mirrors/sidux/kernel/ Germany
http://lug01.eecs.wsu.edu/sidux/kernel Pullman WA, USA
http://sidux.2wl.ru/downloads/kernel Russia
http://ftp.tw.debian.org/pub/Sidux//kernel Asia
http://mirror.aarnet.edu.au/pub/sidux/kernel Australia
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernel South Africa
ftp://debian.co.nz/Sidux/kernel New Zealand

The kernels are available in the following forms:

  • smp - Kernel for the i686 Processor Family with single/dual core or more CPUs
  • 64-smp - Kernel for the AMD64- Processor Family, when using 64 bit sidux

Note down the kernel name you require for your processor (we call this archive kernel.zip, but its going to have a name like: kernel-2.6.22.3-rc1-slh-smp-2.zip ), make a note of it then:

Its 6 lines to do it manually in a konsole as user:
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.22.3-rc1-slh-smp-2.zip
unzip kernel-2.6.22.3-rc1-slh-smp-2.zip -d kernel-2.6.22.3-rc1-slh-smp-2
cd kernel-2.6.22.3-rc1-slh-smp-2
./install-kernel-sidux.sh (it will ask for your root password)

Now restart your computer. The new kernel will boot up by default. Should the new kernel give problems, you can reboot and choose an older kernel try this stage again.

To check the modules loaded into the kernel:

ls /sys/module/
or
cat /proc/modules

IMPORTANT: if you need certain driver modules, e.g. for wlan,you have to install the proper .deb packages for them, e.g., if using madwifi, you have to install the .deb package for madwifi: 

dpkg -i madwifi-modules*.deb

See also getting on line and ndiswrapper and for WIFI

The kernel-update is now done. Should you require them, add the latest graphics driver which can be installed by referring to 3D drivers, If you have an ATI or NVIDIA card, the first reboot to the new kernel should be made to runlevel 3 (this is done by adding 3 to the Grub-Boot-Line)

The semi automated method to install kernels is to use smxi :

apt-get update
apt-get install sidux-scripts
smxi

Removing old kernels

After successfull installation of the new kernel, the kernel.zip (and the kernel archive) can be deleted, however its recommended that you keep them for a few days in case you strike problems and therefore boot to an older kernel as listed in the grub screen

Old kernels can be removed from the system. To do so, you can use the script kernel-remover which is in the package sidux-kernelhacking! This script allows to you nominate which kernels you wish to remove.
