Live-cd upgrade

The facility to "live-cd' upgrade a sidux installation does not exist. Before proceeding with installing sidux BACK-UP YOUR DATA! including your bookmarks and emails.

It is highly recommended that you have a separate data partition. The benefits in terms of disaster recovery, stability of your data are unmeasurable.

Therefore your $HOME becomes a place where basic application configurations are kept. or to put it another way, a container for applications to store their settings.

ALWAYS BACK-UP YOUR DATA including your bookmarks and emails!

Hard Disk Installed Dist-Upgrade

You have upgraded your install with the latest sidux Live Cd and now you want to ensure that you have the latest versions of packages and security updates from here on in.

With sidux its called 'dist-upgrade' and is done over the net

The only supported method to dist-upgrade is as follows:

NEVER EVER do a dist-upgrade nor upgrade whilst in X
Always check Current Warnings on the sidux main web site. These warnings are there for a reason because of the nature of unstable and updated on a daily basis.
Log out of KDE. 
Go to Textmode by doing Ctrl+Alt+F1 
logon as root, and then in type init 3 
apt-get update
apt-get dist-upgrade
run fix-fonts
run get-sidux-binary-gfx to reinstall grahpics drivers if xorg updated and you have such a graphics adapter
otherwise do
init 5 && exit
NEVER DIST-UPGRADE [or UPGRADE] with adept, synaptic or aptitude SEE Upgrade of an Installed System ] There is one exception!! du-fixes-h2, now called smxi, which is short for sidux-maintenance:
apt-get update
apt-get install sidux-scripts
smxi

For new users, the !h2 smxi script may provide help by semi-automating the dist-upgrade process
