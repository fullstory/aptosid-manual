Actualizando el BIOS con FreeDOS

Puede desear o necesitar actualizar el BIOS del PC, cuando el fabricante de la placa madre anuncia alguna mejora en el software del BIOS. El programa de instalación es normalmente ofrecido como una aplicación de MS-DOS.

Este us una manera de actualizar el BIOS desde un USB en Linux. Esto funciona con llaves, palitos, y con los tarjetas micro/mini/SD (con el adaptador adecuada).

Primero, el BIOS tiene que permitir inicialización desde USB -y ser compatible con discos duros USB. Algunas formaciones de BIOS solamente aceptan USB, floppies, CD-ROMs o diapositivos ZIP, aunque útil, puede hacer más dificil el actualización; sin embargo, puede ser que no tiene otra opción (en particular los netbooks).

Necesitan tres cosas:

  1. Un palito USB, preferiblemente de <2 GB (FAT16 no permite más de 2 GB), y una instalación completo de FreeDOS sola necesita 5.8 MB. El FAT16 es el formato recomendada, en vista de que el FAT32 no es detectible por todos los BIOS.
  2. Un medio de instalación de FreeDOS, fdbasecd.iso (8MB) debe ser suficiente.
  3. qemu (# apt-get install qemu), qemu se requiere para el instalador, basicamente el BIOS emulado por qemu hace que el pendrive USB parece a FreeDOS como un disco duro normal, así que puede instalar normalmente (y evita quemar un CD para el FreeDOS).
Esto es absolutamente critico: a ningún momento montar el pendrive USB. Tenga mucho cuidado para selecionar el nodo de diapositivo corecto, de otra manera, todos los datos en el disco equivocado serán borrado sin recurso; por ejemplo, su disco duro principal.

Enchufa el pendrive USB, recuerda, no se monta el pendrive. Revisa el dmesg (las ultimas mensajes, si acaba de enchufarlo), para determinar cual nodo de diapositivo fue asignada al pendrive USB, vamos asumir el /dev/sdb .

Limpia el pendrive USB, todos los datos sera borrado, puede limpiar todo en vez de solamente el primer 16 MB. 

$ sux
Contraseña:
dd if=/dev/zero of=/dev/sdb bs=1M count=16
16+0 records in
16+0 records out
16777216 bytes (17 MB) copied, 2.35751 s, 7.1 MB/s

Iniciando FreeDOS con qemu

Así que el DOS no tiene conocimiento de pendrive USB, necesita una manera de convincer el instalador de FreeDOS a reconocer el pendrive como un disco duro ordinario. En un LiveBoot, el BIOS del sistema lo hace por nosotros, pero tiene que inventar para que el qemu lo reconoce:

como usuario $:
qemu -hda /dev/sdb -cdrom /ruta/al/fdbasecd.iso -boot d

ctrl-alt comutador entre el toma del raton y teclado, permitiendo cambiar entre escritorios para leer de nuevo cualquier instruciones que sale en cada etapa.

Esto se inicia el FreeDOS DC y mapea el pendrive USB el diapositiva en bruto como el disco duro primario maestro (aquí la capabilidad de qemu de eumular el BIOS causa que el pendrive USB parece a DOS como un disco duro normal.) Seleccione el instalador desde el menu de arranque de la instancia de FreeDOS virtualizado:

1) Continua a inicializar FreeDOS desde el CD-ROM
1
enter

Sigue escogiendo el preseleccionada, 1 y/o selecionando Si cuando le pregunta.

La instalador pide que se reinicia - pero no lo haga todavia, hay que corregir dos (2) errores en el instalador de FreeDOS para el "mbr" y el "bootmenu". Tipea la letra n.

Escribiendo un sector de inicio a el pendrive USB

La primer error a corregir es la del "mbr", como sigue:

fdisk /mbr 1

La segunda error a corregir el el "bootmenu" en el nuevo fdconfig.sys, ejecuta:

cd \
edit fdconfig.sys

y modifica la línea de arranque del "command.com" a lo siguiente:

1234?SHELLHIGH=C:\FDOS\command.com C:\FDOS /D /P=C:\fdauto.bat /K set 
(actualmente no cambia este comando, solo anteponer "1234?" al inicio de la línea (antes de  SHELLHIGH==C:\FDOS\command.com .....
NOTA que es de leer: 1234?

No cambiar nada más, porque la línea depende en su configuración de instalación.

Guardalo y salir de "edit":

[alt]+[f]

Una vez de regreso en el prompt de la línea de comando, puede salir de qemu.

Verifica si el qemu arranque el pendrive USB.

qemu -hda /dev/sdb -boot c

El pendrive USB ahora esta iniciable y contiene una completa instalación de FreeDOS de 5.4 MB, para "flashing", escribiendo en el BIOS. Realmente, debe reiniciar sin drivers alguna, (opción del menu, 4), porque cargando los archivos himem.sys y emm386, puede interferir con los utilidades de "flashing" para el BIOS!:

Actualizando el BIOS

Asumiendo que el PC esta ejecutando, encufe el pendrive USB con FreeDOS, montarlo y bajar los archivos de BIOS requerido, como recomendado por el fabricante de la placa madre/BIOS, hacia el pendrive USB con FreeDOS, y desmontar el pendrive.

Apagar el corriente electrico del PC, enchufe el pendrive USB con FreeDOS, y sigue el procedimiento proveedo por el fabricante de la placa madre/BIOS.
