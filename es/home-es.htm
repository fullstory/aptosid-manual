Respaldando la partición /home

Antes de mudar el /home debe respaldar, como raíz, todo debajo del /home: 

tar cvzpf  somewhere/home.tar.gz /home

Para extraer:

tar xvpf somewhere/home.tar.gz

rdiff es un método alternativa para respaldar.

Este puede servir si desea actualizar el /home de una previa instalación a una nueva.

Mudando /home

Mudando /home puede ser realizado de dos maneras: una con le LiveCD y la otra con la línea de comandos, ninguna es difícil.

Debido a que el sistema necesita la information UUID, necesita obtener el UUID de la nueva partición, si no lo tiene notado.

La manera más fácil de agregar este información nueva: comentarlo antes de mudar el /home, en /etc/fstab.

Inicia el PC y en el dialogo del menú de Grub entra 1 y oprima la tecla "Enter". Este lleva a init (runlevel) 1, lo cual es usuario-solitario y el /home corriente no esta montado, entonces es seguro de trabajar con el.

En el tty, login como root y montar el nueva partición de /home, por ejemplo:

mount /dev/sdxX /media/new-home
(or whatever partition the new home is going to be)
cp -pr /home /media/new-home

Proxima, como raíz, edita /etc/fstab para reflejar el cambio a la nueva ubicación de /home: 

mcedit /etc/fstab

Ahora, quita el comento (borrar el #) donde esta el nuevo /home, y comentar (insertar un #) donde esta el viejo /home, guardar con el F2 entonces salir con el F10 y reinicia el PC.
