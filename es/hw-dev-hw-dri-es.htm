Datos para hardware con necesidades no-libres

Debido a la complejidad de la ley, sidux solamente proveerá software Debian dfsg-free.

Drivers de Fuente Abierto para nVidia, ATI, Radeon y Intel

Los drivers de fuente abierto para nVidia, ATI, Radeon y Intel están instalado con sidux.

Intel drivers should work perfectly for 2D and 3D video acceleration as the drivers are included in the Intel free series.

Si estaba usando drivers proprietario, pero desea revertir a drivers de fuente abierto, primero tiene que editar: /etc/X11/xorg.conf . ¡¡SIN EMBARGO, TIENE QUE HACER UN RESPALDO ANTES!! La edición incorrecto del /etc/X11/xorg.conf, resultará en muchas lagrimas por el editor.

Para editar /etc/X11/xorg.conf, clic derecho, Abrirlo con permisos de raíz (root). Busca la sección que dice SECTION DEVICE y cambiarlo a leer nv ó ati Ó intel (to name but a few).

¡¡La edición de "xorg.conf" es enteramente a su riego!!

sources.list

sidux solamente contiene software dfsg-free, probablemente desea agregar "contrib y non-free" al /etc/apt/sources.list.d/debian.list y /etc/apt/sources.list.d/sidux.list o /etc/apt/sources.list y asegurar que tiene acceso a internet:

# Ver sources.list(5) para más información, especialmente
# recuerda que solamente puede usar http, ftp o URIs de archivos
# CDROMs se maneja con el herramienta: apt-cdrom
# El directorio /etc/apt/sources.list.d/ provee una manera de agregue
# entradas a sources.list en archivos separados que terminan con ".list".

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Drivers binarios, de fuente cerrado para: nVidia con dmakms

Preparar el modulo:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Entonces activar "Dynamic Module-Assistant Kernel Module Support" (dmakms) para Nvidia, así que la próxima vez que el kernel de Linux está actualizado, un modulo Nvidia será preparado también, sin intervención manual. Para lograr eso, agregue nvidia-kernel-source a el archivo de configuración del /etc/default/dmakms :

echo nvidia-kernel-source >> /etc/default/dmakms

Entonces:

m-a a-i nvidia-kernel-source

Seguido por:

apt-get install nvidia-glx

Después modificar el "xorg.conf":

sed -i s/nv\"/nvidia\"/ /etc/X11/xorg.conf

Reiniciar el PC para que la instalación del modulo se hará efecto.

Fallo del cargo del Modulo

Si el módulo de Nvidia falla en cargar, para cualquier razón, [nuevo componente de xorg, una problema de la sistema de archivos or si la sistema X no arranca después de una reinicialización]:

modprobe nvidia

Entonces, reinicializar el computadora.

Si el módulo todavia sigue fallando a cargar:

 m-a a-i -f nvidia-kernel-source

Este se reconstruir el módulo , y entonces, reiniciar.

Leer:

$ /usr/share/doc/dmakms

ATI, Radeon and other video drivers

2D Drivers

Los drivers para el x servidor de X.Org (ver xserver-xorg para una descripción más completo) proveas apoyo en 2D para los tarjetas de Nvidia Riva, TNT, GeForce, y Quadro y el ATI Mach, Rage, Radeon, y FireGL cards, junta con los atimisc, r128 y radeon sub-drivers. Radeon and Intel both support 2d acceleration (textured xv) for video playback.

3D Drivers

Some ATI cards also support 3D, so far up to the r5xx chipsets.

To automatically inherit newly packaged non-free firmware when they get updated for 2D and 3D video cards:

apt-get install firmware-linux

Then reboot the computer.

Firmware drivers

El firmware requerida puede ser determinado con el siguiente enumeración del dispositivo / firmware, o a través del script nuevo, fw-detect (empaquetada en sidux-scripts).

$ fw-detect

Es buena practica de hacer: modprobe -r <modulename>, antes de ejecutar: modprobe <modulename>, especialmente para Intel.

Escribiendo en particiones NTFS con ntfs-3g

Precaución: Aunque el driver ntfs-3g es declarada de ser estable, nunca utilizarlo sin realizar un respaldo externo, y por supuesto no en sistemas de producción o de trabajo! Si lo hace, la perdida de data sera su culpa, entonces usarlo a su propio riesgo!

Abre una consola y entra las siguientes comandos: Ver Particionando el DD - Nombramiento del Disco

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para salir del consola, entrar: exit

Ahora, el volumen de NTFS debe ser montado rw (leer/escribir) y debe ser posible de escribir datos en el.
¡Pero, una vez más, Precaución! Usalo solamente para situaciones de emergencia, no esta recomendada para uso diario.
