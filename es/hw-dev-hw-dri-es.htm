nVidia 3d Drivers

Hay 3 métodos de instalar los drivers 3d_nVidia en el sistema

  1. install-binary-gfx

    El nuevo script de instalación de controladores gráficas se le llama: install-binary-gfx. Para conseguir el script, ejecuta get-sidux-binary-gfx una vez, en el konsole como root. Precaución: usa el MÉTODO 1a ó el 1b.

    get-sidux-binary-gfx
install-binary-gfx -a     (para construir sidux de la fuente)

    El script escoge el driver correcto para su tarjeta, construya una deb, instalarlo y automáticamente cambiar el xorg.conf como corresponde. Para el nuevo driver a funcionar, tiene que rearrancar el servidor X (Ctrl+Alt+Backspace).

    Opciones

    Puede ser que hay opciones que deseas activar manualmente:

     Uso: ./install-binary-gfx [opciones]
-a | --auto            Modo Automático
-b | --build           Construye MOD solamente   (nvidia|nvidia-legacy|fglrx)
-d | --detect          Detecta tarjeta solamente
-u | --update          Actualizar Script
-v | --verbose         Verbose

    Ejemplos:

    install-binary-gfx -a         (detecta la tarjeta y instalar drivers de Sid automáticamente.)
install-binary-gfx -b nvidia  (construye el modulo-nvidia....deb en /usr/src. Lo mismo para el fglrx y nvidia-legacy.)
install-binary-gfx -d         (detecta la tarjeta y despliegue la información.)
install-binary-gfx -u         (actualizar este Script. )

    Detección de averías

    Si este Script falla con su tarjeta, por favor ejecutar el script en modo debug y postear la salida tanto como el archivo de registro /var/log/sidux-TARJETA-installer.log, en el foro de Sidux, o utilizar http://paste.debian.net y ir a #sidux para darnos el enlace para recibir ayuda a través del IRC.

  2. sgfxi / du-fixes-h2.sh

    cd /usr/local/bin ; wget -Nc http://techpatterns.com/sgfxi
chmod +x sgfxi

    sgfxi actualmente apoya los drivers de ATI fglrx y Nvidia. También apoya convertiendo hacia o desde los drivers libre de xorg, tales como: ati, intel, ó nv. El script du-fixes-h2.sh utilices el sgfxi como el motor de instalación de drivers graficas, pero simplemente agregue apoyo de algunas opciones para los usuarios.

    Ver el sgfxi -h para una lista completa de opciones. Los opciones serán actualizado sobre la marcha. Cuando utilzar la opción verá también una lista de todos los drivers apoyada al momento, tanto como una listado de las especificaciones del sistema corriente, kernel y versión de xorg. El driver preseleccionada es aquel que se instala si ningún argumentos están dado para cambiarlo. En el caso de tarjetas viejos que requiere un driver legato, el script decidirá el driver apropiado.

    Cuando ejecuta el script, el se pare para decirle que driver será instalado. Puede aceptarlo, o salir y ejecutarlo de nuevo con un parametro, si desea algo diferente.

    O, bajar el !h2, du-fixes-h2.sh, entonces: usarlo para instalar solamente los controladores de 3d

     du-fixes-h2.sh -kiwdt

nVidia X.Org X server - driver nativo para tarjeta nv

Este driver para el servidor X de X.Org (ver xserver-xorg para más información) provea apoyo para tarjetas NVIDIA Riva, TNT, GeForce y Quadro.

Para instalarlo:

Salir de KDE, entrar al modo texto a través
de Ctrl+Alt+F1
login como root y entrar: init 3
(este debe decir que el X no esta ejecutando), entonces:
# apt-get update
# apt-get install xserver-xorg-video-nv
# init 5 && exit

Nota: que este no es el mismo que el driver 'nvidia' que es solamente binario que agregue apoyo 3D, pero es solamente binario y no esta apoyado.

Próxima, tiene que editar /etc/X11/xorg.conf, SIN EMBARGO, TIENE QUE HACER UN RESPALDO PRIMERO, por que una edición incorrecto resultará en muchos lágrimas a su parte.

Para editar /etc/X11/xorg.conf, clic derecho, Abrirlo con permisos de raíz (root). Busca la sección que dice SECTION DEVICE y cambiarlo el driver a leer nv

Monitores

Esta a punto de tirarse de cabeza en archivos de configuración de xorg, por favor, Por Favor, POR FAVOR, haga un respaldo de "/etc/X11/xorg.conf", antes de ir más adelante!

Reconfiguración del ATI con un sólo monitor

Single head :    aticonfig --initial --input=/etc/X11/xorg.conf

Monitores Dual

ATI Nativo - Método 1

http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=19794#19794 tiene algunas xorg.conf que funciona con el driver libre de Radeon.

NOTA: Necesitará la información de la configuración del segundo monitor. Para esto, necesita desenchufar un monitor, arranca con el liveCD para gnererar la información en el xorg.conf, y guardar una copia de esta información, entonces haga los mismos pasos con el otro monitor.

ATI Nativo - Método 2

Con este método, tiene que configurar la resolución del monitor primario, primero, entonces el segundo utilizará el mismo configuración. Así que tiene que asegurar que la resolución para la primera puede ser manejada para la segundo.

En la sección:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

agregue

Option "MergedFB"
Option "CRT2Position" "RightOf"

Para que terminará así:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Para información de configuración completo, ver http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html

NOTA: El enlace arriba utiliza el driver de Radeon, pero es conocido a funcionar con el driver de ATI.

ATI no-libre

get-sidux-gfx

Proximamente, tiene que salir del servidor X:

ctrl+alt+F1

entra su clave de raíz:

Entonces:

aticonfig --dtop=horizontal

Si los monitores terminan fuera de lugar, puede hacerse:

aticonfig --dtop=horizontal,reverse

Por favor, lea la ayuda, lo cual es muy comprensivo, con opciones y ejemplos:

Nota: Ambos monitores necesita tener la misma resolución. Verifica en el manual de los monitores y IGUALAR la resolución más alto de ambos.

Debido a la complejidad de la ley, sidux solamente proveerá software dfsg-free.

sidux solamente contiene software dfsg-free, probablemente desea agregar "contrib y non-free" al /etc/apt/sources.list y asegurar que tiene acceso a internet:

# Ver sources.list(5) para más información, especialmente
# recuerda que sólo puedo usar http, ftp o URIs de archivos.
# CDROMs están manejado a través del herramienta apt-cdrom.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

  • tarjetas de Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan, bajar el firmware desde http://www.kazer.org/acx-firmware-20060207.tar.bz2 y colocarlo en /lib/firmware/.

Escribiendo en particiones NTFS con ntfs-3g

Precaución: Aunque el driver ntfs-3g es declarada de ser estable, nunca utilizarlo sin realizar un respaldo externo, y por supuesto no en sistemas de producción o de trabajo! Si lo hace, la perdida de data sera su culpa, entonces usarlo a su propio riesgo!

Abre una consola y entra las siguientes comandos: Ver Particionando el DD - Nombramiento del Disco

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit

Ahora, el volumen de NTFS debe ser montado rw (leer/escribir) y debe ser posible de escribir datos en el.
