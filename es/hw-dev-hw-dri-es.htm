Datos para hardware con necesidades no-libres

Debido a la complejidad de la ley, sidux solamente proveerá software Debian dfsg-free.

Drivers de Fuente Abierta para nVidia, ATI, Radeon e Intel

Los drivers de fuente abierta para nVidia, ATI, Radeon e Intel vienen instalados con sidux.

Los controladores de Intel deben funcionar perfectamente para aceleración 2D y 3D, puesto que estos drivers están incluido en las series libres de Intel.

Si estaba usando drivers proprietarios pero desea pasar a drivers de fuente abierta, primero tiene que editar: /etc/X11/xorg.conf . ¡¡ATENCIÓN: TIENE QUE HACER UN RESPALDO ANTES!! La edición incorrecta de /etc/X11/xorg.conf puede darle muchos problemas.

Para editar /etc/X11/xorg.conf, ábralo con permisos de root. Busque la sección SECTION DEVICE y cámbielo donde lea nv o ati o intel (para nombrar sólo algunos).

¡¡La edición de "xorg.conf" corre enteramente a su riesgo!!

sources.list

sidux solamente contiene software dfsg-free, por lo que probablemente deseará agregar "contrib" y "non-free" al /etc/apt/sources.list.d/debian.list y /etc/apt/sources.list.d/sidux.list o /etc/apt/sources.list y asegurarse de que tiene acceso a internet:

# Ver sources.list(5) para más información, especialmente
# recuerda que solamente puede usar http, ftp o URIs de archivos
# CDROMs se maneja con el herramienta: apt-cdrom
# El directorio /etc/apt/sources.list.d/ provee una manera de agregar
# entradas a sources.list en archivos separados que terminan con ".list".

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Controladores binarios no libres para nVidia con dmakms

NOTE: Use apt-cache search nvidia y apt-cache show <package> para determinar el controlador correcto para usted. Básicamente son posibles dos elecciones: controladores actuales de 3D para Debian Sid y controladores antiguos de 3D para Debian Sid.

Para los controladores actuales 3d nvidia ≥ GeForce 6xxx :

Prepare el módulo:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Entonces active "Dynamic Module-Assistant Kernel Module Support" (dmakms) para Nvidia, de forma que la próxima vez que el kernel de Linux sea actualizado, un módulo Nvidia será también preparado, sin intervención manual. Para lograr esto, agregue nvidia-kernel-source al archivo de configuración del /etc/default/dmakms :

echo nvidia-kernel-source >> /etc/default/dmakms

Entonces:

m-a a-i nvidia-kernel-source

Seguido por:

apt-get install nvidia-glx

Después modifique el "xorg.conf":

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Reinicie el PC para que la instalación del módulo tenga efecto.

Cuando xorg reciba un update, sólo necesitará reinstalar nvidia-glx:

apt-get install --reinstall nvidia-glx

Cuando haya un nuevo controlador Debian sid nvidia, hágalo accesible mediante apt:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Reinicie el PC para que la instalación del módulo tenga efecto.

Para controladores antiguos 3D nvidia ≥ GeForce 5xxx :
m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Cambie dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Fallo en la carga del Módulo

Si el módulo de Nvidia falla al cargar por cualquier razón [nuevo componente de xorg, un problema del sistema de archivos o si el sistema X no arranca después de una reinicialización]:

modprobe nvidia

A continuació vuelva a iniciar la computadora.

Si el módulo todavia sigue fallando al cargar:

 m-a a-i -f nvidia-kernel-source

Esto reconstruirá el módulo , y entonces reinicie.

Lea:

$ /usr/share/doc/dmakms

Puede también leer nvnews para otras opciones.

ATI, Radeon y otros controladores de Video

2D Drivers

Los controladores para el servidor gráfico X.Org (vea xserver-xorg para una descripción más completa) proporcionan apoyo en 2D para las tarjetas de Nvidia Riva, TNT, GeForce, y Quadro y las tarjetas ATI Mach, Rage, Radeon, y FireGL, junto con las Atimisc, r128 y Radeon sub-drivers. Ambos, Radeon e Intel, soportan aceleración 2d (textured xv) para video.

3D Drivers

Algunas tarjetas ATI además soportan 3D, al menos hasta la serie de chips r5xx.

Para recibir automáticamente nuevo firmware no-libre cuando se actualicen las tarjetas de video de 2D y 3D:

apt-get install firmware-linux

Entonces, reinicie el computador.

Firmware drivers

El firmware necesario puede ser determinado con la siguiente enumeración del dispositivo /firmware, o a través del nuevo script fw-detect (empaquetado en sidux-scripts).

$ fw-detect

Es una buena practica hacer: modprobe -r <modulename>, antes de ejecutar: modprobe <modulename>, especialmente para Intel.

Escribiendo en particiones NTFS con ntfs-3g

Precaución: Aunque el driver ntfs-3g ha sido declarado como estable, nunca lo utilice sin realizar previamente una copia de seguridad externa, y por supuesto no en sistemas de producción o de trabajo! Si lo hace, la pérdida de datos será culpa suya; úselo a su propio riesgo!

Abra una consola y entre los siguientes comandos: (ver Particionando el DD - Nombramiento del Disco)

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para salir del consola, entrar: exit

Ahora, el volumen de NTFS debe ser montado como rw (leer/escribir) y debe ser posible escribir datos en él.
¡Pero, una vez más, Precaución!Úselo solamente para situaciones de emergencia, no está recomendado para un uso diario.
