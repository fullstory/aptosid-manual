nVidia 3d Drivers

Hay 3 métodos de instalar los drivers 3d_nVidia en el sistema

  1. install-binary-gfx

    El nuevo script de instalación de controladores gráficas se le llama: install-binary-gfx. Para conseguir el script, ejecuta get-sidux-binary-gfx una vez, en el konsole como root. Precaución: usa el MÉTODO 1a ó el 1b.

    get-sidux-binary-gfx
install-binary-gfx -a     (para construir sidux de la fuente)

    El script escoge el interprete correcto para su tarjeta, construya una deb, instalarlo y automáticamente cambiar el xorg.conf como corresponde. Para el nuevo interprete a funcionar, tiene que rearrancar el servidor X (Ctrl+Alt+Backspace).

    Options

    Puede ser que hay opciones que deseas activar manualmente:

     Uso: ./install-binary-gfx [opciones]
-a | --auto            Modo Automático
-b | --build           Construye MOD solamente   (nvidia|nvidia-legacy|fglrx)
-d | --detect          Detecta tarjeta solamente
-u | --update          Actualizar Script
-v | --verbose         Verbose
--aiglx                Activar AIGLX

    Ejemplos:

    install-binary-gfx -a         (detecta la tarjeta y instalar drivers de Sid automáticamente.)
install-binary-gfx -a --aiglx (detecta la tarjeta y instalar drivers de Sid automáticamente y activar AIGLX.) 
install-binary-gfx -b nvidia  (construye el modulo-nvidia....deb en /usr/src. Lo mismo para el fglrx y nvidia-legacy.)
install-binary-gfx -d         (detecta la tarjeta y despliegue la información.)
install-binary-gfx --aiglx    (activar AIGLX.)
install-binary-gfx -u         (actualizar este Script. Por favor, note que este no será realizado por el "dist-upgrade" mientras que
 tiene el "contrib" para el repositorio de sidux, activado en el /etc/apt/sources.list.)

    Detección de averías

    Si este Script falla con su tarjeta, por favor ejecutar el script en modo debug y postear la salida tanto como el archivo de registro /var/log/sidux-CARD-installer.log, en el foro de Sidux, o utilizar http://paste.debian.net y ir a #sidux para darnos el enlace para recibir ayuda a través del IRC.

  2. sgfxi / du-fixes-h2.sh

    cd /usr/local/bin ; wget -Nc http://techpatterns.com/sgfxi
chmod +x sgfxi

    sgfxi currently supports ATI fglrx and Nvidia drivers. It also supports converting from or to xorg free drivers like ati, intel, or nv. du-fixes-h2.sh uses sgfxi as its graphics driver installer engine, but simply adds some option support for users.

    See sgfxi -h for complete option list. Options will be updated as time goes on. When you use the -h option you will also see a list of all currently supported drivers, as well as a brief printout of your current system specifications, kernel, xorg version. The default driver is the driver that is installed if no arguments are used to override it. Unless your card is too old, and requires a legacy driver, in which case the script will it determine for you.

    When you run the script, it will stop, and tell you what driver it is going to install. You can accept that, or exit and redo it with an override option if you want something different.

    O, bajar el !h2, du-fixes-h2.sh, entonces: usarlo para instalar solamente los controladores de 3d

     du-fixes-h2.sh -kiwdt

  3. Use Nvidia binary installer

    Instalar desde la fuente.

nVidia X.Org X server - interprete nativo para tarjeta nv

Este driver para el servidor X de X.Org (ver xserver-xorg para más información) provea apoyo para tarjetas NVIDIA Riva, TNT, GeForce y Quadro.

Para instalarlo:

Salir de KDE, entrar al modo texto a través
de Ctrl+Alt+F1
login como root y entrar: init 3
(este debe decir que el X no esta ejecutando), entonces:
# apt-get update
# apt-get install xserver-xorg-video-nv
# init 5 && exit

Nota: que este no es el mismo que el driver 'nvidia' que es solamente binario que agregue apoyo 3D, pero es solamente binario y no esta apoyado.

Próxima, tiene que editar /etc/X11/xorg.conf, SIN EMBARGO, TIENE QUE HACER UN RESPALDO PRIMERO, por que una edición incorrecto resultará en muchos lágrimas a su parte.

Para editar /etc/X11/xorg.conf, clic derecho, Abrirlo con permisos de raíz (root). Busca la sección que dice SECTION DEVICE y cambiarlo el interprete a leer nv

Escribiendo en particiones NTFS con ntfs-3g

Precaución: ntfs-3g es un interprete beta nueva, nunca utilizarlo sin un respaldo externo, y por supuesto no en un sistemas de producción o de trabajo! Si lo hace, la perdida de data sera su culpa, entonces usarlo a su propio riesgo!

Abre una consola y entra las siguientes comandos: Ver Particionando el DD - Nombramiento del Disco

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit
exit

Ahora, el volumen de NTFS debe ser montado rw (leer/escribir) y debe ser posible de escribir datos en el.

Datos para hardware con necesidades no-libres

Debido a la complejidad de la ley, sidux solamente proveerá software dfsg-free.

En vista de que sidux solamente contiene software dfsg-free, probablemente deseas agregar el contrib y non-free al /etc/apt/sources.list como en el ejemplo siguiente; y asegurar que tiene acceso a internet:

# Ver sources.list(5) para más información, especialmente
# recuerda que sólo puedo usar http, ftp o URIs de archivos.
# CDROMs están manejado a través del herramienta apt-cdrom.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • ATi Radeon gráficas: aceleración 3D para tarjetas más viejos hasta el r35x, deben funcionar, tarjetas de Radeon X1xxx más nuevo necesitan drivers no-libres para funcionamiento acelerado, por favor usa el get-sidux-binary-gfx para bajar scripts de instalación para estas tarjetas - apoyo inicial para el kernel 2.6.20 ha sido incorporado y necesita ser comprobado (puede que no funciona bajo todos las circunstancias con el kernel 2.6.20, por eso hay el kernel 2.6.19.x disponible en nuestras servidoras, todavía.)
  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • AVM ISDN/ ADSL PCI/ USB Karten: el driver de fuente cerrado de AVM no está compatible con el kernel 2.6.19.x hasta ahora, por favor baja el kernel 2.6.18.x (http://sidux.net/files/kernel/) por ahora y utiliza el: 
     m-a a-i avm.
  • Broadcom/ bcm43xx wlan: 
    apt-get install bcm43xx-fwcutter.
  • Eagle USB ADSL modem: baja el firmware desde http://eagle-usb.org/ueagle-atm/non-free/ y colocarlo en /lib/firmware/.
  • DVB firmwares para varios tarjetas de TV DVB con todos los carácteristicas (la mayoría de las tarjetas más económicas no necesitan esto): bajar el firmware requerida (verifica en el dmesg) desde http://www.linuxtv.org/downloads/firmware/ y ubicarlo en el /lib/firmware/.
  • tarjetas de hostap basado en 11 MBit/s wlan con firmware cargable (p.e., D-Link DWL-520 rev. E1 y otros): http://www.oakcourt.dyndns.org/~andrew/dwl520e1.html
  • tarjetas de Intel ipw2100, 11 MBit/s wlan: bajar el firmware desde http://ipw2100.sf.net/ y ubicarlo en /lib/firmware/.
  • tarjetas de Intel ipw2200, 54 MBit/s wlan: bajar el firmware desde http://ipw2200.sf.net/ y ubicarlo en /lib/firmware/.
  • tarjetas de Intel ipw3945, 54 Mbit/s wlan: apt-get install ipw3945d firmware-ipw3945 
    apt-get install ipw3945d firmware-ipw3945
  • tarjetas de Intersil prism54, 54 MBit/s wlan: bajar el firmware desde http://prism54.org/firmware/ y colocarlo en /lib/firmware/.
  • nVidia gráficas: aceleración de 3D no es posible con drivers libres todavía, favor utilizar get-sidux-binary-gfx para bajar scripts de instalación para estas tarjetas (quizás este todavía no funciona bajo todos las circunstancias con el kernel 2.6.20, por eso kernel 2.6.19.x están todavía disponible en nuestras servidoras).
  • RaLink rt61 54 MBit/s wlan, bajar el firmware desde http://www.ralinktech.com/ralink/Home/Support/Linux.html
  • tarjetas de Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan, bajar el firmware desde http://www.kazer.org/acx-firmware-20060207.tar.bz2 y colocarlo en /lib/firmware/.
  • tarjetas de ZyDAS zd1201 11 MBit/s wlan: 
     apt-get install zd1201-firmware
  • tarjetas de ZyDAS zd1211 54 MBit/s wlan: bajar el firmware desde http://sourceforge.net/project/showfiles.php?group_id=129083 y ubicarlo en /lib/firmware/.

NOTA: Estamos verificando si podemos proveer paquetes para algunas de estos dispositivos, pero el estado legal no es necesariamente fácil de entender.
