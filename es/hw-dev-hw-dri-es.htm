Drivers no-libres, binarios para: Nvidia / ATI

Hay 4 maneras de instalar los drivers 3d al sistema:

  1. install-binary-gfx

    El nuevo script de instalación de controladores gráficas se le llama: install-binary-gfx. Para conseguir el script, ejecuta get-sidux-binary-gfx una vez, en el consola como root. Precaución: usa el MÉTODO 1a ó el 1b.

    get-sidux-binary-gfx
install-binary-gfx -a  (para construir sidux de la fuente)

    El script escoge el driver correcto para su tarjeta, construya una deb, instalarlo y automáticamente cambiar el xorg.conf como corresponde. Para el nuevo driver a funcionar, tiene que rearrancar el servidor X (Ctrl+Alt+Backspace).

    Opciones

    Puede ser que hay opciones que deseas activar manualmente:

     Uso: ./install-binary-gfx [opciones]
-a | --auto            Modo Automático
-b | --build           Construye MOD solamente (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          Detecta tarjeta solamente
-u | --update          Actualizar Script
-v | --verbose         Verbose

    Ejemplos:

    install-binary-gfx -a         (detecta la tarjeta y instalar drivers de Sid automáticamente.)
install-binary-gfx -b nvidia  (construye el modulo-nvidia....deb en /usr/src. Lo mismo para el fglrx y nvidia-legacy.)
install-binary-gfx -b ati     (construye el modulo-ati....deb en /usr/src. Lo mismo para el fglrx y ati-legacy.)
install-binary-gfx -d         (detecta la tarjeta y despliegue la información.)
install-binary-gfx -u         (actualizar este Script. )

    Detección de averías

    Si este Script falla con su tarjeta, por favor ejecutar el script en modo debug y postear la salida tanto como el archivo de registro /var/log/sidux-TARJETA-installer.log, en el foro de Sidux, o utilizar http://paste.debian.net y ir a #sidux para darnos el enlace para recibir ayuda a través del IRC.

  2. sgfxi / !h2 smxi (mantenimiento de sidux)

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxi actualmente apoya los drivers de ATI fglrx y Nvidia. También apoya convertiendo hacia o desde los drivers libre de xorg, tales como: ati, intel ó nv. El script !h2 smxi utilices el sgfxi como el motor de instalación de drivers gráficas, pero simplemente agregue apoyo de algunas opciones para los usuarios.

    El driver preseleccionada es aquel que se instala si ningún argumentos están dado para cambiarlo. En el caso de tarjetas viejos que requiere un driver legato, el script decidirá el driver apropiado..

    Cuando ejecuta el script, el se pare para decirle que driver será instalado. Puede aceptarlo, o salir y ejecutarlo de nuevo con un parametro, si desea algo diferente.

    También puede ejecutar el instalador sgfxi utilizando el !h2 smxi, lo cual usa el sgfxi para instalar los drivers gráficas. Primera, lea el !h2 smxi página principal para más información, entonces, ejecutar este comando:

    smxi -kiwdt

    sgfxi-h: Ver el sgfxi -h para una lista completa de opciones. Los opciones serán actualizado sobre la marcha. Cuando utilizar la opción verá también una lista de todos los drivers apoyada al momento, tanto como una listado de las especificaciones del sistema corriente, kernel y versión de xorg. Aquí hay unas opciones util para sgfxi:

    -c Usa el modo Compuesto (solamente tarjetas de nVidia).
-d Remover "IgnoreDisplayDevices", por ejemplo para permitir TV (solamente tarjetas de nVidia).
-h Ver este menú de ayuda.
-n Automáticamente instalar el driver correcto de xorg nv nativo, intel, o ati, para su sistema.
   Limpia viejo drivers binarios, actualiza xorg.conf con la nueva información del nuevo driver,
	 y limpia información viejo de nvidia y fgrlx del xorg.conf
-N Fuerza la instalación de un driver especifica de xorg.
Sintaxis: sgfxi -N < una de los drivers de la lista, tales como,  -N radeon >
   Drivers apoyado de Xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   También configurar xorg.conf según sea requerida. Configuración de radeon xorg 3d automáticamente.
-o Sobre llevar el driver preseleccionada con una especifica, es un sobrepuesto absoluto
   - syntax:   -o 9746 (para nVidia, no se usa el parte  1.0- ) o:  -o 8.32.5 (para fglrx)
-r  Remover las líneas de modos del xorg.conf
-t  Activar la VistaGemela dinámica (solamente nVidia)

  3. Para instalación en el liveCD de un driver no-libre:

    sudo sgfxi

    Este consigues el script actualizada, entonces:

    ctrl+alt+f1

    entonces, ejecutar

    sgfxi

    sgfxi -h,
para ver más opciones

    X es rearrancada despues que el driver esta instalada

Drivers libres y gratis para nVidia, ATI, Radeon y Intel

  1. sgfxi

    También puede usar el sgfxi para mover entre drivers libre y no-libre; o para desactualizar a drivers nativos libres: Comandos claves son:

    -n		Automáticamente instalar el driver nativo correcto de xorg nv, intel, o ati, para su sistema.
			Limpia data de drivers binarios viejos, actualizar xorg.conf con la información del nuevo driver,
			y limpia data de nvidia y fgrlx del xorg.conf
-N		Fuerza la instalación de un driver especifica para xorg.
Sintaxis: sgfxi -N < una de los drivers de la lista, tales como -N radeon >
		Drivers apoyado de Xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
		También configurar el xorg.conf como sea requerida. Configuración completa y automática de radeon xorg 3d.

    Primero:

    apt-get update
apt-get install sidux-scripts
sgfxi

    Entonces, para instalar o revertir a un driver nativo

    sgfxi -n
ó
sgfxi -N <xorg driver>

  2. sgfxi / !h2 smxi

    También puede instalar sgfxi a través del !h2 smxi. Solamente arranca el script con las opciones abajo indicado y ir directo a la sección de instalación de gráficas. Si está usando smxi, no necesita instalar el sgfxi, porque el smxi-h2 lo instalará automáticamente.

    !h2 smxi página web principal para más información, entonces ejecuta este comando:

    smxi -kiwdt

Resoluciones y Monitores

Cambiando la Resolución de la Pantalla

En una consola como root:

change-res 1024x768
ó  1600x1200 1400x1050 1280x1024 1152x864 800x600

fix-res 1024x768
ó 1600x1200 1400x1050 1280x1024 1152x864 800x600

Re-arranque el X, por hacer:

ctrl+alt+backspace (Ctrl-Alt-BorraAtras)

Monitores

Esta a punto de tirarse de cabeza en archivos de configuración de xorg, por favor, Por Favor, POR FAVOR, haga un respaldo de "/etc/X11/xorg.conf", antes de ir más adelante!

Todo esto puede cambiarse cuando el xorg 7.3 aparece en los últimos meses del 2007.

Reconfiguración del ATI con un sólo monitor

Single head:    aticonfig --initial --input=/etc/X11/xorg.conf

Monitores Dual

ATI Nativo - Método 1

http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=19794#19794 tiene algunas xorg.conf que funciona con el driver libre de Radeon.

NOTA: Necesitará la información de la configuración del segundo monitor. Para esto, necesita desenchufar un monitor, arranca con el liveCD para gnererar la información en el xorg.conf, y guardar una copia de esta información, entonces haga los mismos pasos con el otro monitor.

ATI Nativo - Método 2

Con este método, tiene que configurar la resolución del monitor primario, primero, entonces el segundo utilizará el mismo configuración. Así que tiene que asegurar que la resolución para la primera puede ser manejada para la segundo.

En la sección:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

agregue

Option "MergedFB"
Option "CRT2Position" "RightOf"

Para que terminará así:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Para información de configuración completo, ver http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html

NOTA: El enlace arriba utiliza el driver de Radeon, pero es conocido a funcionar con el driver de ATI.

ATI no-libre

Primeramente, instalar el driver no-libre, (fglrx)

get-sidux-gfx
ó
sgfxi / smxi

Por favor, referirse a "Instalando Drivers en 3D" para instrucciones completas

Proximamente, tiene que salir del servidor X:

ctrl+alt+F1
tipear: root
entra su clave de raíz:
init 3

Entonces:

aticonfig --dtop=horizontal
init 5 && exit

Si los monitores terminan fuera de lugar, puede hacerse:

aticonfig --dtop=horizontal,reverse

Por favor, lea la ayuda, lo cual es muy comprensivo, con opciones y ejemplos:

aticonfig --help

Nota: Ambos monitores necesita tener la misma resolución. Verifica en el manual de los monitores y IGUALAR la resolución más alto de ambos.

Aquí, en http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=20231#20231 puede encontrar información adicional.

Para regresar o desactovar de monitores duales a un monitor sencillo:

aticonfig --initial --input=/etc/X11/xorg.conf

nvidia

Para el nvidia no-libre, utilizar el configurator de xorg para nvidia http://www.sorgonet.com/linux/nv-online/ y modificar, según necesita, los archivos de xorg.

Datos para hardware con necesidades no-libres

Debido a la complejidad de la ley, sidux solamente proveerá software dfsg-free.

sidux solamente contiene software dfsg-free, probablemente desea agregar "contrib y non-free" al /etc/apt/sources.list y asegurar que tiene acceso a internet:

# Ver sources.list(5) para más información, especialmente
# recuerda que sólo puedo usar http, ftp o URIs de archivos.
# CDROMs están manejado a través del herramienta apt-cdrom.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • ATi Radeon gráficas: aceleración 3d para tarjetas más viejos, hasta r35x debe funcionar; tarjetas Radeon más nuevas, como el X1xxx, necesitan drivers no-libre para rendimiento acelerado, los cuales no estan, todavia, compatible con el X.org 7.2. Depending on the particular chipset "xmodule=vesa" or "xmodule=fbdev" as cheatcode might be in order for later Radeon X1xxx models.

  • DVB firmwares para varios tarjetas de TV DVB con todos los carácteristicas (la mayoría de las tarjetas más económicas no necesitan esto): bajar el firmware requerida (verifica en el dmesg) desde http://www.linuxtv.org/downloads/firmware/ y ubicarlo en el /lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945/ 4965 iwlwifi, 54 Mbit/s wlan Please remove iwlwifi-ucode and iwlwifi-3945, if already installed.: 
    apt-get install firmware-iwl3945

    apt-get install firmware-iwl4965

    apt-get update

update-rc.d ipw3945d start 39 S . start 36 0 6 .
LANG= m-a a-i --text-mode ipw3945-source
echo "blacklist iwlwifi" > /etc/modprobe.d/mac80211
  • tarjetas de Intersil prism54, 54 MBit/s wlan: bajar el firmware desde http://prism54.org/firmware/ y colocarlo en /lib/firmware/.
  • nVidia gráficas: aceleración de 3D no es posible con drivers libres todavía, favor utilizar get-sidux-binary-gfx para bajar scripts de instalación para estas tarjetas.
  • RaLink rt61 54 MBit/s wlan: 
    apt-get install rt61-fwcutter
  • rt2x00 2.0.3 support for RaLink cards supporting wpasupplicant, remove /etc/modprobe.d/mac80211 to prefer it over RaLink legacy drivers
  • tarjetas de Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan, bajar el firmware desde http://www.kazer.org/acx-firmware-20060207.tar.bz2 y colocarlo en /lib/firmware/.
  • ZyDAS zd1201 11 MBit/s wlan: 
    apt-get install zd1201-firmware
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Escribiendo en particiones NTFS con ntfs-3g

Precaución: Aunque el driver ntfs-3g es declarada de ser estable, nunca utilizarlo sin realizar un respaldo externo, y por supuesto no en sistemas de producción o de trabajo! Si lo hace, la perdida de data sera su culpa, entonces usarlo a su propio riesgo!

Abre una consola y entra las siguientes comandos: Ver Particionando el DD - Nombramiento del Disco

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit

Ahora, el volumen de NTFS debe ser montado rw (leer/escribir) y debe ser posible de escribir datos en el.
