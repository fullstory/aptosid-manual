Conectándose en línea / como configurar el red

Si tiene un servidor DHCP en el LAN y el computador esta conectado a el mientras se arranca el liveCD, los parámetros del red debe ser configurado automáticamente -si no es así, tiene que ejecutar el dialogo sidux para configuración del red: Pincha el Kmenu >sidux >Network/Internet >Networkcard configuratión.

O -si le gusto la via gráfica, puede abrir una consola, escribiendo:

sudo
netcardconfig

y sigue las instrucciones.

Activando el Ndiswrapper

sidux tiene todos los paquetes requeridos ya instalados: el modulo del kernel y el ndiswrapper-utils.

Es necesario un archivo de su instalación de "Windows", el *.inf file para el interprete, encontrarlo y cargarlo:

ndiswrapper -i driver.inf

carga el modulo del ndiswrapper:

modprobe ndiswrapper

y comprobar si esta instalado correctamente:

iwconfig 
wlan0 essid 'ESSID'

revisa, si el punto de acceso fue encontrado:

iwlist wlan0 scan

entonces, puede configurar la tarjeta con:

netcardconfig

Para PCs de escritorio con 2 NICs (tarjetas de red) (Compartiendo el internet)

Para configurarlo para conectarse al internet con un laptop (o otra PC) con una solución mínimo y sencillo:

Abre una consola y cambia a root con "sux" y ejecutar los siguientes pasos:

Configurar la primer NIC (tarjeta de red) para poder conectar al internet.

tcardconfig

Para el segundo NIC escogimos una dirección fijo dentro de un dominio privado, por ejemplo:

192.168.1.1

Entonces configurarlo con:

netcardconfig

Por favor, nota: Su IP debe ser en el mismo sub-red (subnet) que los IPs que serán asignado por el DHCP.

Si por error, un gateway ha sido especificado mientras se configura sus interfaces al red, tendrás que ejecutar el script:

remove-gateway.sh

Configurando DHCP/DNS:

apt-get install dnsmasq

Tiene que editar el archivo a través de agregar una línea en /etc/dnsmasq.conf

dhcp-range=192.168.1.50,192.168.1.150

Deja el resto del archivo como esta.

El servidor DHCP en el PC ahora puede ser iniciado con:

/etc/init.d/dnsmasq start

Si los clientes están ahora fijada a DHCP, una dirección IP sera asignado automáticamente

Para enrutar una petición de una cliente al internet, tiene que ejecutar el script:

/usr/sbin/masquerade.sh

Después de esto, cada cliente debe ser capacitado para conectar al internet.

Para tener esta configuración pre-seleccionada cuando se arranca el PC: Activar el dnsmasq en el proceso de arranque con:

update-rc.d -f dnsmasq defaults

Ejecutar el script, masquerade, con: 

echo "/usr/sbin/masquerade" >>  /etc/init.d/bootmisc.sh

Conectando con un módem telefónico de 56k

KDE tiene un programa como frente para un módem telefónico llamado KPPP, Herramienta Telefónica para el Internet, que se encuentra en el main menú bajo el Internet.

Esta aplicación tiene un manual de ayuda incorporado y esta contiene una guía comprensiva para configurar el módem para permitirle conectarse en línea.

A Mini Firewall

Si se necesita una pequeña script de muro de fuego, probar esto:

wget http://rebelhomicide.demon.nl/scripts/mini-miyagi.sh
chmod 755 mini-miyagi.sh 
apt-get remove --purge {firestarter, guarddog}

Ejecutarlo con lo siguiente:

./mini-miyagi.sh

Para parar el mini-miyagi:

/etc/init.d/mini_miyagi_fw.sh stop
