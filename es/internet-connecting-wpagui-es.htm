wpa Roaming GUI

wpa-roaming es un método con que puede navegar y conectar a redes inalambricos, dentro y fuera de una ambiente gráfico del escritorio. Este es muy conveniente para usuarios de notebooks.

wpasupplicant es un programa no solo para dirigirse a redes inalambricos, pero también todos redes inalambricos. En combinación con los /etc/network/interfaces "clasicos", es una manera muy poderoso para conectar a redes sin hacer nada. Lo hace automáticamente si la red es abierto o la red esta agregado a un archivo de configuración. Aun más conveniente: si permite enchufes calientes, la conección corecto es fijado, completamente automatizado después del login (en un ambiente de escritorio o una sessión no gráfico de X).

Básicas

Para poder usar wpa-roaming con wpasupplicant en un ambiente GUI tiene que tener dos paquetes instalados:
* wpasupplicant
* wpagui

apt-get update && apt-get install wpasupplicant wpagui

wpasupplicant es mandatorio, wpagui es una adición muy conveniente para redes controlado por GUI en ambientes donde desea cambiar el punto de acceso.

Usando wpa-gui sin configuración de red

Como raíz, modificar el /etc/network/interfaces para contener estos líneas. El nombre del interfaz puede variar:

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
iface default inet dhcp

Como raíz también puede editar directo el /etc/wpa_supplicant/wpa_supplicant.conf:

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        key_mgmt=NONE
}

El proximo paso asegura el wpa_supplicant.conf de acceso indebido. Esto es necesario, porque claves secretos de redes privados están guardado en este archivo:

chmod 600 /etc/wpa_supplicant/wpa_supplicant.conf

Ejecutandolo

Prerequisitos:

* Asegurar que eres miembro del grupo netdev
*Reiniciar la red o reiniciar el computador

El usuario creaedo durante la instalación de sidux, ya es miembro del grupo, netdev. Cualquier nuevos usuarios creados, tiene que ser agregado al grupo netdev por el usuario root.
Entonces, recargar el wpasupplicant como root con:

wpa_action <interface> reload

En muchos casos después del primer configuración de wpa-roam, una conexión necesita ser parada y el interfaz inalambrica tiene que ser activada. Puede usar estos comandos como raíz:

ifdown $interface  ## para una conección de red ejecutando
/etc/init.d/networking restart  ## configuración nuevamente cargado -no siempre necesario
ifup wlan0  ## wlan0 es el nombre común para un interfaz inalambrica. Toma el nombre desde /etc/network/interfaces

Esta empieza el wpa_gui como usuario (necesita la ruta completa):

Usando wpa-gui con configuración de red

Con la ayuda de IDString y Priority puede designar a cual red la computador esta conectado al iniciar. La prioridad más alta es 1000, la prioridad más bajo es 0. Tiene que agregar el IDString a /etc/network/interfaces también.

El sitaxis para /etc/network/interfaces.

Primero es por la conección a servidores DHCP, el segundo es si esta provisto con una dirección IP fijo. Para ajustar la configuración:

# id_str="home_dhcp"
iface home_dhcp inet dhcp

# id_str="home_static"
iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0
        network 192.168.0.0

        broadcast 192.168.0.255
        gateway 192.168.0.1
Ejemplos Practicos

Si desea ser conectado automáticamente a su WLAN de residencia en casa, darle el IDString = "home" y la priority = "15". Si esta viajando, y desea que el laptop conecta a cualquier red disponible, sin clave de acceso, Darle el IDString = "stalk" y priority = "1" (muy bajo). Pero, por favor, siempre verifica si la conección es legal y desconecta si obviamente no esta intendido de ser gratis o libre acceso.

Stanzas de ejemplos en /etc/network/interfaces:

# /etc/network/interfaces -- archivo de configuración para ifup(8), ifdown(8)

# El interfaz loopback
# agregado automáticamente cuando actualizando
auto lo
iface lo inet loopback

allow-hotplug eth0
iface eth0 inet dhcp

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf

iface home inet dhcp
iface stalk inet dhcp

Ejemplo /etc/wpa_supplicant/wpa_supplicant.conf (SSID y claves cambiado o solamente explicado):

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        ssid="my_ssid"
        scan_ssid=1
        psk=123ABC  ## aquí viene el clave de acceso en código hexadecimal!!
#       psk="password_in_ascii"   ## no necesita el clave en ascii
        key_mgmt=WPA-PSK
        pairwise=TKIP
        group=TKIP
        auth_alg=OPEN
        priority=15
        id_str="home"
}

network={
        ssid=""
        scan_ssid=1
        key_mgmt=NONE
        auth_alg=OPEN
        priority=1
        disabled=1   ## no automatic connection, one needs wpa_cli or wpa_gui
        id_str="stalk"
}

Si "disabled=1" no será conectado automáticamente a un bloque definido de red (WLANs abiertos), tendrá que iniciar roaming a través de wpa_gui o wpa_cli. Para roaming automatica no usa el opción, o comentar la línea con la opción "disabled" con una #.

WEP Encripción

Si desea agregue redes encriptado WEP al wpa_supplicant.conf permanentemente, el sintaxis es:

network={
	ssid="example wep network"
	key_mgmt=NONE
	wep_key0="abcde"
	wep_key1=0102030405
	wep_tx_keyidx=0
}
Notas:
1. Facil de reutilizar

Una vez configurado, puede reusar la configuración en otros laptops o de escritorios con tarjetas WLAN. Solamente copiar /etc/network/interfaces (ajuste el nombre del interface si requerida) y /etc/wpa_supplicant/wpa_supplicant.conf a la nueva computador. No hay necesidad de instalar nada después de eso.

2. Respaldar

Es bueno de respaldar /etc/network/interfaces y /etc/wpa_supplicant/wpa_supplicant.conf, pero encriptar el respaldo porque contiene información sensible. . Una manera facil es un archivo protegido con clave como el 7zip, zip, rar o otros.

3. Facil acceso a wpa_gui

Para crear un icono en el escritorio, haga clic derecho en kmenu > internet > wpa_gui y escoger, Enviar al escritorio.
