wpa Roaming GUI

wpa-roaming es un método con el que puede navegar y conectarse a redes inalámbricas, con o sin interfaz gráfica de escritorio. Esto es muy conveniente para usuarios de ordenadores portátiles.

wpasupplicant es un programa que no sirve sólo para conectarse a redes inalambricas con la clave WPA, sino que puede conectarse también a cualquier red inalámbrica. Combinado con la configuración "clásica" de /etc/network/interfaces ofrece una manera muy potente para conectarse a redes sin hacer nada más. Lo hace automáticamente si la red es abierta o está agregada a un archivo de configuración. Aun más conveniente: si el usuario permite hotplugging ("enchufe en caliente"), la conexión corecta queda fijada de forma completamente automática después del login (en un ambiente de escritorio o en una sessión no gráfica de X).

Las Bases

Para poder usar wpa-roaming con wpasupplicant en un ambiente GUI tiene que tener dos paquetes instalados:
* wpasupplicant
* wpagui

apt-get update && apt-get install wpasupplicant wpagui

wpasupplicant es obligatorio; wpagui es un complemento muy conveniente para redes si desea cambiar el punto de acceso en ambientes controlados por GUI.

Usando wpa-gui sin configuración de red

Como root, modifique el /etc/network/interfaces para contener estas líneas. El nombre del interfaz puede variar:

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
iface default inet dhcp

También como root puede editar directamente el /etc/wpa_supplicant/wpa_supplicant.conf:

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        key_mgmt=NONE
}

En la etapa siguiente se protege a wpa_supplicant.conf de un acceso indebido. Esto es necesario, porque la claves secretas de las redes privadas están guardadas en este archivo:

chmod 600 /etc/wpa_supplicant/wpa_supplicant.conf

Usándolo

Prerequisitos:

* Asegúrese de que es miembro del grupo netdev
* Reiniciar la red o reiniciar el computador

El usuario creado durante la instalación de sidux ya es miembro del grupo netdev. Cualquier nuevo usuario que se haya creado tiene que ser agregado al grupo netdev por el administrador.
A continuación recargue el wpasupplicant [como root] con:

wpa_action <interface> reload

En muchos casos después de la primera configuración de wpa-roam, la conexión necesita ser parada y el interfaz inalámbrico reactivado. Puede usar estos comandos como root:

ifdown $interface  ## detiene una conexión de red que se esté ejecutando
/etc/init.d/networking restart  ## configuración nuevamente cargada; no siempre es necesario
ifup wlan0  ## wlan0 es el nombre común para el interfaz inalámbrico. Toma el nombre desde /etc/network/interfaces

Iniciando wpa_gui

Esta orden lanza el wpa_gui como usuario (necesita la ruta completa):

/usr/sbin/wpa_gui

Usando wpa-gui con configuración de red

Con la ayuda de IDString y Priority puede designar a qué red se conectará el ordenador al iniciar. La prioridad más alta es 1000, la prioridad más baja es 0. Tiene que agregar el IDString a /etc/network/interfaces también.

La sitaxis para /etc/network/interfaces.

Primero la conexión a servidores DHCP, después la dirección IP fija si la tiene. Para ajustar la configuración:

# id_str="home_dhcp"
iface home_dhcp inet dhcp

# id_str="home_static"
iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0
        network 192.168.0.0

        broadcast 192.168.0.255
        gateway 192.168.0.1
Ejemplos Prácticos

Si desea estar conectado automáticamente a la WLAN de su casa, escriba el IDString = "home" y la prioridad = "15". Si está viajando y desea que el ordenador portátil se conecte a cualquier red disponible, sin clave de acceso, dele el IDString = "stalk" y prioridad = "1" (muy baja). Pero, por favor, siempre verifique si la conexión es legal y desconecte si obviamente no es gratis o de libre acceso.

Ejemplos en /etc/network/interfaces:

# /etc/network/interfaces -- archivo de configuración para ifup(8), ifdown(8)

# El interfaz loopback
# agregado automáticamente cuando actualizando
auto lo
iface lo inet loopback

allow-hotplug eth0
iface eth0 inet dhcp

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf

iface home inet dhcp
iface stalk inet dhcp

Ejemplo de /etc/wpa_supplicant/wpa_supplicant.conf (SSID y claves cambiados o solamente comentado):

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        ssid="my_ssid"
        scan_ssid=1
        psk=123ABC  ## aquí viene el clave de acceso en código hexadecimal!!
#       psk="password_in_ascii"   ## no necesita el clave en ascii
        key_mgmt=WPA-PSK
        pairwise=TKIP
        group=TKIP
        auth_alg=OPEN
        priority=15
        id_str="home"
}

network={
        ssid=""
        scan_ssid=1
        key_mgmt=NONE
        auth_alg=OPEN
        priority=1
        disabled=1   ## no automatic connection, one needs wpa_cli or wpa_gui
        id_str="stalk"
}

Si "disabled=1" no será conectado automáticamente a un bloque definido de red (WLANs abiertos) y tendrá que iniciar roaming a través de wpa_gui o wpa_cli. Para roaming automático no use esta opción, o comente la línea con la opción "disabled" con una #.

WEP Encriptación

Si desea agregar redes encriptadas WEP al wpa_supplicant.conf permanentemente, la sintaxis es:

network={
	ssid="example wep network"
	key_mgmt=NONE
	wep_key0="abcde"
	wep_key1=0102030405
	wep_tx_keyidx=0
}
Notas:
1. Facil de reutilizar

Una vez esté configurado, puede volver a usar la configuración en otros ordenadores portátiles o de escritorio con tarjetas WLAN. Solamente copie /etc/network/interfaces (ajuste el nombre del interface si se lo pide) y /etc/wpa_supplicant/wpa_supplicant.conf en el nuevo ordenador. No hay necesidad de instalar nada más después de eso.

2. Respaldar

Es bueno hacer copias de seguridad de /etc/network/interfaces y /etc/wpa_supplicant/wpa_supplicant.conf, pero encripte el respaldo porque contiene información sensible. Una manera fácil es un archivo protegido con clave como el 7zip, zip, rar u otros.

3. Facil acceso a wpa_gui

Para crear un icono en el escritorio, haga clic derecho en kmenu > internet > wpa_gui y escoja 'Enviar al escritorio'.

4. SSIDs Escondidos

Los SSIDs escondidos son detectados cuando scan_ssid=1 está definido en el bloque de red.

5. Algoritmos de utentificación

Para la encriptación WPA/WPA2 necesita:

auth_alg=OPEN

Para la encriptación estática WEP necesitaría:

auth_alg=SHARED
