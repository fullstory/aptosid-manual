Clave root del modo vivo

Por favor note: Cuando ejecuta algo con permisos de raíz, debe saber que esta haciendo! Para navegar sobre una LAN no se requiere acceso de raíz.

En el LiveCD de sidux, la clave root no esta fijado. Si quiere ejecutar un programa que requiere permisos de root, hay varias opciones:

Fijar un clave root (temporario):

Abre una consola / terminal de comandos:

sidux@0[sidux]$ sudo passwd
Entra nueva el clave UNIX:
Tipear de nueva clave UNIX:
passwd: Password actualizado con éxito
sidux@0[sidux]$

ahora puede usar esta clave durante el resto de la sesión con el LiveCD de sidux.

Ejecuta el programa con 

sudo
ó
su-me
Oprimir Alt+ F2, y Entra 
sudo

un espacio y después el nombre de la programa deseada, por ejemplo:

sudo gparted

Ejecuta el programa de un consola-raíz:

Abre una consola / concha de comandos:

sidux@0[sidux]$ su
root@0[sidux]# gparted (o lo que deseas...)

To get out of being root in the konsole type:

exit

or just click the top right hand corner to close the konsole.

Instalando software ejecutando el LiveCD

apt-get update
apt-get install your-preferred-package

Nota: Cuando salio del Live-CD, ningún cambio esta guardado.

Como escribir en particiones NTFS con ntfs-3g

Precaución: ntfs-3g es un intérprete beta nuevo, nunca utilizarlo sin un respaldo externo, y por supuesto no en un sistemas de producción o de trabajo! Si pierdas datos, será su culpa, entonces usar a su propio riesgo!

Abrir una consola y entra los siguientes comandos: Ver Particionando el DD - Nombramiento del Disco

su
umount /dev/XdXX
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para salir de la consola, tipea: exit

Ahora, el volumen NTFS debe ser montado rw (leer/escribir) y debe ser posible de escribir datos en el. ¡Pero, una vez más, Precaución! experimental!
