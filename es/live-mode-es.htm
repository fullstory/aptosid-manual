Clave root del modo vivo

Por favor, tome buena nota: Cuando ejecute algo con permisos de root debe saber lo qué esta haciendo! Para navegar por internet no se requiere un acceso de root.

En el LiveCD de sidux, la clave root no existe. Si quiere ejecutar un programa que requiera permisos de root, tiene varias opciones:

Fijar un clave root (temporal)

Abra una consola/terminal de órdenes:

sidux@0[sidux]$ sudo passwd
Entre la nueva clave UNIX:
Escriba otra vez la clave UNIX:
passwd: Password actualizado con éxito
sidux@0[sidux]$

ahora puede usar esta clave durante el resto de la sesión con el LiveCD de sidux.

Ejecutando un programa desde un terminal de root

Escribiendo sux en una consola, conseguirá privilegios de root con la posibilidad de ejecutar aplicaciones en el entorno gráfiico X.

Abra una consola/terminal (también llamada "shell"):

sidux@0[sidux]$ sux
root@0[sidux]# gparted (o lo que desee...)

Algunas aplicaciones de KDE requiren dbus-launch antes de escribir el nombre de la aplicación:

dbus-launch <Application>

Otra opción paraa todos los gestores de escritorios es: 

Alt+F2 su-to-root -X -c <Application>

Por ejemplo:

Alt+F2 su-to-root -X -c gparted

Para salir del modo raiz (root) en la consola, entre:

exit

o haga clic en la esquina arriba a la derecha para cerrar la consola.

o entre Ctrl-D para cerrar la consola :).

Instalar software ejecutando el LiveCD

apt-get update
apt-get install su-paquete-preferido

Nota: Cuando salga del Live-CD, ningún cambio será guardado, salvo que usted haya activado fromiso and persist.

Como escribir en particiones NTFS con ntfs-3g

Precaución: Aunque el driver ntfs-3g ha sido declarado como estable, no lo utilice sin realizar previamente una copia de seguridad externa, y por supuesto nunca en sistemas de producción o de trabajo! Si lo hace, la pérdida de datos será culpa suya; úselo a su propio riesgo!

Abra una consola y entre los siguientes comandos: Ver Particionando el DD - Nombramiento del Disco

sux
umount /dev/XdXX
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para salir de la consola, tipea: exit o Ctrl-D

Ahora, el volumen NTFS debe ser montado rw (leer/escribir) y debe ser posible escribir datos en él. ¡Pero, una vez más, Precaución! Úselo en situaciones de emergencia. No está recomendado para su uso diario.
