Tamaños de particiones y ejemplos

Para uso normal, recomendamos el ext3. Es el sistema de archivos escogido por sidux y es bien mantenido y apoyado.

Con la Manejador de Particiones gparted los disco duros se puede particionar y/o formatear. Este programa tiene un interface gráfico y es auto-explanatorio.

Gparted puede reducir o mover particiones, particiones NTFS también. Gparted no esta completamente desarrollada y es experimental, pero si lo utiliza en una manera ordenada, no debe encontrar problemas. Ver la documentación completo de gparted. Cambios a particiones-ntfs puede ser realizada con herramientas propietarias tales como, Partition Magic (TM) y Acronis (TM).

SIEMPRE RESPALDE SUS DATOS!

Si una partición parece como montada, tenemos que cerrar la manejador de particiones y desmontar el dispositivo (pinchar derecho en el icono del dispositivo montado en el escritorio->desmontar). Si tiene una partición swap, desmontarlo con:"sudo swapoff -a" en una Konsole. Realizados estos, podemos arrancar el manejador de particiones, de nuevo. En principio, 3 GB es más que suficiente para una instalación en el DD, pero no sera muy divertido. Un monto más razonable sera unas 5 a 8 GB. Para aquellos nuevos a Linux, sugerimos solamente 2 particiones para empezar (root y swap), porque este simplificas la primera instalación bastante. Es mejor de establecer una partición extra para /home.

Usuarios avanzados pueden tener algunas otras particiones adicionales para /var, /tmp, /opt, /usr/local, ...etcétera, para razones especiales. Tanto detalle ahora, nos llevara demasiado lejo de nuestra sujeto. Realmente debe tener una partición swap (equivalente al swapfile de Windows, pero es mucho más efectivo). Para uso normal, el swap debe ser el doble del RAM instalado. [Mas de 1GB probablemente no es necesario.] La sistema de archivos estándar de sidux es el ext3.

Para intercambiar datos con una instalación de Windows, tiene que usar VFAT (FAT32). Quien quiere usar XFS para el / (root) tiene que crear una partición extra de /boot con ext2; o usa el Lilo en vez de Grub (porque Grub no arranca bien con el XFS). El instalador a este punto solamente apoya el XFS en /, solamente editando el archivo ~/.siduxconf.

Quien quiera crear más de 1 partición debe escribir los nombres para referencia más luego! Después de realizar estos, guarda los cambios (el icono del floppy) y cerrar el manejador de particiones. Aquí hay unos ejemplos sencillas para DD de diferentes tamaños y diferentes tipos de uso, todo redondeado.

Configuración de Pruebas - 60GB para un PC donde Windows requiere bastante espacio
Disco Tamaño Sist.Archivos Puntos Monturas/Sistema
hda1 20GB NTFS WindowsXP
hda5 30GB FAT32 Data for Win and Linux
hda6 8GB ext3 /
hda7 500MB Swap Swap
TV y Video-PC - 200GB sin Windows para archivos grandes (ej. Video) XFS es mejor.
Disco Tamaño Sist.Archivos Puntos Monturas/Sistema
hda1 8GB ext3 /
hda5 6GB ext3 /home
hda6 5GB ext3 /var
hda7 180GB XFS /video
hda8 500MB Swap Swap
Standard-PC con Windows 120GB para usar el mejor de ambos mundos.
Disco Tamaño Sist.Archivos Puntos Monturas/Sistema
hda1 20GB NTFS WindowsXP
hda5 8GB ext3 /
hda6 20GB ext3 /home
hda7 70GB FAT32 Intercambio de datos WindowsXP-Linux

Juegos Linux - y Media-PC 160GB
Disco Tamaño Sist.Archivos Puntos Monturas/Sistema
hda1 18GB ext3 /
hda5 20GB ext3 /home
hda6 30GB ext3 /opt
hda7 90GB ext3 /data
hda8 2GB ext3 /var
hda9 500MB Swap Swap
Linux y Windows en 20 GB
Disco Tamaño Sist.Archivos Puntos Monturas/Sistema
hda1 5GB NTFS Win-System
hda2 10GB FAT32 Windows-Programas y Datos para Windows y Linux
hda5 5GB ext3 /
hda6 500MB Swap Swap

Esta alocación es recomendable cuando hay poco espacio en el DD (ej. en un Laptop). Windows se instala en una partición con 3-5 GB con nada más en el. Todos los programas y archivos están en la segundo partición, que es más grande. Quien tiene más espacio puede hacer este partición más grande o dividir programas-Windows y Datos en 2 particiones.

En general

Para sidux sin ningúna aplicaciones multimedia ni juegos, 5 GB es suficiente, para datos tiene la partición de FAT32. Hay muchas maneras de particionar el disco duro. Estos ejemplos son suficiente para darle una comienzo. Le hace más sentido de agregar un segundo disco duros para mejorar el rendimiento, comodidad y seguridad. En cualquier caso, recuerda que Windows siempre tiene que estar instalado en el primer disco duro!
