Tamaños de particiones y ejemplos

Para un uso normal, recomendamos el sistema de archivos ext4.

Con el Gestor de Particiones gparted los disco duros se puede particionar y/o formatear. Este programa tiene una interfaz gráfica amigable que se explica por sí misma.

Gparted también puede reducir o mover particiones y además puede manipular particiones NTFS [con una precaución: una vez alterada una partición NTFS, se tiene que reiniciar inmediatamente el sistema operativo antes de hacer cualquier otra operación.] Vea la documentación completa de gparted. Se pueden realizar cambios en particiones-ntfs con herramientas de software propietario tales como Partition Magic (TM) y Acronis (TM).

¡SIEMPRE, RESPALDE SUS DATOS!

Si una partición parece como montada, desmóntela, ya sea con un click derecho en el icono de la partición en el Gparted o a través de una termninal, por ejemplo:

umount /dev/sda1

La partición de intercambio puede ser desmontada en una terminal con: 

swapoff -a

En principio, 5 GB es más que suficiente para una instalación en un disco duro, pero no será muy agradable. Un minimo razonable para una instalación será de unos 12 GB. Para aquellos que sean nuevos en Linux, sugerimos solamente 2 particiones al principio: root y swap. Esto simplifica la primera instalación considerablemente. Luego establecerá más particiones para /home y otros datos.

Realmente debe tener una partición de intercambio (equivalente al swapfile de Windows pero es mucho más efectivo). Para uso normal, la partición swap debe ser 2 veces la cantidad de RAM instalada.

Para intercambiar datos con una instalación de Windows, debe utilizar vfat (fat32) o el ext2 en vista que ya existe un driver de MS Windows™ para leer ext2 [XFS no está soportado]. Vea Sistema de ficheros instalable Ext2 para MS Windows y también Escribiendo en particiones NTFS con ntfs-3g.

Es aconsejable tomar nota de los nombres de las particiones para posterior referencia.

Algunos ejemplos sencillos de distintos tamaños de particiones:
1 TB para arranque dual MS Windows y Linux
Disco Tamaño Sistema de ficheros Punto de montaje/Sistema
sda1 50 GB NTFS Sistema MS Windows
sdb1 100 GB ext4 / (incluye home)
sda3 300 GB FAT32/ext2 Datos para los sistemas MS Windows y Linux
sda2 550 GB ext4 Datos para Linux
sda4 2 GB Linux Swap Linux Swap
Disco duro de 120 GB con MS Windows y arranque dual con Linux:
Disco Tamaño Sistema de ficheros Punto de montaje/Sistema
sda1 30 GB NTFS Sistema MS Windows
sda2 20 GB ext3 /
sda3 20 GB ext3 /home
sdb1 48 GB FAT32/ext2 Intercambio de datos entre MS Windows y Linux
sda4 2 GB Linux Swap Linux Swap
60 GB para arranque dual MS Windows y Linux:
Disco Tamaño Sistema de ficheros Punto de montaje/Sistema
sda1 24 GB NTFS Sistema MS Windows
sda2 10 GB FAT32/ext2 Datos para MS Windows y Linux
sda3 10 GB FAT32/ext2 Datos para MS Windows y Linux
sdb1 14 GB ext4 / (incluye home)
sda4 2 GB Linux Swap Linux Swap
Disco duro de 200GB:
Disco Tamaño Sistema de ficheros Punto de montaje/Sistema
sda1 20 GB ext4 /
sda2 20 GB ext4 /home
sda3 158 GB ext2/3/4 datos
sda4 2 GB Linux Swap Linux Swap
Disco duro de 160GB:
Disco Tamaño Sistema de ficheros Punto de montaje/Sistema
sda1 20GB ext4 /
sda2 20GB ext4 /home
sda3 59GB ext4 datos
sdb1 59GB ext4 data
sda4 2 GB Linux Swap Linux Swap
En general

Hay muchas manera de hacer la partición de sus discos duros. Estos ejemplos deben ser suficientes para empezar.

Es prudente comprar un disco duro USB para hacer copias de seguridad regularmente por si uno de sus discos duros fallase. En arranque dual con MS Windows™, ponga sienpre MS en el primer disco duro/partición.

Para otras opciones de partición, vea Particiones en volúmenes LVM - Administrador de Volúmenes Lógicos y también Instalación cifrada.
