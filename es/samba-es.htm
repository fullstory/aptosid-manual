Configurando sidux para utilizar SAMBA (Windows) Directorios Compartidos desde máquinas remotos

Haga todos los comandos como root (en un Terminal o Consola) y los URLs en Konqueror (ejecuta Konqueror como usuario normal).

servidor = nombre del servidor o el IP de la máquina Windows
share = nombre del carpeta siendo compartido

En KDE - entra el URL en Konqueror smb://server o el URL completo: smb://server/share

En una consola puede ver los shares (compartidos) ubicado en el servidor ejecutando:

smbclient -L server

Para montar una dirección compartido, un share, en un directorio -(con pleno acceso para TODOS los usuarios) recuerdan eso: El punto de montaje tiene que existir. Si no lo existe, primero tiene que crear n directorio así (el Nombre es arbitrario):

mkdir -p /mnt/server_share

Entonces, monta el share - sistema de archivos remoto VFAT:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777 //server/share /mnt/server_share

o una sistema de archivos NTFS:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777,lfs //server/share /mnt/server_share

Para terminar la conexión, utiliza:

umount /mnt/server_share

Si deseas hacer un entrada en /etc/fstab para hacer la montura más fácil, entonces inserta la siguiente linea en aquel archivo:

//server/share  /mnt/server_share       cifs  defaults,username=your_username,password=**********,file_mode=0777,dir_mode=0777   0   0

Como configurar sidux como un servidor de Samba

En vista de que Samba no esta en el LiveCD, necesitará hacer lo siguiente para tener acesco a samba:

su
apt-get update
apt-get install samba
Instalaciones en el DD:

Porque Samba no esta preinstalado, necesitará hacer lo siguiente para tener aceso a Samba:

sux
apt-get update
apt-get install samba

En una instalación al Disco Duro, es necesario ajustar la configuración de Samba. Aquí esta un ejemplo sencillo. Si deseas saber más sobre el uso de Samba y la configuración de un servidor Samba en Linux, se recomiendo lea la documentación de Samba..

Para ajustar la configuración de samba, haga los siguiente:

Abra el archivo: /etc/samba/smb.conf en un editor (por ejemplo, kedit) y entre esto:

# Cambios globales - Propuesto con todo tan sencillo
#como posible, sin claves, funciona como el Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Share sin permisión de escribir -importante si sistemas de archivos NTFS están de ser compartidos!

[WINDOWS]
comment = Particion de Windows
browseable = yes
writable = no
path = /media/hda1 # <-- ajuste a su partición
public = yes

# Compartiendo una partición con permisos de escritura -la  partición tiene que ser montado
# writable (escribirle) - que tiene sentido con el FAT32.

[DATA]
comment = Partición de Data (primer partición extendido)
browseable = si
writable = si
path = /media/hda5
public = si
Revisando las comparticiones de Samba

Para fijar las comparticiones sin tomar en cuenta la seguridad en Samba, haga los siguientes comandos (por ejemplo para una configuración en LAN):

Fijar el directorio y su contenido, por lo menos: -rwxr-xr-x:

ls -la pathTo/dirShareName/*

Si no, haga:

chmod -R 755 pathTo/dirShareName

Si lo desea que sea escribible:

chmod -R 777 dirShareName

Una manera de asegurar que la compartición es funcionando: (no se olvida de iniciar el servidor):

smbclient -L localhost

Debe ver algo así:

 smbclient -L localhost
Password:
Domain=[HOME] OS=[Unix] Server=[Samba 3.0.26a]

        Sharename       Type      Comment
        ---------       ----      -------
        IPC$            IPC       Servicio IPC (3.0.26a)
        MaShare         Disk      commentario
        print$          Disk      Drivers de impresoras
Domain=[MSHOME] OS=[Unix] Server=[Samba 3.0.26a]

Si no configuraste un clave de pase, solo oprima ENTER

No olvide de guardar. Puede iniciar/detener Samba con:

/etc/init.d/samba start

y:

/etc/init.d/samba stop

Además, puede iniciar/detener Samba al arranque: Ejecuta estos comandos:

update-rc.d samba defaults

Ahora Samba se iniciar cuando el computador se arranque y se para cuando el computador se apague normalmente.
