smxi: sintaxis de comandos

Para utilizar el herramienta de administración multifuncional contribuido por la comunidad de sidux, smxi, que significa mantenimiento de sidux, necesita instalar los scripts:

To install smxi (if its not presently in sidux-scripts)

wget -qO /usr/local/bin/smxi http://techpatterns.com/smxi
make it executable:
chmod +x /usr/local/bin/smxi
To start, type, as root: 
smxi
and follow the prompts :: script homepage - http://techpatterns.com/h2

or to just install the 3d driver script

wget -qO /usr/local/bin/sgfxi http://techpatterns.com/downloads/distro/sgfxi
make it executable:
chmod +x /usr/local/bin/sgfxi
sgfxi
To see options:
sgfxi -h

Caractericticas y funciones básicas del script smxi

El script tiene muchos caracteristicas, muchos de los cuales parecen una sóla vez a los usuarios, otros que siempre están presentada. Después de una verificación inicial del sistema, y realizar algunas preguntas para configuraciones, estos son las opciones principales que sean presentada:

  • Información del Script. Este sección muestra el kernel instalada, la fecha del ultimo uso de smxi para actualizar el sistema, la ultima ejecución, y algunas otras informaciones. Puede ser util, especialmente en sistemas que no están actualizada frecuentemente (recomendado: siempre realiza un dist-upgrade por lo menos una (1) vez al mes). Más de un mes puede ser más riesgoso.)
  • Instalación del Kernel. Ofrece el kernel estable, corriente como la preseleccionada, y ofrece opciones para la instalación de kernels en prueba tanto como otras más viejas. Si ejecuta el smxi con la opción -K, la primer kernel alistado en estable, será el kernel de prueba actual, si existe.
  • sección de advertencias del dist-upgrade: Ofrece advertencias en vivo referente a problemas corrientes y pasados. Después de acordar a seguir (dato: sid es casi nunca en la verde, estado sin problemas, es normalmente amarillo, que es seguro con los arreglos).
  • Arreglos antes del "dist-upgrade", si existe, ejecutara ahora.
  • Advertencias de archivos de configuración: veras una noticia con un aviso de actualizar estos configuraciones. Revisalos, memorizalos, anotarlos. Si la actualización preguntas si desea mantener o cambiar su archivo de configuración, diga y en vez del normal n.
  • Sección de "pre dist-upgrade", incluya manejo de errores para avisar si la actualización salio sin o con errores.
  • Arreglos después de la actualización (post dist-upgrade) , si los hay, ahora será ejcutado.
  • Opciones varios, que incluye:
    • fixfonts - presenta un juego de opciones para arreglar tipos de letras. Bueno para después de la instalación inicial, poco necesitado después de eso.
    • package-install - ofrece conjuntos de paquetes comunes para instalación, incluyendo cosas no-libres, como instaladores del flash y googleearth, y instaladores de suites ofimaticas. Nota: el instalador de OpenOffice.org ofrece selecciones de paquetes de idiomas, y algunas otras cosas que quizás no conocen. Además paquetes de utilidades y otras opciones de paquetes.
    • package-removal - listas de grupos de varios paquetes que puede desear quitar si no lo necesitan: como wifi, componentes en idiomas que no usan, etcétera.
    • clean-up-stuff - varios opciones de limpieza, tales como el eliminador de kernel (nota: siempre deja por lo menos un kernel viejo instalado cuando usa sidux para evitar problemas futuras posibles)
    • miscellaneous-tweaks - cosas como configuración popular de mozilla, afinaciones para el manejador de archivos y otras cosas que puede ser agregado sobre tiempo. Revisanlo.
    • continue - Sigue. Continuar a los preguntas sobre la instalación de drivers gráficas
    • quit - Salir directamente de smxi si has completado su uso. Nota: el instalador de gráficas tiene una opción para arrancar Kde y/o init 5, así que puede usar eso para regresar al escritorio gráfica (GUI).
  • graphics installer - esté utilizas el instalador sgfxi de sidux para instalar "nvidia", usualmente los drivers propietario de ATI fglrx (ATI ha sido eratico sobre el año pasada, pero mayormente funciona). Además, ofrece instalar el driver de xorg, y una opción para arrancar X/su escritorio gráfica.

Como usar los opciones del script smxi

Opciones: todos juntos: 

-deghiIklmrstuwx

o por separado: 

-giktw -f -x

Puede ejecutar este script con cualquier combinación de estos opciones.

Pueden juntarlos, así: -ik, o separarlos, como estos: -w -x

Cada opción o grupo de opciones, tiene que empezar con una:

 -

Ejemplos:

 smxi -ikw

omitir la información del sistema, instalación del kernel y la precaución.

smxi script: opciones: saltar o omitir partes del script

Lo siguiente permite omitir parte del script smxi:

  • -d Omitir el dist-upgrade si también escoge la opción w.
  • -e Omitir los preguntas 'Please hit enter to continue'.
  • -g Omitir los preguntas sobre instalación gráfica.
  • -i Omitir la sección de información del sistema.
  • -k Omitir la pregunta de instalación del kernel.
  • -r Omitir la verificación para el ultimo, rearranque y bajando la última versión del script. Este evita el rearranque del script. No esta recommendada para la mayoria de usuarios, ecepto cuando están rearrancando despues de instalar un kernel y ejecutar el escript de nuevo.
  • -s Omitir algunas partes del script que require una conexión internet. Evitar hacer esto, porque se deshabilita casi todo la funcionalidad del script.
  • -S Omitir la prueba para una conexión internet. Utilizarlo si el script falla la prueba de conexión pero sabe que la conexión esta bien. (Este no esta recommendado y el exito no esta garantizado.)
  • -t Omitir la sección de opciones después del "dist-upgrade".
  • -w Omitir la sección de las advertencias y precauciones acerca del dist-upgrade.

Por ejemplo:

smxi -kwid

seguir directamente a la sección de Opciones Miscelaneos, después del dist-upgrade. smxi -ki saltar sobre la pregunta de instalación del kernel y la sección de información.

Opciones avanzada del smxi

Los siguientes son opciones avanzadas del script, y son para hacer cosas fuera de lo común, pero pueden ser muy útil:

  • -I Fija un nivel diferente para el rearranque del X. Se require este sintaxis: -I 3
    Por favor, NOTA: El número usado es el nivel de init (iniciación) en que desea arrancar el script.
  • -K Instalación Avanzada del Kernel. Si hay kernel(s) nueva de prueba, serán mostrada en las opciones de instalación del kernel. NOTA: ¡Instalar kernels de prueba a su propio riesgo! (por apoyo de hardware nuevo)
  • -l Crear un registro de fecha y hora de todo los data de update y dist-upgrade. Los archivos de registro se escribe en /var/log/smxi/
  • -m Ejecuta la opción de cambiar los espejos de Debian en el /etc/apt/sources.list
  • -M Ejecuta la opción para cambiar los reflejos preseleccionado del kernel para smxi
  • -p Ejecuta el "apt-get update" sin usar los pdiffs. Bueno para conexiones rapidos.
  • -P Ejecuta el script con los configuraciones para conexión, se require este sintaxis:
    -P http://[<username><:password>@]<server><:port>
  • -U Fuerza un update (actualización) del script antes que lo ejecuta de nuevo.
  • -x Ejecuta caracteristicas extras del script, que no forman parte del núcleo de funcionalidad de smxi.
  • -h Imprima en pantalla la menú de Ayuda de smxi.
  • -vImprima información de la versión y el sistema, incluyendo versión del distro, kernel actual, última fecha de apt update y dist-upgrade, y el último uso de smxi.

Opciones del smxi script configurado por el usuario

Puede tener que el script realiza otras tareas a través de editar su archivo de configuración: /etc/smxi.conf

Por favor, NOTA: que este archivo no existe hasta después de la primer ejecución y terminación del script. Entonces si desea agregar algunas de estas cosas y no se encuentra el archivo, sólo crearlo.

  • sgfxi= <algunas opciones normal de sgfxi, que no están estándar en el instalador gráfica>, por ejemplo, si agregue la línea:
    sgfxi=b
to
/etc/smxi.conf,

    hace que su instalación de nvidia automaticamente utiliza el argumento t, además que el estándar -c
    Nota que estos argumentos estan apoyado por sgfxi, no por el smxi, smxi solamente los agregue estos a los argumentos que envia a sgfxi cuando realiza la instalación gráfica. Esto no apoyo ningun argumento / opción, que se require información extra, como: -o 8.33.6, por ejemplo, solamente opciones de una sola letra para la instalación gráfica de sgfxi.
  • hold-install=<lista de paquetes separados por ^> - Este opción permite agregar paquetes a la función automática de "hold/release" del script, que ayuda en evadir problemas con paquetes temporaralmente roto, o de guardar paquetes que no desear actualizar, sin olvidar de ellos después. Por ejemplo:
    Iniciar: hold-install=nano^libgtk2.0
    Nota: Importante: ningun espacios, solamente usa el "^" como separador entre nombres de paquetes!
