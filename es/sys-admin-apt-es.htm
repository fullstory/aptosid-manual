Pequeña Guía de Recetas-APT y 'sources.list'

¿Que quiere decir APT?

APT es el mnemotécnico para el Herramienta de Empaque Avanzado (Advanced Packaging Tool) y es una colección de programas y scripts que ayuda tanto el sysadmin (en este caso, root) con la instalación y gestión de archivos-deb, pero igualmente informa a la sistema mismo de saber que esta instalado.

sources.list

El sistema "APT" - necesita un archivo de configuración que contiene información acerca de las locaciones de paquetes instalable y actualizable y es comunmente referido como sources.list.

Sources are contained in a directory or folder named:
/etc/apt/sources.list.d/

Adentro este directorio hay 2 archivos llamada:
/etc/apt/sources.list.d/debian.list y
/etc/apt/sources.list.d/sidux.list

Este tiene el beneficio de más rápido cambios de espejos (automatizado), y lo hace más seguro de reemplazar listas.

También, puede agregar su propios /etc/apt/sources.list.d/*.list archivos

Todos los ISOs de sidux tienen los siguientes repositorios pre-seleccionado:

# Free University Berlin/ spline (Student Project LInux NEtwork), Germany
deb ftp://ftp.spline.de/pub/sidux/debian/ sid main contrib non-free fix.main fix.contrib fix.non-free
#deb-src ftp://ftp.spline.de/pub/sidux/debian/ sid main contrib non-free fix.main fix.contrib fix.non-free

Entradas adicionales para necesidades no-libres siempre puede ser encontrado en nuestro, siempre actualizado sidux.list and debian.list Desde el 28/02/2008 0800 hrs UTC, están estos:

# Ver sources.list(5) para más información, especialmente
# recuerda que solamente puede usar http, ftp o URIs de archivos
# CDROMs se maneja con el herramienta: apt-cdrom

NOTA: En este ejemplo, el ftp.us, indica que esta usando los repositorios Alemanes. Puede editar este archivo, como raíz, para usar repositorios más cerca físicamente a usted, simplemente por cambiar el país, por ejemplo, cambia ftp.us.* a ftp.es.* ó ftp.ar.*. Muchos países tiene reflejos locales que ayudan a conservar el ancha de banda (bandwidth) y mejorar velocidades.

Para poder conseguir información actualizada sobre los paquetes, el APT utiliza un base de datos. Esta base de datos contiene el paquete pero también los demás paquetes que este requiere para poder funcionar (llamada dependencias). El programa apt-get usa esta base de datos cuando instala los paquetes escogido para resolver todos las dependencias, y así, garantiza que el paquete escogido funciona. La actualización de este base de datos se realiza con el comando: apt-get update.

# apt-get update

Revertir (downgrade) un paquete

A veces es necesario revertir a una versión previa de un paquete, debido a una falla mayor con la versión que acaba de instalar.

Hold (aguantar) un(as) paquetes

echo package hold|dpkg --set-selections

Para quitar un "Hold" de un paquete

echo package install|dpkg --set-selections

Para listar paquetes en "Hold" (aguantada):

dpkg --get-selections | grep hold

Revertiendo paquetes

Los pasos utilizando kmahjongg como ejemplo:

  1. activar los fuentes de "testing" en /etc/apt/sources.list 
    (borrar el #) para que parece 
    deb http://ftp.nl.debian.org/debian/ testing main contrib non-free
  2. apt-get update
  3. entonces,
    # apt-get install -t testing kmahjongg
  4. Ponga el paquete nuevamente instalado en "hold" con:
    echo kmahjongg hold|dpkg --set-selections
  5. entonces, comentar la fuente de "testing" en el /etc/apt/sources.list 
  6. apt-get update

Cuando desea actualizar a la versión más reciente, haga esto:

echo kmahjongg install|dpkg --set-selections
apt-get update
apt-get install kmahjongg

Actualización de una sistema instalado- dist-upgrade

Si van a actualizar el sistema completo, se realizó a través de un dist-upgrade. Siempre debe revisar los Current Warnings en sidux página de web principal, y revisa las advertencias contra las paquetas que su sistema quiere actualizar con dist-upgrade. Si necesita aguantar o hold un paquete, ver la referencia en Downgrade and Hold a package

El uso de 'upgrade' solamente, no esta recomendada en 'debian sid'.

Frecuencia de dist-upgrade

Realizar el dist-upgrade tan routinamente que puedes, idealmente, una vez a la semana o cada dos semanas esta bien, y por lo minimo una vez al mes para estar seguro. Un dist-upgrade realizado cada 2 a 3 meses será considerado como el limite de seguridad. Para paquetes no-estándar, o auto-compilado, tiene que tener más cuidado con la actualización, porque estos puede causar fallas en su sistema debido a incompatibilidades.

Solamente bajar sin instalar

Una opción poco conocido y muy util es la bandera "-d" para revisar cuales paquetes están involucrado:

apt-get dist-upgrade -d

El -d permite bajar los paquetes sin instalarlos, mientras sigue en X desde una consola. Además, le permite realizar el dist-upgrade en init 3 a una hora más a su conveniencia. Este permite revisar las advertencias en la lista, porque, sabiamente, le da la opción de proceder o abortar el proceso.

Estos avisos están allí por una razón --debido a la naturaleza de "Debian sid/unstable" y que se actualiza diariamente.

NUNCA HAGA el DIST-UPGRADE (ni UPGRADE) con ADEPT o SYNAPTIC

Si no va a "init 3", bueno, lo siento, "usted ha sido advertido."

¿Por qué recomendamos SOLAMENTE "apt-get" para "dist-upgrade"?

Los gestores de paquetes "adept", "aptitude", "synaptic" y "kpackage" no están siempre capaz de acomodar las grandes cantidades de cambios que sucede en Sid (cambios de dependencias, de nombres, en los scripts de los mantenedores, etcetera). Esta no es una falla de los desarroladores ni de estos herramientas, los cuales están excelente para la rama estable de Debian. Simplemente, no está apta para las necesidades especiales de Debian Sid.

Utiliza lo que le gusta para buscar paquetes, pero recomendamos utiliza el "apt-get" para "install", "remove", "purge" y "dist-upgrade".

Los gestores de paquetes "adept", "aptitude", "synaptic" y "kpackage" están no-deterministica para selecciones complejos de paquetes: cuando esta se mezcla con un conjunto de paquetes tan cambiables como es Sid, y aun peor, un repositorio externo de calidad cuestionable (no lo recomendamos estos, pero esta una realidad en sistemas de usuarios) y pronto estará acercando al disastre. La otra renglon a notar es que todos estos gerenciadores de paquetes necesita ejecutar en el nivel 5, y/o en X, y haciendo un dist-upgrade (ni upgrade) en nivel 5 ó en X no esta recomendado). Puede dañar el sistema al punto ireparable, quizás no hoy, ni mañana, pero en tiempo, sucederá.

Pero el "apt-get" hace exactamente lo que pides de el. Si hay algun estallido, puede encontrar el punto del error para depurar y arreglar la causa. Si a caso el "apt-get" quiere remover la mitad de la sistema (debido a transiciones de librerias) es una decisión que el admin (probablemente usted) tiene que hacer después de leer todos los mensajes.

Este es el razón porque la sistema "Debian builds" se utiliza el apt-get y ningunas de los demás herramientas de paquetes.

Buscando un paquete con kio-apt

Utilizando el kio-slave, Konqueror provee una manera sin-peligro para utilizar un GUI para buscar paquetes en APT. Es solamente una maquina de buscada de paquetes, sin ningun opcion de instalar ni actualizar (upgrade).

Es una buen alternativa a la función de buscada de otras gestores de paquetes, tales como, Adept, Aptitude, Synaptic y Kpackage. Así no cae en la temptación de hacer un dist-upgrade en X o instalar paquetes utilzando herrmamientas no apto para Debian Sid.

Puede registrar el base de datos de paquetes tanto, dentro de APT, y/o con el 'package search' de la página de Debian,. Le hace una buena resumen de lás páginas con la información de apt-cache policy y apt-cache show.

Se provee:

  • apt:/search? expresión:registra el base de datos de paquetes para 'expresión'. Como el apt-cache search
  • apt:/show? paquete: muestra información detallada acerca de 'paquete'. Como el apt-cache show. Además muestra la política.
  • apt:/política? paquete: muestra la política para 'paquete'. Como apt-cache política.
  • apt:/list? paquete: lista las archivos en 'paquete'.

Instalación

sux
apt-get update && apt-get install kio-apt

Entonces, ejecuta konqueror y entrar:

apt:/

y oprima la tecla Enter. Ve la descripción para todos los posibles opciones que incluye buscando con una forma compacta o extendida.
