Pequeña Guía de Recetas-APT y "sources.list"

¿Que quiere decir APT?

APT es el mnemotécnico para el Herramienta de Empaque Avanzado (Advanced Packaging Tool) y es una colección de programas y scripts que ayuda tanto el sysadmin (en este caso, root) con la instalación y gestión de archivos-deb, pero igualmente informa a la sistema mismo de saber que esta instalado.

sources.list

Por eso, el sistema "APT" necesita un archivo-config: /etc/apt/sources.list, que contiene información acerca de las locaciones de paquetes instalable y actualizable. Un ejemplo de tal archivo desde una instalación típica de sidux es como lo siguiente:

Todos los ISOs de sidux tienen los siguientes repositorios pre-seleccionado:

deb http://ftp.de.debian.org/debian unstable main
deb http://sidux.com/debian/ sid main fix.main

Entradas adicionales para necesidades no-libres siempre puede ser encontrado en nuestro, siempre actualizado sources.list. Desde el 27/02/2007 0800 hrs UTC, están estos:

# Ver sources.list(5) para más información, especialmente
# Recuerda que solamente puede utilizar http, ftp o URIs de archivos
# CDROMs está manejado por el herramienta apt-cdrom. 

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# Testing
# deb http://ftp.de.debian.org/debian testing main contrib non-free
# deb-src http://ftp.de.debian.org/debian testing main contrib non-free

# Experimental
# deb http://ftp.de.debian.org/debian experimental main contrib non-free
# deb-src http://ftp.de.debian.org/debian experimental main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

NOTA: En este ejemplo, el ftp.de, indica que esta usando los repositorios Alemanes. Puede editar este archivo, como raíz, para usar repositorios más cerca físicamente a usted, simplemente por cambiar el país, por ejemplo, cambia ftp.de.* a ftp.es.* ó ftp.ar.*. Muchos países tiene reflejos locales que ayudan a conservar el ancha de banda (bandwidth) y mejorar velocidades.

Para poder conseguir información actualizada sobre los paquetes, el APT utiliza un base de datos. Esta base de datos contiene el paquete pero también los demás paquetes que este requiere para poder funcionar (llamada dependencias). El programa apt-get usa esta base de datos cuando instala los paquetes escogido para resolver todos las dependencias, y así, garantiza que el paquete escogido funciona. La actualización de este base de datos se realiza con el comando: apt-get update.

# apt-get update
Get:1 http://sidux.com sid Release.gpg [189B]
Get:2 http://sidux.com sid Release.gpg [189B]
Get:3 http://sidux.com sid Release.gpg [189B]
Get:4 http://ftp.de.debian.org unstable Release.gpg [189B]
Get:5 http://sidux.com sid Release [34.1kB]
Get:6 http://ftp.de.debian.org unstable Release [79.6kB]

Presumiendo que la base de datos de Apt es actualizada, y el nombre de la paquete se conoce, entonces el siguiente comando instalará un paquete llamado "qemu" y todas sus dependencias: (más adelante, encontraras como buscar y encontrar paquetes.)

Removiendo un paquete

De igual manera, no debe sorprender que lo siguiente desinstalará "gaim", sin embargo, no se remueve las dependencias instalados arriba, este se cubre más tarde en el uso de deb-orphan.

# ejemplo de apt-get remove
apt-get remove gaim
Leyendo listas de paquetes... Listo
Construyendo el árbol de dependencias... Listo

  gaim gaim-irchelper
0 actualizado, 0 nuevamente instalado, 2 para quitar y 310 no actualizado.

Después de desempacar 4743kB de espacio será liberado.
Desea continuar [S/n]?
(Leyendo base de datos ... 174409 archivos y directorios actualmente instalados.)
Removiendo gaim-irchelper ...
Removiendo gaim ...

En este último caso, los archivos del paquete 'gaim' (básicamente como esta configurado) no fueran borrado de la sistema. Serán utilizado si vuelva instalar la misma paquete de nuevo y se utiliza la misma configuración.

Si deseaba quitar los archivos de configuración también, ejecuta este comando:

# apt-get remove --purge gaim
Leyendo listas de paquetes... Listo
Construyendo el árbol de dependencias... Listo
Los siguientes paquetes serán REMOVIDO:
  gaim*
0 actualizado, 0 nuevamente instalado, 1 para quitar y 309 no actualizado.
Necesito bajar 0B de archivos.
Después de desempacar 4678kB de espacio será liberado.
Desea continuar [S/n]? Y
(Leyendo base de datos ... 174405 archivos y directorios actualmente instalados.)
Removiendo gaim ...
Purgando archivos de configuración de gaim ...
#

Aquí, el archivo /etc/gaim no fue solamente borrado, pero también todos los archivos de configuración fueran borrado/purgado. Sin embargo, si un aplicación contiene archivos que no fue creado durante la instalación del paquete siendo purgado, ellos se quedan y no serán borrados.

Revertir (downgrade) un paquete

A veces es necesario revertir a una versión previa de un paquete, debido a una falla mayor con la versión que acaba de instalar. Los pasos utilizando kmahjongg como ejemplo:

  1. activar los fuentes de "testing" en /etc/apt/sources.list 
    (borrar el #) para que parece 
    deb http://ftp.nl.debian.org/debian/ testing main contrib non-free
  2. entonces,
    # apt-get install -t testing kmahjongg
  3. Ponga la paquete nuevamente instalado en "hold" con:
    echo kmahjongg hold|dpkg --set-selections
  4. entonces, comentar la fuente de "testing" en el /etc/apt/sources.list

Cuando desea actualizar a la versión más reciente, haga esto:

echo kmahjongg install|dpkg --set-selections
apt-get update
apt-get install kmahjongg

Actualización de una sistema instalado

Si desea actualizar toda el sistema, este se realiza a través de un "dist-upgrade". Es extremadamente importante, primero, de verificar cuales son los paquetes que serán actualizado:

Si no hay programas de X o de KDE involucrado en el "dist-upgrade", puedes actualizar en el interfaz gráfica que estas utilizando, por ejemplo, KDE, Gnome o lo que sea. Programas que están ejecutando debe ser detenido.

De otra manera, la actualización debe ser llevado a cabo en una consola de texto (en este caso, podía bajar los paquetes con el "-d" y luego completar la actualización a un momento más oportuno en una consola de texto.) Esta pequeña inconveniencia puede ahorrar horas de trabajo más tarde.

Para bajar los paquetes solamente (sin instalarlos ni configurarlos) utiliza el parámetro "-d".

# apt-get -d dist-upgrade
#

The -d allows you to download the dist-upgrade without installing it whilst in X from the konsole and allows you to do the dist-upgrade in init 3 at a later time. It wisely gives you the option to proceed or abort like this

apt-get dist-upgrade -d
xxx upgraded, 30 newly installed, 25 to remove and 2 not upgraded.
Need to get 342MB of archives.
After unpacking 3916kB of additional disk space will be used.
Do you want to continue [Y/n]?
Y
(Y will download the upgraded packages without touching the installed system)

After the 'dist-upgrade -d' has downloaded, and wish to complete the dist-upgrade, immediately or at a later time, please follow these instructions:

ABSOLUTAMENTE NUNCA debe realizar un "upgrade" ni "dist-upgrade" mientras que este en el ambiente gráfico X. Siempre revise la sección "Current Warnings" en la página web principal de sidux. Estos avisos están allí por una razón --debido a la naturaleza de "Debian sid/unstable" y que se actualiza diariamente. Salir de KDE. Entre el modo texto por tipear Ctrl+Alt+F1; entonces login como "root" y después teclear "init 3"; apt-get update && apt-get dist-upgrade; run fix-fonts; run fglrx o nvidia script si xorg fue actualizado y tienes tal dispositivo (driver) gráficos, sino haga "init 5 && exit" [NUNCA HAGA EL DIST-UPGRADE (NI UPGRADE) CON ADEPT O SYNAPTIC] There is one exception!! du-fixes-h2, now called smxi, which is short for sidux-maintenance:

apt-get update
apt-get install sidux-scripts
smxi

For new users, the !h2 smxi script may provide help by semi-automating the dist-upgrade process

Si no va a "init 3", bueno, lo siento, "usted ha sido advertido."

¿Por qué recomendamos SOLAMENTE "apt-get" para "dist-upgrade"?

Los gestores de paquetes "adept", "aptitude", "synaptic" y "kpackage" no están siempre capaz de acomodar las grandes cantidades de cambios que sucede en Sid (cambios de dependencias, de nombres, en los scripts de los mantenedores, etcetera). Esta no es una falla de los desarroladores ni de estos herramientas, los cuales están excelente para la rama estable de Debian. Simplemente, no está apta para las necesidades especiales de Debian Sid.

Utiliza lo que le gusta para buscar paquetes, pero recomendamos utiliza el "apt-get" para "install", "remove", "purge" y "dist-upgrade".

Los gestores de paquetes "adept", "aptitude", "synaptic" y "kpackage" están no-deterministica para selecciones complejos de paquetes: cuando esta se mezcla con un conjunto de paquetes tan cambiables como es Sid, y aun peor, un repositorio externo de calidad cuestionable (no lo recomendamos estos, pero esta una realidad en sistemas de usuarios) y pronto estará acercando al disastre. The other item to note is that all of these types of GUI package managers need to run in init 5, and/or, in X, and in doing a dist-upgrade in init 5 and/or X , (or even an 'upgrade' which is not recommended), you will end up damaging up your system beyond repair, maybe not today or tomorrow, in time you will.

Pero el "apt-get" hace exactamente lo que pides de el. Si hay algun estallido, puede encontrar el punto del error para depurar y arreglar la causa. Si a caso el "apt-get" quiere remover la mitad de la sistema (debido a transiciones de librerias) es una decisión que el admin (probablemente usted) tiene que hacer después de leer todos los mensajes.

Este es el razón porque la sistema "Debian builds" se utiliza el apt-get y ningunas de los demás herramientas de paquetes.

Hay una sóla excepcion!!: sidux maintenance por h2

Searching for a package with kio-apt

Using kio-slave, Konqueror provides a non-dangerous way of using a GUI to search for packages in APT. It is a GUI search engine for packages only, without any install or upgrade options.

This is a good alternative to other package managers search functions like in adept, aptitude, synaptic and kpackage, so you will not get tempted to dist-upgrade in X or install packages using tools that are not suited for Debian sid.

You can search the package database both within APT, and/or the Debian 'package search' on the Debian web site, It makes a good summary of the pages with the information from apt-cache policy and apt-cache show.

It provides:

  • apt:/search? expression:searches the package databse for 'expression'. Like apt-cache search
  • apt:/show? package: shows detailed information about 'package'. Like apt-cache show. It prints the policy too.
  • apt:/policy? package: shows the policy for 'package'. Like apt-cache policy.
  • apt:/list? package: lists the files in 'package'.

Installation

sux
apt-get update && apt-get install kio-apt

Then konqueror and type in:

apt:/

and hit the enter key. See description for all possible options which include searching with a compact form or an extended form..

A complete description of the APT-System can be found at APT-HOWTO
