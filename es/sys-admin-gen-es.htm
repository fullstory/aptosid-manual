Activar servicios en sidux

insserv (Pontos 2008-04 y en adelante): Para iniciar/parar servicios ya instalados:

Lee /usr/share/doc/insserv/README.Debian, las notas de la versión y las páginas man ciudadosamente:

 $ man insserv 
 $ man invoke-rc.d 
 $ man update-rc.d 
 google LSB headers

Todo acceso a los scripts de iniciación por los scripts de los mantenedores de paquetes de Debian debe ser realizado a través de invoke-rc.d.

Para 'iniciar (start)':

invoke-rc.d <service> start

Para 'parar (stop)':

invoke-rc.d <service> stop

Para 'reiniciar (restart)':

invoke-rc.d <service> restart

Para prevenir que un servicio arranque al inicio

update-rc.d <service> remove
[will remove all startup links]

Para asegurar que un servicio arranque al iniciar el sistema [no siempre requerida]

update-rc.d <service> defaults
[crear enlaces de arranque]

Instalaciones Viejos (antes de Pontos 2008-04)

insserv en sistemas instalada antes de Pontos 2008-04

Advertencia: Instalaciones viejos no recibirá insserv automáticamente, debido a enlaces simbolicas o initscripts de usuarios NO es recommendado instalar insserv en instalaciones anterior a Pontos. Este paquete debe ser utilizado con cuidado, porque incorectos o faltante LSB dependencias de initscritps existentes puede resultar en un sistema que no arranca. Instalando insserv en una instalación existente puede requerir cambios manuales.

Lea las documentación de Debian, y las notas de la versión antes de instalar y activar el insserv.

Gestor de Inicio: (Boot-Up Manager - bum) - Herramienta de Configuración de Servicios Gráficas

Si el logico de arranque de una sistema Debian no esta muy claro y familiar, no debe jugar con los symlinks, permisos y etcétera. Para evitar desconfigurar la sistema, el Boot-Up Manager ayudará en automatizar la configuración.

Boot-Up Manager es un Editor GUI de Configuración de Niveles de Ejecución que permite la configuración de cuales servicios de init están invocado cuando inicia o reinicia el sistema. Se muestra una lista de todos los servicios que pueden ser arrancada al inicio. Puede cambiar el estato de servicio, si o no, de servicios individuales.

apt-get install bum

Para utilizar el Boot-Up Manager GUI:

$ sux
password:
bum

Documentación detallada sobre el Boot-Up Manager (bum).

Matando un servicio o proceso

pkill es muy útil porque es legible por humanos y puede funcionar tanto como usuario y como root en un terminal o en el ttyX.

pkill -n service

Si no esta seguro de como deletrar el proceso o servicio que desea matar: pkill <tab> <tab> provea una lista

htop es un muy buen alternativo. (Intenta: killall -1 primero; killall -9 es el ultimo recurso).

sidux Niveles de Ejecución - init

Esta es la lista de runlevels para el sistema operativo *sidux*. Nota que si es diferente que los runlevels del Debian estable:

  • init 0 encendido el PC
  • init 1 modo un sólo usuario (single user mode)
  • init 2 modo Multi-Usario sin Red, y/o a parar o no entrar X,
  • init 3 modo Multi-Usario con Red, no ejecutando el sistema de ventanas-X, y/o a parar o no entrar X
  • init 4 a parar o no entrar X
  • init 5 modo Multi-Usario con Red ejecutando el sistema de ventanas-X, y/o a empezar X
  • init 6 rearrancar

Claves de raíz perdidos

No se puede recuperar un clave perdido, pero si se puede establecer una nueva.

Primero, arranque desde el LiveCD.

Como raíz, montar su partición-raíz (por ejemplo: /dev/sdb2)

mount /dev/sdb2

Ahora, chroot hacia su viejo partición-raíz con y fijar un nuevo clave de acceso:

chroot /media/sdb2 passwd

Fuentes de letras en sidux

Configuración correcto de dpi - Filosofia Básica

Configuraciones de dpi son problematica para divinar, pero actualmente, están realizado perfectamente por X, si el DisplaySize (tamaño de pantalla) provea las dimensiones fisicas en la sección "Monitor Section" de /etc/X11/xorg.conf.

Resoluciones Correctos y la Frecuencia de actualización

Cada monitor tiene una combinación perfecto suyo, pero, desafortunamente, no todos reporta los valores DCC correctos, y a veces, necesita tomarse un control manual.

Drivers Correctos de Dispositivos Graficos

Algunas de las más nuevas tarjetas de ATI y Nvidia simplemente no juega bien con los drivers libres de Xorg, y la unica solución razonable en estos casos son los drivers commercial de fuente cerrado. sidux no lo instalarlo previamente por razones legales, pero las soluciones puede encontrarse aquí.

Pre-selecciones de Fuentes de letras, Renderización y Tamaños

sidux utiliza fuentes pre-seleccionadas (Debian) que estan probada de ser muy balanceada, cambios en la selección de fuentes puede/podría deteriorar la calidad de la renderización. Pero hay unas opciones poderosos en Debian (apart from KDE>systemsettings ) que puede proveer renderización perfecto con otros fuentes, también. Seas consciente que cada fuente tiene solamente unas pocas tamaños perfectos, y que otras tamaños puede tener conflictos.

Variando el tamaño del "dpi" on el comando puede ser de ayuda:

fix-dpi-kdm

Debe mostrar el DPI para la tamaño del pantalla, pero puede jugar con esto también. Tendras que cambiar a "init 3" y despues a "init 5" para que funciona, o hacer un rearranque.

Después de cambiar el tipo de fuente de letra o el DPI (en la configuración de X o Firefox/Iceweasel), puede ser que necesita algunas ajustes para conseguir resultados que le gusta, especialmente después de un cambio de tipo Bitmap a una de TrueType o, viceversa:

dpkg-reconfigure fontconfig-config

Escoge nativo y autohinter en automaitica. De otra manera, variarlo a ver los resultados.

Si eso no funciona, puede que tiene que rehacer el cache de fuentes de letras, con lo siguiente:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Aplicaciones basada en GTK como el Firefox/Iceweasel

Las aplicaciones basada en GTK, son problematic en lo general con las fuentes de letras preseleccionadas por Kde. Este se puede resolver con las siguientes pasos:

apt-get install gtk2-engines gtk-qt-engine-kde4

In System Settings >Appearance you will have a new menu item called GTK Styles and Fonts. Set 'GTK Styles' to use 'Clearlooks' and have the 'GTK Fonts' set to use 'KDE fonts' or experiment with the various options to suit your preferences.

Este puede mejorar la renderización de letras en aplicaciones GTK

CUPS

KDE tiene una sección incorporado en su Ayuda, sin embargo, los dist-upgrade's a veces causa algunas fallas en el CUPS. Aquí esta una solución conocido y probada:

modprobe lp
echo lp >> /etc/modules 
apt-get remove --purge cupsys cups
apt-get install cups
	ó
apt-get install cups cups-driver-gutenprint hplip

Entonces, en el navigador (browser) de internet:

http://localhost:631

Asegurar que CUPS esta ejecutando:

/etc/init.d/cups restart

Otra trampa es cuando configurar CUPS por el GUI, sube una casilla pidiendo el clave de root, pero la casilla ya contiene su nombre de usuario, entonces cuando entra su clave de usuario -no funciona. Lo que realmente esta pidiendo es que cambio su nombre de usuario a root y entrar su clave de root.
