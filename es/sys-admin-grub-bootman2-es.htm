Para escribir Grub de nuevo

Para escribir un nuevo instancia de Grub al MBR del disco duro, por ejemplo para una nueva versión:

/usr/sbin/grub-install --recheck --no-floppy /dev/sda

Este línea puede requerir ejecución más de una vez, hasta que esta convencido que es lo que deseas.

El sector de arranque (MBR) fue sobre-escrito por Windows/o como recuperar el GRUB

Para reescribir Grub al MBR y/o recuperar a Grub en general, rearranca el sistema con el sidux live-cd, primero:

  1. Para identificar y confirmar los particiones (ejem: [h,s]d[a..]X) todos las acciones siguientes se requieran privilegios de raíz, entonces para volverse raíz (# root): 
    $ sux
  2. Cuando esta en raíz: 
    fdisk -l
cat /etc/fstab
    Eso es para saber los nombres correctos.
  3. Cuando acierto la partición correcto, crea el punto de montaje: 
    mkdir -p /media/[hdxx,sdxx,diskx]
  4. y ahora montarlo: 
    mount /dev/xdxx /media/xdxx
  5. Proximo, escribir Grub al MBR del primer disco duro (generico): 
    /usr/sbin/grub-install --recheck --no-floppy --root-directory=/media/xdxx /dev/sda

GRUB el gestor de inicialización - que es?

NOTA: El Grub usa "sdax" para los kernels 2.6.22 y mayor
y la línea:
initrd (hd0,x)/sidux/miniroot.gz
ha cambiado a:
initrd		/boot/initrd.img-2.6.xx.x-slh-smp-x

En general

Grub es un gestor de inicialización que puede arrancar sistemas operativas desde particiones distintas. Grub no depende en una lista de bloques para conseguir los archivos requerido para arrancar, y puede leer un grupo de archivos inherentemente: ext2, ext3, reiserfs, ... (Linux), Fat32, Fat16 (Windows), UFS (BSD), Minix y muchas otras.

Grub sabe de extensiones-BIOS (que significa que puede arrancar particiones arriba de la barrera de 8 Gigabyte!) y no tiene una barrera a los 1024-cilindros. Grub realmente no importa dondes resida el kernel físicamente, solamente necesita la declaración de la partición y la ruta al kernel a arrancar (que puede ser un enlace simbólica). Después de un cambio en la configuración (por ejemplo, el "menu.lst") el Grub no necesita ser ejecutada como Lilo. La nueva configuración sera aplicada al próxima inicialización.

sidux usa Grub para arrancar desde el CD, y cuando se instala al disco duro, el Grub es la preferida. Sin embargo, el Grub puede ser instalado después sin problema. Grub se instala al MBR de una consola como root con el siguiente comando:

grub-install --root-directory=/media/hdaX /dev/hdY

/media/hdaX significa la partición montada para leer, en la cual, en la directorio "/boot/grub" los archivos necesarios (stage2, stage1_5xxx) serán ubicada. El archivo de configuración "menu.lst" también tiene que ser creada en este directorio. /dev/hdY es el disco duro donde el Grub será instalada en el MBR, normalmente /dev/hda.

Nombres de discos / Nombres de dispositivos

Por favor, Nota: El uso de hda es en proceso de reemplazo, por este razón, debe empezar sustituyiendo hda con sda para los kernels 2.6.22 y después.

Grub no sabe la diferencia entre los discos SCSI y IDE. (hd0) es siempre la primer disco duro que es primero en la secuencia de arranque (según la configuración del BIOS), sin importar si es IDE o SCSI. La manera de contar de Grub es poco usual: la primer partición de la primer DD es (hd0,0). Particiones primarios están enumerado de 0-3, los particiones lógicas están enumerado desde 4 para arriba.

Grub Linux Etiqueta
(fd0) /dev/fd0 Floppy (bajo Windows)
(hda0) /dev/hda primer DD
(hd0,0) /dev/hda1 Part.1 en hd1
(hd0,5) /dev/hda6 Extendido part. en hd1

Todas las denominaciones para dispositivas para Grub están en paréntesis (). Ten cuidada con las entradas en la línea del Kernel [en el menu.lst]. A diferencia de Grub, el Kernel necesita los datos contando en la manera normal para Linux. Las entradas en el siguiente ejemplo, ambos apuntan a la misma partición: la cuarta partición en el primer disco duro. La primera línea en notación de Grub, y la segunda es en la notación típica de Linux:

root (hd0,3)
kernel          /boot/vmlinuz root=UUID=bf955abf-ce1f-45a3-9ee5-93b5c5dc4fe0 ro quiet vga=791
initrd          /boot/initrd.img

Modo de comando (Grub-Shell)

Una característica muy cómodo es el Modo de Comando de Grub. Se le entra, tecleando el "c", y abre el modo de comando de Grub. Este permita buscar y arrancar diferentes sistemas y archivos (p.e., Kernel) aún con una menu.lst con errores o faltas.

Para hacer un poco de investigación, puede ejecutar Grub de una sistema ejecutando, desde una consola-raíz, con "grub". En el modo de comando Grub puede ver todos los posibles comandos con "help". Ayuda en detalla para cada comando es disponible con help comando. Entonces, regresa a la consola con "quit" (modo de comando Grub es terminado). En el modo de comando Grub la tecla Tab tiene una función similar a la de bash, lo cual es un gran ayuda a los dedos. Comandos, nombres de discos y nombres de archivos puede ser autocompletado con la tecla Tabulador.

Grub le da una oportunidad de ubicar el Kernel antes del arranque actual con el "find" , por ejemplo, con el siguiente comando:

find /boot/vmlinuz

Grub muestra las particiones que tiene un Kernel con el nombre "vmlinuz" en el directorio "/boot":

(hd0,3) (hd1,0)

Todos los particiones disponibles puedan ser mostrada con "geometry", por ejemplo:

geometry (hd0)

muestra esto:

"drive 0x80: C/H/S = 3648/255/63, El número de sectors = 58605120, /dev/hda Partición num: 0,

Sistema arch. es fat, partición tipo 0xb Partición num: 2,

Después del Kernel o un existente initrd (o miniroot.gz) ha sido ubicado, puede arrancar con "boot".

Arrancada Dual y Múltiple

NOTA: El Grub usa sdax para los kernels 2.6.22 y después

Para habilitar GRUB para arrancar multiples sistemas operativos en muy sencilla con el uso de chainloading. Si por ejemplo, desea ejecutar MS Windows (TM), Debian etch junta con un sidux 'fromiso', necesita ajustar el GRUB un poco. Entonces, editar el archivo /boot/grub/menu.lst y agregue los siguientes líneas después de la línea ### END DEBIAN AUTOMAGIC KERNELS LIST:

####MS Windows (TM) perhaps even MS Vista (TM)
Title Windows (95/98/XP/Vista
rootnoverify (hd0,1)
makeactive
chainloader +1

### Debian etch
title Debian etch
kernel (hd0,3)/debian/vmlinuz root=/dev/hda4 ro (or whatever)
initrd (hd0,3)/debian/miniroot.gz (or (whatever)

### ISO boot
title sidux 32bit from ISO
kernel (hd0,5)/sidux/vmlinuz boot=fll quiet vga=791 fromiso=/sidux/sidux.iso
initrd (hd0,5)/sidux/miniroot.gz

Algunas necesitan/desean varios instalaciones de Windows. Windows siempre quiere arrancar desde el disco 1. Tendremos que esconder una partición y activar otra. Los siguientes comandos debe ser entrada en el modo de comandos Grub y, si tiene éxito, copiarlos al "menu.lst". Este debe ser realizado por cada partición-Windows. No olvido de hacer un respaldo! ;-)

hide (hd0,0) # partición escondido 1 del disco 1
unhide (hd0,1) # hace partición 2 del disco 1 visible
rootnoverify (hd0,1) # hace partición 2 del disco 1 el sist. archiv.-raíz
makeactive # hace partición 2 del disco 1 activo
chainloader +1 # apunta al 1er sector de la partición de arranque
boot # arranca el sistema operativa

Si deseas arrancar una segundo Windows desde el 2do disco, virtualmente intercambias el disco 1 y 2. Así, Windows veas un disco 1 simulado:

map (hd0) (hd1)
map (hd1) (hd0)

Peligros

Como Grub no tiene ningún barrera de 1024-cilindros y puede arrancar sistemas arriba de los 1024 cilindros, el "/boot/grub/stage2" tiene que estar accesible por el BIOS (tiene que estar antes los 1024 cilindros). El Stage2 no puede ser movido después de instalación (por ejemplo, el Defrag puedo intentarlo). Datos de configuración serán escritos que no serán congruente con realidad. Es razonable de hacer el stage2 lea-solamente (tiene que estar en el directorio corriente):

Dos: attrib +r +s stage2
LINUX: # chmod a -w ./stage2
