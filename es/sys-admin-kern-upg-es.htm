Instalar un nuevo kernel

Los kernels de sidux estan en apt, Por favor, esgoje el reflejo sidux más cercana a su ubicación, y modifica su sources.list:

Los kernels están disponibles en las siguientes formas:

  • sidux-686 sidux-686 - kernel para la Familia de Procesadores i686 con nucleo sencillo/dual ó más CPUs
  • sidux-amd64 kernel para sidux de 64 bit
Son 4 pasos de hacerlo manualmente en una consola:

1. Cambiar al usuario root en una consola:

sux
apt-get update

2. Para instalar el último versión del kernel:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. Ya la actualización está realizada. Ahora, si los requieres, agregue el último driver gráfica para tarjetas de ATI, NVIDIA o INTEL. Primero, actualiza el script:

sgfxi -U

4. Ahora, reinicia el computador para reincicar con el nuevo kernel (debe inicializar por defecto).

Para actualizar los drivers de 3D, ejecuta:

sgfxi

Alternativamente, en la primer inicialización con el nuevo kernel, vaya directamente al nivel de ejecución 3 (puede hacerlo agregando 3 al final de la línea de comando de Grub) y entonces, ejecuta sgfxi.

Información más detallada esta disponible aquí.

Si el nuevo kernel presenta problemas, puede reinicializar y escoger un kernel más viejo.

Modulos

Para saber cual de los modulos necesita, el siguiente comando muestra una lista de los modulos disponibles corientemente. Copia este línea al consola/terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Para tener una descripción completo de cada modulo, copia este línea al consolo/terminal:

apt-cache search 2.6.*.slh.*-sidux

Para instalar los modulos requeridos (por ejemplo, virtual-ose y qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EJEMPLO)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EJEMPLO)

Para revisar los modulos cargado en el nuevo kernel:

ls /sys/module/
or
cat /proc/modules

IMPORTANTE: si necesita cierto driver modulo, e.g., para el wlan, tiene que instalar el paquete .deb apropriado para ello. Como ejemplo, para el Agere et131x adaptador de ethernet: 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb solamente es utilizada si ya tiene los debs localmente, pero aun así, si estan en el apt-cache, el apt-get es la mejor opción.

Otras métodos de instalación del kernel

smxi

El script smxi ofrece un método integrado para instalar kernels, drivers, y algunas otras scripts para la administración del sistema:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Además, puede instalar kernels con el centro de control GUI, el siduxcc: lo cual está ubicada en el Menú de Inicio: KDE-Start-Menu>System>sidux Control Centre. siduxcc y también ofrece varios otro aplicaciones GUI para la administración de sistema

Removiendo kernels viejos

Después de la instalación exitosa de un nuevo kernel, los viejso puede ser removidos; sin embargo, es recomendada que lo dejas por unas días por si acaso se encuentra algún problema, y así necesita iniciar con un kernel más viejo, como están listado en la pantalla de inicia del Grub

Lo kernels viejos puede ser removido del sistema. Para realizarlo, puedo usar el script kernel-remover que esta en el paquete "sidux-kernelhacking". Este script tiene que ser instalado, y permite que seleccione los kernels que desea remover.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Alternativamente, puede usar el siduxcc, que utilizas el script de "kernel-remover ".
