Instalando un nuevo kernel

Los kernels de sidux están aqui y están disponible en las siguientes formas:

  • up - Kernel para la familia de procesadoras i686
  • smp - Kernel para la familia de procesadoras i686 con nucleo (core) dual o más CPUs
  • 64-smp - Kernel para la familia de procesadoras AMD64, cuando utiliza el sidux 64 bit

Haga una nota del nombre del kernel que requiere (aqui lo llaman archive kernel.zip, pero tendra un nombre como: kernel-2.6.20.4-rc1-slh-smp-1.zip), entonces anotalo:

Para instalar como usuario
Se requiere 5 líneas para hacerlo manualmente en una konsola:
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.20.4-slh-up-1.zip
unzip kernel-2.6.20.4-slh-smp-1.zip -d kernel-2.6.20.4-rc1-slh-smp-1
cd kernel-2.6.20.4-rc1-slh-smp-1
./install-kernel-sidux.sh (it will ask for your root password)
Para instalar como usuario raíz (root)
sux
cd /usr/src
wget -Nc http://sidux.com/files/kernel/kernel-2.6.20.4-slh-up-1.zip
unzip kernel-2.6.20.4-slh-up-1.zip  -d kernel-current
cd /usr/src/kernel-current
sh install-kernel-sidux.sh

Ahora, rearranca el computador. El nuevo kernel será preseleccionada para arrancar. Si este nuevo presenta problemas, rearrancará y seleccione un kernel más viejo.

IMPORTANTE: Si necesita modulos de drivers particulares, por ejemplo, para wlan, tendra que instalar el paquete deb apropiado. Si está usando el madwifi, tendra que instalar el paquete para madwifi: # dpkg -i madwifi-modules*.deb

La actualización del kernel ya esta realizado. Si se requiere, puede agregar el driver de graficos más actualizado que puede ser instalado según los drivers de 3D drivers. Si tiene una tarjeta ATI o NVIDIA, el primer arranque con el nuevo kernel debe ser a nivel 3 (runlevel 3 este se realiza agregando 3 a la línea de comando de GRUB)

La manera semi-automatica de instalar kernels es con el uso del script, du-fixes-h2.sh:

Remover kernels viejos

Depués de una instalación exitoso de un nuevo kernel, el kernel.zip (y los archivos del kernel) puede ser borrados, aunque, esta recomendado que espera algunas días, por si acaso tiene problemas; y así puede arrancar con un kernel más viejo, como esta listado en la pantalla del GRUB al inicio.

Los kernels viejos puede ser removido del sistema. Para realizarlo, puedo usar el script kernel-remover lo cual esta en la paquete, sidux-kernelhacking. Este script permite que puede seleccionar los kernels que deseas remover.
