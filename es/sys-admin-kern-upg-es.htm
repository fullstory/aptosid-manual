Instalar un nuevo kernel

Los kernels de sidux se encuentran aquí. Selecciona una cerca de su ubicación:

Main Mirrors
http://debian.tu-bs.de/project/sidux//kernel Germany
http://sidux.com/files//kernel Germany
http://sidux.net/files//kernel Germany
ftp://ftp.spline.de/mirrors/sidux/kernel/ Germany

These mirrors can be as much as 24 hours behind the main mirrors
http://www.mirrorservice.org/sites/sidux.com/sidux/kernel United Kingdom
http://ftp.heanet.ie/pub/sidux//kernel Ireland
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernel  TN, USA
http://ftp-mirror.internap.com/pub/sidux/kernel  GA. USA
http://ftp.tw.debian.org/pub/Sidux//kernel Asia
http://mirror.aarnet.edu.au/pub/sidux/kernel Australia
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernel South Africa

Los kernels están disponibles en las siguientes formas:

  • sidux-686 - Kernel para la familia de procesadoras i686 con nucleo (core) single/dual o más CPUs
  • sidux-amd64 - Kernel para la familia de procesadoras AMD64, cuando utiliza el sidux 64 bit
Its 4 steps to do it manually in a konsole:
apt-get update

1. Get the last version kernel:

wget -O - http://sidux.com/debian/pool/main/l/linux-2.6-sidux 2>/dev/null | awk "/linux-image.*$(dpkg-architecture -qDEB_HOST_ARCH_CPU)/" | sed -e 's/^.*linux\-image\-//' -e's/_.*\.deb.*$//'

2. Find which modules you need. Following command gives you the list of current available modules:

apt-cache search 2.6.24-2.6.24.2.slh.6-sidux| awk '/modules/{print $1}'

To get a full description of each module:

apt-cache search 2.6.24-2.6.24.2.slh.6-sidux

3. To Install the kernel, headers and the required modules (for instance virtual-ose, and qc-usb):

apt-get install  linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686
apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.6-sidux-686
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.6-sidux-686

Ahora, rearranca el computador. El nuevo kernel será preseleccionada para arrancar. Si este nuevo presenta problemas, rearrancará y seleccione un kernel más viejo.

Para verificar los modulos cargada en el kernel:

ls /sys/module/
ó
cat /proc/modules

IMPORTANTE: Si necesita modulos de drivers particulares, por ejemplo, para wlan, tendra que instalar el paquete deb apropiado. As an example, for webcams and the gspca module, you have to install the .deb package for gspca: 

dpkg -i gspca-modules*.deb

Ver conectandose en línea y ndiswrapper y también este para WIFI

La actualización del kernel ya esta realizado. Si se requiere, puede agregar el driver de graficos más actualizado que puede ser instalado según los drivers de 3D drivers. Si tiene una tarjeta ATI o NVIDIA, el primer arranque con el nuevo kernel debe ser a nivel 3 (runlevel 3 este se realiza agregando 3 a la línea de comando de GRUB)

La manera semi-automatica de instalar kernels es con el uso del script, du-fixes-h2.sh:now called smxi, which is short for sidux-maintenance:
apt-get update
apt-get install sidux-scripts
smxi

You can also do this with the GUI control centre siduxcc: KDE-Start-Menu>System>sidux Control Centre

Quitar kernels viejos

Depués de una instalación exitoso de un nuevo kernel, el kernel.zip (y los archivos del kernel) puede ser borrados, aunque, esta recomendado que espera algunas días, por si acaso tiene problemas; y así puede arrancar con un kernel más viejo, como esta listado en la pantalla del GRUB al inicio.

Los kernels viejos puede ser removido del sistema. Para realizarlo, puedo usar el script kernel-remover lo cual esta en la paquete, sidux-kernelhacking. Este script permite que puede seleccionar los kernels que deseas remover.
