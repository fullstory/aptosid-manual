Instalar un nuevo kernel

Los kernels de sidux estan en apt, Por favor, escoge el reflejo sidux más cercana a su ubicación, y modifica su sources.list:

Los kernels están disponibles en las siguientes formas:

  • sidux-686 sidux-686 - kernel para la Familia de Procesadores i686 con nucleo sencillo/dual o más CPUs
  • sidux-amd64 kernel para sidux de 64 bit
Pasos para hacerlo manualmente en una consola:

1. Cambiar al usuario root en una consola:

sux
apt-get update

2. Para instalar el último versión del kernel:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

Reiniciar para utilizar el kernel nuevo

Si el nuevo kernel presenta problemas, puede reinicializar y escoger un kernel más viejo.

Módulos

Para saber cual de los módulos necesita, el siguiente comando muestra una lista de los módulos disponibles corientemente. Copia este línea al consola/terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Para tener una descripción completo de cada módulo, copia este línea al consolo/terminal:

apt-cache search 2.6.*.slh.*-sidux

Para instalar los módulos requeridos (por ejemplo, virtual-ose y qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EJEMPLO)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EJEMPLO)

Para revisar los módulos cargado en el nuevo kernel:

ls /sys/module/
or
cat /proc/modules

Instalando módulos con "Dynamic Module-Assistant Kernel Module Support" (dmakms)

dmakms es útil para instalar módulos de kernel que no estaba pre-compilado para el kernel de sidux, y es diseñada para automátizar la instalación de módulos del kernel con el "module-assistant" (m-a) cuando están actualizando o instalando un nuevo kernel de Linux.

apt-get install dmakms module-assistant

Antes de activar el "Dynamic Module-Assistant Kernel Module Support" (dmakms), primero útiliza el "module-assistant" para instalar el/los modulo(s) deseada para el kernel Linux actualmente ejecutando. Para más información acerca del "module-assistant" por favor leer su página manual:

man m-a

El nombre del paquete compatible con el module-assistant entonces, necesita ser agregado a /etc/default/dmakms, para que el proceso de preparar y instalar los mismos modulos para cada nuevo kernel Linux puede ser automátizado.

Ejemplo: Instalando el modulo speakup con el module-assistant

Asegurar que el sources.list tiene contrib non-free agregada en la línea de sources en: /etc/apt/sources.list.d/debian.list 

apt-cache search speakup-s
speakup-source - Source of the speakup kernel modules

Entonces, preparar el módulo:

m-a prepare
m-a a-i speakup-source

Ahorra, activar para speakup, el Dynamic Module-Assistant Kernel Module Support, así que la próxima vez que el kernel Linux está actualizada, un módulo speakup es preparada para el también, sin intervención manual. Para hacer eso, agregue speakup-source al archivo de configuración /etc/default/dmakms . 

mcedit /etc/default/dmakms
speakup-source

Repetir el mismo proceso para cualquier otro paquete de módulo kernel compatible con module-assistant.

Un paquete de linux-headers tiene que ser instalada para cada paquete de linux-image para que el module-assistant compilar los módulos del kernel nuevo.

Fallo al cargar Módulo

Si el módulo falla a cargar para cualquier razón [una nuevo componente de xorg, problema de sistema de archivos o si el sistema-X no arranca después de una reinicialización] :

modprobe <module>

Entonces, reiniciar el computador.

Si todavía falla en cargar:

 m-a a-i -f module-source

Este reconstruir el módulo entonces reiniciar.

Como lo Funciona:

El "Dynamic Module-Assistant Kernel Module Support" consta de una sola initscript (/etc/init.d/dmakms) que ejecuta a inicialización o por un script llamada después de instalación de nuevos paquetes de linux-image de Debian.

En cada inicialización, el /etc/init.d/dmakms esta ejecutada para descubrir si cada módulo listado en /etc/default/dmakms ha proveedo un paquete módulo para el kernel Linux actualmente ejecutando, y llamando el module-assistant para contruir e instalar un paquete módulo según requerido.

Cuando un nuevo paquete de linux-image de Debian está instalada, /etc/init.d/dmakms está ejecutada vía un script de engancho de postinst con dos argumentos, 'start' y el 'version string' del kernel Linux para el cual debe preparar los nuevo paquetes de módulos. En este momento, los paqutes módulos listado en el archivo /etc/default/dmakms están procesado con module-assitant, y los paquetes derivados están planificado para instalación al momento de apagar la maquina. El razón para que la instalación ocurre al momento de apagar es porque es necesario asegurar que el apt/dpkg no están en uso por ningún otro proceso.

$ /usr/share/doc/dmakms

Otras métodos de instalación del kernel

siduxcc

Además, puede instalar kernels con el centro de control GUI, el siduxcc: lo cual está ubicada en el Menú de Inicio: KDE-Start-Menu>System>sidux Control Centre. siduxcc y también ofrece varios otro aplicaciones GUI para la administración de sistema

Removiendo kernels viejos (kernel remover)

Después de la instalación exitosa de un nuevo kernel, los viejso puede ser removidos; sin embargo, es recomendada que lo dejas por unas días por si acaso se encuentra algún problema, y así necesita iniciar con un kernel más viejo, como están listado en la pantalla de inicia del Grub

Lo kernels viejos puede ser removido del sistema. Para realizarlo, puedo usar el script kernel-remover que esta en el paquete "sidux-kernelhacking". Este script tiene que ser instalado, y permite que seleccione los kernels que desea remover.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Alternativamente, puede usar el siduxcc, que utilizas el script de "kernel-remover ".
