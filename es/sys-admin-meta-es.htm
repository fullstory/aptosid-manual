Meta-Paquetes

Durante la instalación al disco duro, también puede escoger Meta-Paquetes de varios aplicaciones y juegos para incluir en su instalación.

El instalador de Meta-Paquetes también esta disponible después de la instalación al disco duro en:
Kmenu >sidux >sidux-meta-installer

Hasta el momento, la lista incluye los siguientes:
+ Discos: escanear y reparar particiones
+ Educación
+ Juegos
+ Idiomas
+ Multimedia (VDR)
+ Redes
+ No-libre
+ Oficina
+ Computadores más viejos
+ Servidor
+ Gestor de ventanas

Nota: Después que ejecuta el instalador de meta-paquetes, veras una línea como lo siguiente

pega su código aquí: (no necesario para instalación normal, sólo para preconfig)

Solamente necesita oprimir la tecla: Entra.

Este list esta siendo actualizado de vez en cuando (revisa el Instalador-Meta en el menú de sidux. La lista esta corriente al 20/05/2007

Meta-Instalador y Proxies

El Meta-instalador tiene que ser ejecutado como root, y si la configuración no esta en el archivo apt.conf, no pueden ser leido. Hay dos maneras de activar el uso de un proxy, y el Método 1 es los mejor porque es permanente:

Método 1

Definir el proxy en el archivo apt.conf, editarlo como root con kwrite /etc/apt/apt.conf y agregar los siguiente:

Acquire{
HTTP::proxy "http://192.168.1.1:3128";
FTP::proxy "http://192.168.1.1:3128";
}

Método 2

Abrir una consola con sux para volverse raiz para fijar los variables del entorno, http_proxy y ftp_proxy, entonces llama al Meta-instalador. Tendra que hacer esto cada vez que desea acesar el Meta-instalador:

export http:proxy=http://192.168.1.1:3128"
export ftp_proxy="ftp://192.168.1.1:3128/"
