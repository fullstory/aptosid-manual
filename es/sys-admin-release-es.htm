Notas sobre "Rolling"

sidux esfuerza para hacer una ruta de actualización constante sin fallas para Debian y los paquetes proveido de sidux, sin embargo, quedan algunas cosas que puede mejorar con intervención manual.

Este página solamente será actualizado después de cada versión estable. La última versión estable es Pontos.

Advertencia de Seguridad de sidux/ Enclavamiento $HOME

Mientras trabajando en el instalador de sidux, una potencial asunto de escalación de privilegios locales ha sido descubierto, que deja una copia del archivo de configuración del instalador en el sistema nueva. Este archivo de configuración contiene semillas para los "hashes" de claves MD5 de los cuentos de raíz y el usuario primero (UID 1000) y es legible para usuarios locales. Esto no revela las claves directamente, pero si puede ser usado para un ataque de fuerza bruta contra ellos.

Versiones Afectadas:

  • 2007-01 Χάος (Chaos)
  • 2007-02 Τάρταρς (Tartaros)
  • 2007-03/ 2007-03.1 Γάια (Gaia)
  • 2007-04/ 2007-04.5 Έρως (Eros)
  • 2008-01 Νυξ (Nyx)
  • 2008-02 Έρεβος (Erebos)
  • 2008-03 Ουρέα (Ourea)

La Corrección:

find /root /home -maxdepth 2 -type f -name '.sidconf*' -delete

Especialmente en sistemas de multi-usuarios, es recommendado además de cambiar las claves de raíz y para el primer usuario (UID 1000), que puede ser realizado ejecutando "passwd" como raíz. La razón para este error ha sido encontrado y la versión de sidux 2008-04 "Πόντος" no está afectada.

Debian crea directorios "home" con permisos de acceso 0751, sidux cambia este acción preseleccionada y aplica 0700 para nueva instalaciones. 

sed -i 's/^DIR_MODE=.*/DIR_MODE=0700/' /etc/adduser.conf
dpkg-reconfigure adduser
(denies "system-wide readable home directories")

Capa de Compatibilidad de ALSA OSS

Empezando con alsa-driver 1.0.17.dfsg-3, los modulos de compatibilidad de OSS no están cargado por omissión. Mientras que este no posar problema en sistemas modernos, algunás aplicaciones comercial o viejos puede todavia requerir soporte para OSS. Dada que la emulación para OSS suministrado en kernels con soporte ALSA es algo limitado (nada de mezcla basado en software, entonces solamente puede ser una sola aplicación accesando /dev/dsp a la vez), os aplicaciones compatible con ALSA deben ser preferidos en lugar de OSS, pero si compatibilidad con OSS es requerida hay dos alternativas disponibles:

  • aoss wrapper
    /usr/bin/aoss es parte del paquete alsa-oss y puede envolver aplicaciones legatos para atrapar llamadas a OSS a través de mapeo transparentemente del API de OSS en ALSA; que permite la aplicación legato aprovechar la mayoria de caracteristicas de ALSA.
  • in-kernel ALSA emulation
    mejor compatibilidad (emulado /dev/dsp, /dev/audio y /dev/mixer device nodes están creado), pero este no permite mezclas basadas en software, entonces solamente puede tener un aplicación de sonido activo a la vez.

Para la mayoria de aplicaciones es preferido de iniciar la aplicaciones afectado con "aoss <program>" para mantener las ventajas de mezcla basada en software y acceso concurrente al hardware de sonido, pero puede ser casos que requiere la capa de compatibilidad en el kernel. Este puede ser activado, si es necesario, a través de cargar los modulos de snd-pcm-oss, snd-mixer-oss y snd-seq-midi. 

for i in snd-pcm-oss snd-mixer-oss snd-seq-midi; do
	grep -q "^${i}" /etc/modules  || echo "${i}" >> /etc/modules
	grep -q "^${i}" /proc/modules || modprobe "${i}"
done

Broadcom b43 wlan

Kernels >2.6.25 require revisiones de firmware ≥4.150.10.5 (internal revision "410.2160 (2007-05-26 15:32:10)") del firmware Broadcom, mientras que los viejos (hasa el kernel 2.6.24) requiere 4.80.53.0 (internal revision "351.126 (2006-07-29 05:54:02)"), que ha sido instalado por revisiones más viejos del b43-fwcutter. b43-fwcutter no puede actualizar imagenes existente de firmware sin invervención manual. 

dpkg --purge b43-fwcutter
rm -rf /lib/firmware/b43 /lib/firmware/b43legacy
apt-get update
apt-get install b43-fwcutter

console-setup

Las mapas de teclados de la consola están manejado por "console-setup". 

apt-get install console-setup
dpkg-reconfigure console-setup

gspca2

El kernel 2.6.27 tiene el nuevo gspca2 driver basada en v4l2, que reemplaza el viejo v4l1, que es requerido para un gran numero de webcams diferentes. Desafortunamente, la mayoria de programas en el espacio del usuario no están consciente del nuevo API v412 para dispositivas anteriormente apoyado por el gspca, mientras tanto, necesitan ser iniciado con este: "LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <program>". 

apt-get update
apt-get install libv4l-0

AS USER
$ LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <program>

insserv

insserv en sistemas instalado antes del Pontos 2008-04

Advertencia: Instalaciones viejos no recibira insserv automáticamente, debido a los enlaces simbolicos e initscripts de los usuarios. NO está recommendada instalar insserv en instalaciones anteriores a Pontos. Este paquete debe ser usado con cuidado, porque uso incorrecto o falta de dependencias de LSB de initscripts existentes puede llegar a una sistema que NO inicia. Instalando insserv en una instalación existente puede requerir algunas cambios manuales..

Lea lea las notas de documentación de Debian, y las notas de la versión antes de instalar y activar insserv.

Paquetes Obsoletas

Como parte del desarrollo continua de sidux, un número de paquetes han sido reemplazado con alternativas mejores o están redundante..

  • capi-isdnconf, obsoleta.
  • configure-xawtv, obsoleta.
  • gprsconnect, reemplazada por umtsmon.
  • ipw2100-firmware, reemplazada por firmware-ipw2x00.
  • ipw2200-firmware, reemplazada por firmware-ipw2x00.
  • modemlink, reemplazada por umtsmon y wvdial.
  • ppp-scripts, reemplazada por umtsmon.

El siguiente comando es capaz de encontrar paquetes no disponibles desde cualquier repositorio de apt activo y puede indicar información sobre paquetes obsoletos, pero ten cuidado con paquetes instalados manualmente. apt-show-versions es una paquete separado y no necesariamente esta preinstalado. 

$ apt-show-versions | awk '/No available version in archive/{print $1}'
