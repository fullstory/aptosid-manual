Actualizar con el LiveCD

La facilidad de realizar una actualización con el LiveCD de una instalación de sidux no existe. Antes de proceder instalando sidux de nuevo, RESPALDA SUS DATOS! incluyendo los atajos, favoritos del web y su correo electrónico.

Es altamente recomendado que tiene una partición separado de datos. Los beneficios en términos de recuperación de un desastre, en la estabilidad de sus datos están enormes.

Por lo tanto, el $HOME vuelve un lugar de guardar los configuraciones básicas. O de otra manera, mayormente una contenedor para que los aplicaciones guardan sus configuraciones.

SIEMPRE RESPALDA SUS DATOS, incluyendo los atajos, favoritos del web y su correo electrónico!

Actualización de la distribución (dist-upgrade) de una instalación en el Disco Duro

Has actualizado la instalación con el último LiveCD y ahora desea asegurar de tener las versiones y actualizaciones de seguridad de la último momento de ahora en adelante.

Con sidux se les llama, 'dist-upgrade', y es realizado en línea. [Banda ancha es mejor]

El único método apoyado es lo siguiente:

NUNCA realiza un "upgrade" ni "dist-upgrade" mientras se este en el ambiente gráfico X

Antes, siempre revisa la sección "Current Warnings" en la página web principal de sidux. Estos avisos están allí por una razón --debido a la naturaleza de "Debian Sid/Unstable" y están actualizados diariamente. Salga del KDE. Entre al modo texto presionando Ctrl+Alt+F1; entonces login como "root" y después teclear "init 3"; apt-get update && apt-get dist-upgrade; run fix-fonts; run fglrx o nvidia script si el xorg fue actualizado y tienes tal dispositivo gráficos, sino haga "init 5 && exit"

[NUNCA HAGA EL DIST-UPGRADE (NI UPGRADE) con adept, aptitude o synaptic]
VER: Actualización de un Sistema Instalado Hay una excepcion!! du-fixes-h2

Para nuevos usuarios, este script puede proveer un ayuda al semi-automatizar el proceso del dist-upgrade !h2 d-u fixes
