sidux Guía Rápido

ESTE ES MUY IMPORTANTE: sidux, como un Live-CD de Linux, es basado en tecnología de alta compresión, (inicialmente desarrollada por Knoppix; sin embargo, no es lo mismo que Knoppix) y por eso, cuidado especial es requerido al momento de quemar el imagen ISO. Sólo usa media de CD [o DVD+RW] de alta calidad, y quemarlo en modo-DAO (disco a la vez) / (algunas lo llaman SAO, sesión a la vez) y a la velocidad más bajo que ofrece el quemador.

sidux es 100% compatible con Debian sid. Los paquetes sidux que sobrellevan paquetes en Debian sid siempre son temporales y están utilizado hasta que el problema esta arreglado en los repositorios de sid. Los paquetes extras de sidux incluye cosas como scripts y herramientas de administración.

La estabilidad de Debian sid/inestable

sid es un sobre nombre para los repositorios de Debian inestable. Debian sid es un repositorio frecuentemente actualizado para estar sincronizada con los versiones últimos y mejores de las versiones de los paquetes mantenidos. A raíz de las actualizaciones frecuentes, menos comprobación de los paquetes es posible debido a la periodo muy corto entre los lanzamientos de desarrolladores aguas arriba y los mantenedores de los paquetes de distribución de Debian.

El kernel de sidux

esta optimizado para ayudar en evitar problemas, agregue nuevo funcionalidad, o configurar para rendimiento más rápido y/o más estabilidad, originando de la última kernel desde http://www.kernel.org/. .

El kernel esta disponible desde aquí: Actualizando el kernel

Manejo de Paquetes

sidux cumple con el Debian sid normal, común y corriente. Debian use el apt-get y dpkg para manejar los paquetes .deb de software.

Debian sid tiene más de 20,000 paquetes, así que debe ser posible encontrar lo que busca con lo siguiente: Busca para paquete con apt-cache o Busca de paquete con kio-apt

Para instalar un paquete: apt-get install <nombre de paquete> Instalando un nuevo paquete.

apt-get se usa para el manejo de paquetes en conjunto con Debian y otros repositorios identificado por los archivos en /etc/sources.list.d/*

Los repositorios de Debian sid están actualizado normalmente dos veces al día, así que la lista de paquetes disponibles cambias muy rápido. Es esencial ejecutar el apt-get update antes de instalar cualquier nuevo paquete o antes de ejecutar apt-get dist-upgrade para estar al día con la lista de paquetes en el servidor.

Variaciones de Debian, Fuente y RPMs

Instalación de aplicaciones desde el fuente no están apoyado. Si realmente necesita compilar y utilizar un aplicación no disponible en los repositorios, hazlo como un usuario, y colocarlo en su directorio /home sin instalarlo al sistema. El uso de checkinstall o conversión de RPMs con alien (y otro programas semejantes) a un deb tampoco están apoyado.

Algunas variaciones de Debian utilizan locaciones diferentes para archivos que puede resultar en inestabilidad de sistema, y algunas paquetes no se instala debido a dependencias que no pueden resolverse el apt-get o dpkg, debido a esquemas diferentes de nombramiento o de numeración. Por ejemplo, una versión diferente de glibc puede causar que la aplicación no se ejecuta.

Actualizando el Sistema: dist-upgrade

apt-get dist-upgrade es la manera recomendado y apoyado para actualizar sidux. El uso de cualquier GUI para actualizar sidux, no esta soportado. Actualización de un Sistema Instalado: dist-upgrade

Un dist-upgrade esta apoyado solamente afuera de X windows. Ejecutando "$ init 3" desde el gestor de ventanas (KDE, XFCE, etc.) o en un terminal virtual (ctrl-alt-F1, ctrl-alt-F2, etc.) termina la ejecución de X y permite que realiza la actualización con seguridad.

Configuración de Redes

Ceni es una herramienta de configuración para redes que puede configurar rápidamente la tarjeta red o inalámbrico. La función inalámbrico puede escanear para redes, usa el wep y wpa para encriptación, utilizar herramientas-inalámbrica o el wpa_supplicant para configuración inalámbrica. La conexión a ethernet es directo si usas el dhcp (asignación automática de direcciones ip) o puede configurarlo manualmente al netmask para el nameserver.

Se ejecuta Ceni con el comando Ceni o ceni . Si no esta instalado, puede instalarlo con el "apt-get install ceni".

Otros Gestores de Ventanas

KDE, XFCE y Fluxbox son los gestores preseleccionado para sidux. El Gnome no esta soportado por sidux. El Gnome es un gestor de ventanas en los repositorios de sid que es conocido de ser inestable y puede causar problemas al intentar actualizar.
