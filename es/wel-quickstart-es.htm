Guía Rápida de aptosid

ESTO ES MUY IMPORTANTE: aptosid, como Live-CD de Linux, está basado en tecnología de alta compresión, y por eso se requiere un cuidado especial en el momento de quemar la imagen ISO. Sólo use un CD [o DVD+RW] de alta calidad, y quémelo en modo-DAO/SAO (disco/sesión-a-la-vez) y a la velocidad más baja según el tipo de disco, idealmente a no más de 8x

El objetivo de aptosid es ser 100% compatible con Debian sid; sin embargo, aptosid puede proveer arreglos temporales de vez en cuando. El repositorio de aptosid contiene paquetes específicos de aptosid que incluyen kernels, scripts, herramientas y documentación de aptosid.

La estabilidad de Debian sid/inestable

sid es un sobrenombre para los repositorios de Debian inestable. Debian sid es un repositorio frecuentemente actualizado para estar sincronizado con las últimas y mejores versiones de los paquetes mantenidos. Debido a las actualizaciones frecuentes, es posible una menor comprobación de los paquetes por causa del periodo muy corto entre los lanzamientos de los desarrolladores y mantenedores de los paquetes de distribución de Debian.

El kernel de aptosid

El kernel (núcleo) está optimizado para ayudar a evitar problemas, añadir nuevas funcionalidades o ser configurado para un rendimiento más rápido y/o con mayor estabilidad, originado en el último kernel desde http://www.kernel.org/ .

El kernel esta disponible desde aquí: Actualizando el kernel

Manejo de Paquetes

aptosid sigue las normas de empaquetamiento de Debian y utiliza apt y dpkg para gestionar los paquetes de software con Debian y otros repositorios identificado por los archivos que se hallan en /etc/sources.list.d/*

Debian sid tiene más de 20.000 paquetes, así que debe ser posible encontrar lo que busca con lo siguiente: Búsqueda de paquetes con apt-cache o con Debian Package Search GUI application.

Para instalar un paquete haga: apt-get install <nombre de paquete> Instalando un nuevo paquete.

Los repositorios de Debian sid son actualizados normalmente dos veces al día, así que la lista de paquetes disponibles cambia muy rápidamente. Es esencial ejecutar el apt-get update antes de instalar cualquier nuevo paquete o antes de ejecutar apt-get dist-upgrade para estar al día con la lista de paquetes en el servidor.

Uso de los repositorios de otras distribuciones basadas en Debian, Código fuente y RPMs

La instalación de aplicaciones a partir de la fuente no está apoyada por aptosid. Si realmente necesita compilar y utilizar una aplicación no disponible en los repositorios, hágalo como un usuario, y coloquelo en su directorio /home sin instalarlo en el sistema. El uso de checkinstall o conversión de RPMs con alien (y otros programas semejantes) a un deb tampoco está apoyado.

Otras distribuciones muy conocidas basadas en Debian que "reempacan" los repositorios de Debian para sus propios repositorios, a menudo utilizan localizaciones diferentes de los archivos que puede resultar en la inestabilidad del sistema, y algunos paquetes no se instalarán debido a dependencias que no pueden resolverse con apt-get o dpkg, debido a esquemas diferentes de nombres o de numeración. Por ejemplo, una versión diferente de glibc puede causar que la aplicación no se ejecute.

Use los repositorios de Debian para instalar los paquetes de software que necesite, ya que con otros repositorios posiblemente no podrá recibir soporte.

Actualizando el Sistema: dist-upgrade

apt-get dist-upgrade es la manera recomendada y apoyada para actualizar aptosid. El uso de cualquier GUI para actualizar aptosid no se recomienda y no está soportado. Actualización de un Sistema Instalado: dist-upgrade

Un dist-upgrade debe realizarse siempre fuera de X windows, y no estará apoyado si se hace desde un gestor de ventanas. Ejecutando "$ init 3" desde el gestor de ventanas (KDE, XFCE, etc.) o en un terminal virtual (ctrl-alt-F1, ctrl-alt-F2, etc.) termina la ejecución de X y permite realizar la actualización con seguridad.

Configuración de Redes

Ceni es una herramienta de configuración para redes que puede configurar rápidamente la tarjeta ya sea de red o inalámbrica. La función inalámbrica es capaz de escanear redes, usar el wep y wpa para encriptación, utilizar herramientas inalámbricas o el wpa_supplicant para la configuración inalámbrica. La conexión a ethernet es directa si usa el dhcp (asignación automática de direcciones ip) o puede configurar manualmente el netmask y el nameserver.

Ceni se ejecuta con el comando Ceni o ceni . Si no está instalado, puede instalarlo con "apt-get install ceni".

Otros Gestores de Ventanas

KDE, XFCE y Fluxbox son los gestores preseleccionados para aptosid. Gnome no esta soportado por aptosid. Gnome es un gestor de ventanas que es conocido en los repositorios de sid por ser inestable y puede causar problemas al intentar actualizar. Puede que algunos usuarios en los foros, wiki y el IRC de aptosid tengan experiencia y puedan estar dispuestos a ayudarle. Si no, deberá confiar en sus propios recursos.
