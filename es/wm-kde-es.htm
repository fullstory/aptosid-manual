Algunas aplicaciones de KDE que no están incluido en sidux KDE Lite

Puede que se requiere activar "non-free" en /etc/apt/sources.list.d/debian.list

konq-plugins - "plugins" para Konqueror.

Este paquete contiene una variedad de "plugins" útiles para Konqueror, el gestor de archivos, navegador de internet y visor de documentos para KDE. Muchos de estos "plugins" aparece en el ménu de herramientas de Konqueror.

Los puntos más interesantes para navegando la red incluye traducciones de páginas web, archivando de páginas, recargando automáticamente, analisis de la estructura del HTML y CSS, una herramienta de buscadas, una barra al lado de noticias, acceso rápido a opciones comunes, markadores, un monitor de programas colgadas, indicador de disponibilidad de microformatos, un herramienta de marcadores para del.icio.us, y integración con aKregator el lector de alimientos RSS.

Los puntos más interesantes para navegación de directorios incluye filtros para directorios, creación de galerías de imagenes, compresión y extracción de archivos, copiar/mudar rápido, un tocador de media en el barra al lado, una barra de información de archivos, un ayudante de directorios de media, un visualizador gráfico del uso del disco, conversiones y transformaciones de imagenes.

kdeaddons - plugins y applets agregados proveedo con KDE

Este metapaquete incluye una colección de "plugins" y "applets" que viene con las versión oficial de KDE.

unrar - Extractor para archivos .rar (versión no-libre)

El unrar puede extraer uno o unos archivos desde un archivo .rar. Si desea crear archivos .rar, instalar el paquete rar.

Se traba

En tal situación, no siempre necesita el botón de reseteo. Eso puede dañar el sistema de archivos o llegar a una perdida de datos. De cualquier forma el sistema de archivos no será limpio después de un apagado duro (sistema de archivos no limpio)

Primero, intenta cambiar a una consola de texto Ctrl-Alt-F1 o rearrancar el servidor-X con Ctrl-Alt-Retroceso (backspace), (si ninguna de estos dos funciona, todavía hay esperanza):

La tecla, ImprPant/PetSis (PrintScr/SysReq), a la derecha arriba del teclado) puede ayudar rearrancar limpiamente una sistema congelada.

La siguiente secuencia de combinaciones de tecladas puede dar buen resultado:
* Alt+PetSis+r (debe regresar control del teclado)
* Alt+PetSis+s (pide un sync)
* Alt+PetSis+e (Envía "term" a todos procesos menos "init")
* Alt+PetSis+i (Envía muere a todos procesos menos "init")
* Alt+PetSis+u (sistemas de archivos están montada leer solamente, que prevea el fsck al rearrancar)
* Alt+PetSis+b (rearrancar el sistema, sin los pasos previo, sera como un apagón [hard reset]).

Es mejor de dar cada paso unas segundos para completar, terminando todos los procesos puede llevar algún tiempito. Las letras a recordar puede ser ayudada con este mnemotécnico:
"Raising Skinny Elephants Is Utterly Boring".

Recientemente, alguien sugiera la siguiente, que tiene, quizás, más sentido:
"Reboot System Even If Utterly Broken".

Problemas con el Login bajo KDE

El contenido del directorio /tmp es normalmente vaciado en cada arranque, y algunos directorios y socketas requerido por el servidor-X también están borrado.

Normalmente, durante el proceso del arranque, el script x11-common de X-Org trata de recrear estas cosas.

Puede suceder que estos scripts no sean llamados durante el proceso del arranque. En este caso, puede recrear los enlaces necesitado con lo siguiente:

# X-ORG: # dpkg-reconfigure x11-common

KDE necesita una alocación de 5% de la partición donde reside /tmp para archivos temporales al login. Si esta ejecutando con una partición que es 95% lleno, no será posible hacer el login con KDE y sera enviado a un ttyX.

Lo mismo cuando KDE esta en un bucle y no de deja hacer el login. Una solución is login a un ttyX para borrar o mover aplicaciones o archivos a otra partición.

Alternativamente, puede usar otra gestor de ventanas de X que no requiere tanto espacio del sistema (por ejemplo, fluxbox que esta presente en una instalación de sidux), o chroot usando el liveCD/DVD para limpiar el partición para permitir arrancar con el KDE.

Por esto, 85% es el uso maximo absoluto recomendado para una partición que KDE utilizas para sus archivos temporales (15% free).

Instalando Arte y Temas de KDE para sidux

Para instalar la última arte de sidux a una instalación existenete de sidux:

apt-get install sidux-art-kde-xxxx sidux-art-wallpaper-xxxx
(donde xxxx es el nombre del lanzamiento, por ejemplo, sidux-art-kde-erebos)

Esto instalará el papel tapiz y temas de sidux

Vaya al Centro de Control > Aparencia y Temas > Gestor de Temas y escoger sidux-art tema/papel tapiz de la lista.

Para activar o cambiar temas de KDM

kdmtheme es una gestionador para temas de KDM. Este provee un Modulo de Control KDE (KCM) que permite la fácil instalación, desinstalación y cambio a las temas KDM.

 apt-get install kdmtheme

Entonces, reiniciar el Centro de Control

Activar Temas manualmente

Todos estos cambios tiene que ser realizado como root.

Corrientemente, los temas-KDM no están activado al arranque en Debian.

Agregue
UseTheme=true
línea en el [X-*-Greeter]-stanza en su archivo  /etc/kde3/kdm/kdmrc.
Cambia temas

Si activado, KDM utiliza su tema de círculos, para cambiar esto, solamente cambia este línea con:

Theme=/usr/share/apps/kdm/themes/circles

a

Theme=/usr/share/apps/kdm/themes/
Desactivar temas

Si desea desactivar los temas solamente cambia a UseTheme=false.

NOTA: Tiene que reinicializa KDM después de realizar cambios a kdmrc.
