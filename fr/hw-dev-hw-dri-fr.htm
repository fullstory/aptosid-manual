Pilotes exécutables non-libres, pour Nvidia / ATI

Quatre méthodes sont utilisables pour installer les pilotes 3D concernés sur votre système

  1. sgfxi / !h2 smxi

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxi supporte actuellement les pilotes ATI fglrx et Nvidia. Il est également capable de gérer la conversion depuis ou vers les pilotes libres xorg comme ati, intel, nv. !h2 smxi utilise sgfxi comme moteur d'installation de pilotes graphiques, en y adjoignant des fonctions de facilitation pour les utilisateurs.

    Le pilote par défaut sera celui déjà installé, si aucun argument n'est passé au programme pour en changer. Sauf si votre carte était trop ancienne et avait besoin d'un pilote générique, auquel cas le script le déterminerait pour vous.

    En cours de fonctionnement après que vous l'ayez lancé, le script s'arrêtera pour vous avertir du pilote qu'il aura choisi d'installer. Vous pourrez valider son choix, ou bien sortir et recommencer en utilisant une option de remplacement si vous souhaitez quelque chose de différent.

    Vous pouvez également lancer l'installeur sgfxi en utilisant le centre de contrôle de sidux !h2 smxi, qui se sert de sgfxi pour installer les pilotes graphiques. Consultez pour ce faire la page web de !h2 smxi où vous trouverez des informations complémentaires. Lancez cette commande ensuite :

    smxi -kiwdt

    sgfxi-h: En utilisant l'option -h vous pourrez afficher une liste de pilotes actuellement gérés, ainsi qu'un bref résumé des spécificités de votre système, et notamment noyau, version de xorg. Voici quelques options utiles pour sgfxi :

    -c Utilisation du mode composite (cartes nVidia seulement).
-d Enlève IgnoreDisplayDevices, accepte une TV par exemple (cartes nVidia seulement).
-h Affiche ce menu d'aide.
-n Installe automatiquement le pilote xorg adapté à votre système, comme nv, intel ou ati.
   Nettoie les anciens pilotes binaires, met xorg.conf à jour avec les nouvelles informations de pilote et purge tout reste de nvidia ou fgrlx de xorg.conf.
-N Force l'installation d'un pilote xorg spécifique. 
Syntaxe: sgfxi -N <un des pilotes de la liste, comme -N radeon>
   Pilotes Xorg gérés: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Configure également xorg.conf comme requis. Complète automatiquement la configuration 3D xorg radeon.
-o Remplace défaut: installe un pilote spécifique, il s'agit d'un remplacement absolu - syntaxe :
   -o 9746 (pour nVidia, n'utilisez pas le préfixe 1.0-) ou: -o 8.32.5 (pour fglrx)
-r Efface les mode lines de xorg.conf
-t Active la vue jumelle dynamique (nVidia seulement)

  2. install-binary-gfx

    Le nouveau script pour installer les pilotes de cartes graphiques s'appelle install-binary-gfx. Pour l'obtenir lancez get-sidux-binary-gfx, depuis une console root.

    get-sidux-binary-gfx
install-binary-gfx -a (pour construire le source sid)

    Le script déterminera quel pilote utiliser pour votre carte, construira un paquet .deb, l'installera et modifiera votre fichier de configuration xorg.conf en conséquence. Pour que la prise en compte du nouveau pilote soit effective, il vous faudra redémarrer votre serveur X (Ctrl+Alt+Retour).

    Options

    Il existe des options qu'il vous sera possible d'activer manuellement:

     Usage: ./install-binary-gfx [options]
-a | --auto            Mode automatique
-b | --build           Construction du module seule (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          Détection de la carte seule
-u | --update          Mise-à-jour du script
-v | --verbose         Mode bavard

    Exemples

    install-binary-gfx -a (Ceci détecte votre carte et installe automatiquement les pilotes pour Sid.)
install-binary-gfx -b nvidia (Ceci se contente de construire le .deb du module nvidia dans /usr/src. La commande fonctionne également avec fglrx et nvidia-legacy.)
install-binary-gfx -b ati (Ceci se contente de construire le .deb du module ati dans /usr/src.La commande fonctionne également avec fglrx et nvidia-legacy.)
install-binary-gfx -d (Ceci détecte votre carte pour ensuite afficher l'information.)
install-binary-gfx -u (Ceci mettra le script à jour. )

    En cas de problème

    Si le script échoue avec votre carte, alors lancez-le en mode debug et postez le résultat comme le fichier de log /var/log/sidux-CARD-installer.log sur le Forum de sidux ou bien utilisez http://paste.debian.net, puis connectez-vous au canal IRC #sidux en fournissant le lien afin de recevoir une assistance immédiate.

  3. Pour installer les pilotes propriétaires sur le live-CD:

    sudo sgfxi

    pour récupérer la dernière version du script ; puis:

    ctrl+alt+f1

    et lancez

    sgfxi

    sgfxi -h,
pour voir les options

    X est relancé sitôt le pilote installé.

Les pilotes libres xorg pour nVidia, ATI, Radeon, Intel

  1. sgfxi

    Vous pouvez utiliser sgfxi pour alterner entre pilotes libres et pilotes propriétaires ; la permutation fonctionne dans les deux sens. Les commandes clés sont :

    -n Installe automatiquement le pilote xorg nv, intel, or ati adapté à votre système.
   Nettoie les anciens pilotes binaires, met xorg.conf à jour avec les nouvelles informations de pilote et purge tout reste de nvidia ou fgrlx de xorg.conf.
-N Force l'installation d'un pilote xorg spécifique.
Syntaxe: sgfxi -N <un des pilotes de la liste, comme -N radeon>
   Pilotes Xorg gérés: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Configure également xorg.conf comme requis. Complète automatiquement la configuration 3D xorg radeon.

    Première étape

    apt-get update
apt-get install sidux-scripts
sgfxi

    Pour installer ou revenir à un pilote xorg

    sgfxi -n
OU
sgfxi -N <xorg driver>

  2. sgfxi / !h2 smxi

    Utilisez sgfxi à partir de !h2 smxi. Lancez simplement le script en utilisant les options qui suivent et vous serez conduits directement à la section d'installation des pilotes graphiques. Si vous utilisez smxi vous n'avez pas besoin d'installer sgfxi, car smxi l'installe automatiquement.

    Consultez pour ce faire la page web de !h2 smxi où vous trouverez des informations complémentaires. Lancez cette commande ensuite :

    smxi -kiwdt

  3. La voie de la difficulté

    Les pilotes pour le serveur graphique X.Org - consultez xserver-xorg pour une description plus détaillée - fournissent un support 2D pour les cartes graphiques NVIDIA Riva, TNT, GeForce, et Quadro, ainsi que pour les cartes ATI Mach, Rage, Radeon, et FireGL au moyen des sous-pilotes 'atimisc', 'r128' et 'radeon'. Ceux-ci gèrent en outre la série free Intel en 3D.

    Pour les installer:

    Sortez de KDE pour passer en mode texte en utilisant Ctrl+Alt+F1
Devenez l'administrateur root
puis entrez init 3
(ceci assure que le serveur X soit arrêté)
apt-get update
apt-get install xserver-xorg-video-nv
OU
apt-get install xserver-xorg-video-ati
init 5 && exit

    Notez que ceux-ci n'ont rien à voir avec les pilotes 'nvidia/ati' qui étaient seulement fournis sous forme d'exécutables, et qui géraient la 3D. Ceux-ci ne sont plus supportés.

    Il vous faut ensuite éditer votre fichier /etc/X11/xorg.conf, APRÈS TOUTEFOIS AVOIR PRIS LA PRÉCAUTION D'EN FAIRE UNE SAUVEGARDE, car l'édition incorrecte de /etc/X11/xorg.conf peut vous causer de gros, gros soucis.

    Pour éditer /etc/X11/xorg.conf, vous devez être root. Trouvez-y la section intitulée Section "Device", et changez-y le pilote graphique à la ligne Driver par nv OU ati

    VOUS ÊTES PRÉVENUS ! L'ÉDITION DE XORG.CONF PRÉSENTE DES RISQUES ! C'EST À VOUS ET À VOUS SEUL DE LES ASSUMER !

    Plus d'informations sur X.Org ici

Changer la résolution de vos écrans et moniteurs

Changer la résolution d'écran

Ceci s'opère depuis une console root :

siduxcc

et suivez les instructions, ou par l'interface graphique:Kmenu>System>sidux Control Centre>Display

Si X ne radémarre pas automatiquement, faites un radémarrage de X en entrant 

ctrl+alt+retour arrière

Double Affichage

Vous êtes sur le point de vous immerger dans les fichiers de configuration de xorg. S'il vous plait ! Faites une sauvegarde de votre fichier xorg.conf, avant d'aller plus loin !

Les instructions détaillées ici pourraient changer d'ici la fin 2007 lorsque xorg 7.3 sera sorti.

Pilote libre ATI - Méthode 1

L'article http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=19794#19794 propose quelques fichiers xorg.confs qui fonctionnent avec le pilote libre 'radeon'.

AVERTISSEMENT: Vous devrez vous munir de l'information nécessaire à la configuration de votre second moniteur. Un moyen d'y parvenir, consiste à démarrer sur le Live-CD pour générer un fichier xorg.conf, de copier celui-ci, puis de le faire à nouveau avec le second moniteur.

Pilote libre ATI - Méthode 2

Par cette méthode, vous allez régler en premier la résolution du premier moniteur, puis le second utilisera la même configuration. Vous devez donc vous assurer que votre second moniteur supporte les paramètres que vous aurez fixé pour le premier et donc avoir ajusté ceux-ci en conséquence.

Dans la section:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

ajoûtez

Option "MergedFB"
Option "CRT2Position" "RightOf"

De sorte que la strophe se présente comme suit:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Pour consulter l'information exhaustive de configuration rendez-vous sur http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html (en)

AVERTISSEMENT: Le lien ci-dessus utilise 'radeon' mais la technique a déjà fonctionné avec le pilote 'ati'.

Pilote propriétaire ATI (Pilote fglrx)

Commencez par installer le pilote propriétaire ATI (fglrx)

get-sidux-gfx
ou
sgfxi / smxi

Reportez-vous à la section dédiée à l'installation des pilotes 3D pour des instructions complètes.

Ensuite: Vous devrez sortir de X

ctrl+alt+F1
entrez: root
entrez votre mot-de-passe:
init 3

Ensuite:

aticonfig --dtop=horizontal
init 5 && exit

Si le cadrage de vos moniteurs est incorrect, vous pouvez faire

aticonfig --dtop=horizontal,reverse

Lisez l'aide, svp ; elle est bien conçue et accessible, et comporte des exemples et des options:

aticonfig --help

Avertissement: Les deux moniteurs doivent avoir la même résolution. Aussi consultez leurs documentations techniques respectives afin de fixer pour ceux-ci la résolution maximale que ceux-ci puissent supporter EN MÊME TEMPS.

L'article du forum http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=20231#20231 pourra vous apporter de l'information complémentaire. (en)

Pour revenir en arrière et/ou désactiver le double-affichage

aticonfig --initial --input=/etc/X11/xorg.conf

Nvidia

Pour obtenir un double affichage avec les pilotes propriétaires Nvidia, utilisez simplement l'outil de configuration xorg de nvidia http://www.sorgonet.com/linux/nv-online/ et modifiez vos fichiers xorg en conséquence.

Indications à propos des matériels nécessitant un support non-libre

Du fait de complexités juridiques, le live-cd sidux n'est fourni qu'avec des outils entièrement libres.

De ce fait, vous souhaiterez probablement ajoûter les sources contrib/ non-free à votre fichier /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list et vous configurer un accès à Internet:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in seperate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • Les cartes graphiques Ati Radeon : l'accélération 3D pour les cartes antérieures jusqu'à r35x devrait fonctionner. Les plus récentes cartes Radeon X1xxx nécessitent des pilotes propriétaires pour l'accélération des performances, mais ceux-ci ne sont pas encore compatibles avec X.org 7.2. En fonction du GPU dont vous êtes équipé, les modules "xmodule=vesa" ou "xmodule=fbdev" utilisés comme codes de démarrage peuvent vous servir afin de pouvoir utiliser les modèles de type Radeon X1xxx.
  • Atheros/ "madwifi" wifi/wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wifi/wlan: 
    apt-get install atmel-firmware
  • Broadcom, bcm43xx/ b43/ b43 legacy wlan: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • modem Eagle USB ADSL: récupérez le firmware depuis http://eagle-usb.org/ueagle-atm/non-free/ et placez-le dans le répertoire /lib/firmware/.
  • Firmwares DVB pour certaines cartes haut-de-gamme TV DVB (la plupart des cartes bon marché n'en auront pas besoin): récupérer le firmware adapté (consultez dmesg) depuis http://www.linuxtv.org/downloads/firmware/ et placez-le dans le répertoire /lib/firmware/.
  • Intel ipw2100, 11 MBit/s wifi/wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wifi/wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan désinstallez iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 and firmware-iwl4965 s'ils sont déjà installés, svp.: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54 MBit/s wifi/wlan: récupérer le firmware adapté depuis http://prism54.org/firmware/ et placez-le dans /lib/firmware/.
  • Cartes graphiques Nvidia: l'accélaration 3D n'est pas encore possible avec les pilotes libres. S.v.p., utilisez get-sidux-binary-gfx or sgfxi pour mettre en place des scripts d'installation pour ces cartes.
  • RaLink rt61 and rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    désinstallez rt61-fwcutter, rt73-fwcutter, firmware-rt61 and firmware-rt73, s'ils sont déjà installés

  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wifi/wlan, récupérer le firmware adapté depuis http://www.kazer.org/acx-firmware-20060207.tar.bz2 et placez-le dans /lib/firmware/.
  • ZyDAS zd1201 11 MBit/s wifi/wlan: 
    apt-get install zd1201-firmware
  • ZyDAS zd1211 54 MBit/s wifi/wlan: 
    apt-get install zd1211-firmware

Nous nous efforçons de fournir des paquetages pour tout firmware librement redistribuable depuis nos dépots d'outils non-libres, mais tous les fournisseurs ne l'autorisent pas.

Comment accéder aux partitions NTFS en écriture avec ntfs-3g

Soyez avertis: Si le pilote ntfs-3g est désormais réputé 'stable', n'en usez jamais sans disposer d'une sauvegarde externe de vos données, et quoiqu'il arrive, jamais sur un système de production ! Si vous le faites quand-même, et vous perdez des données, ce sera votre faute ; vous l'utilisez à vos risques et périls !

Ouvrez une console/terminal et entrez-y la commande suivante: (reportez-vous à Partitionnement des DDurs - Identification des disques

su
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Pour sortir de la console entrez : exit

Dès lors votre volume NTFS peut être monté rw (read and write) et vous devriez être en mesure d'y écrire des données. Mais encore une fois, soyez prudent ! Usez-en dans les situations d'urgence ; cette méthode n'est pas recommandée pour un usage permanent quotidien.
