aptosid-*.iso - Mot-de-Passe root en mode Live

Faites bien attention : du moment où vous exécutez des tâches avec des privilèges root, vous devez être en mesure de comprendre ce que vous êtes en train de faire ! Pour surfer sur Internet ou sur le réseau, pas besoin de droits d'administration...

aptosid-*.iso default password

Sur le Live-CD de aptosid aucun mot de passe n'est assigné à root. Si vous voulez lancer un programme qui nécessite des privilèges d'administrateur, vous disposez de plusieurs choix :

Simple method, just type:

sux
Assigner un mot-de-passe (provisoire) à root

Ouvrez un terminal/console:

aptosid@0[aptosid]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
aptosid@0[aptosid]$

vous pouvez alors utiliser ce mot de passe durant le reste de votre session avec le Live-CD aptosid.

sudo is not supported on hard disk installations, see sudo and sux.

Démarrer un programme depuis un terminal administrateur

Avec la commande sux, vous pouvez lancer application de X avec le privilège de l'administrateur.

Ouvrez un terminal/console:

aptosid@0[aptosid]$ sux
root@0[aptosid]# gparted (ou ce qui vous chante...)

Certaines applications KDE requièrent dbus-launch devant le nom de l'application :

dbus-launch <Application>

Une autre variante qui fonctionne pour la grande majorité des gestionnaires de bureaux est :

Alt+F2 su-to-root -X -c <Application>

Par exemple:

Alt+F2 su-to-root -X -c gparted

Pour sortir du mode root dans la console, entrez :

exit

ou bien faites Control+D au clavier ou encore cliquez juste sur l'icône de fermeture de la fenêtre pour fermer la console.

Dans le cas d'un lock-out sur une aptosid iso-*. vous devez faire ce qui suit et suivez le prompt pour définir un mot de passe:

alt+ctrl+F1
sudo passwd

Une fois que le mot de passe est actif, faites ce qui suit:

alt+ctrl+F7

Installer des logiciels durant votre session en mode Live

apt-get update
apt-get install le_paquet_désiré

Attention: à la fermeture du Live-CD, aucun changement ne sera gardé, sauf si vous activez fromiso et persist.

Comment accéder aux partitions NTFS en écriture avec ntfs-3g

Soyez avertis: Si le pilote ntfs-3g est désormais réputé 'stable', n'en usez jamais sans disposer d'une sauvegarde externe de vos données, et quoiqu'il arrive, jamais sur un système de production ! Si vous le faites quand même et perdez des données, ce sera votre faute ; vous l'utilisez à vos risques et périls !

Ouvrez une console/terminal et entrez-y la commande suivante : (reportez-vous à Partitionnement des DDurs - Identification des disques

) 
sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Pour sortir de la console entrez : exit

Dès lors votre volume NTFS peut être monté rw (read and write, autrement dit lecture et écriture) et vous devriez être en mesure d'y écrire des données. Mais encore une fois, soyez prudent ! Usez-en dans les situations d'urgence ; cette méthode n'est pas recommandée pour un usage permanent quotidien.
