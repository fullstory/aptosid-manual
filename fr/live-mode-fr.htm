Mot-de-Passe root en mode Live

Faites bien attention : du moment où vous exécutez des tâches avec des privilèges root, vous devez être en mesure de comprendre ce que vous êtes entrain de faire ! Pour fureter sur Internet ou sur réseau, pas besoin de droits d'administration...

Sur le Live-CD de sidux aucun mot-de-passe n'est assigné à root. Si vous voulez lancer un programme qui nécessite des privilèges d'administrateur, vous disposez de plusieurs choix :

Assigner un mot-de-passe (provisoire) à root:

Ouvrez un terminal/console:

sidux@0[sidux]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

vous pouvez alors utiliser ce mot-de-passe durant le reste de votre session avec le Live-CD sidux.

Lancer le programme avec

sudo

ou 

su-me

Appuyez sur Alt+ F2, puis entrez

sudo

avant le nom du programme que vous désirez lancer ; par exemple :

sudo gparted

Lancer le programme depuis une console root:

Ouvrez un terminal/console:

sidux@0[sidux]$ su
root@0[sidux]# gparted (ou ce qui vous chante...)

Pour sortir du mode root dans la console, entrez:

exit

ou bien faites Control+D au clavier ou encore cliquez juste sur l'icône de fermeture de la fenêtre pour fermer la console.

Installer des logiciels durant votre session en mode Live

apt-get update
apt-get install le_paquet_désiré

Attention: à la fermeture du Live-CD, aucun changement ne sera gardé.

Comment accéder aux partitions NTFS en écriture avec ntfs-3g

Soyez avertis: Si le pilote ntfs-3g est désormais réputé 'stable', n'en usez jamais sans disposer d'une sauvegarde externe de vos données, et quoiqu'il arrive, jamais sur un système de production ! Si vous le faites quand-même, et vous perdez des données, ce sera votre faute ; vous l'utilisez à vos risques et périls !

Ouvrez une console/terminal et entrez-y la commande suivante: (reportez-vous à Partitionnement des DDurs - Identification des disques

su
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Pour sortir de la console entrez : exit

Dès lors votre volume NTFS peut être monté rw (read and write) et vous devriez être en mesure d'y écrire des données. Mais encore une fois, soyez prudent ! Usez-en dans les situations d'urgence ; cette méthode n'est pas recommandée pour un usage permanent quotidien.
