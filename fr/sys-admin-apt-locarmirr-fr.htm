Mise à niveau d'ordinateurs avec bande passante/ vitesse de connection limitée

Pour ceux qui utilisent plusieurs ordinateurs et/ou qui ont en plus une faible bande passante ou encore pour ceux qui ont une connection tres lente et qui souhaitent mettre à jour leurs ordinateurs, il exite des solutions pour les maintenir à jour que ce soit dans un réseau local temporaire ou permanent.

La solution est d'utiliser un dossier local d'archives sur un des ordinateurs comme mirroir de téléchargement que les autres ordinateurs utiliseront pour leur mise à niveau. Vous sauvegardez ainsi de la bande passante pour vos plus importantes tâches quotidiennes.

Prérequis

Assurez vous d'avoir 6 gigas d'espace libre disponible pour le cache.

Utilisation d'approx comme mirroir local d'archives

Lorsque l'ordinateur client cherchera les fichiers ceux du cache seront disponibles, pourvu que vous ayez effectué apt-get update, dist-upgrade -d ou dist-upgrade sur l'ordinateur hébergeant un serveur approx.

Etape 1: Configurer le Serveur pour ordinateurs Clients afin d'utiliser approx

apt-get install approx

mcedit /etc/approx/approx.conf

Modifier le fichier approx.conf afin qu'il utilise les mirroirs en ligne:

# Here are some examples of remote repository mappings.
# See http://www.debian.org/mirror/list for mirror sites.


debian http://ftp.iinet.net.au/debian/ << modifier afin d'utiliser le mirroir debian local
sidux http://sidux.net/debian/

Appliquer la même syntaxe aux autres dèpots que vous souhaitez ajouter à votre mirroir local.

Démarrez le serveur approx:

update-inetd --enable approx

S'il ne se lance pas, redémarrer l'ordinateur serveur sur lequel est installé approx, vu que approx est réputé être parfois têtu à se lancer.

Après avoir redemarré lancer apt-get update et dist-upgrade ou dist-upgrade -d. Ceci pour être sûr qu'approx peux accéder aux dernières mises à jours disponibles pours les ordinateurs clients, autrement certains paquets installés sur ceux ci ne seront pas sur le serveur hôte. Si c'est le cas approx téléchargera les paquets appropriés.

Les paquets téléchargés s'accumulent dans /var/cache/approx qui sera chargé après la première utilisation des clients.

Etape 2: Configurer les clients afin qu'ils utilisent le serveur approx

Tout d'abord: Modifier le fichier/etc/apt/sources.list.d/*.list afin qu'il utilise approx comme mirroir debian et sidux.

Avec mcedit, décommenter vos habituels liens URLs (placer un# devant chaque ligne) et ajouter les lignes suivantes et sauver, exemple:

Sources list Debian
mcedit /etc/apt/sources.list.d/debian.list

#deb your current debian mirror

deb http://approx:9999/debian/ sid main contrib non-free
Sources list sidux 
mcedit /etc/apt/sources.list.d/sidux.list

#deb your current sidux mirror

deb http://approx:9999/sidux/ sid main fix.main
Autres sources lists

Faire pareil que précédement pour les autres sources.list requis.

Serveurs Proxy

Editer ensuite /etc/hosts pour ajouter le proxy local afin d'acceder à l'adresse IP de votre serveur:

mcedit /etc/hosts

10.1.1.X approx

Maintenant exécuter apt-get update et dist-upgrade ou dist-upgrade -d. A chaque première utilisation sur les clients vous constaterez des lenteurs et même des délais de connexion dépassés, mais insistez. A force d'utilisation vous y parviendrez à long terme.
