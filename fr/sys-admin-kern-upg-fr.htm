Installer les nouveaux noyaux

Les nouveaux noyaux sont distribués par apt. Choisissez s'il vous plait le mirroir sidux le plus proche et modifiez vos sources.list (sous /etc/apt):

Les noyaux sont disponibles sous les formes suivantes:

  • sidux-686 - Noyau pour processeur de famille i686 doté d'un simple/double/coeur et plus de CPU
  • sidux-amd64 - Noyau pour sidux 64 bit
4 étapes sont nécessaires pour installer le noyau manuellement en mode console:

1. Dans la console passez en mode Administrateur (su ou sux:

# apt-get update

2. Pour installer la dernière version du noyau:

# apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. Mise à jour du noyau effectuée. Dans le cas que vous auriez besoin de dernier pilote de la carte graphique ATI, nVidia ou Intel, mettez d'abord à jour le script:

sgfxi -U

4.Redémarrez votre ordinateur avec le nouveau noyau (cela doit être par défaut).

Pour mettre à jour le pilote 3D:

sgfxi

Alternativement, après le premier démarrage du nouveau noyau rendez vous directement au niveau d'exécution 3 (ça se réalise en ajoûtant 3 à la ligne de commande de Grub ou en entrant la commande 'init 3' une fois logué en mode console à partir de KDM) et lancez sgfxi.

Voir l'information plus detaillée de sgfxi

Au cas où il y ait des problèmes avec le nouveau noyau, vous pouvez rédémarrer et selectionner l'ancien noyau.

Modules

Pour trouver les modules dont vous avez besoin, la commande suivante vous liste les modules actuellement disponibles. Copiez cette ligne dans votre terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Pour avoir la description complète de chaque module, copiez cette ligne dans votre terminal

apt-cache search 2.6.*.slh.*-sidux

Pour installer les module requis (exemple avec virtualbox-ose et qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXEMPLE)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXEMPLE)

Pour vérifier si les modules sont chargés dans le noyau:

ls /sys/module/
ou
cat /proc/modules

IMPORTANT: si votre configuration a besoin de certains modules de pilotes spécifiques (par exemple pour le wifi) vous devrez installer les paquet .deb qui leurs sont appropriés. Exemple: Pour le Agere et131x adaptateur ethernet: 

apt-get install et131x-modules-*

Dans le cas seulement où vous auriez déjà le paquet du module pour et131x, vous pouvez utiliser la commande dpkg -i et131x-modules*.deb. Mais même dans ce cas, s'il est disponible par apt-cache, la méthode avec apt-get est préférable.

Voir aussi Activation de Ndiswrapper et WIFI-WLAN

Autre façon d'installer le noyau

smxi

Le script smxi vous offre une méthode intégrée pour installer le noyau, les pilotes et de nombreux autre choses pour l'administration du système:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Vous pouvez installer le noyau aussi à partir du centre de contrôle graphique siduxcc: depuis le KDE-Start-Menu > System > sidux Control Centre. siduxcc vous offre également d'autres applications graphiques pour l'administration du système

Effacer les anciens noyaux

Si l'installation du nouveau noyau est un succès, les anciens noyaux peuvent être effacées ; il est toutefois recommandé de les conserver quelques jours afin d'être vraiment sûr que tout aille bien et de garder la possibilité de redémarrer sur un noyau antérieur dans le menu de grub.

Ces noyaux anciens peuvent être désinstallés du système. Vous pouvez procéder en utilisant le script kernel-remover intégré au paquetage sidux-kernelhacking! Ce script permet de déterminer quels noyaux vous souhaitez désinstaller.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Alternativement, vous pouvez utiliser siduxcc, qui utilise le script kernel-remover.
