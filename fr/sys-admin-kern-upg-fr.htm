Installer les nouveaux noyaux

Les noyaux sont publiés ici. Choisissez le miroir le plus proche de vous, svp:

Main Mirrors
http://debian.tu-bs.de/project/sidux/kernels Allemagne
http://sidux.net/files//kernel Allemagne
ftp://ftp.spline.de/mirrors/sidux/kernel/ Allemagne

These mirrors can be as much as 24 hours behind the main mirrors
http://www.mirrorservice.org/sites/sidux.com/sidux/kernels Royaume-Uni
http://ftp.heanet.ie/pub/sidux//kernel Ireland
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernels TN, USA
http://ftp-mirror.internap.com/pub/sidux/kernel  GA. USA
http://sidux.2wl.ru/downloads/kernels Russie
http://ftp.tw.debian.org/pub/Sidux/ Asie
http://mirror.aarnet.edu.au/pub/sidux/kernel Australia
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernels Afrique du Sud

Les noyaux sont disponibles sous les formes suivantes:

  • sidux-686 - Noyau pour processeur de famille i686 doté d'un simple/double/coeur et plus de CPU
  • sidux-amd64 - Noyau pour Processeur de la famille AMD64- , pour sidux 64 bit uniquement
Ce sont les 4 étapes pour installer le noyau manuelement en mode console:
apt-get update

1. Pour obtenir la dernière version du noyau:

wget -O - http://sidux.com/debian/pool/main/l/linux-2.6-sidux 2>/dev/null | awk "/linux-image.*$(dpkg-architecture -qDEB_HOST_ARCH_CPU)/" | sed -e 's/^.*linux\-image\-//' -e's/_.*\.deb.*$//'

2. Pour trouver les modules dont vous avez besoin, la commande suivante vous liste les modules actuellement disponibles. Copiez cette ligne dans votre terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Pour avoir la décription complète de chaque module, copiez cette ligne dans votre terminal

apt-cache search 2.6.*.slh.*-sidux

3. Pour installer le noyau, les entêtes du noyau et les modules requis (par exemple virtualbox-ose et qc-usb)

apt-get install  linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686
apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686

4 Redémarrez votre ordinateur une fois l'opération terminée. Le nouveau noyau sera utilisé lors du nouveau démarrage par défaut. Celui-ci poserait-il problème, vous n'auriez qu'à redémarrer en choisissant un noyau antérieur, et recommencer l'opération.

Pour vérifier si les modules sont chargés dans le noyau:

ls /sys/module/
or
cat /proc/modules

IMPORTANT: si votre configuration a besoin de certains modules de pilotes spécifiques (par exemple pour le wifi) vous devrez installer les paquet .deb qui leurs sont appropriés. As an example, for the Agere et131x ethernet adapter: 

apt-get install et131x-modules-*

Dans le cas seulement où vous avez déjà le paquet du module pour et131x, vous pouvez utiliser la commande dpkg -i et131x-modules*.deb. Mais même dans ce cas, s'il est disponible par apt-cache, le méthode avec apt-get est préférable.

Reportez-vous également à se connecter à internet et ndiswrapper ainsi qu'à WIFI

Après la mise-à-niveau de votre noyau, si vous en avez besoin, , ajoûtez-y les dernier pilotes graphiques installables. Si vous avez une carte graphique ATI ou Nvidia, le premier redémarrage avec un nouveau noyau doit s'opérer en niveau d'exécution 3 (ça se réalise en ajoûtant 3 à la ligne de commande de Grub).

Une méthode semi-automatique d'installation des noyau consiste à utiliser smxi :

apt-get update
apt-get install sidux-scripts
smxi

Vous pouvez faire la même chose dans le centre de controle en graphique siduxcc de KDE-Start-Menu

Effacer les anciens noyaux

Si l'installation du nouveau noyau est un succès, son .zip (et ses archives) peuvent être effacées ; il est toutefois recommandé de les conserver quelques jours afin d'être vraiment sûr que tout aille bien et de garder la possibilité de redémarrer sur un noyau antérieur.

Ces noyaux anciens peuvent être désinstallés du système. Vous pouvez y procéder en utilisant le script kernel-remover intégré au paquetage sidux-kernelhacking! Ce script permet de déterminer quels noyaux vous souhaitez désinstaller.

kernel-remover
