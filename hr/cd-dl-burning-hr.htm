Povući aptosid-CD

OVO JE VRLO VAŽNO: aptosid, kao Linux LIVE-CD, je bazirana na tehnologiji visoke kompresije, (i zbog toga, dobro pazite kad pečete ISO-datoteku. Samo koristite CDs visoke kvalitete [ili DVD+RW] i pecite sa DAO-modusom (disk-at-once) i ne brže nego x8.

UVIJEK OSIGURAJTE PODATKE!

Download Mirrors

Molimo Vas da koristite vama najbliži. Mirrors with codeboxes underneath them get updated as soon as changes are uploaded (refer to /etc/apt/sources.list.d/aptosid.list).

Definitions of files on each mirror

Each aptosid mirror has following files:

MD5SUM
MD5SUM.asc
SHA256SUM
SHA256SUM.asc
SOURCES
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

The .manifest file lists all the packages contained in the specific iso. The .iso is the image to download. The .md5 and .sha256 files are to check the integrity of the iso.

The .asc files are signature files, so you can verify that the checksums (.md5 .sha256) have not been changed and that they can be used to verify the integrity of the iso.

The sources tar is only for those who require a static copy of the source code for redistribution purposes to comply with the licensing requirements. Read the SOURCES file for more information.

md5-sum i provjera

OVO JE VRLO VAŽNO: aptosid, kao Linux LIVE-CD, je bazirana na tehnologiji visoke kompresije, (i zbog toga, dobro pazite kad pečete ISO-datoteku. Samo koristite CDs visoke kvalitete [ili DVD+RW] i pecite sa DAO-modusom (disk-at-once) i ne brže nego x8.

To je zbroj za datoteku koji je korišten da provjerite integritet datoteka. md5sum datoteke je uspoređen s poznatim zbrojem. Na taj način možete provjeriti jeli se vaša datoteka promijenila ili da li je u kvaru; to je savjetodavno za podatke s interneta.

Ako je md5 zbroj povučene datoteke isti kao i zbroj u md5-datoteci, možete biti sigurni da je datoteka u redu. Pod Linux dobijete zbroj sa:(treba moment da se izračuna)

$ md5sum file_to_check

Zbroj će biti napisan na konzoli.
md5sum (486 kb) isto možete provjeriti md5sum Windows.

ISO-datoteke aptosida su uvijek ponuđene s md5-datotekom i trebaju uvijek biti provjerene prije korištaja. To garantira, ako imate problema, ne trebate ih tražiti u povučenim datotekama, i to drži forum čisto od problema zbog pokvarenih ISO-datoteka. To bi trebali obaviti u konzoli. Otvorite direktorij s ISO-datotekom i md5-zbrojem i upišite naredbu da provjerite md5-zbroj. e.g:

Download the following files from the mirror:

MD5SUM
MD5SUM.asc
SHA256SUM
SHA256SUM.asc

In a terminal:

$ md5sum aptosid.iso

Ako ne pašu, dobit će te grešku:

"aptosid.iso: Error
md5sum: Warning: calculated checksum does not match!"

Ako je povučena datoteka u redu, program se zatvori bez vijesti.

If you wish to be able to verify the validity of the .asc files do the following as $USER:
Example using:
aptosid-2009-02-aether-kde-lite-i386-200907141544.iso

1) Import "Stefan Lippers-Hollmann's" (slh's) public key from a key-server:

$ gpg --keyserver wwwkeys.eu.pgp.net --recv-keys F80994F6
gpg --edit-key F80994F6
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Now you can verify the integrity of the checksum with:

$ gpg --verify MD5SUM.asc

If you have classified the key as trustworthy using the necessary gpg commands, then this is the output you should get with "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
$ gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

You will get 'Good signature' with a WARNING if you did not choose to classify the key:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model

Kako pržiti CD s Windows?

OVO JE VRLO VAŽNO: aptosid, kao Linux LIVE-CD, je bazirana na tehnologiji visoke kompresije, (i zbog toga, dobro pazite kad pečete ISO-datoteku. Samo koristite CDs visoke kvalitete [ili DVD+RW] i pecite sa DAO-modusom (disk-at-once) i ne brže nego x8.

Možete isto pržiti CD s Windows. Povučena datoteka mora se Pržiti kao ISO-datoteka. Ako je Winrar (ili bilo koji drugi archivni program) spojen sa .ISO-datotekama, moglo bi biti problema, zato što je on onda korišten za ISO datoteke. Od ISO-datoteke morate pržiti CD.

Imate više dobrih opcija da pržiti s Windows.

Open Source pržilica za Windows

cdrtfe :kompatibilan za Windows 9x/ME/2000/XP (testiran sa Win95, Win98SE, Win2000, WinXP) samo za Win9x/ME: ASPI Layer (na primjer Adaptec ASPI 4.60)

LinuxLive USB Creator, an open source project, offers a GUI for MS Windows™, which enables an install of a aptosid-i386.iso (32 bit) to a USB stick. You need to use fromhd bootline/cheatcode on the boot line.

Closed Source pržilica za Windows
  • Nero (ne Nero express!)
  • Drugi dobar freever program je CD/DVD Burner XP pro
  • Burncdcc od terabyteunlimited ali samo može da peče ISOs..

Pržiti CD s Linux

OVO JE VRLO VAŽNO: aptosid, kao Linux LIVE-CD, je bazirana na tehnologiji visoke kompresije, (i zbog toga, dobro pazite kad pečete ISO-datoteku. Samo koristite CDs visoke kvalitete [ili DVD+RW] i pecite sa DAO-modusom (disk-at-once) i ne brže nego x8.

Ako već radite s Linux na vašem računalu možete koristiti konzolu ili sa instaliranim burning programima.

Sa aptosid ili drugim Linux distribucijom imate K3b kao standard cd-burning alat. Otvorite ga, pod "Tools" -> "burn CD-Image..." i otvorite vašu ISO-datoteku za peći (e.a. aptosid.iso).

K3B izračuna na početku md5-zbroj za ISO-datoteke (treba mu moment za to). Ako je md5-zbroj povučene datoteke isti kao i zbroj u md5-datoteki, možete biti sigurni da je datoteka u redu, i možete pržiti datoteku sa klikom na "Start".

Problems with burning mostly happen with a frontend. You can burn directly from the shell with burniso.

Also see Writing aptosid to a USB/SSD stick with any Linux, MS Windows or Mac OS X system (32 and 64 bit).
