Postavke Time-poslužitelja

Kao prvo u konzoli kao root

apt-cache search ntp
apt-get update && apt-get install  ntp ntp-doc
update-rc.d -f ntp defaults
izvodite update-rc.d poslje malo konfiguracije

Nađite dokumente na vašem sistemu

/usr/share/doc/ntp-doc/html/index.html
i upišite u favorite!

To je veliki doc, i ne treba vam sve, ali možete sve da pročitate.

ntp neće biti aktiviran prije reboota, ali trebali bi postaviti vrijeme sto točnije prije toga

ntp će dobiti svoje vrijeme iz liste poslužitelja u /etc/ntp.conf, koja je glavna datoteka za postavke,

Oboje ntpdate i ntpd daemon [zvan ntp] izpitavaju listu time-poslužitelja blizu vrha /etc/ntp.conf.. ovdje je lista kao primjer:

 pool.ntp.org atlas za više 100 -sloj NTP servera.
# Tvoj server će birati različiti set svako vrijeme kada budete startali .
#  *** Molimo da mislite  na spajanje sa fondom! ***
#  ***  http://www.pool.ntp.org/#join  ***
server 192.168.3.24
server ntp.blueyonder.co.uk
server uk.pool.ntp.org
server 1.uk.pool.ntp.org
server 2.uk.pool.ntp.org
server 0.europe.pool.ntp.org
server 1.europe.pool.ntp.org
server 2.europe.pool.ntp.org

Prvi je drugo računalo u istoj mreži, na kojemu je isto ntp izvođen [na tome je 'server 192.168.3.1']

Drugi je time-poslužitelja vašeg ISP s kojim ste spojeni.

Sljedeći su nekoliko poslužitelja od uk.pool.ntp.org, onda nekoliko evropskih. Vaši isp-name-poslužitelji su isto puno puta time-poslužitelji, možete provjeriti sa:

ntpdate -v

To neće nista promijeniti, ali će vam pokazati rezultat za vrijeme, nešto kao:

# ntpdate -v 192.168.3.24
19 Sep 19:09:27 ntpdate[13329]: ntpdate 4.2.2@1.1532-o Wed Aug  9 12:08:54 UTC 2006 (1)

A full list of ntp timeservers is here http://www.pool.ntp.org/

Onda hoćete da dopustite ulaz na vaše lokalna računala

# Lokalni korisnici mogu ispitati ntp poslužitelja izbliza.
restrict 127.0.0.1 nomodify
restrict 192.168.3.0/24

Sad hoćete prijenos:

# Ako hoćete da nudite vrijeme vašem lokalnim subnetu, promijenite sljedeću liniju.
# (Opet, adresa je samo primjer.)
broadcast 192.168.3.255

ntp.conf datoteka je malo čudna, otvorena je kao diff ako samo kliknete na nju .. prije ntp starta, morate postaviti vrijeme, s

# ntpdate -u -b uk.pool.ntp.org
19 Sep 19:19:33 ntpdate[15641]: step time server 62.3.200.116 offset 0.001523 sec

Onda startujete ntp, kao servis, da startuje s svakim startom [isto tako,s reboot]. Ako je ntp već malo radio, izvodite:

ntpq -pn

Ako je sve u redu, trebali bi vidjeti nešto slično ovome:

Zvijezda na trećoj liniji u tom primjeru: *82.219.3.1, pokazuje aktivnog time-poslužitelja. To znaći sad imate točno vrijeme, i koristi port 123. Primjer za iptables liniju je:

$IPT -A udp_inbound -p UDP -s 0/0 --destination-port 123 -j ACCEPT
$IPT -A INPUT -j ACCEPT -p tcp --dport 123
