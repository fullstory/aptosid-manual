Konfigurirati sidux da koristi SAMBA (Windows) dijeljenje direktorija na udaljenim računalima

Izvodite sve naredbe kao root (u terminalu ili konzoli) Upišite URL u Konqueror (Konqueror izvodite kao normalan korisnik).

server = ime poslužitelja ili IP od Windows računala
share = ime dijeljenjog direktorija

U KDE - Konqueror upišite URL smb://server ili kompletnu URL smb://server/share

U konzoli možete vidjeti dijeljenje direktorije na poslužitelju s:

smbclient -L server

Da mountate dijeljenji direktorij -(sa full access za SVE korisnike) zapamtite ovo: Mountpoint mora egzistirati. Ako ne, morate kreirati prvo direktorij kao ovoga (ime je neodređeno):

mkdir -p /mnt/server_share

Onda mountate dijeljenji direktorij - udaljeni file-sistem VFAT:

mount -t smbfs -o username=Administrator,fmask=777,dmask=777 //server/share /mnt/server_share

ili udaljeni file-sistem NTFS:

mount -t smbfs -o username=Administrator,fmask=777,dmask=777,lfs //server/share /mnt/server_share

Da prekinite vezu, koristite:

umount /mnt/server_share

Ako hoćete da postavite u /etc/fstab da olakšate mount, onda upišite sljedeću liniju u tu datoteku:

//server/share  /mnt/server_share       smbfs  defaults,username=your_username,password=**********,fmask=777,dmask=777   0   0

Kako postaviti sidux da bude Samba-Server

Zato što Samba nije na sidux Live-CD, morate obaviti sljedeće da dobijete samba ulaz:

su
fix-unionfs
apt-get install samba
HD-instalacija:
apt-get install samba

Na HD-instalaciji je važno da namjestite Samba konfiguraciju. Ovdje je mali primjer. Ako hoćete vise da znate o korištaju Sambe i postavkama Linux Samba poslužitelja preporučljivo je da pročitate Samba dokumentaciju..

Da namjestite samba-konfiguraciju obavite sljedeće:

otvorite datoteku /etc/samba/smb.conf u editoru (kao kedit ili kwrite) i upišite ovo:

# Globalne promjene - prijedlog je sve jednostavno sto moguće - bez lozinki, radi kao Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Dijeljenje bez prava za pisanje - važno ako dijelite NTFS file-sisteme!

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/hda1 # <-- namjestite na vašu particiju
public = yes

# Dijeljenje s pravima za pisanje - particiju morate mountati
# prava za pisanje - na primjer za FAT32.

[DATA]
comment = Data Partition (First extended partition)
browseable = yes
writable = yes
path = /media/hda5
public = yes

Ne zaboravite memorirati. Sad možete startovati/stopirati sambu sa

/etc/init.d/samba start

i

/etc/init.d/samba stop

Isto možete startovati/stopirati sambu automatski na startu. Izvodite sljedeće:

update.rc.d samba defaults

Sad samba startuje na boot i stopira kad ugasite računalo.
