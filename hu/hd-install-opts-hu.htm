Rendszerindítás CD képfájlból ("fromiso") - Overview

Normál használat esetén az ext4 fájlrendszert javasoljuk; ez az alapértelmezett sidux fájlrendszer.

A "fromiso" cheat-kód megadásával a rendszert egy, a gépeden lévő fájlrendszeren (ext2/3/4) található ISO fájlból is elindíthatod, amely sokkal gyorsabb, mint CD-ről futtatni. (Ekkor a telepítési idő is az eredeti töredéke lesz).

A "fromiso" természetesen sokkal gyorsabb, mint egy CD/DVD olvasó, ráadásul az CD-ROM eszköz is szabadon marad. Alternatívaként használható a QEMU is.

fromiso with Grub2

sidux has provided a grub2 file named 60_fll-fromiso to generate a fromiso entry to the grub2 menu. The only file to configure fromiso is called grub2-fll-fromiso and is found in /etc/default/grub2-fll-fromiso..

First open a terminal and become root with:

sux
apt-get update
apt-get install grub2-fll-fromiso

Then open an editor, which may be kwrite, mcedit, vim or another that you prefer:

mcedit /etc/default/grub2-fll-fromiso

Next uncomment (remove the # ) the lines you need to be operative and replace the default instructions within the "quote marks" with your preferences.

For example, compare this altered grub2-fll-fromiso with the default, (the highlighted lines are the changed lines for instructional purposes):

# Defaults for grub2-fll-fromiso update-grub helper
# sourced by grub2's update-grub
# installed at /etc/default/grub2-fll-fromiso by the maintainer scripts

#
# This is a POSIX shell fragment
#

# specify where to look for the ISO
# default: /srv/ISO ### Note: This is the path to the directory that contains the ISO,
 it is not to include the actual sidux-*.iso file.###
FLL_GRUB2_ISO_LOCATION="/media/disk1part4/sidux-iso"

# array for defining ISO prefices --> sidux-*.iso
# default: "sidux- fullstory-"
FLL_GRUB2_ISO_PREFIX="sidux-"

# set default language
# default: en_US
FLL_GRUB2_LANG="en_AU"

# override the default timezone.
# default: UTC
FLL_GRUB2_TZ="Australia/Melbourne"

# kernel framebuffer resolution, see
# http://manual.sidux.com/de/cheatcodes-vga-de.htm#vga
# default: 791
#FLL_GRUB2_VGA="791"

# additional cheatcodes
# default: noeject
FLL_GRUB2_CHEATCODE="noeject nointro"

Save and close the editor, then run in the terminal

update-grub

Your grub2 grub.cfg will be updated to see the different ISOs you have placed in the directory you specified and will be available on your next boot up.

fromiso with grub-legacy

Lépések:

Válassz ki egy megfelelő helyet, ahová az ISO, illetve a továbbiakban szükséges plusz két fájlt fel tudod másolni. Hozd létre az alap könyvtárat, pl. "/media/sdaX", (sda1 vagy sda2, sda5 vagy 6, használd a neked a megfelelőt), ebben hozz létre egy "sidux" nevű alkönyvtárat és másold bele az ISO fájlt:

$ sux
mkdir -p /media/sdaX/sidux
mv sidux.iso /media/sdaX/sidux

Csatold fel az ISO képfájlt (vagy használd az mc-t [midnight commander], mivel az be tud lépni az .iso képfájlokba, így ki tudod másolni a szükséges fájlokat):

mount -t iso9660 -o loop,ro /media/sdaX/sidux/sidux.iso /mnt

Másold a vmlinuz és initrd.img fájlokat a felcsatolt ISO fájlból a könyvtárba, behelyettesítve a saját könyvtárnevet/merevlemez eszköznevet az alábbi példába:

cp /mnt/boot/vmlinuz0.686 /media/sda5/sidux/vmlinuz0.686
cp /mnt/boot/initrd0.686 /media/sda5/sidux/initrd0.686
umount /mnt

Következhet a grub testreszabása, a /boot/grub/menu.lst fájl szerkesztésével. A következő sorok hozzáadása szükséges a ### END DEBIAN AUTOMAGIC KERNELS LIST sor UTÁN: FIGYELEM, a grub sorokban nem megengedett a * vagy egyéb wildcard használata! Helyettesítsd be saját merevlemez eszköznevedet a példába!

### ISO boot
title sidux 32bit from ISO
kernel (hd0,4)/sidux/vmlinuz0.686 boot=fll quiet vga=791 fromiso=/sidux/sidux.iso
initrd (hd0,4)/sidux/initrd0.686

A következő boot-kor a grub menüjében lesz egy új menüpont, az ISO képfájlról való bootoláshoz. Ha később frissíted az ISO képfájlt, illetve a vmlinuz and intird.img fájlokat, a grub menu.lst fájlját nem kell már szerkesztened.

Általános tudnivalók a fromiso állandó rendszerként való használatához

fromiso, mint állandó rendszer (persist) a merevlemezen

Lehetőség van rra, hogy az előbb leírt fromiso live rendszer megőrizze a beállításait a merevlemezen. When the fromiso uses ext2/ext3/ext4 the default is simply:

persist

When the fromiso uses a vfat filesystem, then you must use a file that contains a linux filesystem and the cheatcode will look like:

persist=/sidux/sidux-rw

A sidux az aufs fájlrendszert használja az úgynevezett "copy on write", felülírás helyett másolás működéshez. Ez teszi lehetővé, hogy az ISO fájlban lévő fájlok és mappák megváltoztathatók legyenek - ilyenkor az eredetiekről másolat készül a memóriába, és a változtatás is ott tárolódik. A persist boot paraméter segítségével a változtatott fájlok ugyanazon a merevlemez partíción fognak tárolódni, ahol a fromiso ISO képfájl.

A fromiso segítségével olyan live rendszert kapunk, amely automatikusan rendelkezik minden olyan tulajdonsággal, amelyet a sidux live ISO tartalmaz. Ennek előnye például az automatikus hardverfelismerés, de ez egyben azt is jelenti, hogy minden egyes bootoláskor létrehozza ugyanazokat a fájlokat, hacsak ezt nem szabályozzuk újabb boot paraméter kódokkal.

A persist használata az olyan kiegészítő kódokkal, mint a noxorgconf és nonetwork, biztosítja azt, hogy ezek a fájlok nem jönnek létre újra és újra minden boot során.

A persist használatával lehetőség nyílik arra is, hogy csomagokat telepítsünk az apt segítsévégel, és minden telepített alkalmazás vagy adat a következő bootoláskor is rendelkezésre fog állni.

fromiso és persist használata bootolható USB-sticks/SSD-cards

Az állandó rendszer ideálisan használható egy bootolható USB pendrive-ra való telepítéssel (az install-usb-gui eszköz használatával), amelyen megtalálható minden fájl és szoftver, amely a napi használathoz szükséges. Your files will be stored in a subfolder on the drive.

DOS és Windows 9x rendszereken a FAT fájlrendszer az alapértelmezett, és az USB meghajtók is leggyakrabban ilyen fájlrendszerrel formázva kerülnek forgalomba. A persist funkció ebben az esetben egyetlen nagy fájlt hoz létre, amely loop device-ként egy teljes fájlrendszert fog magában foglalni.

Az USB-sticks/SSD-cards meghajtóknál ajánlott az ext2 vagy a VFAT fájlrendszerek használata; mivel legvalószínűbben ezek biztosítják a több platformos elérést abban az esetben, ha az adatokat vissza kell menteni. (Az MS Windows™-hoz létezik támogatás az adatcseréhez.) A flash meghajtó dokumentációjában megtalálható, hogy az adott típust hány írási műveletre tervezték.

ext2 fájlrendszerek

Ezen fájlrendszerek esetén a teljes partíció felhasználásra kerül, és a /fll könyvtár automatikusan létrejön a bootolások között megőrzendő adatok tárolására. A tárolható adatok mennyiségét csak a pendrive kapacitása korlátozza.

VFAT fájlrendszer

VFAT használata esetén az állandó rendszer egy minimum 100Mb-os, maximum 2Gb-ra növekvő fájl létrehozásával lehetséges. A fájl neve kötelezően sidux-rw.

Example of creating persist after initial installation

If you are not sure of the mount point, mount the stick and run ls -lh /media to provide a list of all your system mount points. Look for something like drwxr-xr-x 6 username root 4.0K Jan 1 1970 disk. If your output states differently then replace "/media/disk" in line with your requirement, (for example "/media/sdc1"):

disk="/media/disk"

Set the size of the persist partition:

size=1024

Make a directory on the stick:

mkdir $disk/sidux

Run the code to make the persist partition:

dd if=/dev/zero of=$disk/sidux/sidux-rw bs=1M count=$size &&  echo 'y' | LANG=C /sbin/mkfs.ext2 $disk/sidux/sidux-rw &&  tune2fs -c 0 "$disk/sidux/sidux-rw"

A Windows NT/2000/XP (TM) alatt általánosan használt NTFS partíciók egyáltalán NEM HASZNÁLHATÓK állandó fájlok tárolására alkalmas perzisztens partícióként.

A sidux telepítésére USB/SSD devices meghajtóra

A telepítés elvégzése USB-stick/SSD-card merevlemezre hasonlóan egyszerű, mint a merevlemezre való telepítés. Lásd a következő útmutatót:

Requirements:

Any PC that has a USB 2.0 protocol and supports booting from USB/SSD.

A sidux.iso image.

3 kinds of installation to USB/SSD
  • 1) fromiso ; sidux specific: sidux-on-a-stick
  • 2) Full (the full installation to a USB/SSD behaves as normal HD installation and is done through the normal installer).
  • 3) RAW device write. This is ideal when using any Linux, MS Windows or Mac OS X operating system and wish to install sidux-on-a-stick, (with caveats).

USB/SSD fromiso Installation, sidux-on-a-stick

Pre format your usb device with ext2 or fat32 before proceeding (at least a 2 gig capacity) . The device should have 1 partition only and as some BIOSes are temperamental this must be marked bootable.

If using a GUI formatting application like gparted, please ensure that you first delete the existing partition, then recreate the partition before formatting.

The fromiso installation is done through Menu>System>install-sidux-to-usb. On a LIVE-CD you can also click on sidux Installer Icon and choose Install to USB.

Lehetőség van a nyelv, időzóna és egyéb boot paraméterek kiválasztására, illetve az állandó rendszer (persist) kiválasztására egy jelölőnégyzettel.

A telepítés végeztével elkészül a bootolható USB/SSD. Ha a 'persist' opciót nem választottuk ki, erre később is van lehetőség a persist opció hozzáadásával a grub képernyőn. (If vfat, it is probably best to start again though).

Parancssori példa:
 fll-iso2usb -D /dev/sdb -f none --iso /home/sidux/sidux.iso -p -- lang=no tz=Pacific/Auckland

This installs the iso image to the USB device sdb with persist, Norwegian language localisation and Pacific/Auckland (NZL) time on the grub default line.

Az X.Org (videokártya, billentyűzet, egér) beállítások, illetve a hálózati beállítások nem kerülnek eltárolásra - ez ideálissá teszi az USB lemezt arra, hogy több számítógépen is használhassuk.

További információk a testreszabással kapcsolatban:

$ man fll-iso2usb

Full installation to a USB/SSD (behaves as normal HD installation)

The recommended mininmum size of a USB-stick/SSD-card is:
sidux "LITE" needs 2.5 gig PLUS you need data space,
sidux "FULL" needs 4 gig PLUS you need data space

Pre format your device with ext2 and partition the USB-stick/SSD-card as you would a standard PC.

Start the installation from the Live-ISO and choose the partition on the USB/SSD-device, where sidux is to be installed, for example sdbX and follow the sidux installer prompts. Read Installing to your HD

To boot from your USB/SSD 'Boot from USB' must be enabled in your BIOS.

Other note worthy points are:

  • A USB-stick/SSD-card install will be usually be bound to the PC that initiated the original installation. If it is your intention to be able to use the USB/SSD stick on other PCs, it should not have non-free graphics drivers and cheatcodes pre configured, with the exception to probably have the vesa cheatcode hardcoded into the menu.lst, (all to be attended to after a successful installation).
  • After booting the USB-stick/SSD-card to another PC, you will need to alter fstab to access the PCs hard drives.
  • "fromiso" with "persist" is a better option should portability be the intention.

Full installation to a USB Hard Disk Drive like an installation to a partition

A USB Hard Disk Drive has one quite good and appealing application, (particularly to new users coming from MS or another distro), and that is you can install sidux to a USB HDD, plug it in without needing to configure a PC for dual boot (repartitioning, grub alterations etcetera).

Start the installation from the Live-ISO, (or from a USB-stick/SSD-card), as a standard install, not a USB install and choose the partition on the device, where sidux is to be installed, for example sdbX and follow the sidux installer prompts. Grub must be written to the USB HDD partition.

Read Installing to your HD

Other note worthy points are:

  • A USB HDD install will be usually be bound to the PC that initiated the original installation. If it is your intention to be able to use the USB HDD on other PCs, the USB HDD should not have non-free graphics drivers and cheatcodes pre configured, with the exception to probably have the vesa cheatcode hardcoded into the menu.lst, (all to be attended to after a successful installation).
  • If you want to use the installation on another machine be aware that you must make some adjustments. You will need to alter fstab to access the PCs hard drives, maybe xorg.conf and probably the network configuration.v

sidux telepítése VirtualBox alá

A szükséges lépések:

  • 1. Merevlemez képfájl (harddisk image) létrehozása a VirtualBox számára
  • 2. A virtuális gép elindítása az ISO-ról
  • 3. A rendszer telepítése a virtuális lemezre

Rendszer követelmények

Ajánlott memória: 1 GB . Ideális esetben 512 Mb jut a vendég operációs rendszernek és 512 Mb a gazdagépnek (kevesebb memóriával is elindul, de a rendszer teljesítménye csökken).

Lemez terület: Míg maga a VirtualBox program kis helyet foglal (egy tipikus telepítés nagyjából 30 MB-ot igényel), a virtuális gépek virtuális merevlemezei igen méretes tárhelyet igényelnek - például a MS Windows XP (TM) telepítése után a helyigény sok gigabájtra is nőhet. A sidux VirtualBoxban való használatához 5 GB lemezképfájl szükséges, plusz némi swap terület.

A Virtualbox telepítése

apt-get update
apt-get install virtualbox-ose-qt virtualbox-ose-source dmakms module-assistant

Then prepare the module:

m-a prepare
m-a a-i virtualbox-ose-source

Then activate Dynamic Module-Assistant Kernel Module Support for virtualbox, so that the next time the Linux kernel is updated a virtualbox module is prepared for it too, without manual intervention. To do that, add virtualbox-ose-source to the /etc/default/dmakms configuration file. 

mcedit /etc/default/dmakms
virtualbox-ose-source

Then reboot the computer.

A sidux telepítése a virtuális gépre

A VirtualBox varázslója használandó a sidux virtuális gépének létrehozására. Ezután a sidux normál telepítési utasításait kell követni.

sidux telepítése QEMU alá

  • 1. Merevlemez képfájl létrehozása a QEMU számára
  • 2. Az ISO bootolása a QEMU-val
  • 3. A rendszer telepítése a QEMU képfájlba

A QT GUI tool is available to help you configure QEMU:

apt-get install qtemu

Merevlemez képfájl létrehozása

A QEMU futtatásához szükséged lesz egy merevlemez képfájlra. Ez a fájl hordozza a virtuális merevlemez teljes tartalmát.

A lemez elkészítése:

qemu-img create -f qcow sidux.qcow 3G

A parancs hatására létrejön a "sidux.qcow" nevű képfájl. A "3G" paraméter határozza meg a lemez méretét - jelen esetben 3 GB. Használható az M utótag, ha megabájtban kényelmesebb megadni a méretet (például "256M"). Nem kell túl sokat foglalkoznod a lemez méretével - a qcow formátum tömöríti a képfájlt, így annak üres területei nem növelik a fájl méretét.

Az operációs rendszer telepítése

Ez az első alkalom, hogy el kell indítanod az emulátort. Megjegyzés: amikor a QEMU ablakon belül kattintasz, az egérmutatót elkapja az emulátor. Az elengedéshez nyomd meg a következő billentyűkombinációt:

Ctrl+Alt

Ha rendszerindító floppy lemezt kell használnod, így futtasd a QEMU-t:

qemu -floppy sidux.iso -net nic -net user -m 512 -boot d sidux.qcow

Ha a CD-ROM a boot-eszköz, akkor így:

qemu -cdrom sidux.iso -net nic -net user -m 512 -boot d sidux.qcow

Most telepíthető a sidux, a valós merevlemezre való telepítéshez hasonlóan.

A rendszer futtatása

A futtatáshoz szükséges parancs:

qemu [lemez_képfájl]

Hasznos gyakorlat lehet az úgynevezett átfedő (overlay) képfájlok használata. Ezzel a módszerrel egyszer kell létrehozni egy merevlemez képfájlt, majd utasítjuk a QEMU-t, hogy a változásokat egy külső fájlban tárolja. Így rendkívül egyszerű visszaállítani a kiinduló állapotot.

Az overlay képfájl létrehozásához írjuk a parancssorba:

qemu-img create -b [[alap_képfájl]] -f qcow [[overlay_képfájl]]

Helyettesítsd be a merevlemez képfájlt az alap_képfájl helyére (a mi esetünkben sidux.qcow). Ezután így indítsd a quemut:

qemu [overlay_képfájl]

Az eredeti képfájl érintetlen marad. A módszer hátránya, hogy az alap képfájlt nem szabad átnevezni, vagy elmozgatni, mert az overlay képfájl az eredeti névre és helyre hivatkozik.

Valódi partíció, mint a merevlemez képfájl elsődleges partíciója

Lehetséges a QEMU alatt a rendszer valamely valós partícióját is használni (ahogyan egy valódi számítógép elindításakor is megadható a root partíció). Ez a linux szoftveres RAID linear módjával és egy loopback eszköz kombinációjával tehető meg. (ehhez szükség lesz a linear.ko kernel modulra). A trükk abból áll, hogy a QEMU alatt használni kívánt valódi merevlemez partíció elé dinamikusan egy új master boot recordot (MBR) fűzünk.

Tegyük fel, hogy van egy egyszerű lecsatolt /dev/sdaXN partíciónk tetszőleges fájlrendszerrel, amelyet a QEMU lemezképfájl részévé szeretnénk tenni. Először létre kell hoznunk egy kis fájlt, amely az MBR-t fogja tartalmazni:

dd if=/dev/zero of=/mbr/fajl/helye count=32

Ennek hatására egy 16 Kb-os (32 * 512 bájt) fájl lett létrehozva. Fontos, hogy ne legyen túl kicsi (bár az MBR csak 512 bájtos blokkot igényelne), mert a kisebb fájl kisebb blokkméretet eredményez majd a szoftveres RAID eszközön, az viszont rossz hatással lenne a teljesítményre. Adjunk hozzá egy loopback eszközt az MBR fájlhoz:

losetup -f /mbr/fajl/helye

Tegyük fel, hogy a létrejött eszköz a /dev/loop0 lett (azaz ez az elsőnek létrejött loopback eszközünk). A következő lépés, hogy elkészítjük az "összefűzött" MBR + /dev/sdaN diszk képfájl párost, a szoftveres RAID segítségével:

 modprobe linear
 mdadm --build --verbose /dev/md0 --chunk=16 --level=linear --raid-devices=2 /dev/loop0 /dev/sdaN

Eredményként egy /dev/md0 eszközt kapunk, amelyet a QEMU lemezképfájlként fog tudni használni (ne felejts el megfelelő jogokat beállítani a fájlon, hogy a QEMU hozzáférhessen). Az utolsó (és szintén trükkös) lépés, hogy beállítjuk a lemez adatait (lemez geometria és partíciós tábla), úgy, hogy az MBR-ben az elsődleges partíció kezdőpontja megegyezzen az egyik /dev/sdaN eszközével a /dev/md0 tömb belsejében (példánkban a 16 * 512 = 16384 bájt eltolása). Ezt az fdisk paranccsal végezheted el a gazdagépen (nem az emulátorban! A QEMU alapértelmezett nyers lemez detektáló rutinja gyakran eredményez nem kilobájtra kerekíthető offszeteket (mint pl. 31.5 KB az előző bekezdésben), amit a szoftveres RAID nem tud lekezelni. Így tehát a gazdagépen add ki):

 fdisk /dev/md0

Hozz létre egy elsődleges partíciót a /dev/sdaN-hez igazodva, és addig igazgasd a 's'ector parancsot az e'x'pert menüből, amíg az első cilinder (ahol az első partíció kezdődik), megegyezik az MBR méretével. Végül a 'w'rite parancs elmenti az eredményt a fájlba: készen is vagyunk. Van egy partíciód, amit a saját rendszereden is felmountolhatsz és a QEMU is lekezeli, mint lemez képfájlt:

 qemu -hdc /dev/md0 [...]

Természetesen bármilyen rendszerbetöltő telepíthető a QEMU-ban erre a képfájlra, mivel a az eredeti /boot/sdaN partíció tartalmazza a szükséges eszközöket.

A QEMU Accelerator (gyorsító) Modul használata

A QEMU fejlesztői készítettek egy opcionális kernel modult, ami felgyorsítja a QEMU-t, ezáltal gyakran közel natív teljesítményt biztosítva. Ehhez a következő opcióval töltsük be a modult:

major=0

A szükséges /dev/kqemu eszköz létrehozásának automatizálására a következő parancs szolgál:

echo "options kqemu major=0" >> /etc/modprobe.conf

Ez kiegészíti a modprobe.conf-ot, amely biztosítja hogy a modul mindig a megfelelő opcióval töltődik be.

qemu [...] -kernel-kqemu

Ezzel a teljes virtualizációt engedélyezzük, amely érezhető sebességnövekedést eredményez.

A QEMU indítása:

qemu -cdrom /tmp/pkg/sidux-debug.iso -net nic -net user -m 512
