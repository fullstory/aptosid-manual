Nem nyílt forráskódú meghajtókat igényló hardverek

A különböző országokban alkalmazott jogszabályok összetettsége miatt a sidux live-cd csak ún. dfsg-free szoftvereket szállít.

Szabad forráskódú meghajtók nVidia, ATI, Radeon és Intel videokártyákhoz

A nyílt forráskódú meghajtók a felsorolt típusú videokártyákhoz telepítve vannak a sidux rendszerben.

Intel drivers should work perfectly for 2D and 3D video acceleration as the drivers are included in the Intel free series.

Amennyiben zárt forráskódú meghajtókat használtál és szeretnél visszatérni nyílt forráskódúakra, szerkesztened kell a /etc/X11/xorg.conf fájlt, DE ELŐTTE KÉSZÍTS MENTÉST A FÁJLRÓL, mert a /etc/X11/xorg.conf helytelen szerkesztése rengeteg szenvedést tud okozni.

Az /etc/X11/xorg.conf szerkesztéséhez root jogok kellenek. Keresd meg SECTION DEVICE bekezdést és változtasd meg annak driver részét nv, ati vagy intel szövegre (to name but a few).

A xorg.conf szerkesztését csak saját felelősségedre végezd!!

További információk az Intel   ati radeon   radeonhd   nv  X.Org

Forráslista

Mivel a sidux iso csak dfsg-free szoftvert tartalmaz, valószínűleg ki kell majd bővíteni a rendszert contrib / non-free forrásokkal is a /etc/apt/sources.list.d/debian.list és /etc/apt/sources.list.d/sidux.list vagy /etc/apt/sources.list fájlokban:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Bináris, zárt forráskódú meghajtó az nVidia kártyákhoz dmakms-sel

NOTE: Use apt-cache search nvidia and apt-cache show <package> to ascertain the correct driver for you. There are basically 2 choices, current Debian Sid 3D drivers and legacy Debian Sid 3D drivers.

For current 3d nvidia drivers ≥ GeForce 6xxx :

A modul előkészítése:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Ezután aktiválni kell az úgynevezett dmakms (Dynamic Module-Assistant Kernel Module Support) támogatást az nVidia-hoz, ennek eredményeképpen a kernel frissítésekor az nVidia meghajtó is automatikusan fog frissülni. Ehhez az nvidia-kernel-source sort hozzá kell adni a /etc/default/dmakms konfigurációs fájlhoz:

echo nvidia-kernel-source >> /etc/default/dmakms

Majd:

m-a a-i nvidia-kernel-source

Ezt követően:

apt-get install nvidia-glx

A következő lépés az xorg.conf módosítása:

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Indítsd újra a számítógéped a módosítások érvénybe lépéséhez.

When xorg updates you only need to reinstall nvidia-glx:

apt-get install --reinstall nvidia-glx

When a new Debian sid nvidia driver becomes available thru apt:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Reboot PC for installation of the module to take effect.

For legacy 3d nvidia drivers ≥ GeForce 5xxx :
m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Alter dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Sikertelen modul betöltés

Ha az nVidia modul nem töltődik be valamely okból :

modprobe nvidia

Majd indítsd újra a számítógépet.

Ha a modul még mindig nem tud betöltődni:

 m-a a-i -f nvidia-kernel-source

Ez újraépíti a modult. Ezek után indítsd újra a számítógépet.

További olvasmány:

$ /usr/share/doc/dmakms

You can also read nvnews for other options.

ATI, Radeon and other video drivers

2D Drivers

Az X.Org X szerver meghajtói (lásd xserver-xorg további meghatározásért) 2D támogatást biztosítanak NVIDIA Riva, TNT, GeForce, és Quadro kártyákhoz, valamint az ATI Mach, Rage, Radeon, és FireGL kártyáihoz sorban az atimisc, r128 és radeon al-meghajtókkal. Radeon and Intel both support 2d acceleration (textured xv) for video playback.

3D Drivers

Some ATI cards also support 3D, so far up to the r5xx chipsets.

To automatically inherit newly packaged non-free firmware when they get updated for 2D and 3D video cards:

apt-get install firmware-linux

Then reboot the computer.

Firmware meghajtók

A szükséges firmware-ek a következőkben tárgyalt eszköz / firmware lista alapján, illetve az újonnan kifejlesztett fw-detect szkript segítségével telepíthetőek (ez a sidux-scripts csomagban található).

$ fw-detect

Általános szabályként javasolt a modprobe -r <modulename> futtatása a modprobe előtt.

NTFS-partíciók írása ntfs-3g segítségével

Figyelem: Bár az ntfs-3g kernel modul stabilnak minősített, sose használd külső biztonsági mentés nélkül és természetesen ne éles rendszeren! Ha megteszed, a te hibád ha adatvesztés következik be, tehát csak saját felelősségedre használhatod!

Nyiss egy parancssort és add ki a következő parancsokat (Lásd A merevlemez partícionálása - A lemezek elnevezése fejezetet):

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Kilépés a konzolból: exit

Az NTFS-köteted fel lett csatolva rw és most már írni is fogod tudni. De még egyszer figyelmeztetünk! Csak vészhelyzetben használd, nem javasolt a napi szintű alkalmazása.
