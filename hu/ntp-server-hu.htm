Időszerver beállítása

Az első lépés parancssorban, root felhasználóként:

apt-cache search ntp
apt-get update && apt-get install  ntp ntp-doc
update-rc.d -f ntp defaults
futtasd az update-rc.d-t a beállítások elvégzése után

A dokumentáció megtalálható a gépeden az alábbi helyen:

/usr/share/doc/ntp-doc/html/index.html
Javasoljuk, hogy készíts rá könyvjelzőt!

Ez egy nagyon alapos és nagy méretű dokumentáció, a beállításhoz nem lesz szükség az egész végigolvasására.

Az ntp szolgáltatás elindulásához újraindítás szükséges, és még az újraindítás előtt érdemes a pontos időt kézzel beállítani.

Az ntp szolgáltatás a /etc/ntp.conf fájlban meghatározott szerverektől kéri le a pontos időt, tehát ezt a fájlt fogjuk leginkább szerkeszteni.

Mind az ntpdate, mind az ntp daemon (ennek neve ntp) a /etc/ntp.conf fájl elején lévő szerver listából fog választani. Egy példa szerver lista:

 A pool.ntp.org-hoz több, mint 100 alacsonyabb stratum-ú NTP szerver van rendelve,
# A helyi szerver minden indításkor más csoportot választhat ezekből
#  *** Fontold meg az NTP szerverek szövetségéhez való csatlakozást! ***
#  ***  http://www.pool.ntp.org/#join  ***
server 192.168.3.24
server ntp.blueyonder.co.uk
server uk.pool.ntp.org
server 1.uk.pool.ntp.org
server 2.uk.pool.ntp.org
server 0.europe.pool.ntp.org
server 1.europe.pool.ntp.org
server 2.europe.pool.ntp.org

Az első egy lokális hálózaton futó másik gép, amely szintén ntp-t futtat [azon a gépen ez a sor így néz ki: 'server 192.168.3.1']

A második az Internet-szolgáltató időszerverének címe.

Ezután következik egy angliai időszerver, majd pár európai, a biztonság kedvéért. Egyébként az internet-szolgáltatók névszerverei általában időszerverként is működnek, ezt az alábbi módon lehet leellenőrizni:

ntpdate -v

A fenti sor nem végez beállítást, mindössze egy idő-eredménnyel tér vissza, pl.:

# ntpdate -v 192.168.3.24
19 Sep 19:09:27 ntpdate[13329]: ntpdate 4.2.2@1.1532-o Wed Aug  9 12:08:54 UTC 2006 (1)

Az ntp időszerverek teljes listája itt található: http://www.pool.ntp.org/

Az időszerver szolgáltatás engedélyezése a lokális hálózaton lévő gépeknek

# A lokális hálózaton a felhasználók ne piszkálhassák a szervert
restrict 127.0.0.1 nomodify
restrict 192.168.3.0/24

Az időszervert behirdetjük a többi gép számára:

# Ha a helyi gépek számára szeretnél időszerverként működni, használd a következő sort
# (Természetesen saját hálózatod paramétereit helyettesítsd be.)
broadcast 192.168.3.255

Az ntp.conf fájl kicsit fura, ha egérrel nyitod meg, a rendszer diff fájlnak ismeri fel. Az ntp szolgáltatás elindítása előtt be kell állítanod az időt, pl.

# ntpdate -u -b uk.pool.ntp.org
19 Sep 19:19:33 ntpdate[15641]: step time server 62.3.200.116 offset 0.001523 sec

Az ntp szolgáltatásként való indításához (minden rendszerindításkor elindul) a következőt kell tenned:

ntpq -pn

Ha minden rendben történt, ehhez hasonlót kell látnod:

A harmadik sorban látható csillag ( *82.219.3.1 ) az aktív idő szervert jelzi, amelyet elsődlegesként kezelünk. Ennek hatására folyamatosan megtartja a pontos időt, a 123-as portot használva. Egy példa iptables sor a tűzfal engedélyezéséhez:

# Network Time Protocol (NTP) Server
$IPT -A udp_inbound -p UDP -s 0/0 --destination-port 123 -j ACCEPT
$IPT -A INPUT -j ACCEPT -p tcp --dport 123
