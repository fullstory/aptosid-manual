A sidux beállítása távoli gépeken található SAMBA (Windows) megosztások elérésére

Minden parancsot root-ként (egy terminálon, vagy Konsole ablakban) kell kiadni, az URL-eket a Konqueror cím mezőjébe kell beírni (a Konquerort normál felhasználóként kell futtasd).

server = a Windows gép neve, vagy IP címe
share = a megosztott erőforrás (pl. könyvtár) neve

KDE alatt, a Konquerorba írd be az URL-t: smb://server, vagy a teljes címet: smb://server/share

Egy konzol ablakban a távoli gépen található erőforrásokat a következő parancs segítségével listázhatod ki:

smbclient -L server

Ha egy megosztást fel akarsz mountolni egy könyvtárba (teljes jogokkal minden felhasználó számára), tartsd szem előtt: a célkönyvtárnak (mountpoint) léteznie kell. Ha nem létezik, az alábbiak szerint hozd létre (tetszőleges névvel):

mkdir -p /mnt/server_share

Ezután mountolható a megosztás - távoli VFAT file-rendszer:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777 //server/share /mnt/server_share

vagy egy távoli NTFS file-rendszer:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777,lfs //server/share /mnt/server_share

Egy kapcsolat bontása így történik::

umount /mnt/server_share

Ha mountolás egyszerűbbé tétele érdekében egy bejegyzést kívánsz elhelyezni az /etc/fstab-ban, add hozzá a következő sort:

//server/share  /mnt/server_share       cifs  defaults,username=your_username,password=**********,file_mode=0777,dir_mode=0777   0   0

Hogyan lesz a sidux Samba kiszolgáló

Mivel a samba nem található meg a sidux live CD-jén, a következő lépések szükségesek a használatához:

su
apt-get update
apt-get install samba
Merevlemezre telepített rendszer esetén:

Mivel a samba nincs előtelepítve, a következő lépsek segítségével telepíthető:

sux
apt-get update
apt-get install samba

Telepített rendszer esetén szükséges a Samba kiszolgáló beállításain igazítani. Íme egy egyszerű példa. Ha többet szeretnél tudni a Samba használatáról, illetve a Samba kiszolgáló Linuxon való üzemeltetéséről, ajánlatos elolvasnod a Samba dokumentációt..

A samba-konfiguráció beállítása a következőképpen történik:

nyisd meg a /etc/samba/smb.conf beállítófájlt egy szövegszerkesztőben. (pl. kedit, vagy kwrite) és írd bele az alábbiakat:

# Általános beállítások - Javasolt mindent a lehető legegyszerűbbre venni:
# Jelszó nélküli hozzásférés, mint a Windows 9x rendszerekben

[global]
security = share
workgroup = WORKGROUP

# Egy megosztás írási jogok nélkül - ez fontos, ha NTFS file-rendszer kerül megosztásra!

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/sda1 # <-- ide a saját NTFS partíciódat írd
public = yes

# Írási joggal megosztott partíció - már eleve írási jogokkal kell felmountolni.
# Alkalmazható például FAT32 file-rendszer esetén.

[DATA]
comment = Data Partition (first extended Partition)
browseable = yes
writable = yes
path = /media/sda5
public = yes
A megosztások ellenőrzése

Biztonsági megfontolások nélküli samba megosztásokat (pl. otthoni, helyi hálózatban) az alábbi parancsokkal készíthetsz elő:

A könyvtárakra és azok tartalmára legalább -rwxr-xr-x jogsultságokat alkalmazz:

ls -la pathTo/dirShareName/*

Ha nem a kívánt jogokat látod, állítsd be:

chmod -R 755 pathTo/dirShareName

Ha írhatóvá is kívánod tenni:

chmod -R 777 dirShareName

Íme egy módszer a megosztásaid működésének ellenőrzésére: (ne felejtsd el elindítani előtte a szervert):

smbclient -L localhost

Valami ehhez hasonlót kell kapnod:

 smbclient -L localhost
Password:
Domain=[HOME] OS=[Unix] Server=[Samba 3.0.26a]

        Sharename       Type      Comment
        ---------       ----      -------
        IPC$            IPC       IPC Service (3.0.26a)
        MaShare         Disk      comment
        print$          Disk      Printer Drivers
Domain=[MSHOME] OS=[Unix] Server=[Samba 3.0.26a]

Ha nem állítottál be jelszót, csak üss egy ENTER-t

Ne felejtsd el elmenteni a beállításokat. A samba szervert ezután elindítani és leállítani a következő módon lehet:

/etc/init.d/samba start

és:

/etc/init.d/samba stop

A samba automatikusan elindítható rendszerindításkor. Ehhez add ki a következő parancsot:

update-rc.d samba defaults

Mostantól a boot során a samba elindul és szabályosan leáll rendszerleállításkor.

A sambáról további tudnivalókat ezen a címen találhatsz..
