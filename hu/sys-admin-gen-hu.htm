Szolgáltatások engedélyezése

insserv (Pontos 2008-04 and forward ): To start/stop already installed services:

Read /usr/share/doc/insserv/README.Debian, the release notes and the man pages carefully:

 $ man insserv
 $ man invoke-rc.d
 $ man update-rc.d
 google LSB headers

For 'start':

/etc/init.d/<service> start

For 'stop':

/etc/init.d/<service> stop

For 'restart:

/etc/init.d/<service> restart

To prevent services running at-startup

update-rc.d <service> remove
[will remove all startup links]

To make sure a service runs at boot [not always required]

update-rc.d <service> defaults
[will make startup links]

To read the list of what services are running:

ls /etc/rc5.d

S means the service will start.
K means service will not start.

Older Installs (before Pontos 2008-04)

insserv on installed systems before Pontos 2008-04

Warning: Older installs will not get insserv automatically, due to users own initscripts or symlinks. It is NOT recommended to install insserv on installations prior to Pontos. This package should be used with care, as incorrect or missing LSB dependencies of existing initscripts can lead to an unbootable system. Installing insserv on an existing install may require some manual changes.

Read read the Debian documentation notes, and release notes before installing and enabling insserv.

Boot-Up Manager (bum) - Graphical services configuration tool

If the logic of a debian system boot up sequence is not very clear and familiar to you, you should not play with symlinks, permissions and so forth. In order to avoid messing up your system. Boot-Up Manager will help you to automate your configuration.

Boot-Up Manager is a GUI runlevel configuration editor which allows you to configure what init services are invoked when the system boots up or reboots. It displays a list of every service which could be started at boot. You can toggle individual services on and off..

apt-get install bum

To use the Boot-Up Manager GUI:

$ sux
password:
bum

Detailed documentation on Boot-Up Manager (bum).

Szolgáltatás, vagy processz kilövése (kill)

A pkill program igen hasznos, mivel "emberi fogyasztásra" alkalmas módon kommunikál, mind egyszerű felhasználóként, mind root-ként használható egy terminál ablakban, vagy tty konzolon.

pkill -n service

Ha nem tudod biztosan hogyan írják az adott processz, vagy szolgáltatás nevét, a pkill <tab> <tab> ad egy listát az elérhető nevekről.

A htop szintén egy jól használható alternatíva. (A killall -9 az utolsó lehetőséged).

sidux futási szintek - init

Íme a sidux operációs rendszer futási szintjei, ám felhívjuk a figyelmed, hogy különböznek a Debian stabil változatának futási szintjeitől:

  • runlevel 0 powers off the PC.
  • runlevel 1 single user (safety mode). Dæmons like apache and sshd are stopped. Do not go to this level via remote access.
  • runlevel 2 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows
  • runlevel 3 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows (this is where dist-upgrades are actioned).
  • runlevel 4 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows
  • runlevel 5 Multi-User mode with the network running the X-windows system, and/or to start X-windows.
  • runlevel 6 reboot.
  • runlevel S where early boot time services are executed on a 'once only basis'. You cannot switch to it after it has been run.

Elveszett root jelszó

Visszanyerni nem tudod, csak beállítani egy új jelszót.

Először indítsd el a Live-CD rendszert.

Rootként csatold fel a rendszered root partícióját (pl. /dev/sdb2)

mount /dev/sdb2 /media/sdb2

Ezután chroot-olj a saját root-partíciódra végül állítsd be az új jelszót:

chroot /media/sdb2
passwd

Betűkészletek sidux alatt

Megfelelő dpi beállítások - alap filozófia

A DPI beállításokat eléggé problémás megállapítani, de történetesen az X ezt tökéletesen el tudja végezni, ha az /etc/X11/xorg.conf "Monitor Section" részében a DisplaySize változóban megadjuk a képernyő fizikai méreteit.

Megfelelő felbontási és frissítési értékek

Minden monitornak meg van a maga tökéletes beállítás-kombinációja, de sajnos nem mindegyik adja meg magáról a helyes DDC értékeket, így néha kézzel kell azokat felülbírálnunk.

Megfelelő grafikus kártya meghajtók

Néhány újabb ATI és Nvidia kártya egyszerűen nem működik jól a szabad Xorg meghajtókkal, így ilyen esetben az egyetlen használható megoldás a kereskedelmi, zárt meghajtók használata. A sidux jogi okokból nem telepíti őket előre. A megoldás itt olvasható.

Alapértelmezett betűtípusok, megjelenítési módjuk és méreteik

A sidux (a Debian által) előre kiválasztott szabad betűtípusokkal érkezik, melyek kipróbáltan kiegyensúlyozott megjelenést nyújtanak, a felhasználók által összeválogatott font-készletek esetlegesen ronthatnak ezen a minőségen. De van a Debianban néhány hatékony eszköz (apart from KDE>systemsettings ) melyek segíthenek kiigazítani más betűkészletek pontatlan renderelését is. Tartsd szem előtt, hogy minden betűtípusnak van néhány valóban tökéletes megjelenítésű mérete, ám más méretekben esetleg csúnyán mutatnak.

Segítségedre lehet, ha egy kicsit eljátszol a dpi méretekkel, az alábbi program használatával:

fix-dpi-kdm

Meg kell mutatnia a képernyő megfelelő DPI értékét, de megadthatsz neki kísérletképpen más értéket is. Újra kell indítanod az X-et, pl init 3 és vissza init 5, vagy újraindítod a gépet.

Miután megváltoztatod a betűtípusokat vagy a DPI-t (az X-ben, vagy a Firefox/Iceweasel-ben), némi utánigazítást kell végezned, hogy az eredmény valóban megfelelő legyen, különösen ha fontkészleteket átváltottad bitmap fontokról True Type készletekre vagy vissza:

dpkg-reconfigure fontconfig-config

A native és autohinter opciókat automatikusra tedd. Esetleg kísérletezhetsz vele a legjobb eredmény érdekében.

Újra kell majd építened a font cache-t:

fc-cache -f -vv

Ha ez nem működik, akkor újra kell telepítened a font cache alapértelmezett beállító-állományát tartalmazó csomagot:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
GTK alapú alkalmazások, mint a Firefox/Icewasel

A GTK alapú programok általában problémásak a KDE alapértelmezett beállításaival. A dolog a következőképpen orvosolható:

apt-get install gtk2-engines system-config-gtk-kde gtk-qt-engine gtk2-engines-qtcurve

In System Settings >Appearance you will have a new menu item called GTK Styles and Fonts. Set 'GTK Styles' to use 'Clearlooks' and have the 'GTK Fonts' set to use 'KDE fonts' or experiment with the various options to suit your preferences.

Ez valószínűleg megjavítja a betűk megjelenítését a gtk alkalmazásokban.

modprobe lp

apt-get remove --purge cupsys cups

http://localhost:631

Sound in sidux

Sound is muted by default in sidux.

The KDE version uses Kmix and XFCE uses Mixer.

Often it is only a matter of clicking on the sound icon in the taskbar and unchecking the 'Mute box'.

Kmix

In Kmix you will need to activate the options you prefer for channel options, Kmix>Setting>Configure Channels. Or in a terminal:

$ kmix
XFCE

In XFCE run the mixer application and add some controls via the Multimedia>Mixer and click the Select Controls box. Or in a terminal:

$ xfce4-mixer

Alsamixer

If you prefer to use Alsamixer, it is in the alsa-utils package:

apt-get update
apt-get install alsa-utils
exit

Set your preferred sound settings as $user from the terminal:

$ alsamixer

See also the sidux wiki.
