A Grub újratöltése

A grub újratöltése az MBR-be, például egy újabb verzió telepítése esetén:

/usr/sbin/grub-install --recheck --no-floppy /dev/sda

Lehet, hogy ezt a parancsot többször is ki kell adnod, amíg biztos lehetsz abban, hogy sikeresen lefutott.

A grub helyreállítása Windows által felülírt MBR esetén

Ha ilyen esetben akarod helyreállítani a grub-ot, a számítógépet egy sidux live-cd használatával indítsd el.

  1. A partícióid azonosításához (pl. [h,s]d[a..]X) minden alábbi utasítást root-ként kell kiadni, tehát szerezzünk jogokat: 
    $ sux
  2. Már root-ként add ki: 
    fdisk -l
cat /etc/fstab
    Így megkaphatod a megfelelő neveket.
  3. Miután azonosítottad a megfelelő partíciót, hozz létre számára egy mount point-ot: 
    mkdir -p /media/[hdxx,sdxx,diskx]
  4. Mountold fel: 
    mount /dev/xdxx /media/xdxx
  5. Most már vissza tudod tölteni az elsődleges merevlemezed MBR területére a grub-ot: 
    /usr/sbin/grub-install --recheck --no-floppy --root-directory=/media/xdxx /dev/sda

GRUB a boot-menedzser - mi ez?

Általános tudnivalók

A Grub egy boot-menedzser ami képes különböző fájlrendszerekből operációs rendszereket betölteni. A Grub nem hagyatkozik a blokk-listára, hogy megtalálja a rendszer elindításához szükséges fájlokat és alapból sok fájlrendszert képes olvasni: ext2, ext3, reiserfs, ... (Linux), Fat32, Fat16 (Windows), UFS (BSD), Minix és még sok más.

A Grub ismeri a BIOS bővítményeket (ami azt jelenti, hogy a (8 GB-nál magasabban lévő partíciókat is be tudja bootolni!) és nincs benne az ismert 1024-cilinder-korlát. A Grub-ot tényleg nem érdekli fizikailag hol van a kernel, amire szüksége van az a partíciók meghatározása és az elindítandó kernelhez vezető elérési útvonal (ami akár szimbolikus link is lehet). Ha módosul a konfiguráció (pl. a menu.lst) nem szükséges semmilyen programot futtatni mint pl. Lilo esetében, az új konfiguráció a következő indításkor máris használható.

A sidux Grub-ot használja a CD-ről történő induláshoz, és merevlemezre telepítés esetén is a Grub az alapértelmezett rendszertöltő. Mindazonáltal a Grub utólag is problémamentesen telepíthető. A Grub az MBR-be a következő, root-ként kiadott parancs segítségével telepíthető:

grub-install --root-directory=/media/sdaX /dev/sdY

A /media/sdaX egy olvashatóra felmountolt partíciót jelent, melyen egy "/boot/grub" nevű könyvtárban megtalálhatók a szükséges fájlok (stage2, stage1_5xxx). A "menu.lst" konfigurációs fájlt ugyanebben a könyvtárban kell létrehozni. A /dev/sdY az a merevlemez, amelynek MBR-jébe a Grub-ot telepíteni akarod, alapértelmezésben a /dev/sda.

Meghajtó-nevek/Eszköznevek

A Grub nem tesz különbséget SCSI és IDE merevlemezek között. A (hd0) mindig az első diszk, amely első helyen szerepel a (BIOS beállítások szerinti) boot-sorrendben, függetlenül attól, hogy IDE vagy SCSI. Szokatlan partíció-számozási mód is; az első hd első partíciója (hd0,0). Az elsődleges partíciók 0-3-ig kerülnek számozásra, a logikai partíciók 4-től felfelé.

Grub Linux Címke
(fd0) /dev/fd0 Floppy (Windows alatt)
(hd0) /dev/sda első hd
(hd0,0) /dev/sda1 A hd0 1. partíciója
(hd0,5) /dev/sda6 Logical part. a hd 1-en

A Grub alatt minden eszköznevet zárójelekbe () kell tenni. Ám vigyázz a kernel-sorok utáni bejegyzésekkel! A Grub-bal ellentétben a Kernel a normál linux-jelölések szerint várja az adatokat. A következő példa sorai ugyanarra a partícióra mutatnak, név szerint az első merevlemez negyedik partíciójára. Az első sor a Grub jelölése szerinti, a második egy tipikus linux-bejegyzés:

root (hd0,3)
kernel          /boot/vmlinuz root=UUID=bf955abf-ce1f-45a3-9ee5-93b5c5dc4fe0 ro quiet vga=791
initrd          /boot/initrd.img

Parancs-mód (Grub-Shell)

Egy nagyon kényelmes szolgáltatás a parancs-mód. A "c" billentyű megnyomásával érhető el, amikor is feljön a grub-shell. Lehetővé teszi, hogy megkeress és elindíts különböző rendszereket és fájlokat (pl. kernel) még hibás menu.lst esetén is.

Egy kis próbálkozás érdekében, rootként bármikor elindíthatod a Grub-ot a "grub" parancs kiadásával. A Grub-shellben kiadott "help" megmutatja az összes, rendelkezésre álló parancsot. Parancsonként részletes súgó kérhető a help "parancsnév" beírásával. A Linux parancssorhoz való vésszatérés a "quit" parancs megadásával történik (A Grub-Shell bezáródik). A Grub-shellben a tab-billentyű hasonló funkcióval bír, mint a Linux parancsértelmezőiben, ami nagy segítség. A parancsok, eszköznevek és fájlnevek automatikusan kiegészíthetők a tab-billentyűvel.

A Grub megadja a lehetőséget, hogy a "find" segítségével megkeresd a kernelt és az initrd-t mielőtt bármit megpróbálnál bootolni, mint a következő példa esetében is:

find /boot/vmlinuz

A Grub megmutatja azt a partíciót amelyiken a "/boot" könyvtárban megtalálható a "vmlinuz" nevű kernel:

(hd0,3) (hd1,0)

Az összes elérhető partíció megjeleníthető a "geometry" segítségével, például:

geometry (hd0)

a parancs kimenete:

"drive 0x80: C/H/S = 3648/255/63, The number of sectors = 58605120, /dev/sda Partition num: 0,
Filesystem type is fat, partition type 0xb Partition num: 1,
Filesystem type is fat, partition type 0xb Partition num: 2,
Filesystem type is fat, partition type 0xc Partition num: 4,
Filesystem type is ext2fs, partition type 0x83 Partition num: 5,
Filesystem type unknown, partition type 0x82 Partition num: 6,
Filesystem type is fat, partition type 0xb"

Miután megtaláltad a kernelt vagy egy valós initrd-t (vagy miniroot.gz-t), a "boot" parancs elindíthatja őket.

Dual- és multi-boot

Több operációs rendszer indításának engedélyezése nagyon egyszerű a Grub-ban, a chainloading használatával. Ha például adott egy MS Windows (TM) és egy Debian Etch OS, mellette egy sidux 'fromiso' telepítés, egy kicsit állítanod kell a grubon. Tehát, szerkeszd /boot/grub/menu.lst fájlt és a ### END DEBIAN AUTOMAGIC KERNELS LIST sor után írd be a következőket:

####MS Windows (TM) perhaps even MS Vista (TM)
Title Windows (95/98/XP/Vista
rootnoverify (hd0,1)
makeactive
chainloader +1

### Debian etch
title Debian etch
kernel (hd0,3)/debian/vmlinuz root=/dev/sda4 ro (or whatever)
initrd (hd0,3)/debian/miniroot.gz (or (whatever)

### ISO boot
title sidux 32bit from gaiaISO
kernel (hd0,3)/sidux3/vmlinuz-2.6.22.1-slh-smp-7 boot=fll quiet vga=791 fromiso=/sidux3/siduxgaia.iso
initrd (hd0,3)/sidux3/initrd.img-2.6.22.1-slh-smp-7

Néhány felhasználónak különböző verziójú MS Windows (TM) rendszerekre van szüksége. A MS Windows (TM) mindig az első diszkről akar bootolni. Így hát az egyik partíciót el fogjuk rejteni és aktiválunk helyette egy másikat. A következő utasításokat a Grub-parancsértelmezőben kell kiadni és ha sikeresen lefutottak, akkor beírhatók a menu.lst fájlba. Ezt minden Windows-partícira el kell végezni. Ne felejts el biztonsági másolatot készíteni:

hide (hd0,0) # az 1. diszk 1. partíciójának elrejtése
unhide (hd0,1) # az 1. diszk 2. partíciójának láthatóvá tétele
rootnoverify (hd0,1) # az 1. diszk 2. partícióját beállítjuk gyökér-fájlrendszernek
makeactive # aktiváljuk az 1. diszk 2. partícióját
chainloader +1 # rámutatunk az indító partícióra
boot # elindítjuk az operációs rendszert

Ha egy második Windowst akarsz indítani a második lemezről, virtuálisan meg kell cserélned az 1. és 2. lemezeket. Ezáltal a Windows egy szimulált 1. diszket fog látni:

map (hd0) (hd1)
map (hd1) (hd0)

Buktatók

Bár Grub nem ütközik az 1024 cilinderes akadályba, tehát az 1024. cilinder felett elhelyezkedő rendszereket is képes elindítani, a "/boot/grub/stage2" fájlt a BIOS-nak el kell tudni érnie (az 1024. cilinder alatt kell lennie). A stage2 nem helyezhető át a telepítés után (pl. egy defragmentáló megkísérelheti). A telepítéskor rögzített adatok többé nem felelnének meg a valós állapotnak. Ez megelőzhető a stage2 írásvédetté tételével (a megfelelő könyvtárban állva):

Dos: attrib +r +s stage2
LINUX: # chmod a -w ./stage2

A hivatalos Grub GyIK.
