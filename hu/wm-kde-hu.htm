Some useful KDE applications not preinstalled on sidux KDE Lite

You may need to have non-free enabled in /etc/apt/sources.list.d/debian.list

konq-plugins - plugins for Konqueror.

This package contains a variety of useful plugins for Konqueror, the file manager, web browser and document viewer for KDE. Many of these plugins will appear in Konqueror's Tools menu.

Highlights for web browsing include web page translation, web page archiving, auto-refreshing, HTML and CSS structural analysis, a search toolbar, a sidebar news ticker, fast access to common options, bookmarklets, a crash monitor, a microformat availability indicator, a del.icio.us bookmarks sidebar, and integration with the aKregator RSS feed reader.

Highlights for directory browsing include directory filters, image gallery creation, archive compression and extraction, quick copy/move, a sidebar media player, a file information metabar/sidebar, a media folder helper, a graphical disk usage viewer and image conversions and transformations.

unrar - Unarchiver for .rar files (non-free version)

Unarchiver for .rar files (non-free version) Unrar can extract files from .rar archives. If you want to create .rar archives, install package rar.

Asztali felület fagyások

Hasonló helyzetekben nincs mindig szükséged a reset gomb megnyomására. Ez adatvesztéshez vagy a fájlrendszer károsulásához vezethet. Akárhogy is, a fájlrendszer sosem lesz tiszta egy erős újraindítás után (a fájlrendszer nem tiszta)

Először, próbálj meg átmenni terminálba alt-ctl-F1 vagy indítsd újra az X-szervertalt-ctl-backspace, (Ha az előző két megoldás egyike sem működik, még mindig van remény):

A SYSRQ-billentyű (print-billentyű, a billentyűzet jobb felső részén található) segíthet abban hogy tisztán újraindíthass egy lefagyott rendszert.

A következő billentyű-kombináció sorozatának lehetőségei:
* ALT+SYSRQ+R (vissza adja a billentyűzet vezérlését)
* ALT+SYSRQ+S (végrehajt egy szinkronizációt)
* ALT+SYSRQ+E (Term jel küldése minden folyamatnak, kivéve init)
* ALT+SYSRQ+I (Kill jel küldése minden folyamatnak, kivéve init)
* ALT+SYSRQ+U (fájlrendszerek mountolása csak olvasásra, meggátolja az fsck-t újraindításnál)
* ALT+SYSRQ+B (rendszer újraindítása, az előző lépések kihagyásával, ez az úgy nevezett 'hard reset').

A legjobb ha mindegyik lépésnek adsz pár másodpercet hogy befejeződjön, mert például a folyamatok bezárása eltarthat egy darabig. A szükséges betük könnyedén megjegyezhetőek a következő módon: "Reboot(újraindítás) System(rendszer) Even(egyenlő) If(ha) Utterly(teljesen) Broken(törött)"

Bejelentkezési problémák KDE alatt

A /tmp mappa tartalma normál esetben, minden újraindításnál kitisztul, szóval az X-szerver néhány mappája és foglalata is letőrlődik.

Alapesetben, a boot-folyamatnál, az x11-common parancs létrehozza ezeket a dolgokat az X-Org számára.

Meglehet, hogy ezek a parancsok nem lettek előhívva a boot folyamán. Hogy újból létrehozzuk a szükséges linkeket, írjuk be:

# X-ORG: # dpkg-reconfigure x11-common

A KDE-nek általában 5% helyre van szüksége, ahol a /tmp mappa elhelyezkedik, ahhoz hogy a szükséges fájlok bejelentkezéskor létrejöhessenek. Ha 95% -os telített partícióval rendelkezel, nem fogsz tudni bejelentkezni, ehelyett visszadob majd egy tty-re.

Ugyanez érvényesül a kdm újraindulgatásra, ami nem fog beengedni. a megoldás, hogy bejelentkezel tty-n és letörölsz/letisztítasz pár nem használatos alkalmazást vagy fájlt.

Alternatív megoldásként használhatsz egy másik X ablak kezelőt, ami nem igényel annyi helyet (például a fluxbox ami már a sidux telepítésnél is elérhető), vagy chroot -olhatsz a sidux live-CD/DVD -t használva hogy letisztítsd a partíciód hogy bejelentkezhess KDE-be.

85% is the recommended absolute maximum for a partition that KDE accesses for its /tmp files.(15% free).

Installing sidux KDE Art and Themes

To install the latest sidux-art to an existing installation:

apt-get install sidux-art-kde-xxxx sidux-art-wallpaper-xxxx
(Where xxxx is the name of the release for example sidux-art-kde-erebos)

This will install the sidux wallpaper and themes

Go to your Control Centre > Appearance and Themes > Theme manager and choose sidux-art theme/wallpaper from the list.

How to enable or change KDM-themes

A kdmtheme egy téma kezelő a KDM részére. Tartalmaz egy KDE VezérlőKözpont Modult (KCM) amely segítségével könnyedén telepítheted, törölheted vagy cserélheted a KDM témáidat.

apt-get install kdmtheme

Then restart the Control Centre

Hogyan engedélyezzük vagy cseréljük a KDM-témákat manually

Minden változást root jogokkal kell végrehajtani.

Témák engedélyezése

Jelenleg a KDM-témák nincsenek alapesetben engedélyezve a Debian rendszereken.

Adj hozzá a következő
UseTheme=true
sort [X-*-Greeter]-stanza a /etc/kde3/kdm/kdmrc fájlhoz.
Témák cseréje

Ha engedélyezve van, a KDM alapesetben engedélyezi a témák cserélgetését. Hogy ezt megváltoztassuk, csak változtassuk meg a következő sort:

Theme=/usr/share/apps/kdm/themes/circles

a következőre

Theme=/usr/share/apps/kdm/themes/
Témák kikapcsolása

Ha ki szeretnéd kapcsolni a témádat, csak válts vissza a UseTheme=false -ra.

NOTE: You have to restart KDM after you've made changes to the kdmrc.
