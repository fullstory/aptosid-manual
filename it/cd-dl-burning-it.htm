Scaricare il CD di sidux

MOLTO IMPORTANTE:sidux, come liveCD Linux, si basa su una tecnologia ad alta compressione, e, per questo motivo, è richiesta particolare attenzione per masterizzare l'immagine ISO. Utilizzare solo supporti CD (o DVD+RW) di alta qualità e masterizzare in modalità DAO (disk-at-once) non più velocemente di 8x.

Definizioni dei file in ogni mirror

Ogni mirror sidux ha i seguenti file:

MD5SUM
MD5SUM.asc
SHA256SUM
SHA256SUM.asc
SOURCES
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

Il file .manifest è la lista di tutti i pacchetti contenuti nella specifica iso. Il file .iso è l'immagine da scaricare. I file .md5 e .sha256 sono utilizzati per controllare l'integrità del file iso.

I file .asc sono la firma che permette di verificare che i file .md5 e .sha256 non sono stati modificati e che possono essere a loro volta utilizzati per verificare l'integrità della iso.

I nostri ringraziamenti vanno a tutti quelli che ospitano mirror per i loro sforzi a favore di sidux.

I link agli scaricamenti ed ai mirror si trovano su sidux.com

Se si ha a disposizione un server FTP veloce con sufficiente banda, per favore informare il sidux Forums. In alternativa andare sui forum di sidux o nel canale IRC #sidux su oftc.net

Somma MD5 e verifica

MOLTO IMPORTANTE:sidux, come liveCD Linux, si basa su una tecnologia ad alta compressione, e, per questo motivo, è richiesta particolare attenzione per masterizzare l'immagine ISO. Utilizzare solo supporti CD (o DVD+RW) di alta qualità e masterizzare in modalità DAO (disk-at-once) non più velocemente di 8x.


Si tratta di un controllo utilizzato per verificare l'integrità dei file. La somma md5 corrente del file viene confrontata con una somma nota. In questo modo è possibile verificare se un file è stato danneggiato o modificato; ne è consigliato l'uso in caso di file scaricati da internet.

Se la somma md5 del file scaricato corrisponde a quello del file md5, si può essere sicuri che il file è stato scaricato correttamente. Su Linux si può ottenere la somma con: (il calcolo richiede un po' di tempo)

$ md5sum file_da_controllare

Il risultato della somma verrà scritta sulla console.
Con md5summer (486kb) la somma md5 può essere verificata anche in Windows.

Le immagini ISO di sidux sono sempre fornite con un file somma md5 e dovrebbero essere sempre controllate prima della masterizzazione. Questo garantisce che, in caso di problemi, questi non siano dovuti al file scaricato, e libera i forum da problemi che non possono essere risolti perché l'immagine ISO è danneggiata. Si dovrebbe effettuare il controllo in una console. Basta spostarsi nella cartella che contiene sia l'immagine ISO che il file con la somma md5 e poi eseguire:

Scaricare i seguenti file dal mirror:

MD5SUM
MD5SUM.asc
SHA256SUM
SHA256SUM.asc

In un terminale:

$ md5sum sidux.iso

Se non corrispondono, si ottiene un errore:

"sidux.iso: Error
md5sum: Warning: calculated checksum does not match!"

Se il file scaricato è corretto, il programma termina senza messaggi.

Se si vuole verificare la validità dei file .asc fare le operazioni seguenti come $USER:

Ecco un esempio utilizzando:
sidux-2009-02-aether-kde-lite-i386-200907141544.iso

1) Importare la chiave pubblica di "Stefan Lippers-Hollmann" (slh) da un key-server:

$ gpg --keyserver wwwkeys.eu.pgp.net --recv-keys F80994F6
gpg --edit-key F80994F6
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Adesso si può verificare l'integrità degli altri file con:

$ gpg --verify MD5SUM.asc

Se si è classificata la chiave come affidabile utilizzando i necessari comandi gpg, questo è il risultato che si dovrebbe ottenere con "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

Si otterrà una 'Good signature' con un WARNING se non era stato scelto di classificare la chiave:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u

Come si fa a masterizzare il cd con Windows?

MOLTO IMPORTANTE:sidux, come liveCD Linux, si basa su una tecnologia ad alta compressione, e, per questo motivo, è richiesta particolare attenzione per masterizzare l'immagine ISO. Utilizzare solo supporti CD (o DVD+RW) di alta qualità e masterizzare in modalità DAO (disk-at-once) non più velocemente di 8x.


Naturalmente, si può masterizzare il CD anche con Windows. Il file scaricato va masterizzato come file ISO. Se Winrar (o qualche altro strumento di archiviazione) è associato ai file .ISO, potrebbe essere seccante perché si potrebbe scambiarlo per un archivio. Dal file ISO si deve masterizzare un CD. Non masterizzare il file ISO come disco dati!! È necessario masterizzare il file ISO come Disk-At-Once.

Si hanno a disposizione varie e valide opzioni per masterizzare in Windows.

Programmi di masterizzazione open source per Windows

cdrtfe: compatibile con Windows 9x/ME/2000/XP (testato con Win95, Win98SE, Win2000, WinXP); solo per Win9x/ME: ASPI Layer (per esempio Adaptec ASPI 4.60)

Programmi di masterizzazione closed source e proprietari per Windows
  • Nero (non nero express!)
  • Un altro valido strumento freeware è CD/DVD Burner XP pro
  • Burncdcc da terabyteunlimited può masterizzare solo ISO...

Masterizzare il CD con Linux

MOLTO IMPORTANTE:sidux, come liveCD Linux, si basa su una tecnologia ad alta compressione, e, per questo motivo, è richiesta particolare attenzione per masterizzare l'immagine ISO. Utilizzare solo supporti CD (o DVD+RW) di alta qualità e masterizzare in modalità DAO (disk-at-once) non più velocemente di 8x.


Se si utilizza già Linux sul PC, si dovrebbe masterizzare il cd o da shell o con uno dei programmi di masterizzazione che sono installati.

Con sidux e altre distribuzioni Linux viene fornito K3b come strumento di masterizzazione di default. Avviarlo, scegliere "Extras" -> "Burn CD-Image..." e scegliere il file ISO da masterizzare (per esempio sidux.iso).

Prima di tutto K3B calcola la somma MD5 per il file ISO (questo richiede un po' di tempo). Se il valore mostrato è simile al valore del file MD5 (per esempio sidux.iso.md5), lo scaricamento è stato eseguito con successo e si può masterizzare il file semplicemente cliccando su "Start".

I problemi con la masterizzazione di solito accadono utilizzando una interfaccia grafica (frontend). Si può masterizzare direttamente dalla shell con burniso .
