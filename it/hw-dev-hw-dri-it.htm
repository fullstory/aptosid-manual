Suggerimenti per hardware che necessita di driver non liberi

A causa della complessità della legge, il liveCD di sidux contiene solo dfsg-free software.

Driver a sorgente aperto Xorg per nVidia, ATi, Radeon, Intel

I driver open source Xorg per nVidia, ATi, Radeon, Intel sono preinstallati in sidux.

Intel drivers should work perfectly for 2D and 3D video acceleration as the drivers are included in the Intel free series.

Se si sta utilizzando driver proprietari e si vuol tornare indietro a quelli open source bisogna modificare /etc/X11/xorg.conf, COMUNQUE FARE UNA COPIA DI BACKUP PRIMA, dal momento che modificare /etc/X11/xorg.conf in maniera non corretta risulterà in lacrime versate.

Per modificare il file /etc/X11/xorg.conf cliccarci sopra con il tasto destro ed aprirlo con i permessi di root. Trovare la sezione intitolata SECTION DEVICE e cambiare il display driver in nv OPPURE ati OR intel (to name but a few).

Modificare xorg.conf è interamente a vostro rischio!!!

Sources List (liste dei depositi del software)

Probabilmente si vorranno aggiungere le sezioni contrib / non-free al file /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list o al /etc/apt/sources.list. Poi procurarsi un accesso internet:

# Vedere il file sources.list(5) per maggiori informazioni, in particolare
# ricordare che potete usare solo http, ftp o file URIs
# Le fonti su CDROMs sono gestite mediante lo strumento apt-cdrom.
# La cartella /etc/apt/sources.list.d/ fornisce un modo per aggiungere delle
sorgenti sources.list
# in file separati che confluiscono nel file sources.list

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main
fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main
fix.contrib fix.non-free

Driver a sorgente chiuso, binari per: nVidia con dmakms

NOTE: Use apt-cache search nvidia and apt-cache show <package> to ascertain the correct driver for you. There are basically 2 choices, current Debian Sid 3D drivers and legacy Debian Sid 3D drivers.

For current 3d nvidia drivers ≥ GeForce 6xxx :

Prepara il modulo:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Quindi attivare Dynamic Module-Assistant Kernel Module Support (dmakms) per nvidia, così la prossima volta che il kernel Linux viene aggiornato viene preparato anche un modulo nvidia per questo, senza intervento manuale. Per fare ciò aggiungere nvidia-kernel-source al file di configurazione /etc/default/dmakms:

echo nvidia-kernel-source >> /etc/default/dmakms

Quindi:

m-a a-i nvidia-kernel-source

Seguito da:

apt-get install nvidia-glx

Poi modifica xorg.conf:

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Riavviare il sistema perché venga eseguita l'installazione del modulo.

When xorg updates you only need to reinstall nvidia-glx:

apt-get install --reinstall nvidia-glx

When a new Debian sid nvidia driver becomes available thru apt:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Reboot PC for installation of the module to take effect.

For legacy 3d nvidia drivers ≥ GeForce 5xxx :
m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Alter dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Errore nel caricamento del modulo

Se nvidia non si dovesse caricare per qualsiasi motivo eseguire:

modprobe nvidia

Quindi riavviare il sistema.

Se ancora dovesse dare errore nel caricamento:

m-a a-i -f nvidia-kernel-source

Questo comando ricostruisce il modulo, quindi riavviare.

Leggere:

$ /usr/share/doc/dmakms

You can also read nvnews for other options.

ATI, Radeon and other video drivers

2D Drivers

The drivers for the X.Org X server (see xserver-xorg for a further description) provides support in 2D for NVIDIA Riva, TNT, GeForce, and Quadro cards and the ATI Mach, Rage, Radeon, and FireGL cards along with atimisc, r128, r6xx/r7xx and radeon sub-drivers. Radeon and Intel both support 2d acceleration (textured xv) for video playback.

3D Drivers

Open source 3D drivers are included for the Intel free series. Some ATI cards also support 3D, so far up to the r5xx chipsets.

To automatically inherit newly packaged non-free firmware when they get updated for 2D and 3D video cards:

apt-get install firmware-linux

Then reboot the computer.

Firmware drivers

Il firmware richiesto può essere determinato con la seguente enumerazione device / firmware oppure usando fw-detect, il nuovo script appena sviluppato (contenuto nel pacchetto sidux-scripts).

$ fw-detect

Scrivere su partizioni NTFS con ntfs-3g

Vi avvisiamo: sebbene ntfs-3g sia definito come stabile, non usarlo mai per dei salvataggi esterni, e naturalmente non usarlo su sistemi di produzione! L'eventuale perdita di dati dovuti ad un suo utilizzo non potrà in alcun modo essere imputabile a sidux, quindi stare attenti!

Aprire una shell/console/terminale e immettere i seguenti comandi: leggere Partizionare il disco fisso - Denominazione dei dischi

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx

Per uscire dalla console digitare: exit

Ora il volume NTFS dovrebbe essere montato rw e si dovrebbe essere in grado di scrivere i dati su di esso. Ma, di nuovo, stare attenti! Usarlo soltanto in situazioni di emergenza, non è raccomandato per un uso su base giornaliera.
