Attivare Ndiswrapper

sidux ha gia' installati tutti i paccehtti necessari: il modulo del kernel e ndiswrapper-utils.

Vi serve un file '.inf' di "Windows" per il driver; reperitelo e caricatelo con:

ndiswrapper -i driver.inf

Per controllare se l'installazione e' andata a buon fine usate l'opzione -l:

root@blubb:/cartella/con/i/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

E' importante che risulti un "driver installed" e un "device present" nella risposta dell'opzione -l. Se non risultano entrambi qualcosa e' andato storto (controllate i messaggi di errore emessi da -i e/o dmesg), perche' se ci sono errori a questo punto, non avrete la connessione wlan.

Caricate il modulo ndiswrapper con:

modprobe ndiswrapper

Per controllare se e' stato creato un dispositivo wifi, digitate:

iwconfig

Per controllare se e' stato trovato un punto di accesso wireless:

iwlist wlan0 scan

a questo punto potete configurare la scheda di rete wifi con:

/usr/sbin/netcardconfig
oppure, per le versioni sidux piu' recenti digitate:
ceni
in una konsole. Oppure usate il centro di controllo sidux digitando in konsole:
siduxcc

In caso di aggiornamento del kernel, dovrete reinstallare il modulo ndiswrapper. Usando ovviamente il numero del vostro kernel corrente:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Poi caricatelo con modprobe:

modprobe ndiswrapper

Non dovrete reinstallare i dirver windows in quanto questi sono separati dal modulo del kernel.

Potete anche scrivere 'ndiswrapper' in /etc/modules per caricarlo all'avvio

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

Potete fare la procedura mediante il centro di controllo di sidux a interfaccia grafica - siduxcc - che trovate nel menu di KDE

Ndiswrapper Avanzato

ndiswrapper e' un modulo che abilita Linux ad usare i driver WLAN di Windows. E' necessario in quanto alcune schede di rete wireless non offrono un driver open source o libero, rendendo cosi' difficoltosa l'implementazione della scheda in Linux. Ndiswrapper gira attorno al problema, 'avvolgendo' il driver Windows e traslandone gli input e gli output dalla scheda a Linux.

Identificare il driver necessario

Prima di tutto vi serve il driver windows appropriato per la vostra scheda WLAN. Queste schede hanno numerosi e differenti chip set per lo stesso modello e quindi il solo modo di identificare accuratamente una scheda e' quello di utilizzare il device id (USBID/PCIID).

Per trovare questo 'id' aprite un terminale:

Per un adattatore WLAN su USB
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

Come vedete nell'esempio di cui sopra, lo USBID per l'adattatore WLAN D-Link Corp. [hex] DWL-122 e' 2001:3700.

Per un adattatore WLAN (mini) PCI / PCMCIA 
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

Questo 'id' ha il seguente schema 'vendor:device' pari a 168c:0024 per questa scheda Atheros.

Ora cercate su: ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list una voce con lo 'id' di cui sopra. Se siete fortunati troverete una voce corrispondete e un driver raccomandato.

Vi serviranno i file INF e SYS di quel driver. La cosa migliore e' trovare un file ZIP da espandere con unzip. Se trovate un file EXE dovrebbe essere un archivio auto-estraente (usate wine per l'estrazione) oppure un file CAB (estraete con cabextract). Ora mettete i file in una nuova cartella della vostra home.

Attivare Ndiswrapper e installare i driver

Entrate nella cartella con i file del driver windows :

bla@blubb:~$cd /cartella/con/i/driver/windows
bla@blubb:~$su
root@blubb:/cartella/con/i/driver/windows# ndiswrapper -i <driver.inf>
installing net5416

Per controllare se l'installazione e' avvenuta con successo, usate l'opzione -l:

root@blubb:/cartella/con/i/driver/windows# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

E' importante che otteniate un "driver installed" e un "device present" come risposta all'opzione -l. Se non le vedete entrambe allora qualcosa e' andato storto (controllate i messaggi di errore mostrati dall'opzione -i e/o dmesg), perche' se ci sono degli errori qui, non si potra' proseguire.

Se non sono segnalati errori, carichiamo il modulo del kernel: 

root@blubb:/da/qualsiasi/cartella# modprobe ndiswrapper

Se non otterrete nessun messaggio dopo questo comando, allora tutto e' andato bene, se invece vedrete un qualche messaggio allora controllate con dmesg per ulteriori informazioni.

il comando "iwconfig" dovrebbe ora mostrare un dispositivo wireless:

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

Se volete caricare ndiswrappper all'avvio della macchina, scrivete semplicemente ndiswrapper in /etc/modules, a mano oppure usate: 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
Reinstallare Ndiswrapper dopo un aggiornamento del kernel

Dopo un aggiornamento del kernel dovrete reinstallare il modulo del kernel di ndiswrapper. Il file .deb e' localizzato in /usr/src/kernel-downloads/ <versione-del-kernel>/ndiswrapper-modules- <versione >.deb,. Per esempio per il kernel , 2.6.21.5-slh-smp-2, dovrete fare:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Dopo l'installazione, caricatelo con:

root@blubb:/da/qualsiasi/cartella# modprobe ndiswrapper

Non dovrete reinstallare i driver windows in quanto la parte driver e' separata dal modulo del kernel.

Informazioni generali

Per configurare il nuovo dispositivo usate netcardconfig o sidux-networkconfig (o 'Ceni' nelle sidux piu' recenti). Un dettagliato how-to puo' essere trovato qui .

Se trovate un altro driver o uno nuovo, fate qualche prova. Puo' causare problemi, un kernel panic nel peggiore dei casi, ma provandolo potrete aiutare il progetto ndiswrapper e altri utenti ad avere benefici dai migliori driver per le varie schede. E dopo una prova avvenuta con successo non dimenticatevi di aggiornare/completare le informazioni sulla lista di ndiswrapper.
