La catena LAMP in sidux

L'acronimo LAMP si riferisce ad un insieme di programmi liberi comunemente usati insieme per far girare siti o sever web dinamici
Linux, il sistema operativo
Apache, il server web
MySQL, il sistema di gestione dei database (o un server database)
Perl, PHP, and/or Python, Linguaggi di scripting

ATTENZIONE: Mai usare il PC dedicato al lavoro quotidiano come server web! Usate un apposito PC per quella funzione e per nessun'altra!

Usi di un server:
a) un server locale di prova per progettisti web senza connessione internet, che e' lo scopo di questo capitolo;
b) un server privato ' chiuso' connesso alla rete;
c) un server web privato pienamente aperto all'internet
d) un server web commerciale. Questo e' oltre lo scopo del nostro manuale

Requisiti minimi

Almeno 256 mega di ram. Una quantita' inferiore puo' causare molti problemi in quanto un server mysql richiede molta ram per funzionare correttamente. In caso di scarsita' di RAM, Mysql produrra' un errore tipo "cannot connect to mysql.sock" "non posso connettermi a mysql.sock.

Il pacchetto generico per server di sidux

Il pacchetto generico per server di sidux consiste di un installazione pre-pacchettizzata mediante l'installatore meta-package. Le applicazioni che installa sono:

apache2
apache2-utils
apache2-mpm-prefork
php5 php5-common
mysql-server
mysql-common
libapache2-mod-php5
php5-mysql
phpmyadmin

ATTENZIONE 

apt-get remove --purge splashy

Perche' splashy inceppa sempre mysql.

Il file di configurazione di Apache e' localizzato in:/etc/apache2/apache2.conf e la vostra cartella web /var/www

Per controllare se php e' installato e avviato correttamente, create semplicemente un file test.php nella cartella /var/www che includa la funzione phpinfo() esattamente come mostrato sotto.

mcedit /var/www/test.php

# test.php
<?  phpinfo(); ?>

Puntate il navigatore web su:

http://localhost 
or 
http://vostro_IP:80

Questo dovrebbe mostrare tutta la vostra configurazione php e le impostazioni predefinite.

Potete modificare i valori che vi interessano e impostare dei domini virtuali usando il file di configurazione di apache.

Se volete provare la vostra installazione aprite il navigatore internet e digitate la seguente stringa nella barra degli indirizzi

http://vostro_indirizzo_IP/apache2-default/

Se otterrete un messaggio di benvenuto, la vostra installazione e' corretta.

la cartella 'root' predefinita dei documenti per apache2 e' /var/www. Per cambiarla createne un'altra dove preferite e collegatela a /var/www nel modo seguente: 

mkdir /home/myself/www
ln -s /home/myself/www /var/www

Eseguendo i comandi mostrati sopra sarete in grado di modificare il vostro sito web dalla vostra home e come utente normale.

Proteggere in via predefinita i file del Server

Un aspetto a volte incompreso di Apache e' la funzionalita' dell'accesso predefinito. Questa, se non fate qualcosa per cambiarla, sarebbe il modo in cui il server puo' trovare il percorso verso un file attraverso le normali regole che mappano gli URL, e quindi inviarlo ai client che hanno richiesto quegli URL.

Consideriamo il seguente esempio:

 1. # cd /; ln -s / public_html
 2. Accessing http://localhost/~root/

Questo consentirebbe ai client di 'passeggiare' a loro piacimento attraverso l'intero filesystem! Per sistemare questa cosa aggiungete il seguente blocco alla configurazione del vostro server:

<Directory />
   Order Deny,Allow
   Deny from all
</Directory>

Questo impostera' impedira' l'accesso predefinito a locazioni del filesystem. Aggiungete dei corretti blocchi di <Directory> per permettere l'accesso solo alle aree che volete. Ad esempio,

<Directory /usr/users/*/public_html>
    Order Deny,Allow
    Allow from all
</Directory>
<Directory /usr/local/httpd>
    Order Deny,Allow
    Allow from all
 </Directory>

Ponete particolare attenzione alla interazione alle direttive <Location> e <Directory> ; per esempio anche se <Directory /> nega l'accesso, una direttiva <Location /> potrebbe sovrascriverla.

Siate guardinghi anche sul far girare giochi con la direttiva UserDir; impostarla in modo tipo "./" potrebbe avere lo stesso effetto, per root, come il primo esempio sopra. Se state usando Apache 1.3 o versione superiore, e' fortemetne raccomandata l'inclusione della seguente stringa nei file di configurazione del vostro server:

UserDir disabled root

# apache2-ssl-certificate

Avviate lo script 'a2enmod ssl' ad es.

# a2enmod ssl

Questo generera' un (link) collegamento simbolico tra mods- available e mods- enabled

Fate una copia del file '/etc/apache2/sites-available/default' nella cartella /etc/apache2/sites-available/ - e chiamatelo 'ssl'

# cp /etc/apache2/sites-available/default /etc/apache2/sites-available/ssl

Create un sym-link a questo nubo file per questo uso

#  ln -s /etc/apache2/sites-available/ssl /etc/apache2/sites-enabled/                     
(or)
#a2ensite ssl

Se volete cambiare qualsiasi impostazioni della configurazione di base fatelo in /etc/apache2/apache2.conf e se volete cambiare la root dei documenti predefinita fatelo nel file /etc/apache2/sites-available/default e riavviate il server apache.

Per riavviare il server Apache usate il seguente comando:

#/etc/init.d/apache2 restart

Ora bisogna cambiare l'indirizzo della porta in /etc/apache2/ports.conf che in via predefinita stara' in ascolto sulla porta 80 mentre ora con SSL occorre impostare l'ascolto sulla porta 443

 Listen 443

Modificate il file /etc/apache2/sites-available/ssl (o qualsiasi nome che avrete dato al file di configurazione del vostro nuovo siti ssl) e cambiate la porta 80 in 443 nel nome del sito.

Aggiungete le due stringhe seguenti da qualche parte nel file /etc/apache2/apache2.conf file

  SSLEngine On

  SSLCertificateFile /etc/apache2/ssl/apache.pem

Modificate 'SSLCertificateFile' /etc/apache2/ssl/apache.pem e immettete le locazioni del file di certificato e del file della chiave certificata. Vedete l'esempio sotto

SSLCertificateFile /etc/apache2/ssl/online.test.net.crt
SSLCertificateKeyFile /etc/apache2/ssl/online.test.net.key

Impostate 'ServerSignature' a off, seguite questi passi e modificate il file /etc/apache2/apache2.conf aggiungendo le seguenti due stringhe

  ServerSignature Off
 ServerTokens ProductOnly

Se volete permettere tipi di file indice differenti, controllate le seguenti stringhe nel file /etc/apache2/apache2.conf 

 DirectoryIndex index.html index.cgi index.pl index.php index.xhtml index.shtml

Riavviate iil server apache

 /etc/init.d/apache2 restart

Ora dovreste avere un server di prova, se vi venisse voglia di connettervi ad interent con esso, NON FATELO!... Usate un altro PC puramente dedicato ad essere un internet web server!
