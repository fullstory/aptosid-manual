aptosid-*.iso - Password di root (amministratore) in modalità liveCD

Notare che ogni volta che si esegue qualcosa con i permessi di root si deve aver chiaro che cosa si sta facendo! Per navigazioni web in una internet non è richiesto un accesso come root.

aptosid-*.iso default password

Sul liveCD aptosid non è impostata alcuna password di root. Se si vuole avviare un programma che richiede i privilegi di root si hanno le seguenti scelte:

Simple method, just type:

sux
Impostare una password di root (temporanea)

Aprire una console/shell:

aptosid@0[aptosid]$ sudo passwd
Enter new UNIX password (immettere la nuova password UNIX):
Retype new UNIX password (ridigitare la nuova password UNIX):
passwd: password updated successfully (password aggiornata con successo)
aptosid@0[aptosid]$

Ora si può usare questa password per il resto della sessione live di aptosid.

sudo non è supportato nelle installazoni su hard disk, vedere sudo e sux.

Avviare un programma da una console di root

Utilizzando sux si farà diventare root l'utente con i privilegi Xapp (funzioneranno le applicazioni ad interfaccia grafica lanciate da root).

Aprire una console/shell:

aptosid@0[aptosid]$ sux
root@0[aptosid]# gparted (o quello che si vuole...)

Alcune applicazioni KDE richiedono dbus-launch prima del nome dell'applicazione:

dbus-launch <applicazione>

Un'altra opzione generica comune a tutti i maggiori Desktop Manager è:

Alt+F2 su-to-root -X -c <applicazione>

Per esempio:

Alt+F2 su-to-root -X -c gparted

Per uscire dalla condizione di root digitare nella console:

exit

Oppure più semplicemente cliccare l'angolo in alto a destra della finestra per chiudere la console.

Nel caso di un blocco su aptosid-*.iso c'è bisogno di fare le azioni seguenti e seguire le indicazioni per impostare una password:

alt+ctrl+F1
sudo passwd

Una volta che la password è attiva, eseguire:

alt+ctrl+F7

Installare un software anche in una sessione liveCD, eccetto se si abilita fromiso e persist.

apt-get update
apt-get install nome-del-pacchetto-che-serve

Nota: quando si uscirà dalla sessione liveCD, non verrà mantenuto nessun cambiamento.

Come scrivere su una partizione NTFS con ntfs-3g

Attenzione: sebbene ntfs-3g sia dichiarato ufficialmente "stabile", non usarlo mai senza un salvataggio esterno, e naturalmente non su un sistema di produzione! Se lo si fa, una eventuale perdita di dati non sarà da imputare a aptosid, quindi il suo uso è rischioso!

Aprire una shell e immettere i seguenti comandi: Leggere anche: Partizionare il disco fisso - Assegnare nomi al disco

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx

Per uscire dalla console scrivere: exit

Ora il volume NTFS dovrebbe essere montato in scrittura (rw) e si dovrebbe essere in grado di immagazzinare dati su di esso. Ma ancora, stare attenti! Usarlo solo in situazioni di emergenza, non è raccomandato per un uso giornaliero.
