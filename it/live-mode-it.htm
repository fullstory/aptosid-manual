Password di root (amministratore) in modalità liveCD

Notare che ogni volta che si esegue qualcosa con i permessi di root si deve aver chiaro che cosa si sta facendo! Per navigazioni web in una LAN non è richiesto un accesso come root.

Sul liveCD sidux non è impostata alcuna password di root. Se si vuole avviare un programma che richiede i privilegi di root si hanno le diverse scelte seguenti:

Digitare sux farà diventare root l'utente con i privilegi Xapp (funzioneranno le applicazioni a interfaccia grafica lanciate da root).

Impostare una (temporanea) password di root:

Aprire una console/shell:

sidux@0[sidux]$ sudo passwd
Enter new UNIX password (immettere la nuova password UNIX):
Retype new UNIX password (ridigitare la nuova password UNIX):
passwd: password updated successfully (password aggiornata con successo)
sidux@0[sidux]$

Ora si può usare questa password per il resto della sessione live di sidux.

Lanciare il programma con:

sudo
or
sux
or
kdesu

Premere Alt+ F2, e immettere:

sudo

con in mente il programma che serve, ad esempio:

sudo gparted

Avviare un programma da una root-shell:

aprire una console/shell:

sidux@0[sidux]$ sux
root@0[sidux]# gparted (o quello che si vuole...)

Per uscire dalla condizione di root digitare nella console:

exit

o semplicemente cliccare l'angolo in alto a destra della finestra per chiudere la console.

Installare software anche in una sessione liveCD

apt-get update
apt-get install nome-del-pacchetto-che-serve

Nota: quando si uscirà dalla sessione liveCD, non verrà mantenuto nessun cambiamento.

Come scrivere su una partizione NTFS con ntfs-3g

Attenzione: sebbene ntfs-3g sia dichiarato ufficialmente "stabile", non usarlo mai senza un salvataggio esterno, e naturalmente non su un sistema di produzione! Se lo si fa, una eventuale perdita di dati non sarà da imputare a sidux, quindi il suo uso è rischioso!

Aprire una shell e immettere i seguenti comandi: Leggere anche: Partizionare il disco fisso - Assegnare nomi al disco

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx

Per uscire dalla console scrivere: exit

Ora il volume NTFS dovrebbe essere montato in scrittura (rw) e si dovrebbe essere in grado di immagazzinare dati su di esso. Ma ancora, stare attenti! Usarlo solo in situazioni di emergenza, non è raccomandato per un uso giornaliero.
