Dimensionamento delle Partizioni ed Esempi

Per un uso normale si raccomanda ext3; è il filesystem predefinito di sidux ed è ben mantenuto.

I dischi vengono partizionati e/o formattati con il gestore delle partizioni GParted. Il programma ha un'interfaccia grafica ed è facilmente comprensibile.

Gparted può anche ridurre oppure muovere partizioni ma anche manipolare partizioni NTFS (con un avviso speciale: una volta modificata la partizione NTFS si deve riavviare immediatamente prima di fare qualsiasi altra operazione). Leggete la documentazione completa per gparted. Le modifiche alle partizioni ntfs possono essere eseguite anche con strumenti proprietari come Partition Magic (TM) e Acronis (TM).

SALVARE SEMPRE I DATI PRIMA di modificare le partizioni!

Nel caso che una partizione dovesse esere montata, smontarla, incluso lo swap, e, con un click del tasto destro del mouse sull'icona della partizione in gparted oppure attraverso un terminale. Per esempio:

umount /dev/sda1

La partizione di swap può essere smontata in un terminale con:

swapoff -a

All'inizio, 5 GB sono più che sufficienti per una installazione su hard disk, ma non si avrà abbastanza divertimento con questa onfigurazione. Una installazione ragionevole dovrebbe disporre come minimo di 12 GB di spazio. Per chi è nuovo di linux suggeriamo soltanto 2 partizioni per iniziare (root/home e swap) dal momento che si semplifica la prima installazione, succesivamente si possono creare partizioni aggiuntive per una /home separata e partizioni solo per i dati.

Si dovrebbe avere sempre una partizione di swap (equivalente al file di swap di windows, solo che è molto più efficace). Per un utilizzo normale, la partizione di swap dovrebbe essere pari al doppio della quantità della RAM.

Per lo scambio dei dati con una installazione windows si dovrebbe usare vfat (fat32) oppure ext2 dal momento che è disponibile un driver per MS Windows™ per lo scambio dei dati (XFS non è supportato).

Chiunque crei più di una partizione dovrebbe scriversi il nome per un futuro riferimento!

Ecco alcuni semplici esempi per differenti dimensioni di hd e differenti utilizzi, il tutto arrotondato.

Configurazione per provare Linux - 60GB per un pc, dove Windows usa abbastanza spazio
Disk Size Filesystem Mountpoint/System
sda1 20GB NTFS WindowsXP
sda5 30GB FAT32 Dati per Win e Linux
sda6 8GB ext3 /
sda7 500MB Swap Swap

sda5 6GB ext3 /home
sda6 5GB ext3 /var
sda7 178.5GB ext3 /video
sda8 1GB Swap Swap
Un Pc Standard con Windows 120GB per usare il meglio dei due mondi.

sda5 8GB ext3 /

sda7 70GB FAT32 Dati per WindowsXP-Linux

PC Linux per gioco e Media 160GB
Disk Size Filesystem Mountpoint/System
sda1 18GB ext3 /
sda5 20GB ext3 /home
sda6 30GB ext3 /opt
sda7 90GB ext3 /data

Linux e Windows su 20 GB
Disk Size Filesystem Mountpoint/System
sda1 5GB NTFS Win-System
sda2 10GB FAT32 Programmi e dati per Windows e Linux
sda5 5GB ext3 /
sda6 500MB Swap Swap

Quest'ultima allocazione è consigliabile quando c'è poco spazio sul disco fisso (vecchi pc e portatili). Windows è installato in una partizione separata da 3-5 GB senza programmi aggiuntivi. Tutti i programmi e i file di dati sono nella seconda e più grande partizione. Chi avesse un po' più di spazio su disco può dedicarlo a questa partizione o dividere i programmi Windows dai dati con 2 partizioni.

Generale

Ci sono molti modi di partizionare il disco fisso. Questi esempi dovrebbero bastare per iniziare. Ha anche senso mettere un secondo disco per migliorare le prestazioni, la comodità e la sicurezza. In questo caso MS Windows (TM) va sempre installato sul primo disco fisso/partizione!
