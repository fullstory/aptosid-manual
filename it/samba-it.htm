Configurare sidux per usare condivisioni SAMBA (Windows) di macchine remote

Eseguite tutti i comandi come root (in un Terminal o Konsole) Mettete gli URL in Konqueror (eseguite Konqueror come utente normale).

In questa pagina, dove trovate:

- server = nome_server o indirizzo IP della macchina windows
- share = nome della cartella condivisa

In KDE - Konqueror scrivete nella barra degli indirizzi URL smb://server o lo URL completo smb://server/share

In una konsole potete vedere le condivisioni localizzate su un dato server con:

smbclient -L nome_server

Per montare una condivisione in una cartella (con pieno accesso per TUTTI gli utenti) ricordate che il punto di mount, cioe' la cartella dove verra' visualizzata la condivisione, deve esistere. Altrimenti dovete prima creala (il nome e' arbitrari, in questo caso e' server_share):

mkdir -p /mnt/server_share

Ora montate la condivisione, un filesystem remoto VFAT:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777 //server/share /mnt/server_share

o un filesystem remoto NTFS :

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777,lfs //server/share /mnt/server_share

Per chiudere la connessione alla condivisione, smontatela con:

umount /mnt/server_share

Se volete mettere una voce in /etc/fstab per facilitare la procedura inserite la seguete stringa nel file suddetto:

//server/share  /mnt/server_share       cifs  defaults,username=your_username,password=**********,file_mode=0777,dir_mode=0777   0   0

Come impostare sidux come server Samba

su un live-cd sidux :

Siccome samba non e' incluso nel sidux live-cd, se vi serve dovete installarlo con i seguenti comandi:

su
apt-get update
apt-get install samba
su una sidux installata :

siccome samba non e' preinstallato, anche in questo caso se serve dovrete installarlo con i seguenti comandi:

sux
apt-get update
apt-get install samba

Su una sidux installata e' necessario sistemare la configurazione di Samba. Ecco un semplice esempio. Se volete saperne di piu' sull'uso di Samba e sulle impostazioni di un Server Samba Linux e' consigliabile leggere la Documentazione Samba ..

Per sistemare la configurazione di samba dovete fare come segue:

Aprite il file /etc/samba/smb.conf in un elaboratore di testo (come kedit o kwrite) e scrivete questo:

# Variazioni globali- Proposta 'tutto semplice' per quanto possibile
#no password, comportamento come Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Condivisione senza permessi di scrittura -importante se i filesystem condivisi sono NTFS!

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/hda1 # <-- correggete 'hda1' mettendo la vostra partizione
public = yes

# Condivisione di partizione con permessi di scrittura - la partizione deve essere montata
# in modalita' scrivibile - ha senso con FAT32, ad esempio.

[DATA]
comment = Data Partition (first extended Partition)
browseable = yes
writable = yes
path = /media/hda5
public = yes
Controllare le condivisioni in samba

Per impostare le condivisioni in samba lasciando perdere le questioni sulla sicurezza eseguite i seguenti comandi (esempio per una impostazione di rete LAN):

Impostate le cartelle e i file contenuti, con almeno i seguenti permessi: -rwxr-xr-x:

ls -la percorso_verso_/nome_cartella_condivisa/*

Se no eseguite:

chmod -R 755 percorso_verso_/nome_cartella_condivisa

Se la volete in scrittura:

chmod -R 777 percorso_verso_/nome_cartella_condivisa

Un modo per assicurarvi che la vostra condivisine stia funzionando: (non dimenticatevi di avviare il server):

smbclient -L localhost

Dovreste vedere qualcosa tipo:

 smbclient -L localhost
Password:
Domain=[HOME] OS=[Unix] Server=[Samba 3.0.26a]

        Sharename       Type      Comment
        ---------       ----      -------
        IPC$            IPC       IPC Service (3.0.26a)
        MaShare         Disk      comment
        print$          Disk      Printer Drivers
Domain=[MSHOME] OS=[Unix] Server=[Samba 3.0.26a]

Se non avevate impostata una password, premete semplicemente INVIO

Non dimenticate di salvare. Potete avviare/fermare samba con:

/etc/init.d/samba start

e:

/etc/init.d/samba stop

potete anche avviare/fermare samba automaticamente all'avvio. Eseguite questa chiamata:

update.rc.d samba defaults

Ora samba si avviera' assieme al sistema e si fermera' quando lo spegnerete.
