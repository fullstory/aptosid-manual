Una piccola guida a ricette per APT

Cosa significa APT?

APT e' una abbreviazione per Advanced Packaging Tool (strumento avanzato di pacchettizzazione) ed e' una collezione di programmi e script che aiutano sia l'amministratore del sistema (nel vostro caso, root) nell'installazione e la gestione dei file .deb sia, al contempo, il sistema stesso a sapere cosa c'e' installato.

Sources List o fonti software

Il sistema "APT" necessita di un file di configurazione che contiene le informazioni riguardo alle locazioni dei pacchetti installabili e aggiornabili, e ci si riferisce comunemente a lui come sources.list.

Sources are contained in a directory or folder named:
/etc/apt/sources.list.d/

Dentro la cartella ci sono due file chiamati:
/etc/apt/sources.list.d/debian.list e
/etc/apt/sources.list.d/sidux.list

Sono due perche' si beneficia cosi' di un facile (automatizzato) cambio dei mirror, e rende piu' sicuro rimpiazzare le liste.

Potete anche aggiungere il vostro personale file /etc/apt/sources.list.d/*.list

Tutte le ISO sidux usano in via predefinita le seguenti fonti software :

# Free University Berlin/ spline (Student Project LInux NEtwork), Germany
deb ftp://ftp.spline.de/pub/sidux/debian/ sid main contrib non-free fix.main fix.contrib fix.non-free
#deb-src ftp://ftp.spline.de/pub/sidux/debian/ sid main contrib non-free fix.main fix.contrib fix.non-free

Ulteriori indirizzi per programmi non liberi si possono trovare in queste liste sempre aggiornate sidux.list and debian.list Al giorno 28/02/2008 0800 hrs UTC esse sono:

NOTATE: In questo esempio ftp.us indica l'uso di depositi software USA.Potete modificare questo file, come root, per usare i depositi fisicamente piu' vicini a voi, semplicemente cambiando il paese come ad esempio, da ftp.us a ftp.it, o ftp.uk. La maggior parte dei paesi hanno dei mirror locali in modo da preservare la banda e migliorare la velocita'.

Per riuscire ad gestire le informazioni aggiornate sui pacchetti, APT usa un database. Esso contiene i pacchetti ma anche quali altri pacchetti sono necessati per un dato pacchetto per funzionare (chiamate dipendenze). Il programma apt-get usa questo database mentre installla i pacchetti sche scegliere per risolvere le dipendenze e garantire che quel dato pacchetto possa poi lavorare. L'aggiornamento del database viene esegito con il comndo apt-get update.

# apt-get update
	(which returns )

Get:2 http://sidux.com sid Release.gpg [189B]
Get:3 http://sidux.com sid Release.gpg [189B]
Get:4 http://ftp.de.debian.org unstable Release.gpg [189B]
Get:5 http://sidux.com sid Release [34.1kB]
Get:6 http://ftp.de.debian.org unstable Release [79.6kB]

Installare un nuovo pacchetto

Presumendo che il database di Apt sia aggiornato e il nome del pacchetto sia conosciuto, allora il seguente comando installa un pacchetto chiamato qemu, per esempio, e tutte le sue dipendenze: (piu' avanti vedrete come cercare e trovare pacchetti.)

Cancellare un pacchetto

Analogamente non destera' sorpresa che il seguente comando rimuova il pacchetto (gaim, in questo caso); comunque esso non rimuovera' le dipendenze eventualmente installate col programma. Questo lavoro verra' spiegato dopo in deb-orphan 

# apt-get remove esempio
apt-get remove gaim
Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim gaim-irchelper
0 upgraded, 0 newly installed, 2 to remove and 310 not upgraded.
Need to get 0B of archives.
After unpacking 4743kB disk space will be freed.
Do you want to continue [Y/n]?
(Reading database ... 174409 files and directories currently installed.)
Removing gaim-irchelper ...
Removing gaim ...

In questo caso i file di configurazione del pacchetto 'gaim' (fondamentalmente come il suo setup) non verra' cancellato dal sistema. Potrete riusarli nel caso decideste di reistallare lo stesso pacchetto e trovaste comodo usare le medesime impostazioni

Se invece voleste cancellare anche i file di configurazione, lanciate questo comando:

# apt-get remove --purge gaim
Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim*
0 upgraded, 0 newly installed, 1 to remove and 309 not upgraded.
Need to get 0B of archives.
After unpacking 4678kB disk space will be freed.
Do you want to continue [Y/n]? Y
(Reading database ... 174405 files and directories currently installed.)
Removing gaim ...
Purging configuration files for gaim ...

In questo caso i file nella cartella /etc/gaim vengono cancellati assieme anche a tutti i file di configurazione vengono puliti/cancellati. Se comunque una applicazione viene sospesa o ci sono file non creati durante la installazione del pacchetto ceh stiamo disintallando, questi non verranno cancellati.

Tornare alla versione precedente o mettere in attesa un pacchetto

A volte e' necessario ritornare ad una versione precedente di un pacchetto, o metterlo in sospensione a causa magari di un difetto o un problema insorto con l'ultima versione installata.

Mettere in sospensione un paccheto

echo package hold|dpkg --set-selections

Togliere dalla sospensione un pacchetto

echo package install|dpkg --set-selections

Elencare i pacchetti in sospensione:

dpkg --get-selections | grep hold

Tornare ad una versione precedente

I passi mostrati usano kmahjongg come esempio:

  1. abilitate/decommentate le fonti 'testing' nel file /etc/apt/sources.list 
    (in pratica cancellate il simbolo #) cosicche' vengano lette 
    deb http://ftp.nl.debian.org/debian/ testing main contrib non-free
  2. apt-get update
  3. apt-get install -t testing kmahjongg
  4. Mettete in sopensione il pacchetto appena installatocon:
    echo kmahjongg hold|dpkg --set-selections
  5. commentate (aggiungete # davanti alla riga) le fonti 'testing' nel file /etc/apt/sources.list 
  6. apt-get update

Quando volete avere l'ultima versione del pacchetto, fate semplicemente questo:

echo kmahjongg install|dpkg --set-selections
apt-get update
apt-get install kmahjongg

Aggiornare un intero sistema installato - dist-upgrade

L'aggiornamento dell'intero sistema viene eseguito mediante il comando dist-upgrade. Dovreste sempre controllare prima la voce 'Current Warning' sul sito principale di sidux, e controllare i 'segnali di pericolo' ivi esposti riguardo ai pacchetti che il vostro sistema vorrebbe aggiornare ad una versine superiore (dist-upgrade, appunto). Se volete tenere un certo pacchetto o metterlo in sospensione - hold - fate riferimento a Tornare alla versione precedente o mettere in attesa un pacchetto

Non e' raccomandato fare solo un 'upgrade' di 'debian sid'.

La frequenza del dist-upgrade

fate dist-upgrade abitudinalmente quando potete, idealmente ogni una o due settimane, almeno una volta al mese per essere al sicuro. Un dist-upgrade initiated solo ogni 2 o tre mesi dovrebbe essere considerato fuori limite riguardo alla sicurezza. Per pacchetti non standard o autcompilati dovrete prestare molta piu' attenzione con l'innalzamento di versione perche' possono corrompere il vostro sistema in ogni momento a causa delle incompatibilita'.

Just downloading

apt-get dist-upgrade -d

Ragioni per non utilizzare altro che apt-get per il dist-upgrade

I gestori di pacchetti come adept, aptitude, synaptic and kpackage non sono in gradi di tenere conto di un grande ammontare di variazioni che avvengono in Sid (cambi di dipendenze, di nomi, degli script dei manutentori, ...). Non per colpa degli sviluppatori o degli strumenti di gestione dei pacchetti; gli sviluppatori scrivono strumenti eccellenti e favolosi per la branca stabile di Debian ma semplicemente non utilitzzabilil per le speciali esigenze della Debian Sid.

Usate dunque cio' che volete per cercare un pacchetto ma 'attaccatevi' ad apt-get per realmente installare/rimuovere/fare dist-upgrade

I gestori di pacchetti come adept, aptitude, synaptic and kpackage sono come minimo non-teterministici (per una selezione complessa di pacchetti), miscelate questo fatto con una situazione in rapido cambiamento come la dEbian Sid e anche peggio, con l'uso di un deposit sofware esterni di discutibile qualita' (che noi non raccomandiamo ma che sono una realta' sui sistemi dei vosti utenti) e vedrete che siete sull'ordo del disastro. L'altra cosa da notare e' che questi tipi di gestori di pacchetti in modalita' grafica girano in init 5, e/o in X, e facendo un dist-upgrade in init 5 e/o X, (o anche un 'upgrade' che non e' racomandato), finirete con il danneggiare il vostro sistema oltre l apossibilita' di riparazione, forse non o ggi o domani, ma sarete in tempo a vederlo.

apt-get invece fa strettamente cio' che gli si chiede, se c'e' un qualsiasi danno potete definirlo con esattezza e fare il debug / o riparare la causa, se apt-get volesse rimuovere mezzo sistema (a causa di transizioni di librerie) e' il momento per l'amministratore (cioe' voi) di dare una occhiata severa al tutto.

E' questa la ragione per cui le derivate debian usano apt-get e non gli altri gestori di pacchetti.

Cercare un pacchetto con kio-apt

Usando kio-slave, Konqueror fornische un modo esente da pericoli di usare un GUI per cercare pacchetti in APT. Esso e' solo un sistema di ricerca a interfaccia grafica (GUI) per i pacchetti, senza nessuna opzione per installarli o aggiornarli.

E' una buona alternativa alle funzioni di ricerca degli altri gestori di pacchetti come quelle incluse in adept, aptitude, synaptic and kpackage, cosi' non srete tentati di fare un dist-upgrade in X o installare pacchetti che non sono adatti per la Debian sid.

Potete cercare nella base dati dei pacchetti sia all'interno di APT, e/o con il 'package search' sul sito web di Debian, Esso fa una buon sommario delle pagine con le informazioni da apt-cache policy e apt-cache show.

Esso fornisce:

  • apt:/search? expression: cerca nella base dati dei pacchetti con il parametro 'expression'. Come apt-cache
  • apt:/show? nome_pacchetto: mostra informazini dettagliate riguardo a 'nome_pacchetto'. Come apt-cache show. Visualizza anche la linea politica del pacchetto.
  • apt:/policy? nome_pacchetto: Mostra la linea politica del pacchetto 'nome-pacchetto'. Come apt-cache policy.
  • apt:/list? package: elenca i file contenuti in 'nome-pacchetto'.

Installazione

sux
apt-get update && apt-get install kio-apt

Poi aprite konqueror e digitate in:

apt:/

e premete invio. Vedrete la descrizione di tutte le possibili opzioni che includono la ricerca in forma compatta o estesa..
