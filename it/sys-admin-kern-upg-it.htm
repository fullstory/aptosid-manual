Installare nuovi kernel

I kernel sidux sono in apt. Per favore scegliete il deposito mirror di sidux piu' vicino a voi e modificate conseguentemente le vostre fonti software:

I kernel di sidux son disponibili nelle seguenti versioni:

  • sidux-686 sidux-686 - kernel per la famiglia di processori i686 tipo singolo/doppio core o piu' CPU
  • sidux-amd64 kernel sidux per 64 bit
4 passi per l'installazione manuale del kernel usando la konsole:

1. In una konsole autenticatevi come root, poi:

apt-get update

2. Installare l'ultima versione del kernel:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. L'aggiornametno del kernel-update e' fatto. Se vi servono aggiungete gli ultimi driver grafici delle schede ATI, NVIDIA o INTEL. Prima aggiornate lo script:

sgfxi -U

4.Ora riavviate il computer avviando cosi' il nuovo kernel (dovrebbe farlo come opzione predefinita).

Per aggiornare i driver 3D lanciate:

sgfxi

In alternativa, al primo avvio del nuovo kernel andate direttametne al runlevel 3 (si puo' fare aggiungendo 3 alla stringa di boot di Grub) e poi avviate sgfxi.

Trovate qui informazioni piu' dettagliate su sgfxi

Se il nuovo kernel vi desse dei problemi, potete riavviare il pc scegliendo nel grub il vecchio kernel gia' funzionante.

Se non voleste aggiornare il kernel ad ogni lancio dist-upgrade, mentre viene disponibile, mettete i meta in 'hold' con siduxcc o smxi.

Moduli

Per trovare i moduli che vi servono, il seguente comando vi fornisce la lista del moduli correntemente disponibili; copiate questa stringa nella vostra console/terminale :

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Per avere una completa descrizione di ogni modulo, copiate la seguente stringa nella vostra console/terminale:

apt-cache search 2.6.*.slh.*-sidux

Per installare il modulo che vi serve (per esempio virtual-ose, e qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)

Per controllare i moduli caricati nel kernel:

ls /sys/module/
or
cat /proc/modules

IMPORTANTE: Se vi servono certi moduli driver, es. per rete wireless, dovete installare l'appropriato pacchetto .deb. Ad esempio, per l'adattatore di rete Agere et131x: 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb e' usato solo se avete gia' localmente i pacchetti .deb, e se questi sono gia' nella cartella apt-cache; alla fine apt-get e' l'opzione migliore.

Leggete anche mettersi on line e ndiswrapper e la parte per il WIFI su questo manuale

Altri metodi per installare il kernel

smxi

Lo smxi script offre un metodo integrato per instalalre i kernels, driver e diversi altri script di amministrazione del sistema:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Potete anche installare i kernel mediante il centro di controllo sidux a interfaccia grafica siduxcc: che nel menu' e' localizzato in KDE-Start-Menu>Sistema>sidux Control Centre; siduxcc offre anche parecchie altre applicazioni a interfaccia grafica di amministrazione del sistema

Rimozione di vecchi kernel

Dopo l'installazione con successo di un nuovo kernel, i vecchi possono essere rimossi, comunque e' raccomandato tenerli per alcuni giorni nel caso si evidenzino problemi con il nuovo per cui potrete riavviare con uno dei vecchi kernel presenti nella lista di avvio di grub

Per rimuovere i vecchi kernel potete usare lo script kernel-remover che si trova nel pacchetto sidux-kernelhacking. Questo script deve essere installato e vi permette di scegliere il kernel da rimuovere.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

In alternativa potete usare siduxcc, il quale usa lo script kernel-remover.
