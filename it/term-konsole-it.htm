Definizione di terminale/konsole

Un terminale, chiamato anche console e "konsole" in KDE, è un programma che rende possibile interagire direttamente con il S.O. Linux immettendo vari comandi che sono poi eseguiti immediatamente. Vi si fa spesso riferimento chiamandolo "shell" o "command line" - "linea di comando", il terminale è uno strumento davvero potente di cui vale davvero la pena di ottenere alcune conoscenze di base sul suo uso.

In sidux si può trovare il terminale/konsole accanto al K-menu, sotto il simbolo di una schermo nero di un PC. A seconda del tema di icone che si è scelto esso può contenere l'immagine di una shell. Si troverà la stessa icona nel K-menu sotto al voce "Sistema".

Quando si apre una finestra di terminale verrà presentata una riga (chiamata "prompt") che avrà il formato seguente:

username@hostname:~$

Si dovrebbe riconoscere il nome utente (username) come il nome con cui si è entrati nel sistema. Il simbolo ~ (tilde) indica che si è nella cartella home e $ indica che si è nel sistema con i privilegi di utente normale. Alla fine si vedrà il cursore. Questa è la linea di comando dove immettere i comandi che si vogliono eseguire.

Molti comandi devono essere eseguiti con i privilegi di root (amministratore). Per ottenerli si deve digitare: sux al prompt e premere invio. Verrà chiesta la password di root. Digitarla e premere ancora invio (si noterà che mentre si digita questa password nulla viene mostrato a schermo).

Se la password è corretta il prompt cambierà così:

root@hostname:/home/nomeutente#

ATTENZIONE: Mentre si è nel sistema come root, esso non si opporrà nel fare cose potenzialmente pericolose come cancellare file importanti, ecc. . Si deve essere assolutamente sicuri riguardo alle cose che si stanno facendo perché è davvero possibile danneggiare seriamente il sistema.

Notare che il $ è cambiato in # (hash). In un terminale/konsole il simbolo # indica sempre che si è nel sistema con i privilegi di root. In questo manuale vengono ritenute implicite le cose da fare che precedono i simboli $ o # negli esempi. Così un comando come:

# apt-get install qualcosa

significa: aprire un terminale, diventare root (con sux) e immettere il comando al prompt #. (non digitare il simbolo #)

Qualche volta la konsole e/o il terminale in esecuzione può corrompersi, allora digitare:

reset

e premere invio.

Se quanto visualizzato da una konsole e/o da un terminale sembra distorto si può intervenire premendo ctrl+l, che ridisegna la finestra del terminale. Questa distorsione si presenta per lo più mentre si lavora con programmi che usano l'interfaccia ncurses, come irssi

Occasionalmente una konsole e/o un terminale può sembrare bloccato, tuttavia non lo è, e qualsiasi cosa si digita verrà poi eseguito in sequenza. Questa situazione può essere causata dalla pressione accidentale di ctrl+s. In questo caso provare ctrl+q per sbloccare il terminale.

Il comando sux

Numerosi comandi necessitano di essere eseguiti con i privilegi di root. Per ottenerli digitare:

sux

L'usuale comando per diventare root è "su" ma se si utilizza sux, si possono avviare applicazioni con interfaccia grafica (applicazioni GUI/X11) dalla linea di comando, cioè permette a root di lanciare applicazioni grafiche, poiché sux è un involucro (wrapper) attorno al comando standard "su" che trasferisce le credenziali X all'utente cui punta.

Esempi di attività con applicazioni X11 avviate via sux possono essere: usare un elaboratore di testo come kwrite o kate per modificare file di root; fare partizionamenti con gparted o lanciare un file manager a interfaccia grafica (Xapp) come dolphin o thunar.

Alcune applicazioni KDE richiedono dbus-launch prima del nome dell'applicazione:

dbus-launch <applicazione>
Opzioni da tastiera in KDE

Per avviare krunner in KDE:

Alt+F2

oppure tasto destro nel desktop e scegliere:

Esegui comando

quindi:

kdesu <applicazione>
Opzioni da tastiera in Xfce

Per avviare Esegui Comando in Xfce:

Alt+F2

oppure tasto destro nel desktop e scegliere:

Esegui comando

quindi:

gksu <applicazione>
Altre opzioni dei Desktop Window Manager

Un'altra opzione da tastiera generica e comune a tutti i maggiori Desktop Manager è:

Alt+F2

quindi:

su-to-root -X -c <applicazione>

Tutte le opzioni da tastiera di sui sopra possono essere avviate nel terminale.

sudo non è supportato

sudo non è abilitato per default nella installazione su hard disk. È disponibile per l'uso mentre si è avviato da livecd dal momento che non è definita nessuna password di root. La ragione di questa scelta è che se qualcuno ottiene le password degli utenti, non ottiene immediatamente privilegi da super utente e fa cambiamenti potenzialmente dannosi al sistema.

Un altro problema di sudo è che induce l'utente ad avviare una applicazione di root con la configurazione da utente che può causare cambiamenti o sovrascrittura di permessi e questo potrebbe rendere un'applicazione inutilizzabile dall'utente stesso. Utilizzare sux, kdesu, gksu oppure su-to-root -X -c come raccomandato!

Essere root

ATTENZIONE: Mentre si è nel sistema come root, esso non si opporrà nel fare cose potenzialmente pericolose come cancellare file importanti, ecc. . Si deve essere assolutamente sicuri riguardo alle cose che si stanno facendo perché è davvero possibile danneggiare seriamente il sistema.

In nessuna circostanza si dovrebbe essere come root in una console/terminale per lanciare applicazioni che un utente normale usa per il suo lavoro quotidiano, coma spedire posta, creare fogli elettronici o navigare in internet e cosi via.

Aiuto nella Linea di Comando

Ebbene, esiste. Molti comandi e programmi Linux sono forniti con il proprio manuale chiamato "man page" o "manual page" accessibile dalla linea di comando. La sintassi è:

$ man nome-comando

oppure

$ man -k <keyword>

Questo porta in console la "manual-page" per quel comando. Navigare su e giù con i tasti del cursore. Ad esempio provare:

$ man apt-get

Per uscire dalle man-pages digitare semplicemente q

Un'altra utile facilitazione è il comando "apropos". Fondamentalmente abilita a cercare le man-pages per un comando se ad esempio non se ne ricorda la completa sintassi. Per esempio si può provare:

$ apropos apt-

Viene visualizzata la lista di tutti i comandi per il gestore dei pacchetti "apt". L'utilità "apropos" è uno strumento abbastanza potente, ma la sua descrizione in dettaglio è al di là dello scopo di questo manuale. Per i dettagli sul suo uso leggere la sua man-page.

Lo script e come usarlo

Uno script di shell è un modo conveniente per raggruppare in un file molti comandi. Immettendo il nome del file script ogni comando verrà eseguito in sequenza. sidux è fornito di parecchi ed utili script per facilitare la vita degli utenti.

Se lo shell-script si trova nella cartella corrente lanciarlo con:

./nome_dello_shell-script

Alcuni script richiedono un accesso di root (sux) in un terminale e altri no, dipende interamente da ciò che fa lo script.

Installazione di Script e procedura di esecuzione

Usare wget per scaricare il file script mettendolo dove è stato raccomandato (per esempio può richiedere di essre messo in /usr/local/bin) ; si può usare la funzionalità copia/incolla via mouse del nome del file direttamente nella finestra della konsole, dopo essersi autenticati con sux

Esempio dell'uso di wget come utente root (sux)
sux
cd /usr/local/bin
wget nome-dello-script

Ora occorre rendere eseguibile il file

chmod +x  nome-dello-script

Si può usare anche un browser per scaricare uno script file e metterlo dove indicato, comunque si dovrà sempre renderlo eseguibile.

Esempio dell'uso di wget come utente normale

Per scaricare un file nella directory $HOME come utente normale '$':

$ wget http://bluewater.sidux.com/shell-script-test/test-script.sh

$ chmod +x test-script.sh

Per lanciare uno script, aprire un terminale/konsole, scrivere il nome dello script appena scaricato (preceduto da ./):

$ ./test-script.sh

Si dovrebbe vedere a schermo la seguente frase:

Congratulations user
You successfully downloaded and executed a bash script!
