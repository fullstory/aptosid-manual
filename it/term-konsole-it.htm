Definizione di terminale/konsole

Un terminale, chiamato anche console e 'konsole' in KDE, e' un programma che rende possibile interagire direttamente con il S.O. Linux immettendo vari comandi che sono poi eseguit immediatamente. Vi si fa spesso riferimento chiamandolo 'shell' o 'command line' - 'linea di comando', il terminale e' uno strumento davvero potente di cui vale davvero la pena di ottenere alcune conoscenze di base sul suo uso.

In sidux potete trovare il terminale/konsole accanto al K-menu, sotto il simbolo di una schermo nero di un PC. A seconda del tema di icone che avete scelto esso puo' contenere l'immagine di una shell. Troverete la stessa icona nel K-menu sotto al voce "Sistema".

Quando aprite una finestra di terminale vi verra' presentato una riga (chiamata 'prompt') che avra' il formato seguente:

username@hostname:~$

Dovreste riconoscere il nome utente (username) come il nome con cui siete entrati nel sistema. Il simbolo ~ (tilde) indica che siete nella vostra cartella home e $ indica che siete nel sistema con i privilegi di utente normale. Alla fine vedrete il cursore. Questa e' la vostra linea di comando dove immetterete i comando che volete eseguire.

Molti comandi devono essere eseguiti con i privilegi di root (amministratore). Per ottenerli dovete digitare: sux al prompt e premere invio. Vi sara' chiesta la password di root. Digitatele e premete ancora invio (noterete che mentre voi digitate questa password nulla viene mostrato a schermo).

Se la password e' corretta il prompt cambiera' cosi':

root@hostname:/home/username#

ATTENZIONE: Mentre siete nel sistema come root, esso non vi fermera' nel fare cose potenzialmente pericolose come cancellare file importanti ecc. Dovete essere assolutamente sicuri riguardo alle cose che state facendo perche' e' davvero possibile danneggiare seriamente il vostro sistema.

Notate che il $ e' cambiato in # (hash). In un terminale/konsole il simbolo # indica sempre che siete nel sistema con i privilegi di root. In questo manuale vengono ritenute implicite le cose da fare che precedono i simboli $ o # negli esempi. Cosi' un comando come:

# apt-get install something

significa: aprite un terminale, diventate root (con sux) e immettete il comando al prompt #. (non digitate il simbolo #)

Qualche volta la konsole e/o il terminale in esecuzione puo' corrompersi, allora digitate:

reset

e premete invio.

Se quanto emesso da una konsole e/o da un terminale sembra distorto potete intervenire premendo ctrl+l, che ridisegna la finestra del terminale. Questa distorsione si presenta per lo piu' mentre si lavora con programmi che usano l'interfaccia ncurses, comes irssi

Occasionalmente una konsole e/o un terminale puo' sembrare bloccato, tuttavia non lo e', e qualsiasi cosa digitate verra' poi eseguito in sequenza. Questa situazione puo' essere causata dalla pressione accidentale di ctrl+s. In questo caso provate ctrl+q per sbloccare il terminale.

Il comando sux

Numerosi comandi necessitano di essere eseguiti con i privilegi di root. Per ottenerli digitate:

sux

L'usuale comando per diventare root e' 'su' ma se userete invece sux, potrete lanciare applicazioni con interfaccia grafica (applicazioni GUI/X11) dalla linea di comando, cioe' permette a root di lanciare applicazioni grafiche, poiche' sux e' un involucro (wrapper) attorno a comando standard 'su' che trasferisce le vostre credenziali X all'utente cui punta.

Esempi di attivita' con applicazioni X11 lanciate via sux possono essere: usare un elaboratore di testo come kwrite o kate per modificare file di root; fare partizionamenti con gparted o lanciare un file manager a interfaccia grafica (Xapp) come konqueror. Potete anche alterare dei file di root cliccando il file col tasto destro del mouse e scegliendo 'modifica come root' e immettendo la password (di root) che naturalmente vi verra' chiesta in quanto verra' richiamato kdesu.

Tutto questo significa che, diversamente da 'sudo', non puo' semplicemente venirsene chiccessia sul vostro sistema a digitare 'sudo' e fare cambiamenti potenzialmente dannosi.

ATTENZIONE: Mentre siete nel sistema come root, esso non vi fermera' nel fare cose potenzialmente pericolose come cancellare file importanti ecc. Dovete essere assolutamente sicuri riguardo alle cose che state facendo perche' e' davvero possibile danneggiare seriamente il vostro sistema.

In nessuna circostanza dovreste essere come root in una console/terminale per lanciare applicazioni che un utente normale usa per il suo lavoro quotidiano, coma spedire posta, creare fogli elettronici o navigare l'internet e cosi via.

Aiuto nella Linea di Comando

Ebbene, c'e'. Molti comandi e programmi Linux sono forniti con il proprio manuale chiamato "man page" o "manual page" accessibile dalla linea di comando. La sintassi e':

$ man nome-comando

Questo porta in console la 'manual-page' per quel comando. Navigate su e giu' con i tasti del cursore. Ad esempio provate:

$ man apt-get

Per uscire dalle man-pages digitate semplicemente q

Un'altra utile facilitazione e' il comando "apropos" . Fondamentalmente vi abilita a cercare le man-pages per un comando se ad esempio non ne ricordate la completa sintassi. Per esempio potete provare:

$ apropos apt-

Vi si presentera' la lista di tutti i comandi per il gestore dei pacchetti 'apt'. La utilita' 'apropos' e' uno strumento abbastanza potente, ma la sua descrizione in dettaglio e' al di la' dello scopo di questo manuale. Per i dettagli sul suo uso leggete la sua man-page.

Lo script e come usarlo

Uno script di shell e' un modo conveniente per raggruppare in un file molti comandi. Immettendo il nome del file script ogni comando verra' eseguito in sequenza. Sidux e' fornito con parecchi e utili script per facilitare la vita degli utenti.

Se lo shell-script si trova nella cartella corrente lanciatelo con

./nome_dello_shell-script

Alcuni script richiedono un accesso di root (sux) in un terminale e altri no, dipende interamente da cio' che fa lo script.

Installazione di Script e procedura di esecuzione

Usate wget per scaricare il file script mettendolo dove e' stato raccomandato che sia messo (per esempio puo' richiedere che lo mettiate in /usr/local/bin) ; potete usare la funzionalita' copia/incolla via mouse del nome del file direttamente nella finestra della konsole, dopo esservisi autenticati con sux

Esempio dell'uso di wget come utente root (sux) 
sux
cd /usr/local/bin
wget nome-dello-script

Ora occorre rendere eseguibile il file

chmod +x  nome-dello-script

Potete usare anche un browser per scaricare uno script file, poi mettetelo dove e' indicato che sia messo ma comunque dovrete sempre renderlo eseguibile.

Esempio dell'uso di wget come utente normale

Per scaricare un file nella vostra cartella $HOME come utente normale '$':

$ wget http://bluewater.sidux.com/shell-script-test/test-script.sh

$ chmod +x test-script.sh

Per lanciare uno script, aprite un terminale/konsole, scrivete il nome dello script appena scaricato (preceduto da ./) :

$ ./test-script.sh

Dovreste vedere a schermo la seguente frase:

Congratulations user
You successfully downloaded and executed a bash script!
