aptosid - Guida di avvio rapido

MOLTO IMPORTANTE: aptosid, come liveCD Linux, si basa su una tecnologia ad alta compressione, e, per questo motivo, è richiesta particolare attenzione per masterizzare l'immagine ISO. Utilizzare solo supporti CD (o DVD+RW) di alta qualità e masterizzare in modalità DAO (disk-at-once) non più velocemente di 8x.

aptosid cerca di essere al 100% compatibile con Debian sid, tuttavia di volta in volta aptosid può fornire delle temporanee correzioni. Il repository di apt di aptosid contiene i propri pacchetti specifici che includono anche il kenel, gli script, le utilità e la documentazione.

Debian sid / stabilità della unstable

sid è una 'etichetta' per i depositi 'unstable' di Debian. Debian sid è un deposito frequentemente aggiornato, rapidamente in sincronia con le ultime e più grandi versioni 'upstream' del software mantenuto. A causa dei frequenti aggiornamenti vi si trovano pacchetti meno complessivamente testati per il breve periodo tra la versione 'upstream' e quella Debian ufficiale dei pacchetti.

Kernel aptosid

Il kernel è ottimizzato da aptosid per eliminare problemi, aggiungere nuove funzionalità oppure ottenere migliori prestazioni, aumentare la stabilità ed è messo a punto a partire dall'ultimo kernel rilasciato da http://www.kernel.org/ .

Il kernel ha un sito mirror qui: Aggiornare il kernel

Gestione dei pacchetti

aptosid è conforme alla pacchettizzazione Debian ed utilizza apt e dpkg per la gestione del software tramite i repository debian ed altri listati in /etc/sources.list.d/*

Debian sid ha oltre 20.000 pacchetti, si dovrebbe trovare quello che si vuole usando Ricerca pacchetti con apt-cache oppure con l'applicazione Debian Package Search GUI.

Per installare il pacchetto che serve scrivere in console: apt-get install <nome pacchetto> Vedi 'Installare un nuovo pacchetto'

I repository di Debian sid possono essere aggiornati fino a 4 volte al giorno quindi avviare apt-get update è fondamentale prima di installare ogni pacchetto nuovo o prima di avviare apt-get dist-upgrade per mantenere aggiornate le liste dei pacchetti sul server.

Utilizzare i repository di altre distribuzioni basate su Debian, sorgenti ed RPM

Installazioni da codice sorgente non sono supportate. Se davvero avete bisogno di compilare la vostra applicazione, fatelo come utente normale e mettete il programma nella vostra home e non installatelo nel sistema. L'uso di checkinstall e la conversione da pacchetti RPM con alien (e altri convertitori simili) a .deb non sono supportati.

Altre famose, (e meno famose), distribuzioni basate su Debian che ripacchettano i pacchetti Debian per i propri repository, spesso usano differenti locazioni di file per varie applicazioni e la cosa può causare instabilità nel sistema e qualche pacchetto potrebbe non installarsi a causa di dipendenze non risolvibili derivate da schemi differenti di assegnazione dei nomi ai pacchetti o per strani numeri di versione. Per esempio, una differente versione di glibc può impedire l'avvio di una applicazione.

Usare i repository Debian per installare i pacchetti software necessari, dal momento che altri repository molto probabilmente non possono essere supportati.

Aggiornare di versione l'intero sistema - dist-upgrade

apt-get dist-upgrade è la modalità per aggiornare un intero sistema aptosid. Non è supportato nessun sistema ad interfaccia grafica per questo lavoro. Aggiornare un sistema installato - dist-upgrade

Un dist-upgrade funziona solo al di fuori dell'interfaccia grafica X. Lanciando "init 3" da una console nel vostro window manager (KDE, XFCE, ecc.) o in un terminale virtuale raggiunto con (ctrl+alt+f1, ctrl+alt+f2, ecc.) si uscirà da X e si potrà lanciare con sicurezza il dist-upgrade.

Configurazione della rete

Ceni è uno strumento per configurare rapidamente e senza problemi la scheda di rete ethernet o la scheda wireless. La funzione wireless può fare la scansione delle rete, usare wep e wpa per la crittografia e usare i wireless-tools o wpa_supplicant per la configurazione del wireless. La configurazione della scheda ethernet è immediata usando dhcp (assegnazione automatica di indirizzo ip) oppure si può impostare manualmente tutto, dalle netmask ai nameserver.

Ceni si avvia con il comando Ceni o ceni . Se non è installato, un 'apt-get install ceni' lo installerà.
