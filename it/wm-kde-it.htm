Alcune applicazioni utili di KDE non preinstallati in sidux KDE Lite

Si potrebbe aver bisogno di abilitare non-free in /etc/apt/ debian.sources.list

konq-plugins - plugin per Konqueror

Questo pacchetto contiene diversi plugin utili per Konqueror, il gestore file, web browser e visualizzatore di documenti di KDE. Molti di questi plugin compariranno nel menu Strumenti di Konqueror.

Utilità per il web browsing compreso traduzione pagine web, archiviazione pagine web, aggiornamento automatico, analisi strutturale di HTML e CSS, barra di ricerca, barra laterale per le notizie, accesso veloce alle opzioni più comuni, preferiti, monitor di crash, un indicatore di disponibilità microformato, una barra laterale per i preferiti del.icio.us, integrazione con il lettore di feed RSS aKregator.

Utilità per la lettura delle directory compreso un filtro, creazione di galleria di immagini, compressione ed estrazione, sposta/copia veloce, barra laterale per lettore audio, barra laterale per informazioni sui file, un supporto per le directory media, un visualizzatore grafico dell'uso del disco e conversioni e trasformazioni di immagini.

unrar - Estrazione di archivi .rar (versione non-free)

Estrazione di archivi .rar (versione non-free). Unrar può estrarre file da archivi .rar. Se si vuole creare un archivio .rar installare il pacchetto rar.

Sistema bloccato

In questa situazione non sempre dovete premere il tasto reset. Ciò può danneggiare il filesystem o portare ad una perdita di dati. In ogni modo il filesystem non sarà pulito dopo un hard reset (filesystem not clean)

Per prima cosa cercate di andare in una console con alt-ctrl-F1 o di riavviare il server X con alt-ctrl-backspace, (Se entrambe queste opzioni non funzionano c'è ancora speranza):

Il tasto SYSRQ (Tasto Stamp o Print, in alto a destra sulla tastiera) può aiutare a risvegliare in modo pulito un sistema bloccato.

Sono possibili le seguenti combinazioni di tasti:
* ALT+SYSRQ+R (dovrebbe ripristinare il controllo della tastiera)
* ALT+SYSRQ+S (sblocca un sync)
* ALT+SYSRQ+E (invia un comando di term a tutti i processi meno init)
* ALT+SYSRQ+I (invia un comando di kill a tutti i processi meno init)
* ALT+SYSRQ+U (i filesystem sono montati in sola lettura, non fa fsck (file system check) al riavvio)
* ALT+SYSRQ+B (riavvia il sistema senza i passi precedenti, questo è un 'hard reset').

È meglio concedere a ogni passo alcuni secondi per completarsi; per chiudere tutti i processi, per esempio, può servire un po' di tempo. Le lettere dei comandi di cui sopra associate al tasto 'Stamp' possono essere ricordate ricorrendo all'acronimo (in inglese): "Reboot System Even If Utterly Broken" (Riavvia il Sistema (E)anche In Ultra Blocco)

Problemi di accesso al sistema sotto KDE

Il contenuto della cartella /tmp directory viene solitamente cancellato ad ogni avvio, così anche alcune cartelle e agganci (sockets) che servono all'X-server vengono cancellati.

Normalmente, durante il processo di avvio, lo script x11-common per X-Org ricrea queste cose.

Se per qualche ragione questi script non vengono richiamati durante il processo di avvio è possibile ricreare i necessari link lanciando:

# X-ORG: # dpkg-reconfigure x11-common

KDE necessita di un 5% di allocazioni libere nella partizione in cui risiede la cartella /tmp per i file temporanei creati all'accesso. Se state lavorando con una partizione piena al 95% potreste non essere in grado di entrare in KDE e verrete fatti scendere alla tty.

Lo stesso succede con il con il kdm in looping che non vi lascia entrare. Una soluzione è autenticarsi in una tty (o richiamarla con un ctrl-alt-F1 ad es.) così potrete cancellare o ripulire alcune applicazioni o file non più necessari

Alternativamente, per accedere al sistema potete usare un gestore X window che non richiede così tanto spazio libero al sistema (per esempio, il gestore X windows fluxbox è già presente in una installazione sidux), o fare chroot usando un live-CD/DVD di sidux per ripulire la partizione per poi riuscire ad avviare di nuovo KDE.

Per una partizione usata da KDE per il suo /tmp è raccomandato che sia occupata per un massimo dell'85%. (15% libero).

Installare sidux KDE Art ed i Temi

Per installare i più recenti sidux-art in un sistema esistente:

apt-get install sidux-art-kde-xxxx sidux-art-wallpaper-xxxx
(Dove xxxx è il nome della release, per esempio sidux-art-kde-momos)

Questo installerà lo sfondo sidux ed i temi

Per cambiare lo sfondo:

Tasto destro del mouse sul desktop, scegliere Impostazioni di Desktop. Sotto la voce Sfondo alla voce "Tipo:" impostarla su Immagine, viene fornita una lista dalla quale scegliere uno sfondo da visualizzare (la lista compare accanto alla voce "Immagine:"). Si può anche scegliere una immagine extra tra quelle elencate cliccando il pulsante Sfoglia accanto alla lista delle immagini e scegliendo un file in qualunque parte del PC.

Per cambiare la schermata di accesso:

Per prima cosa aprire Impostazioni di sistema con i privilegi di amministratore (root):

Alt  + F2 (per far comparire krunner)

kdesu systemsettings

Adesso cliccare sulla scheda Avanzate e quindi su Gestione degli accessi, scegliere la scheda Tema e selezionare il tema preferito. Per attivare il nuovo gestore di accesso si deve riavviare il computer.

Display Managers - dm

Installare altri ambienti desktop insieme a quello preinstallato:

Quando si installa un altro ambiente desktop insieme a quello dell'installazione corrente, (per esempio si è installato sidux-kde.iso e adesso si vuole aggiungere l'ambiente desktop Xfce oppure LXDE), un gestore di sessioni probabilmente verrà installato insieme ad esso oppure si potrebbe averlo installato da soli (gdm, slim oppure qualche altro pacchetto dm).

Il problema è che si tornerà alla configurazione di default per i runlevel Debian con la conseguenza che non si potrà più entrare in runlevel 3 per arrestare X e fare dist-upgrade.

La soluzione è:

apt-get update
apt-get install --reinstall distro-defaults
update-rc.d <dm> remove
update-rc.d <dm> defaults

Esempi per update-rc.d <dm> defaults e notare il punto . :

update-rc.d kdm start 24 5 . stop 01 0 1 2 3 4 6 .

update-rc.d gdm start 24 5 . stop 01 0 1 2 3 4 6 .

update-rc.d slim start 01 5 . stop 01 0 1 2 3 4 6 .
