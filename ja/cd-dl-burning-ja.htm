重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

ミラー上のファイルの説明

siduxのミラーには次のようなファイルがあります:

sidux-2009-01-ouranos-kde-lite-amd64-200902151304.amd64.manifest
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.md5
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.md5.asc
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.sha256
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.sha256.asc

.manifestのファイルはISOイメージの中に含まれているパッケージの一覧です。.isoはダウンロードするためのISOイメージファイルです。.md5と.sha256はISOイメージの統合性をチェックするためのファイルです。

.ascは署名ファイルで、(.md5や.sha256の)チェックサムが変わっていないことを確認するために使われます。

.asc ファイルが有効であることを確認したい場合はユーザーモードで次のようにします。

1) Stefan Lippers-Hollmann (通称slh)のパブリックキーをキーサーバーからインポートします:

$ gpg --keyserver wwwkeys.eu.pgp.net --recv-keys F80994F6
gpg --edit-key F80994F6
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) 次にチェックサムの統合性を次のように確認します:

$ gpg --verify sidux-2009-01-ouranos-kde-full-i386-amd64-200902151348.iso.md5.asc sidux-2009-01-ouranos-kde-full-i386-amd64-200902151348.iso.md5

gpgコマンドでキーを信用できるものと分類したならば、このコマンドの出力に"5 trusted ultimately"と表示されるはずです:

$ gpg: Signature made Sun 15 Feb 2009 04:55:55 PM CET using DSA key ID F80994F6
$ gpg: checking the trustdb
$ gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
$ gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

キーを信用できるものと分類しなかった場合は出力として'Good signature'が警告とともに表示されます。:

$ gpg: Signature made Sun 15 Feb 2009 04:55:55 PM CET using DSA key ID F80994F6
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"
$ gpg: WARNING: This key is not certified with a trusted signature!
$ gpg:          There is no indication that the signature belongs to the owner.
$ Primary key fingerprint: 64C3 6120 DA8D 91E7 378B  E79F 3916 C431 F809 94F6
siduxをホストしてくれているすべてのミラーサイトに感謝を捧げます.

ダウンロードのリンクとミラーについてはこちらにもどうぞ sidux.com

Bittorentを使うダウンロード

Bittorentはファイル・シェアリングのプロトコールで、sidux-CDにも搭載されています。 これを使えばファイルをインターネットからダウンロードしている間に同時にアップロードもすることができます。 実際のダウンロードのリンクは sidux.com から探してください。　例:

# btdownloadcurses --max_upload_rate 12 ftp://dummy/sidux/sidux-200xxxxx/sidux-200xxxxx.iso.torrent

コンソールですぐにダウンロードがはじまります。このコマンドをスタートするディレクトリにISOファイルを書き込めるだけの(約400MB -2GBの)スペースがあることを確認してください。

ダウンロードが終了してもファイをル他のユーザとシェアできるよう、しばらくはプログラムをそのまま走らせていてください。これはサーバーの負担を軽くして、ファイルの普及を助けることになります。 詳しくはBrianの BitTorent FAQ とガイドを読んでください。

ダウンロードが終わったら忘れずにmd5sumでダウンロードを検証してください。

もしスピードと通信量の十分なFTPサーバを自由に使える人がいたら、ぜひこちらのsidux teamに連絡してください。 もしくはsidux Forums か oftc.netのIRC-Channelでもかまいません。

md5sumとファイルの検証

重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

いつもデータのバックアップを!

md5sumとはファイルのデータのチェック・サムで、これを使ってファイルの完全性をチェックできます。 手元のファイルのmd5sumとオリジナルのチェック・サムとが一致することでファイルが変更または破損されていないことを確認できます。インターネットを通じてダウンロードしたファイルの検証に使ってください。

ダウンロードしたファイルのmd5sumがmd5ファイル内の値と一致していれば、ダウンロードが成功したとみなせます。Linuxのシステムでは(少々時間がかかりますが）次のコマンドでチェック・サムが計算できます

$ md5sum チェックするファイル

計算結果はコンソールに表示されます。
md5sum(486 kb)を使えばWindowsでもmd5sumが計算できます。

siduxのISOイメージはいつもmd5sumが一緒についてきますから、CDに焼き付ける前に必ずチェックするべきです。そうすればもし後で問題が起こっても、ダウンロードの問題ではないことが明らかになり、ISOファイルの欠損から引き起こされた問題でフォーラムにサポートを求めるケースを避けることができます。コンソールでまずISOファイルのあるディレクトリに移動し、次の例のコマンドでmd5sumをチェックします:

$ md5sum -c sidux.iso.md5

合計が一致しないと次のように表示されます:

"sidux.iso: Error
md5sum: Warning: calculated checksum does not match!"

ダウンロードされたファイルが検証されればプログラムは何も表示せずに終了します。

WindowsでCDを焼くには?

重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

もちろんWindowsからでもCDを焼くことはできます。ダウンロードしたファイルはISOファイルとして焼かなければいけません。Winrar（または他のアーカイバ）がISOファイルにリンクされているとファイルがアーカイブとして扱われて面倒になるかもしれません。ISOファイルはそのままCDに焼かなければいけません。

WindowsでCDを焼くにはいろいろな方法があります。

Windowsのオープン・ソースのCD書き込みプログラム

cdrtfe :Windows 9x/ME/2000/XP上で動きます (Win95, Win98SE, Win2000, and WinXPでテストされています) Win9x/MEではASPIレイヤー(Adaptec ASPI 4.60など)上のみ作動します。

Windowsのクローズド・ソースでプロプライエタリ・ソフトのCD書き込みプログラム
  • Nero (nero expressではダメ!)
  • 他にフリーウェアのなかなかよいツールとしては CD/DVD Burner XP pro があります。
  • terabyteunlimited のBurnCDCCはISOファイルのCDへの書き込み専用です。

LinuxでCDを焼くには

重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

すでにあなたのコンピュータでLinuxを使っている場合は、シェルから、またはインストールされているCD書き込みプルグラムを使えばCDを焼くことができます。

siduxや、多くのLinuxのディストロがデフォルトのCD書き込みプログラムとしてK3bを搭載しています。このプログラムをオープンし"Extras" -> "burn CD-Image..." そして焼こうとしているISOファイル(例.sidux.iso)を選択してください。

K3Bは始めにISOファイルのMD5-sumを計算します(これにしばらく時間がかかります)。この計算の結果とMD5ファイルの内容が一致したらダウンロードは成功したとして、"Start"を押せばCDの書き込みが始まります。

Problems with burning mostly happen with a frontend. You can burn directly from the shell with burniso.

XDELTA-Patchを使ってsidux-CDを作る

重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

xdeltaはすでに存在するISOイメージと変更ファイルを使って新しいISOイメージを作ることができるプログラムです。 また、これを使えって再び長いダウンロードなしで小さな修正などを行うことなどもできます。

ターミナル/コンソールからrootとして:

apt-get update
apt-get install xdelta

一例として、553MBのISOイメージに対して変更ファイルpatch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta はたったの127MBだけです。

たとえば:

  • ハードディスク上にISOイメージ、仮にsidux.isoがあったとします
  • 変更ファイルが patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdeltaだったら:

ターミナル/コンソールからユーザーとして:

$ cd /path/to/iso/and/patch
$ xdelta patch patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta

このプログラムはパッチから必要なファイルネームを読み出して使います。 md5sumのチェックも行います。ただしハードディスクの残りスペースはチェックしないので気をつけてください。

Windowsでsidux-LiveCDにパッチする場合はWindows用の xdelta.exeを使ってください。 Windowsで xdelta を使う際の情報はこちらへ
