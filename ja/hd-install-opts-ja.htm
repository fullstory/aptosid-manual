"fromiso"で起動する - Overview

普通のユーザーにはext4を勧めます; しっかりメンテナンスされていて、aptosidのデフォルトのファイルシステムです。

このチートコード(起動オプション)でext2/3/4のパーティション上のisoファイルから起動することができ、CDから起動するよりずっと速く動きます。 (ハードディスクへのインストールも"fromiso"を使えばずっと早く終わります)。

"fromiso"はCDやDVDドライブよりずっと早く、また起動中にCD/DVDドライブを他の目的に使える利点があります。 他の方法としてQEMUを使うこともできます。

必要条件:

* ちゃんと作動するGrub (フロッピー、ハードドライブ上、またはライブCD)
* aptosidのISOイメージ: aptosid.isoとext2/3/4などのLinuxふぁいるシステム
* Unixのファイルシステムはケースセンシティブなので、もしイメージファイルの名がaptosid.ISOのように変更されていたらコマンドもその通りに変える必要があります。

fromiso と Grub2

aptosid は fromiso のエントリーを Grub2 のメニューに加えるために 60_fll-fromiso というファイルを提供しています。fromiso のために設定が必要なのはただ一つ、/etc/default/grub2-fll-fromiso のフォルダーにある grub2-fll-fromiso というファイルだけです。

まずターミナルを開きsux のコマンドで root になり:

sux
apt-get update
apt-get install grub2-fll-fromiso

次に好きなエディタ(kwrite、mcedit、vim、その他):

mcedit /etc/default/grub2-fll-fromiso

次に、有効にしたい設定のコメント行の頭の # をとり、 "引用記号" にはさまれたデフォルトに必要な変更を加えます。

例として、下に変更を加えたあとの altered grub2-fll-fromiso を示します。(変更があった行がハイライト で表示されています):

# Defaults for grub2-fll-fromiso update-grub helper
# sourced by grub2's update-grub
# installed at /etc/default/grub2-fll-fromiso by the maintainer scripts

#
# This is a POSIX shell fragment
#

# specify where to look for the ISO
# default: /srv/ISO ### Note: This is the path to the directory that contains the ISO,
 it is not to include the actual aptosid-*.iso file.###
FLL_GRUB2_ISO_LOCATION="/media/disk1part4/aptosid-iso"

# array for defining ISO prefices --> aptosid-*.iso
# default: "aptosid- fullstory-"
FLL_GRUB2_ISO_PREFIX="aptosid-"

# set default language
# default: en_US
FLL_GRUB2_LANG="en_AU"

# override the default timezone.
# default: UTC
FLL_GRUB2_TZ="Australia/Melbourne"

# kernel framebuffer resolution, see
# http://manual.aptosid.com/de/cheatcodes-vga-de.htm#vga
# default: 791
#FLL_GRUB2_VGA="791"

# additional cheatcodes
# default: noeject
FLL_GRUB2_CHEATCODE="noeject nointro"

ファイルを保存してエディタを終了し、ターミナルで次のコマンドを実行します。

update-grub

Grub2 の grub.cfg がアップデートされ、次の起動時から新しく指定したディレクトリのISOが表示され、そこから起動することができるようになります。

fromiso と persist について

ハードディスクからの fromiso と persist

fromiso の設定と persist の起動オプションを組み合わせれば、ハードディスクを使った持続的なライブシステムができます。デフォルトとして使う起動オプションは:

persist

fromiso が VFATのファイルシステムを使う場合は Linux のファイルシステムを内容としてふくむファイルを使う必要があります。起動オプションは次のようになります:

persist=/aptosid/aptosid-rw

aptosid は aufs を使って、いわゆる "copy on write" (書き込み時コピー)という方法で新しいファイルをメモリーに保存することで、 ISOイメージ上のファイルシステムに新しいファイルやフォルダーを作成したり、もとからあるファイルに変更を加えたりできるようにしています。起動オプションに persist を使うと、新しいファイルをハードディスクの fromiso のISOイメージのあるのと同じパーティションに保存します。

fromiso でライブシステムを起動すると aptosid のライブISOの自動機能が実行され、ハードウェアが自動的に設定されたりする利点がありますが、それは同時に、起動のたびに同じファイルを繰り返し作成している、ということです。

ところが起動オプションに、noxorgconf, nonetwork といった aptosid 独自のオプションと合わせて persist を使うことで、同じファイルを起動のたびに作成する必要がなくなります。

起動オプションに persist を使うということは、apt を使ってパッケージをインストールしたり、プログラムや保存したデータを次に起動したときにも使えるということでもあります。

USB-sticks/SD/flash-cardsからの fromiso と persist

持続モードの理想的な使い方は install-usb-gui ツールを使って必要なファイルやソフトウェアをあらかじめ用意した起動フラッシュメモリを作ることかもしれません。 Your files will be stored in a subfolder on the drive.

persist を市販のフラッシュドライブのデフォルトで、DOSやWindows9xで広く使われていた FAT ファイルシステムで使うにはひとつの大きなファイルをループデバイスとして作成し、フォーマットします。

USB-sticks/SD/flash-cardsなどのフラッシュドライブでは ext2 と VFAT システムを推薦します。MS Windows™用のドライバもあるためデータ交換に使うときより多くのプラットフォームで使うことができ、またもしもの場合のデータ救済の可能性も高いでしょう。Read/Writes to flash type drives are contingent on the specifications of your USB USB-sticks/SD/flash-cards.

ext2 ファイルシステム

ext2/3 の場合、パーティション全体が使われます。現行システムのrootが使われ ライブモードのメカニズムとして/fll というディレクトリが作成され残りのスペースが持続モードのファイル保存用に使われます。

VFAT ファイルシステム

VFAT が使われると、持続モードのファイル保存スペースとして最大２Gb、最小100Mbが使えます。(それ以下では役にたたないでしょう)　このファイルにaptosid-rw.

最初のインストールが終わってから持続モードに設定する例

マウントポイント名について曖昧に感じるときは、まずフラッシュメモリをマウントして ls -lh /media のコマンドを実行します。出力の中から drwxr-xr-x 6 username root 4.0K Jan 1 1970 disk のような行を探します。もし disk ではなく他の名前でマウントされている場合は "/media/disk" をその名に合わせて変更してください。(例 "/media/disk-1").

この例で続けると、 df -h のコマンドで確かめることができます:

/dev/sdc2             3.4G  4.0K  3.4G   1% /media/disk
/dev/sdc1             4.1G  1.1G  2.8G  28% /media/disk-1

つまり:

disk="/media/disk-1"

持続モードのパーティションのサイズを決める:

size=1024

フラッシュメモリ上にディレクトリを作る:

mkdir $disk/aptosid

次の持続モードパーティション作成のコマンドを実行する:

dd if=/dev/zero of=$disk-1/aptosid/aptosid-rw bs=1M count=$size &&  echo 'y' | LANG=C /sbin/mkfs.ext2 $disk-1/aptosid/aptosid-rw &&  tune2fs -c 0 "$disk-1/aptosid/aptosid-rw"

Windows NT/2000/XP(TM)のインストールに使われている NTFS システムでは持続モードはできません。

aptosidを USB/SD/flash ドライブにインストール

aptosid をUSBスティックやSD/フラッシュカードといった外部ディスクにインストールするのは普通のハードディスクにインストールするのと同じように簡単です。

必要条件:

USB 2.0 プロトコルとUSB/SD/flashからの起動をサポートしているPC。

aptosidのISOファイル.

USB/SD/flash にインストールする３つの方法
  • 1) fromiso ; aptosid 独自のフィーチャー: aptosid-on-a-stick
  • 2) Full (USB/SD/flash にハードドライブの場合と同じように通常のインストーラでインストールする).
  • 3) RAW デバイス として書き込みする。これはPCをその他のLinux、MS Windows、Mac OS X といったオペレーティングシステムで使っていて、aptosid をフラッシュドライブから起動する。(うまくいかない場合があります)

USB/SD/flash への fromiso インストール aptosid-on-a-stick

まず準備としてUSB/SD/flashデバイスを ext2 または FAT32 にフォーマットします。(２GB以上) このデバイスにはパーティションが一つだけになるようにして、このパーティションを起動可能にします。(BIOSによってはこれをしないと起動できないことがあるため) 。

GParted のような GUI アプリケーションでフォーマットするときは、まず元々あったパーティションを削除して、その後でパーティションを新しく作ってフォーマットしてください。

ハードディスクにインストールされている aptosid のシステムで USB のfromiso ドライブを作る

fromiso USB インストールはハードディスクにインストールされたシステムからはMenu>System>install-aptosid-to-usb

USB fromiso from a aptosid-*.iso:

で実行できます。ライブCDの場合はaptosid Installer アイコン からインストーラを実行して、Install to USBを選びます。

Options:

デフォルトで使用する言語、タイムゾーンその他のオプションや persist モードで起動するかなどをチェックボックスで設定します。

これで機動可能なUSB/SD/flashデバイスができました。あとからpersist モードを使いたくなった場合は機動時のGRUBのコマンド行に persist を加えるか、または menu.lst の #kopt= の行に persist と書き込みます。. . (VFAT のフラッシュメモリならば多分はじめからインストールし直したほうがいいでしょう。)

コンソールからのインストールの一例
 fll-iso2usb -D /dev/sdb -f none --iso /home/aptosid/aptosid.iso -p -- lang=no tz=Pacific/Auckland

このコマンドは ISO イメージをsdbにあたる USB デバイスに parsist と のオプションで、ノルウェー語とニュージーランドの太平洋/オークランド時間をGRUBのデフォルトに設定してインストールします。

Xウィンドウズの設定(ビデオカード、キーボード、マウスなど）やネットワークインターフェイスファイルは保存されないので、いろいろなコンピュータで起動するライブシステムに最適です。

カスタマイズオプションなどの詳細は次のコマンドのmanページを見てください:

$ man fll-iso2usb

USB/SD/flash への通常の(ハードディスクへのインストールと同じ)フルインストール

最小限のサイズとしてUSB/SD/flash デバイスは:
"LITE" バージョンのaptosidはシステムに2.5GB
"FULL" バージョンのaptosid はシステムに 4 GBe

程度は必要でしょう。これにユーザーとして必要なデータスペースを計算してサイズを決めてください。

準備としてデバイスを ext2 にフォーマットしてから普通のハードディスクと同じようにパーティションします。

ライブCDからインストーラを起動してUSB/SD/flashデバイスのパーティションを aptosid のインストール先として選びます。インストーラのダイアログ上で例としてばsdbX を選び、インストーラの指示に従ってインストールします。詳しくはハードディスクへのインストールを参照。

USB/SD/flashデバイスから起動するには BIOS の設定で 'Boot from USB' が有効になっていなくてはなりません。

その他のポイントとして:

  • USB/SD/flash へのフルインストールは通常インストールを行ったPCのみで使えます。他のPCでも使いたい場合はnon-freeのグラフィックドライバーは使わず、特別な起動オプションも使わないようにします。例外として起動オプションに vesa を使うのはハードウェアの違うPCで起動するのに便利です。(これらの設定はますインストールがうまくいってから起動後に設定します。)
  • 他のPCで USB/SD/flash から起動する場合、そのPCのハードディスクにアクセスするにはfstabを修正する必要があるでしょう。
  • 他のPCからも起動することを念頭に入れている場合は "fromiso" と "persist" のオプションを使うほうがいいでしょう。

USBのハードディスクのパーティションにフルインストールする

USB ハードディスクへのインストールには（とくにMSや他のディストロから来た新しいユーザーにとって）魅力があります。aptosid をUSB ハードディスクにインストールすれば、そのPCを複数のシステムで起動するための(パーティショニング、GRUB設定などの)ややこしい作業を避けることができるからです。

ライブモードからインストーラを起動して USBインストールではなく、普通のインストールをはじめて aptosid システムをインストールしたいパーティション、例として sdbX を選び、インストーラの指示に従います。GRUBはUSBハードディスクのパーティションにインストールします。

詳しくはハードディスクへのインストール参照。

その他のポイントとして:

  • USB/SD/flash へのフルインストールは通常インストールを行ったPCのみで使えます。他のPCでも使いたい場合はnon-freeのグラフィックドライバーは使わず、特別な起動オプションも使わないようにします。例外として起動オプションに vesa を使うのはハードウェアの違うPCで起動するのに便利です。(これらの設定はますインストールがうまくいってから起動後に設定します。)
  • このシステムを他のPCでも使いたい場合は、ハードディスクのアクセスのため fstab を変更したり、X ウィンドウの設定のため xorg.conf を修正したり、ネットワーク設定を変更したりする必要があるでしょう。

Full installation to bootable GPT removable devices (behaves as a normal HD installation)

Refer to Partitioning with gdisk for GPT disks and then follow the instructions for Installation options - HD, USB, VM and Cryptroot.

Bootable (U)EFI removable devices

Applicable from the aptosid 2011-02 release.

If you want to boot using EFI without burning optical media, then you need a vfat partition containing a portable EFI bootloader /efi/boot/bootx64.efi. The aptosid amd64 isos include such a file and a grub configuration which it can load. To prepare a stick to boot this way, simply copy the contents of the aptosid iso to the root of a vfat formatted usb stick. You should also mark the partition as bootable using a disk partitioning tool.

Of course simply copying the files onto a vfat usb stick will not let you boot it on a traditional bios system, however it is quite easy to enable this using syslinux and install-mbr. All you need to do is run (without the stick being mounted): 

syslinux -i -d /boot/isolinux /dev/sdXN
install-mbr /dev/sdX

A stick prepared this way, will boot both by EFI to the plain grub2 menu and by traditional bios to the graphical gfxboot menu.

One of the advantages of having a stick created this way, as opposed to a raw stick created due to using isohybrid, is that you can edit the boot files on the stick to add your preferred options automatically.

For traditional BIOS systems you can edit the /boot/isolinux/syslinux.cfg file and/or the /boot/isolinux/gfxboot.cfg file. For EFI systems you can edit the /boot/grub/x86_64-efi/grub.cfg file.
