non-freeのバイナリードライバー: nVidia / ATI

3Dドライバーをインストールするには4つの方法があります

  1. sgfxi / !h2 smxi

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxiスクリプトは今のところATIのfglrxとnVidiaのドライバーをサポートしています。 Xorgのatiやnvといったフリードライバーとの交換もサポートしています。 smxiスクリプトはsgfxiをグラフィックドライバーのインストールエンジンとして使っていますが、いくつかのオプションの選択肢を与えているだけです。

    パラメータなしでスクリプトを実行するとデフォルトのドライバーがインストールされます。 古いカードを使っていて過去のドライバーをインストールする必要がある場合をのぞいてはスクリプトが自動的に適切なドライバーを選択します。

    スクリプトを実行すると、一旦ストップして、これからどのドライバーをインストール使用としているかを告げます。 これを受け入れるか、またはここで実行を停止して別のオプションで他のドライバーを指定することもできます。

    !h2のsmxiスクリプトからグラフィックドライバーをインストールすることもできます。 smxiからsgfxiが実行されます。 詳しくはまず!h2 smxi ホームページ を読んでからこのコマンドを実行してください。

    smxi -kiwdt

    sgfxi-h: -hオプションを使えば今サポートされているドライバの表と、合わせてカーネルやXorgのバージョンやシステムについての情報をを表示します。 sgfxiのオプションをいくつか紹介します。

    -c コンポジットモードで使う (nVidiaカードのみ)
-d IgnoreDisplayDevicesオプションを取り除く, TVも選べるようになる (nVidiaカードのみ)
-h ヘルプメニューの表示
-n Xorgのnv, intel, またはatiのドライバーからシステムに適合するものを自動的に選択してインストール
   古いバイナリードライバーのファイルを消し、xorg.confからnVidiaやfgrlxについての情報を取り除き、
			新しいドライバーの情報を書き込む
-N 特定のバージョンのドライバーをインストールする
   シンタックス: sgfxi -N <リストから選んだドライバー, 例： -N radeon>
   サポートされているXorgドライバー: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   xorg.confの必要な設定も自動的に書き込みます。 Radeon Xorgの3D設定も自動的に行います
-o デフォルトの代わりに特別なドライバーをインストールする、絶対的なドライバー指定する. 
   シンタックス:
   -o 9746 (nVidiaのドライバーでは1.0-の部分は書きません) または: -o 8.32.5 (fglrx)
-r xorg.confからモードに関する記述を削除する
-t ダイナミックツウィンヴューを有効にする (nVidiaのみ)

  2. install-binary-gfx

    グラフィックドライバーをインストールする新しいスクリプトは: install-binary-gfxといいます。 このスクリプトをダウンロードするにはコンソールからrootとして

    get-sidux-binary-gfx
install-binary-gfx -a (to build sid source)

    このスクリプトはカードに合うドライバーを選択し、debファイルを作成し、インストールしさらにxorg.confも自動的に調整します。 新しいドライバーを有効にするには(Ctrl+Alt+Backspace)でXウィンドウを再スタートします。

    オプション

    次のようなオプションが選べます:

     Usage: ./install-binary-gfx [options]
-a | --auto            Automatic mode　# 自動モード
-b | --build           MOD Build only (nvidia|nvidia-legacy|ati|ati-legacy|fglrx) # モジュールの作成のみ
-d | --detect          Detect Card only	# ハードウェア探査のみ
-u | --update          Update Script    # スクリプトのアップデート
-v | --verbose         Verbose          # 詳細を出力

    例

    install-binary-gfx -a (カードを探査してsidのドライバーを自動的にインストール)
install-binary-gfx -b nvidia (nvidia-module....deb を /usr/src に作成。 同様にfglrxやnvidia-legacyのモジュールも作成できます)
install-binary-gfx -b ati (ati-module....debを/usr/srcに作成)
install-binary-gfx -d (カードを探査しデータを表示します)
install-binary-gfx -u (スクリプトをアップデート)

    困ったときは

    Scriptが成功しなかったときはデバッグモードで実行し、表示された出力とログファイル/var/log/sidux-CARD-installer.logをsiduxフォーラムまたはhttp://paste.debian.netにペーストしてIRCの#siduxに行き、リンクを示してアドバイスを求めてください

  3. ライブCDへのバイナリーnon-freeドライバーのインストール:

    sudo sgfxi

    これで最新のスクリプトをダウンロードし:

    Ctrl+Alt+F1

    そして

    sgfxi

    を実行します

    sgfxi -h,
to see options

    ドライバーのインストールのあとXが再スタートします

  4. nVidia/ATIのバイナリーインストーラ
    nVidiaから直接バイナリードライバーインストーラをダウンロードします
ATIから直接バイナリードライバーインストーラをダウンロードします

nVidia, ATI, Radeon, Intel用のフリーXorgドライバー

  1. sgfxi

    sgfxiを使ってフリーからnon-freeのドライバーへの切り替えや、またフリーのドライバーに戻すことなどもできます。 主なコマンドは:

    -n ハードウェアに適合したXorgのnv、intel、またはatiドライバーを自動的にインストールします。
   古いバイナリードライバーのファイルを消し、xorg.confからnVidiaやfgrlxについての情報を取り除き、
   新しいドライバーの情報を書き込む
-N 特定のバージョンのドライバーをインストールする
   シンタックス: sgfxi -N <リストから選んだドライバー, 例： -N radeon>
   サポートされているXorgドライバー: 
    ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   xorg.confの必要な設定も自動的に書き込みます。 Radeon Xorgの3D設定も自動的に行います。

    まず

    apt-get update
apt-get install sidux-scripts
sgfxi

    次にXorgのドライバーをインストールまたは再インストールするには

    sgfxi -n
または
sgfxi -N <xorg driver>

  2. sgfxi / !h2 smxi

    sgfxiを!h2のsmxiから使用できます。 smxiスクリプトを次のオプションで実行すればグラフィックインストーラの部門に直行します。 smxiを使っている人はsgfxiを特別にインストールすることは要りません。 smxiが自動的にインストールしますから。

    詳しくは!h2 smxiのホームページ を読んで、それからこのコマンドを実行します:

    smxi -kiwdt

  3. 自分でインストールしたい人は

    X.OrgのXサーバー(詳しくはxserver-xorgを見てください)はNVIDIAのRiva、TNT、GeForce、QuadroのカードとATIのMach、Rage、Radeonの２Dと、'atimisc'、'r128'や'radeon'といったサブドライバーを併用することによりFireGLcardsの２Dをサポートしています。 同時にIntelのfreeシリーズの３Dもサポートしています。

    インストールするには:

    KDEからログアウトしCtrl+Alt+F1でテキストモードに入り
rootとしてログインし
init 3とタイプします
(Xが終了したというメッセージが出るはずです)
apt-get update
apt-get install xserver-xorg-video-nv
または
apt-get install xserver-xorg-video-ati
init 5 && exit

    3DサポートのバイナリーのnVidia/ATIドライバーとは同じではないところに注意、ただしバイナリードライバーはサポートされません。

    次に/etc/X11/xorg.confを編集しますが, このファイルの編集中にを間違えをすると泣かされるハメになりますからまず必ずバックアップのコピーを作ってください。

    /etc/X11/xorg.confを編集するには右クリックしてrootとして開きます。 Section "Device"の項目を探しDriverをnvまたはatiに変えます。

    警告しましたよ!! XORG.CONFの編集のリスクは自分で負ってください!!

non-freeのドライバーが必要なハードウェアについてのヒント

複雑な法律的な理由のためsiduxのライブCDはdfsg-free (DebianのFree Software Guidelineの立場からfreeとみなされるもの）だけを搭載しています。

Sources List

siduxのISOにはdfsg-freeのソフトウェアしか含まれていないため、 /etc/apt/sources.list.d/debian.list または/etc/apt/sources.list にcontribやnon-freeを加えてネットに接続してドライバーをダウンロードする必要があるでしょう:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Video Cards
  • ATi Radeon: r35xまでの古いカードは3Dアクセレーションができるするはずですが、それ以降のRadeon X1xxxカードでアクセレーションを実現するにはnon-freeドライバーが必要です。 最新のRadeon X1xxxモデルは内蔵のチップによっては起動オプションに"xmodule=vesa"か"xmodule=fbdev"を使う必要があるかもしれません。
Firmware
  • どのファームウェアをインストールする必要があるかを調べるには、ユーザーとして次のコマンドを実行します:
$ fw-detect

インストールすべきドライバのリストとコマンドが表示されます。なにも必要ないか、すでにインストールされている場合は何も表示されません。

  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • Broadcom, bcm43xx/ b43/ b43 legacy wlan: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • Broadcom Blutonium Bluetooth chipset (BCM2033): 
    apt-get install bluez-firmware
  • Eagle USB ADSL modem: ファームウェアをhttp://eagle-usb.org/ueagle-atm/non-free/ からダウンロードして /lib/firmware/に保存します。
  • 多機能の DVB TVカード用のDVBファームウェア(大抵の安価なTVカードには要りません): (dmesgをチェックして)必要なファームウェアをhttp://www.linuxtv.org/downloads/firmware/からダウンロードして /lib/firmware/に保存します。
  • IVTV firmware:fetch the firmware from http://dl.ivtvdriver.org/ivtv/firmware/ and place it under/lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan iwlwifi-ucodeまたはiwlwifi-3945、firmware-iwl3945、firmware-iwl4965がすでにインストールしてあったら削除してください:
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54 MBit/s wlan: ファームウェアをhttp://prism54.org/firmware/からダウンロードして /lib/firmware/に保存します。
  • nVidia graphics: フリードライバーではまだ3Dアクセレーションはできません。 get-sidux-binary-gfx または sgfxi からスクリプトをダウンロードしてインストールしてください。
  • RaLink rt61 and rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    rt61-fwcutter、rt73-fwcutter、firmware-rt61またはfirmware-rt73がすでにインストールしてあったら削除してください。

  • Ricoh R5U870 OEM webcams:fetch the firmware from http://mediati.org/r5u870/r5u870-0.11.0.tar.gz and copy the r5u870_*.fw files to /lib/firmware/.
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan: 
    m-a a-i acx100
    ファームウェアをhttp://www.kazer.org/acx-firmware-20060207.tar.bz2からダウンロードして/lib/firmware/に保存します。
  • ZyDAS zd1201 11 MBit/s wlan: Fetch the firmware from http://surfnet.dl.sourceforge.net/sourceforge/linux-lc100020/zd1201-0.14-fw.tar.gz and place it under /lib/firmware/.
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

ntfs-3gを使ってNTFSパーティションに書き込む

警告: ntfs-3gは'stable'であると表示はされていますが,データの外部バックアップなしに使ったり、重要な仕事用のシステムに使うのは避けるべきです。 敢えてそのような使い方をしてデータを失っても自業自得ですから自分の責任で使うこと！

ターミナルを開き次のコマンドを入力します: ハードディスクのパーティション - ディスクの名前についてを参照

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
コンソールを終了するには: exit

これであなたのNTFSパーティションはrw(読み書きモード)でマウントされていて、データの書き込みができるるはずです。 警告を繰り返しますが これは緊急の場合に使うもので、日々の仕事につかうのにはお勧めできません！
