non-freeのドライバーが必要なハードウェアについてのヒント

sidux のライブ ISO には dfsg-free のソフトウェアだけが提供されています。

Sources List

siduxのISOにはdfsg-freeのソフトウェアしか含まれていないため、 /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list にcontribやnon-freeを加えてネットに接続してドライバーをダウンロードする必要があるでしょう。編集を加えたあとの行は、ミラーの選択によって多少の違いはありますが、おおよそ次のようになります:

# Debian
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main fix.main
#deb-src http://sidux.com/debian/ sid main fix.main

Apt のレポジトリソースを変更したあとで実際にパッケージの検索やインストールを始める前にまず apt-get update のコマンドでパッケージデータベースを更新する必要があります。( 新しくパッケージをインストールする前に習慣としてまず apt-get update を実行するようにしましょう。)

nVidia, ATI/AMD, Radeon, Intel用のXorgのオープンソースドライバを使う

nVidia, ATI/AMD, Radeon, Intel用のXorgのオープンソースドライバはsiduxに標準装備されています。

ノート: オープンソースのドライバを使う場合一般的に xorg.conf は必要ありません。

ATIやnVidiaなどの商業ドライバを使っていて、オープンソースのドライバに戻したいというときには /etc/X11/xorg.conf.d/xx-xxxx.conf をrootとしてテキストエディタで開いて、SECTION DEVICE のセクションを探し、ディスプレイドライバの行を実際のグラフィックカードの種類に応じて nv または radeon または intel (などなど)のように変更します。

Xorg.conf の編集のリスクは自分の責任で

詳しい情報は Intel   ATI/AMD   nv  X.Orgで調べてください。

nVidiaのクローズソースバイナリードライバーをdmakmsで自動インストール & xorg.conf.d

実際にサポートされている nVidia のGPUの一覧表などは NVIDIA Linux Graphics Driver download pageを見てください。

その他のオプションについてnvnews なども参考になります。

新しくインストールしたシステムは /etc/X11/xorg.conf.d のディレクトリがあることを確認して、そこに 20-nvidia.conf というファイルを加えます。以前からのインストールも次のようにして最新の設定にできます 

mkdir /etc/X11/xorg.conf.d
touch /etc/X11/xorg.conf.d/20-nvidia.conf

そしてお好みのテキストエディタ(kwrite、kate、mousepad、mcedit、vi、vimなどなど) でそのファイル 20-nvidia.conf を開いて:

<editor> /etc/X11/xorg.conf.d/20-nvidia.conf

次の内容のエントリーを加えます:

#
Section "Device"
         Identifier      "Device 0"
         Driver          "nvidia"
EndSection
# This is a trailing line, it is needed so that End Section is not the last line

複数のグラフィックカードが搭載されているPCでは、PCIを確認して 20-nvidia.conf に書き込む必要があります。まず:

lspci | grep -i vga

このコマンドはおおよそ次のような行を出力します:

01:00.0 VGA compatible controller:

上の場合、01:00.0 のBusID を'Driver' の行の下に加えます。その際シンタックスが二つめの０を削って、コロン(':')が加えられたPCI:x:y:z: の形になることに注意してください。つまり上の例では次のようになります:

BusID		 "PCI:1:0:0"

nVidia ドライバのインストール

ノート: apt-cache search nvidia と apt-cache show <package> のコマンドを実行して、どのドライバを使うべきかを確認してください。nVidia ドライバには。現行の Debian Sid 3D ドライバーと、旧式のGPU用の旧式(legacy type) Debian Sid 3D ドライバの二つのタイプがあります。

現行の nVidia 3D ドライバ ≥ GeForce 6xxx :

まずモジュールの準備:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

次にnvidiaドライバーのためのdmakms(Dynamic Module-Assistant Kernel Module Support)を有効にして、今後カーネルが更新されたときnvidiaモジュールも自動的に準備されるようにします。そのためには/etc/default/dmakms の設定ファイルにnvidia-kernel-sourceを加えます。:

echo nvidia-kernel-source >> /etc/default/dmakms

それから:

m-a a-i nvidia-kernel-source

さらに続けて:

apt-get install nvidia-glx

Reboot PC for installation of the module to take effect.

#### -->

PCを再起動してモジュールが有効になっているか確かめます。

Xorg のアップデートの際には nvidia-glx パッケージを再インストールするだけです。:

apt-get install --reinstall nvidia-glx

Debian sid に新しいバージョンの nVidia ドライバが入って、apt コマンドでインストールできるようになったら:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

そのあと PC を再起動するとモジュールが有効になります。

旧式の nVidia ドライバの名前は Debian では次のように区別しています。

  • nvidia-kernel-legacy-71xx is for GeForce 2
  • nvidia-kernel-legacy-96xx is for GeForce 4
  • nvidia-kernel-legacy-173xx is for GeForce 5
旧式の 3D ドライバを ≤ GeForce 5xxx のGPUで使う場合の例:

他の旧式ドライバを使うときには 173xx を他の番号に取り替えてください。

m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Alter dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Xorg がアップデートされたときは nvidia-glx-legacy を再インストールするだけです:

apt-get install --reinstall nvidia-glx-legacy-173xx

モジュールのロードに失敗したら

いずれかの原因で nVidia のモジュールのロードが失敗したときは:

modprobe nvidia

そしてコンピュータを再起動します。

それでもまだモジュールがロードされなければ:

 m-a a-i -f nvidia-kernel-source

or

 m-a a-i -f nvidia-kernel-legacy-173xx-source

このコマンドでモジュールがもう一度構築されます。そして再起動.

くわしくはこの文書を読んでください:

$ /usr/share/doc/dmakms

ATI/AMD, Radeon and other video drivers

2D Drivers

X.OrgのXサーバー(詳しくはxserver-xorgを見てください)はNVIDIAのRiva、TNT、GeForce、QuadroのカードとATIのMach、Rage、Radeonの２Dと、atimisc、r128やradeonといったサブドライバーを併用することによりFireGL cardsの２Dをサポートしています。 Radeon and Intel both support 2d acceleration (textured xv) for video playback.

3D Drivers

ATI のカードにはxserver-xorg-video-radeonで 3D ((とKDEアニメーション)をサポートしているものもあります。今のところ r700 のチップセットまでがサポートされています。

2D または 3D のビデオカードなど、新しくパッケージに加えられた non-free のファームウェアを自動的にインストールするには:

apt-get install firmware-linux

そして PC を再起動します。

Firmware drivers

必要なファームウェアは新しく開発された fw-detect というスクリプトを使えば、デバイスとファームウェアを列挙します。(パッケージ: sidux-scripts)

$ fw-detect

fw-detect のコマンドは、必要なファームウェアをインストールし、有効化の設定するためのコマンドを出力します:

例:

#Detected driver that requires firmware to operate
#Follow these instructions to obtain the correct firmware
# and activate the zd1211rw driver:
apt-get update
apt-get install zd1211-firmware
modprobe -r zd1211rw
modprobe zd1211rw

git のレポジトリからファームウェアをインストールするには:

apt-get install git-core
USB のフラッシュメモリから non-free ファームウェアをインストール

インターネットに接続するためのファームウェアをインストールするときなど、他のオンラインシステムで必要な .deb ファイルをダウンロードして USB フラッシュメモリに保存して、別のコンピュータに移すことができます。http://cdimage.debian.org/cdimage/unofficial/non-free/firmware/sid/current/ のリンクから zip または tar.gz のフォーマットでダウンロードして、firmware という名前のフォルダーに解凍すると、いくつかの .deb ファイルになります。次に firmware-linux-free*.deb をダウンロードして先ほどの firmware フォルダーに保存し、これを USB メモリースティックに保存して、ファームウェアのインストールが必要なコンピュータに移動して:

dpkg -i firmware-linux-nonfree*.deb

のコマンドでインストールできます。

合法的に頒布できるファームウェアはパッケージにしてこちらのnon-freeレポジトリに用意していますが、すべてのベンダーが許可してくれるわけではありません。

ntfs-3gを使ってNTFSパーティションに書き込む

警告: ntfs-3gは'stable'であると表示はされていますが,データの外部バックアップなしに使ったり、重要な仕事用のシステムに使うのは避けるべきです。 敢えてそのような使い方をしてデータを失っても自業自得ですから自分の責任で使うこと！

ターミナルを開き次のコマンドを入力します: ハードディスクのパーティション - ディスクの名前についてを参照

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
コンソールを終了するには: exit

これであなたのNTFSパーティションはrw(読み書きモード)でマウントされていて、データの書き込みができるるはずです。 警告を繰り返しますが これは緊急の場合に使うもので、日々の仕事につかうのにはお勧めできません！
