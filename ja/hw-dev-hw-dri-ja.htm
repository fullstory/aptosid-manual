non-freeのバイナリードライバー: nVidia / ATI

  1. sgfxi / smxi

    sidux コミュニティーに支えられた機能豊富なシステム管理ツール smxi/sgfxi (sidux-maintenanceの略)を使ってインストールします。 詳しくはこちらsmxi と sgfxi のシンタックス一覧へ.

  2. nVidia/ATIのバイナリーインストーラ
    nVidiaから直接バイナリードライバーインストーラをダウンロードします
ATIから直接バイナリードライバーインストーラをダウンロードします

nVidia, ATI, Radeon, Intel用のXorgのオープンソースドライバを使う

nVidia, ATI, Radeon, Intel用のXorgのオープンソースドライバはsiduxに標準装備されています。 クローズソースのドライバのインストールの際にオープンソースドライバを消してしまっている場合はsgfxiを使って元に戻すことができます。

X.OrgのXサーバー(詳しくはxserver-xorgを見てください)はNVIDIAのRiva、TNT、GeForce、QuadroのカードとATIのMach、Rage、Radeonの２Dと、'atimisc'、'r128'や'radeon'といったサブドライバーを併用することによりFireGLcardsの２Dをサポートしています。 同時にIntelのfreeシリーズの３Dもサポートしています。

インストールするには:

KDEからログアウトしCtrl+Alt+F1でテキストモードに入り
rootとしてログインし
init 3とタイプします
(Xが終了したというメッセージが出るはずです)
apt-get update
apt-get install xserver-xorg-video-nv
または
apt-get install xserver-xorg-video-ati
または
xserver-xorg-video-intel
init 5 && exit

カードによっては3Dがサポートされていることもありますが、たいていは2Dのみのサポートです。

次に/etc/X11/xorg.confを編集しますが, このファイルの編集中にを間違えをすると泣かされるハメになりますからまず必ずバックアップのコピーを作ってください。

/etc/X11/xorg.confを編集するには右クリックしてrootとして開きます。 Section "Device"の項目を探しDriverをnvまたはatiに変えます。

警告しましたよ!! XORG.CONFの編集のリスクは自分で負ってください!!

non-freeのドライバーが必要なハードウェアについてのヒント

複雑な法律的な理由のためsiduxのライブCDはdfsg-free (DebianのFree Software Guidelineの立場からfreeとみなされるもの）だけを搭載しています。

Sources List

siduxのISOにはdfsg-freeのソフトウェアしか含まれていないため、 /etc/apt/sources.list.d/debian.list または/etc/apt/sources.list にcontribやnon-freeを加えてネットに接続してドライバーをダウンロードする必要があるでしょう:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Video Cards
  • ATi Radeon: r35xまでの古いカードは3Dアクセレーションができるするはずですが、それ以降のRadeon X1xxxカードでアクセレーションを実現するにはnon-freeドライバーが必要です。 最新のRadeon X1xxxモデルは内蔵のチップによっては起動オプションに"xmodule=vesa"か"xmodule=fbdev"を使う必要があるかもしれません。
Firmware
  • どのファームウェアをインストールする必要があるかを調べるには、ユーザーとして次のコマンドを実行します:
$ fw-detect

インストールすべきドライバのリストとコマンドが表示されます。なにも必要ないか、すでにインストールされている場合は何も表示されません。

  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • Broadcom, b43/ b43 legacy wlan: 
    apt-get install b43-fwcutter
  • Broadcom Blutonium Bluetooth chipset (BCM2033): 
    apt-get install bluez-firmware
  • Eagle USB ADSL modem: ファームウェアをhttp://eagle-usb.org/ueagle-atm/non-free/ からダウンロードして /lib/firmware/に保存します。
  • 多機能の DVB TVカード用のDVBファームウェア(大抵の安価なTVカードには要りません): (dmesgをチェックして)必要なファームウェアをhttp://www.linuxtv.org/downloads/firmware/からダウンロードして /lib/firmware/に保存します。
  • IVTV:ファームウェアをhttp://dl.ivtvdriver.org/ivtv/firmware/からダウンロードして/lib/firmware/に保存します。
  • Intel ipw2100 and ipw2200, 11/54 MBit/s wlan: 
    apt-get install firmware-ipw2x00
(obsolete ipw2100-firmware and ipw2200-firmware ought to be purged)
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan iwlwifi-ucodeまたはiwlwifi-3945、firmware-iwl3945、firmware-iwl4965がすでにインストールしてあったら削除してください:
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54 MBit/s wlan: ファームウェアをhttp://prism54.org/firmware/からダウンロードして /lib/firmware/に保存します。
  • nVidia graphics: フリードライバーではまだ3Dアクセレーションはできません。 get-sidux-binary-gfx または sgfxi からスクリプトをダウンロードしてインストールしてください。
  • RaLink rt61 and rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    rt61-fwcutter、rt73-fwcutter、firmware-rt61またはfirmware-rt73がすでにインストールしてあったら削除してください。

  • Ricoh R5U870 OEM webcams:ファームウェアをhttp://mediati.org/r5u870/r5u870-0.11.0.tar.gzからダウンロードしてr5u870_*.fwに該当するファイルを/lib/firmware/にコピーします.
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan: 
    m-a a-i acx100
    ファームウェアをhttp://www.kazer.org/acx-firmware-20060207.tar.bz2からダウンロードして/lib/firmware/に保存します。
  • ZyDAS zd1201 11 MBit/s wlan: ファームウェアをhttp://surfnet.dl.sourceforge.net/sourceforge/linux-lc100020/zd1201-0.14-fw.tar.gzからダウンロードして/lib/firmware/に保存します。
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

合法的に頒布できるファームウェアはパッケージにしてこちらのnon-freeレポジトリに用意していますが、すべてのベンダーが許可してくれるわけではありません。

ntfs-3gを使ってNTFSパーティションに書き込む

警告: ntfs-3gは'stable'であると表示はされていますが,データの外部バックアップなしに使ったり、重要な仕事用のシステムに使うのは避けるべきです。 敢えてそのような使い方をしてデータを失っても自業自得ですから自分の責任で使うこと！

ターミナルを開き次のコマンドを入力します: ハードディスクのパーティション - ディスクの名前についてを参照

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
コンソールを終了するには: exit

これであなたのNTFSパーティションはrw(読み書きモード)でマウントされていて、データの書き込みができるるはずです。 警告を繰り返しますが これは緊急の場合に使うもので、日々の仕事につかうのにはお勧めできません！
