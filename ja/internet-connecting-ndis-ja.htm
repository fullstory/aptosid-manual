Ndiswrapperの有効化

siduxにはすでに必要なパッケージがすべてインストールされています: カーネルモジュールとndiswrapper-utils

apt-get update
apt-get install ndiswrapper-modules-2.6-sidux-i686 (or ndiswrapper-modules-2.6-sidux-amd64)
Do it yourself - 自分でやること

手持ちのカードに必要なドライバーを見つけ、次のコマンドでインストールします。:

ndiswrapper -i driver.inf

インストールがうまくいったかをチェックするには-lオプションを使います:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

ここで-lの出力に"driver installed"と"device present"が表示されているのを確認するのが大切です。 この両方が表示されていなければ何かがうまくいかなかったということです。 (-iオプションとdmesgをチェックします)、ここでの表示がうまくいかなければwlan接続はできません。

ndiswrapper-moduleのロード:

modprobe ndiswrapper

wifiデバイスが作成されたのを確かめるには::

iwconfig

アクセスポイントが見つかったかを確かめるには:

iwlist wlan0 scan

そしてカードの設定をするには:

ceni

カーネルをアップデートするごとにndiswrapperを再インストールする必要があります。 カーネルの番号を新しいカーネルのものに変えて実行してください:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

そしてmodprobeでロードします:

modprobe ndiswrapper

Windowsのドライバーはカーネルモジュールと一緒ではないので改めてインストールする必要はありません。

/etc/modulesにndiswrapperと書いておけば起動の際自動的にロードされます

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

KDEーStart-MenuからGUIのsiduxccコントロールセンターを使っても同じことができます。

Ndiswrapper上級編

ndiswrapperとはLinuxでWindowsのWLANドライバーが使えるようにするモジュールです。 これが必要になるのはいくつかのWLANカードのメーカーがオープンソースまたはフリーのドライバーを提供せず、Linux用のドライバーの実現を難かしくしているためです。 Ndiswrapperはこの問題をWindows用のドライバを"wrapping"して、カードのLinuxの間の入出力を変換することによって解消しています。

必要なドライバーの識別

まず第一にWLANカードに合った正しいWindows用のドライバーが必要です。 WLANカードには同じ名のモデルでも様々なチップセットが使われているため、正確に識別できるただ一つの方法はデバイスID(USBID/PCIID)を確認することです。

この'ID'を得るためにはターミナルから:

USBのWLANアダプターの場合
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

この例のD-Link Corp. [hex] DWL-122アダプターのIDは2001:3700です。

(mini)PCIまたはPCMCIAのWLANアダプターの場合
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

ここでは'ID'はメーカー:デバイスの形をとっています。 上の例では168c:0024がAtherosカードのIDです。

そして次に ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list の中からこの'ID'の項目を探します, 運がよければちょうどぴったりの項目と、このカードのために推薦されるドライバーが見つかるでしょう。

このドライバーのinfファイルとsysファイルが必要です。 もしくはzipファイルを入手してそこからunzipで展開します。 自己解凍型のEXEファイル(WINEを使って解凍）またはCABファイル(cabextractを使って解凍)を入手した場合はホームディレクトリの下に新しいディレクトリを作ってすべてのファイルをそこに置くのがいいでしょう。

Ndiswrapperの有効化とドライバーのインストール

まずWindowsのドライバーファイルのあるディレクトリに移ります:

bla@blubb:~$cd /folder/with/driver
bla@blubb:~$su
root@blubb:/folder/with/driver# ndiswrapper -i <driver.inf>
installing net5416

インストールがうまくいったかをチェックするには-lオプションで:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

ここで-lの出力に"driver installed"と"device present"が表示されているのを確認するのが大切です。 この両方が表示されていなければ何かがうまくいかなかったということです。 (-iオプションとdmesgをチェックします)、ここでの表示がうまくいかなければwlan接続はできません。

カーネルモジュールをロードするには: 

root@blubb:/some/folder# modprobe ndiswrapper

すべてが順調の時はなにも表示されません。 なにか出力があったときはdmesgをチェックして詳しい情報を探します。

iwconfigのコマンドでワイアレスデバイスが表示されるはずです:

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

ndiswrappperをシステム起動時に自動的にロードしたい場合は/etc/modulesにndiswrapperを加えます。 エディタで書き込むかまたは次のコマンドを使います: 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
カーネルアップデート後のndiswrapperを再インストール

After A kernel update you have to reinstall the ndiswrapper kernel module. 

apt-get update
apt-get install ndiswrapper-modules-2.6-sidux-i686 (or ndiswrapper-modules-2.6-sidux-amd64)

インストールが終わったらこのコマンドでロード:

root@blubb:/some/folder# modprobe ndiswrapper

Windowsのドライバーはカーネルモジュールと一緒ではないので改めてインストールする必要はありません。

ついでに

ネットワーク関連のデバイスを設定するには Ceniを使います。

使い方の詳細はこちら .

新しいドライバー、または別のドライバーを見つけたときはぜひそれもテストしてください。 なにか問題が(最悪の場合カーネルパニックが)起きるかもしれませんが、こういったことの積み重ねで様々なカードに最適のドライバーを選べるようになりdiswrapperプロジェクトに役立ちます。 テストがうまくいったときはリスト の追加/補完も忘れないように。
