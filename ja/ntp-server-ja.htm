タイムサーバーの設定

まずコンソールからrootとして

apt-cache search ntp
apt-get update && apt-get install  ntp ntp-doc
update-rc.d -f ntp defaults
その後あれこれの設定が済んだら
run update-rc.d

説明の文書もインストールされています。

/usr/share/doc/ntp-doc/html/index.html
ブックマークしましょう!

とても大きな文書で多くの話題を網羅しているのですべてを読む必要はありません。

ntpは再起動するまでスタートしませんが、再起動する前になるべく正確に時間を合わせるようにしましょう。

ntpは /etc/ntp.confの中のリストのサーバーと時刻を合わせます。 他のサーバーを選ぶにはこのファイルを変更します。

ntpdateコマンドとntpdデーモン(普通はntpと呼ばれます)の両方とも/etc/ntp.confのタイムサーバーのリストの上から順に現在時刻の同期を試みます。 リストの一例をあげます:

 pool.ntp.org maps to more than 100 low-stratum NTP servers.
# Your server will pick a different set every time it starts up.
#  *** Please consider joining the pool! ***
#  ***  http://www.pool.ntp.org/#join  ***
server 192.168.3.24
server ntp.blueyonder.co.uk
server uk.pool.ntp.org
server 1.uk.pool.ntp.org
server 2.uk.pool.ntp.org
server 0.europe.pool.ntp.org
server 1.europe.pool.ntp.org
server 2.europe.pool.ntp.org

一番上は同じネットワーク上でntpが動いているコンピュータです。

二つめはあなたが接続しているISPのタイムサーバーです。

その後に続くのはイギリスとヨーロッパ大陸のタイムサーバーのいくつかです 運がよければこの設定のままでも使えるかもしれません。 ところで、多くの場合ISPのネームサーバーがタイムサーバーも兼ねています。 これを確かめるには:

ntpdate -v

このコマンド自体は設定の変更をしませんが、次の例のような出力が表示されます:

# ntpdate -v 192.168.3.24
19 Sep 19:09:27 ntpdate[13329]: ntpdate 4.2.2@1.1532-o Wed Aug  9 12:08:54 UTC 2006 (1)

世界のntpサーバー一覧はこちら http://www.pool.ntp.org/

このコンピュータに他のコンピュータからのntpアクセスを許可するには

# Local users may interrogate the ntp server more closely.
restrict 127.0.0.1 nomodify
restrict 192.168.3.0/24

ブロードキャスト設定:

# 同じLANにあるコンピュータに現在時刻を送るには次の行を編集します。
# (アドレスは一例にすぎません。)
broadcast 192.168.3.255

ntp.confファイルの取扱いはちょっと変わっていて、クリックしただけでは現行の時刻との時間差だけが扱われるため、初めてntpをスタートする前に一度次のようにコマンドから時刻の設定を行います。

# ntpdate -u -b uk.pool.ntp.org
19 Sep 19:19:33 ntpdate[15641]: step time server 62.3.200.116 offset 0.001523 sec

その後で(再)起動の度にntpがスタートするように設定します。 しばらくサーバーとして使ってみてから:

ntpq -pn

を実行すると、ちゃんと作動している場合つぎの例のような表示があります:
