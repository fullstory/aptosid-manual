smxiスクリプトのコマンドシンタックス

sidux コミュニティーに支えられた機能豊富なシステム管理ツール smxi (sidux-maintenanceの略）は次のスクリプトでインストールできます:

smxiをインストールするには:

cd /usr/local/bin; wget techpatterns.com/smxi.zip; unzip smxi.zip && smxi
そして英語の指示にしたがってください。

3Dドライバをインストールするスクリプトだけをインストールするには

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
オプションを表示するには:
sgfxi -h

smxiスクリプトの特色と基本的な機能

このスクリプトにはいろいろな機能があります。初めて実行されたときだけユーザーの目に触れるものもあれば、実行のたびに表示されるものもあります。 最初にシステムチェックといくつかの質問がありそれから核心の部分がはじまります:

  • スクリプトの情報 このセクションでは作動中のカーネルのバージョン、最後にこのスクリプトがアップデートに使われた日時などの情報が表示されます。 この情報はあまり頻繁にアップデートされないシステムで現状を確認するのに役に立ちます。 (一ヶ月に一度はアップグレードしましょう。 それ以上だと問題が起こりやすくなります)
  • カーネルインストール。 ここでは現行のカーネルをデフォルトに、上級オプションとしてそれ以外の以前のカーネルの自動インストールの選択肢があります。 smxiを-Kオプションでスタートすると現行のテストカーネルがあったらそれを第一の選択として表示します。
  • dist-upgradeの警告セクション 現在起こっている、または最近起こったsiduの問題についての警告をライブで表示します。 (sidはまったく問題のない青信号の状態になることは滅多にありません。 大抵は安全ではあるが問題の解決法も用意された黄信号の状態です。)
  • dist-upgrade前のフィックス, 必要があれば実行されます。
  • 設定ファイルについての注意 設定を以前の状態で保持するか、書き換えるかの質問があります。普通はnでノーと答えますが、yでイエスと答えれば設定が維持されます。
  • dist-upgradeセクション, アップデートがうまくいかなかった時の問題解決コマンドも用意されています。
  • dist-upgrade後のフィックスも必要があれば実行されます。
  • 種々のオプション ここには:
    • フィックスフォント - フォントをフィックスするオプションです。 インストール後は役に立ちますが、その後は滅多に必要としません。
    • パッケージインストール - よく使われるパッケージのグループをインストールするオプションです。 non-freeのフラッシュプレーヤーやグーグルアース、オフィスのインストーラなど。 ノート: OpenOffice.orgのインストーラは言語パックその他のあまり知られていない便利なユーティリティのインストールもサポートしています。
    • パッケージの除去 - Wifiやドイツ語のサポートなどのパッケージグループで、あなたには必要のないものがあったらをシステムから取り除くオプションです。
    • クリーンアップ - いらなくなったカーネルの整理などさまざまなクリーンアップオプションがあります。 (ノート: 将来の問題に備えて以前のカーネルを最低一つは消さずにとっておきましょう。)
    • 種々の微調整 - 人気のMozillaやファイルハンドラーの設定など、時とともに変わっていくオプションです。 時々チェックしてください
    • 続き - グラフィックドライバのインストールにはいります。
    • 終了 - これ以上smxiを使ってやることがなかったらそこから先を続ける必要はありません。 ノート: グラフィックドライバのインストールが終わるとそこからinit 5に入ってKDEをスタートするオプションがあるのでそこからデスクトップに戻ることができます。
  • グラフィックインストーラ - このセクションはnvidiaやATIのfglrxといった企業ドライバのインストールを行いますが、(ATIはここ一年ほど不安定ですが、たいていは使えます)他にXorgのドライバのインストールやXのデスクトップをスタートするオプションもあります。

smxiスクリプトpオプションの使い方

オプションを全部一緒に: 

-deghiIklmrstuwx

または一つ一つ別個に: 

-giktw -f -x

オプションはどんな組み合わせでも使えます。

オプションを-ik のように一まとめに書いてもよく、それとも -w -x のように一つ一つ別々に使っても構いません

オプションの一つ一つ、またはまとめたグループは:

 -

の文字に続きます

例:

 smxi -ikw

システム情報とカーネルインストールと警告セクションをスキップします

smxiスクリプトオプション: スクリプトの一部をとばす

次のオプションでスクリプトの一部を省略できます:

  • -d -wオプションと一緒に選ばれるとdist-upgradeを省略。
  • -e 'Please hit enter to continue'(Enterを押してください)を省略。
  • -g グラフィックインストーラを省略。
  • -i 最初の情報セクションを省略。
  • -k カーネルインストールを省略。
  • -r スクリプトの最新バージョンの確認、ダウンロードと再スタートのセクションを省略。 普通のユーザーにはお勧めできませんが、たとえばsmxiでカーネルをインストールしたあと再起動してまたsmxiを実行する場合などはもう一度最新バージョンを確認する必要はありません。
  • -s インターネットへの接続を必要とする部分をすべて省略。 ほとんどすべての機能が省略されてしまうのであまり意味がありません。
  • -S インターネットへの接続の確認を省略。 （ちゃんと接続されていることがわかっているのに接続テストがうまくいかない時などに使えますがいつもうまくいくとはかぎらないのであまりお勧めできません)
  • -t dist-upgrade後のオプションを省略。
  • -w dist-upgradeの警告セクションを省略。

一例:

smxi -kwid

このオプションだとdist-upgradeが終わるとすぐに種々のオプションのセクションに移ります。 smxi -ki だと最初の情報セクションとカーネルインストールを省略します。

smxiの上級オプション

これから紹介するのは上級オプションです。 めったに使うことはありませんが、ときには役に立つこともあるでしょう:

  • -I Xの再始動のInitレベルを変更する: -I 3のようなシンタックスで指定します
    ノート: この番号はXをスタートするinitレベルの番号です。
  • -K 上級のカーネルインストール。 もし新しいテストカーネルがあったらこれをインストールオプションとして表示する。 テストカーネルのインストールは自分のリスクと責任で行ってください!
  • -l すべてのアップデートとdist-upgradeについて時刻付きのログを書く。 ログファイルの場所は /var/log/smxi/
  • -m /etc/apt/sources.listに設定されているDebianのミラーを変更する
  • -M カーネルをダウンロードするデフォルトミラーを変更する
  • -p apt-get update をpdiffを使わないように実行する。 早いネット接続のとき便利です。
  • -P Proxy設定のネット接続をつかって実行する。 シンタックスは:
    -P http://[<username><:password>@]<server><:port>
  • -U スクリプトを更新して再スタートする。
  • -x スクリプトの核となる部分以外の機能を実行する。
  • -h smxiスクリプトのHelpメニューを表示する。
  • -v スクリプトのバージョン、ディストロバージョン、作動中のカーネルのバージョン、最後に apt-get update や apt-get dist-upgrade が行われた日時、最後にsmxiを実行した日時などのシステム情報を表示する。

ユーザーが設定するsmxiスクリプトオプション

/etc/smxi.confという設定ファイルを編集すればこのスクリプトの働きを変更できます

初めてsmxiが実行されるまではこのファイルはありません。 初めての実行のオプションを設定したい場合はこのファイルを新しく作ってください。

  • sgfxi=<sgfxiのオプションでグラフィックインストーラの標準オプションではないもの>, たとえば次の行を/etc/smxiに加えると:
    sgfxi=b

    nVidiaドライバのインストールでは標準の -cオプションに加えてさらに t オプションが使われます.
    このオプションはsgfxiスクリプトがサポートするオプションで、smxiのオプションではありません。 smxiはグラフィックインストールでsgfxiを実行するときに単にこのオプションを加えるだけです。 この方法ではたとえば -o 8.33.6 のようにパラメータとして追加の情報が必要なオプションは設定できません。 ただ一文字だけで足りるオプションだけです。
  • hold-install=< ^>マークで区切られたパッケージ名のリスト - このオプションでsmxiスクリプトのホールド/リリース機能を使って一貫性のないパッケージなどを一時的な問題を避けるためアップグレードの対象からはずします。 この設定ファイルに書いておくだけでうっかりしてアップグレードしてしまうことを避けられます。 例:
    hold-install=nano^libgtk2.0
    ノート: 重要: スペースは入れずに^の文字だけをパッケージ名の区切りに使います!

sgfxi

To install closed source 3d drivers:

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
To see options:
sgfxi -h

sgfxiスクリプトは今のところATIのfglrxとnVidiaのドライバーをサポートしています。 Xorgのatiやnvといったフリードライバーとの交換もサポートしています。 smxiスクリプトはsgfxiをグラフィックドライバーのインストールエンジンとして使っていますが、いくつかのオプションの選択肢を与えているだけです。

パラメータなしでスクリプトを実行するとデフォルトのドライバーがインストールされます。 古いカードを使っていて過去のドライバーをインストールする必要がある場合をのぞいてはスクリプトが自動的に適切なドライバーを選択します。

スクリプトを実行すると、一旦ストップして、これからどのドライバーをインストール使用としているかを告げます。 これを受け入れるか、またはここで実行を停止して別のオプションで他のドライバーを指定することもできます。

のsmxiスクリプトからグラフィックドライバーをインストールすることもできます。 smxiからsgfxiが実行されます。 詳しくはまず smxi ホームページ を読んでからこのコマンドを実行してください。

smxi -kiwdt

sgfxi-h: -hオプションを使えば今サポートされているドライバの表と、合わせてカーネルやXorgのバージョンやシステムについての情報をを表示します。 sgfxiのオプションをいくつか紹介します。

-c コンポジットモードで使う (nVidiaカードのみ)
-d IgnoreDisplayDevicesオプションを取り除く, TVも選べるようになる (nVidiaカードのみ)
-h ヘルプメニューの表示
-n Xorgのnv, intel, またはatiのドライバーからシステムに適合するものを自動的に選択してインストール
   古いバイナリードライバーのファイルを消し、xorg.confからnVidiaやfgrlxについての情報を取り除き、
			新しいドライバーの情報を書き込む
-N 特定のバージョンのドライバーをインストールする
   シンタックス: sgfxi -N <リストから選んだドライバー, 例： -N radeon>
   サポートされているXorgドライバー: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   xorg.confの必要な設定も自動的に書き込みます。 Radeon Xorgの3D設定も自動的に行います
-o デフォルトの代わりに特別なドライバーをインストールする、絶対的なドライバー指定する. 
   シンタックス:
   -o 9746 (nVidiaのドライバーでは1.0-の部分は書きません) または: -o 8.32.5 (fglrx)
-r xorg.confからモードに関する記述を削除する
-t ダイナミックツウィンヴューを有効にする (nVidiaのみ)

オープンソースのドライバに戻すにはs

nVidia, ATI, Radeon, Intel用のXorgのオープンソースドライバはsiduxに標準装備されています。 クローズソースのドライバのインストールの際にオープンソースドライバを消してしまっている場合は sgfxi を使って元に戻すことができます。まず

cd /usr/local/bin && wget -Nc http://techpatterns.com/downloads/distro/sgfxi && chmod +x sgfxi && sgfxi
オプションを表示するには:
sgfxi -h

Xorgのドライバにもどすには

sgfxi -n
または
sgfxi -N <Xorg driver>

Key commands are:

-n ハードウェアに適合したXorgのnv、intel、またはatiドライバーを自動的にインストールします。
   古いバイナリードライバーのファイルを消し、xorg.confからnVidiaやfgrlxについての情報を取り除き、
   新しいドライバーの情報を書き込む
-N 特定のバージョンのドライバーをインストールする
   シンタックス: sgfxi -N <リストから選んだドライバー, 例： -N radeon>
   サポートされているXorgドライバー: 
    ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   xorg.confの必要な設定も自動的に書き込みます。 Radeon Xorgの3D設定も自動的に行います。
コンソールについて知っておきたいこと

ターミナルウィンドウで上下にスクロールしたいときはShift+PgUp/Shift+PgDownを使います。 流れて読めなくなってしまった表示などを遡って読むことができますが、ふつう6スクリーンくらい前までという制限があります
