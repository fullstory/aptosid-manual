Enabling services in sidux

insserv (Pontos 2008-04 and forward ): To start/stop already installed services:

Read /usr/share/doc/insserv/README.Debian, the release notes and the man pages carefully:

 $ man insserv 
 $ man invoke-rc.d 
 $ man update-rc.d 
 google LSB headers

All access to the init scripts by Debian packages’ maintainer scripts should be done through invoke-rc.d.

For 'start':

invoke-rc.d <service> start

For 'stop':

invoke-rc.d <service> stop

For 'restart:

invoke-rc.d <service> restart

To prevent services running at-startup

update-rc.d <service> remove
[will remove all startup links]

To make sure a service runs at boot [not always required]

update-rc.d <service> defaults
[will make startup links]

Older Installs (before Pontos 2008-04)

insserv on installed systems before Pontos 2008-04

警告: 古いインストールには ユーザーの initscript やシンボリックリンクなどに適応できないため、insserv は自動的に設定されません。Pontos 以前にインストールされたシステムに insserv をインストールすることは推奨しません。このパッケージの使用には注意が必要で、既存の initscript に LSB 依存関係の間違いや欠如があるとシステムが起動できなくなるおそれがあります。古いシステムに insserv をインストールするにはいくつかの作業が必要です。

insserv をインストールし、有効二する前に必ず Debian documentation notesとリリースノートを読んでください。

インストールされているサービスはdist-upgradeの度にアップデートされます。 dist-upgradeの前にはスタートしないように設定されたサービスもdist-updateのあとには自動的にスタートするように設定されます。

サービスを単発でスタートしたり、起動時にサービスが自動的にスタートするように、またはしないように設定するには

invoke-rc.d service start

または

rcconf

rcconfのノート: カーソルを上下に移動するにはUp/Downキー、選択/非選択はSpace、TabでOK(またはCancel)をハイライトにしてEnterで設定を有効にして終了

sidux Control Centreを使って設定することもできます。

サービスまたはプロセスを強制終了(kill)するには

pkillは読みやすい出力の便利なコマンドで、ターミナルまたはttyからユーザーとしてでもrootとしてでもどちらでも使えます。

pkill -n service

強制終了したいサービスまたはプロセスの名前に自身がない場合は pkill <tab> <tab> でリストを表示します

代わりにhtopを使うのもいいアイデアです。 (どうにも困ってしまったらkillall -9も使えます)。

siduxのランレベル - init

ここにsiduxオペレーティングシステムのランレベルの一覧を載せます。 Debian stableとは違っていることに注意してください:

  • init 0 PCの電源off
  • init 1 シングルユーザーモード
  • init 2 ネットワークなしのマルチユーザーモード、Xのストップまたはスタートしないときに
  • init 3 ネットワークありのマルチユーザーモード、XをストップまたはXをスタートしないときに
  • init 4 XをストップまたはXをスタートしないとき
  • init 5 ネットワークありのマルチユーザーモードでXウィンドウシステムが作動、Xをスタートするコマンド
  • init 6 再起動

失ったパスワードを回復することはできませんが、新しいパスワードを設定することはできます。

まずはライヴCDから起動。

rootとして問題のrootパーティションをマウントする(例として /dev/hdb2)

# mount /dev/hdb2
問題のrootパーティションにchrootします
# chroot /media/hdb2 passwd

siduxのフォント

正しいDPI設定のしかた - 理念

DPIの設定を推測するのは容易ではありませんが、/etc/X11/xorg.confの"Monitor Section"に実際のスクリーンのサイズを書いてあればXは自動的に正しい設定を選びます。

正しい解像度とリフレッシュレート設定

個々のモニターにはそれぞれに最適の組み合わせがありますが、残念なことにすべてのモニターが正しいDCC値をリポートするわけではないため、時にはユーザーが設定する必要もあります。

正しいグラフィックアダプタードライバ

新しいATIやNvidiaのカードはフリーのXorgドライバではうまく作動しないため、そいうい場合はメーカーのクローズドソースドライバを使うことが唯一の合理的な解決法です。 siduxは法律的な理由でそういうドライバを自動的にインストールしません。 そういう場合の解決法はこちらをご覧ください

デフォルトのフォント、レンダリングとサイズ

siduxはDebianで選ばれて非常にバランスのいいことが実証されたフリーフォントを使っています。 ユーザーの趣味で他のフォントを入れるとレンダリングの質が落ちることがあります。 そのような時のために(kcontrolから選ぶことの他にも)いくつかの強力なオプションがDebianに用意されていて他のフォントも美しく描写することができます。 ただ、どのフォントでも完璧に表示できるサイズは限られていて、他のサイズにはうまく対応できないことを考慮してください。

DPIサイズを設定するコマンドでちょっと遊んでみるのもいいかもしれません:

fix-dpi-kdm

このコマンドはあなたのスクリーンサイズのDPIを表示しますが、このコマンドでしばらく遊んでみることを勧めます。 新しい設定を見るためには一度init 3にしてまたinit 5にもどりXを再スタートする必要があります。 再起動でも同じ結果がみれます

(XまたはFirefox/Iceweaselの)フォントを変更した後で、とくにBitmapフォントとTruTypeフォントを取り替えた後など、きれいなフォントを表示するために若干の調整が必要な場合があります。 次のコマンドを実行します:

dpkg-reconfigure fontconfig-config

Nativeとautohinter on automaticを選びましょう。 それともいろいろ試して結果をみるtのもいいアイデアです。

それでもダメなときは次のコマンドでフォントキャッシュを作り直します:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Firefox/Icewaselなど、GTKを使っているアプリケーション

GTKを使ってるアプリケーションは一般にKDEのデフォルトと相性がよくありません。 この問題はつぎのコマンドで解決できます:

apt-get install gtk2-engines gtk-qt-engine

KDEのコントロールセンターにgtkのメニューが追加されているはずです。 ここでclearlooksとKDEフォントを使うように設定します。

過去にgtk-qt-engineのためシステムが不安定になる問題がありました。 2006年以降にはこの問題は起きていませんが、上の方法を試してシステムがおかしくなった場合はgtk-qt-engineをromoveまたはpurgeしてください。 設定は$HOMEディレクトリに残りますから依存関係のために設定が消えることはありません:

apt-get remove gtk-qt-engine
または
apt-get remove --purge gtk-qt-engine

上記の方法でgtkのアプリケーションのフォントを修正できる*かも*しれない、というだけです

CUPS

KDEにはCUPSについて膨大なヘルプがありますが、dist-upgradeによってCUPSがうまく作動しなくなることがあります。 そういうときは:

modprobe lp
echo lp >> /etc/modules 
apt-get remove --purge cupsys cups
apt-get install cups
	または
apt-get install cups cupsys-driver-gutenprint hplip

その後ブラウザで:

http://localhost:631

CUPSがスタートしているのを確認します。 :

/etc/init.d/cups restart

もう一つGUIでCUPSの設定を行う場合に小さい落とし穴があります。 ダイアログが開き、ユーザー名が表示されパスワードの入力を待ちますが、ここにユーザーのパスワードを入れてもうまくいきません。 ここではユーザー名のかわりにrootとタイプしてrootのパスワードを入力しないといけません。
