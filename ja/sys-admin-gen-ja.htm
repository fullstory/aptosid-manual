sidux システム上でサービスを使うには

insserv (Pontos 2008-04 以降のリリース): すでにインストールされているサービスをスタート/ストップするには:

/usr/share/doc/insserv/README.Debianと insserv の man ページを注意深く読んでください:

 $ man insserv
 $ man invoke-rc.d
 $ man update-rc.d
 google LSB headers

スタートするには:

/etc/init.d/<service> start

ストップするには:

/etc/init.d/<service> stop

For 'restart:

/etc/init.d/<service> restart

起動の際に自動的にスタートしないようにするには:

update-rc.d <service> remove
[このサービスのスタートアップリンクがすべて削除されます]

起動の際に必ず自動的にスタートするようにするには [このコマンドは必要とは限りません]

update-rc.d <service> defaults
[このサービスのスタートアップリンクが作成されます]

To read the list of what services are running:

ls /etc/rc5.d

S means the service will start.
K means service will not start.

Pontos 2008-04 以前のリリース

Pontos 2008-04 以前のシステムでも insserv がインストールされていたら前の項目にしたがってください

警告: 古いインストールには ユーザーの initscript やシンボリックリンクなどに適応できないため、insserv は自動的に設定されません。Pontos 以前にインストールされたシステムに insserv をインストールすることは推奨しません。このパッケージの使用には注意が必要で、既存の initscript に LSB 依存関係の間違いや欠如があるとシステムが起動できなくなるおそれがあります。古いシステムに insserv をインストールするにはいくつかの作業が必要です。

insserv をインストールし、有効にする前に必ず Debian documentation notesとリリースノートを読んでください。

ブートアップマネージャ(bum) - サービス設定用のGUIツール

Debian のブートアップシーケンスの論理によく親しんでいない人はシンボリックリンクやパーミッションなどを気軽にいじったり、実験したりするべきではありません。システムが滅茶苦茶になる恐れがあります。ブートアップマネージャを使って設定を自動的に行った方が安全です。

ブートアップマネージャはランレベル設定のGUIエディタで、システムの起動の際にどのサービスをスタートするかなどの設定ができます。起動時の自動スタートが可能なサービスの一覧が表示されて、個々のサービスのOn/Off設定ができます。

apt-get install bum

ブートアップマネージャを使うには:

$ sux
password:
bum

ブートアップマネージャ(bum)の詳細はこちらへ.

サービスまたはプロセスを強制終了(kill)するには

pkillは読みやすい出力の便利なコマンドで、ターミナルまたはttyからユーザーとしてでもrootとしてでもどちらでも使えます。

pkill -n service

強制終了したいサービスまたはプロセスの名前に自身がない場合は pkill <tab> <tab> でリストを表示します

代わりにhtopを使うのもいいアイデアです。 (どうにも困ってしまったらkillall -9も使えます)。

siduxのランレベル - init

ここにsiduxオペレーティングシステムのランレベルの一覧を載せます。 Debian stableとは違っていることに注意してください:

  • runlevel 0 powers off the PC.
  • runlevel 1 single user (safety mode). Dæmons like apache and sshd are stopped. Do not go to this level via remote access.
  • runlevel 2 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows
  • runlevel 3 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows (this is where dist-upgrades are actioned).
  • runlevel 4 Multi-User mode with the network running, with X-windows not running, and/or to stop or not enter X-windows
  • runlevel 5 Multi-User mode with the network running the X-windows system, and/or to start X-windows.
  • runlevel 6 reboot.
  • runlevel S where early boot time services are executed on a 'once only basis'. You cannot switch to it after it has been run.

rootのパスワードを忘れてしまったら

失ったパスワードを回復することはできませんが、新しいパスワードを設定することはできます。

まずはライヴCDから起動。

rootとして問題のrootパーティションをマウントする(例として /dev/sdb2)

mount /dev/sdb2 /media/sdb2

							
問題のrootパーティションにchrootします and set a new password:


chroot /media/sdb2 passwd

siduxのフォント

正しいDPI設定のしかた - 理念

DPIの設定を推測するのは容易ではありませんが、/etc/X11/xorg.confの"Monitor Section"に実際のスクリーンのサイズを書いてあればXは自動的に正しい設定を選びます。

正しい解像度とリフレッシュレート設定

個々のモニターにはそれぞれに最適の組み合わせがありますが、残念なことにすべてのモニターが正しいDCC値をリポートするわけではないため、時にはユーザーが設定する必要もあります。

正しいグラフィックアダプタードライバ

新しいATIやNvidiaのカードはフリーのXorgドライバではうまく作動しないため、そいうい場合はメーカーのクローズドソースドライバを使うことが唯一の合理的な解決法です。 siduxは法律的な理由でそういうドライバを自動的にインストールしません。 そういう場合の解決法はこちらをご覧ください

デフォルトのフォント、レンダリングとサイズ

siduxはDebianで選ばれて非常にバランスのいいことが実証されたフリーフォントを使っています。 ユーザーの趣味で他のフォントを入れるとレンダリングの質が落ちることがあります。 そのような時のために(apart from KDE>systemsettings )いくつかの強力なオプションがDebianに用意されていて他のフォントも美しく描写することができます。 ただ、どのフォントでも完璧に表示できるサイズは限られていて、他のサイズにはうまく対応できないことを考慮してください。

DPIサイズを設定するコマンドでちょっと遊んでみるのもいいかもしれません:

fix-dpi-kdm

このコマンドはあなたのスクリーンサイズのDPIを表示しますが、このコマンドでしばらく遊んでみることを勧めます。 新しい設定を見るためには一度init 3にしてまたinit 5にもどりXを再スタートする必要があります。 再起動でも同じ結果がみれます

(XまたはFirefox/Iceweaselの)フォントを変更した後で、とくにBitmapフォントとTruTypeフォントを取り替えた後など、きれいなフォントを表示するために若干の調整が必要な場合があります。 次のコマンドを実行します:

dpkg-reconfigure fontconfig-config

Nativeとautohinter on automaticを選びましょう。 それともいろいろ試して結果をみるtのもいいアイデアです。

それでもダメなときは次のコマンドでフォントキャッシュを作り直します:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Firefox/Icewaselなど、GTKを使っているアプリケーション

GTKを使ってるアプリケーションは一般にKDEのデフォルトと相性がよくありません。 この問題はつぎのコマンドで解決できます:

apt-get install gtk2-engines system-config-gtk-kde gtk-qt-engine gtk2-engines-qtcurve

System Settings >Appearance のメニューに新しくGTK Styles and Fontsが入りました。 'GTK Styles' に 'Clearlooks' を選び、'GTK Fonts' を'KDE fonts' に設定します。もしくは 自分で色々なオプションを試してみてください。.

上記の方法でgtkのアプリケーションのフォントを修正できる*かも*しれない、というだけです

CUPS

KDEにはCUPSについて膨大なヘルプがありますが、dist-upgradeによってCUPSがうまく作動しなくなることがあります。 そういうときは:

modprobe lp
echo lp >> /etc/modules
apt-get remove --purge cupsys cups
apt-get install cups
	または
apt-get install cups cups-driver-gutenprint hplip

その後ブラウザで:

http://localhost:631

CUPSがスタートしているのを確認します。 :

/etc/init.d/cups restart

もう一つGUIでCUPSの設定を行う場合に小さい落とし穴があります。 ダイアログが開き、ユーザー名が表示されパスワードの入力を待ちますが、ここにユーザーのパスワードを入れてもうまくいきません。 ここではユーザー名のかわりにrootとタイプしてrootのパスワードを入力しないといけません。

Sound in sidux

Sound is muted by default in sidux.

The KDE version uses Kmix and XFCE uses Mixer.

Often it is only a matter of clicking on the sound icon in the taskbar and unchecking the 'Mute box'.

Kmix

In Kmix you will need to activate the options you prefer for channel options, Kmix>Setting>Configure Channels. Or in a terminal:

$ kmix
XFCE

In XFCE run the mixer application and add some controls via the Multimedia>Mixer and click the Select Controls box. Or in a terminal:

$ xfce4-mixer

Alsamixer

If you prefer to use Alsamixer, it is in the alsa-utils package:

apt-get update
apt-get install alsa-utils
exit

Set your preferred sound settings as $user from the terminal:

$ alsamixer

Then as root:

alsactl store

See also the sidux wiki.
