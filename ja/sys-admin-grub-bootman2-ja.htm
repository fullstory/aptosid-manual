To rewrite Grub

新しいバージョンのGrubでGrubローダーをハードMBRに書き込むには:

/usr/sbin/grub-install --recheck --no-floppy /dev/sda

このコマンドは、本当に実行したいということをプログラムが納得してくれるまで何度か実行する必要があるかもしれません。

grubの復旧 または MBRがWindowsのブートローダーに上書きされてしまったとき

GrubをMBRに書き直したり、またはGrubを修復したりするには、まずsiduxのライブCDからブートすることが必要です:

  1. Grubをインストールするパーティション([h,s]d[a..]Xなどの名前)を確認するには、次の様なコマンドをrootとして実行します。まずroot(#)になるために: 
    $ sux
  2. rootとして次のコマンドを実行します: 
    fdisk -l
cat /etc/fstab
    この中で / のパーティションの名前を見つけてください。
  3. パーティションの名前が確認できたら、まずマウントポイントを作成します: 
    mkdir -p /media/[hdxx,sdxx,diskx]
  4. そしてマウントします: 
    mount /dev/xdxx /media/xdxx
  5. 最初のハードディスクのMBRにGrubを再度書き込むコマンドの例: 
    /usr/sbin/grub-install --recheck --no-floppy --root-directory=/media/xdxx /dev/sda

MBRからgrubを消すには

簡単です: ターミナルからrootとして

lilo -M /dev/hda -s /dev/null

を実行するだけです。

ブートマネージャGRUBとは

概説

ノート: Grubで2.6.22以降のカーネルを起動するコマンドが変更され、以前の
initrd (hd0.x)/sidux/miniroot.gz
は新しく
initrd		/boot/initrd.img-2.6.xx.x-slh-smp-x
になりました。

Grubは様々なOSをそれぞれのパーティションから起動できるブートマネージャです。 　Grubはシステムの起動に必要なファイルを見つけるのにブロックリストに依存することなく、多くのファイルシステムを直接読むことができます。 デフォルトで読めるファイルシステムに: ext2, ext3, reiserfs, ... (Linux), Fat32, Fat16 (Windows), UFS (BSD), Minix その他いろいろ

GrubはBIOS-extensionsを認識し(つまり8Gbめより上のパーティションも起動できる!)1024iシリンダーの壁もありません。 Grubは実際どこにカーネルのファイルがあるかは一切気にせず、パーティションの宣言とと起動すべきカーネルへのパス(シンボリックリンクでもOK)だけが必要です。 設定ファイル(menu.lst)が変更されても、Liloのように一度プログラムを実行する必要なく、新しい設定は次回のブートから有効になります。

siduxはライブCDからの起動にも、HDにインストールされた時にもGrubをデフォルトのブートマネージャとして使っていますが、Grubは後からでも問題なくインストールすることができます。 GrubをMBRにインストールするにはrootとして次のコマンドを実行します:

grub-install --root-directory=/media/hdaX /dev/hdY

このコマンドで/media/hdaX と書かれているのは、マウントされ、読むことができ、そこの"/boot/grub"のディレクトリに起動に必要なファイル(stage2, stage1_5xxx)が置かれるパーティションの名前です。 同じ場所に"menu.lst"も用意されていなければなりません。 /dev/hdY はMBRにGrubをインストールしようとしているハードディスクで、普通なら/dev/hdaでしょう。

ドライブネームとデバイス名

注意: hda という名は使われなくなりつつあります。 そのため2.6.22以降のカーネルではhda をsda に変えていく必要があります。

GrubはSCSIとIDEのハードディスクを区別しません。 (hd0)はいつでも（BIOS設定の)ブート順で最初のハードディスクで、IDEかSCSIかを問いません。 数え方は変わっていて、最初のハードディスクの最初のパーティションは(hd0,0)と呼びます。 プライマリパーティションは０から３、ロジカルパーティションは４以降の番号で呼ばれます。

Grub Linux Label
(fd0) /dev/fd0 フロッピー
(hda0) /dev/hda HD no.1
(hd0,0) /dev/hda1 hd1のパーティションno.1
(hd0,5) /dev/hda6 hd1の拡張パーティション

Grubのためのデバイスネームはカッコ()でくくります。 カーネルの行にあらわれるデバイス名に注意してください。 Grubとは対照的にカーネルはデータとして標準のlinux式のデバイス名が必要です。 下の例でにでてくるデバイス名は同じパーティション、つまり最初のハードでぃすの四番めのパーティションを指します。 一行目はGrubの書き方で、二行目は典型的なlinuxの書き方が使われています

root (hd0,3)
kernel          /boot/vmlinuz root=UUID=bf955abf-ce1f-45a3-9ee5-93b5c5dc4fe0 ro quiet vga=791
initrd          /boot/initrd.img

コマンドモード (Grubシェル)

とても便利な機能として、コマンドモードがあります。 (Escでメニューから抜け出て）"c"をタイプすることでGrubシェルが現れます．たとえmenu.lstに間違いがあったとしてもここから他のシステムのカーネルを起動することができます。

ちょっとしたテストを行うためなら、起動中のシステムのrootシェルからでもGrubを実行できます。 Grubシェルから"help"とタイプするだけですべてのコマンドを見ることができます。 それぞれのコマンドについてのもっと詳しい説明はhelp command", で見ることができます。 Linuxシェルにもどるには"quit"とタイプすることでGrubシェルを終了します。 GrubシェルではキーがLinuxシェルのように機能して入力を手伝ってくれます。 コマンド、ドライブ名、ファイル名などがキーを押すことで自動的に補填されます。

Grubの"find"コマンドを使えば、OSを実際に起動する前にカーネルやinitrdファイルを見つけることができます。 次のコマンドを実行します

find /boot/vmlinuz

するとGrubは"/boot"ディレクトリに"vmlinuz"という名のカーネルファイルがあるパーティションを次のように表示します:

(hd0,3) (hd1,0)

"geometry"コマンドを使えば利用できるすべてのパーティションを表示できます。

geometry (hd0)

を実行するとたとえば次のように表示されます:

"drive 0x80: C/H/S = 3648/255/63, The number of sectors = 58605120, /dev/hda Partition num: 0,
Filesystem type is fat, partition type 0xb Partition num: 1,
Filesystem type is fat, partition type 0xb Partition num: 2,
Filesystem type is fat, partition type 0xc Partition num: 4,
Filesystem type is ext2fs, partition type 0x83 Partition num: 5,
Filesystem type unknown, partition type 0x82 Partition num: 6,
Filesystem type is fat, partition type 0xb"

カーネルまたはinitrd(またはminiroot.gz)が見つかったら、"boot"コマンドでそこから起動できます。

このモードでは"menu.lst"の内容を起動の際に変更することができます。 これはCDから起動される場合や、"menu.lst"が書き込み禁止になっていても実行できます。 このモードに入るにはまずキーを押し、Returnのあと"e"キーを押します。 そのままタイプすれば行の最後にパラメータが入力されますが、カーソルを前に移動すれば行の途中を編集することももできます。

Grubから複数のOSを起動するのはchainloadを使えばとても簡単にできます。 たとえばMS Windows(TM)とDebian etchとsiduxのfromisoインストールを起動したかったらGrubファイル/boot/grub/menu。lst の### END DEBIAN AUTOMAGIC KERNELS LISTの行の下に次のように付け加えます:

####MS Windows (TM) perhaps even MS Vista (TM)
Title Windows (95/98/XP/Vista
rootnoverify (hd0,1)
makeactive
chainloader +1

### Debian etch
title Debian etch
kernel (hd0,3)/debian/vmlinuz root=/dev/hda4 ro (or whatever)
initrd (hd0,3)/debian/miniroot.gz (or (whatever)

### ISO boot
title sidux 32bit from gaiaISO
kernel (hd0,3)/sidux3/vmlinuz-2.6.22.1-slh-smp-7 boot=fll quiet vga=791 fromiso=/sidux3/siduxgaia.iso
initrd (hd0,3)/sidux3/initrd.img-2.6.22.1-slh-smp-7

いくつかのMS Windows(TM)を一つのPCにインストールしたい人もいるでしょう。 ところがMS Windows(TM)はいつでも最初のドライブから起動したがるため、他のディスクから起動するには先行するパーティションを隠したり、別のパーティションをアクティブにしたりする必要があります。 そのためつぎのようなコマンドをまず試しにGrubのコマンドシェルから実行してみて、うまくいったらmenu.lstにコピーします。 これはWindowsのパーティションごとに行う必要があります。 データのバックアップも忘れずに:

hide (hd0,0)         # disk1のpartition1を隠す(不可視モード)
unhide (hd0,1)       # disk1のpartition2が隠されていたら可視モードにする
rootnoverify (hd0,1) # disk1のpartition2をrootファイルシステムにする
makeactive           # disk1のpartition2をアクティブにする
chainloader +1       # スタートパーティションの最初のセクターを起動ポイントにする
boot                 # OSを起動する

二つめのハードディスクからMS Windows(TM)をスタートしたい場合、ディスク1とディスク2を仮想スワップして、Windowsが最初のディスクからスタートしたように思い込ませます:

map (hd0) (hd1)
map (hd1) (hd0)

落とし穴

Grubには1024シリンダの壁がないので1024シリンダ以降にあるシステムを起動できますが、/boot/grub/stage2 はBIOSから読めるようになっていないと(つまり1024以内のシリンダにないと)起動しません。 Stage2はインストール後には動かせなくなるので、(たとえばデフラグメントプログラムなどが)このファイルを動かすと設定データと適合せず起動できなくなることがあります。 そのためStage2は書き込み防止にして置いた方が安全です。(Stage2のあるディレクトリから)次のコマンドを実行します:

Dos: attrib +r +s stage2
LINUX: # chmod a -w ./stage2
