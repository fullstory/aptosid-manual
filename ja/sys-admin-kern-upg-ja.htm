カーネルのインストール

siduxのカーネルはaptを使ってインストールします。
一番近いsiduxのレポジトリーミラーを使うようにsourceファイルを編集してください。


siduxのカーネルには次のような種類があります:

  • sidux-686 sidux-686 - シングルまたはデュアルコアのi686系のプロセッサー用カーネル
  • sidux-amd64 64bitのsiduxカーネル
カーネルのアップデートは次の４つのステップで行います:

1. コンソールでまずrootになります:

apt-get update

2. 常に最新カーネルをインストールするには:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. これでカーネルのアップデートはできました。ATIやnVidia,Intelなどの最新グラフィックドライバーが必要な場合はそれもインストールします。まずスクリプトをアップデートします:

sgfxi -U

4.ここでコンピュータを再起動して新しいカーネルでシステムを立ち上げます。(デフォルトで最新カーネルが選ばれるはずです)。

3Dドライバーのアップデートには:

sgfxi

もう一つの方法としては新しいカーネルの最初の起動でランレベル３に入るようにします。そのためには起動オプションに 3 を加えてください。そしてrootとしてログインしてから sgfxi を実行します。

グラフィックドライバーのインストールの詳細はこちらへ

新しいカーネルで何か問題があったら、再起動の際に以前のカーネルを選ぶことができます。

モジュール

どのモジュールをインストールするかを決めるには次のコマンドを実行するとインストールできるモジュールがすべて表示されます。モジュールのパッケージ名をコマンドラインにペーストして使います:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

各モジュールの説明が見たい場合は次のコマンドをコンソールにペーストして実行します:

apt-cache search 2.6.*.slh.*-sidux

必要なモジュールをインストールするには(例としてここではvirutualbox-oseとqc-usbのモジュールをインストールします):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)

モジュールがカーネルにロードされているのを確認するには:

ls /sys/module/
または
cat /proc/modules

重要事項: (wlanなど)特別なモジュールが必要な場合はそのための.debパッケージをインストールする必要があります。例としてAgereのet131xイーサーネットアダプターのモジュールをインストールするには: 

apt-get install et131x-modules-*

.debパッケージがすでにローカルディスクにある場合はdpkg -i et131x-modules*.debでインストールできますが、その場合でもapt-cacheが使える場合はapt-get でインストールした方がいいです。

詳しいことはNdiswrapper と WIFI-WLANのページを参照

カーネルをインストールする別の方法

smxi

smxi スクリプトにはカーネルやドライバーののインストールやその他多くのシステムアドミン用の機能が統合されています:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

GUIの siduxコントロールセンター からもカーネルのインストールができます。KDE-Start-Menu>System>sidux Control Centre. から実行します。その他のシステムアドミン用のGUIモジュールも実行できます。

古いカーネルの削除

新しいカーネルのインストールに成功したら、古いカーネルは削除してもかまいませんが、新しいカーネルに問題があった場合にはGrubの起動メニューから古いカーネルでも起動できるように、少なくとも数日間は古いカーネルを維持することをお勧めします。

古いカーネルはシステムから削除できます。そのためには sidux-kernelhackingというパッケージのkernel-remover というスクリプトをつかいます。このスクリプトは削除できるカーネルをリストアップしてどれを削除するかを質問します。

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

siduxコントロールセンターからkernel-removerスクリプトを実行することもできます。smxiでカーネルをインストールした場合はsmxiのカーネル削除機能を使ってください.
