カーネルのインストール

siduxのカーネルはここからダウンロードできます。 一番近いのミラーを選んでください:

Main Mirrors
http://debian.tu-bs.de/project/sidux//kernel ドイツ
http://sidux.com/files//kernel ドイツ
http://sidux.net/files//kernel ドイツ
ftp://ftp.spline.de/mirrors/sidux/kernel/ ドイツ

These mirrors can be as much as 24 hours behind the main mirrors
http://www.mirrorservice.org/sites/sidux.com/sidux/kernel イギリス
http://ftp.heanet.ie/pub/sidux//kernel アイルランド
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernel  テネシー州, USA
http://ftp-mirror.internap.com/pub/sidux/kernel  GA. USA
http://ftp.tw.debian.org/pub/Sidux//kernel 台湾 
http://mirror.aarnet.edu.au/pub/sidux/kernel オーストラリア
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernel 南アフリカ

二つのタイプのカーネルがあります:

  • sidux-686 - i686 プロセッサー・ファミリーのシングル/ダブル・コア（またはそれ以上）の CPUs
  • sidux-amd64 - AMD64 プロセッサー・ファミリーのCPU用の64bitのsidux
次の4つのステップでコンソールからカーネルがアップデートできます:
apt-get update

1. 最新のバージョンをチェックするには:

wget -O - http://sidux.com/debian/pool/main/l/linux-2.6-sidux 2>/dev/null | awk "/linux-image.*$(dpkg-architecture -qDEB_HOST_ARCH_CPU)/" | sed -e 's/^.*linux\-image\-//' -e's/_.*\.deb.*$//'

2. モジュールのリストを表示するコマンド。この中に必要なモジュールがあったらパッケージ名を記録してください。

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

モジュールの説明も見たかったら:

apt-cache search 2.6.*.slh.*-sidux

3. カーネル、ヘッダーと必要なモジュール(ここでは例としてvirtualbox-oseとqc-usb)をインストールするコマンド:

apt-get install  linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686
apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686

4. 続いてコンピュータを再起動します。 新しいカーネルがデフォルトで起動されます。 新しいカーネルの起動に問題があったらもう一度起動して一つ前のカーネルを選んで試すことができます。

カーネルにロードされたモジュールを調べるには:

ls /sys/module/
または
cat /proc/modules

重要事項: wlanなどのドライバ・モジュールが必要な場合、そのモジュールの.debパッケージをインストールする必要があります。 たとえばmadwifiを使っているならmadwifi用の.debパッケージが必要です: 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb is only used if you have the debs locally already, but even in that case, if they are in the apt-cache, apt-get is a better option.

インターネットへの接続／ndiswrapperとWIFI-WLAの項を参照

カーネルのアップデートはこれで終わりです。 グラフィック・ドライバば必要な場合, 最新のドライバをインストールします ATI か NVIDIA のカードを使っている場合、新しいカーネルをインストールした後の最初の起動は、ドライバのインストールのため(起動の際のGrubのカーネル・オプションに3を付け加えて) init 3のレベルで始める必要があります。

smxi を使って半自動化されたカーネル・アップデート:

apt-get update
apt-get install sidux-scripts
smxi

KDEーStart-MenuからGUIのsiduxccコントロールセンターを使っても同じことができます.

古いカーネルを取り除くには

新しいカーネルのインストールに成功したら kernel.zip (と古いカーネル)を消しても構いませんが、数日の間は何かの問題に備えてGrubスクリーンから選べる古いカーネルをとっておくことを勧めます。

古いカーネルはシステムから取り除くことができます。 sidux-kernelhackingというパッケージに含まれるkernel-remover というスクリプトを使ってください。 このスクリプトでどのカーネルを取り除くか選択できます。

kernel-remover
