メタパッケージ

メタパッケージインストーラは次のコマンドでインストールできます。

apt-get install install-meta

そしてKmenu>Utilities>metapackage-installerで実行できます。

The meta-packages installer should only be used if the system has been recently dist-upgraded, so that whenever you run it, it only installs the packages needed and doesn't initiate a partial dist-upgrade at an inopportune time.

次のような項目がインストールできます:

  • Disk: パーティションの走査と修復のツール
  • Education: 教育的ソフト
  • Games: ゲーム
  • Language: 日本語、中国語、韓国語、クロアチア語、ギリシャ語、ポーランド語、ロシア語などの言語をサポートするパッケージや OpenOffice.org、iceweaselなどでのこれらの言語をサポートするためのパッケージ
  • LaTex:
  • Multimedia -VDR:
  • Network: Tor と Privoxy
  • Non_free : いろいろなドライバー
  • Older_Computer: 軽量なアプリとウィンドウマネージャ
  • Office: Gnumeic、Abiword、Koffice、 OpenOffice.org
  • Server: sidux LAMP stack
  • WindowManagers:
    - icewm - (軽量デスクトップ)
    - lxde - (軽量デスクトップ)
    - Fluxbox - (軽量デスクトップ)
    - fvwm-crystal - (軽量デスクトップ)
    - Xfce4 - (軽量デスクトップ)

ノート: メタインストーラが実行された後で次のような表示が出るかもしれません。

paste your code here: (not necessary for normal install, only for preconfig)

気にしないでただ Enterを押してください。

メタインストーラとプロクシ

HTTPプロクシを通してインターネットに接続されているコンピュータの場合、メタインストーラの実行に問題が起こるかもしれません。 その場合、まずメタインストーラは必ずrootとして実行してください。

apt.confファイルのプロクシ設定を変える必要があります。 プロクシ設定を変えるのに二つの方法を紹介しますが、方法１のほうが持続的である点で優れています:

方法 1

プロクシを apt.conf ファイルの中に定義する。 rootとして kwrite で/etc/apt/apt.conf を開き、次のように入力します:

Acquire{
HTTP::proxy "http://192.168.1.1:3128";
FTP::proxy "http://192.168.1.1:3128";
}

方法 2

ターミナル/コンソールを開きsux をつかってrootになり、http_proxy と ftp_proxy の環境変数をセットしそのターミナルからメタインストーラをスタートします。 他の場所からスタートすると設定した環境変数が使われないので効果がありません。 この方法だと設定が保存されないためメタインストーラを実行するたびに同じ作業を繰り返す必要があります。:

export http_proxy="http://192.168.1.1:3128"
export ftp_proxy="ftp://192.168.1.1:3128/"
