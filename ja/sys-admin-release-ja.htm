Rolling Notes

sidux は Debian と sidux独自のパッケージの Dist-upgrade によって、再インストールを必要としない円滑なローリングリリースを提供するよう努力していますが、それでもいくつか自動化できずにユーザーの手を借りないとできないことも起こります。

このページは sidux の新しいリリースごとに旧リリースをローリングアップデートするユーザーのために書き換えられます。現在のリリースはコードネーム Pontos です。

安全のためのアドバイス

sidux インストーラの改良作業の際に、ユーザー権能の乱用に利用されかねない、安全上の欠陥、HDへのインストーラの設定ファイルのうち、インストール先の新しいrootと、デフォルトのユーザー(UID 1000)のパスワードのMD5暗号化された文字列が書き込まれたものがインストール後にも残っているのが発見されました。 この文字列はパスワードそのものというわけではありませんが、パスワードを破るためのブルートフォース攻撃に利用される可能性のあるものです。

問題のあるリリース:

  • 2007-01 Χάος (Chaos)
  • 2007-02 Τάρταρς (Tartaros)
  • 2007-03/ 2007-03.1 Γάια (Gaia)
  • 2007-04/ 2007-04.5 Έρως (Eros)
  • 2008-01 Νυξ (Nyx)
  • 2008-02 Έρεβος (Erebos)
  • 2008-03 Ουρέα (Ourea)

修正コマンド:

find /root /home -maxdepth 2 -type f -name '.sidconf*' -delete

とくに複数のユーザーが使うシステムでは、念のためrootと、(UIDが1000である)最初のユーザーのパスワードを変更することを勧めます。rootおよびそユーザーとして、"passed" のコマンドで変更できます。この問題の原因は発見されているため、sidux 2008-04 "Πόντος" には問題ありません。

Debian はホームディレクトリを 0751 のアクセス許可で作成しますが、sidux の新規インストールでは 0700 をデフォルトに設定しています。

sed -i 's/^DIR_MODE=.*/DIR_MODE=0700/' /etc/adduser.conf
dpkg-reconfigure adduser
(denies "system-wide readable home directories")

ALSA の OSS コンパチブルレイヤー

alsa-driver のバージョン1.0.17.dfsg-3以降、OSS コンパチビリティモジュールはデフォルトでロードされなくなりました。新しい Linux システムでは何も問題がありませんが、古いソフトウェアや、商業的なアプリケーションには今でも OSS サポートを要求するものがあります。カーネル内蔵の ALSA エミュレーションは限られているため、(ソフトウェアベースのミキシングがないため一つのアプリケーションだけしか /dev/dsp にアクセスできないことなど) ALSA コンパチブルなアプリケーションを使うことが望ましいですが、どうしても OSS コンパチブルのソフトをつかわないといけないと困る場合、２つの方法があります。

  • aoss wrapper
    /usr/bin/aoss は alsa-oss パッケージに含まれていて、古いアプリケーションの OSSコールをラップして OSS ABI をALSAに透明に中継します。この方法で通常 ALSA の利点をいかすことができます。
  • カーネル内蔵 ALSA エミュレーション
    バイナリーコンパチビリティはこちらのほうが優れています。(/dev/dsp がエミュレートされているため, /dev/audio と /dev/mixer のデバイスが作られます), ただこの方法ではソフトウェアベースのミキシングはできないため、ただ一つのアプリケーションの音声しか有効になりません。

たいていのアプリケーションはソフトウェアベースのミキシングとハードウェアへの同時アクセスのできるよう "aoss <program>" で起動するほうがいいですが、まれにカーネル内蔵コンパチブルレイヤーを使わないといけないケースがあります。これが必要なときは snd-pcm-oss、 snd-mixer-oss とsnd-seq-midi のモジュールを次のコマンドでロードします。 

for i in snd-pcm-oss snd-mixer-oss snd-seq-midi; do
	grep -q "^${i}" /etc/modules  || echo "${i}" >> /etc/modules
	grep -q "^${i}" /proc/modules || modprobe "${i}"
done

Broadcom b43 wlan

2.6.25 以降のカーネルは、4.150.10.5 以降の Broadcom ファームウェアレビジョン (内部バージョン "410.2160 (2007-05-26 15:32:10)") が必要です。以前の (2.6.24までの) カーネルは以前の b43-fwcutter がインストールした 4.80.53.0 (内部バージョン"351.126 (2006-07-29 05:54:02)")を使っていました。b43-fwcutter は自動的にファームウェアをアップデートできないため、次のように手動でアップデートする必要があります。

dpkg --purge b43-fwcutter
rm -rf /lib/firmware/b43 /lib/firmware/b43legacy
apt-get update
apt-get install b43-fwcutter

コンソール設定

コンソールのキーマップは"console-setup" で行います。 

apt-get install console-setup
dpkg-reconfigure console-setup

gspca2

2.6.27 カーネルには新しい v4l2 ベースの gspca2 ドライバーが内蔵されていて、以前の v4l1 ベースの gspca 外部ドライバと交換されましたが、多くのウェブカメラが旧式のドライバーを必要としています。 多くのユーザースペースのプログラムは残念ながらまだ新しい v4l2 API を使えないため "LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <program>" が必要です。. 

apt-get update
apt-get install libv4l-0

AS USER
$ LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <program>

insserv

Pontos 2008-04 以前にインストールされたシステムで insserv を使うには

警告: 古いインストールには ユーザーの initscript やシンボリックリンクなどに適応できないため、insserv は自動的に設定されません。Pontos 以前にインストールされたシステムに insserv をインストールすることは推奨しません。このパッケージの使用には注意が必要で、既存の initscript に LSB 依存関係の間違いや欠如があるとシステムが起動できなくなるおそれがあります。古いシステムに insserv をインストールするにはいくつかの作業が必要です。

insserv をインストールし、有効二する前に必ず Debian documentation notesとリリースノートを読んでください。

不要になったパッケージ

siduxの開発が進むにつれて、いくつかのパッケージが不要になったり、より良いソフトウェアと交換されたりしています。次のパッケージが不要になりました。

  • capi-isdnconf.
  • configure-xawtv
  • gprsconnect (umtsmonに交替）
  • ipw2100-firmware (firmware-ipw2x00に交替)
  • ipw2200-firmware (firmware-ipw2x00に交替)
  • modemlink (umtsmon と wvdialに交替)
  • ppp-scripts (umtsmonに交替)

Apt のレポジトリーにもう存在しないパッケージを表示して、不要になったパッケージをさがすヒントにするのに次のコマンドが使えます。ただし、apt以外の方法でインストールされたプログラムが必要としているパッケージもあるかもしれないので注意。 apt-show-versions は独自のパッケージなのでまだインストールされていないかもしれません。

$ apt-show-versions | awk '/No available version in archive/{print $1}'
