ターミナル/コンソールとは

ターミナル、またはコンソールとは、さまざまなコマンドを入力し、実行することでLinuxのOSと直接対話することのできるプログラムです。 他の呼び名としてしばしば 'shell' または 'command line'も使われます。 KDEではkonsoleというプログラムが用意されています。 ターミナルは非常に強力な道具として基本的な使い方を知っておく価値があります。

aptosidではターミナル/コンソールはKメニューの横にモニターのアイコンで表示されています。 えらんだテーマによってはアイコンの中に貝の絵が入っているかもしれません。同じアイコンをKメニューの"System"の中から選ぶこともできます。

ターミナルのウィンドウを開くと次のようなプロンプトが表示されます:

username@hostname:~$

usernameのところにあなたのログインネームが表示されているはずです。 ~ は今あなたのホーム・ディレクトリにいること、$はユーザーの権利でログインしていることを示しています。 最後にカーソルが表示されています。 これが実行したいコマンドを入力するためのコマンドラインです。

実行にあたってrootの権利を必要とするコマンドがたくさんあります。 そのためにはコマンドラインにsux とタイプしてEnterを押してください。 rootパスワードを聞かれますのでタイプしてEnterを押してください。 (パスワードの入力中はタイプした文字は表示されません)

もしパスワードが正しければ次のようなプロンプトが表示されます。

root@hostname:/home/username#

警告: rootとしてログインしている間、重要なファイルの消去といった危険なコマンドもそのまま実行されます。 システムを害する可能性も十分にありますから、どんなコマンドを実行しようとしているか完全に把握したうえで実行してください。

プロンプトの$マークが# に代わることに注目してください。 ターミナル/コンソール上ではいつでも# はrootの権利でログインしていることを示します。 このマニュアルでは$と#のマークの前に表示されるプロンプトを省略します。 次のコマンドは:

# apt-get install something

ターミナルを開き,(suxで)rootになり,コマンドを#のプロンプトに入力するという意味です。 (#をタイプしないように)

時にコンソールの画面がおかしくなってしまうことがあります。 そういう時は:

reset

とタイプしてEnterを押してください。

ターミナル/コンソールの表示が歪んで見えるときがあります。 そういうときはctrl+l, を押してターミナル・ウィンドウの再描画をすることで直ることがあります。 表示の歪みはncoursesのインターフェイスを使っているプログラム(irssiなど)で起こることが多いです

ターミナル/コンソールはときに凍ってしまって何の反応もしないように見えるのに、実際は動いていてタイプされたコマンドが実行されている、ということがあります。 これは誤ってctrl+sを押して締まったときに起こり、その場合 ctrl+qを押してターミナルのロックを解除することで直ります。

コンソールの user:~$ と root:# の プロンプト:の色分け

コンソールのプロンプトを色分けすることでuser~$のつもりでうっかりroot # として破滅的なコマンドを実行してしまったりすることを防いだり、マーカーとして100行ほど前に実行したコマンドもすぐ見付らるようしたりすることができます。

デフォルトでは user~$ と root# のプロンプトは同じ色ですが、どちらの色も簡単に変えることができます。

基本色の色指定コード:

(00;XXのシンタックス)
[00;30] Black   黒
[00;31] Red     赤
[00;32] Green   緑
[00;33] Yellow  黄色
[00;34] Blue    青
[00;35] Magenta マゼンタ
[00;36] Cyan    シアン
[00;37] White   白
[[00;XX] を [01;XX] に変えるとカラーバリエーションが得られます].
ユーザーとしての ~$ プロンプトの色を変えるには:

$ プロンプトのユーザーとして、お好みのテキストエディタで:

$ <editor> ~/.bashrc

39行目あたりの次のようなコメント行の先頭の # を消して有効なコマンドにします:

force_color_prompt=yes

53行目あたりの(一例として) 01;32m, のような色指定コード文字列のある行を探してお好み色のコードに変えます。

例として シアン [01;36m\]の user~:$ プロンプトにするには、3箇所の[01;XXm\] を編集して次のようにします:

PS1='${debian_chroot:+($debian_chroot)}\[\033[01;36m\]\u@\h\[\033[01;36m\]:\[\033[01;36m\]\w\[\033[00m\]\$ '

新しい設定は次回のターミナルログインの時から有効になります。

root としての # プロンプトの色を変えるには:
sux
<editor> /root/.bashrc

39行目あたりの次のようなコメント行の先頭の # を消して有効なコマンドに>します:

force_color_prompt=yes

53行目あたりの(一例として) 01;32m, のような色指定コード文字列のある行>を探してお好み色のコードに変えます。

例として 赤 [01;31m\]の root:# プロンプトにするには、3箇所の[01;XXm\] を編集して次のようにします:

PS1='${debian_chroot:+($debian_chroot)}\[\033[01;31m\]\u@\h\[\033[01:31m\]:\[\033[01;31m\]\w\[\033[00m\]\$ '

新しい設定は次回のターミナルログインの時から有効になります。

ターミナルの背景色を変えるには

コンソールターミナルの背景色やフォントを変えるにはターミナルプログラムのメニューのオプションから探してください。

色設定に凝りはじめると限りなく様々な設定ができますが、あまり凝らずになるべくシンプルな設定にしておくようにおすすめします。

About sux

rootの特権を持って実行する必要があるコマンドがたくさんあります。 そのためには:

sux

とタイプします。

rootになるためのコマン ドとして普通は'su'を使いますが、代わりにsuxを使う ことでX11のアプリケーションもコマンドラインから実行することができます。 suxはwrapperとして標準のsuを呼ぶ前にユーザーのXクレデンシャル(資格証明)やディスプレイの情報などををサーバに送ります。 ( sudoの欄参照).

suxを使ってX11のアプリケーションを実行する例としてkwriteやkateといったテキストエディタでrootのファイルを編集したり、dolphin や thunar といったファイルマネージャーでシステム管理したり、gpartedでパーティションを作ることなどがあげられます。

KDEのアプリケーションにはプログラム名の前にdbus-launch をつける必要のあるものもあります:

dbus-launch <Application>
KDE のキーボードオプション

KDE の krunner をスタートするには:

Alt+F2

またはデスクトップを右クリックしてから次のメニューを選びます:

Run Command

そして:

kdesu <Application>
XFCE のキーボードオプション

XFCE の Run Command をスタートするには:

Alt+F2

またはデスクトップを右クリックしてから次のメニューを選びます:

Run Command

そして:

gksu <Application>
Other Desktop WIndow Manager options

もう一つ、たいていのデスクトップマネージャーでも使えるオプションとして:

Alt+F2

then:

su-to-root -X -c <Application>

上記のコマンドはターミナルにタイプして実行します。

sudo はサポートされていません

ハードディスクにインストールされたシステムでは sudo コマンドはデフォルトでサポートされていません。ライブCDではrootのパスワードがないのでsudo を使っても問題がありません。sudo をサポートしない理由として、侵入者がもし一人のユーザーのパスワードを手にしても、すぐにスーパーユーザーの権利でシステムを破壊することができなくなるからです。

sudo のもう一つの問題点として、ユーザーの設定でスーパーユーザー用のコマンドを実行しているうちに、ファイルの権利設定を知らない間に変更してしまうことがあることです。このためにその後コマンドをユーザーとして実行できなくなってしまうような例もあります。sudo の代わりにsux, kdesu, gksu または su-to-root -X -c などを使いましょう!

rootになることの意味

警告: rootとしてログインしている間、重要なファイルの消去といった危険なコマンドもそのまま実行されます。 システムを害する可能性も十分にありますから、どんなコマンドを実行しようとしているか完全に把握したうえで実行してください。

E-mailを送ったり計算表を作ったり,ネットのサーフィングなど普通のユーザーの権利で日々使っているプログラムをrootとしてターミナル/コンソールから実行することはどんな場合であれ絶対に避けるべきです。

コマンドライン・ヘルプ

ちゃんとあります。 大抵のLinuxのコマンド/プログラムはコマンドラインから呼び出せる"man page"または"manual page"という名のマニュアルを備えています。 呼び方は:

$ man "command"

or

$ man -k <keyword>

これでそのコマンドのマニュアルページが読めます。 カーソルキーで上下にスクロールできます。 試しに次のコマンドを実行してみましょう:

$ man apt-get

マニュアルページから抜け出すには qをタイプしてください。

もう一つの便利なコマンドに "apropos" があります。 これはコマンドの名前やシンタックスをを正確に思い出せない時にマニュアルページをサーチするコマンドです。 試しに次のコマンドを実行してみましょう:

$ apropos apt-

パッケージマネージャ'apt'のすべてのコマンドがリストとして表示されます。 この'apropos'は非常に強力なツールで使い方のすべてを紹介するスペースはありませんので、詳しいことはマニュアルページを読んでください。

スクリプトとその使い方

シェル・スクリプトとはいくつかのコマンドを一つのファイルの中にまとめる便利なものです。 ファイル名をタイプすることでコマンドが順番に実行されます。 aptosidには便利なスクリプトがいくつかついてきます。

シェルスクリプトがカレント・ディレクトリにあるときは次のように実行します

./スクリプトの名前

スクリプトの書かれた目的に応じてrootのアクセス権(sux)が必要なものと特に必要としないものがあります。

スクリプトのインストールと実行の手順

wgetを使ってスクリプト・ファイルをダウンロードします。 指示にしたがってファイルをどこに保存するか決めてください。 (たとえば /usr/local/binのディレクトリに保存するように、など), suxでrootとしてログインした上でマウスを使ってファイル名をコピーしコンソール・ウィンドウにペースとすることもできます。

wgetを使ってrootアクセス(sux)を必要とするダウンロードをする手順の一例 
sux
cd /usr/local/bin
wget script-name

このファイルを実行可能モードのにする必要があります

chmod +x  script-name

ブラウザを使ってスクリプト・ファイルをダウンロードし指示された場所に移動・保存する手もありますが、この場合でもやはり実行可能モードにする必要があります。

ユーザーとしてwgetを使う一例r

ファイルをユーザー($)として$HOMEディレクトリに保存するには:

$wget http://bluewater.aptosid.com/shell-script-test/test-script.sh

$chmod +x test-script.sh

スクリプトを実行するにはターミナル/コンソールを開き、スクリプトの名を次のように入力します:

$./test-script.sh

次のように表示されるはずです:
