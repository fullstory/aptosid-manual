siduxクイックスタートガイド

重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

siduxはDebian sidと100%コンパチブルであることを目指していますが、ときに一時的なホットフィックスを使ってDebianのパッケージの代わりのパッケージを提供することがあります。 siduxのレポジトリーにはアート、カーネル、スクリプト、ツール、文書などのsidux独自のパッケージが置いてあります。

Debian sid/unstableの安定性

sidはDebianのunstableレポジトリの名前です。Debian sidは頻繁にアップデートされアップストリームの最新のソフトウェアバージョンとシンクロナイズされます。このアップデートの頻繁さのために、アップストリームからDebianのパッケージになるまでの短い期間にパッケージをとことんテストすることはできません。

siduxカーネル

siduxではDebianのカーネルを使っていません。siduxのカーネルは http://www.kernel.org/ の最新のカーネルに、様々なハードウェアに対応し、問題点を避け、スピードと安定性を増すように変更を加えsiduxのために最適化されたカーネルです。

カーネルのミラーは: カーネルのアップグレードのページにあります。

パッケージ管理

siduxはDebianのパッケージシステムに準拠して /etc/sources.list.d/* で設定されるDebianその他のレポジトリー上のパッケージをaptとdpkgのコマンドを使ってインストールします。

Debian sidには二万を越える数のパッケージがありますから、使いたいプログラムを apt-cache or with Debian Package Search GUI application.。

パッケージをインストールするにはapt-get install <パッケージ名>のコマンドを使います。パッケージのインストールを参照。

Debian sidのレポジトリは一日に平均して4回ほどアップデートされるため、インストールできるパッケージのリストと現行バージョンの情報はあっという間に古くなってしまいます。そのためapt-get dist-upgradeの前にまず必ずapt-get update を実行して、サーバーのパッケージリストについてののデータを更新するすることが肝心です。

Using other Debian based distributions' repositories, Source and RPMs

ソースコードからのインストールはサポートされません。どうしてもアプリケーションを自分でコンパイルしなくてはならないときはユーザーとして作成し、システムにはインストールせずにホームディレクトリに置いてください。CheckInstallの使用やalienといったツールを使ってRPMをdebにコンバートしてインストールすることなどもサポートされません。

Debian based distributions' often - から派生するいろいろなディストロがありますが、ファイルの置き場所などの違いによってシステムが不安定になったり、パッケージのネーミングやバージョンの違いのために依存関係が解決せず、インストールに失敗したりすることがあります。別バージョンのglibcを使っているためにアプリケーションが実行できないこともしばしば起こります。

Upgrading the system - dist-upgrade

siduxのアップグレードではapt-get dist-upgrade が公式にサポートされている方法です。GUIのフロントエンドを使ったアップグレードはsiduxではサポートされていません。システムのアップグレード -- dist-upgrade

Xウィンドウの上からのdist-upgradeはサポートされていません。(KDEやXFCEなどの)ウィンドウマネージャか、またはバーチャルターミナル(Ctrl+Alt+F1, Ctrl+Alt+F2など)から'init 3'を実行することでXが終了し、安全にアップグレードできるようになります。

ネットワーク設定

Ceni というネットワーク設定ツールで手早くLANカードやワイアレスカードを設定できます。ワイアレス機能はネットワークのスキャンWEPまたはWPAを使った暗号化ができ、設定のためのツールやWPAサプリカントを起動します。DHCP(IPアドレスの自動設定)を使っている場合のLAN設定は簡単で、そうでない場合はIPアドレスやネットマスク、ネームサーバーなどを手動で設定します。

Ceniは Ceniまたはceni のコマンドで実行されます。まだインストールされていないときはapt-get install ceniのコマンドでインストールできます。

他のウィンドウマネージャ

KDE、XFCEとfluxboxはsiduxにデフォルトで搭載されています. Gnomeはsiduxではサポートされていません。ウィンドウマネージャとしてのGnomeはsidのレポジトリでは不安定で、しばしば問題をおこすことで知られています。siduxフォーラムやウィキ、IRCチャットにはGnomeの経験のあるユーザーもいるので個人的にサポートしてくれることもあるかもしれませんがそうでない場合は自分でなんとかするしかありません。

日本語サポート

sidux リリースにはささやかながら日本語のサポートが組み込まれています。

ライブCDの起動の際にF4を押して「日本語」を選ぶか、または起動オプションにlang=jaを加えることで、日本語ロケールがデフォルトになり、インストーラなどのメッセージが日本語表示されるようになります。(sidux-kde-full のLive-DVDには日本語のフォントも入っていて、F4のオプションで日本語システムがスタートしますが、それ以外のLive-CDではデフォルトのロケールは設定されますが、起動後に日本語のフォントその他をインストールする必要があります。)

追加のフォント、日本語入力環境、KDEの日本語化などは日本語サポートのウィキページにしたがって、スクリプトまたはapt-getのコマンドでインストールできます。

注意：日本語環境のインストールの前にネットワークの設定が必要です

詳しくは日本語サポートのウィキページをご覧ください。
