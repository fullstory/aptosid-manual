siduxクイックスタートガイド

重要事項！: siduxはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。（これはもともとKnoppixによって開発されたものですが、Knoppixと同一のものではありません）そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

siduxはDebian sidと100%コンパチブルです。Debianパッケージの代わりにsiduxパッケージが使われるのはsidのレポジトリの問題が解決されるまでの一時的な対策としてだけです。sidux独自の追加パッケージはスクリプトやツールなどが入っています。

Debian sid/unstableの安定性

sidはDebianのunstableレポジトリの名前です。Debian sidは頻繁にアップデートされアップストリームの最新のソフトウェアバージョンとシンクロナイズされます。このアップデートの頻繁さのために、アップストリームからDebianのパッケージになるまでの短い期間にパッケージをとことんテストすることはできません。

siduxカーネル

siduxではDebianのカーネルを使っていません。siduxのカーネルは http://www.kernel.org/ の最新のカーネルに、様々なハードウェアに対応し、問題点を避け、スピードと安定性を増すように変更を加えsiduxのために最適化されたカーネルです。

カーネルのミラーは: カーネルのアップグレードのページにあります。

カーネルzipアーカイブにはinstall-kernel-sidux.shというスクリプトがあり、これを使って必要なパッケージをインストールし、新しくインストールされたカーネルをデフォルトに設定します。

注意 apt-getでダウンロードできるDebianのカーネルには上に述べたsiduxカーネルのような修正や最適化、前もってコンパイルされたモジュールなどがありませんから、sidux上で使わないでください。

パッケージ管理

siduxはDebian sidに完全に準拠しています。Debianはapt-getとdpkgをソフトウェアパッケージの管理に使っています。

Debian sidには二万を越える数のパッケージがありますから、使いたいプログラムを apt-cache や kio-apt などのツールを使って見つけてください。

パッケージをインストールするにはapt-get install <パッケージ名>のコマンドを使います。パッケージのインストールを参照。

apt-get を使って /etc/sources.list.d/* のファイルに指定されたDebianその他のレポジトリからパッケージをインストールします。

Debian sidのレポジトリは平均して毎日一二度アップデートされるためパッケージのリストやバージョンのデータはすぐに古くなってしまいます。そのためパッケージのインストールやapt-get dist-upgradeの前にまず必ずapt-get update を実行して、サーバーのパッケージリストについてののデータを更新するすることが肝心です。

その他のDebian、ソースコード、RPMパッケージからのインストール

ソースコードからのインストールはサポートされません。どうしてもアプリケーションを自分でコンパイルしなくてはならないときはユーザーとして作成し、システムにはインストールせずにホームディレクトリに置いてください。CheckInstallの使用やalienといったツールを使ってRPMをdebにコンバートしてインストールすることなどもサポートされません。

Debianから派生するいろいろなディストロがありますが、ファイルの置き場所などの違いによってシステムが不安定になったり、パッケージのネーミングやバージョンの違いのために依存関係が解決せず、インストールに失敗したりすることがあります。別バージョンのglibcを使っているためにアプリケーションが実行できないこともしばしば起こります。

Upgrading the system - dist-upgrade

siduxのアップグレードではapt-get dist-upgrade が公式にサポートされている方法です。GUIのフロントエンドを使ったアップグレードはsiduxではサポートされていません。システムのアップグレード -- dist-upgrade

Xウィンドウの上からのdist-upgradeはサポートされていません。(KDEやXFCEなどの)ウィンドウマネージャか、またはバーチャルターミナル(Ctrl+Alt+F1, Ctrl+Alt+F2など)から'init 3'を実行することでXが終了し、安全にアップグレードできるようになります。

smxi

smxiスクリプトはdist-upgradeを実行するもう一つの優れた方法で、最新のライブCDにはすでに搭載されています。依存関係の壊れたパッケージの取扱いができ、問題を起こすパッケージはメンバーによってモニターされ自動的にアップデートのリストから外したり、大きな問題については緊急の修正を行うこともあります。このプログラムの機能をフルに生かすにはsiduxのフォーラムをしっかり読んでください。

smxiはdist-upgradeのアシストの他に、3Dのグラフィックドライバーやカーネルを簡単にインストールできるオプションやvirtualbox-oseやquemuなどの自動インストールなど様々な機能があります。.

smxiのコマンドシンタックス

sgfxi - グラフィックドライバーインストーラ (nVidia/ATiの商業ドライバーまたはXorgのオープンドライバー)

このツールを使えばnVidiaやATIのグラフィックカードの全機能を使えるnvidiaまたはfglrxドライバーを自動的にインストールできます。

NVIDIA - Debianのnvidiaパッケージのインストールにはgfxiの使用をおすすめします。このスクリプトは通常新しいドライバーの発表当日にアップデートされ、3Dが正しく作動するようにxorg.confのオプションを設定します。

ATI - sidのレポジトリのfglrxドライバーが、現行バージョンのXウィンドウとカーネルとの組み合わせで正しく作動する場合はsgfxiで自動インストールできます。fglrxドライバーの命名法やコンパチビリティなどが新しいバージョンごとに大きく変わるため、ATIのサポートは期待にそえないことがよくあります。フォーラムやIRCチャットで状況を確認してください。

sgfxiのオプション-n <driver> または -N <driver> のコマンドを使ってオープンソースのドライバと簡単に交換することができます。

ネットワーク設定

Ceni というネットワーク設定ツールで手早くLANカードやワイアレスカードを設定できます。ワイアレス機能はネットワークのスキャンWEPまたはWPAを使った暗号化ができ、設定のためのツールやWPAサプリカントを起動します。DHCP(IPアドレスの自動設定)を使っている場合のLAN設定は簡単で、そうでない場合はIPアドレスやネットマスク、ネームサーバーなどを手動で設定します。

Ceniは Ceniまたはceni のコマンドで実行されます。まだインストールされていないときはapt-get install ceniのコマンドでインストールできます。

他のウィンドウマネージャ

KDEとfluxboxはsiduxにデフォルトで搭載されています. Gnomeはsiduxではサポートされていません。ウィンドウマネージャとしてのGnomeはsidのレポジトリでは不安定で、しばしば問題をおこすことで知られています。siduxフォーラムやウィキ、IRCチャットにはGnomeの経験のあるユーザーもいるので個人的にサポートしてくれることもあるかもしれませんがそうでない場合は自分でなんとかするしかありません。XFCEは公式にはサポートされていませんが古いコンピュータのユーザーなどで使っている人は多いのでフォーラム/ウィキ/IRCでサポートを受けられるケースが多いでしょう。
