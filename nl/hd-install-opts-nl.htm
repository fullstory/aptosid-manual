Opstarten met "fromiso"

Voor normaal gebruik adviseren wij het ext3 bestandssysteem. Dat is het standaard bestandssysteem voor sidux en wordt goed onderhouden (maintained).

Met deze cheatcode kun je opstarten van een iso-bestand die op een partitie staat, dit is vele malen sneller dan van een CD op te starten. (HD installatie met "fromiso" neemt ook minder tijd in beslag. Sommige hebben het over minder dan 4 minuten, maar met 6-8 minuten is het nog steeds enorm snel).

Dit is natuurlijk veel sneller dan vanaf CD- /DVD-drive en houdt de cd/dvd-drive beschikbaar voor andere toepassingen. Je kunt natuurlijk ook gebruik maken van VMware als alternatief.

Benodigdheden:

* een functionerende grub (op een diskette, een HD-Installatie of van een Live-CD)
* een sidux ISO bestand bijv.: sidux.iso en een Linux bestandssysteem zoals ext2/3
* Als je het bestand sidux.ISO noemt, moeten de commando's aangepast worden want Linux/unix benamingen zijn hoofdlettergevoelig.

Eros en Gaia Live-cd:

Eerst kiezen we een plek voor het iso-bestand en de 2 andere bestanden die nodig zijn. Hiervoor maken we een basis-map, bijv. "/media/sda6" (of sda7,8 ...), maken hierin de map "sidux" aan en plaatsen het iso-bestand daarin:

# mkdir /media/sda6/sidux
# mv sidux.iso /media/sda6/sidux

Dan gaan we naar de map en mounten het ISO image-bestand (een tussenstap via /mnt/test):

# mkdir -p /mnt/test
# mount -t iso9660 -o loop,ro /media/sdax/sidux/sidux.iso /mnt/test

Nu kopiëren we de vmlinuz en intrid,img bestanden van de gemounte iso image naar de map (hieronder de juiste kernel variables voor x invullen):

# cp /mnt/test/boot/vmlinuz-2.6.23.9-rc1-slh-smp-1  /media/sda6/sidux/
# cp /mnt/test/boot/initrd.img-2.6.23.9-rc1-slh-smp-1 /media/sda6/sidux/

Hierna passen we grub aan. We openen het bestand /boot/grub/menu.lst als root en voegen de volgende tekst toe NA de ### END DEBIAN AUTOMAGIC KERNELS LIST: NB: Er zijn geen wildcards (*) toegestaan in grub voor bestandsnamen! 

### ISO boot
title sidux 32bit from ISO
kernel (hd0,4)/sidux/vmlinuz-2.6.23.9-rc1-slh-smp-1 boot=fll quiet vga=791 fromiso=/sidux/sidux.iso
initrd (hd0,4)/sidux/initrd.img-2.6.23.9-rc1-slh-smp-1

Bij de volgende hebben we een nieuw menu item in grub om de de iso image te starten.

Tartaros and Chaos Live-cds:

Eerst kiezen we een plek voor het iso-bestand en 2 bestanden die we nodig hebben, zodat we kortere namen krijgen. Hiervoor creëeren we een basis-directory: bijvoorbeeld /dev/hda7, Creëer nu de directory "sidux" en kopieer de iso-bestand hiernaar toe via:

# mkdir /media/hda7/sidux
# mv sidux.iso /media/hda7/sidux

Nu gaan we naar die directory en mounten we de iso image als volgt:

# mkdir -p /mnt/test
# mount -t iso9660 -o loop,ro /media/hda5/sidux/sidux.iso /mnt/test

Vervolgens kopiëren we de bestanden miniroot.gz en vmlinuz van de gemounte iso image naar onze directory dmv:

# cp /mnt/test/boot/vmlinuz /media/hda5/sidux/
# cp /mnt/test/boot/miniroot.gz /media/hda5/sidux/

Hierna passen we grub aan. We openen het bestand /boot/grub/menu.lst als root en voegen de volgende tekst toe NA de ### END DEBIAN AUTOMAGIC KERNELS LIST: NB: Er zijn geen wildcards (*) toegestaan in grub voor bestandsnamen! 

### ISO boot
title sidux 32bit from ISO
kernel (hd0,4)/sidux/vmlinuz boot=fll quiet vga=791 fromiso=/sidux/sidux.iso
initrd (hd0,4)/sidux/miniroot.gz

Bij de volgend herstart v/d pc hebben we een nieuw menu item in grub om de iso image te booten.

fromiso en persist (eros)

Met eros kun je een blijvend live systeem op een CDRW schijf hebben door een combinatie van een fromiso opstelling met de blijvende cheatcode.

sidux gebruikt aufs om "copy on write" op je cd mogelijk te maken, dit om het je toe testaan om nieuwe bestanden en folders te schrijven en bestaande te updaten in het geheugen. Met de persist cheatcode, kan eros je nieuwe bestanden opslaan op dezelfde harddisk partitie waar je ook je fromiso image staat.

Gebruikmakend vanpersist geeft je een live systeem met alle automatische features van de sidux live cd. Dit heeft als voordeel dat alle hardware automatisch geconfigueerd wordt, maar betekend ook dat de bestanden bij elke nieuwe boot opnieuw aangemaakt worden..

Als voorbeeld worden je instellingen voor X (videokaart, toetsenbord, muis) en je netwerk interface bestanden niet bewaard, het kan wel op elke pc gebruikt worden, maar het is lastig als je specifiek een configuratit van je netwerk moet hebben, want dit moet elke keer weer ingesteld worden.

Misschien is de ideale gebruik van de "persistence mode" in samenhang met de nieuwe install-usb-gui tool om je eigen bootable flash drive met je eigen bestanden en software te maken.

Je bestanden worden dan in een subfolder op de drive opgeslagen. Deze subfolder wordt aangemaakt als deze niet bestaat wanneer de fromiso en persist cheatcodes gebruikt worden.

persist op een FAT filesysteem, meestal gebruikt voor dos/Windows 9x installatie's en meestal standaard op flash devices gevonden, eisen van je om 1 enkele grote bestand aan te maken als een loop device, je moet deze bestand dan formatteren.

Zoals gewend raadt sidux aan om een ext3 partitie te gebruiken om je iso en andere bestanden op te slaan..

Het bestand moet de volgende naam hebben sidux-rw en geplaatst worden in de root directory van je FAT drive. Uitgaande van het feit dat je de gebruikte disc mount als een schrijfbare disc op /media/sdb1 en je wil een 1GB groot bestand creeeren als je persistent drive dan kun je de volgende commando's gebruiken om dit uit te voeren (geen root rechten nodig als je naar het bestand kan schrijven).

disc="/media/sdb1"
size=1024
dd if=/dev/zero of=${disc}/sidux-rw bs=1M count=${size} && echo 'y' | /sbin/mkfs.ext3 ${disc}/sidux-rw

NTFS partitie's, normaal gebruikt bij Windows NT/2000/XP (TM) installatie's, kunnen NIET gebruikt worden voor de persistence mode.

Installatie van sidux op een USB-Stick of USB-HD

De installatie van sidux op een USB-stick of een USB-HD is net zoo eenvoudig als de normale installatie op een harde schijf in de pc. Voer het uit conform de volgende stappen van onderstaande handleiding.

De aanbevolen minimale grootte van een usb stick is:
sidux "LITE" heeft 1.2GB nodig EN je hebt ruimte voor data (home/Mijn_documenten) nodig,
sidux "FULL" needs 2.1GB nodig EN je hebt ruimte voor data nodig

We starten de normale installatie en kiezen de partitie op de USB-device, waarop sidux geïnstalleerd moet worden - hieronder noemen wij dat "sda1", maar elke andere beschikbare partitie op een USB-disk of stick kan ook gebruikt worden.

De partitie moet geformatteerd zijn met een linux-bestandssysteem. Geadviseerd wordt hiervoor het ext3 bestandssysteem. Een reeds bestaande partitie wordt geformatteerd door de installer!

Alle data op deze partitie worden dus overschreven de het formatteren!

Je kunt nu opstarten van je USB-HD/Stick. 'Boot from USB' moet in je BIOS op enabled staan.

Installing sidux with VirtualBox

VirtualBox is ideaal voor 32bit machines. Voor 64bit gebruik dan QEMU.

  • 1. creeer een harddisk image voor VirtualBox
  • 2. start de iso op met VirtualBox
  • 3. installeren op de harddisk image

Vereisten

Aanbevolen RAM: 1 GB : Ideaal zou zijn 512 MB voor het gast os en 512 MB voor de host. (het kan op minder draaien, maar verwacht dan geen enorme performance).

Hard disk ruimte: VirtualBox zelfs heeft niet veel nodig, bij een normale installatie ongeveer 30 MB harddisk ruimte. De virtual machines zullen echter veel grotere bestanden op de harddisk opeisen, deze ziet de virtual machine dan als zijn eigen harddisk. Dus, om bijvoorbeeld MS Windows XP (TM) te installeren, zul je een bestand nodig hebben van meerdere GBs! To have sidux in VirtualBox you need to allocate a 5 gig image plus a swap allocation.

Indien je een nieuwe kernel geinstalleerd hebt, doe dan eerst een dist-upgrade voordat je virtual machine modules gaat installeren, anders krijg je een gcc mismatch foutmelding, omdat de module niet gebouwd kan worden.(Dit geld voor elke externe module)

Het mechanisme:

Installeer de virtualbox pakket
Klik op Applications > System Tools >InnoTek VirtualBox - Virtual Machine
Maak een VM, plaats een Operating Systeem-CD in je CDRom drive en installeer en doe een reboot van het gast OS.

Installatie:

apt-get update
apt-get install virtualbox-ose 
m-a a-i virtualbox-ose

Hierna sla je de informatie van het huidige besturingssysteem op via : 

virtualbox-ose-module-$(uname -r)

Je moet het volgende ook uitvoeren om virtualbox te kunnen starten als betreffende gebruiker:

adduser <username> vboxusers

Het installeren van sidux in een virtual machine

Gebruik virtualbox's wizard om een nieuwe virtual machine voor sidux te creeeren. volg dan de instructies voor een normale sidux installatie.

VirtualBox heeft een goed gedocumenteerde PDF Help document, deze kun je vanaf hun site downloaden.

Installeren van sidux met QEMU

  • 1. Creëer een harddiskimage voor qemu
  • 2. Start de iso op met qemu
  • 3. Installeer op de image

Het creëeren van de harddisk image

Om qemu op te starten heb je waarschijnlijk een harddisk image nodig. Dit is een bestand welke de inhoud van de geëmuleerde harddisk opslaat.

Gebruik het volgende commando:

qemu-img create -f qcow sidux.qcow 3G

Om het image file genaamd "sidux.qcow" te creëeren. De "3G" parameter specificeerd de grote van de disk - in dit geval 3 GB. Je kunt de waarde M gebruiken voor megabytes (bijvoorbeeld "256M"). Je moet je geen zorgen maken over de disk grootte - het qcow formaat komprimeerd de image zo dat de lege ruimte niet aan de grote van het bestand wordt toegevoegd.

Installeren van het besturingssysteem

Dit is de eerste keer dat je de emulator moet starten. Een ding moet je onthouden, wanneer je in het qemu window klikt, wordt de muispointer gebruikt in dit window, om deze weer los te laten moet je op Ctrl+Alt drukken.

Indien je een opstartdiskette nodig heb, start dan Qemu als volgt op:

qemu -floppy sidux.iso -net nic -net user -m 512 -boot d sidux.qcow

Indien je CD-ROM opstartbaar is, start dan Qemu als volgt op:

qemu -cdrom sidux.iso -net nic -net user -m 512 -boot d sidux.qcow

Installeer nu sidux alsof je een installatie uitvoert op een echte harddisk.

Het systeem starten

Om het systeem te starten voer je het volgende commando uit:

qemu [hd_image]

Het is een goed idee om overlay images te gebruiken. Op deze manier hoef je maar 1 keer een harddisk image te creëeren en vertel je Qemu om wijzigingen op te slaan in een extern bestand. Zo kom je van alle instabiliteit af, omdat je altijd terug kunt naar de vorige systeem status.

Het creÃ«eren van een overlay image, doe je als volgt:

qemu-img create -b [[base''image]] -f qcow [[overlay''image]]

Vervang de harddisk image = base_image door, in ons geval sidux.qcow. Hierna start je qemu middels het commando:

qemu [overlay_image]

De originele image wordt hierdoor niet gebruikt. Een probleem, de base image kan niet hernoemd of verplaatst worden, want de overlay onthoudt het volledige pad naar de base image.

Gebruik maken van een echte partitie als een enkele primaire partitie voor de hard disk image

Soms wil je een systeem partitie gebruiken met qemu (Bijvoorbeeld, als je wilt opstarten van je echte systeem of om qemu een speciale partitie te laten gebruiken als zijn root partitie). Je kunt dit doen door gebruik te maken van software RAID in lineaire mode (je hebt de linear.ko kernel driver nodig) en een loopback device: de truck is dat je dynamisch een master boot record (MBR) append aan een echte partitie welke je aan de qemu raw disk image wilt koppelen.

Stel je voor dat je een unmounted /dev/hdaN partitie met een bestandssysteem hebt welke je deel wilt maken van een qemu disk image. Als eerste moet je een klein bestand aanmaken waarin de MBR komt te staan, dit doen we met het volgende commando:

dd if=/dev/zero of=/path/to/mbr count=32

Hier wordt een 16 KB (32 * 512 bytes) bestand aangemaakt. Het is belangrijk om het bestand niet te klein te maken (ook al heeft de MBR maar 512 bytes nodig), omdat hoe kleiner het bestand is, deze te kleiner moet de chunk size van de software RAID device zijn, wat weer invloed kan hebben op de performance. Hierna maak je een loopback device aan in de MBR bestand, middels:

losetup -f /path/to/mbr

Laten we aannemen dat het resulterende resultaat het device /dev/loop0 is, omdat we toch nog niet van een andere loopback devices gebruik maakten. De volgende stap is om de "merged" MBR + /dev/hdaN disk image te gebruiken als software RAID:

 modprobe linear
 mdadm --build --verbose /dev/md0 --chunk=16 --level=linear --raid-devices=2 /dev/loop0 /dev/hdaN

De aangemaakte /dev/md0 is wat je gaat gebruiken als qemu raw disk image (vergeet niet om de rechten goed te zetten, zodat qemu bij het bestand kan). De laatste stap (een beetje moeilijke) is het zetten van de disk configuratie (disk geometry en partitie tabellen) zo dat de primary partitie start punt in de MBR overeenkomt met de /dev/hdaN in de /dev/md0 (een offset van precies 16 * 512 = 16384 bytes in dit voorbeeld). Doe dit door gebruik te maken van fdisk op de host machine, niet in de emulator: de standaard raw disc detectie routine van qemu resulteert vaak in een niet-kilobyte afgeronde offsets (zoals 31.5 KB, zoals weer gegeven in vorige sectie) dit kan niet gemanaged worden door de software RAID code. Doe het volgende vanaf het hostsysteem:

 fdisk /dev/md0

Hier creëeren we een enkele primaire partitie welke overeenkomt met /dev/hdaN, en spelen hier met de 's'ector commando van het 'x'pert menu totdat we de eerste cylinder (waar de eerste partitie start), overeenkomt met de grote van de MBR. Daarna doen we een , 'w'rite om het geheel op te slaan en je bent klaar. Je hebt nu een partitie die je direct kunt mounten vanaf je hostsysteem, en ook als onderdeel van je qemu disk image en wel als volgt:

 qemu -hdc /dev/md0 [...]

Je kunt natuurlijk op een veilige manier elke bootloader op deze disk image zetten voor het gebruik met qemu, vooruitgesteld dat de originele /boot/hdaN partitie de benodigde tools bevat.

QEMU Acceleratie Module gebruiken

De ontwikkelaars van qemu hebben een optionele kernel module gemaakt om qemu sneller te laten lopen, soms bijna op je echte pc snelheid. Deze module moet je laden met de optie:

major=0

Om het automatisch creëeren van de benodigde /dev/kqemu device uit te voeren, moet je het volgende commando gebruiken:

echo "options kqemu major=0" >> /etc/modprobe.conf

Dit zal het bestand modprobe.conf aanpassen om ervoor te zorgen dat deze module optie elke keer wordt gebruikt bij het laden van deze module.

qemu [...] -kernel-kqemu

Dit activeert de volledige virtualisatie en zal de snelheid behoorlijk verbeteren.

Om qemu te activeren doen we:

qemu -cdrom /tmp/pkg/sidux-debug.iso -net nic -net user -m 512
