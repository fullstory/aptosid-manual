nVidia 3d drivers

Er zijn 3 manieren om de 3D nVidia drivers op je systeem te installeren:

  1. install-binary-gfx

    Het nieuwe installatiescript voor de grafische drivers heet: install-binary-gfx. Om dat script te downloaden, typ in konsole als root:

    get-sidux-binary-gfx
install-binary-gfx -a (om de sid source te bouwen)

    Deze zal de goede driver voor jouw kaart uitzoeken, een .deb opbouwen, deze installeren, en je xorg.conf automatisch veranderen. Om de nieuwe driver goed te laten werken, moet je wel X opnieuw opstarten. (Ctrl+Alt+Backspace).

    Opties

    De volgende opties kunen gebruikt worden:

     Usage: ./install-binary-gfx [options]
-a | --auto            Automatische modus
-b | --build           alleen bouw van een module MOD Build only (nvidia|nvidia-legacy|fglrx)
-d | --detect          alleen detectie van de kaart 
-u | --update          update script
-v | --verbose         met meldingen

    Voorbeelden

    install-binary-gfx -a (hiermee worden je kaart gedetecteerd en de sid drivers automatisch geïnstalleerd)
install-binary-gfx -b nvidia (hiermee wordt de nvidia-module....deb in /usr/src aangemaakt. Dat werkt ook zo voor fglrx en nvidia-legacy.)
install-binary-gfx -d (hiermee worden je kaart gedetecteerd en de informatie afgedrukt.)
install-binary-gfx -u (hiermee wordt een update voor het script uitgevoerd.)

    Hulp bij problemen

    Als het script niet goed werkt voor je kaart, laat het a.u.b. in de debug mode lopen en stuur het log-bestand /var/log/sidux-CARD-installer.log naar het Sidux Forum. Een andere mogelijkheid is het droppen in http://paste.debian.net, ga naar de chat #sidux en plaats daar de link om hulp in IRC chat te krijgen.

  2. sgfxi / du-fixes-h2.sh

    cd /usr/local/bin ; wget -Nc http://techpatterns.com/sgfxi
chmod +x sgfxi

    sgfxi ondersteunt op dit moment ATI fglrx en Nvidia drivers. Verder kunnen conversies van en naar xorg free drivers uitgevoerd worden zoals ati, intel of nv. du-fixes-h2.sh gebruikt sgfxi als graphics driver installer omgeving, maar voegt eenvoudige opties toe voor de gebruikers.

    Met sgfxi -h kan de complete lijst voor de opties getoond worden. Na verloop van tijd zullen de opties geactualiseerd worden. Met de -h optie kan je bovendien een lijst van alle nu ondersteunde drivers zien en de systeem specificatie, kernel, xorg version afdrukken. De default driver wordt geïnstalleerd als verder geen opties/arguments worden gebruikt. Verder worden oudere kaarten, die een legacy driver nodig hebben, automisch geconfigureerd.

    Als het script gestart wordt, stopt de installatie even en informeert je welke driver geïnstalleerd wordt. Dat kan je accepteren of stoppen met de installatie en het script opnieuw uitvoeren met een andere instelling als je bij nader inzien de installatie nog wilt aanpassen.

    Of gebruik de h2 d-u-fixes. Gebruik de onderstaande optie om het het h2 script alleen de 3d drivers te installeren.

    du-fixes-h2.sh -kiwdt

  3. Gebruik de Nvidia binary installer

nVidia X.Org X server-Native nv display driver

Deze driver voor de X.Org X server (zie: xserver-xorg voor een uitgebreidere beschrijving) zorgt voor ondersteuning voor: NVIDIA Riva, TNT, GeForce, en de Quadro cards.

Om hem te installeren:

Verlaat KDE en ga naar tekst modus door: Ctrl+Alt+F1
log in als root
typ: init 3
apt-get update
apt-get install xserver-xorg-video-nv
init 5 && exit

Let er wel op dat dit niet dezelfde driver is als de binary-only 'nvidia' driver, die 3D ondersteuning heeft, maar binary-only en verder niet ondersteund wordt.

Daarna moet je /etc/xorg.conf veranderen, maar MAAK EERST EEN BACKUP!, want als de veranderingen niet goed werken, zit je zonder backup met de brokken.

Om /etc/xorg.conf te veranderen: rechtsklik erop, open de file met rootpermissies. Vind de sectie: SECTION DEVICE en verander de display driver naar lees nv

ATI X.Org X server-Native ati Driver

Deze driver voor de X.Org X server geeft ondersteuning voor de ATI Mach, Rage, Radeon, en FireGL series. Hij levert de 'atimisc', 'r128' en 'radeon' sub-drivers.

Om hem te installeren:

Verlaat KDE en ga naar tekst modus door: Ctrl+Alt+F1
log in als root
typ: init 3
apt-get update
apt-get install xserver-xorg-video-nv
init 5 && exit

Let er wel op dat dit niet dezelfde driver is als de door ATI geleverde, binary-only 'fglrx' driver, die toegevoegde 3D ondersteuning geeft aan sommige nieuwere Radeon kaarten, en dus niet door ATI ondersteund wordt.

Daarna moet je /etc/xorg.conf veranderen, maar MAAK EERST EEN BACKUP!, want als de veranderingen niet goed werken, zit je zonder backup met de brokken.

Native ATI - methode 1

Lees hier meer over bekende goed werkende configuratie-bestanden xorg.confs met de free radeon driver.

Noot: Hiervoor heb je informatie nodig over de configuratie van de tweede monitor. Om dat snel te doen, trek 1 monitor eruit en boot met de liveCD om een correct xorg.conf bestand aan te maken. Dat bestand kan gekopieerd worden. Hierna kan dezelfde actie met de andere monitor gedaan worden.

Native ATI - methode 2

Met deze methode, moet eerst de resolutie van de primaire monitor vastgelegd worden. Hierna kan de tweede monitor deze configuration voor de tweede gebruikt worden. Dat betekent dat de maximale/gewenste configuratie van de eerste monitor zo moet worden vastgelegd, dat de tweede monitor hiermee kan omgaan.

In de sectie

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

voeg in

Option "MergedFB"
Option "CRT2Position" "RightOf"

aan het einde:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Hier staat uitgebreidere informatie over de configuratie.

Tips voor hardware met 'non-free' benodigdheden

Vanwege de bepalingen in de verschillende wetten, zal sidux live-cd alleen dfsg-free software aanbieden en ondersteunen.

Hoewel sidux allen dfsg free software bevat, wil je misschien desondanks non-free software toevoegen aan je /etc/apt/sources.list als die nodig is.

# See sources.list(5) for more information, especialy
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • ATi Radeon graphics: 3d acceleration for older cards up to r35x should work, newer Radeon X1xxx cards need non-free drivers for accelerated performance which are not yet compatible with X.org 7.2.
  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • AVM ISDN/ ADSL PCI/ USB cards: AVM's closed source drivers are not compatible with kernel 2.6.21 yet.
  • Broadcom/ bcm43xx wlan: 
    apt-get install bcm43xx-fwcutter.
  • Eagle USB ADSL modem: gebruik de firmware van http://eagle-usb.org/ueagle-atm/non-free/ en plaats deze onder /lib/firmware/.
  • DVB firmware voor verschillende DVB TV kaarten (de meeste budget cards hebben dat niet nodig): download de nodige firmeware (controleer dmesg) via http://www.linuxtv.org/downloads/firmware/ en plaats het onder /lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945/ iwlwifi, 54 Mbit/s wlan: 
    apt-get install iwlwifi-ucode
    You can activate the old non-free driver, if iwlwifi creates problems: 
    apt-get update
apt-get install ipw3945d firmware-ipw3945
update-rc.d ipw3945d start 39 S . start 36 0 6
LANG= m-a a-i --text-mode ipw3945-source
echo "blacklist iwlwifi" > /etc/modprobe.d/mac80211
  • Intersil prism54, 54 MBit/s wlan: download de nodige firmeware http://prism54.org/firmware/ en plaats het onder /lib/firmware/.
  • nVidia graphics: 3d acceleration is niet beschikbaar met free drivers op dit moment, gebruik a.u.b. get-sidux-binary-gfx en gebruik scripts voor deze kaarten
  • RaLink rt61 54 MBit/s wlan,: 
    apt-get install rt61-fwcutter
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan, download de nodige firmeware http://www.kazer.org/acx-firmware-20060207.tar.bz2 en plaats het onder /lib/firmware/.
  • ZyDAS zd1201 11 MBit/s wlan: 
    apt-get install zd1201-firmware
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Wij onderzoeken nog of wij sommige drivers kunnen intergeren in sidux. Helaas is de wettelijke status van deze drivers nogal ingewikkeld.

Schrijven naar NTFS-partitions met ntfs-3g

Wees gewaarschuwd: ntfs-3g is een vroege beta-driver; dus gebruik hem nooit zonder een externe backup, en gebruik hem dus ook niet op productiesystemen/voor je werk! Als je het toch doet, is het je eigen verantwoordelijkheid als je gegevens verloren gaan, dus: gebruik ervan is op eigen risico!

Open een shell en voer de volgende commando's in: Zie daarvoor: Partitioneren van je Harddisk - Partitie aanduiding

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit
exit

Je NTFS-Volume zou nu gemount moeten zijn met rw permissies en je zou er nu gegevens op moeten kunnen opslaan. Maar nogmaals: wees gewaarschuwd! Het is experimenteel!
