Binaire, niet-gratis drivers voor Nvidia / ATI

Er zijn 3 manieren om de 3D nVidia drivers op je systeem te installeren:

  1. sgfxi / smxi

    Gebruik voor de installatie het multi-function administration tool van de sidux community smxi/sgfxi. De naam is een afkorting voor sidux-maintenance. Hier zie je meer informatie over smxi en sgfxi Syntax Command Sets.

  2. Gebruik van de Nvidia/ATI binary installer
    Installeer vanaf de source van Nvidia of ATI

Open Source Xorg drivers voor nVidia, ATI, Radeon, Intel

Open Source Xorg driver voor nVidia, ATI, Radeon, Intel zijn standaard ge誰nstalleerd in sidux. Als je deze drivers hebt overschreven met closed source drivers, kan je dat ongedaan maken met sgfxi.

De drivers voor X.Org X server (zie xserver-xorg voor meer informatie) ondersteunen in NVIDIA Riva, TNT, GeForce en Quadro kaarten en de ATI Mach, Rage, Radeon en FireGLcards van ATI met 'atimisc', 'r128' en 'radeon' sub-drivers. Er wordt ook ondersteuning voor de Intel free series in 3d aangeboden.

Om hem te installeren:

Verlaat KDE en ga naar tekst modus door: Ctrl+Alt+F1
log in als root
typ: init 3
apt-get update
apt-get install xserver-xorg-video-nv
OF
apt-get install xserver-xorg-video-ati
OF
xserver-xorg-video-intel
init 5 && exit

Bij sommige kaarten wordt 3D ondersteuning aangeboden, standaard is de 2D weergave.

Daarna moet je /etc/xorg.conf veranderen, maar MAAK EERST EEN BACKUP!, want als de veranderingen niet goed werken, zit je zonder backup met de brokken.

Om /etc/xorg.conf te veranderen: rechtsklik erop, open de file met rootpermissies. Vind de sectie: SECTION DEVICE en verander de display driver naar nv OF ati.

JE BENT GEWAARSCHUWD!! XORG.CONF VERANDEREN IS HELEMAAL VOOR EIGEN RISISCO!!

Tips voor hardware met 'non-free' benodigdheden

Vanwege de bepalingen in de verschillende wetten, zal sidux live-cd alleen dfsg-free software aanbieden en ondersteunen.

Sources List

Hoewel sidux allen dfsg free software bevat, wil je misschien desondanks non-free software toevoegen aan je /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list or /etc/apt/sources.list als die nodig is.

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Schrijven naar NTFS-partitions met ntfs-3g

Wees gewaarschuwd: ntfs-3g is een vroege beta-driver; dus gebruik hem nooit zonder een externe backup, en gebruik hem dus ook niet op productiesystemen/voor je werk! Als je het toch doet, is het je eigen verantwoordelijkheid als je gegevens verloren gaan, dus: gebruik ervan is op eigen risico!

Open een shell en voer de volgende commando's in: Zie daarvoor: Partitioneren van je Harddisk - Partitie aanduiding

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Om de konsole te verlaten type: exit
exit

Je NTFS-Volume zou nu gemount moeten zijn met rw permissies en je zou er nu gegevens op moeten kunnen opslaan. But again, be warned! Use it in emergency situations, it is not recommended for use on a daily basis.
