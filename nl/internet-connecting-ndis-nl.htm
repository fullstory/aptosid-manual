Activeren van Ndiswrapper

sidux heeft alle benodigde pakketten reeds geïnstalleerd: de kernel module en de ndiswrapper-utils.

Je hebt een "Windows" inf-file van de driver nodig, vind dezen en laad het middels:

ndiswrapper -i driver.inf

Om te controleren of de installatie goedgegaan is, gebruik je de optie -l:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Het is belangrijk dat te zien is "driver installed" en "device present" zoals hierboven getoond. Als dat niet zo is, dan is iets niet correct geïnstalleerd en is geen WLAN-verbinding mogelijk. Controleer dan de de foutmeldingen met ndiswrapper -i en/of dmesg.

Laad hierna de ndiswrapper-module:

modprobe ndiswrapper

en controleer of het goed geïnstalleerd is middels commando:

iwconfig

controleer of het access-punt gevonden is:

iwlist wlan0 scan

hierna kun je de kaart configureren middels:

/usr/sbin/netcardconfig

Als je een update van je kernel uitvoert, moet je ndiswrapper herinstalleren. Pas het nummer van je kernel handmatig aan aan je gebruikte kernel:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Dan wordt het geladen met modprobe:

modprobe ndiswrapper

De Windows drivers moeten niet nogmaals geïnstalleerd worden omdat het driver gedeelte apart van de kernel module werkt.

Je kan ook ndiswrapper toevoegen aan /etc/modules om ndiswrapper tijdens het starten te laden.

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

You can also do this with the GUI control centre siduxcc in KDE-Start-Menu

Ndiswrapper Advanced

Ndiswrapper is een module waarmee in Linux Windows WLAN drivers gebruikt kunnen worden. Dat kan nodig zijn omdat sommige WLAN-card producenten geen open source of free driver aanbieden. Dat maakt de implementatie van de WLAN-card in Linux ingewikkeld. Ndiswrapper probeert dat probleem op te lossen door "wrapping" (omhullen) van de Windows driver en vertaalt output en input van de WLAN-card en Linux.

Identificatie van de nodige driver

Hiervoor is een goed werkende Windows driver voor de WLAN-card nodig. WLAN-cards hebben een groot aantal verschillende chip sets voor 1 model en de enge manier om de card correct te identificeren is met het device id (USBID/PCIID).

OM dat 'id' te verkrijgen wordt een terminal geopend:

Voor USB WLAN adaptor
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

Zoals in bovenstaand voorbeeld te zien is, is de USBID voor de D-Link Corp. [hex] DWL-122 adaptor 2001:3700.

Voor een (mini) PCI / PCMCIA WLAN adaptor
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

Hierboven wordt naast 'id' het schema vendor:device getoond en hierna de id 168c:0024 voor de Atheros card.

Hierna kan gezocht worden met: ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list of iets te vinden is voor het gevonden 'id'-nummer. Als je geluk hebt, zie je voor je WLAN-card een aanbevolen (recommanded) driver.

Je hebt de INF en SYS bestanden van driver nodig. Als het meezit is er een ZIP bestand dat eenvoudig met een unzip-programma uit lezen en te pakken is. Als er alleen een EXE bestand beschikbaar is, dan is het een self-extracting bestand en kan wine gebruikt worden om het uit te pakken. Is het een CAB bestand kan het uitpakken met cabextract gedaan worden. Geadviseerd wordt om alle bestanden uit te pakken in een nieuwe map in je eigen home-map of -partitie.

Activeren van Ndiswrapper en installeren van de driver

Ga naar de map met de windows driver bestanden:

bla@blubb:~$cd /folder/with/driver
bla@blubb:~$su
root@blubb:/folder/with/driver# ndiswrapper -i <driver.inf>
installing net5416 ...
root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Zoals eerder vermeld, het is belangrijk dat "driver installed" en "device present" te zien zijn. Als dat niet zo is, dan is iets niet correct geïnstalleerd en is geen WLAN-verbinding mogelijk. De volgende stappen werken dan ook niet.

Laden van de kernel module: 

root@blubb:/some/folder# modprobe ndiswrapper

Als je hierop geen reactie ziet, werkt alles goed. Als er meldingen komen, gebruik dmesg voor nadere informatie.

"iwconfig" laat dan een wireless device zien bij wlan0:

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

Als ndiswrappper standaard geladen moet worden, dan kan een opdracht voor ndiswrapper in /etc/modules, geschreven worden, met een tekstverwerker per hand of door: 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
Herinstallatie van Ndiswrapper na een kernel update

Na een kernel update moet ndiswrapper opnieuw in de kernel module geïnstalleerd worden. Het deb bestand is te vinden in /usr/src/kernel-downloads/ <kernel-version >/ndiswrapper-modules- <version >.deb. Bijvoorbeeld voor kernel 2.6.21.5-slh-smp-2 voer de dpkg als volgt uit:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Na de installatie moet ndiswrapper geladen worden door:

root@blubb:/some/folder# modprobe ndiswrapper

Een herinstallatie is niet nodig als de Windows driver niet afhankelijk zijn van de kernel module.

Basis informatie

Om een nieuwe netwerkcard te installeren kan netcardconfig of sidux-networkconfig gebruikt worden. Meer details (how-to) kunnen hier gevonden worden.

Als je een andere/nieuwere driver voor je WLAN-card ziet, test hem of hij geschikt is. Dat kan problemen veroorzaken op je systeem, een kernel panic in een worst case situatie maar je helpt hiermee het ndiswrapper project en anderen om de beste drivers voor verschillende WLAN-cards te vinden. Vergeet niet om te melden bij ndiswrapper list als de andere/nieuwere driver goed werkt.

Meer informatie staat in de wiki.
