Online komen / hoe configureer ik het netwerk

Als je beschikt over een DHCP server in je LAN en je computer is hiermee verbonden, terwijl je van de Live-CD aan het opstarten bent, dan zullen je netwerk-instellingen automatisch ingesteld worden - zo niet, dan moet je het sidux network configuration dialog starten middels een klik op Kmenu>sidux>Network/Internet>Networkcard configuration.

Of - als je het niet op de grafische manier wilt doen, dan moet je een terminal openen en het volgende commando's uitvoeren: 

netcardconfig

en volg dan de aanwijzingen.

For a HD install you can use 'Kmenu>sidux>Network/Internet>Networkcard configuration', and follow the prompts, or use a konsole, (as user$), with:

/usr/sbin/netcardconfig

and follow the prompts.

Activeren van Ndiswrapper

sidux heeft alle benodigde pakketten reeds geinstalleerd: de kernel module en de ndiswrapper-utils.

Je hebt een "Windows" inf-file van de driver nodig, vind dezen en laad het middels:

ndiswrapper -i driver.inf

laad de ndiswrapper-module:

modprobe ndiswrapper

en controleer of het goed geinstalleerd is middels commando:

iwconfig

controleer of het access-punt gevonden is:

iwlist wlan0 scan

hierna kun je de kaart configureren middels:

/usr/sbin/netcardconfig

Als je een update van je kernel uitvoert, moet je ndiswrapper herinstalleren. Pas het nummer van je kernel handmatig aan aan je gebruikte kernel:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Dan wordt het geladen met modprobe:

modprobe ndiswrapper

De Windows drivers moeten niet nogmaals geïnstalleerd worden omdat het driver gedeelte apart van de kernel module werkt.

Je kan ook ndiswrapper toevoegen aan /etc/modules om ndiswrapper tijdens het starten te laden.

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

Er is meer informatie in de wiki.

Desktop-PCs met 2 NIC's (Internet delen)

Het configureren om op internet te komen met een laptop (of een andere pc) met een minimale maar simpele oplossing voer je als volgt uit:

Open een terminal en wordt root-gebruiker via het "sux" commando om de volgende stappen te kunnen uitvoeren:

Configureer de eerste nic (netwerkkaart) zo dat je het internet kunt bereiken.

/usr/sbin/netcardconfig

Voor de tweede nic kiezen we een vaste (statische) IP-Adres in een prive domain, bv.

192.168.1.1

Configureer deze dan ook met

/usr/sbin/netcardconfig

Let OP: Je IP adres moet in dezelfde subnet zitten als het IP-adres die je verkregen hebt via de DHCP server.

Indien door een fout een Gateway gespecificeerd was tijdens de configuratie van je netwerkkaarten, dan moet je het volgende script uitvoeren:

remove-gateway

DHCP/DNS Instellen:

apt-get install dnsmasq

Je moet het bestand /etc/dnsmasq.conf wijzigen door het toevoegen van de regel:

dhcp-range=192.168.1.50,192.168.1.150

De rest laat je zo als het is.

De DHCP-Server op de Desktop-PC kan nu gestart worden middels 

/etc/init.d/dnsmasq start
.

Indien de clients ingesteld staan op DHCP, dan moeten ze nu automatisch een IP-Adres ontvangen.

Om de client's aanvraag te routeren naar het Internet moet je nog het volgende script opstarten:

/usr/sbin/masquerade

Na deze actie kan elke client het internet benaderen.

Om deze configuratie standaard te laden bij het opstarten van je Desktop-PC: Activeren we dnsmasq tijdens het boot-proces middels commando:

update-rc.d -f dnsmasq defaults

Start het masquerade-script op via:

echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Verbinding maken via een 56k inbel Modem

KDE heeft een grafische tool voor inbelmodems, genaamd KPPP Internet Dial-up Tool, deze kan gevonden worden in het K-menu en dan onder het menu Internet.

De applicatie heeft een interne help functie, welke je helpen kan om je modem zo in te stellen dat je een verbinding met het internet kan maken.

Firewalls

Een firewall is in principe niet nodig als je verbinding via een router wordt gemaakt. Maar een firewall heeft een belangrijke rol voor de veiligheid als de internetverbinding per ADSL-usb-modem of via een dial-up modem tot stand komt:

apt-get install guarddog
OR
apt-get install firestarter

Guarddog is een gebrukiersvriendelijke firewall voor KDE onder Linux. De beste manier om hiermee te beginnen is het lezen van de korte gebruikershandleiding.

Firestarter - een Open Source firewall met grafische omgeving voor Linux. [Als je de graphische omgeving verlaat, blijft de firewall actief op de achtergrond.]
