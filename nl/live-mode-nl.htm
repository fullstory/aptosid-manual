Live Mode root wachtwoord

LET OP: Waar je ook iets met root rechten uitvoert, weet wat je aan het doen bent!! Voor websurfen over het LAN is geen root toegang nodig.

Op de sidux LiveCD is geen root wachtwoord ingesteld. Als je een programma met root-rechten wilt uitvoeren dan kun je dit om meerdere manieren doen:

Stel een (tijdelijke) root-wachtwoord in:

Open een console/ terminal:

sidux@0[sidux]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

nu kun je dit wachtwoord gebruiken tijdens deze sidux-Live-sessie, bij herstart (v/d livecd) moet je hem opnieuw instellen.

Start het programma via

sudo
of
su-me
Druk op Alt+ F2, en enter " 
sudo

voer je gewenste programma uit bijvoorbeeld:

sudo gparted

start het programma via de root-shell:

Open een console/terminal:

sidux@0[sidux]$ su
root@0[sidux]# gparted (of wat je ook wilt opstarten...)

Om uit het konsole te komen type je:

exit

en dan

exit
(waarnaar het konsole zich zal gaan afsluiten)

Installing software whilst on a Live-CD

apt-get update
apt-get install your-preferred-package

Note: when you power off the Live-CD, no changes will be kept

Storing permanent changes to a HD using the Live-cd

Permanent $HOME

home  /dev/sda1/sidux.img  mount loopback file as /home/sidux

unionrw

De eerste kan aangemaakt worden middels cheatcode:

unionrw

Het unionrw cheat kan in verschillende manieren gebruikt worden:

  1. Verwijst naar een partitie op de harddisk
  2. Verwijst naar een loop bestand (zoals een ext2 loopback bestand)
  3. Verwijst naar een directory

Alle drie manieren werken om een permanente opslag van wijzigingen op te slaan op een harddisk naar je keuze - indien je altijd dezelfde cheatcode gebruikt, dan kun je een op maat ingestelde programma's selectie hebben. Dit slaat niet je /home op, om dit uit te voeren heb je de home cheatcode nodig.

Zoals genoemd zijn er drie cheatcode's , de derde is:

unionro

unionro

Deze is ongeveer hetzelfde als unionrw, maar je bent niet instaat om bestanden te overschrijven op de plek waar je ze hebt opgeslagen. Echter kun je een nieuwe unionrw en de unionro cheatcode gelijktijdig gebruiken. Niet iedereen vindt dit bruikbaar, maar bedenk wel dat dit allemaal experimenteel is. Bedenk ook dat geen enkele van deze cheatcodes genoemd wordt in het help bestand, welke je met F1 kunt oproepen bij het starten van sidux.

Hoe kan ik op NTFS-partities schrijven met ntfs-3g

Wees gewaarschuwd: ntfs-3g is in een beta driver, gebruik het nooit zonder je data gebackupped te hebben, en gebruik het niet op productie systemen! Doe je dit wel, dan is het op eigen risico!!!

Open een shell en voer de volgende commando's uit: Zie ook Partitioning your HD - Disk Naming

su
fix-unionfs
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit
exit

Nu is je NTFS-Volume gemount als rw en zou je er data op kunnen schrijven. Maar nogmaals wees gewaarschuwd het is experimentele software!
