aptosid-*.iso - Live Mode root wachtwoord

LET OP: Waar je ook iets met root rechten uitvoert, weet wat je aan het doen bent!! Voor websurfen over het internet is geen root toegang nodig.

aptosid-*.iso default password

Op de aptosid LiveCD is geen root wachtwoord ingesteld. Als je een programma met root-rechten wilt uitvoeren dan kun je dit om meerdere manieren doen:

Eenvoudige methode, gebruik :

sux
Stel een (tijdelijke) root-wachtwoord in

Open een konsole/terminal:

aptosid@0[aptosid]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
aptosid@0[aptosid]$

nu kun je dit wachtwoord gebruiken tijdens deze aptosid-Live-sessie, bij herstart (v/d livecd) moet je hem opnieuw instellen.

sudo wordt niet ondersteund bij installaties op een harde schijf, zie sudo en sux.

Een programma vanuit een root-shell laten draaien

Met sux kan je als root grafische programma's starten (Xapp privileges).

Open een konsole/terminal:

aptosid@0[aptosid]$ sux
root@0[aptosid]# gparted (of wat je ook wilt opstarten...)

Sommige KDE applicatie benodigen dbus-launch voor de applicatie-start:

dbus-launch <Application>

Een andere algemene optie voor alle Desktop Managers is:

Alt+F2 su-to-root -X -c <Application>

bijvoorbeeld:

Alt+F2 su-to-root -X -c gparted

Om de root-modus in de konsole te verlaten:

exit

ALT-F4 kan ook gebruikt worden of gebruik sluiten in het konsolevenster rechtsboven in de hoek.

Je moet het volgende doen in geval van een "lockout" op een aptosid-*.iso en de prompts volgen om het wachtwoord aan te maken :

alt+ctrl+F1
sudo passwd

Wanneer het wachtwoord actief is, doe :

alt+ctrl+F7

Installatie van software in de Live-CD modus

apt-get update
apt-get install your-preferred-package

Let op: als de Live-CD modus wordt beëindigd, worden deze installaties niet bewaard, behalve wanneer je fromiso and persist hebt geactiveerd.
