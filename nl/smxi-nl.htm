De syntax van !h2 smxi commando's

Basisfuncties van het smxi script

Het script bevat verschillende functies. Sommige instellingen voor de functies worden 1 keer aan de gebruiker gevraagd, andere functies vragen elke keer een keuze van de gebruiker. Na de systeem controle en wat vragen over de configuratie worden de onderstaande opties getoond:

  • Script information. In dit stuk worden je kernel, het laatste gebruik van het script voor de update van het systeem en nog aanvullende informatie getoond. Dat is met name voor systemen van belang die niet vaak een update uitvoeren (geadviseerd wordt: upgrade minimaal 1 keer per maand anders kunnen risico's ontstaan).
  • Kernel install. Hiermee wordt de laatste stable kernel als standaard voorgesteld. De uitgebreide opties geven je de mogelijkheid om een andere kernel-versie te kiezen (ook oudere versies). Als je smxi start met de -K optie, dan wordt de laatste "current testing kernel" in de lijst aangeboden als 1e "kernel in stable".
  • dist-upgrade waarschuwings sectie: Geeft live waarschuwingen van huidige en afgelopen sid problemen. Nadat je akkoord gaat (hint: sid is bijna nooit in het groen, probleem vrije status, het is meestal geel, veilig maar met fixes).
  • pre dist-upgrade fixes Voor de beschikbare fixes die voorafgaand aan de upgrade uitgevoerd moeten worden.
  • config file alerts: you will see a 'update these configuration files' information piece. Take a quick look at those, and if the upgrade asks if you want to keep or replace your config file for that item, say y instead of the normal n.
  • dist-upgrade section. Dit gedeelte bevat de foutcontrole en laat je weten of de laatste upgrade zonder problemen werd afgerond of dat tijdens de installatie foutmeldingen kwamen.
  • post dist-upgrade fixes De foutmeldingen worden hierdoor verholpen.
  • miscellaneous options including:
    • fixfonts - hiermee worden font fixing opties aangeboden. Dat is vaak nodig na de eerste installie van het systeem, later is dat bijna nooit meer nodig.
    • package-install - groepen voor package install opties, incl. non-free pakketten zoals flash en Google-Earth en Office installers. Note: De OpenOffice.org installer stelt language pack selections voor en een paar aanvullende programma's. Kijk in de lijst welke pakketten beschikbaar zijn.
    • package-removal - hierin worden verschillende groepen van pakketten getoond die je misschien wilt verwijderen, zoals wifi, Duitse onderdelen, etc.
    • clean-up-stuff - verschillende opruim-opties, zoasl kernel remover (NB: bewaar altijd de laatste goed werkende oudere kernel, hierdoor kan je 'een stap terug' doen als met de laatste kernel problemen optreden.)
    • miscellaneous-tweaks - verschillende Mozilla instellingen ook voor de omgang met verschilende bestanden. Lees de lijst voor de andere beschikbare scripts.
    • continue - hierme ga je verder na de graphics installer
    • quit - afsluiten van smxi als je geen verdere installaties wilt uitvoeren. NB: De graphics installer stelt ook de kde/init 5 optie voor. Hiermee kan je weer terug naar je grafische omgeving.
  • graphics installer - hiermee wordt de sgfxi graphics installer van sidux gestart om nvdia en ati fglrx proprietaire driver te installeren (ati heeft voor een aantal foutmeldingen in het laatste jaar gezorgd, maar in principe werken deze drivers). Verder wordt de installatie van de xorg driver aangeboden en de terugkeer naar je grafische omgeving x/desktop.

Handleiding voor de smxi script opties

Opties compleet: 

-deghiIklmrstuwx

of afzonderlijk: 

-giktw -f -x

Je kan het script met elke combinatie van deze opties uitvoeren.

Je kan dat gezamenlijk uitvoeren: -ik, of afzonderlijk, zoals: -w -x

Elke optie, of een groep van opties, moet beginnen met een min-streepje:

 -

Voorbeeld:

 smxi -ikw

Hierme sla je de systeem informatie, kernel installatie en de waarschuwingen over.

smxi script opties en het overslaan van delen van het script

De volgende opties staan het overslaan toe:

  • -d geen dist-upgrade als ook de w optie is geselecteerd.
  • -e geen bevestigingen zoals 'Please hit enter to continue'.
  • -g zonder graphics installatie.
  • -i zonder system information.
  • -k zonder kernel installatie.
  • -r zonder controle van de laatste keer dat het script werd uitgevoerd en ook het herstarten van het script. Dat wordt niet geadviseerd en is alleen zinvol als net een kernel installatie heeft plaatsgevonden en hierna een reboot moet plaatsvinden.
  • -s zonder delen van het script waarvoor een internetverbinding nodig is. Hierdoor verliest het script de meeste functies.
  • -S zonder controle van de internetverbinding. Als je zeker weet dat een verbinding actief is en alleen de test hiervan mislukt (deze optie wordt niet geadviseerd en het werkt niet altijd).
  • -t zonder de post dist-upgrade opties.
  • -w zonder de waarschuwingen tijdens de upgrade.

Voorbeeld:

smxi -kwid

hiermee ga je direct naar de misc opties na de dist-upgrade. smxi -ki de kernel installatie vragen en de systeem informatie worden niet getoond.

smxi uitgebreide opties

De volgende geavanceerde script opties zijn niet standaard maar kunnen in sommige gevallen zinvol zijn:

  • -I legt een andere init level vast voor de X restart. De syntax hiervoor is: -I 3
    Het cijfer is de init level waarmee het script de grafische omgeving X zal starten.
  • -K Advanced Kernel Install. Als nieuwere test kernels beschikbaar zjn, worden deze getoond in de kernel install opties. De installatie van de advanced kernels is voor eigen risico!
  • -l hiermee worden logs met tijd van alle update stappen en dist-upgrade data vastgelegd. De logfiles bevinden zich in /var/log/smxi/
  • -m om de debian mirrors in de /etc/apt/sources.list te veranderen.
  • -M Runs the option to change default kernel mirrors for smxi
  • -p gebruikt apt-get update zonder pdiffs. Dat is goed voor snelle internetverbindingen.
  • -P start het script met proxy settings voor de verbinding, gebruik de syntax:
    -P http://[<username><:password>@]<server><:port>
  • -U het script wordt geactualiseerd voordat de update start.
  • -x activeert extra script opties die niet bij de hoofdtaken behoren.
  • -h drukt het smxi script help menu af.
  • -v Prints version and system information, including distro version, current kernel, apt update and dist-upgrade last used, and last use of smxi.

Aanpassingen in de smxi script opties

Je kan het script ook andere dingen laten doen door de configuratie aan te passen in /etc/smxi.conf.

NB: Dat configuratie-bestand ontstaat pas nadat het script 1 keer is uitgevoerd. Als het nog niet bestaat kan het bestand handmatig aangemaakt worden.

  • sgfxi=<sommige niet standaard sgfxi opties>, bijvoorbeeld als je in /etc/smxi.conf de de onderstaande regel toevoegd:
    sgfxi=b

    dan wordt je nvidia installatie automatisch gebruikt met de optie t en de standaard-optie -c
    NB: Het zijn opties die door sgfxi ondersteund worden, niet door smxi. smxi voegt deze optie toe en stuurt deze door aan sgfxi als de graphics driver geïnstalleerd wordt. Extra toevoegingen worden niet ondersteund, zoals bijv.: -o 8.33.6. Gebruik dus alleen letter opties voor de sgfxi graphics installatie.
  • hold-install=<pakket-lijst gescheiden door ^> - Deze optie biedt de mogelijkheid om pakketten te integreren in de automatische hold/release functie. Hierdoor kunnen bekende probleme met upgrades van sommige pakketten 'geparkeerd' en andere pakketten beschermd worden (waarvan geen upgrade wenselijk is). Voorbeeld:
    hold-install=nano^libgtk2.0
    NB: Gebruik geen spaties maar alleen een ^ als pakket scheidingsteken!
Andere wetenswaardigheden in de console

Om te scrollen in een terminal/console venster gebruik: shift+pageup/pagedown om te zien wat buiten het bereik van je venster staat. Hiervoor is bepaalde waarde voor het aantal schermen/regels vastgelegd. Voor langzaam scrollen kan shift+pijl_up/pijl_down gebruikt worden.
