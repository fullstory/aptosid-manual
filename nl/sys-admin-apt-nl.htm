Een klein APT-kookboek

Wat betekent APT?

APT is een afkorting voor Advanced Packaging Tool en is dus een verzameling van programma's en scripts, die (de systeembeheerder) root bij de installatie en het beheer van Deb-pakketten ondersteunen.

Het "APT"-systeem heeft als basis hiervoor een bestand nodig /etc/apt/sources.list. Hierin staat informatie over de bronnen waaruit de pakketten gehaald moeten worden. Hieronder wordt voor 1 bestand de sidux installatie uitgelegd.

NB: De instelling van de servers kan als root veranderd worden, door de landen-code aan te passen zoals ftp.us, ftp.uk. In de meeste landen zijn locale Debian-mirrors beschikbaar. Hierdoor kan de gebruiker sneller downloaden en wordt tevens bandbreedte bespaard.

# See sources.list(5) for more information, especialy
# See sources.list(5) for more information, especialy
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.

# Unstable
deb http://ftp.nl.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.nl.debian.org/debian unstable main contrib non-free

# Testing
# deb http://ftp.nl.debian.org/debian testing main contrib non-free
# deb-src http://ftp.nl.debian.org/debian testing main contrib non-free

# Experimental
# deb http://ftp.nl.debian.org/debian experimental main contrib non-free
# deb-src http://ftp.nl.debian.org/debian experimental main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Om te kunnen beoordelen of recentere bestanden/pakketten beschikbaar zijn, wordt de eigen installatie vergeleken met de recente lijst van de bestanden op de Debian-server. Het programma apt-get gebruikt deze informatie, om onderlinge verbanden/afhankelijkheden op te lossen en op deze manier te garanderen, dat de gekozen pakketten werken. De actualisatie van deze datenbase wordt gedaan via het commando apt-get update.

# apt-get update
Get:1 http://sidux.com sid Release.gpg [189B]
Get:2 http://sidux.com sid Release.gpg [189B]
Get:3 http://sidux.com sid Release.gpg [189B]
Get:4 http://ftp.nl.debian.org unstable Release.gpg [189B]
Get:5 http://sidux.com sid Release [34.1kB]
Get:6 http://ftp.nl.debian.org unstable Release [79.6kB]

Hoe kan ik een nieuw pakket installeren?

Als de APT-databank geactualiseerd is en de naam van het pakket bekend is, dan is onderstaand commando voldoende (later wordt getoond, hoe een pakket te vinden is):

# apt-get install qemu

Building dependency tree... Done
The following extra packages will be installed:
  bochsbios openhackware proll vgabios
Recommended packages:

Verwijderen van een pakket:

De manier van werken komt grotendeels overeen met de installatie van een programmapakket. Verder is van belang dat bepaalde programmapakketten, die een onderling verband (afhankelijkheden) hebben, niet gewist worden. Dat wordt later bij deb-orphan uitgelegd.

# apt-get remove gaim
Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim gaim-irchelper
0 upgraded, 0 newly installed, 2 to remove and 310 not upgraded.
Need to get 0B of archives.
After unpacking 4743kB disk space will be freed.
Do you want to continue [Y/n]?
(Reading database ... 174409 files and directories currently installed.)
Removing gaim-irchelper ...
Removing gaim ...

In deze procedure blijven de configuratiebestanden staan. Deze bestanden kunnen zo later bij een herinstallatie van het programmapakket (in dit voorbeeld gaim) hergebruikt worden. Het kan zinvol zijn om de configuratiebestanden ook te verwijderen (bijv. als niet duidelijk is of het 'niet werken' aan de programma-versie of aan de configuratie ligt). Er wordt dan het volgende commando gebruikt:

# apt-get remove --purge gaim
Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim*
0 upgraded, 0 newly installed, 1 to remove and 309 not upgraded.
Need to get 0B of archives.
After unpacking 4678kB disk space will be freed.
Do you want to continue [Y/n]? Y
(Reading database ... 174405 files and directories currently installed.)
Removing gaim ...
Purging configuration files for gaim ...

In het voorbeeld werden de configuratiebestanden met het programmapakket gaim gewist, maar niet de programmapakketten, van welke gaim afhankelijk was (er zijn pakketten die door meerdere applicaties gebruikt worden - die moeten dus ook blijven staan).

Downgrading van een pakket

Het kan nodig zijn om terug te gaan naar een eerdere versie van een pakket (als een grote bug in de meest recente versie zit). De volgende stappen moeten bijv. voor kmahjongg worden doorlopen:

  1. Maak de testing sources beschikbaar in /etc/apt/sources.list 
    (verwijder de #) en daar moet dus staan 
    deb http://ftp.nl.debian.org/debian/ testing main contrib non-free
  2. apt-get install -t testing kmahjongg
  3. Zet het nieuwe pakket op hold door:
    echo kmahjongg hold|dpkg --set-selections
  4. Plaats het hekje # weer voor de testing sources in /etc/apt/sources.list

Als je de laatste versie van een pakket wilt installeren:

echo kmahjongg install|dpkg --set-selections
apt-get update
apt-get install kmahjongg

Upgrade/bijwerken van het geïnstalleerde systeem:

Als het complete systeem bijgewerkt moet worden, dan gebeurt dat via "dist-upgrade" . Hierbij moet worden opgelet, met welke programmapakketten een upgrade uitgevoerd wordt.

Voor upgrades moet standaard de grafische omgeving worden verlaten. De upgrade wordt in een tekstkonsole uitgevoerd. Het is echter mogelijk, de pakketten eerst te downloaden en later pas in de tekstkonsole te wisselen). Door deze kleine maatregelen kan men zich soms uren aan reparaties besparen.

(Alleen) downloaden:

# apt-get dist-upgrade -d

The -d allows you to download the dist-upgrade without installing it whilst in X from the konsole and allows you to do the dist-upgrade in init 3 at a later time. It wisely gives you the option to proceed or abort like this

apt-get dist-upgrade -d
xxx upgraded, 30 newly installed, 25 to remove and 2 not upgraded.
Need to get 342MB of archives.
After unpacking 3916kB of additional disk space will be used.
Do you want to continue [Y/n]?
Y
(Y will download the upgraded packages without touching the installed system)

After the 'dist-upgrade -d' has downloaded, and wish to complete the dist-upgrade, immediately or at a later time, please follow these instructions:

A.u.b. NOOIT "apt-get dist-upgrade" of "apt-get upgrade" met een actieve grafische omgeving X uitvoeren.

Voordat een upgrade wordt uitgevoerd, wordt aanbevolen de sidux-Homepage te bezoeken. Hier staan misschien de recente upgrade-waarschuwingen. Deze waarschuwingen zijn door het systeem Debian sid/unstable noodzakelijk. In de bestandenlijsten worden namelijk dagelijks nieuwe programmapakketten opgenomen.

Volgorde voor de upgrades: afmelden uit de grafische omgeving zoals de KDE desktop, met Ctrl+Alt+F1 in de tekstmodus wisselen; aanmelden als root, vervolgens "init 3" uitvoeren (zonder aanhalingstekens). Hierdoor komen wij in een omgeving zonder toeters en bellen. De volgende opdrachten zijn: apt-get update; apt-get dist-upgrade; fix-fonts. Verder kunnen de scripts voor de installatie van 3D-videokaartdrivers uitgevoerd worden (na een update van xorg en indien deze drivers gebruikt worden, is een herinstallatie van de 3D-videokaartdrivers verstandig). Na de upgrades wordt de opdracht "init 5 && exit" gegeven.

Voor de zekerheid nogmaals onze waarschuwing: a.u.b. NOOIT een dist-upgrade of upgrade met een grafisch programma zoals adept of kpackage uitvoeren! De eenvoudige manier om het systeem bij te werken is m.b.v. !h2 smxi, dat tegenwoordig smxi heet, de afkorting voor sidux-maintenance. 

apt-get update
apt-get install sidux-scripts
smxi

Voor nieuwe users, het !h2 smxi script kan je helpen bij een semi-automatische dist-upgrade.

Het script loodst je stap-voor-stap door de upgrade. Natuurlijk kan je ook zelf met apt-get een upgrade uitvoeren. Als je hierbij niet met init 3 werkt, moet je goed weten wat je doet, je bent bij deze nogmaals gewaarschuwd!

De redenen om niets anders dan apt-get voor een dist-upgrade te gebruiken

Pakket managers zoals adept, aptitude, synaptic en kpackage zijn niet altijd in staat om de grote hoeveelheid aan veranderingen in Sid-systemen volledig en correct te verwerken (veranderde afhankelijkheden, veranderde namen, script veranderingen, ...). Dat is geen fout van de ontwikkelaars van deze hulpmiddelen. Zij hebben prachtige programma's geschreven voor de Debian Stable systemen, maar zij zijn niet gemaakt voor de bijzonderheden van Debian Sid systemen.

Je kan voor het zoeken naar pakketten gebruiken wat je wilt, maar neem uitsluitend apt-get voor installeren/verwijderen/dist-upgrading.

Probleem 1: Pakket managers zoals adept, aptitude, synaptic en kpackage zijn 'non-deterministic' voor complexe pakketten. Als in het dagelijks geactualiseerde aanbod van Debian Sid en in verschillende externe bronnen van verschillende qualiteit (wij gebruiken of adviseren deze niet, maar zij zijn wel de realiteit op een groot aantal user systemen) grafische pakket managers gebruikt worden, kunnen problemen ontstaan die niet eenvoudig gerepareerd kunnen worden. Probleem 2: Deze grafische package managers gebruiken runlevel 5 (init 5) en/of werken in X. Als je dus een dist-upgrade in init 5 en/of X uitvoert (ook een 'upgrade' wordt niet aanbevolen), kunnen problemen voor je systeem ontstaan, misschien niet vandaag of morgen, maar de problemen komen vrijwel zeker op je systeem.

Apt-get doet eenvoudig wat het behoort te doen. Als er tijdens de installatie een probleem ontstaat (afhankelijkheden, ontbrekende pakketten), dan wordt je geïnformeerd voordat de installatie doorgaat. Als apt-get de helft van je systeem wil verwijderen dan moet de administrator (jezelf dus) er serieus naar kijken voordat je 'y' en enter gebruikt.

Dat is de reden waarom Debian builds apt-get gebruiken en geen andere package managers.

De uitzondering naast de handmatige tour via apt-get is !h2 smxi (oude naam, de nieuwe naam is 'smxi'-script). Het voordeel van het smxi-script is ook dat de upgrade/update van de verschillende delen van je systeem stap-voor-stap worden aangeboden. Je ziet bij het starten van het smxi-script overigens ook de melding 'wil je nu naar init 3'. Hiermee heb je nog de tijd om alle lopende programma's te stoppen voordat je verdergaat.
