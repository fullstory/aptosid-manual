Diensten/services starten in sidux

Services worden ge-update wanneer een dist-upgrade uitgevoerd wordt. Elke andere services welke voor de dist-upgrade waren gedeactiveerd worden nu weer geactiveerd.

Om een service sessie te starten, of om het managen welke services geactiveerd of gedeactiveerd moeten zijn tijden het opstarten voer je het volgende uit:

invoke-rc.d service start

of

rcconf

Opmerking voor rcconf: pijltjes gebruiken voor het maken van een selectie/scrollen (op/neer), spatiebalk voor selecteren, tab-toets om bij "Ok" of "Afbreken" te komen, Enter-toets om "Ok" of "Afbreken" uit te voeren.

Je kunt ook gebruik maken van sidux Control Centre

Het stoppen van een service of proces

pkill is erg handig, vooral om dat het goed leesbaar is en in zowel user als root mode kan werken in een terminal of via tty

pkill -n service

Als je niet zeker bent van de correcte spelling van de te killen proces of service, dan geeft pkill <tab> <tab> je een lijst

htop is ook een goed alternatief. (killall -9 is je laatste alternatief).

sidux runlevels - init

Hieronder staat de lijst van sidux operating system runlevels, let op: deze wijken iets af van debain stable runlevels:

  • init 0 zet de pc uit
  • init 1 single user mode
  • init 2 Multi-User mode zonder netwerk, en/of stoppen van X of niet in X gaan,
  • init 3 Multi-User mode met netwerk maar geen X-windows system, en/of stoppen van X of niet in X gaan
  • init 4 Om niet in X te gaan of om X te stoppen
  • init 5 Multi-User mode with netwerk in een draaiend X-windows systeem, en/or X te starten
  • init 6 herstarten van de PC

Als het rootwachtwoord vergeten is

Een vergeten rootwachtwoord kan niet hersteld worden, maar er kan wel een nieuw rootwachtwoord aangemaakt worden.

Hiervoor wordt de Live-CD gebruikt.

Nadat deze gestart is, moet de rootpartitie op de harde schijf door de user root gemount worden (bijv.: /dev/hdb2)

# mount /dev/hdb2

Hierna wordt de opdracht chroot voor de rootpartitie uitgevoerd (chroot=changed root).

# chroot /media/hdb2 passwd

Lettertypes in sidux

Correcte dpi waarden - grondgedachte hierachter

DPI waarden zijn moeilijk te raden, maar worden thans door X goed weergegeven, mits de schermgrootte (DisplaySize) door de physical screen dimensions in /etc/X11/xorg.conf "Monitor Section" wordt ondersteund.

De juiste resoluties en frequenties (refresh rates)

Elke monitor heeft zijn eigen optimale waarden-combinatie. Helaas tonen niet allen de juiste DCC waarden waardoor handmatige aanpassingen nodig kunnen zijn.

De juiste drivers voor de adapter

Sommige nieuwe ATI en Nvidia videokaarten werken gewoon niet met de free Xorg drivers. De enige oplossing is op dit moment de keuze voor een closed source driver. sidux kan deze niet leveren als pre-install vanwege de licenties.

De oplossingen voor deze driverproblemen zijn hier te vinden.

Standaard lettertypes, rendering and grootte

sidux (Debian) gebruikt (pre-selected) standaard free fonts die getest zijn om een optimale middenweg te vinden in de grote hoeveelheid aan opties. De eigen voorkeuren voor lettertypes kunnen de kwaliteit van de beeldconvertering verslechteren. Maar er is een aantal opties in Debian (naast de settings in kcontrol) die de weergave met andere lettertypes verbeteren. Elk lettertype heeft maar een paar echt perfecte groottes, de andere groottes kunnen goed zijn.

Nadat de lettertypes, grootte (of bitmap/truetype) of DPI aangepast zijn (in X of Firefox/Iceweasel), is het verstandig om de weergave hiervan op het scherm te 'herschrijven':

dpkg-reconfigure fontconfig-config

Als dat niet goed werkt, moet de huidige weergave (fonts cache) opnieuw geladen worden:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Firefox/Iceweasel pre-settings

GTK-based apps zijn in algemene zin lastig voor de kde defaults. Dat kan gecorrigeerd worden door:

apt-get install gtk2-engines gtk-qt-engine

In het systeemmenu (control center) is een nieuw gtk menu item toegevoegd. Hier kan gekozen worden voor clearlooks vervolgens behoren de lettertypes de kde fonts te gebruiken.

Als er stabiliteitsproblemen optreden (vroeger kwam dat voor, maar eigenlijk niet meer sinds 2006), wordt de verwijdering van purge gtk-qt-engine geadviseerd. De $HOME-voorkeuren worden hierdoor niet verwijderd en er zijn geen afhankelijkheden of problemen bekend als de gtk-qt-engine gewist wordt, dus:

apt-get remove gtk-qt-engine
of
apt-get remove --purge gtk-qt-engine

Dat kan de problemen met de lettertypes (font rendering) in gtk-prgramma's oplossen.

CUPS

KDE heeft een groot hoofdstuk in de KDE help over CUPS. Door dist-upgrades kunnen problemen met CUPS ontstaan, vandaar dat hieronder een oplossing voor CUPS-problemen wordt getoond:

modprobe lp
echo lp >> /etc/modules 
apt-get remove --purge cupsys cups
apt-get install cups 
	OF
apt-get install cups cupsys-driver-gutenprint hplip

Daarna in een browser (Konqueror,Icedove) in de adresbalk: 

http://localhost:631

Nu CUPS herstarten om er zeker van te zijn dat het actief is:

/etc/init.d/cups restart

Als CUPS via de GUI methode benaderd wordt, komt een dialog box en vraagt om je password. Hoewel de dialog box je user name toont, komt een foutmelding als je je user password invult. Je moet dus de user name veranderen in root en je root password invullen.

Meer informatie hierover is in het sidux forum te vinden.
