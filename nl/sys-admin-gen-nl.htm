Diensten/services starten in sidux

insserv (vanaf Pontos 2008-04 en later): om ge√Ønstalleerde diensten te starten/stoppen:

Lees zorgvuldig /usr/share/doc/insserv/README.Debian, de release notes en de man pages:

 $ man insserv
 $ man invoke-rc.d
 $ man update-rc.d
 google LSB headers

Voor de 'start' van diensten:

/etc/init.d/<service> start

Voor de 'stop' van diensten:

/etc/init.d/<service> stop

Voor een herstart:

/etc/init.d/<service> restart

Om diensten niet tijdens het opstarten van het systeem uit te voeren

update-rc.d <service> remove
[zal alle startup links verwijderen]

Om diensten at boot uit te voeren [niet altijd noodzakelijk]

update-rc.d <service> defaults
[zal de startup links aanmaken]

om de lijst te lezen welke services lopen:

ls /etc/rc5.d

S betekent dat de service gestart wordt.
K betekent dat de service niet gestart wordt.

Boot-Up Manager (bum) - Grafische services configuratie tool

Als de debian systeem start volgorde logica je niet geheel duidelijk en vertrouwd is, dan zou je niet met symlinks, permissies enzovoort moeten gaan spelen, om niet je systeem te verkoeien. Boot-Up Manager zal je helpen om de configuratie te automatiseren.

Boot-Up Manager is een grafische runlevel configuratie editor deze staat het je toe om bij een systeemstart aan te geven welke init services gestart worden. Het toont een lijst met alle services die tijdens een systeemstart gestart kunnen worden. Je kunt de services aan of uit zetten..

apt-get install bum

Om de Boot-Up Manager GUI te gebruiken voer je het volgende uit:

$ sux
password:
bum

Gedetaileerde documentatie van Boot-Up Manager (bum).

Als het rootwachtwoord vergeten is

Een vergeten rootwachtwoord kan niet hersteld worden, maar er kan wel een nieuw rootwachtwoord aangemaakt worden.

Hiervoor wordt de Live-CD gebruikt.

Nadat deze gestart is, moet de rootpartitie op de harde schijf door de user root gemount worden (bijv.: /dev/sdb2)

mount /dev/sdb2 /media/sdb2

Hierna wordt de opdracht chroot voor de rootpartitie uitgevoerd (chroot=changed root) and set a new password:

chroot /media/sdb2 passwd

Lettertypes in sidux

Correcte dpi waarden - grondgedachte hierachter

DPI waarden zijn moeilijk te raden, maar worden thans door X goed weergegeven, mits de schermgrootte (DisplaySize) door de physical screen dimensions in /etc/X11/xorg.conf "Monitor Section" wordt ondersteund.

De juiste resoluties en frequenties (refresh rates)

Elke monitor heeft zijn eigen optimale waarden-combinatie. Helaas tonen niet allen de juiste DCC waarden waardoor handmatige aanpassingen nodig kunnen zijn.

De juiste drivers voor de adapter

Sommige nieuwe ATI en Nvidia videokaarten werken gewoon niet met de free Xorg drivers. De enige oplossing is op dit moment de keuze voor een closed source driver. sidux kan deze niet leveren als pre-install vanwege de licenties.

De oplossingen voor deze driverproblemen zijn hier te vinden.

Standaard lettertypes, rendering and grootte

sidux (Debian) gebruikt (pre-selected) standaard free fonts die getest zijn om een optimale middenweg te vinden in de grote hoeveelheid aan opties. De eigen voorkeuren voor lettertypes kunnen de kwaliteit van de beeldconvertering verslechteren. Maar er is een aantal opties in Debian (apart from KDE>systemsettings ) die de weergave met andere lettertypes verbeteren. Elk lettertype heeft maar een paar echt perfecte groottes, de andere groottes kunnen goed zijn.

Nadat de lettertypes, grootte (of bitmap/truetype) of DPI aangepast zijn (in X of Firefox/Iceweasel), is het verstandig om de weergave hiervan op het scherm te 'herschrijven':

dpkg-reconfigure fontconfig-config

Als dat niet goed werkt, moet de huidige weergave (fonts cache) opnieuw geladen worden:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Firefox/Iceweasel pre-settings

GTK-based apps zijn in algemene zin lastig voor de kde defaults. Dat kan gecorrigeerd worden door:

apt-get install gtk2-engines system-config-gtk-kde gtk-qt-engine gtk2-engines-qtcurve

In System Settings >Appearance you will have a new menu item called GTK Styles and Fonts. Set 'GTK Styles' to use 'Clearlooks' and have the 'GTK Fonts' set to use 'KDE fonts' or experiment with the various options to suit your preferences.

Dat kan de problemen met de lettertypes (font rendering) in gtk-prgramma's oplossen.

CUPS

KDE heeft een groot hoofdstuk in de KDE help over CUPS. Door dist-upgrades kunnen problemen met CUPS ontstaan, vandaar dat hieronder een oplossing voor CUPS-problemen wordt getoond:

modprobe lp
echo lp >> /etc/modules
apt-get remove --purge cupsys cups
apt-get install cups
	OF
apt-get install cups cups-driver-gutenprint hplip

Daarna in een browser (Konqueror,Iceweasel) in de adresbalk: 

http://localhost:631

Nu CUPS herstarten om er zeker van te zijn dat het actief is:

/etc/init.d/cups restart

Als CUPS via de GUI methode benaderd wordt, komt een dialog box en vraagt om je password. Hoewel de dialog box je user name toont, komt een foutmelding als je je user password invult. Je moet dus de user name veranderen in root en je root password invullen.

Meer informatie hierover is in het sidux forum te vinden.

Geluid in sidux

Geluid staat standaard op uit in sidux.

De KDE versie gebruikt Kmix en XFCE gebruikt Mixer.

Meestal is het klikken op de luidspreker icon in de taskbar en het vinkje weghalen voor de 'Dempen box'.

Kmix

In Kmix moet je de opties die je aan wilt hebben voor een kanaal activeren, Kmix>Setting>Configure Channels. Or in a terminal:

$ kmix
XFCE

In XFCE start de mixer applicatie en voeg de opties toe via Multimedia>Mixer and click the Select Controls box. Or in a terminal:

$ xfce4-mixer

Alsamixer

Als je voor Alsamixer kiest, deze zit in het alsa-utils pakket:

apt-get update
apt-get install alsa-utils
exit

Set your preferred sound settings as $user from the terminal:

$ alsamixer

Zie ook de sidux wiki.
