Nieuwe kernels installeren

De sidux kernels zitten in apt. Selecteer aub de dichtsbijzijnde sidux repo mirror en verander je sources:

De kernels zijn beschikbaar in de volgende varianten:

  • sidux-686 sidux-686 - kernel voor de i686 Processor Familie met single/dual core of meerdere CPUs
  • sidux-amd64 kernel voor 64 bit sidux
De kernel installatie benodigd 4 handmatige stappen in een konsole:

1. In een konsole login als root:

apt-get update

2. Om de laatste kernel versie te installeren voer je het volgend uit:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. De kernel-update is nu klaar. Als je ze nodig hebt, voeg dan de laatste grafische drivers, ATI, NVIDIA of INTEL kaarten toe. Maar update eerst het script:

sgfxi -U

4.Nu herstart je je computer en boot je de nieuw geïnstalleerde kernel (Dit is de standaard boot instelling).

Om de 3D drivers te updaten start je:

sgfxi

Alternatief, tijdens de eerste herstart van de nieuwe kernel ga je direct naar runlevel 3 (did doe je door een 3 aan de Grub-Boot-regel toe te voegen) en draai je het programma sgfxi.

Meer gedetaileerde sgfxi informatie vindt je hier.

Geeft de nieuwe kernel problemen, dan kun je altijd herstarten met een oudere kernel versie.

Modules

Om te kijken welke modules je nodig hebt, kun je het volgende commando gaan uitvoeren welke je een lijst geeft van alle huidige modules, kopieer deze regel in je konsole/terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Om een volledige beschrijving van een module te verkrijgen kopieer je de volgende regel naar je konsole/terminal:

apt-cache search 2.6.*.slh.*-sidux

Om de benodigde modules te installeren voer je de volgende commandos uit (bijvooerbeeld voor virtual-ose, en qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (dit is een voorbeeld)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (dit is een voorbeeld)

Met de volgende commando's controleer je welke modules in de kernel geladen zijn:

ls /sys/module/
or
cat /proc/modules

BELANGRIJK: als je bepaalde driver modules nodig hebt, bijvoorbeeld coor wlan, dan moet de de benodigde .deb pakketten geïnstalleerd hebben. Bijvoorbeeld voor de Agere et131x ethernet adapter:

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb wordt alleen gebruikt als je de .deb pakket local hebt. Maar zelfs dan is, als deze modules bestaan in apt-cache, apt-get de beste optie voor de installatie.

Andere kernel installatie methodes

smxi

De smxi script biedt een geïntegreerde manier om kernels, drivers en een aantal andere systeembeheer scripts te installeren:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Je kunt ook kernels installeren via de grafische control center siduxcc: deze staat onder KDE-Start-Menu>System>sidux Control Centre. siduxcc biedt ook andere systeembeheer GUI applicaties.

Verwijderen van oude kernels

Na het succesvol installeren van de nieuwe kernel, kunnen de oude kernels verwijderd worden. Het wordt echter wel aangeraden om de oude kernels eerst een paar dagen te bewaren voor het geval er toch nog problemen optreden met de nieuwe kernel. Zo kan je via grub nog een oude kernel selecteren om daarmee te booten.

Oudere kernels kun je van het systeem verwijderen. Om dit te doen kun je het script kernel-remover gebruiken welke in het pakket sidux-kernelhacking zit. Dit script moet geïnstalleerd worden en stelt je in staat om (ook meerdere) kernels te selecteren die van je systeem verwijderd kunnen worden.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Alternatief: je kan siduxcc gebruiken, welke de kernel-remover script gebruikt.
