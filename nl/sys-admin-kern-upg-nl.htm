Nieuwe kernels installeren

De sidux kernels zijn hier te vinden en zijn in de onderstaande versies beschikbaar:

  • up - Kernel voor de i686 Processor Family
  • smp - Kernel voor de i686 Processor Family met dual core of meer CPUs
  • 64-smp - Kernel voor de AMD64- Processor Family, als je 64 bit sidux gebruikt

Het is verstandig om de kernel naam die je nodig hebt voor je processor op te schrijven (we noemen hem voor de eenvoud kernel.zip, maar de echte namen luiden zoals: kernel-2.6.20.4-slh-up-1.zip ), schrijf de geschikte versie voor je pc op.

Installatie als user
Er zijn maar 5 regels opdrachten in de konsole nodig:
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.20.4-slh-up-1.zip
unzip kernel-2.6.20.4-slh-smp-1.zip -d kernel-2.6.20.4-rc1-slh-smp-1
cd kernel-2.6.20.4-rc1-slh-smp-1
./install-kernel-sidux.sh (it will ask for your root password)
Installatie als root
sux
cd /usr/src
wget -Nc http://sidux.com/files/kernel/kernel-2.6.20.4-slh-up-1.zip
unzip kernel-2.6.20.4-slh-up-1.zip  -d kernel-current
cd /usr/src/kernel-current
sh install-kernel-sidux.sh

Hierna is een herstart van je computer nodig. De nieuwe kernel wordt bij het booten als standaard gebruikt. Als met de nieuwe kernel probleme optreden, kan je rebooten en een oudere kernel gebruiken (deze staat dan onder der nieuwe kernel in het grub-menu)

Belangrijk: als er bepaalde drivers nodig zijn, bijv. voor wlan, moet je de .deb packages gebruiken. De driveres staan in de uitgepakte map van de kernel.zip. Voorbeeld: als je madwifi nodig hebt. Hiervoor moet de .deb package voor madwifi geïnstalleerd worden: # dpkg -i madwifi-modules*.deb

De kernel-update is nu klaar. Als er 3D-drivers nodig zijn, voor grafische kaarten van ATI of NVIDIA, moet je eerst rebooten en de nieuwe kernel starten met runlevel 3. Hiervoor kan je 3 de Grub-Boot-Line veranderen en een 3 aan het einde toevoegen. Let op: als je grub in de grafische modus start, kan je deze door escape verlaten. Hierna kan elk item in het menu vie de lettertoets 'e' (edit) aangepast worden.

De halfautomatische methode voor de installatie van kernels gebruikt du-fixes-h2.sh.

Oude kernels verwijderen

Nadat een nieuwe kernel installatie gelukt is, kunnen de kernel.zip (en het kernel archive) verwijderd worden. Het wordt niet geadviseerd om dat direct op te ruimen. Je kan beter een paar dagen hiermee wachten en nagaan of alle programma's en apparaten prima werken. Als er problemen optreden, ben je namelijk in staat om met een oudere kernel van de grub-menu-lijst te booten.

Oudere kernels kunnen via een script uit het systeem verwijderd worden. Hiervoor kan een script gebruikt worden kernel-remover dat in de package sidux-kernelhacking staat! Het script laat je de kernels zien die verwijderd kunnen worden.
