Nieuwe kernels installeren

De sidux kernels zijn hier te vinden. Gebruik de dichtstbijzijnde locatie:

Main Mirrors
http://debian.tu-bs.de/project/sidux//kernel Germany
http://sidux.com/files//kernel Germany
http://sidux.net/files//kernel Germany
ftp://ftp.spline.de/mirrors/sidux/kernel/ Germany

These mirrors can be as much as 24 hours behind the main mirrors
http://www.mirrorservice.org/sites/sidux.com/sidux/kernel United Kingdom
http://ftp.heanet.ie/pub/sidux//kernel Ireland
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernel  TN, USA
http://ftp-mirror.internap.com/pub/sidux/kernel  GA. USA
http://ftp.tw.debian.org/pub/Sidux//kernel Asia
http://mirror.aarnet.edu.au/pub/sidux/kernel Australia
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernel South Africa

De kernels zijn in de volgende versies beschikbaar:

  • sidux-686 - Kernel voor de i686 Processor Familie met single/dual core of meer CPUs
  • sidux-amd64 - Kernel voor de AMD64- Processor Familie, als je 64 bit sidux gebruikt
Het kan in 4 stappen handmatige via konsole uitgevoerd worden:
apt-get update

1. Verkrijg de laatste kernel versie:

wget -O - http://sidux.com/debian/pool/main/l/linux-2.6-sidux 2>/dev/null | awk "/linux-image.*$(dpkg-architecture -qDEB_HOST_ARCH_CPU)/" | sed -e 's/^.*linux\-image\-//' -e's/_.*\.deb.*$//'

2. Zoek uit welke modules je nodig hebt. Het volgende commando geeft je een lijst van de huidige beschikbare modules, copy this line to your console/terminal::

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Voor een volledige beschrijving van elke module voer je het volgende commando uit, copy this line to your console/terminal::

apt-cache search 2.6.*.slh.*-sidux

3. Om de kernel, headers en de benodigde modules (bijvoorbeeld virtual-ose, en qc-usb) te installeren voer je het volgende uit:

apt-get install  linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686
apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686

4 Hierna is een herstart van je computer nodig. De nieuwe kernel wordt bij het booten als standaard gebruikt. Als met de nieuwe kernel probleme optreden, kan je rebooten en een oudere kernel gebruiken (deze staat dan onder der nieuwe kernel in het grub-menu)

De geladen modules in de kernel kunnen zo opgevraagd worden:

ls /sys/module/
of
cat /proc/modules

Belangrijk: als er bepaalde drivers nodig zijn, bijv. voor wlan, moet je de .deb packages gebruiken. De driveres staan in de uitgepakte map van de kernel.zip. Voorbeeld: als je madwifi nodig hebt. As an example, for the Agere et131x ethernet adapter: 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb wordt alleen gebruikt wanneer je de debs lokaal beschikbaar hebt, maar zelfs in dit geval, als ze in apt-cachce staan is, apt-get een betere optie.

Zie ook Online komen en Activeren van Ndiswrapper en voor WIFI

De kernel-update is nu klaar. Als er 3D-drivers nodig zijn, voor grafische kaarten van ATI of NVIDIA, moet je eerst rebooten en de nieuwe kernel starten met runlevel 3. Hiervoor kan je 3 de Grub-Boot-Line veranderen en een 3 aan het einde toevoegen. Let op: als je grub in de grafische modus start, kan je deze door escape verlaten. Hierna kan elk item in het menu vie de lettertoets 'e' (edit) aangepast worden.

De halfautomatische methode voor de installatie van kernels gebruikt het script du-fixes-h2.sh (oude naam), tegenwoordig heet het script smxi, een afkorting voor sidux-maintenance:
apt-get update
apt-get install sidux-scripts
smxi

Je kunt dit ook doen via de grafische control centre siduxcc tool in KDE-Start-Menu

Oude kernels verwijderen

Nadat een nieuwe kernel installatie gelukt is, kunnen de kernel.zip (en het kernel archive) verwijderd worden. Het wordt niet geadviseerd om dat direct op te ruimen. Je kan beter een paar dagen hiermee wachten en nagaan of alle programma's en apparaten prima werken. Als er problemen optreden, ben je namelijk in staat om met een oudere kernel van de grub-menu-lijst te booten.

Oudere kernels kunnen via een script uit het systeem verwijderd worden. Hiervoor kan een script gebruikt worden kernel-remover dat in de package sidux-kernelhacking staat! Het script laat je de kernels zien die verwijderd kunnen worden.

kernel-remover
