Opmerkingen voor upgrades van geïnstalleerde systemen

sidux probeert zo goed mogelijk een weg te bereiden voor rolling dist-upgrade als het gaat om de eigen pakketten van debian en sidux. Een paar dingen vragen een handmatige aanpassing.

Deze pagina wordt geactualiseerd na elke release, de actuele release is Pontos.

sidux beveiligingadvies / bescherming van $HOME

Tijdens de actualisatie van de sidux installer werd een beveiligingsprobleem (local privilege escalation) ontdekt, omdat een kopie van het tijdelijke configuratiebestand achterbleef op het nieuw geïnstalleerde system. Dat configuratiebestand bevat de salted MD5 password hashes voor root en de eerste user account (UID 1000) en is door locale users leesbaar. Hoewel deze passwords niet direct openbaar zijn, kan het configuratiebestand misbruikt worden bij een brute force attack.

Releases waar het probleem voorkomt:

  • 2007-01 (Chaos)
  • 2007-02 (Tartaros)
  • 2007-03/ 2007-03.1 (Gaia)
  • 2007-04/ 2007-04.5 (Eros)
  • 2008-01 (Nyx)
  • 2008-02 (Erebos)
  • 2008-03 (Ourea)

Correctie:

find /root /home -maxdepth 2 -type f -name '.sidconf*' -delete

Vooral bij multi-user systemen wordt geadviseerd om de passwords voor root en de eesrte user (UID 1000) te veranderen. Dat kan gedaan worden met "passwd" als root/ user. De oorzaak voor het probleem werd gevonden en hierdoor komt het bij sidux 2008-04 Pontos niet meer voor.

Debian maakt de home directories aan met 0751 rechten (access permissions), sidux wijkt hiervan af en gebruikt 0700 rechten voor nieuwe installaties. 

sed -i 's/^DIR_MODE=.*/DIR_MODE=0700/' /etc/adduser.conf
dpkg-reconfigure adduser
(vermijden van "system-wide leesbare home directories")

ALSA OSS compatibiliteitslayer

Vanaf de alsa-driver 1.0.17.dfsg-3 worden de OSS compatibility modules niet meer automatisch geladen. Hoewel dat meestal voor moderne Linux systemen geen probleem is, vereisen sommige commerciële of legacy applicaties nog OSS support. Omdat ALSA's in-kernel emulatie voor OSS beperkingen heeft (geen software based mixing, hierdoor kan slechts 1 applicatie /dev/dsp gebruiken), genieten ALSA compatibele applicaties de voorkeur. Maar als OSS compatibiliteit nodig is, zijn twee alternativen beschikbaar:

  • aoss wrapper
    /usr/bin/aoss maakt deel uit van het alsa-oss pakket en kan "wrap around" legacy applicaties. De OSS oproepen woorden door "transparently mapping" van OSS ABI naar ALSA gestuurd. Hierdoor kunnen de meeste ALSA functies gebruikt worden.
  • in-kernel ALSA emulatie
    regelt een betere binary compatibiliteit (/dev/dsp, /dev/audio en /dev/mixer device nodes worden gecreëerd). Maar hiermee is geen software based mixing mogelijk en er kan maar 1 sound applicatie gebruikt worden.

Voor de meeste applicaties die hiermee te maken hebben, wordt geadviseerd deze via "aoss <program>" te starten om de voordelen van software based mixing en gelijktijdige toegang tot de sound hardware te gebruiken. Maar in sommige situaties waar de in-kernel OSS compatibiliteitslayer nodig is, kan deze geactiveerd worden door de snd-pcm-oss, snd-mixer-oss en snd-seq-midi modules te laden. 

for i in snd-pcm-oss snd-mixer-oss snd-seq-midi; do
	grep -q "^${i}" /etc/modules  || echo "${i}" >> /etc/modules
	grep -q "^${i}" /proc/modules || modprobe "${i}"
done

Broadcom b43 wlan

Kernels >2.6.25 benodigen firmware versies ≥4.150.10.5 (internal revision "410.2160 (2007-05-26 15:32:10)") van de Broadcom firmware. Oudere kernels (tot kernel 2.6.24) benodigen 4.80.53.0 (internal revision "351.126 (2006-07-29 05:54:02)") die geïnstalleerd werd bij oudere b43-fwcutter versies. b43-fwcutter kan geen upgrade van geïnstalleerde firmware images maken, de upgrade moet handmatig geïnstalleerd worden. 

dpkg --purge b43-fwcutter
rm -rf /lib/firmware/b43 /lib/firmware/b43legacy
apt-get update
apt-get install b43-fwcutter

console-setup

De console keymaps worden nu met "console-setup" geïnstalleerd. 

apt-get install console-setup
dpkg-reconfigure console-setup

gspca2

In kernel 2.6.27 zit de nieuwe (v4l2 based) gspca2 driver. Deze vervangt de oudere v4l1 based externe gspca driver, die voor een groot aantal webcams in gebruik is. Helaas gebruiken de meeste userspace programma's nog niet de nieuwe v4l2 API voor devices die voorheen met gspca gebruikt werden. Voorlopig is het nodig om deze programma's met "LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <program>" te starten. 

apt-get update
apt-get install libv4l-0

AS USER
$ LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <program>

insserv

insserv bij geïnstalleerde systemen vóór Pontos 2008-04

LET OP: Oudere installaties worden niet automatisch van insserv voorzien, vanwege de users eigen initscripts of symlinks. Het wordt NIET geadviseerd om insserv te gebruiken bij installaties vóór Pontos. Dit pakket dient met voorzichtigheid gebruikt te worden, omdat incorrecte of ontbrekende LSB afhankelijkheden van bestaande initscripts tot een unbootable systeem kunnen leiden. Voor de installatie van insserv bij bestaande systemen zijn soms handmatige aanpassingen nodig.

Lees de Debian documentatie opmerkingen en de release notes vóórdat insserv geïnstalleerd en geactiveerd wordt.

Overbodige pakketten

Als gevolg van de verdere ontikkeling van sidux is een aantal pakketten vervangen door betere alternatieven of werd overbodig.

  • capi-isdnconf, overbodig.
  • configure-xawtv, overbodig.
  • gprsconnect, vervangen door umtsmon.
  • ipw2100-firmware, vervangen door firmware-ipw2x00.
  • ipw2200-firmware, vervangen door firmware-ipw2x00.
  • modemlink, vervangen door umtsmon en wvdial.
  • ppp-scripts, vervangen door umtsmon.

Het volgende commando zoekt naar pakketten die niet meer beschikbaar zijn in een actief apt repository en kan overbodige pakketten tonen. Hierbij kunnen ook handmatig geïnstalleerde pakketten getoond worden (apt-show-versions is een pakket dat niet standaard op alle systemen geïnstalleerd is). 

$ apt-show-versions | awk '/No available version in archive/{print $1}'
