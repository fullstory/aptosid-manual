sidux Snel Start handleiding

HEEL BELANGRIJK: sidux, als Linux LIVE-CD, werkt met een hoge compressie technologie, en daarom is het verstandig om goed op te letten bij het downloaden en branden van het ISO image-bestand. Gebruik hiervoor goede CD-media [of DVD+RW], brand in DAO-modus (disk-at-once) en niet sneller dan 8x.

sidux probeert 100% compatibel te zijn aan Debian sid, echter kan sidux van tijd tot tijd tijdelijke hot-fixes bevatten. De sidux apt-repository bevat specifieke sidux pakketten welke bestaan uit de sidux kernel, scripts, tools en documentatie.

Over de stabiliteit van Debian Sid

Sid is de benaming voor de onstabiele repositories van Debian. In Debian Sid komen heel frequent nieuwe software versies beschikbaar. Het voordeel van de snelle beschikbaarheid van de laatste software in Debian Sid heeft ook risico's. Vanwege de snelheid van updates van de ontwikkelaars, kunnen de pakketten in het algemeen minder uitgebreid getest worden.

sidux kernel

De kernel is voor sidux geoptimaliseerd om de volgende doelen te bereiken: functionaliteit verbeteren of toevoegen, betere performance, betere stabiliteit. De basis is altijd de laatste kernel van http://www.kernel.org/.

Meer uitleg over kernel is hier te vinden: Opwaarderen van de kernel

Pakket Management

sidux is compliant with Debian packaging and uses apt and dpkg for software package management with debian and other repositories identified by the files in /etc/sources.list.d/*

Debian sid has over 20,000 packages, so you should be able to find the one you would like to use with Package searching with apt-cache or with Debian Package Search GUI application.

Om een pakket te installeren voer je uit: apt-get install <pakketnaam> Installeer een nieuw pakket

Debian Sid's repositories worden 4 times per dag geupdate. Hierdoor kan de lijst van pakketten en hun versienummers snel out-of-date raken. Het uitvoeren van apt-get update is essentieel voordat je welk nieuw pakket dan ook gaat installeren. Dat is ook noodzakelijk voordat je een apt-get dist-upgrade gaat uitvoert om zo up-to-date te blijven met de pakkettenlijst op de servers.

Using other Debian based distributions' repositories, Source and RPMs

Source installaties worden niet ondersteund. Als je echt een applicatie moet compileren, doe het dan als een gebruiker en zet deze apllicatie in je home directory. Gebruik maken van checkinstall en het converteren van RPMs met alien (of soort gelijke tools) naar een deb-bestand worden niet ondersteund.

Debian based distributions' often hebben meestal een andere bestandslocatie voor diverse applicaties welke voor systeem instabiliteit kan zorgen. Sommige pakketten willen niet meer installeren omdat ze onopgeloste afhankelijkheden van de pakketbenaming of andere versienummers hebben. Voorbeeld: een andere versie van glibc kan veroorzaaken dat een applicatie niet wil opstarten die gebruik maakt van glibc.

Opwaarderen van het systeem - dist-upgrade

apt-get dist-upgrade is de gebruikelijke manier voor het upgraden van sidux. Gebruik maken van een grafische client om sidux te updaten wordt niet ondersteund. Upgraden van een geïnstalleerd Systeem - dist-upgrade

Een dist-upgrade wordt alleen ondersteund buiten de grafische X-omgeving. Ga naar een terminal (ctrl+alt+f1, ctrl+alt+f2 etc. kan ook) en type init 3. Hierdoor zal X stoppen, zodat je veilig kan upgraden.

Netwerk Configuratie

Ceni is een netwerk configuratie tool om met weinig moeite je netwerk of draadloos netwerkkaart te configureren. De draadloze functie kan zoeken naar netwerken, gebruik maken van wep en wpa voor encryptie en gebruikmaken van wireless-tools of wpa_supplicant voor de draadloze configuratie. Ethernet is recht-toe-recht-aan als je gebruikmaakt van dhcp (automatische ip-adres toewijzing), maar je kan het ook gerbuiken voor het handmatig instellen van netmasks tot aan nameservers.

Ceni wordt gestart middels het commando Ceni of ceni . Indien het niet geïnstalleerd is, dan kan je dat doen m.b.v.: apt-get install ceni.

Meer informatie over het online komen met Ceni is hier te vinden.

Andere Window Managers

KDE, XFCE en fluxbox zijn standaard in sidux geinstalleerd. Gnome wordt niet door sidux ondersteund. Gnome als een window manager in sid repositories staat erom bekend om onstabiel te zijn en veroorzaakt op dit moment problemen. Er zijn gebruikers in het sidux forums /wiki en op de IRC chat die ervaring met Gnome hebben en die willen je misschien wel helpen (anders kan je altijd zelf proberen).
