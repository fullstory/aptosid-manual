Enkele handige KDE applicatie's die niet voorgeinstalleerd zijn op de sidux KDE Lite versie

Je moet de non-free ingeschakeld hebben in /etc/apt/sources.list.d/debian.list

konq-plugins - plugins voor Konqueror.

Dit pakket bevat een variatie van handige plugins voor Konqueror, de file manager, web browser en document weergever voor KDE. Vele van deze plugins zullen in Konqueror's Tools menu verschijnen.

Hoogepunten voor web browsen inclusief webpagina vertaling, webpagina archivering, auto-verversen, HTML en CSS structurele analysische, een zoek toolbar, een sidebar nieuws ticker, snel toegang to algemene opties, bladwijzers, een crash monitor, een microformaat beschikbaarheids indicator, een del.icio.us bladwijzer sidebar, en integratie met aKregator RSS feed lezer.

Hoogtepunten voor het verkennen van directories inclusief filters, image gallery creeeren, archiveren compressen en uitpakken, snel kopieren/verplaatsen, een sidebar mediaplayer, een file informatie metabar/sidebar, een mediafolder helper, een grafische diskverbruik tool en image converter en vervormers.

kdeaddons - add-on plugins en applets geleverd met KDE

Deze metapackage bevat een collectie van add-on plugins en applets welke geleverd worden bij de officiele KDE release.

unrar - Uitpakker voor .rar bestanden (niet-gratis versie)

Uitpakker voor .rar bestanden (niet-gratis versie) Unrar kan .rar bestanden uitpakken. Indien je .rar bestanden wilt maken, installeer dan het pakket rar.

Desktop 'bevriest'

In een dergelijke situatie heb je niet altijd de reset button nodig. Dit kan tenslotte je bestandssysteem beschadigen of leiden tot dataverlies. In elk geval zal het bestandssysteem niet 'schoon' zijn na een harde reboot ("filesystem not clean")

Probeer eerst naar een tekstconsole te gaan met: alt-ctrl-F1 of start de X-server opnieuw op met: alt-ctrl-backspace. Ook als geen van deze twee opties werkt, is er nog steeds hoop:

De SysRq-toets (print-toets, aan de rechterbovenkant van het toetsenbord) kan je helpen om een 'bevroren' desktop netjes te rebooten.

De hieronderstaande toetsencombinaties zijn mogelijk:

* ALT+SYSRQ+R (geeft je de controle terug over je toetsenbord)
* ALT+SYSRQ+S (leidt tot een sync)
* ALT+SYSRQ+E (stuurt term naar alle processen behalve init)
* ALT+SYSRQ+I (stuurt kill naar alle processen behalve init)
* ALT+SYSRQ+U (bestandssysemen worden slechts readonly gemount, voorkomt fsck tijdens reboot)
* ALT+SYSRQ+B (reboot het systeem; zonder de vorige stappen zou dit een harde reboot zijn).

Geef elke stap een paar seconden de tijd om uitgevoerd te worden; het beëindigen van alle processen duurt bijvoorbeeld wel een tijdje. De hierboven genoemde letters kun je onthouden met het volgende, engelse, ezelsbruggetje: "Reboot System Even If Utterly Broken"

Loginproblemen voor KDE

Normaal gesproken wordt de inhoud van de /tmp directory verwijderd bij elke boot. Sommige directories en sockets die de X-server nodig heeft, worden dus ook verwijderd.

Gewoonlijk worden die door het script 'x11-common' voor X-Org tijdens het boot-proces gewoon aangemaakt.

Het kan zijn dat dit script niet wordt aangeroepen tijdens het boot-proces. Om deze links dan toch te maken, typ je in de console: 

# X-ORG: # dpkg-reconfigure x11-common

KDE heeft 5% ruimte nodig op de partitie waar de directory /tmp ligt, dit omdat er tijdelijke bestanden tijdens het inloggen worden aangemaakt. Dus als je partitie 95% vol is, kun je niet inloggen met KDE en kom je terug in terminal mode (tty).

Hetzelfde geldt voor kdm looping welke je niet toestaat om in te loggen. Een oplossing is om via tty (terminal) in te loggen om ruimte te creëren, door het verwijderen van applicaties / bestanden die je niet meer gebruikt.

Een ander alternatief is een andere X window manager gebruiken die niet zoveel ruimte van het systeem vereist ( bijvoorbeeld fluxbox is ook standaard geinstalleerd bij een sidux install), of doe een chroot gebruikmakend van de sidux live-CD/DVD om de partitie op te schonen, zodat je wel tijdens het booten KDE kan opstarten.

Maximaal een 85% volle partitie wordt aangeraden om KDE te kunnen gebruiken, zodat deze nog 15% ruimte heeft in de /tmp directory voor zijn tijdelijke bestanden.

Installeren van sidux KDE Art en Thema's

Om de laatste sidux-art te installeren op een bestaande sidux installatie voer je het volgende uit:

apt-get install sidux-art-kde-xxxx sidux-art-wallpaper-xxxx
(Where xxxx is the name of the release for example sidux-art-kde-erebos)

Dit installeerd de sidux wallpaper en thema's

Ga naar je Control Centre > Appearance and Themes > Theme manager en kies sidux-art theme/wallpaper uit deze lijst.

Hoe kan ik een KDM-thema veranderen / actief maken

kdmtheme is een themamanager voor KDM. Het levert een KDE Control Module (KOM) die je in staat stelt om makkelijk je KDM-thema's te installeren, te veranderen of te verwijderen. 

apt-get install kdmtheme

Herstart hierna de Control Centre

Thema's handmatig mogelijk maken

KDM-thema's zijn niet standaard ingesteld in Debian. 

 
Voeg:
UseTheme=true
toe in de [X-*-Greeter]-stanza in je /etc/kde3/kdm/kdmrc bestand.
Thema's veranderen

Het standaard-thema van KDM is: cirkels. Om dit te veranderen, wijzig je alleen maar de regel: 

Theme=/usr/share/apps/kdm/themes/circles

in:

Theme=/usr/share/apps/kdm/themes/
Thema's niet (meer) toestaan

Als je geen thema's meer wilt gebruiken, verander je: UseTheme=true gewoon weer in: UseTheme=false

OPMERKING: Als je 'kdmrc' hebt veranderd, moet je wel opnieuw 'kdm' starten.

In de sidux wiki zijn richtlijnen voor het gebruik van KDE Themes te vinden.
