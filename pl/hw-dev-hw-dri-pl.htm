Wskazówki dla sprzętu potrzebującego niewolnego oprogramowania

Ze względu na złożoną sytuację prawną, sidux live-cd będzie zawierał tylko oprogramowanie wolne w sensie dfsg.

Sources List

Ponieważ sidux zawiera tylko oprogramowanie wolne w sensie dfsg, prawdopodobnie będziesz chciał dodać contrib/ non-free do /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list or /etc/apt/sources.list:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Binary, closed source drivers for: nVidia with dmakms

Prepare the module:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Then activate Dynamic Module-Assistant Kernel Module Support (dmakms) for nvidia, so that the next time the Linux kernel is updated a nvidia module is prepared for it too, without manual intervention. To do that, add 'nvidia-kernel-source' to the /etc/default/dmakms configuration file. Then:

m-a a-i nvidia-kernel-source

Next alter xorg.conf:

sed -i s/nv\"/nvidia\"/ /etc/X11/xorg.conf

Reboot PC for installation of the module to take effect.

Read:

$ /usr/share/doc/dmakms

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel are pre-installed with sidux.

Sterowniki dla X.Org X serwer (patrz: xserver-xorg dla dalszego opisu) dostarcza wsparcie dla 2d kart NVIDIA Riva, TNT, GeForce i Quadro oraz kart ATI Mach, Rage, Radeon i FireGL wraz z substerownikami atimisc', 'r128' i 'radeon'. Również udostępnia 3d dla kart Intela.

Aby go zainstalować:

Przełącz się z KDE do trybu tekstowego poprzez Ctrl+Alt+F1
zaloguj się jako root
i wpisz: init 3
(Powinno to spowodować, że X przestanie działać)
apt-get update
apt-get install xserver-xorg-video-nv
LUB
apt-get install xserver-xorg-video-ati
LUB
xserver-xorg-video-intel
init 5 && exit

For some cards it may add 3D support, however it is usually 2D.

If you have been running proprietary drivers and wish to revert back to opensource drivers you need to edit /etc/X11/xorg.conf, HOWEVER YOU MUST MAKE A BACKUP COPY FIRST, as editing /etc/X11/xorg.conf incorrectly will result in many tears by you.

Aby edytować /etc/X11/xorg.conf, kliknij prawym klawiszem w ten plik i otwórz go z prawami roota. Znajdź sekcję SECTION DEVICE i zmień sterownik na nv LUB atiOR ati OR intel.

Editing xorg.conf is entirely at your own risk!!

Firmware drivers

The required firmware can be determined with the following device/ firmware enumeration or by using the newly developed fw-detect script (packaged in sidux-scripts).

$ fw-detect

It is good practice to do a modprobe -r <modulename> before modprobe <modulename>, especially for intel.

Zapisywanie na partycjach NTFS przy pomocy ntfs-3g

Uwaga: Chociaż ntfs-3g jest uznawany za 'stabilny', nigdy nie używaj go bez zewnętrznej kopii zapasowej i oczywiście nie na systemach produkcyjnych. Jeśli to zrobisz, nie bierzemy odpowiedzialności za utratę danych, więc używaj go na własne ryzyko!

Otwórz konsolę i wpisz następujące polecenie: Patrz Partycjonowanie dysku twardego - Nazewnictwo

umount /dev/XdXX
mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX
exit
exit

Teraz twoja partycja NTFS jest zamontowana w trybie rw i będziesz mógł zapisywać na niej dane! Ale pamiętaj! Używaj go tylko w sytuacjach nagłego wypadku, nie zalecamy codziennego użycia tego sterownika.
