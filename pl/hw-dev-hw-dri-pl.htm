Binarne, niewolne sterowniki dla Nvidii / ATI

Istnieją trzy metody instalacji sterowników 3d nVidii

  1. sgfxi / smxi

    Use the sidux community contributed multi-function administration tool, smxi/sgfxi, which is short for sidux-maintenance. To read more refer to smxi and sgfxi Syntax Command Sets.

  2. Użycie binarnego instalatora Nvidia/ATI

    Pobierz instalator binarnych sterowników prosto ze strony Nvidii
Pobierz instalator binarnych sterowników prosto ze strony ATi

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel are pre-installed with sidux. If you have overwitten them with closed source drivers you can revert or roll back to them with sgfxi

Sterowniki dla X.Org X serwer (patrz: xserver-xorg dla dalszego opisu) dostarcza wsparcie dla 2d kart NVIDIA Riva, TNT, GeForce i Quadro oraz kart ATI Mach, Rage, Radeon i FireGL wraz z substerownikami atimisc', 'r128' i 'radeon'. Również udostępnia 3d dla kart Intela.

Aby go zainstalować:

Przełącz się z KDE do trybu tekstowego poprzez Ctrl+Alt+F1
zaloguj się jako root
i wpisz: init 3
(Powinno to spowodować, że X przestanie działać)
apt-get update
apt-get install xserver-xorg-video-nv
LUB
apt-get install xserver-xorg-video-ati
LUB
xserver-xorg-video-intel
init 5 && exit

For some cards it may add 3D support, however it is usually 2D.

Następnie musisz wyedytować /etc/X11/xorg.conf, JEDNAKŻE MUSISZ WCZEŚNIEJ ZROBIĆ KOPIĘ BEZPIECZEŃSTWA, ponieważ niepoprawna edycja /etc/X11/xorg.conf spowoduje, że serwer X już się nie uruchomi.

Aby edytować /etc/X11/xorg.conf, kliknij prawym klawiszem w ten plik i otwórz go z prawami roota. Znajdź sekcję SECTION DEVICE i zmień sterownik na nv LUB ati

UWAGA!! EDYCJĘ XORG.CONF WYKONUJESZ NA WŁASNE RYZYKO!!

Wskazówki dla sprzętu potrzebującego niewolnego oprogramowania

Ze względu na złożoną sytuację prawną, sidux live-cd będzie zawierał tylko oprogramowanie wolne w sensie dfsg.

Sources List

Ponieważ sidux zawiera tylko oprogramowanie wolne w sensie dfsg, prawdopodobnie będziesz chciał dodać contrib/ non-free do /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Video Cards
  • grafika ATi Radeon: akceleracja 3d dla starszych kart do r35x powinna działać, nowsze karty Radeon X1xxx potrzebują niewolnych sterowników dla akceleracji 3D, które nie są jeszcze kompatybilne z X.org 7.2. W zależności od chipsetu konieczne może być uruchomienie systemu z opcją startową "xmodule=vesa" lub "xmodule=fbdev" dla późniejszych modeli Radeon X1xxx.
Firmware
  • To help detect what firmware you need to install, run, as user:
$ fw-detect

This will give you a list of recommendations and what commands to run. It will return 'null' if you do not need anything, or firmware is already installed.

    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan:

  • IVTV firmware:fetch the firmware from http://dl.ivtvdriver.org/ivtv/firmware/ and place it under/lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan Usuń iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 i firmware-iwl4965, jeśli są zainstalowane:.: 
    apt-get install firmware-iwlwifi
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54 MBit/s wlan: pobierz firmware z http://prism54.org/firmware/ i umieść je w katalogu /lib/firmware/.
  • grafika nVidia: akceleracja 3d nie jest jeszcze możliwa ze sterownikami open source, użyj get-sidux-binary-gfx or sgfxi,
  • RaLink rt61 and rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    Please remove rt61-fwcutter, rt73-fwcutter, firmware-rt61 and firmware-rt73, if already installed

  • Ricoh R5U870 OEM webcams:fetch the firmware from http://mediati.org/r5u870/r5u870-0.11.0.tar.gz and copy the r5u870_*.fw files to /lib/firmware/.
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan:

  • ZyDAS zd1201 11 MBit/s wlan: 
    apt-get install zd1201-firmware

Zapisywanie na partycjach NTFS przy pomocy ntfs-3g

Uwaga: Chociaż ntfs-3g jest uznawany za 'stabilny', nigdy nie używaj go bez zewnętrznej kopii zapasowej i oczywiście nie na systemach produkcyjnych. Jeśli to zrobisz, nie bierzemy odpowiedzialności za utratę danych, więc używaj go na własne ryzyko!

Otwórz konsolę i wpisz następujące polecenie: Patrz Partycjonowanie dysku twardego - Nazewnictwo

umount /dev/XdXX
mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX
exit
exit

Teraz twoja partycja NTFS jest zamontowana w trybie rw i będziesz mógł zapisywać na niej dane! Ale pamiętaj! Używaj go tylko w sytuacjach nagłego wypadku, nie zalecamy codziennego użycia tego sterownika.
