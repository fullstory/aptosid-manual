Binarne, niewolne sterowniki dla Nvidii / ATI

Istnieją trzy metody instalacji sterowników 3d nVidii

  1. sgfxi / !h2 smxi

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxi obecnie wspiera sterowniki ATI fglrx i Nvidia. Umożliwia także konwersję z lub do wolnych sterowników xorg takich jak ati, intel, lub nv. !h2 smxi używa sgfxi jako silnika instalatora sterowników graficznych, ale dodaje wsparcie dla niektórych opcji dla użytkowników.

    Domyślny sterownik to ten, który zostanie zainstalowany, jeśli nie zostaną podane żadne argumenty. Jeśli twoja karta jest zbyt stara i wymaga sterowników dla starszych urządzeń, skrypt to wykryje.

    Kiedy uruchomisz skrypt, zatrzyma się, aby powiadomić, który sterownik zamierza zainstalować. Możesz wtedy zaakceptować ten wybór, lub zakończyć działanie skryptu i uruchomić go z opcjami, które umożliwią inną konfigurację.

    Możesz także uruchomić instalator sgfxi przy pomocy !h2 smxi, który używa sgfxi do instalowania sterowników graficznych. Najpierw przeczytaj stronę domową !h2 smxi , aby zdobyć więcej informacji, następnie wykonaj polecenie:

    smxi -kiwdt

    sgfxi-h: Kiedy użyjesz argumentu -h, otrzymasz listę wszystkich aktualnie wspieranych sterowników, jak również krótki wydruk specyfikacji twojego systemu, kernela, wersji xorg. Oto kilka użytecznych argumentów sgfxi:

    -c Używa trybu composite (tylko karty nVidia).
-d Usuwa IgnoreDisplayDevices, pozwala na TV  (tylko karty nVidia).
-h Wyświetla dostępne argumenty.
-n Automatycznie instaluje właściwe, natywne sterowniki xorg: nv, intel, lub ati.
   Usuwa stare sterowniki binarne, uaktualnia xorg.conf o nowe informacje o sterownikach,
   i czyści opcje nvidii i fgrlx z xorg.conf
-N Wymusza instalację określonego sterownika xorg. 
Składnia: sgfxi -N <jeden ze sterowników z listy, jak -N radeon>
   Wspierane sterowniki Xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Również konfiguruje xorg.conf. Automatyczna konfiguracja 3d dla sterownika radeon.
-o Zmienia domyślne działanie: instaluje określony sterownik, jest to absolutna zmiana - składnia:
   -o 9746 (dla nVidia, nie używaj części nazwy 1.0-) lub: -o 8.32.5 (dla fglrx)
-r Usuwa ustawienia mode lines z xorg.conf
-t Uaktywnia dynamiczne twinview (tylko nVidia)

  2. install-binary-gfx

    Nowy skrypt do instalacji sterowników graficznych nazywa się: install-binary-gfx. Aby go pobrać, uruchom w konsoli jako root get-sidux-binary-gfx. 

    get-sidux-binary-gfx
install-binary-gfx -a (użycie kodu źródłowego z repozytorium sida)

    Skrypt wybierze odpowiedni sterownik dla twojej karty, zbuduje deb, zainstaluje go i automatycznie zmodyfikuje xorg.conf stosownie do twojej konfiguracji. Aby zmiany przyniosły efekt musisz zrestartować X (Ctrl+Alt+Backspace).

    Opcje

    Możesz uaktywnić następujące opcje:

     Usage: ./install-binary-gfx [options]
-a | --auto            Tryb automatyczny
-b | --build           Tylko buduj moduły (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          Tylko wykryj kartę
-u | --update          Zaktualizuj skrypt
-v | --verbose         Pokaż komunikaty

    Przykłady

    install-binary-gfx -a (Wykryje twoją kartę i automatycznie zainstaluje sterowniki z sida.)
install-binary-gfx -b nvidia (Tylko zbuduje moduł nvidia - deb in /usr/src. Działa też z fglrx i nvidia-legacy.)
install-binary-gfx -b ati (Zbuduje moduł ati....deb w /usr/src. To samo działa z fglrx i ati-legacy.)
install-binary-gfx -d (Wykryje twoją kartę i wyświetli informacje.)
install-binary-gfx -u (Zaktualizuje skrypt.)

    Rozwiązywanie problemów

    Jeśli skrypt zawiedzie w przypadku twojej karty, uruchom go w trybie wykrywania błędów (debug mode) i zamieść wynik wraz z plikiem /var/log/sidux-CARD-installer.log na forum sidux (TM) lub użyj http://paste.debian.net i wejdź na #sidux podając link, aby otrzymać pomoc na IRC.

  3. Instalacja niewolnych sterowników dla live-cd:

    sudo sgfxi

    To pobierze najnowszy skrypt, następnie:

    ctrl+alt+f1

    i uruchom

    sgfxi

    sgfxi -h,
aby zobaczyć dostępne opcje

    X zostanie uruchomiony ponownie, gdy sterowniki zostaną zainstalowane

Wolne sterowniki xorg dla nVidia, ATI, Radeon, Intel

  1. sgfxi

    Możesz również użyć sgfxi, aby przejść między wolnymi i binarnymi sterownikami: Główne polecenia:

    -n Automatycznie instaluj właściwe, natywne sterowniki xorg: nv, intel, lub ati.
   Usuwa stare sterowniki binarne, uaktualnia xorg.conf o nowe informacje o sterownikach,
   i czyści opcje nvidii i fgrlx z xorg.conf
-N Wymusza instalację określonego sterownika xorg. 
Składnia: sgfxi -N <jeden ze sterowników z listy, jak -N radeon>
   Wspierane sterowniki Xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Również konfiguruje xorg.conf. Automatyczna konfiguracja 3d dla sterownika radeon.

    Najpierw

    apt-get update
apt-get install sidux-scripts
sgfxi

    Aby zainstalować lub przywrócić natywny sterownik

    sgfxi -n
LUB
sgfxi -N <sterownik xorg>

  2. sgfxi / !h2 smxi

    Używaj sgfxi via !h2 smxi. Po prostu uruchom skrypt używając następujących opcji, a wtedy zaprowadzi cię wprost do sekcji instalacji sterowników graficznych. Jeśli używasz smxi, nie potrzebujesz instalować sgfxi, ponieważ smxi instaluje go automatycznie.

    strona domowa !h2 smxi, aby otrzymać więcej informacji, wykonaj te polecenie:

    smxi -kiwdt

  3. Trudniejszy sposób

    Sterowniki dla X.Org X serwer (patrz: xserver-xorg dla dalszego opisu) dostarcza wsparcie dla 2d kart NVIDIA Riva, TNT, GeForce i Quadro oraz kart ATI Mach, Rage, Radeon i FireGL wraz z substerownikami atimisc', 'r128' i 'radeon'. Również udostępnia 3d dla kart Intela.

    Aby go zainstalować:

    Przełącz się z KDE do trybu tekstowego poprzez Ctrl+Alt+F1
zaloguj się jako root
i wpisz: init 3
(Powinno to spowodować, że X przestanie działać)
apt-get update
apt-get install xserver-xorg-video-nv
LUB
apt-get install xserver-xorg-video-ati
init 5 && exit

    Zwróć uwagę, że sterownik ten nie jest taki sam, jak binarny 'nvidia', który umożliwia obsługę 3D, ale jest niewspierany ze względu na brak kodu źródłowego.

    Następnie musisz wyedytować /etc/X11/xorg.conf, JEDNAKŻE MUSISZ WCZEŚNIEJ ZROBIĆ KOPIĘ BEZPIECZEŃSTWA, ponieważ niepoprawna edycja /etc/X11/xorg.conf spowoduje, że serwer X już się nie uruchomi.

    Aby edytować /etc/X11/xorg.conf, kliknij prawym klawiszem w ten plik i otwórz go z prawami roota. Znajdź sekcję SECTION DEVICE i zmień sterownik na nv LUB ati

    UWAGA!! EDYCJĘ XORG.CONF WYKONUJESZ NA WŁASNE RYZYKO!!

    Więcej informacji o X.Org

Dwa monitory

Masz zamiar zanurzyć się w plikach konfiguracyjnych, więc zrób kopię bezpieczeństwa xorg.conf, zanim przejdziesz dalej!

Poniższa treść może ulec zmianie, gdy pojawi się xorg 7.3 pod koniec 2007 r.

Natywny ATI - Metoda 1

Na forum znajdziesz kilka działających xorg.conf z wolnym sterownikiem radeon.

UWAGA: Musisz zdobyć informacje o konfiguracji drugiego monitora. Aby to zrobić, musisz odłączyć jeden monitor i uruchomić komputer z liveCD, aby wygenerować xorg.conf, skopiować go, następnie zrobić to samo z drugim monitorem.

Natywny ATI - Metoda 2

Przy pomocy tej metody, musisz najpierw ustawić rozdzielczość pierwszego monitora, a następnie drugi będzie korzystał z takiej samej konfiguracji, więc musisz ustawić ten pierwszy w rozdzielczość, którą drugi będzie w stanie obsłużyć.

W sekcji:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

dodaj

Option "MergedFB"
Option "CRT2Position" "RightOf"

Więc będzie się kończyć w ten sposób:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Pełna informacja o konfiguracji na stronie http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html

UWAGA: W powyższym linku jest wykorzystany sterownik radeon, ale może działać również ze sterownikami ati.

Instrukcja instalacji sterowników 3D

Następnie: musisz wyjść z sesji X

ctrl+alt+F1
wpisz: root
wpisz swoje hasło:
init 3

Dalej:

aticonfig --dtop=horizontal
init 5 && exit

Jeśli twoje monitory nie zostaną poprawnie skonfigurowane, możesz wpisać

aticonfig --dtop=horizontal,reverse

Przeczytaj pomoc, która bardzo szczegółowo opisuje opcje i podaje przykłady:

aticonfig --help

Uwaga: Oba monitory muszą mieć tę samą rozdzielczość. Sprawdź więc instrukcję obu monitorów i DOPASUJ najwyższą rozdzielczość przez nie obsługiwaną.

Forum może dostarczyć dodatkowych informacji.

aticonfig --initial --input=/etc/X11/xorg.conf

Wskazówki dla sprzętu potrzebującego niewolnego oprogramowania

Ze względu na złożoną sytuację prawną, sidux live-cd będzie zawierał tylko oprogramowanie wolne w sensie dfsg.

Ponieważ sidux zawiera tylko oprogramowanie wolne w sensie dfsg, prawdopodobnie będziesz chciał dodać contrib/ non-free do /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in seperate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Zapisywanie na partycjach NTFS przy pomocy ntfs-3g

Uwaga: Chociaż ntfs-3g jest uznawany za 'stabilny', nigdy nie używaj go bez zewnętrznej kopii zapasowej i oczywiście nie na systemach produkcyjnych. Jeśli to zrobisz, nie bierzemy odpowiedzialności za utratę danych, więc używaj go na własne ryzyko!

Otwórz konsolę i wpisz następujące polecenie: Patrz Partycjonowanie dysku twardego - Nazewnictwo

umount /dev/XdXX
mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX
exit
exit

Teraz twoja partycja NTFS jest zamontowana w trybie rw i będziesz mógł zapisywać na niej dane! Ale pamiętaj! Używaj go tylko w sytuacjach nagłego wypadku, nie zalecamy codziennego użycia tego sterownika.
