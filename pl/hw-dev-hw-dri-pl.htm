Wskazówki dla sprzętu wymagającego niewolnego oprogramowania

Ze względu na złożoną sytuację prawną, sidux live-cd będzie zawierał tylko oprogramowanie wolne w sensie dfsg.

Sterowniki Open Source Xorg do nVidia, ATI, Radeon, Intel

Sterowniki Open Source Xorg dla kart nVidia, ATI, Radeon, Intel są pre-instalowane przy instalacji sidux.

Sterowniki Intela powinny perfekcyjnie pasować do akceleratorów grafiki 2D i 3D. Dołączone sterowniki należą do serii wolnych sterowników Intela.

Jeżeli uruchomiłeś odpowiedni sterownik a chciałbyś przywrócić sterownik opensource, powinieneś zrobić to edytując /etc/X11/xorg.conf, WCZEŚNIEJ MUSISZ ZROBIĆ KOPIĘ BEZPIECZEŃSTWA ponieważ niewłaściwa edycja /etc/X11/xorg.conf może przyspożyć ci sporych kłopotów.

Aby edytować /etc/X11/xorg.conf, kliknij prawym klawiszem w ten plik i otwórz go z prawami roota. Znajdź sekcję SECTION DEVICE i zmień sterownik na nv LUB ati LUB intel (wymieniając tylko niektore).

Edytując xorg.conf robisz to na własne ryzyko!!

Sources List

Ponieważ sidux zawiera tylko oprogramowanie wolne w sensie dfsg, prawdopodobnie będziesz chciał dodać contrib/ non-free do /etc/apt/sources.list.d/debian.list i /etc/apt/sources.list.d/sidux.list lub /etc/apt/sources.list:

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Sterowniki binarne, z kodem zamkniętym do nVidia z dmakms

NOTE: Use apt-cache search nvidia and apt-cache show <package> to ascertain the correct driver for you. There are basically 2 choices, current Debian Sid 3D drivers and legacy Debian Sid 3D drivers.

For current 3d nvidia drivers ≥ GeForce 6xxx :

Przygotowanie modułu:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Następnie należy uaktywnić Dynamic Module-Assistant Kernel Module Support (dmakms) dla nvidia tak, aby następnym razem przy aktualizacji jądra linuksa moduł nvidii był także aktualizowany bez koniecznosci ręcznej ingerencji. Aby tego dokonać, dodaj nvidia-kernel-source do pliku konfiguracyjnego /etc/default/dmakms :

echo nvidia-kernel-source >> /etc/default/dmakms

Następnie:

m-a a-i nvidia-kernel-source

W kolejności:

apt-get install nvidia-glx

Następne zmiany xorg.conf:

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Żeby zobaczyć efekt instalacji modulu należy zrestartować PC.

Po aktualizacji xorg należy tylko przeinstalować nvidia-glx:

apt-get install --reinstall nvidia-glx

Nowy sterownik nvidia do Debian sid staje się dostępny przez:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Zrestartuj PC aby instalacja modułów odniosła skutek.

For legacy 3d nvidia drivers ≥ GeForce 5xxx :
m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Alter dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Błąd załadowania Modułu

Jeżeli z jakiejkolwiek przyczyny nie uda się załadować modułu nvidii:

modprobe nvidia

Potem zrestartuj komputer.

Gdyby moduł się nadal nie ładował:

 m-a a-i -f nvidia-kernel-source

To spowoduje odbudowanie modułu. Zrestartuj komputer.

Czytaj:

$ /usr/share/doc/dmakms

You can also read nvnews for other options.

ATI, Radeon i inne sterowniki video

Sterowniki 2D

Sterowniki dla X.Org X server (dalszy opis patrz xserver-xorg) zapewniają wsparcie dla kart 2D z NVIDIA Riva, TNT, GeForce, oraz Quadro a także ATI Mach, Rage, Radeon i FireGL wraz z atimisc, r128, r6xx/r7xx podwersjami radeon. Radeon jak i Intel suportują akcelerację 2d (textured xv) do odtwarzania wideo.

Sterowniki 3D

Niektóre karty ATI także obsługują 3D, do tej pory do chipsetu r5xx.

Do automatycznego instalowania nowych niewolnych sterowników, kiedy aktualizujemy pakiety sterowników 2D i 3D:

apt-get install firmware-linux

Potem zrestartuj komputer.

$ fw-detect

Zapisywanie na partycjach NTFS przy pomocy ntfs-3g

Uwaga: Chociaż ntfs-3g jest uznawany za 'stabilny', nigdy nie używaj go bez zewnętrznej kopii zapasowej i oczywiście nie na systemach produkcyjnych. Jeśli to zrobisz, nie bierzemy odpowiedzialności za utratę danych, więc używaj go na własne ryzyko!

Otwórz konsolę i wpisz następujące polecenie: Patrz Partycjonowanie dysku twardego - Nazewnictwo

umount /dev/XdXX
mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX
exit
exit

Teraz twoja partycja NTFS jest zamontowana w trybie rw i będziesz mógł zapisywać na niej dane! Ale pamiętaj! Używaj go tylko w sytuacjach nagłego wypadku, nie zalecamy codziennego użycia tego sterownika.
