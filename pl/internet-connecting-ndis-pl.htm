Aktywacja Ndiswrapper

sidux ma już wszystkie potrzebne pakiety zainstalowane: moduł kernela i ndiswrapper-utils.

Potrzebujesz windowsowy plik inf dla sterownika, znajdź go i załaduj:

ndiswrapper -i driver.inf

Aby sprawdzić, czy instalacja poszła dobrze, użyj polecenia:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Ważne, abyś zobaczył komunikaty "driver installed" (sterownik zainstalowany) i "device present" (urządzenie obecne) w wynikach polecenia. Jeśli ich obu nie zobaczysz, oznacza to, że coś poszło źle (sprawdź informacje o błędach z polecenia i/lub z dmesg), ponieważ, jeśli pojawiają się komunikaty o błędach, nie uzyskasz połączenia wlan.

załaduj moduł ndiswrappera:

modprobe ndiswrapper

i sprawdź, czy jest zainstalowany poprawnie:

iwconfig

sprawdź, czy access-point został odnaleziony:

iwlist wlan0 scan

następnie możesz skonfigurować kartę przy pomocy polecenia:

netcardconfig

Jeśli zaktualizujesz kernel, będziesz musiał ponownie zainstalować sterownik. Zmień numer kernela, aby pasował do twojego, obecnie zainstalowanego:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Następnie załaduj moduł przy pomocy modprobe:

modprobe ndiswrapper

Nie musisz instalować ponownie sterowników windows, ponieważ są one niezależne od modułu kernela.

Możesz również wpisać ndiswrapper do /etc/modules, aby uruchamiać go przy starcie systemu

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

Konfigurację możesz wykonać także przy pomocy centrum sterowania siduxcc umieszczonym w menu KDE.

Ndiswrapper - konfiguracja zaawansowana

Ndiswrapper jest modułem, który udostępnia linuksowi sterowniki WLAN systemu Windows. Jest to potrzebne, ponieważ niektórzy producenci kart wlan nie dostarczają sterowników open source, więc implementacja tych kart w linuksie jest trudna. Ndiswrapper omija ten problem poprzez pośrednictwo w komunikacji sterownika windows i systemu linux.

Identyfikacja wymaganego sterownika

Najpierw potrzebujesz odpowiedni sterownik windowsa do twojej karty WLAN. Karty WLAN posiadają wiele różnych chipów dla jednego modelu i jedynym sposobem identyfikacji karty jest sprawdzenie id urządzenia (USBID/PCIID).

Aby znaleźć to 'id', otwórz terminal:

Dla urządzenia USB WLAN
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

Jak widać powyżej, USBID dla D-Link Corp. [hex] DWL-122 adaptor to 2001:3700.

Dla urządzenia (mini) PCI / PCMCIA WLAN
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

To 'id' ma również schemat producent:urządzenie, tutaj jest to 168c:0024 dla karty Atheros.

Teraz szukaj: ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list dla wpisu z tym 'id', jeśli masz szczęście, znajdziesz go i rekomendację dla sterownika.

Potrzebujesz pliki INF i SYS z tego sterownika. Najlepiej, jeśli masz plik ZIP, który możesz rozpakować przy pomocy polecenia unzip. Jeśli masz plik EXE, jest to samorozpakowujące się archiwum (użyj wine, aby rozpakować plik) lub CAB (wypakuj przy pomocy cabextract). Najłatwiej zachować wszystkie pliki w nowym katalogu w home.

Aktywacja Ndiswrapper i instalacja sterownika

Teraz zmień katalog na ten z plikami sterownikami windowsa:

bla@blubb:~$cd /katalog/ze/sterownikiem
bla@blubb:~$su
root@blubb:/katalog/ze/sterownikiem# ndiswrapper -i <driver.inf>
installing net5416 ...
root@blubb:/katalog/ze/sterownikiem# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Ważne, abyś zobaczył komunikaty" driver installed" i "device present". Jeśli nie widziałeś ich obu, oznacza to, że coś poszło źle (sprawdź komunikaty o błędach ndiswrappera i/lub dmesg), ponieważ, jeśli teraz pojawiają się błędy, dalsze kroki nie dadzą spodziewanego rezultatu.

Teraz załaduj moduł kernela: 

root@blubb:/some/folder# modprobe ndiswrapper

Jeśli otrzymasz komunikat o błędach, sprawdź dmesg dla dalszych informacji.

"iwconfig" powinien wykryć urządzenie bezprzewodowe:

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

Jeśli chcesz, aby ndiswrappper ładował się przy starcie, wpisz go do /etc/modules, albo ręcznie, albo użyj polecenia: 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
Ponowna instalacja Ndiswrappera po aktualizacji kernela

Po aktualizacji kernela musisz ponownie zainstalować moduł kernela ndiswrapper. Plik deb jest umieszczony w /usr/src/kernel-downloads/ <wersja-kernela >/ndiswrapper-modules- <wersja >.deb,. Na przykład dla kernela 2.6.21.5-slh-smp-2, musisz wykonać polecenie:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Po instalacji, załaduj go przy pomocy:

root@blubb:/some/folder# modprobe ndiswrapper

Nie musisz ponownie instalować sterowników windowsa, ponieważ sterownik windows jest niezależny od modułu kernela.

Ogólne informacje

Aby skonfigurować nowe urządzenie użyj netcardconfig lub sidux-networkconfig. Szczegółowe how-to możesz znaleźć tu.

Jeśli znajdziesz inny/nowszy sterownik, również go przetestuj. Może to powodować problemy, kernel panic w najgorszym przypadku, ale może to pomóc projektowi ndiswrappera i innym użytkownikom skorzystać z najlepszych sterowników dla różnych kart. Nie zapomnij zaktualizować informacji na liście urządzeń kompatybilnych ndiswrappera po udanym teście.
