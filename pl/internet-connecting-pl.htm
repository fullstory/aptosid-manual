Połączenie z internetem / Konfiguracja sieci

Jeśli masz serwer DHCP w swojej sieci i twój komputer jest do niej podłączony podczas startu Live-CD, to ustawienia sieci powinny być skonfigurowane automatycznie - w przeciwnym razie musisz uruchomić sidux network configuration dialog: Kliknij Kmenu>sidux>Network/Internet>Networkcard configuration.

Lub - Jeśli wolisz, możesz otworzyć terminal i wpisać 

netcardconfig

i postępować według wskazówek.

W systemie zainstalowanym na twardym dysku możesz użyć 'Kmenu>sidux>Network/Internet>Networkcard configuration', a następnie postępować według wskazówek, lub wykorzystać konsolę i (jako użytkownik) wywołać polecenie:

/usr/sbin/netcardconfig

a następnie postępować według wskazówek.

Komputery z dwiema kartami sieciowymi (współdzielenie internetu)

Aby udostępnić internet innemu komputerowi należy:

Otworzyć terminal i zmienić użytkownika na roota przy pomocy "sux", aby móc wykonać następujące kroki.

Konfiguruj pierwszą kartę sieciową, aby połączyć się z internetem.

netcardconfig

Dla drugiej karty sieciowej wybieramy stały adres IP w domenie prywatnej, np.

192.168.1.1

Następnie również skonfiguruj ją przy pomocy:

netcardconfig

Pamiętaj: Twoje IP musi być w tej samej podsieci jak IP, które zostało pobrane z DHCP.

Jeśli przez pomyłkę zdefiniowałeś bramę podczas konfiguracji interfejsów sieciowych, to musisz uruchomić skrypt

remove-gateway

Konfigurowanie DHCP/DNS:

apt-get install dnsmasq

Dodaj linię w pliku /etc/dnsmasq.conf

dhcp-range=192.168.1.50,192.168.1.150

Pozostaw resztę pliku bez zmian.

Serwer DHCP na komputerze z dostępem do internetu może być uruchomiony przy pomocy 

/etc/init.d/dnsmasq start
.

Jeśli klienci skonfigurowani są, aby korzystać z DHCP, adres IP zostanie przydzielony automatycznie.

Aby zapewnić routing zapytań klientów do internetu, uruchom skrypt

/usr/sbin/masquerade

Po tym każdy klient powinien mieć dostęp do internetu.

Aby ustawienia pozostały po ponownym uruchomieniu komputera: Aktywuj dnsmasq w procesie startu przy pomocy

update-rc.d -f dnsmasq defaults

Uruchom skrypt maskarady:

echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Połączenie przy pomocy 56k Dial-up Modem

KDE posiada interfejs do modemów dial up KPPP Internet Dial-up Tool, mieszczący się w głównym menu w dziale Internet.

Aplikacja posiada system pomocy i udostępnia instrukcję konfiguracji modemu.

Firewalle

Firewalle są zwykle niepotrzebne, jeśli jesteś za routerem properly configured, jednakże pełnią bardzo ważną rolę w bezpieczeństwie twojego systemu, jeśli łączysz się z internetem przy pomocy modemu usb adsl lub via dial-up modem:

apt-get install guarddog
LUB
apt-get install firestarter

Guarddog jest bardzo przyjaznym narzędziem konfigurującym firewalla dla KDE. Najlepszym sposobem rozpoczęcia pracy z programem jest przeczytanie krótkich tutoriali, które tu się znajdują.

Firestarter - firewall z graficznym interfejsem dla linuksa.[Kiedy opuścisz interfejs, firewall pozostanie aktywny w tle.]
