Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

Definições dos arquivos em cada espelho

Cada espelho do sidux apresenta os seguintes arquivos:

sidux-2009-01-ouranos-kde-lite-amd64-200902151304.amd64.manifest
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.md5
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.md5.asc
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.sha256
sidux-2009-01-ouranos-kde-lite-amd64-200902151304.iso.sha256.asc

O arquivo .manifest lista todos os pacotes contidos na iso. O arquivo com extensão .iso é a imagem a ser baixada. Os arquivos .md5 e .sha256 são usados na verificação da integridade da ISO.

Os arquivos .asc são arquivos de assinatura, de forma que você possa verificar se a cadeia de caracteres ('checksums') (.md5 .sha256) não foi alterada e pode ser usada para confirmar a integridade da ISO.

Se você quiser verificar a integridade dos arquivos .asc, faça o seguinte como $USUÁRIO:

1) Importe a chave pública de "Stefan Lippers-Hollmann's" (slh):

$ gpg --keyserver wwwkeys.eu.pgp.net --recv-keys F80994F6
gpg --edit-key F80994F6
Comando> trust
Comando> (escolha uma opção, p. ex., a 5 - I trust ultimately [Confio extremamente] ou m [para voltar ao menu principal])
Comando> quit

2) Agora você pode confirmar a integridade do 'checksum' com:

$ gpg --verify sidux-2009-01-ouranos-kde-full-i386-amd64-200902151348.iso.md5.asc sidux-2009-01-ouranos-kde-full-i386-amd64-200902151348.iso.md5

Se você classificou a chave como totalmente confiável, usando os comandos correspondentes do gpg, então esta é a saída que você verá (se tiver escolhido a opção 5 - I trust ultimately, como sugerimos):

$ gpg: Signature made Sun 15 Feb 2009 04:55:55 PM CET using DSA key ID F80994F6
$ gpg: a verificar a base de dados de confiança
$ gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
$ gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

Você vai ver um 'Good signature' (Assinatura boa) acompanhado de um AVISO, se você não tiver classificado a chave:

$ gpg: Signature made Sun 15 Feb 2009 04:55:55 PM CET using DSA key ID F80994F6
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"
$ gpg: WARNING: This key is not certified with a trusted signature! (AVISO:  a confiabilidade da assinatura desta chave não foi verificada)
$ gpg:          There is no indication that the signature belongs to the owner (não há nada que indique que a assinatura pertença ao autor.
$ Primary key fingerprint: 64C3 6120 DA8D 91E7 378B  E79F 3916 C431 F809 94F6

Como baixar com o Bittorrent

Bittorrent é um protocolo para compartilhamento de arquivos que pode ser encontrado no CD do sidux. Ele permite que arquivos sejam baixados (download) e transmitidos (upload) pela internet, ao mesmo tempo. Favor visitar sidux.com para obter os atuais endereços dos torrents. Por exemplo, se você digitar o comando abaixo em um terminal, seu arquivo deve começar a ser baixado logo em seguida:

# btdownloadcurses --max_upload_rate 12 ftp://dummy/sidux/sidux-200xxxxx/sidux-200xxxxx.iso.torrent

Certifique-se de que haja espaço livre suficiente no diretório onde você executou o comando, para acomodar o arquivo ISO (aproximadamente 700 MB).

Quando terminar de baixar, deixe o programa rodando para que o arquivo seja compartilhado com outros usuários. Isto reduzirá a carga nos servidores e os arquivos serão semeados mais rapidamente. Um bom tutorial pode ser encontrado em Brians BitTorrent FAQ and Guide

Não se esqueça de conferir o md5sum para validar o que foi baixado.

Se você ou alguém que você conheça possuir um servidor de FTP rápido, com suficiente tráfico à  disposição, favor informar a equipe do sidux. Alternativamente, podem ser contatados os Fórums do sidux ou o Canal de IRC em oftc.net

Conferência do md5sum e validação

OBSERVAÇÃO MUITO IMPORTANTE: o sidux, como LiveCD do Linux, é baseado em tecnologia de alta compressão (desenvolvida pioneiramente pelo Knoppix, mas não igual à do Knoppix). Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

SEMPRE FAÇA BACKUPS (CÓPIAS) DE SEUS DADOS!

Esta conferência é usada para checar a integridade de arquivos. A soma do md5sum encontrada é comparada com uma soma já conhecida. Assim, você pode verificar se um arquivo foi alterado ou sofreu danos; essa checagem é altamente recomendada para arquivos baixados pela internet.

Se o valor do md5 do arquivo baixado corresponder ao valor do md5 do arquivo original, você pode estar seguro de que o arquivo foi baixado corretamente. No Linux, a verificação é feita com este comando (o cálculo demora um pouquinho):

$ md5sum arquivo_a_verificar

A soma será escrita no próprio terminal.
Esta validação também pode ser feita no Windows. Basta instalar o programa "md5sum" (486 kb).

As imagens ISO do sidux são sempre acompanhadas pelo arquivo de checagem do md5 (ele possui a extensão .md5) e você deve sempre validá-las antes de queimá-las. Isto garante que, se ocorrerem problemas, eles não são causados pelo arquivo que você baixou e ainda mantém o fórum do sidux livre de questões não solucionadas só porque a ISO estava danificada. Isto é feito no terminal. Mude para o diretório onde estão os arquivos com a imagem ISO e com o arquivo .md5 e digite o seguinte comando para validar a soma md5sum:

$ md5sum -c nome_da_iso_do_sidux.iso

Se os valores encontrados não forem os mesmos, você verá uma mensagem de erro:

"nome_da_iso_do_sidux.iso: Error
md5sum: Warning: calculated checksum does not match!" (md5sum: Aviso: soma calculada não confere!)

Se o arquivo tiver sido baixado corretamente, o programa finaliza sem nenhuma mensagem.

Como eu queimo o CD no Windows?

OBSERVAÇÃO MUITO IMPORTANTE: o sidux, como LiveCD do Linux, é baseado em tecnologia de alta compressão (desenvolvida pioneiramente pelo Knoppix, mas não igual à do Knoppix). Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

SEMPRE FAÇA BACKUPS (CÓPIAS) DE SEUS DADOS!

Naturalmente, é possível queimar o CD no Windows, também. O arquivo baixado deve ser queimado como um arquivo ISO. Se o Winrar (ou qualquer outro programa de compressão) estiver associado a arquivos .ISO, podem ocorrer problemas, porque ele será visto como um simples arquivo. Portanto, de posse da imagem ISO, queime logo um CD. Não queime o arquivo ISO como disco de dados!! É necessário queimá-lo como Disk-At-Once

Como queimar o CD no Linux

OBSERVAÇÃO MUITO IMPORTANTE: o sidux, como LiveCD do Linux, é baseado em tecnologia de alta compressão (desenvolvida pioneiramente pelo Knoppix, mas não igual à do Knoppix). Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

SEMPRE FAÇA BACKUPS (CÓPIAS) DE SEUS DADOS!

Se você já roda Linux em sua máquina, você pode queimar o CD tanto pela linha de comando quanto usando algum programa já existente no sistema operacional.

Para tarefas desse tipo, o sidux e outras distribuições instalam o K3b por padrão. Basta abri-lo, clicar em Ferramentas -> Queimar imagem de CD e selecionar o arquivo ISO a ser queimado (p. ex. sidux.iso).

O K3B inicialmente calcula a soma do MD5 para a imagem ISO (o que demora um momento). Se o resultado for igual ao apresentado pelo arquivo MD5 (isto é, o sidux.iso.md5), você pode queimar a imagem, bastando para tal clicar em "Queimar".

Como criar um CD do sidux usando XDELTA-Patch

OBSERVAÇÃO MUITO IMPORTANTE: o sidux, como LiveCD do Linux, é baseado em tecnologia de alta compressão (desenvolvida pioneiramente pelo Knoppix, mas não igual à do Knoppix). Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

SEMPRE FAÇA BACKUPS (CÓPIAS) DE SEUS DADOS!

O xdelta é um programa que monta uma nova imagem ISO a partir de uma previamente existente e acrescenta alguns arquivos modificados, entre outras coisas. Seu objetivo é também incorporar pequenas correções de bugs sem que seja necessário baixar novamente toda a imagem.

Digite no terminal, como root:

apt-get update
apt-get install xdelta

Os arquivos modificados de patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta, por exemplo, têm o tamanho de apenas 5.4 MB, enquanto a imagem ISO soma 553 MB.

Digamos que você tenha:

  • Uma imagem ISO no HD (p. ex. sidux.iso)
  • Um arquivo modificado (p. ex. patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta)

Digite no terminal, como usuário:

$ cd /caminho/para/iso/e/patch
$ xdelta patch patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta

O programa vai extrair os arquivos necessários do patch ("remendo") e conferir o md5. Atenção: ele não verifica se há espaço suficiente no HD.
