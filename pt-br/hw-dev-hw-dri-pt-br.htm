Drivers Proprietários para: Nvidia e ATI

Há quatro maneiras para você instalar drivers 3D em seu sistema:

  1. sgfxi / smxi

    Use a ferramenta administrativa multi-função smxi/sgfxi, que significa 'sidux-maintenance', ou seja, 'manutenção do sidux' e é uma contribuição da comunidade. Para saber mais, leia o tópico Comandos do smxi.

  2. Usar os instaladores da NVidia e da ATI

    Baixe o driver binário direto da NVidia
Baixe o driver binário direto da ATI

Drivers Xorg de Código Aberto para nVidia, ATI, Radeon, Intel

Drivers Xorg de código aberto para nVidia, ATI, Radeon, Intel vêm pré-instalados no sidux. Se você sobrescreveu-os com drivers proprietários e deseja voltar atrás, você pode usar o sgfxi para isso.

Os drivers da X.Org (veja xserver-xorg para uma descrição detalhada) dão suporte em 2D para as placas Riva, TNT, GeForce e Quadro da NVIDIA e Mach, Rage, Radeon e FireGLcards da ATI, além dos sub-drivers 'atimisc', 'r128' e 'radeon'. Há suporte também para a série 3D livre da Intel.

Para instalar:

Pressione Ctrl+Alt+F1 para sair do KDE e entrar no modo texto
Logue-se como root
Digite init 3
(para ter certeza de que o X não está rodando mais)
apt-get update
apt-get install xserver-xorg-video-nv
OU
apt-get install xserver-xorg-video-ati
OU
xserver-xorg-video-intel
init 5 && exit

Como dissemos, o suporte normal é 2D, apesar de haver placas onde é possível o suporte 3D.

A seguir, você precisa editar o arquivo /etc/X11/xorg.conf. ENTRETANTO, FAÇA UMA CÓPIA DELE PRIMEIRO, pois qualquer erro cometido resultará em muitas lágrimas pra você...

Para editá-lo, clique nele com o botão direito e abra-o como root (você pode fazer isso pelo teclado também, é claro). Encontre o texto SECTION DEVICE e mude o driver para nv OU ati

AVISO!!! EDITE O XORG.CONF POR SUA PRÓPRIA CONTA E RISCO!!!

Dicas para hardware que precisa de soluções não-livres

Devido à complexidade da legislação, o LiveCD do sidux é montado com software livre somente.

Sources List

Como a ISO do sidux contém apenas software livre, você provavelmente sentirá necessidade de adicionar repositórios com aplicações não-livres (contrib/ non-free) à sua /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Placas de Vídeo
  • Placa de vídeo ATI Radeon: Aceleração 3D para placas antigas funciona até os modelos r35x; placas mais recentes, modelos Radeon X1xxx, precisam de drivers proprietários, porém estes ainda não são compatíveis com o X.org 7.2. Dependendo do chipset, usar a 'cheatcode' "xmodule=vesa" ou "xmodule=fbdev" pode dar resultado para essas placas mais novas.
Firmware
  • Para ajudá-lo a descobrir qual firmware você precisa instalar, rode, como usuário:
$ fw-detect

Com isto você recebe uma lista de recomendações e quais comandos executar. Retorna "null" se você não necessitar de nada ou se todo o firmware já estiver instalado.

  • Atheros/ "madwifi" wlan: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: 
    apt-get install atmel-firmware
  • Broadcom, bcm43xx/ b43/ b43 legacy wlan: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • Broadcom Blutonium Bluetooth chipset (BCM2033): 
    apt-get install bluez-firmware
  • Modem USB ADSL Eagle: baixe o firmware em http://eagle-usb.org/ueagle-atm/non-free/ e coloque-o no diretório /lib/firmware/.
  • Firmwares DVB para várias placas de TV (placas mais baratas provavelmente não precisarão disso): cheque o dmesg e depois baixe o firmware em http://www.linuxtv.org/downloads/firmware/ e coloque-o no diretório /lib/firmware/.
  • Firmware IVTV: obtenha o firmware em http://dl.ivtvdriver.org/ivtv/firmware/ e coloque-o em/lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan: favor remover iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 e firmware-iwl4965, se já estiverem instalados: 
    apt-get install firmware-iwlwifi
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54 MBit/s wlan: baixe o firmware em http://prism54.org/firmware/ e coloque-o em /lib/firmware/.
  • Placas nVidia: aceleração 3D ainda não é possível com os drivers livres. Favor usar get-sidux-binary-gfx ou sgfxi para baixar os scripts de instalação dessas placas.
  • RaLink rt61 e rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    Favor remover os arquivos rt61-fwcutter, rt73-fwcutter, firmware-rt61 e firmware-rt73, se já estiverem instalados.

  • Webcams OEM Ricoh R5U870: baixe o firmware em http://mediati.org/r5u870/r5u870-0.11.0.tar.gz e copie o arquivo r5u870_*.fw para: /lib/firmware/.
  • Texas Instruments ACX100, 22 Mbit/s/ ACX111, 54 MBit/s wlan: 
    m-a a-i acx100
    baixe o firmware em http://www.kazer.org/acx-firmware-20060207.tar.bz2 e coloque-o no diretório /lib/firmware/.
  • ZyDAS zd1201 11 MBit/s wlan: obtenha o firmware em http://surfnet.dl.sourceforge.net/sourceforge/linux-lc100020/zd1201-0.14-fw.tar.gz e coloque-o em /lib/firmware/.
  • ZyDAS zd1211, 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Tentamos oferecer pacotes para firmware legalmente redistribuível em nossos repositórios 'non-free', mas nem todo fabricante permite isso.

Escrevendo em partições NTFS com ntfs-3g

ATENÇÃO:: Ainda que o ntfs-3g seja considerado 'estável', jamais use-o sem antes fazer cópias de seus arquivos importantes; não o use, também, em sistemas de produção! Se o fizer e perder dados, lembre-se de que foi avisado. Portanto, o risco é todo seu!

Abra um terminal e digite os comandos abaixo. Veja Particionando seu HD - Nomes dos Discos

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para sair do terminal, digite: exit

Agora seu volume NTFS deve estar montado rw e você será capaz de armazenar dados nele. Repetimos, porém: use apenas em situações emergenciais, NUNCA no dia-a-dia!!!
