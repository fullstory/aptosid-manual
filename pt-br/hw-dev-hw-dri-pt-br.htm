Dicas para hardware que precisa de soluções não livres

Devido à complexidade da legislação, o LiveCD do sidux é montado com software livre somente.

Sources List

Como a ISO do sidux contém apenas software livre, você provavelmente sentirá necessidade de adicionar repositórios com aplicações não livres (contrib/ non-free) à sua /etc/apt/sources.list.d/debian.list e /etc/apt/sources.list.d/sidux.list ou /etc/apt/sources.list:

# See sources.list(5) for more information, especially    [Veja sources.list (5) para mais informações]
# Remember that you can only use http, ftp or file URIs     [Lembre-se de que você só pode usar http, ftp ou URIs] 
# CDROMs are managed through the apt-cdrom tool.     [CDROMs são gerenciados pelo apt-cdrom]
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list     [O diretório /etc/apt/sources.list.d/ fornece uma maneira de adicionar]
# entries in separate files that end with sources.list file.     [linhas em arquivos separados, que comporão a sources.list final]

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Drivers binários de código fechado para nVidia, com dmakms

Prepare o módulo:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Então ative o Dynamic Module-Assistant Kernel Module Support (dmakms) para nNidia, de forma que, da próxima vez que o kernel for atualizado, um módulo nVidia também será preparado para ele, sem intervenção manual. Para isso, adicione nvidia-kernel-source ao arquivo de configuração /etc/default/dmakms .

Daí:

m-a a-i nvidia-kernel-source

Seguido de:

apt-get install nvidia-glx

Em seguida, altere o arquivo xorg.conf:

sed -i s/nv\"/nvidia\"/ /etc/X11/xorg.conf

Reinicie seu PC para que a instalação do módulo tenha efeito

Module load failure

Should nvidia fail to load, for whatever reason [new xorg component, a fs problem or if X doesnt start after reboot] :

modprobe nvidia

Then reboot the computer.

Should the module still fail to load:

 m-a a-i -f nvidia-kernel-source

This rebuilds the module then reboot.

Leia:

$ /usr/share/doc/dmakms

Drivers Xorg de código aberto para nVidia, ATI, Radeon e Intel

Drivers Xorg de código aberto para nVidia, ATI, Radeon e Intel vêm pré-instalados no sidux.

Os drivers da X.Org (veja xserver-xorg para uma descrição detalhada) dão suporte em 2D para as placas Riva, TNT, GeForce e Quadro da NVIDIA e Mach, Rage, Radeon e FireGLcards da ATI, além dos sub-drivers 'atimisc', 'r128' e 'radeon'. Há suporte também para a série 3D livre da Intel.

Para instalar:

Pressione Ctrl+Alt+F1 para sair do KDE e entrar no modo texto
Logue-se como root
Digite init 3
(para ter certeza de que o X não está rodando mais)
apt-get update
apt-get install xserver-xorg-video-nv
OU
apt-get install xserver-xorg-video-ati
OU
xserver-xorg-video-intel
init 5 && exit

Como dissemos, o suporte normal é 2D, apesar de haver placas onde é possível o suporte 3D.

Se você tem usado drivers proprietários e quiser reverter para drivers de código aberto, será necessário editar o arquivo /etc/X11/xorg.conf. NO ENTANTO, ANTES DE MAIS NADA FAÇA UMA CÓPIA DELE, de vez que editar incorretamente o /etc/X11/xorg.conf irá resultar em muitas lágrimas para você!!

Para editá-lo, clique nele com o botão direito e abra-o como root (você pode fazer isso pelo teclado também, é claro). Encontre o texto SECTION DEVICE e mude o driver para nv OU ati OU intel.

A edição do xorg.conf é totalmente por sua conta e risco!!!

$ fw-detect

Como escrever em partições NTFS com ntfs-3g

ATENÇÃO:: Ainda que o ntfs-3g seja considerado 'estável', jamais use-o sem antes fazer cópias de seus arquivos importantes; não o use, também, em sistemas de produção! Se o fizer e perder dados, lembre-se de que foi avisado. Portanto, o risco é todo seu!

Abra um terminal e digite os comandos abaixo. Veja Particionando seu HD - Nomes dos Discos

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para sair do terminal, digite: exit

Agora seu volume NTFS deve estar montado rw e você será capaz de armazenar dados nele. Repetimos, porém: use apenas em situações emergenciais, NUNCA no dia-a-dia!!!
