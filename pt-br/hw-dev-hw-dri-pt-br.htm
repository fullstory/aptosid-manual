Dicas para hardware que precisa de soluções não livres

A live-ISO do sidux contém somente software livre.

Sources List

Como a ISO do sidux contém apenas software livre, você provavelmente sentirá necessidade de adicionar repositórios com aplicações não livres (contrib non-free) às suas /etc/apt/sources.list.d/debian.list e /etc/apt/sources.list.d/sidux.list. As linhas que você inserir devem se parecer com estas, dependendo dos espelhos ('mirrors') que você escolheu:

# Debian
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main fix.main
#deb-src http://sidux.com/debian/ sid main fix.main

Após alterar sua sources. list, é necessário que você atualize a base com um apt-get update antes de procurar ou instalar novos pacotes. (o 'apt-get update' deve sempre ser usado antes de instalar qualquer coisa).

Os drivers Xorg de código aberto para nVidia, ATI, Radeon, Intel & xorg.conf

Drivers Xorg de código aberto para nVidia, ATI, Radeon e Intel vêm pré-instalados no sidux.

Os drivers da Intel, incluídos em sua série de drivers livres, devem funcionar perfeitamente com placas de vídeo 2D e 3D.

Nota: os drivers livres geralmente não necessitarm do xorg.conf. Se esse arquivo não existir e você precisar dele por algum motivo, digite em um terminal, como root:

X -configure

Este comando cria um arquivo xorg.conf, com as informações essenciais, coloca-o no diretório /root e o nomeia como xorg.conf.new. O próximo passo é movê-lo e renomeá-lo:

mv /root/xorg.conf.new /etc/X11/xorg.conf

Para editá-lo, clique nele com o botão direito e abra-o como root (você pode fazer isso pelo teclado também, é claro). Encontre o texto SECTION DEVICE e mude o driver para nv OU radeon OU intel (só para citar alguns).

A edição do xorg.conf é totalmente por sua conta e risco!!!

Drivers binários de código fechado para nVidia, com dmakms & xorg.conf

Como o xorg.conf geralmente não é necessário, exceto se você tiver a intenção de rodar drivers de vídeo de código fechado, novas instalações precisam ter certeza de que o arquivo xorg.conf existe em /etc/X11. Se não existir, crie um arquivo temporário em sua $home com o nome de xorg.conf.dmakms e copie o seguinte nele:

# /etc/X11/xorg.conf (Xorg X Window System server configuration file) ('Arquivo de configuração do Xorg')
# Edit this file with caution, and see the xorg.conf(5) manual page. ('Edite este arquivo com cuidado e veja a página de manual do xorg.conf (5)')
# (Type "man xorg.conf" at the shell prompt.) ('Digite "man xorg.conf" no prompt do terminal')
#
Section "Device"
         Identifier      "Device 0"
         Driver          "nvidia"
EndSection

# This is a trailing line, it is needed so that End Section is not the last line

Se você tiver mais de uma placa de vídeo, será necessário descobrir o PCI e incluí-lo no arquivo xorg.conf:

lspci | grep -i vga

Isso deve resultar em uma sintaxe similar a esta:

01:00.0 VGA compatible controller:

Adicione o busid 01:00.0 como uma linha extra debaixo da linha 'Driver'; entretanto, note que a sintaxe é PCI:x:y:z: com um zero sendo desprezado e com a adição dos dois pontos, assim:

BusID		 "PCI:1:0:0"

Salve e mova o arquivo para /etc/X11:

mv xorg.conf.dmakms /etc/X11/xorg.conf

NOTA: Use 'apt-cache search nvidia' e 'apt-cache show <pacote>' para descobrir o driver certo para você. Basicamente, há duas possibilidades: drivers 3D correntes do Debian Sid e drivers 3D legados ('legacy 3D drivers') do Debian Sid.

Para os drivers 3D nVidia correntes: ≥ GeForce 6xxx :

Prepare o módulo:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Então ative o Dynamic Module-Assistant Kernel Module Support (dmakms) para nVidia, de forma que, da próxima vez que o kernel for atualizado, um módulo nVidia também será preparado para ele, sem intervenção manual. Para isso, adicione nvidia-kernel-source ao arquivo de configuração /etc/default/dmakms :

echo nvidia-kernel-source >> /etc/default/dmakms

Daí:

m-a a-i nvidia-kernel-source

Seguido de:

apt-get install nvidia-glx

Em seguida, altere o arquivo xorg.conf:

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Reinicie seu PC para que a instalação do módulo faça efeito.

Quando o Xorg for atualizado, você vai precisar somente de reinstalar nvidia-glx:

apt-get install --reinstall nvidia-glx

Quando um novo driver da nVidia estiver disponível:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Reinicie sua máquina para que a instalação do módulo faça efeito.

Para os drivers nVidia legados ≥ GeForce 5xxx :
m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Altere o dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Quando o xorg for atualizado, você vai precisar de instalar somente o nvidia-glx-legacy:

apt-get install --reinstall nvidia-glx-legacy-173xx

Falha no carregamento do módulo

Se o carregamento do módulo nVidia falhar, por um motivo ou outro (um componente novo do xorg, um problema no sistema de arquivos, uma recusa do X em iniciar):

modprobe nvidia

Então, reinicie o computador.

Se ainda assim o módulo não carregar:

 m-a a-i -f nvidia-kernel-source

ou

 m-a a-i -f nvidia-kernel-legacy-173xx-source

Com isso, o módulo é reconstruído. Agora basta reiniciar.

Leia:

$ /usr/share/doc/dmakms

Você também pode ler nvnews para outras opções.

ATI, Radeon e outros drivers de vídeo

Drivers 2D

Os drivers da X.Org (veja xserver-xorg para uma descrição detalhada) dão suporte em 2D para as placas Riva, TNT, GeForce e Quadro da NVIDIA e Mach, Rage, Radeon e FireGL da ATI, além dos subdrivers atimisc, r128 e radeon. Tanto Radeon quanto Intel dão suporte a aceleração 2D (xv texturizado) para reprodução de vídeo.

Drivers 3D

Algumas placas ATI também dão suporte a 3D (e animações KDE) com xserver-xorg-video-radeon. Até o momento, o suporte atinge chipsets até r700.

Para atualizar automaticamente novos pacotes de firmwares não livres para placas de vídeo 2D e 3D:

apt-get install firmware-linux

Então, reinicie o computador.

Drivers para firmware

O firmware necessário pode ser determinado pelo script recém-disponibilizado fw-detect (empacotado em sidux-scripts).

$ fw-detect

A saída de fw-detect detalha os comandos necessários para instalar e ativar o firmware:

Exemplo:

#Detected driver that requires firmware to operate
#Follow these instructions to obtain the correct firmware
# and activate the zd1211rw driver:
apt-get update
apt-get install zd1211-firmware
modprobe -r zd1211rw
modprobe zd1211rw

Para instalar firmare pelos repositórios git, você precisa de:

apt-get install git-core
pacotes .deb com firmware não-livre para pendrives

Se você tiver de preparar pacotes .deb de firmware e transferi-los para outro computador usando uma pendrive ou algum outro dispositivo USB, você pode baixá-los tanto como arquivos .zip quanto como arquivos tar.gz em http://cdimage.debian.org/cdimage/unofficial/non-free/firmware/sid/current/. Depois, extraia-os para uma pasta com o nome de firmware (devem aparecer vários arquivos .deb). A seguir, baixe firmware-linux-free*.deb e adicione-o a todos os outros debs na pasta. Daí, transfira tudo para a pendrive e desta para o computador que precisa do firmware não livre e:

dpkg -i firmware-linux-nonfree*.deb

Tentamos oferecer pacotes para firmware legalmente redistribuível em nossos repositórios 'non-free', mas nem todo fabricante permite isso.

Como escrever em partições NTFS com ntfs-3g

ATENÇÃO:: Ainda que o ntfs-3g seja considerado 'estável', jamais use-o sem antes fazer cópias de seus arquivos importantes; não o use, também, em sistemas de produção! Se o fizer e perder dados, lembre-se de que foi avisado. Portanto, o risco é todo seu!

Abra um terminal e digite os comandos abaixo. Veja Particionando seu HD - Nomes dos Discos

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Para sair do terminal, digite: exit

Agora seu volume NTFS deve estar montado rw e você será capaz de armazenar dados nele. Repetimos, porém: use apenas em situações emergenciais, NUNCA no dia-a-dia!!!
