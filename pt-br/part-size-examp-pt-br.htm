Tamanhos das Partições e Exemplos

Para uso normal, recomendamos o ext3, que é o sistema de arquivos padrão do sidux; além disso, ele possui uma boa manutenção.

Com o Gerenciador de Partições GParted, você pode formatar e particionar seu HD. O programa possui interface gráfica e é auto-explicativo.

O Gparted também é capaz de diminuir e mover partições, inclusive NTFS. Apesar de ainda não estar completamente desenvolvido e de chamar a si mesmo "experimental", o Gparted não lhe causará problemas se você usá-lo da maneira correta. Sua documentação está aqui. Mudanças em partições NTFS também podem ser feitas com ferramentas proprietárias, como Partition Magic (TM) e Acronis (TM).

SEMPRE FAÇA BACKUPS (CÓPIAS) DE SEUS DADOS!

Se o GParted mostrar que a partição está montada, é necessário desmontá-la primeiro. Para isso, feche o aplicativo, localize na Área de Trabalho o ícone correspondente à partição, clique com o botão direito sobre ele e, no menu de contexto, clique em 'Unmount' (Desmontar). Se se tratar de partição swap, desmonte-a com: "sudo swapoff -a" (sem as aspas) no terminal. Quando terminar, reabra o GParted. Em princípio, 3 GB são suficientes para instalar o sidux no HD, mas tão pouco não lhe proporcionará muita diversão. Um tamanho razoável fica entre 5 e 8 GB. Se você é iniciante no Linux, sugerimos que utilize apenas 2 partições (root e swap), para facilitar sua vida. Se quiser, faça também uma partição extra para /home.

Usuários avançados podem querer partições adicionais para /var, /tmp, ...etc, por razões especiais. Não vamos entrar em detalhes, para não nos desviarmos de nossos objetivos aqui. Você vai mesmo precisar de uma partição swap (semelhante ao 'swapfile' do Windows, porém mais eficiente), cujo tamanho deve ter o dobro de sua memória RAM. E lembre-se: o sistema de arquivos padrão do sidux é o ext3.

Para trocar dados com o Windows, você deve usar vfat (fat32). Se você quiser usar o sistema de arquivos XFS para / (root), você terá de criar uma partição /boot (ext2) ou usar o lilo (porque o GRUB não funciona bem com o XFS). No momento, é preciso editar o arquivo ~/siduxconf se se quiser que o instalador do sidux trabalhe com XFS na /.

Se você criou mais de uma partição (além da swap), escreva seus nomes em um pedaço de papel, porque vai precisar mais tarde. Feito isso, salve as alterações (é o ícone do disquete) e feche o GParted. Listamos abaixo alguns exemplos com diferentes tamanhos de HD e diferentes tipo de uso.

Este último exemplo é recomendável para HDs com pouco espaço. O Windows é instalado em uma partição separada de 3 - 5 GB, onde nada mais é colocado. Todos os programas e arquivos vão para uma partição grande. Se você tiver mais espaço, você pode fazê-la ainda maior ou dividi-la em duas, uma só com os programas do Windows, outra para os dados que serão trocados com o Linux.

Resumindo

Para uma instalação do sidux sem jogos e/ou aplicações multimídia especiais, 5GB são suficientes. Para dados, use partições fat32. Há muitas maneiras de particionar seu HD. Os exemplos dados devem ser o bastante para começar. Uma boa solução é adquirir um segundo HD, para aumentar o rendimento, o conforto e a segurança. Neste caso, o Windows tem sempre de ficar na primeira partição do primeiro HD (Master)!
