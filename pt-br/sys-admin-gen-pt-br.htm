Como habilitar serviços no sidux

insserv (Pontos 2008-04 and forward ): To start/stop already installed services:

Read /usr/share/doc/insserv/README.Debian, the release notes and the man pages carefully:

 $ man insserv 
 $ man invoke-rc.d 
 $ man update-rc.d 
 google LSB headers

All access to the init scripts by Debian packages’ maintainer scripts should be done through invoke-rc.d.

For 'start':

invoke-rc.d <service> start

For 'stop':

invoke-rc.d <service> stop

For 'restart:

invoke-rc.d <service> restart

To prevent services running at-startup

update-rc.d <service> remove
[will remove all startup links]

To make sure a service runs at boot [not always required]

update-rc.d <service> defaults
[will make startup links]

Older Installs (before Pontos 2008-04)

insserv on installed systems before Pontos 2008-04

Warning: Older installs will not get insserv automatically, due to users own initscripts or symlinks. It is NOT recommended to install insserv on installations prior to Pontos. This package should be used with care, as incorrect or missing LSB dependencies of existing initscripts can lead to an unbootable system. Installing insserv on an existing install may require some manual changes.

Read read the Debian documentation notes, and release notes before installing and enabling insserv.

Os serviços são atualizados sempre que você roda um dist-upgrade. Serviços desabilitados antes do dist-upgrade ficam habilitados novamente.

Para rodar um serviço em uma sessão ou para gerenciar quais serviços serão habilitados ou não no arranque, digite no terminal:

invoke-rc.d serviço start

ou

rcconf

Nota sobre o rcconf: use as setas direcionais para mover-se para cima/para baixo, Espaço para marcar/desmarcar, Tab para OK ou Cancelar e ENTER para aplicar as mudanças.

Você também pode usar o sidux Control Centre ('Centro de Controle do sidux').

Como matar um serviço ou processo

pkill é útil porque é fácil de entender e funciona tanto no nível de usuário quanto no de root, no terminal ou no tty:

pkill -n serviço

Se você estiver em dúvida quanto à grafia do processo ou serviço que deseja matar, o pkill <tab> <tab> fornece uma listagem.

htop também é uma boa alternativa. (em último caso, use killall -9).

Runlevels (níveis de execução) no sidux - init

Esta é a lista dos runlevels do sidux; note-se que ela difere da lista do Debian stable:

  • init 0 desligar o PC
  • init 1 modo monousuário
  • init 2 modo multiusuário sem rede e/ou fechar ou não entrar no X
  • init 3 modo multiusuário com rede não rodando o servidor gráfico X e/ou fechar ou não entrar no X
  • init 4 fechar ou não entrar no X
  • init 5 modo multiusuário com rede rodando o servidor gráfico X e/ou iniciar o X
  • init 6 reiniciar o PC

Perda de senha root

Não é possível recuperar uma senha perdida, mas você pode escolher outra:

Primeiro, dê o boot pelo LiveCD.

Como root, monte sua partição "/" (por exemplo, /dev/hdb2):

# mount /dev/hdb2
Agora, chroot em sua velha partição root com:
# chroot /media/hdb2 passwd
e especifique uma nova senha.

Fontes no sidux

Configuração correta do dpi - filosofia básica

A configuração correta do DPI é difícil de descobrir, mas o X faz isso perfeitamente se 'DisplaySize' mostrar as dimensões físicas da tela, na seção "Monitor Section" do arquivo /etc/X11/xorg.conf.

Resoluções corretas e atualizações da tela

Todo monitor tem sua perfeita combinação de configurações, porém, infelizmente, nem todos mostram os valores de DCC corretos, o que acaba por exigir trabalho manual.

Drivers de placas de vídeo

Algumas placas ATI e NVidia mais recentes não funcionam bem com os drivers livres do Xorg e a única solução viável é valer-se dos drivers comerciais de código fechado. O sidux não os pré-instala por razões legais. A solução pode ser encontrada aqui

Fontes padrão, renderização e tamanhos

O sidux usa as fontes livres pré-selecionadas pelo Debian, comprovadamente bem equilibradas. Usar uma seleção diferente pode resultar em deteriorização da qualidade de renderização. Mas há algumas opções poderosas (sem falar das configurações do KControl), que também podem ajudar a conseguir renderizações suaves com outras fontes. Lembre-se, no entanto, que cada fonte possui apenas uns poucos tamanhos perfeitos; outros já não dão um resultado tão bom.

Experimentar com os valores do DPI também pode ajudar:

fix-dpi-kdm

Isso mostra o DPI para sua tela, mas você pode brincar à vontade desde que o faça no init 3 (e depois volte para o init 5 ou simplesmente reinicie sua máquina).

Depois de mudar os tipos de fonte ou os tamanhos do DPI (no X ou no Firefox/Iceweasel), podem ser necessários alguns ajustes até que tudo fique do seu gosto. Isso acontece principalmente quando você muda as fontes de Bitmap para Truetype ou vice-versa:

dpkg-reconfigure fontconfig-config

Selecione Native e Autohinter. Se não gostar, experimente até conseguir os melhores resultados.

Pode ser necessário reconstruir o cache das fontes:

fc-cache -f -vv

Se não funcionar, você terá de reinstalar o pacote com um arquivo de configuração padrão do cache de suas fontes:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Aplicações baseadas em GTK, como Firefox/Icewasel

Em geral, esse tipo de aplicações é problemático com os padrões do KDE. Para resolver, digite como root no terminal:

apt-get install gtk2-engines gtk-qt-engine

Abra o KControl e você encontrará um novo item para GTK. Configure-o para usar Clearlooks e as fontes do KDE.

Caso aconteçam problemas de estabilidade, comuns até (mas não após) 2006, use os comandos abaixo para desinstalar ou purgar o pacote gtk-qt-engine. As configurações serão mantidas em $HOME; não há dependências a remover:

apt-get remove gtk-qt-engine
ou
apt-get remove --purge gtk-qt-engine

Isso DEVE acertar a renderização das fontes em suas aplicações GTK

CUPS

Há uma grande seção dedicada ao CUPS na Ajuda do KDE, porém constantes dist-upgrades podem fazer com que ele passe a ter um comportamento errático. Eis uma solução:

modprobe lp
echo lp >> /etc/modules 
apt-get remove --purge cupsys cups
apt-get install cups 
	OU
apt-get install cups cupsys-driver-gutenprint hplip

Então, abra o navegador Web e digite na barra de endereços: 

http://localhost:631

Assegure-se de que o CUPS está rodando:

/etc/init.d/cups restart

Outra pegadinha: quando você vai configurar o CUPS pela interface gráfica, uma caixa de diálogo pede sua senha root, mas o que vem preenchido nela é seu nome de usuário; então, se você coloca sua senha de usuário, não funciona. Na verdade, o que você tem de fazer é mudar seu nome de usuário para root e entrar sua senha root.
