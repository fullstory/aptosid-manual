Habilitando serviços no sidux

O sidux roda os seguintes serviços, por padrão: cpid, cron, dbus, hotkey-setup, kdm, klogd, sysklogd; eles são atualizados a cada dist-upgrade. Quaisquer outros serviços instalados após um dist-upgrade também ficam ativados.

Para iniciar um serviço em uma sessão, use:

/etc/init.d/service start

Para ativar serviços na inicialização do sistema:

rcconf

Veja a observação a seguir:

Nota sobre o rcconf: Use as setas direcionais para mover-se para cima/para baixo, Espaço para marcar/desmarcar, Tab para OK ou Cancelar e ENTER para aplicar as mudanças

Uma alternativa para o rcconf é:

update-rc.d service defaults

Perda de senha root

Não é possível recuperar uma senha perdida, mas você pode escolher outra:

Primeiro, dê o boot pelo LiveCD

Como root, monte sua partição / (por exemplo, /dev/hdb2)

# mount /dev/hdb2
Agora, chroot em sua velha partição root com:
# chroot /media/hdb2
... e use o comando passwd para escolher uma nova senha:
# passwd

Fontes no sidux

Configuração correta do dpi - Filosofia Básica

A configuração correta do DPI é difícil de descobrir, mas o X faz isso perfeitamente se 'DisplaySize' mostrar as dimensões físicas da tela, na seção "Monitor Section" do arquivo /etc/X11/xorg.conf.

Resoluções corretas e Atualizações da tela

Todo monitor tem sua perfeita combinação de configurações, porém, infelizmente, nem todos mostram os valores de DCC corretos, o que acaba por exigir trabalho manual.

Drivers de Placas de Vídeo

Algumas placas ATI e NVidia mais recentes não funcionam bem com os drivers livres do Xorg e a única solução viável é valer-se dos drivers comerciais de código fechado. O sidux não os pré-instala por razões legais. A solução pode ser encontrada aqui

Fontes padrão, renderização e tamanhos

O sidux usa as fontes livres pré-selecionadas pelo Debian, comprovadamente bem equilibradas. Usar uma seleção diferente pode resultar em deteriorização da qualidade de renderização. Mas há algumas opções poderosas (sem falar das configurações do KControl), que também podem ajudar a conseguir renderizações suaves com outras fontes. Lembre-se, no entanto, que cada fonte possui apenas uns poucos tamanhos perfeitos, outros já não dão um resultado tão bom.

Experimentar com os valores do DPI também podem ajudar:

fix-dpi-kdm

Isso mostra o DPI para sua tela, mas você pode brincar à vontade desde que o faça no init 3 (e depois volte para o init 5 ou simplesmente reinicie sua máquina).

Depois de mudar os tipos de fonte ou os tamanhos do DPI (no X ou no Firefox/Iceweasel), podem ser necessários alguns ajustes até que tudo fique do seu gosto. Isso acontece pricipalmente quando você muda as fontes de Bitmap para Truetype ou vice-versa:

dpkg-reconfigure fontconfig-config

Selecione Native e Autohinter. Se não gostar, experimente até conseguir os melhores resultados.

Se, no fim, você não ficar satisfeito, vai ser necessário rezafer o cache das fontes:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Aplicações baseadas em GTK, como Firefox/Icewasel

Em geral, esse tipo de aplicações é problemático com os padrões do KDE. Para resolver, digite como root no terminal:

apt-get install gtk2-engines gtk-qt-engine

Abra o KControl e você encontrará um novo item pata GTK. Configure-o para usar Clearlooks e as fontes do KDE. fonts.

Caso aconteçam problemas de estabilidade, comuns até (mas não desde) 2006, use os comandos abaixo para desinstalar ou purgar o pacote gtk-qt-engine. As configurações serão mantidas em $HOME; não há dependências a remover:

apt-get remove gtk-qt-engine
or
apt-get remove --purge gtk-qt-engine

Isso DEVE acertar a renderização das fontes em suas aplicações GTK

CUPS

Há uma grande seção dedicada ao CUPS na Ajuda do KDE, porém constantes dist-upgrades podem fazer com que ele passe a ter um comportamento errático. Eis uma solução:

modprobe lp
echo lp >> /etc/modules 
apt-get remove --purge cupsys 
apt-get install cupsys 
	OU
apt-get install cupsys cupsys-driver-gutenprint hplip

Então, abra o navegador Web e digite na barra de endereços: 

localhost:631

Assegure-se de que o CUPS está rodando: /etc/init.d/cups restart

Outra pegadinha: quando você vai configurar o CUPS pela interface gráfica, uma caixa de diálogo pede sua senha root, mas o que vem preenchido nela é seu nome de usuário; então, se você coloca sua senha de usuário, não funciona. Na verdade, o que você tem de fazer é mudar seu nome de usuário para root e entrar sua senha root.

Runlevels (níveis de execução) no sidux - init

Esta é a lista dos runlevels do sidux; note-se que ela difere da lista do Debian stable:

  • init 0 desligar o PC
  • init 1 modo single user (usuário único)
  • init 2 modo múlti-usuário sem rede e/ou fechar ou não entrar no X
  • init 3 modo múlti-usuário com rede não rodando o servidor gráfico X e/ou fechar ou não entrar no X
  • init 4 fechar ou não entrar no X
  • init 5 modo múlti-usuário com rede rodando o servidor gráfico X e/ou iniciar o X
  • init 6 reiniciar o PC
