Instalando novos Kernels

Os kernels do sidux ficam aqui. Favor escolher o espelho mais perto de você:

Espelhos principais
http://debian.tu-bs.de/project/sidux/kernels Alemanha
http://sidux.net/files//kernel Alemanha
ftp://ftp.spline.de/mirrors/sidux/kernel/ Alemanha

Os espelhos abaixo podem estar atrasados em até 24 horas em relação aos espelhos principais
http://www.mirrorservice.org/sites/sidux.com/sidux/kernels Reino Unido
http://ftp.heanet.ie/pub/sidux//kernel Irlanda
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernels Tennessee, EUA
http://ftp-mirror.internap.com/pub/sidux/kernel  GA. EUA
http://ftp.tw.debian.org/pub/Sidux/ Ásia
http://mirror.aarnet.edu.au/pub/sidux/kernel Austrália
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernels África do Sul

Os kernels estão disponíveis nas seguintes formas:

  • sidux-686 - Para processadores da família i686 com single/dual core ou mais CPUs
  • sidux-amd64 - Para processadores da família AMD64, quando se está usando a versão 64 bit do sidux
Siga estes 04 passos para realizar a operação manualmente, no console:
apt-get update

1. Baixe a versão mais recente do kernel:

wget -O - http://sidux.com/debian/pool/main/l/linux-2.6-sidux 2>/dev/null | awk "/linux-image.*$(dpkg-architecture -qDEB_HOST_ARCH_CPU)/" | sed -e 's/^.*linux\-image\-//' -e's/_.*\.deb.*$//'

2. Encontre os módulos que você necessita. O comando abaixo lista os módulos disponíveis atualmente, copy this line to your console/terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Se quiser ver a descrição completa de cada módulo, copy this line to your console/terminal:

apt-cache search 2.6.*.slh.*-sidux

3. Para instalar o kernel, 'headers' (cabeçalhos) e os módulos necessários (por exemplo, virtual-ose e qc-usb):

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686
apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686

4.Agora, reinicie seu computador e o novo kernel será carregado automaticamente, no lugar do anterior. Se ele lhe trouxer problemas, reinicie e escolha um kernel mais antigo.

Para verificar os módulos carregados no kernel:

ls /sys/module/
ou
cat /proc/modules

IMPORTANTE: se você faz uso de certos módulos de drivers (p. ex., wlan), você terá de reinstalar os pacotes .deb apropriados. As an example, for the Agere et131x ethernet adapter: 

apt-get install et131x-modules-*

Veja também Configurando a rede e ndiswrapper e WIFI

A atualização do kernel está pronta. Reinstale o driver da placa de vídeo 3D, se você usa uma. Se ela for da ATI ou NVIDIA, na primeira vez que você reiniciar a máquina com o novo kernel é necessário parar no runlevel 3 (é só digitar 3 na tela inicial do Grub).

Há um método seguro e semi-automatizado para se instalar um novo kernel: o script smxi:
apt-get update
apt-get install sidux-scripts
smxi

You can also do this with the GUI control centre siduxcc in KDE-Start-Menu

Removendo kernels antigos

Depois de o novo kernel ter sido instalado com sucesso, o kernel.zip e o correspondente arquivo descompactado podem ser apagados. No entanto, é recomendável que você os guarde por alguns dias, para qualquer eventualidade

Recomendamos também que você mantenha sempre o kernel anterior, para o caso do novo não rodar satisfatoriamente. Havendo problemas, basta selecioná-lo na lista que o GRUB mostra em sua tela de abertura. Já os kernels antigos podem ser removidos do sistema. Para isso, use o script kernel-remover, que está no pacote sidux-kernelhacking! Ele permite que você escolha quais kernels deseja remover.

kernel-remover
