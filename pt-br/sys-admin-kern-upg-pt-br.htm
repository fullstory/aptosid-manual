Instalando novos Kernels

Os kernels do sidux ficam no APT. Favor escolher o espelho mais próximo e alterar sua sources.list.:

Os kernels estão disponíveis nos seguintes formatos:

  • sidux-686 - kernel para a família de processadores i686, com dual core ou não e mais CPUs
  • sidux-amd64 - kernel para a plataforma de 64 bit
Eis os quatro passos a serem seguidos para instalar um novo kernel manualmente, pelo terminal:

1. Abra o terminal e torne-se root; depois digite:

apt-get update

2. Para instalar a última versão do kernel:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3. A atualização do kernel está feita. Se você tiver uma placa ATI, nVidia ou Intel, será necessário reinstalá-la. Primeiro, atualize o script:

sgfxi -U

4.Agora, reinicie seu computador com o novo kernel (é o que está na primeira linha do GRUB).

Para atualizar os drivers 3D da placa de vídeo, execute:

sgfxi

Alternativamente, quando reiniciar o micro com o novo kernel pela primeira vez, vá direto pra o runlevel 3 (basta digitar 3 na Linha de Comando do GRUB) e dai rodar o sgfxi.

Informações mais detalhadas sobre o sgfxi aqui

Se o novo kernel trouxer problemas, você pode reiniciar a máquina e escolher um kernel antigo.

Módulos

Para saber quais os módulos que você precisa, o comando a seguir lista os módulos disponíveis; copie-o para o terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Copie a linha seguinte para o terminal, caso queira uma descrição completa de cada módulo:

apt-cache search 2.6.*.slh.*-sidux

Para instalar os módulos necessários (por exemplo, virtual-ose e qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXEMPLO)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXEMPLO)

Para checar os módulos carregados no kernel:

ls /sys/module/
ou
cat /proc/modules

IMPORTANTE: Se precisar de certos módulos, como wlan, você precisa instalar os pacotes .deb correspondentes para eles. Por exemplo, para o adaptador ethernet Agere et131x: 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.deb somente é possível se você já tiver os pacotes .deb baixados em sua máquina. Mesmo assim, se eles estiverem no apt-cache, o apt-get é uma opção melhor.

Outros métodos de instalação do kernel

smxi

O script smxi proporciona um método integrado de instalação de kernels e drivers; são instalados, ainda, vários scripts relacionados com tarefas administrativas:

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Você pode também instalar kernels pela interface gráfica siduxcc , que fica no KMenu>Sistema>sidux Control Centre. O siduxcc possui diversas outras aplicações gráficas referentes a administração do sistema

Removendo kernels antigos

Depois de um novo kernel instalado com sucesso, os kernels antigos podem ser apagados, ainda que seja prudente ficar com pelo menos um deles por mais alguns dias, pois pode acontecer de você enfrentar imprevistos com o novo. Dessa forma, se for preciso, basta selecioná-lo na tela do Grub.

Para remover kernels antigos do sistema, use o script kernel-remover , que está no pacote sidux-kernelhacking. Este script precisa estar instalado e permite que você indique quais kernels deverão ser apagados.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

Alternativamente, você pode usar o siduxcc, que também usa o script kernel-remover.
