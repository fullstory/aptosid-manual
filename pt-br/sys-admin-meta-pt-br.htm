Meta-Packages (Meta-Pacotes)

Durante a instalação no HD, você pode ainda incluir meta-pacotes de várias aplicações e jogos.

Se preferir, isso pode ser feito mais tarde, pois o instalador desses meta-pacotes também está disponível em: Kmenu>Utilities>sidux-meta-installer

Até agora, a lista inclui:

Educação:
Jogos:

Multimídia - VDR:
Redes: - Tor e Privoxy
Não-livres : - Drivers diversos
OpenOffice.org : - Para a ISO Lite do sidux
Micros Antigos: - Aplicações e gerenciadores de janelas leves
Escritório : - Gnumeic + Abiword, KOffice e OpenOffice.org
Servidor: - sidux LAMP
Gerenciadores de Janelas:
- icewm (desktop leve)
- Fluxbox (desktop leve)
- fvwm-crystal (desktop leve)
- Xfce4 (desktop leve)

Nota: Ao rodar o meta-instalador, você verá esta linha:

paste your code here: (not necessary for normal install, only for preconfig)
(isto é, 'cole seu código aqui: (não é necessário para uma instalação normal, apenas para preconfig')

Tudo que você precisa fazer é pressionar enter.

O Meta-Instalador e Proxies

Se seu computador estiver conectado à Internet via HTTP-Proxy, é possível que você encontre dificuldades ao rodar o meta-instalador. (Lembre-se: o meta-instalador necessita de privilégios de root.)

Você precisa alterar suas configurações proxy no arquivo apt.conf. Há duas maneiras de se fazer isso: (o Método 1 é melhor, pois é permanente)

Método 1

Defina o proxy no arquivo apt.conf, abra o /etc/apt/apt.conf com o kwrite e adicione o seguinte:

Acquire{
HTTP::proxy "http://192.168.1.1:3128";
FTP::proxy "http://192.168.1.1:3128";
}

Método 2

Abra um terminal, use sux para se tornar root e configure as variáveis de ambiente http_proxy e ftp_proxy. Por fim, rode o meta-installer NA MESMA SESSÃO DE TERMINAL ONDE VOCÊ CONFIGUROU AS VARIÁVEIS PROXY ACIMA, do contrário elas não farão efeito... Você terá que fazer isso sempre que quiser acessar o meta-instalador:

export http:proxy=http://192.168.1.1:3128"
export ftp_proxy="ftp://192.168.1.1:3128/"
