Notas sobre atualização contínua ('Rolling Notes')

O sidux tenta fornecer um dist-upgrade indolor para os pacotes Debian e sidux. No entanto, sempre restam algumas coisinhas que podem ser melhoradas com intervenção manual.

Esta página só será atualizada após o lançamento de cada versão nova. A versão atual é Pontos.

Aviso de segurança do sidux/restrições de leitura da $HOME

Enquanto o instalador do sidux era totalmente remodelado, descobriu-se um potencial problema de escalação de privilégios locais, que deixa uma cópia do arquivo de configuração do instalador no novo sistema. Neste arquivo também estão contidas indicações para os 'hashes' das senhas de root e da conta do primeiro usuário (UID 1000) e pode ser lida pelos usuários locais. Ainda que não revele as senhas diretamente, isso pode ser usado para um ataque de força bruta contra elas.

Versões afetadas:

  • 2007-01 Chaos
  • 2007-02 Tartaros
  • 2007-03/ 2007-03.1 Gaia
  • 2007-04/ 2007-04.5 Eros
  • 2008-01 Nyx
  • 2008-02 Erebos
  • 2008-03 Ourea

Correção:

find /root /home -maxdepth 2 -type f -name '.sidconf*' -delete

Principalmente em sistemas multiusuário, recomendamos trocar a senha tanto do root quanto do primeiro usuário (UID 1000). Isso pode ser feito digitando "passwd" (sem as aspas) como root, em um terminal. O motivo desse problema já foi encontrado e o sidux 2008-04 Pontos (Pontos) não foi afetado.

O Debian cria diretórios /home com permissão de acesso 0751. O sidux diverge desse padrão ao usar 0700 para novas instalações.

sed -i 's/^DIR_MODE=.*/DIR_MODE=0700/' /etc/adduser.conf
dpkg-reconfigure adduser
(nega "sistema totalmente aberto para leitura" aos diretórios home")

Camada de compatibilidade ALSA OSS

Começando com o alsa-driver 1.0.17.dfsg-3, os módulos de compatibilidade do OSS não são mais carregados por padrão. Enquanto isso não traz problemas em modernas versões do Linux, algumas aplicações antigas ('legacy') ou comerciais podem ainda precisar do suporte ao OSS. Dado que o ALSA compilado no kernel oferece uma emulação OSS bastante limitada (não há nenhuma mixagem por software, daí que apenas uma aplicação pode acessar /dev/dsp de cada vez), as aplicações compatíveis com o ALSA têm sempre preferência. Mas caso haja necessidade de compatibilidade com o OSS, existem duas alternativas disponíveis:

  • "capa" aoss
    /usr/bin/aoss é parte do pacote alsa-oss e pode envolver aplicações legadas para capturar chamadas OSS ao mapear de forma transparente a API OSS no ALSA; assim, torna-se possível aproveitar a maioria dos recursos do ALSA sem maiores problemas
  • emulação ALSA direto do kernel
    melhor compatibilidade (criam-se 'nós de dispositivos' (device nodes) /dev/dsp, /dev/audio e /dev/mixer), mas não é possível mixagem por software, portanto só é possível uma aplicação ativa de cada vez.

Na maioria dos casos, é preferível abrir as aplicações afetadas com "aoss <programa>" para manter as vantagens de mixagem por software e acesso concomitante à placa de som, pois devem ser raras as ocasiões em que seja necessária a camada de compatibilidade OSS fornecida direto do kernel. Mas se você precisar, ative isso carregando os módulos snd-pcm-oss, snd-mixer-oss e snd-seq-midi. 

for i in snd-pcm-oss snd-mixer-oss snd-seq-midi; do
	grep -q "^${i}" /etc/modules  || echo "${i}" >> /etc/modules
	grep -q "^${i}" /proc/modules || modprobe "${i}"
done

Broadcom b43 wlan

Kernels >2.6.25 requerem revisões de firmware ≥4.150.10.5 (revisão interna "410.2160 (2007-05-26 15:32:10)") do firmware da Broadcom, enquanto versões mais antigas (até o kernel 2.6.24) requerem 4.80.53.0 (revisão interna "351.126 (2006-07-29 05:54:02)"), que foi instalada por revisões anteriores do b43-fwcutter. O b43-fwcutter não pode atualizar por si mesmo o firmware já existente, portanto a atualização é feita com intervenção manual: 

dpkg --purge b43-fwcutter
rm -rf /lib/firmware/b43 /lib/firmware/b43legacy
apt-get update
apt-get install b43-fwcutter

console-setup

Os mapas de teclado do Terminal agora são gerenciados pelo "console-setup". 

apt-get install console-setup
dpkg-reconfigure console-setup

gspca2

O Kernel 2.6.27 vem com o driver gspca2 v4l2 (que substitui o antigo driver externo gspca v4l1), necessário por um largo número de diferentes webcams. Infelizmente, muitos programas comumente utilizados pelos usuários finais não tomaram conhecimento, ainda, da nova API v412 para os dispositivos anteriormente cobertos pelo gspca; daí que, por enquanto, eles têm de abertos com: "LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <programa>". 

apt-get update
apt-get install libv4l-0

Como usuário:
$ LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <programa>

insserv

insserv em sistemas anteriores ao Pontos 2008-04

Atenção: instalações mais antigas não recebem o insserv automaticamente, devido aos scripts de inicialização ou links simbólicos dos usuários. Recomendamos NÃO instalar o insserv em versões anteriores ao Pontos. Este pacote deve ser usado com cuidado, pois dependências LSB incorretas ou faltantes podem levar a um sistema incapaz de ser inicializado. Instalar o insserv em um sistema já instalado requer algumas mudanças manuais.

Leia as notas de documentação do Debian, e as notas de lançamento antes de instalar e habilitar o insserv.

Pacotes obsoletos

Como parte do contínuo desenvolvimento do sidux, vários pacotes foram substituidos por alternativas melhores, ou tornaram-se redundantes.

  • capi-isdnconf, obsoleto.
  • configure-xawtv, obsoleto.
  • gprsconnect, substituido por umtsmon.
  • ipw2100-firmware, substituido pelo firmware-ipw2x00.
  • ipw2200-firmware, substituido pelo firmware-ipw2x00.
  • modemlink, substituido por umtsmon e wvdial.
  • ppp-scripts, substituido por umtsmon.

O comando abaixo é capaz de encontrar pacotes não disponíveis em nenhum dos repositórios APT e pode dar indicações sobre pacotes obsoletos. Entretanto, seja cuidadoso com os pacotes instalados manualmente. O apt-show-versions é um pacote separado e não é necessariamente préinstalado. 

$ apt-show-versions | awk '/No available version in archive/{print $1}'
