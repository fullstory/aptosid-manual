Guia Rápido do sidux

OBSERVAÇÃO MUITO IMPORTANTE: o sidux, como LiveCD do Linux, é baseado em tecnologia de alta compressão (desenvolvida pioneiramente pelo Knoppix, mas não igual à do Knoppix). Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

O sidux é totalmente compatível com o Debian sid. Pacotes do sidux que substituam pacotes problemáticos do Debian sid são sempre temporários e usados apenas até o momento em que cheguem aos repositórios novas versões desses pacotes, já livres dos bugs. Existem, no entanto, pacotes que são específicos do sidux. Eles incluem scripts e várias ferramentas.

Estabilidade do Debian sid/instável

sid é um apelido dado aos repositórios instáveis do Debian. O Debian sid é atualizado freqüentemente e traz sempre as versões mais recentes dos pacotes mantidos. Por causa dessa alta freqüência de atualizações, não é possível fazer uma grande quantidade de testes nos pacotes, entre o momento em que eles chegam aos repositórios e o momento em que são disponibilizados.

O kernel do sidux

O kernel é otimizado pelo sidux, para ajudar a evitar problemas e adicionar novas funcionalidades. Também é configurado para obter um desempenho melhor e mais estável. Todas essas otimizações são feitas a partir do site http://www.kernel.org/.

Há um espelho para o kernel aqui: Atualizando o kernel

Gerenciamento de pacotes

Conforme dito acima, o sidux é totalmente compatível com o Debian sid. O Debian usa o 'apt-get' e o 'dpkg' para gerenciamento de software.

O Debian sid possui mais de 20.000 pacotes, portanto você, com certeza, encontrará o que precisa. Para isso, use o Procurando Pacotes com o Apt-cache ou o Procurando Pacotes com o Kio-apt

Para instalar o pacote apt-get install <package name>: Instalando um Novo Pacote

O apt-get é usado para o gerenciamento de pacotes em conjunto com os repositórios Debian e outros, identificados pelos arquivos em /etc/sources.list.d/* .

Os repositórios do Debian sid são atualizados uma ou duas vezes por dia, em média. Com isso, a listagem de pacotes disponíveis fica defasada rapidamente. Rodar apt-get update é essencial antes de instalar quaisquer novos pacotes ou antes de um apt-get dist-upgrade ; assim, você mantém seu sistema sempre sincronizado com os pacotes existentes no servidor.

Variantes do Debian, Fontes e RPMs

Instalações a partir dos arquivos fonte não são possíveis. Se você tiver mesmo de compilar sua aplicação, faça-o como usuário, na sua pasta /home, sem instalar no sistema. Também não há suporte para o uso do 'checkinstall' e conversões de RPMs para .deb com o 'alien' (ou outro programa desse tipo).

Variantes do Debian muitas vezes colocam os arquivos das aplicações em locais diferentes, o que pode causar instabilidades no sistema. Alguns pacotes, inclusive, nem conseguem ser instalados, devido a dependências não resolvidas causadas pelo uso de nomes ou números de versões diferentes para os pacotes.Por exemplo, uma versão diferente da 'glibc' pode fazer com que a aplicação nem seja executada.

Atualizando Todo o Sistema - Dist-upgrade

apt-get dist-upgrade é a maneira padrão de atualizar o sidux. Usar qualquer 'front-end' (interface gráfica) para atualizar o sidux não é possível. Atualizando todo o sistema - dist-upgrade

Um 'dist-upgrade' só é possível fora do ambiente X. Para parar o X, basta digitar "init 3" (sem as aspas) no terminal do KDE, XFCE ou qualquer que seja o gerenciador de janelas que você esteja usando. Você pode também usar um terminal virtual (ctrl+alt+f1, ctrl+alt+f2, etc).

smxi

O smxi passou a ser incluído nos últimos LiveCDs do sidux. Esta é uma grande alternativa para rodar um 'dist-upgrade' confiável. Ele é monitorado por membros do sidux para evitar atualizações de pacotes que possam quebrar o sistema e inclui soluções para problemas maiores, caso eles surjam. Dê uma olhada no fórum do sidux, se estiver interessado em conhecer todas suas funcionalidades.

O smxi, além de assistir os usuários nas atualizações, também permite uma rápida e fácil instalação de 3d video drivers y novos kernels. Ele ainda oferece um certo grau de automatização na instalação do virtualbox-os e do qemu.

Comandos do smxi

sgfxi - Instalador de drivers de placas de vídeo (instala os drivers de código aberto e da nVidia e ATI)

Esta ferramenta automatiza a instalação dos drivers nvidia e fglrx, necessários para aproveitar todos os recursos de sua placa de vídeo.

NVIDIA - O sgfxi é altamente recomendado para os pacotes nvidia do Debian. Normalmente ele é atualizado no mesmo dia em que um novo driver é adicionado aos repositórios. O sgfxi modifica automaticamente o arquivo xorg.conf, para que você tenha um 3D rodando com perfeição.

ATI - O sgfxi funciona se os pacotes fglrx no Debian sid funcionarem com a versão do X instalada e se combinarem com o kernel atual. Devido à natureza aleatória da compatibilidade e do esquema de nomes de cada nova versão do fglrx, o trabalho da ATI é, no mínimo, bem abaixo das expectativas. Verifique o fórum e/ou o IRC do sidux.

Você pode reverter para os drivers de código aberto usando o sgfxi, com os comandos -n <driver> ou -N <driver>.

Configuração da rede

O Ceni é uma ferramenta que permite a configuração rápida de sua rede ou placa wireless, sem nenhuma complicação. A opção 'wireless' procura pelas redes disponíveis, usa encriptação WEP e WPA e usa os pacotes wireless-tools ou wpa_supplicant para configuração. Já a conexão com fios (Ethernet) é automática se você usar dhcp. Se não for o caso, você pode configurá-la manualmente, desde as máscaras de rede aos nomes de servidores.

Execute o Ceni com o comando Ceni ou ceni . Se ele não estiver instalado, faça-o com o comando 'apt-get install ceni'.

Outros gerenciadores de janelas

O KDE e o Fluxbox são os padrões no sidux. O sidux não dá suporte ao Gnome porque seus pacotes no Debian sid são, sabidamente, muito instáveis e, no estágio atual, podem causar o colapso de todo o sistema. Alguns usuários nos fórums, /wiki e IRC do sidux podem, quem sabe, ajudá-lo. Do contrário, você terá de se arranjar sozinho. Já o caso do XFCE é diferente: apesar de também não haver suporte para ele, é bem provável que você encontre quem possa ajudá-lo nos canais citados, porque o XFCE é bastante popular entre usuários com computadores antigos.
