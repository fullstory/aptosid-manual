Definitions of files on each mirror

Each sidux mirror has following files:

MD5SUM 
MD5SUM.asc  
SHA256SUM   
SHA256SUM.asc
SOURCES
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

The .manifest file lists all the packages contained in the specific iso. The .iso is the image to download. The .md5 and .sha256 files are to check the integrity of the iso.

The .asc files are signature files, so you can verify that the checksums (.md5 .sha256) have not been changed and that they can be used to verify the integrity of the iso.

Mulţumiri tuturor celor ce găzduiesc locaţiile de descărcare pentru efortul lor legat de sidux.

Verificarea cu suma md5sum şi validarea

FOARTE IMPORTANT: sidux , ca LIVE-CD, este bazat pe tehnologie de înaltă compresie , şi din această cauză este nevoie de grijă deosebită când se scrie o imagine ISO . Folosiţi doar CD-uri sau DVD-uri de bună calitate pe care să le scrieţi doar în DAO-mode (disk-at-once) şi nu cu viteză mai mare de 8X.

Aceasta este o sumă de control a fişierelor şi este folosită pentru a verifica integritatea lor după descărcare. Suma de control curentă este comparată cu o sumă deja ştiută . În acest fel se verifică dacă fişierul a fost schimbat sau deteriorat ; procedeu folosit în special pentru fişierele descărcate de pe internet .

Dacă suma de control a fişierului descărcat corespunde cu cea din fişierul md5sum puteţi fi siguri că fişierul a fost descărcat corect . În Linux obţineţi suma de control cu comanda ( va dura puţin până este calculată ) : 

$ md5sum file_to_check

Rezultatul va fi scris în consolă .
Cu programul `md5summer` (486 kb) suma de control md5sum poate fi verificată şi în Windows .

Imaginile tip ISO de sidux au întotdeauna ataşat un fisier md5sum cu ajutorul căruia puteţi verifica integritatea imaginii înainte de a o scrie pe disc. Se garantează astfel , în cazul în care apar probleme , că ele nu se datorează descărcării defectuoase a fişierelor şi astfel forumul se ţine liber de probleme care nu pot fi rezolvate din cauză că fişierele ISO au fost deteriorate la descărcare .Verificare se face în consolă . Schimbaţi în directorul unde se află fişierul ISO şi MD5 şi lansaţi comanda . ex. :

Download the following files from the mirror:

MD5SUM 
MD5SUM.asc  
SHA256SUM   
SHA256SUM.asc

In a terminal:

$ md5sum sidux.iso

Dacă suma de control şi fişierul md5sum nu au aceaşi valoare veţi primi mesajul :

"sidux.iso: Error
md5sum: Warning: calculated checksum does not match!"

Dacă fişierul descărcat este corect programul se va termina fără nici un mesaj .

If you wish to be able to verify the validity of the .asc files do the following as $USER:
Example using:
sidux-2009-02-aether-kde-lite-i386-200907141544.iso

1) Import "Stefan Lippers-Hollmann's" (slh's) public key from a key-server:

$ gpg --keyserver wwwkeys.eu.pgp.net --recv-keys F80994F6
gpg --edit-key F80994F6
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Now you can verify the integrity of the checksum with:

$ gpg --verify MD5SUM.asc

If you have classified the key as trustworthy using the necessary gpg commands, then this is the output you should get with "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

You will get 'Good signature' with a WARNING if you did not choose to classify the key:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID F80994F6
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model

Scrierea CD-ului în Windows ?

FOARTE IMPORTANT: sidux , ca LIVE-CD, este bazat pe tehnologie de înaltă compresie , şi din această cauză este nevoie de grijă deosebită când se scrie o imagine ISO . Folosiţi doar CD-uri sau DVD-uri de bună calitate pe care să le scrieţi doar în DAO-mode (disk-at-once) şi nu cu viteză mai mare de 8X.

Bineînţeles CD -ul poate fi scris şi în Windows . Fisierul descărcat trebuie scris ca un fişier ISO. Dacă Winrar ( sau un alt program de arhivare ) este asociat cu fişierele .ISO el va fi arătat ca o arhivă . Din acest fişier ISO trebuie scris CD -ul .

Există câteva opţiuni pentru a scrie discul în Windows.

Programe Open Source pentru Windows

cdrtfe :compatibil cu Windows 9x/ME/2000/XP ( testat în Win95, Win98SE, Win2000, WinXP) doar în Win9x/ME: lucrează ASPI Layer (ex. Adaptec ASPI 4.60)

Programe de scriere cu licentă în Windows
  • Nero ( însă NU nero express!)
  • O altă unealtă bună care este freeware CD/DVD Burner XP pro
  • Burncdcc de la terabyteunlimited care scrie doar fişiere ISO.

Scrierea CD -ului în Linux

FOARTE IMPORTANT: sidux , ca LIVE-CD, este bazat pe tehnologie de înaltă compresie , şi din această cauză este nevoie de grijă deosebită când se scrie o imagine ISO . Folosiţi doar CD-uri sau DVD-uri de bună calitate pe care să le scrieţi doar în DAO-mode (disk-at-once) şi nu cu viteză mai mare de 8X.

Dacă deja folosiţi linux pe computer puteţi scrie CD-ul fie folosind consola fie un program de scriere deja instalat .

În sidux sau alte distribuţii de Linux K3b este unealta implicită de scriere CD-uri. Deschideţi-l , selectaţi "Extras" -> "burn CD-Image..." şi alegeţi fişierul ISO de scris ( ex. sidux.iso ).

K3B iniţial calculează suma de control MD5 pentru fişierul ISO ( va dura un moment ). Dacă suma găsită este similară cu suma din fişierul MD5 ( ex. sidux.iso.md5 ) , descărcarea a fost terminată cu succes şi puteţi scrie fişierul apăsând "Start".

Problems with burning mostly happen with a frontend. You can burn directly from the shell with burniso.
