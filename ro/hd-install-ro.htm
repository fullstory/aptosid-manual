Pregătirea pentru instalarea pe Hard Disc

Pentru uzul curent recomandăm tipul de fişiere ext3, este sistemul implicit în sidux şi este foarte bine întreţinut.

Înaintea instalării vă rugam deconectaţi toate memoriile USB , camerele , etc . Instalarea pe dispozitive USB necesită paşi suplimentari. Pin editarea fişierul de instalare : ~/.sidconf se pot schimba opțiunile de instalare inplicite, de exemplu folosirea unui sistem de fişiere diferit și/sau instalarea sistemul pe mai multe partiţii. De exemplu creerea unei partiţii separate pentru directorul/repertoarul home/.

Este foarte recomandat să aveţi o partiţie separată pentru date . În termeni de stabilitate și de recuperare a datelor, ca urmare a unui dezastru, beneficiile sunt incomesurabile.

De aceea directorul $HOME devine locul unde configurările aplicaţiilor de bază sunt ţinute , sau, să spunem altfel, este un container unde aplicaţiile păstrează configurațiile/setările personale și este locul unde sunt păstrate toate datele personale.

Reinstalarea de aplicații utilizabile pe un alt calculator

Pentru a crea o listă cu aplicaţiile deja instalate, în scopul de a le 'pune' pe o alta mașină, sau în anumite cazuri chiar pe propria mașină, rulați comanda urmatoare într-un terminal (sau in consolă): 

dpkg -l|awk '/^ii/{ print $2 }'|grep -v -e ^lib -e -dev -e $(uname -r) >/home/username/installed.txt

Apoi copiați fișierul text rezultat pe o cheie usb, sau pe orice alt periferic mobil (cdrom ) .

Copiați acest fișier în repertoriul $HOME pe a doua mașină, și folosiți această listă ca referință pentru instalarea aplicaţiilor necesare.

RAM și Swap

Pe calculatoarele cu mai puţin de 512 mb ram trebuie să aveţi o partiţie swap . Mărimea acesteia nu trebuie să fie mai mică de 128 mb (informaţiile date de cfdisk NU ar trebui crezute atâta vreme cât el calculeaza în baza 10 ) dar nici o partiţie swap mai mare de 1G nu are rost .

Vă rugăm consultati : Partiţionarea Hard Discului

ÎNTOTDEAUNA FACEŢI SALVĂRI DE SIGURANŢĂ A DATELOR ! including your bookmarks and emails! See Back-Up with rdiff and Back-Up with rsync. Another option is sbackup (needs installing).

Instalarea pe Hard Disc este mai confortabilă iar sistemul mai rapid decât atunci când rulează de pe CD.

Setați (în BIOS) CD-ROM-ul ca prim dispozitiv de boot. La majoritatea computerelor puteţi intra în BIOS apăsând tasta [del] în timpul procesului de boot-are (în unele versiuni de BIOS se poate pur şi simplu să alegeți dispozitivul de pe care să se încarce sistemul la pornire, de ex. la un BIOS tip AMI, folosim tasta F11 sau F8).

În cele mai multe cazuri acum sidux trebuie să pornească. În caz contrar, puteţi folosi opțiunile de boot, (numite cheatcodes, sau parametri de boot) care pot fi scrise direct în linia de comandă a managerului de boot. Folosirea parametrilor de boot (ex. pentru rezoluţia ecranului sau limba selectată) poate salva un timp prețios în faza de configurare de după instalare . A se vedea Cheatcodes sau Resoluții VGA (Un exemplu util este folosirea tastei F4, la pornirea de pe Live CD, pentru a alege limba viitorului sistem).

Choosing the language for your installation

Language Installs with KDE-full

Select your main language from the grub menu (F4) in the kde-full release, to install the localisations for the desktop and many applications while booting.

This ensures they are also present after installing sidux, while only installing the required languages for the given system. The amount of memory required for this feature depends on the language and sidux may refuse to install the given language packs automatically with insufficient RAM and the boot sequence will be continued in English language but with the desired locales settings (currency, date and time format, keyboard charsets). 512 MB memory or more should be safe for all supported languages, which are:

Default - Deutsch (German)
Default - English (US)
*Dansk (Danish)
*Greek (Greek)
*Español (Spanish)
*English (GB)
*Français (French)
*Hrvatski (Croatian)
*Italiano (Italian)
*Nihongo (Japanese)
*Nederlands (Dutch)
*Polski (Polish)
*Português (Portuguese BR and PT)
*Română (Romanian)
*Rusian (Russian)

The language selection depends on the availability of sidux-manual translations, get involved to add your language.

Other Language installs with KDE-lite
  1. Select your main language from the grub menu (F4). (See also sidux specific Live-CD Cheatcodes). The Language files themselves are not on the Live-CD so the system will fall back to default English. However, this will make the correct language configuration needed for your preferred language and therefore no need to make any changes into the system, aside from the installation of the missing language files.
  2. Start the installation.
  3. Install to HD and reboot.
  4. After HD install, install the language of your choice and applications via the install-meta. If the packages are not available via the meta-installer, install manually via apt-get. If you want to, you can remove any unwanted languages. 
    apt-get install install-meta
    and select metapackage-installer from the Kmenu > Utilities.

Another option is to apt-get install install-meta whilst on the live CD and prepare your required language files, before initiating the HD instalation. This will require at least 512mb of RAM

First Time boot up to the HD

After booting up for the first time you will discover that sidux has forgotten its network configuration. The network can be comfortably set up from Kmenu > Internet > Ceni. For additional WIFI/WLAN roaming please read this.

Programul de instalare sidux

1. Programul de instalare este pornit din icoana Desktop, KMenu> System>sidux-installer, sau de la consolă folosind :

sux
install-gui.bash

2. După ce aţi citit (şi înţeles ) mesajul de avertizare , trecem la alegerea partiţiei .

Now choose where the installation is supposed to go to and we establish the mount points. Partitions for which we do not establish mount points for are mounted by the installer (the swap partition will be automatically mounted, when the system starts).

All other partitions will be placed as a /media/ partition. Here you have to select the root partition ("/") for your sidux installation. However, it is at this moment you can also choose to create a data partition. A 'left-click' will activate your choices for each partition.

Have you backed up your data?

If you haven't partitioned your hard drive yet, do it in the Start Part.-manager and have a look at Partitioning your HD using Gparted or, if you want to use the shell, read Partitioning your HD

You may also start them from the konsole/terminal

sux
gparted

sau

su
cfdisk

3. sidux folosește Grub ca manager de boot, deci vom instala Grub în MBR! Dacă aici veţi alege altceva trebuie să ştiţi foarte bine ce faceţi.

Grub recunoaşte şi alte sisteme de operare precum (ex. Windows) şi le adaugă la meniul de boot.

Mai departe veţi putea să modificaţi Grub în această fereastră .

4. Mai departe vom alege numele utilizatorului , parola de utilizator şi parola de root (va rugăm să le ţineţi minte !). To add additional users, do so after installation via the terminal with adduser..

5.Acum alegeţi numele pe care să-l aibă calculatorul după instalare (puteţi să-l numiţi cum vreţi - 'Hostname : acesta poate conţine doar litere şi cifre ; dar nu trebuie să înceapă cu un numar '.

După aceasta puteţi alege dacă cupsys sau ssh să pornească automat sau nu .

6. Această atenționate aste ultima ocazie de a corija parametrii de instalare. Reverificați acești parametrii cu mare atenție, după care clic pe Save Config și Next ...

În aceasta etapă a instalării se poate schimba/edita fișierul de configurație, dupa care instalarea poate folosi acest (nou) fișier. Instalatorul nu face nici o verificare, deci orice clic pe back in instalator duce la pierderea fișierul de configurație, așa că nu dați clic pe back în instalator dacă nu doriți să pierdeți fișierul de configurație editat manual .

Editarea fișierului (~/.sidconf) este recomandată utizatorilor experimentați in linux, care doresc să adauge schimbări specifice la fișierul de configurație, sau au o schemă de partiționare specială care poate fi ignorată de verificarea automată a instalerului.

Should the installer detect that you are trying to install over an old $HOME it will warn you and will not let you proceed with the installation until you choose another user name.

Pentru a începe instalarea clic pe Begin Installation. Întregul proces de instalare pe mașini noi este între 5-15 minute, iar pe vechi mașini 60 minute.

Daca procesul de instalare 'pare' oprit, nu disperati, lăsați ceva timpul suplimentar să curgă ...

Sfărșit ! Reboot-ți , scoateți cdrom-lui, si boot-ați în noul sistem instalat pe disc.

La prima pornire a sistemului veţi descoperi că sidux a pierdut configurarea reţelei .Va trebui să faceţi o nouă configurare ( Wlan, Modem, ISDN,...).

Dacă prima dată reţeaua a fost detectată automat (DHCP) chiar folosind un DSL-Router, acum va trebui să o reactivaţi prin comanda: 

ceni

Uneltele de configurare pot fi găsite şi în : Kmenu>sidux>Internet>ceni. Deasemeni vedeţi : Internet şi rețele

To add users to your installation

To add a new user with automatic group permissions granted, as root: 

adduser <newuser> (just press enter, it should take care of the complexities)
You will get asked to type in the password twice

sidux specific icons (like the manual and IRC icons) need adding manually. To add them you will need to right click the application in the Kmenu to add it to the desktop.

kuser can create new user as well, however you will need to manually adjust the group permissions for that user.

To delete a user

deluser <user>

man deluser

Despre sux

Multe comenzi trebuie executate ca super utilizator sau root. Comanda care face această trecere este: 

sux

Cu toate că su este comanda obișnuită pentru a deveni super utilizator, folosirea comenzi sux vă va permite să rulați aplicații de tip GUI / X11 în linia de comadă, și permite super utilizatorului să pornească aplicații grafice în numele utilizatorului care o lansează, întru-cât sux este un wrapper al comenzii standard su, cate permite transferul credențialelor X către utilizatorul care o lansează.

Exemple de utilizare de aplicații X11, prin sux, sunt lansarea de editoare precum kwrite sau kate, partiționarea cu gparted sau utilizarea unui manager de fișiere ca konqueror. Se poate deasemenea fi folosit pentru modificatea de fișiere, cu un clic dreapta, alegerea 'edit-as-root' cu promt de parolă root, care în background apelează transparent la comanda kdesu .

Toate acestea spre deosebire de 'sudo', care permite la oricine care are acces la clavier să scrie 'sudo' și să facă schimbări în sistem foarte probabil nedorite sau nefaste.

ATENȚIE: O dată logat ca utilizator root, sistemul nu vă va înpiedica să faceți lucruri care pot pericrita sitemul, ca de exemplu ștergerea de fișiere importante, etc ..., deci trebuie să fiți absolut sigur de ceeea ce vreți să faceți, întru-cât riscul de a vă ruina sistemul este maxim.

În nici un caz, nu trebuie să executați ca root în consolă/terminal aplicații tipice pentru un utilizator normal, ca de exemplu trimeterea de email-uri, crearea de spreadsheets sau navigarea pe net si așa mai departe.
