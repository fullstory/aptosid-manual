Trucuri pentru dispozitive ce necesită drivere non-free

Din cauza complexităţii legii sidux live-cd va oferi doar software dfsg-free .

Sources List

Din cauză că sidux conţine doar software dfsg free va trebui să adăugaţi contrib/ non-free la /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list sau /etc/apt/sources.list şi să-i asiguraţi accesul la internet:

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Drivere de sursa inchisa, binare pentru: nVidia cu dmakms

Pregateste modulul:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Apoi activeaza Dynamic Module-Assistant Kernel Module Support (dmakms) pentru nvidia, asfel incat urmatoarea data cand kernel-ul Linux este actualizat un modul nvidia este pregatit deasemeni, fara interventie manuala. Ca sa efectuati aceasta, adaogati nvidia-kernel-source la /etc/default/dmakms dosarul configiratie.

Apoi:

m-a a-i nvidia-kernel-source

Urmat de:

apt-get install nvidia-glx

Apoi schimbati xorg.conf:

sed -i s/nv\"/nvidia\"/ /etc/X11/xorg.conf

Reboot PC pentru ca instalarea modulului sa fie efectuata.

Citeste:

$ /usr/share/doc/dmakms

Drivere de sursa deschisa Xorg pentru nVidia, ATI, Radeon, Intel

Drivere de sursa deschisa Xorg pentru nVidia, ATI, Radeon, Intel sunt preinstalate cu sidux.

Driverele X.Org pentru serverele X (vedeţi xserver-xorg pentru descriere) oferă suport 2d pentru placile NVIDIA Riva, TNT, GeForce, şi Quadro , pentru plăcile ATI Mach, Rage, Radeon, şi FireGL plus pentru sub-driverele 'atimisc', 'r128' şi 'radeon'. Pentru seriile Intel free se oferă suport în 3d.

Pentru instalare :

Faceţi Log out din KDE intrând în Textmode şi apăsând Ctrl+Alt+F1
logare ca root 
şi apoi tastaţi: 
init 3
( aceasta ar trebui să ateste că serverul X nu mai funcţionează)
apt-get update
apt-get install xserver-xorg-video-nv
sau
apt-get install xserver-xorg-video-ati
sau
xserver-xorg-video-intel
init 5 && exit

Pentru anumite card-uri s-ar putea adaoga suport 3D,dar de obicei este suficient 2D.

Daca ati folosit drivere de sursa inchisa si doriti sa reveniti la drivere de sursa deschisa va trebui sa editati /etc/X11/xorg.conf,DAR MAI INTAI TREBUIE SA FACETI O COPIE "BACK UP", deoarece editand /etc/X11/xorg.conf incorect va rezulta in multe lacrimi varsate de dvs.

Pentru a edita /etc/xorg.conf faceţi clic dreapta pe el şi-l deschideţi cu permisiune root . Găsiţi secţiunea care citeşte SECTION DEVICE şi schimbaţi driverul de display pentru a fi nv sau ati. intel.

Editing xorg.conf is entirely at your own risk!!

Drivere Firmware

$ fw-detect

Scrierea pe partitii NTFS cu ntfs-3g

Atenţie: Chiar dacă ntfs-3g este considerat ca 'stable', niciodată nu-l folosiţi fără salvarea prealabilă a datelor şi, bineînteles , nu pe sisteme productive! Dacă nu ţineţi cont de aceasta vina pentru pierderea de date vă aparţine, folosiţi pe risc propriu !

Deschideţi o consolă şi introduceţi următoarele comenzi: Vedeţi Partiţionarea HardDiscurilor - Numirea Discurilor

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Pentru a ieși din poziția de root tastați:
exit

Acum partiţia NTFS ar trebui să fie montată rw şi să puteţi scrie date pe ea. Dar încă o dată, aveţi grijă ! Utilizaţi doar în situaţii de urgentă, nu este recomandat pentru utilizarea zilnică.
