Trucuri pentru dispozitive ce necesită drivere non-free

A sidux live-ISO will only provide dfsg-free software.

Sources List

Din cauză că sidux conţine doar software dfsg free va trebui să adăugaţi contrib/ non-free la /etc/apt/sources.list.d/debian.list and /etc/apt/sources.list.d/sidux.list sau /etc/apt/sources.list şi să-i asiguraţi accesul la internet:

# Debian
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free

Drivere de sursa deschisa Xorg pentru nVidia, ATI, Radeon, Intel

Drivere de sursa deschisa Xorg pentru nVidia, ATI, Radeon, Intel sunt preinstalate cu sidux.

Intel drivers should work perfectly for 2D and 3D video acceleration as the drivers are included in the Intel free series.

Note: If xorg.conf does not exist, and you require it to be present for some reason, in a terminal run:

X -configure

This command writes an xorg.conf and places it in /root directory and names it xorg.conf.new and contains the essential information. Next step is move and rename it:

mv /root/xorg.conf.new /etc/X11/xorg.conf

Pentru a edita /etc/xorg.conf faceţi clic dreapta pe el şi-l deschideţi cu permisiune root . Găsiţi secţiunea care citeşte SECTION DEVICE şi schimbaţi driverul de display pentru a fi nv sau radeon. intel (to name but a few).

Editing xorg.conf is entirely at your own risk!!

Mai multe informaţii despre Intel   ati radeon   radeonhd   nv  X.Org

Drivere de sursa inchisa, binare pentru: nVidia cu dmakms

New installs will need to ensure that xorg.conf exists in /etc/X11. If it does not exist, create a file called xorg.conf.dmakms and copy into it the following:

# /etc/X11/xorg.conf (Xorg X Window System server configuration file)
# Edit this file with caution, and see the xorg.conf(5) manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
Section "Device"
         Identifier      "Device 0"
         Driver          "nvidia"
	 BusID		 "PCI:1:0:0"
 EndSection

Save and move the file to /etc/X11:

mv xorg.conf.dmakms /etc/X11/xorg.conf

NOTE: Use apt-cache search nvidia and apt-cache show <package> to ascertain the correct driver for you. There are basically 2 choices, current Debian Sid 3D drivers and legacy Debian Sid 3D drivers.

For current 3d nvidia drivers ≥ GeForce 6xxx :

Pregateste modulul:

apt-get install nvidia-kernel-source nvidia-kernel-common dmakms

Apoi activeaza Dynamic Module-Assistant Kernel Module Support (dmakms) pentru nvidia, asfel incat urmatoarea data cand kernel-ul Linux este actualizat un modul nvidia este pregatit deasemeni, fara interventie manuala. Ca sa efectuati aceasta, adaogati nvidia-kernel-source la /etc/default/dmakms dosarul configiratie:

echo nvidia-kernel-source >> /etc/default/dmakms

Apoi:

m-a a-i nvidia-kernel-source

Urmat de:

apt-get install nvidia-glx

Apoi schimbati xorg.conf:

sed -i -e s/nv\"/nvidia\"/ -e s/vesa\"/nvidia\"/ /etc/X11/xorg.conf

Reboot PC pentru ca instalarea modulului sa fie efectuata.

When xorg updates you only need to reinstall nvidia-glx:

apt-get install --reinstall nvidia-glx

When a new Debian sid nvidia driver becomes available thru apt:

m-a a-i nvidia-kernel-source
apt-get install --reinstall nvidia-glx

Reboot PC for installation of the module to take effect.

For legacy 3d nvidia drivers ≥ GeForce 5xxx :
m-a a-i nvidia-kernel-legacy-173xx-source && apt-get install nvidia-glx-legacy-173xx dmakms

Alter dmakms:

echo nvidia-kernel-legacy-173xx-source >> /etc/default/dmakms

Module load failure

Should nvidia fail to load, for whatever reason:

modprobe nvidia

Then reboot the computer.

Should the module still fail to load:

 m-a a-i -f nvidia-kernel-source

This rebuilds the module then reboot.

Citeste:

$ /usr/share/doc/dmakms

You can also read nvnews for other options.

ATI, Radeon and other video drivers

2D Drivers

Driverele X.Org pentru serverele X (vedeţi xserver-xorg pentru descriere) oferă suport 2d pentru placile NVIDIA Riva, TNT, GeForce, şi Quadro , pentru plăcile ATI Mach, Rage, Radeon, şi FireGL plus pentru sub-driverele atimis', r128 şi radeon. Radeon and Intel both support 2d acceleration (textured xv) for video playback.

3D Drivers

Some ATI cards also support 3D, so far up to the r5xx chipsets.

To automatically inherit newly packaged non-free firmware when they get updated for 2D and 3D video cards:

apt-get install firmware-linux

Then reboot the computer.

Drivere Firmware

Firmware-ul necesar poate fi determinat cu urmatorul dispozitiv/ firmware enumerare sau folosind recentul text dezvoltat fw-detect (gasit in sidux-scripts).

$ fw-detect

The output of fw-detect describes the commands needed to install and activate the firmware:

Example:

#Detected driver that requires firmware to operate
#Follow these instructions to obtain the correct firmware
# and activate the zd1211rw driver:
apt-get update
apt-get install zd1211-firmware
modprobe -r zd1211rw
modprobe zd1211rw

To install firmware from git repos, you need to:

apt-get install git-core

Should you need to prefetch firmware .debs, to put on a usb-key for example, you can download them as either a zip or tar.gz file from http://cdimage.debian.org/cdimage/unofficial/non-free/firmware/sid/current/

Vom verifica dacă putem oferi măcar unele din pachetele de instalare pentru aceste dispozitive, dar demersurile legale nu sunt întotdeauna uşoare.

Scrierea pe partitii NTFS cu ntfs-3g

Atenţie: Chiar dacă ntfs-3g este considerat ca 'stable', niciodată nu-l folosiţi fără salvarea prealabilă a datelor şi, bineînteles , nu pe sisteme productive! Dacă nu ţineţi cont de aceasta vina pentru pierderea de date vă aparţine, folosiţi pe risc propriu !

Deschideţi o consolă şi introduceţi următoarele comenzi: Vedeţi Partiţionarea HardDiscurilor - Numirea Discurilor

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Pentru a ieși din poziția de root tastați:
exit

Acum partiţia NTFS ar trebui să fie montată rw şi să puteţi scrie date pe ea. Dar încă o dată, aveţi grijă ! Utilizaţi doar în situaţii de urgentă, nu este recomandat pentru utilizarea zilnică.
