Activarea Ndiswrapper

sidux are toate pachetele deja instalate : modulul de kernel şi utilitaţile ndiswrapper .

Aveţi nevoie de un "Windows" inf-file pentru driver , găsiţi-l şi încărcaţi-l :

ndiswrapper -i driver.inf

Pentru a verifica dacă instalarea s-a efectuat cu succes , verificaţi cu -l:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Este important să vedeţi "driver installed" şi "device present" în mesajul de după -l.Dacă nu vedeţi aceste două înseamnă că ceva a mers prost (verificaţi mesajele de eroare cu -i şi/sau dmesg), pentru că dacă aveţi un mesaj de eroare aici nu veţi avea conexiune wlan .

încărcaţi modulul ndiswrapper :

modprobe ndiswrapper

şi verificaţi dacă s-a încărcat corect :

iwconfig

verificaţi dacă a fost găsit punctul de acces :

iwlist wlan0 scan

apoi puteţi configura placa de reţea cu :

/usr/sbin/netcardconfig

Dacă doriţi update de kernel , va trebui să-l reinstalaţi . Schimbaţi numărul de kernel pentru a se potrivi kernelului curent :

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Apoi încărcaţi-l cu modprobe:

modprobe ndiswrapper

Nu va trebui să reinstalaţi şi driverul de windows deoarece partea cu driverul de windows este separată de kernel module.

Puteţi deasemeni scrie ndiswrapper în /etc/modules pentru încărcarea ndiswrapper la startup

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

Ndiswrapper - Avansat

ndiswrapper este un modul care-i permite sistemului de operare Linux să folosească drivere WLAN din Windows . Acesta este necesar deoarece unii producători de plăci wlan nu oferă şi un driver free sau open source ceea ce face utilizarea acelei plăci în Linux dificilă. Ndiswrapper rezolvă această problemă "wrapping" driverul Windows şi traducând intrarile şi ieşirile din placă pentru Linux.

Identificarea driverului necesar

Mai întâi aveţi nevoie de un driver de Windows potrivit pentru placa dumneavoastră WLAN . Plăcile WLAN au multe şi diferite chip sets chiar la acelaşi model şi singura cale de a identifica corect placa este după device id (USBID/PCIID).

Pentru a obţine acest 'id' deschideţi un terminal :

Pentru un adaptor USB WLAN 
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

Cum vedeţi în exemplul de mai sus , USBID pentru adaptorul D-Link Corp. [hex] DWL-122 este 2001:3700.

Pentru un adaptor (mini) PCI / PCMCIA WLAN 
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

Acest 'id' are schema vendor:device deasemeni , aici 168c:0024 pentru placa Atheros .

Acum căutaţi la : ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list o intrare cu acel 'id' , iar dacă sunteţi norocoşi veţi găsi o intrare corespondentă şi o recomandare de driver.

Aveţi nevoie de fişierele INF şi SYS pentru acel driver. Cel mai bine veţi găsi o arhivă ZIP pe care s-o despachetaţi cu unzip. Dacă găsiţi un fişier EXE este o arhivă auto-extractivă (folosiţi wine pentru extracţie) sau un CAB (extrageţi cu cabextract). Cea mai uşoară cale este să stocaţi fişierele într-un director nou în home.

Activarea Ndiswrapper şi instalarea driverului

Schimbaţi în directorul cu fişierele driverului de windows :

bla@blubb:~$cd /folder/with/driver
bla@blubb:~$su
root@blubb:/folder/with/driver# ndiswrapper -i <driver.inf>
installing net5416 ...
root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Este foarte important să aveţi "driver installed" şi "device present" în ieşirea de după -l. Dacă nu aveţi aceste două înseamnă că ceva a mers prost (verificaţi mesajele de eroare de la -i şi/sau dmesg), pentru că dacă primiţi un mesaj greşit aici înseamnă că pasul nu a reuşit .

Acum încărcaţi kernel module: 

root@blubb:/some/folder# modprobe ndiswrapper

Dacă nu primiţi nici un mesaj aici înseamnă că totul e în regula , dacă primiţi ceva verificaţi dmesg pentru informaţii suplimentare .

"iwconfig" ar trebui acum să vă arate un dispozitiv wireless :

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

Dacă doriţi ca ndiswrapper să se încarce la start up trebuie să scrieţi ndiswrapper în /etc/modules, fie manual , fie folosind : 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
Reinstalarea Ndiswrapper după kernel update

După kernel update va trebui să reinstalaţi ndiswrapper kernel module. Fişierul deb este localizat la /usr/src/kernel-downloads/ <kernel-version >/ndiswrapper-modules- <version >.deb,. Deci, de exemplu , pentru kernelul 2.6.21.5-slh-smp-2, va trebui să faceţi :

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

După instalare încărcaţi-l cu :

root@blubb:/some/folder# modprobe ndiswrapper

Nu va trebui să reinstalaţi driverul de windows deoarece acesta este separat de kernel module.

Informaţii Generale

Pentru configurarea noului dispozitiv folosiţi netcardconfig sau sidux-networkconfig. Un how-to detaliat poate fi găsit aici .

Dacă găsiţi un alt/mai nou driver vă rugăm testaţi-l şi pe acela. Acesta poate cauza probleme , un kernel panic în cel mai rău caz , dar ajutaţi astfel proiectul ndiswrapper şi alţi utilizatori să beneficieze de cel mai bun driver pentru diferite plăci . Nu uitaţi deasemeni să completaţi/updataţi lista ndiswrapper după un test cu succes.
