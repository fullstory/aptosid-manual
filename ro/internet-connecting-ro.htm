Conectarea Online / Configurarea Retelei

Daca exista in reteaua dumneavoastra un server DHCP la care este conectat calculatorul in timpul pornirii cu Live CD setarile de retea se vor face automat - in caz contrar va trebui sa porniti dialogul de configurare a retelei in sidux: Apasati Kmenu>sidux>Network/Internet>Networkcard configuration.

Sau - daca nu va place modul grafic , deschideti o consola , tastati : 

sudo
netcardconfig

si urmati instructiunile.

Activarea Ndiswrapper

sidux are toate pachetele deja instalate : modulul de kernel si utilitatile ndiswrapper .

Aveti nevoie de un "Windows" inf-file pentru driver , gasiti-l si incarcati-l :

ndiswrapper -i driver.inf

incarcati modulul ndiswrapper :

modprobe ndiswrapper

si verificati daca s-a incarcat corect :

iwconfig
iwconfig wlan0 essid 'ESSID'

verificati daca a fost gasit punctul de acces :

iwlist wlan0 scan

apoi puteti configura placa de retea cu :

netcardconfig

Computerele cu doua placi de retea ( pentru Internet sharing )

Configurarea pentru conectarea la internet cu un laptop (sau alt pc) printr-o solutie simpla :

Deschideti un terminal si schimbati pe utilizatorul root cu "sux" pentru a putea executa urmatorii pasi .

Configurati prima placa de retea pentru accesul la internet :

netcardconfig

Pentru a doua placa de retea vom alege o adresa IP fixa intr-un anumit domeniu , ex. :

192.168.1.1

Si deasemeni o vom configura cu :

netcardconfig

Tineti minte : Adresa IP data trebuie sa fie in acelasi subnet ca si adresa IP ce este atribuita automat prin DHCP .

Daca din greseala s-a specificat o valoare pentru Gateway in timpul configurarii interfetelor de retea , va trebui sa rulati scriptul :

remove-gateway.sh

Setarea DHCP/DNS :

apt-get install dnsmasq

Va trebui sa editati fisierul /etc/dnsmasq.conf adaugand urmatoarea linie :

dhcp-range=192.168.1.50,192.168.1.150

Restul ramane asa cum este .

Acum serverul DHCP de pe computerul gazda poate fi pornit cu : 

/etc/init.d/dnsmasq start
.

Daca computerul client este setat DHCP , o adresa IP va fi atribuita automat .

Pentru a accesa internetul , de pe computerele client , mai trebuie rulat scriptul : 

/usr/sbin/masquerade.sh

Dupa aceasta orice computer client ar trebui sa se poata conecta la internet .

Pentru a avea aceasta configuratie incarcata automat la pornire pe computerul gazda : activati dnsmasq in procesul de boot cu :

update-rc.d -f dnsmasq defaults

Rulati masquerade-script cu :

echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Conectarea printr-un 56k Dial-up Modem

KDE contine o unealta pentru conectarea prin modem numita KPPP Internet Dial-up Tool , care se gaseste in meniul principal --> Internet .

Aplicatia are atasat un manual intern ce ofera un ghid foarte intuitiv pentru setarea modemului pentru accesul online .

Mini-Firewall

Daca aveti nevoie de un firewall script incercati urmatoarele (necesar doar pentru o conectare directa la internet , nu e necesar daca rulati din spatele unui router ):

wget http://rebelhomicide.demon.nl/scripts/mini-miyagi.sh
chmod 755 mini-miyagi.sh 
apt-get remove --purge {firestarter,guarddog}

Rulati :

./mini-miyagi.sh

Pentru a opri mini-miyagi :

/etc/init.d/mini_miyagi_fw.sh stop
