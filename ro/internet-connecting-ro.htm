Conectarea Online / Configurarea Reţelei

Dacă există în reţeaua dumneavoastră un server DHCP la care este conectat calculatorul în timpul pornirii cu Live CD setările de reţea se vor face automat - în caz contrar va trebui să porniţi dialogul de configurare a reţelei în sidux: Apăsaţi Kmenu>sidux>Network/Internet>Networkcard configuration.

Sau - dacă nu vă place modul grafic , deschideţi o consolă , tastaţi : 

netcardconfig

şi urmaţi instrucţiunile.

For a HD install you can use 'Kmenu>sidux>Network/Internet>Networkcard configuration', and follow the prompts, or use a konsole, (as user$), with:

/usr/sbin/netcardconfig

and follow the prompts.

Activarea Ndiswrapper

sidux are toate pachetele deja instalate : modulul de kernel şi utilitaţile ndiswrapper .

Aveţi nevoie de un "Windows" inf-file pentru driver , găsiţi-l şi încărcaţi-l :

ndiswrapper -i driver.inf

încărcaţi modulul ndiswrapper :

modprobe ndiswrapper

şi verificaţi dacă s-a încărcat corect :

iwconfig

verificaţi dacă a fost găsit punctul de acces :

iwlist wlan0 scan

apoi puteţi configura placa de reţea cu :

/usr/sbin/netcardconfig

Should you update your kernel, you will need to reinstall it. Change the kernel number to suit your current kernel:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

Then load it with modprobe:

modprobe ndiswrapper

You don't have to reinstall the windows drivers as the windows driver part is seperate from the kernel module.

You can also write ndiswrapper to /etc/modules to load ndiswrapper at startup

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

See also more on the wiki

Computerele cu două plăci de reţea ( pentru Internet sharing )

Configurarea pentru conectarea la internet cu un laptop (sau alt pc) printr-o soluţie simplă :

Deschideţi un terminal şi schimbaţi pe utilizatorul root cu "sux" pentru a putea executa următorii paşi .

Configuraţi prima placă de reţea pentru accesul la internet :

/usr/sbin/netcardconfig

Pentru a doua placă de reţea vom alege o adresă IP fixă într-un anumit domeniu , ex. :

192.168.1.1

Şi deasemeni o vom configura cu :

/usr/sbin/netcardconfig

Ţineţi minte : Adresa IP dată trebuie să fie în acelaşi subnet ca şi adresa IP ce este atribuită automat prin DHCP .

Dacă din greseală s-a specificat o valoare pentru Gateway în timpul configurării interfeţelor de reţea , va trebui să rulaţi scriptul :

remove-gateway

Setarea DHCP/DNS :

apt-get install dnsmasq

Va trebui să editaţi fişierul /etc/dnsmasq.conf adăugând următoarea linie :

dhcp-range=192.168.1.50,192.168.1.150

Restul ramâne aşa cum este .

Acum serverul DHCP de pe computerul gazdă poate fi pornit cu : 

/etc/init.d/dnsmasq start
.

Dacă computerul client este setat DHCP , o adresă IP va fi atribuită automat .

Pentru a accesa internetul de pe computerele client , mai trebuie rulat scriptul : 

/usr/sbin/masquerade.

După aceasta orice computer client ar trebui să se poată conecta la internet .

Pentru a avea această configuraţie încărcată automat la pornire pe computerul gazdă : activaţi dnsmasq în procesul de boot cu :

update-rc.d -f dnsmasq defaults

Rulaţi masquerade-script cu :

echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Conectarea printr-un 56k Dial-up Modem

KDE conţine o unealtă pentru conectarea prin modem numită KPPP Internet Dial-up Tool , care se găseşte în meniul principal --> Internet .

Aplicaţia are ataşat un manual intern ce oferă un ghid foarte intuitiv pentru setarea modemului pentru accesul online .

Firewalls

Firewalls nu sunt în mod normal necesare dacă rulaţi din spatele unui router , oricum au un rol foarte important dacă doriţi să vă conectaţi la internet cu un modem usb adsl sau via dial-up modem :

apt-get install guarddog
SAU
apt-get install firestarter

Guarddog este un firewall cu o interfaţă prietenoasă pentru KDE rulând în Linux . Cea mai bună cale de a începe este să citiţi toate tutorialele , începând chiar cu primul

Firestarter - Un firewall Open Source , cu interfaţă grafică pentru Linux [ iar când veţi părăsi interfaţa grafică firewall-ul va rămâne activ în fundal ].
