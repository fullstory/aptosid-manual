GUI wpa Roaming

Metoda wpa-roaming permite conectarea la rețele wireless (fără fir) în mod grafic sau în mod consolă. O metodă foarte utilă utilizatorilor de laptop.

Programul wpasupplicant poate aloca chei WPA pentru o întreagă rețea wireless. În combinație cu interfața "clasică" /etc/network/interfaces, devine o metodă forte de conectare la rețele net, fară nici o altă intervenție suplimentară. Conectarea se face automat, dacă rețeaua este deschisă sau dacă rețeaua este agaugată la fișierul de configurație. În plus dacă opțiunea hotplug este activată, imediat după logarea în X sau în consolă conexiunea este automat activată.

Pachete de bază

Pentru a utiliza wpa-roaming cu wpasupplicant în mediul grafic trebuiesc instalate aceste pachete:
* wpasupplicant
* wpagui

apt-get update && apt-get install wpasupplicant wpagui

pachetul wpasupplicant este absolut necesar, pe când wpagui este doar un GUI ce permite controlul rețelei în cazul în care este necesar să fie schimbat punctul de acces la net.

Utilizarea wpa-gui fără a configura rețeaua

Ca utilizator root adaptați /etc/network/interfaces ca să conțină aceste linii. Numele interfeței poare fi diferit:

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
iface default inet dhcp

Apoi editați liniile următoare din /etc/wpa_supplicant/wpa_supplicant.conf:

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        key_mgmt=NONE
}

Următorul pas protejează fișierul wpa_supplicant.conf de intervenții nedorite. Aceast pas este necesar, pentru că cheile secrete de acces în rețelele private sunt salvate în acest fișier:

chmod 600 /etc/wpa_supplicant/wpa_supplicant.conf

Mod de utilizare

Faza preliminară:

* Asigurați-vă că faceți parte din grupul netdev
* Reporniți rețeaua sau reboot-ați

User-ul creat la instalarea sidux, este deja membru al grupului netdev. Orice alt utilizator nou creat trebuie adăugat grupului netdev de către root.
Acum relansați wpasupplicant ca root cu:

wpa_action <interface> reload

În multe cazuri, după prima setare a wpa-roam, o conexiune network existentă trebuie oprită și interfața wireless trebuie activată. Puteți folosi aceste comenzi ca root:

ifdown $interface  ## oprește o conexiune network are rulează deja
/etc/init.d/networking restart  ## încarcă configurația network nou creată - nu-i necesar întotdeauna
ifup wlan0  ## wlan0 este doar un nume generic pentru o interfață wireless. Luați numele corect din fișierul /etc/network/interfaces

Utilizarea wpa-gui împreună cu configurarea rețelei

Cu ajutorul IDString și a Priority se poate specifica la ce rețea va fi conectat la boot-are acest calculator/box. Prioritatea maximă este 1000, iar minimă este 0. Deasemenea trebuie să adăugați IDString la /etc/network/interfaces .

Sintaxa fișierului /etc/network/interfaces

Primul exemplu este pentru o configurare prin DHCP, al doilea este pentru alocarea unei interfețe statice. Adaptați la nevoile locale:

# id_str="home_dhcp"
iface home_dhcp inet dhcp

# id_str="home_static"
iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0
        network 192.168.0.0

        broadcast 192.168.0.255
        gateway 192.168.0.1
Exemple practice:

Dacă doriți să vă conectați automat la rețeaua locală WLAN, când sunteți acasă, alegeți IDString "home" și prioritatea "15". Dacă călătoriți și doriți să ca laptop-ul să se conecteze la orice rețea liberă, ne-criptată prin parolă (password), alegeți IDString "stalk" și prioritatea "1" (foarte joasă). Dar vă rugăm să verificați că aveți o conexiune legală și deconectați-vă dacă nu este o conexiune free/liberă.

Exemple stanzas pentru /etc/network/interfaces:

# /etc/network/interfaces -- configuration file for ifup(8), ifdown(8)

# The loopback interface
# automatically added when upgrading
auto lo
iface lo inet loopback

allow-hotplug eth0
iface eth0 inet dhcp

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf

iface home inet dhcp
iface stalk inet dhcp

Exemple pentru /etc/wpa_supplicant/wpa_supplicant.conf (SSID și passwords sunt schimbate sau doar explicate):

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        ssid="my_ssid"
        scan_ssid=1
        psk=123ABC  ##here comes the passphrase in hexadecimal code!!
#       psk="password_in_ascii"   ##you dont need to
        key_mgmt=WPA-PSK
        pairwise=TKIP
        group=TKIP
        auth_alg=OPEN
        priority=15
        id_str="home"
}

network={
        ssid=""
        scan_ssid=1
        key_mgmt=NONE
        auth_alg=OPEN
        priority=1
        disabled=1   ## no automatic connection, one needs wpa_cli or wpa_gui
        id_str="stalk"
}

Cu "disabled=1" nu veți fi conectat automat la un bloc de rețele definit (open WLANs), voi trebuind să inițiați roaming-ul prin wpa_gui sau wpa_cli. Pentru un roaming automat nu folosiți deloc optiunea sau comentați linia ce conține opțiunea "disabled" folosind semnul # la începutul liniei.

Cifrare WEP

Iată un exemplu de fișier wpa_supplicant.conf, dacă doriți să adăugați o rețea la care să vă conectați permanent :

network={
	ssid="example wep network"
	key_mgmt=NONE
	wep_key0="abcde"
	wep_key1=0102030405
	wep_tx_keyidx=0
}
Note:
1. Reutilizarea configurației

În momentul în care a veți o configurație funcțională, o puteți reutiliza usor și pe alt laptop sau desktop care are card de rețea WLAN. Copiați aceste două fișiere /etc/network/interfaces ( ajustați numele interfeței dacă este cazul ) și /etc/wpa_supplicant/wpa_supplicant.conf pe noul computer. Nu este nevoie să mai "installing" nimic după aceasta.

2. Backup-ul configurației

Este bine să salvați aceste două fișiere /etc/network/interfaces și /etc/wpa_supplicant/wpa_supplicant.conf , dar secretizați acest backup întru-cât conține informații sensibile. Acest lucru se face ușor folosind archive protejate prin parolă cu ar fi 7zip, zip, rar sau altele.

3. Acces rapid la wpa_gui

Pentru a crea un icon pe desktop, click-dreapta (rightclick) pe kmenu>internet>wpa_gui și alegeți să-l trimiteți pe desktop.

4. SSIDs ascunse

SSIDs ascunse sunt detectate când scan_ssid=1 este definit în block-ul network.

5. Algoritmi de Autentificare

Pentru criptarea WPA/WPA2 trebuie să:

auth_alg=OPEN

Pentru o criptare statică WEP trebuie să:

auth_alg=SHARED
