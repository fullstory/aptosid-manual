GUI wpa Roaming

Metoda wpa-roaming permite conectarea la rețele wireless (fără fir) în mod grafic sau în mod consolă. O metodă foarte utilă utilizatorilor de laptop.

Programul wpasupplicant poate aloca chei WPA pentru o întreagă rețea wireless. În combinație cu interfața "clasică" /etc/network/interfaces, devine o metodă forte de conectare la rețele net, fară nici o altă intervenție suplimentară. Conectarea se face automat, dacă rețeaua este deschisă sau dacă rețeaua este agaugată la fișierul de configurație. În plus dacă opțiunea hotplug este activată, imediat după logarea in X sau în consolă conexiunea este automat activată.

Pachete de bază

Pentru a utiliza wpa-roaming cu wpasupplicant în mediul grafic trebuiesc instalate aceste pachete:
* wpasupplicant
* wpagui

apt-get update && apt-get install wpasupplicant wpagui

pachetul wpasupplicant este absolut necesar, pe când wpagui este doar un GUI ce permite controlul rețelei în cazul în care este necesar să fie schimbat punctul de acces la net.

Utilizarea wpa-gui făra a configura rețeaua

Ca utilisator root adaptați /etc/network/interfaces ca să conțină aceste linii. Numele interfeței poare fi diferit:

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
iface default inet dhcp

Apoi editați liniile următoare din /etc/wpa_supplicant/wpa_supplicant.conf:

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        key_mgmt=NONE
}

The next step secures wpa_supplicant.conf from unwanted access. This is necessary, because secret keys of private networks are saved in this file:

chmod 600 /etc/wpa_supplicant/wpa_supplicant.conf

Mod de utilizare

Fază preliminară:

* Asigutați-vă că faceți parte din grupul netdev
* Reporniți rețeaua sau reboot-ați

The user created during the installation of sidux, is already a member of the netdev group. Any new users created must be added to the netdev group by root.
Now reload wpasupplicant as root with:

wpa_action <interface> reload

In many cases after the first setup of wpa-roam an existing network connection needs to be stopped and the wireless interface has to be brought up. You can use those commands as root:

ifdown $interface  ## stops a running network connection
/etc/init.d/networking restart  ## network configuration newly loaded - not always needed
ifup wlan0  ## wlan0 is just a common name for a wireless interface. Take the name from /etc/network/interfaces

Utilizarea wpa-gui împreună cu configurarea rețelei

Cu ajutorul IDString și a Priority se poate specifica la ce rețea va fi conectat la boot acest calculator/box. Prioritatea maximă este 1000, iar minimă este 0. Deasemenea trebuie să adăugați IDString la /etc/network/interfaces .

Syntaxa fișierului /etc/network/interfaces

Primul exemplu este pentru o configurare prin DHCP, al doilea este pentru alocarea unei interfețe statice. Adaptați la nevoile locale:

# id_str="home_dhcp"
iface home_dhcp inet dhcp

# id_str="home_static"
iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0
        network 192.168.0.0

        broadcast 192.168.0.255
        gateway 192.168.0.1
Exemple practice

Dacă doriți să vă conectați automat la rețeaua locală WLAN, când sunteți acasă, alegeți IDString "home" și prioritatea "15". Dacă călătoriți , și doriți să ca laptop-ul să se conecteze la orice rețea liberă, ne criptată prin password, alegeți IDString "stalk" și prioritatea "1" (foarte joasă). Dar vă rugăm să verificați că aveți o conecsiune legală și deconectați-vă dacă nu este o conexiune free/liberă.

Exemple stanzas pentru /etc/network/interfaces:

# /etc/network/interfaces -- configuration file for ifup(8), ifdown(8)

# The loopback interface
# automatically added when upgrading
auto lo
iface lo inet loopback

allow-hotplug eth0
iface eth0 inet dhcp

allow-hotplug wlan0
iface wlan0 inet manual
  wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf

iface home inet dhcp
iface stalk inet dhcp

Exemple pentru /etc/wpa_supplicant/wpa_supplicant.conf (SSID și passwords sunt schimbate sau doar explicate):

ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        ssid="my_ssid"
        scan_ssid=1
        psk=123ABC  ##here comes the passphrase in hexadecimal code!!
#       psk="password_in_ascii"   ##you dont need to
        key_mgmt=WPA-PSK
        pairwise=TKIP
        group=TKIP
        auth_alg=OPEN
        priority=15
        id_str="home"
}

network={
        ssid=""
        scan_ssid=1
        key_mgmt=NONE
        auth_alg=OPEN
        priority=1
        disabled=1   ## no automatic connection, one needs wpa_cli or wpa_gui
        id_str="stalk"
}

With "disabled=1" you will not be automatically connected to a defined network block (open WLANs), you have to initiate roaming through wpa_gui or wpa_cli. For automatic roaming don't use the option at all or comment the line with the "disabled" option using a #.

Cifrage WEP

Iată un exemplu de fișier wpa_supplicant.conf, dacă doriți să adaugați o rețea la care să vă conectați permanent :

network={
	ssid="example wep network"
	key_mgmt=NONE
	wep_key0="abcde"
	wep_key1=0102030405
	wep_tx_keyidx=0
}
Note
1. Reutilizarea configurației

În momentul în care a veți o configurație funcțională, o puteți reutiliza usor și pe alt laptop sau desktop care are cărți rețea WLAN. Copiați aceste două fișiere /etc/network/interfaces ( ajustați numele interfeței dacă este cazul ) și /etc/wpa_supplicant/wpa_supplicant.conf pe noul computer. Nu este nevoie să mai "installing" nimic după aceasta.

2. Backup de configurației

Este bine sa salvați aceste două fișiere /etc/network/interfaces și /etc/wpa_supplicant/wpa_supplicant.conf , dar encriptați acest backup întru-cât contin informații sensibile. Acest lucru se face ușor folosind archive protejate prin parolă cu ar fi 7zip, zip, rar sau altele.

3. Acces rapid la wpa_gui

To create a desktop icon, rightclick on kmenu>internet>wpa_gui and choose to send it to the desktop.

4. SSIDs ascunse

SSIDs ascunse sunt detectate when scan_ssid=1 is defined in the network block.

5. Authentification algorithms

For WPA/WPA2 encryption you need:

auth_alg=OPEN

For static WEP encryption you might need:

auth_alg=SHARED
