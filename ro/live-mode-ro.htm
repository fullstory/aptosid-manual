Parola de root in Live Mode

Atentie : Intotdeauna cand executati ceva cu permisiune root trebuie sa stiti foarte bine ce faceti ! Doar pentru navigare pe internet sau in reteaua locala nu este nevoie de permisiune root .

Pe sidux LiveCD implicit nu este setata nici o parola de root .Ca sa rulati un program care cere privilegii de root aveti cateva posibilitati :

Setarea ( temporara ) a unei parole de root :

Deschideti o consola / terminal :

sidux@0[sidux]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

acum puteti folosi aceasta parola pentru restul sesiunii live .

Rulati programul cu 

sudo
or
su-me
Apasati Alt+ F2 , si introduceti 
sudo

in fata numelui programului dorit , ex. :

sudo gparted

Rulati programul dintr-o consola root :

Deschideti o consola / shell :

sidux@0[sidux]$ su
root@0[sidux]# gparted ( sau orice altceva doriti...)

Pentru a iesi din consola tastati :

exit

apoi

exit
( dupa care consola se va inchide )

Instalarea de software cand rulati de pe Live-CD

apt-get update
apt-get install your-preferred-package

Retineti : cand veti opri lucrul de pe Live-CD toate schimbarile se vor pierde .

Stocarea permanenta a schimbarilor pe HD cand rulati de pe Live-cd

Director $HOME permanent 

home  /dev/sda1/sidux.img  mount loopback file as /home/sidux

unionrw

unionrw

Codul unionrw poate fi folosit :

  1. Specificand o partitie de pe Hard Disc
  2. Secificand un fisier loop (cum ar fi un fisier loopback tip ext2 )
  3. Specificand un director

Toate trei caile fac ca schimbarile sa fie stocate permanent in discul specificat - daca veti folosi intotdeauna acelasi cod veti putea face selectie personalizata de programe din destinatia data de utilizator .Aceasta nu stocheaza in directorul /home direct , pentru aceasta va trebui sa folositi codurile de home .

unionro

Cum deja am mentionat exista si un alt cod :

unionro

Acesta este practic ca si unionrw . dar nu aveti posibilitatea sa suprascrieti fisierele acolo unde le-ati stocat . Oricum aveti posibilitatea sa combinati un nou unionrw cod si codul unionro in acelasi timp . Nu oricine va gasi acest lucru folositor , dar , tineti minte , totul este experimental . Si inca ceva : nici unul din aceste coduri nu este mentionat in fisierul help care apare cand apasati F1 atunci cand porniti sidux .

Scrierea pe partitii NTFS folosind ntfs-3g

Atentie !: Chiar daca ntfs-3g este declarat a fi 'stable', niciodata sa nu-l folositi fara o salvare prealabila a datelor , si bineinteles nu pe sisteme productive ! Daca nu tineti cont de acestea vina in cazul pierderii de date este doar a dumneavoastra ,deci il folositi pe risc propriu!

Deschideti o consola si introduceti urmatoarele comenzi : Vezi Partitionarea Hard Discului - Numirea discurilor

su
fix-unionfs
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit
exit

Acum partitia NTFS ar trebui sa fie montata rw si sa puteti stoca date pe ea. Inca odata , atentie ! Utilizati-l doar in situatii de urgenta , nu este recomandata utilizarea zilnica.
