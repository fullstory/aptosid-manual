Instalarea de noi kerneluri

Kernelurile sidux sunt in apt. Alegeti va rog cea mai apropiata oglinda repo de sidux si schimbati sursele:

Kernelurile sunt disponibile in urmatoarele forme:

  • sidux-686 sidux-686 - kernelul pentru i686 Procesor de familie cu miezul singular/dublu sau cu mai muti CPUs
  • sidux-amd64 kernel pentru 64 bits sidux
Cele patru trepte de efectuare manuala in consola:

1. In consola schimba la root:

apt-get update

2.Instalarea ultimei versiuni de kernel:

apt-get install linux-image-2.6-sidux-686 linux-headers-2.6-sidux-686

3.Aducerea la zi a kernel-ului este acum efectuata. Daca aveti nevoie adaogati ultimul driver grafic, cartusele video, ATI, NVIDIA sau INTEL. Mai intii aduceti la zi scenariul:

sgfxi -U

4.Acum re-porniti computerul si boot-ati noul kernel (ar trebui sa-l booteze prin default).

Sa aduceti la zi driverele 3D rulati:

sgfxi

In mod alternativ,la primul reboot al noului kernel mergi direct la nivelul de rulare numarul 3 (aceasta se face prin adaogarea de 3 la Grub-Boot-Line) si apoi rulati sgfxi.

Informatii mai detailate de sgfxi se afla aici

Daca noul kernel este problematic, puteti sa rebutati si sa alegeti un kernel mai vechi.

Module

Ca sa aflati de care module aveti nevoie, urmatoarea comanda va ofera lista modulelorcurente, la zi. Copiati aceasta linie pe consola/terminal:

apt-cache search 2.6.*.slh.*-sidux| awk '/modules/{print $1}'

Pentru o descriere totala a fiecarui modul, copiaza aceasta linie pe consola/terminal:

apt-cache search 2.6.*.slh.*-sidux

Pentru instalarea modulelor necesare(de exemplu virtual-ose, si qc-usb):

apt-get install virtualbox-ose-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)
apt-get install qc-usb-modules-2.6.24-2.6.24.2.slh.7-sidux-686 (EXAMPLE)

Pentru verificarea modulelor incarcate in kernel:

ls /sys/module/
sau
cat /proc/modules

IMPORTANT: Daca aveti nevoie de anumite module drivere, e.g. pentru wlan, trebuie saa instalati pachetele corecte .deb pentru ele. Ca exemplu, pentru Agere adaptorul et131x pentru ethernet : 

apt-get install et131x-modules-*

dpkg -i et131x-modules*.debeste folosit numai daca aveti deja debs locali, dar chiar si atunci , daca sunt in the apt-cache, apt-get este o optiune mai buna.

SVezi de asemenicum sa intri on line si ndiswrapper si pentru WIFI

Alte metode de instalare de kernel

smxi

Un smxi scenariu ofera o metoda integrata de instalare a kernel-urilor, drivere si un numar de alte sisteme de scenerii administrative :

apt-get update
apt-get install sidux-scripts
smxi
siduxcc

Puteti de asemeni sa instalati kernel-uri cu centru de control GUI siduxcc: care este localizat in KDE-Start-Menu>System>sidux Control Centre. siduxcc de asemenea va ofera cateva alte sisteme de aplicatie administrativa GUI

Inlocuirea de kerneluri vechi

Dupa instalarea cu succes a noului kernel,vechile kernel-uri pot si sterse (deleted), insa se recomanda sa le pastrati cateva zile, in cazul in care aveti probleme si deci puteti sa bootati cu un kernel vechi, dupa cum este aratat in ecranul grub

Kernel-urile vechi pot fi eliminate din sistem. Ca sa efectuati aceasta, puteti folosi scenariul kernel-removercare este in pachetul sidux-kernelhacking. Acest scenariu trebuie instalat si va permite sa nomonalizati care din vechile kernel-uri doriti sa fie eliminate.

apt-get update
apt-get install sidux-kernelhacking
kernel-remover

In mod alternativ, puteti folosi siduxcc, care foloseste scenariul kernel-remover .
