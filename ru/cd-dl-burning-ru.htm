Загрузка sidux-CD

ВАЖНАЯ ИНФОРМАЦИЯ: sidux, как Linux LiveCD, основан на технологии архивации, первоначально разработанной для Knoppix, но теперь используемой в усовершенствованной форме. По этой причине нужно уделять особое внимание методу записи ISO-образа. Мы рекомендуем использовать высококачественные носители CD (или DVD+R) для записи в режиме DAO (disk at once) не быстрее чем восьмикратно (8x).

Списки зеркал и ссылки для загрузки на sidux.com

Загрузка через bittorrent.

Bittorrent - это протокол обмена файлами, соответствующая программа включена в состав sidux-CD. Bittorrent позволяет одновременно как загружать файлы самому, так и  предоставлять файлы для загрузки другими. Адреса актуального torrent-файла можно найти на sidux.com.

Загрузить torrent-файл в консоли можно, если выполнить:

# btdownloadcurses --max_upload_rate 12 ftp://dummy/sidux/sidux-200xxxxx/sidux-200xxxxx.iso.torrent

После ввода команды вскоре должна начаться загрузка.

Внимание: В директории, из которой выполняется команда, должно быть достаточно свободного место, примерно 700 МБ.

После окончания загрузки, пожалуйста, оставьте программу   работать дальше, чтобы и другие смоги загрузить Ваш файл. Это позволяет разгрузить наши сервера и ускорить сам процесс загрузки. Хорошее руководство по теме можно найти здесь: Brian's BitTorrent FAQ and Guide.

Пожалуйста, не забывайте проверять md5sum.

Если Вы можете предоставить в распоряжение FTP-сервер достаточного объема и пропускной способности, пожалуйста, свяжитесь с одним из наших администраторов. Альтернативно о своем желании можно сообщить в форуме или на канале IRC.

Сумма md5sum и проверка целостности загруженных файлов

ВАЖНАЯ ИНФОРМАЦИЯ: sidux, как Linux LiveCD, основывается на технологии архивации, первоначально разработанной для Knoppix, но теперь используемой в усовершенствованной форме. По этой причине нужно уделять особое внимание методу записи ISO-образа. Мы рекомендуем использовать высококачественные носители CD (или DVD+R) для записи в режиме DAO (disk at once) не быстрее чем восьмикратно (8x).

Сумма md5sum - это контрольная сумма файла & nbsp, используемая для проверки целостности соответствующего файла. При этом вновь полученная сумма md5sum файла сравнивается с известной предыдущей суммой. Таким образом становится возможно установить, был ли изменен файл или поврежден, что целесообразно при осуществлении загрузок

Если суммы md5sum загруженного файла совпадают, то файл скачан без ошибок. В Linux сумму MD5 файла можно получить так:

$ md5sum имя_файла (который вы хотите проверить)

Затем сумма выводится на консоль. С помощью md5sum (486 KB) MD5-сумма может быть расчитана и под Windows.

ISO-образы sidux всегда предлагаются для загрузки с соответствующей суммой md5sum и всегда должны проверяться перед записью на CD. Это должно выполняться в терминале. Переходят в директорию, содержащую как файл ISO, так и сумму MD5, командой:

$ md5sum -c sidux-имя.iso.md5

Проверяют сумму md5sum. Если обе проверочных суммы не совпадают, будет получено сообщение об ошибке:

sidux-имя.iso: неудача md5sum: предупреждение: 1 от 1 рассчитанной проверочной суммы не совпадают.

Если файл был скачан без ошибок, суммы должны будут совпасть и программа закончится без какого-либо сообщения.

Прожиг LiveCD в Windows

По этой причине нужно уделять особое внимание методу записи ISO-образа. Мы рекомендуем использовать высококачественные носители CD (или DVD+R) для записи в режиме DAO (disk at once) не быстрее чем восьмикратно (8x).

Само собой разумеется, CD может прожигаться также в Windows. Загруженный файл должен прожигаться как ISO файл. Winrar (или другая программа архивного хранения) будучи связан с ISO файлами, позволит видеть ISO-образ как архивный файл. Этот ISO-образ используется для записи CD.

Имеется ряд альтернативных программ для записи ISO в Windows.

Запись ISO-образа под Linux

ВАЖНАЯ ИНФОРМАЦИЯ: sidux, как Linux LiveCD, основывается на технологии архивации, первоначально разработанной для Knoppix, но теперь используемой в усовершенствованной форме. По этой причине нужно уделять особое внимание методу записи ISO-образа. Мы рекомендуем использовать высококачественные носители CD (или DVD+R) для записи в режиме DAO (disk at once) не быстрее чем восьмикратно (8x).

На компьютере с уже установленным Linux, можно создать LiveCD с любой установленной в системе программой для записки дисков. В sidux - k3b - стандартная программа для записки дисков. В пункте меню необходимо выбрать "дополнения" -> "Имидж файл записать ... ", записываемый ISO-образ (например, sidux-Имя.iso); DAO (Disk At Once) как метод для записи дисков.

K3b сначала рассчитывает сумму MD5 ISO-образа (занимает некоторое время). Совпадение расчитанного значения MD5 со значением проверочной суммы MD5 файла, сохраненного при загрузке образа и находящегося в той же самой папке (например, sidux-Имя.iso.md5) означает, что загрузка прошла успешно и файл может быть записан простым щелчком на кнопке "Start".

Нажатие правой кнопкой мыши на строку, содержащую расчитанную сумму и выбор пункта "compare checksums" вызовет дополнительное окно, в котором можно ввести известную MD5 сумму вручную для сравнения.

Актуализация содержания LiveCD с помощью xdelta-патчей

ВАЖНАЯ ИНФОРМАЦИЯ: sidux, как Linux LiveCD, основывается на технологии архивации, первоначально разработанной для Knoppix, но теперь используемой в усовершенствованной форме. По этой причине нужно уделять особое внимание методу записи ISO-образа. Мы рекомендуем использовать высококачественные носители CD (или DVD+R) для записи в режиме DAO (disk at once) не быстрее чем восьмикратно (8x).

xdelta является программой, с помощью которой из имиджа ISO и файла разницы можно создать новый имидж ISO. Это очень удобно, т.к. позволяет включать в записываемый образ все самые свежие обновления и исправления ошибок, без издержек на полную загрузку.

In a terminal/konsole as root:

apt-get update
apt-get install xdelta

Если,предположим, файлы разницы: от patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta объемом 16,5 МБ и соответственно 5,4 МБ, в то время как сам полный ISO-образ будет 553 МБ. (Это - просто пример.)

Пример

  • ISO-Файл лежит на жестком диске, например, sidux.iso
  • Файл разницы, например, patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta

Оба файла должны быть в одной и той же папке.

Применение патча для ISO-образа как пользователь в консоли:

$ cd /path/to/iso/and/patch
$ xdelta patch patch_2007-03-200707231344-gaia_pre2-kde-lite_200708051825-gaia_pre3-kde-lite.xdelta

Программа распаковывает необходимые файлы из патча, проводя при этом проверку md5sum. Осторожно! При этом не проверяется достаточно ли есть места на жестком диске.

По следующей прямой ссылке находится программа xdelta для Windows, посредством которой можно пропачить ISO-образ. Здесь прямая ссылка к программе: xdelta.exe. На той же самой странице имеется информация, но только на английском языке.
