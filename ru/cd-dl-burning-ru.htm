Загрузка sidux-CD

Download-Links und Mirrors findet man unter sidux.com

Загрузка через bittorrent.

Bittorrent - это протокол обмена файлами, программа для того находится на sidux-CD. Он позволяет одновременно загружать файлы, как для себя, так и  для других. Адреса актуального torrent-Файла можно найти на sidux.com.

Загрузить torrent-Файл в консоли можно, например, таким образом выполнить:

# btdownloadcurses --max_upload_rate 12 ftp://dummy/sidux/sidux-200xxxxx/sidux-200xxxxx.iso.torrent

В консоли загрузка должна скоро начинаться.

Внимание: В директории, из которого запускается команда, должно быть достаточно свободного место, примерно 700 МБ.

После окончания загрузки, пожалуйста, оставьте программу  дальше работать, для того чтобы и другие смоги загрузить Ваш файл. Это разгрузит наши серверы и загрузка для других будет протекать быстрее. Здесь имеется хорошее руководство по теме: Brian's BitTorrent FAQ and Guide.

Пожалуйста, не забывайте проверять MD5-сумму.

Если Вы можете предоставить в распоряжение FTP-сервер с достаточным количеством потока информационного обмена, пожалуйста, контактируйте с одним из наших администраторов. Альтернативно о вашем желании можно сообщить в форуме или канале ретрансляции бесед в интернете (IRC-Channel).

Сумма MD5 и проверка целостности загруженных файлов

Сумма MD5 - это контрольная сумма файла. Эта контрольная сумма  используется для проверки целостности принадлежащего файла. При этом временная сумма MD5 файла сравнивается с известной предыдущей суммой. Таким образом, может устанавливаться, был ли изменен файл или был поврежден, что целесообразно при загруженных файлах.

Если сумма MD5 загруженного файла совпадает с суммой, которая указана в файле MD5, то файл загружался правильно. В Linux получают сумму MD5 файла так:

$ md5sum имя файла, который вы хотите проверить

Это продолжается недолго, и сумма выводится затем на консоли. Mit dem md5summer (486 KB) kann die MD5-Summe auch unter Windows geprüft werden.

ISO-обраьы sidux всегда предлагаются с соответствующей суммой MD5 для загрузки и всегда должны проверяться перед запиской на CD. Это должно выполняться в терминале. Переходят в директорию, в которой находятся как файл ISO, так и сумма MD5, и командой:

$ md5sum -c sidux-имя.iso.md5

Проверяют сумму MD5. Если обе проверочных суммы не совпадают, c программы получают сообщение:

sidux-имя.iso: неудача md5sum: предупреждение: 1 от 1 рассчитанной проверочной суммы не совпадают.

Если, однако, загруженный файл правилен, то программа заканчивается без какого-либо сообщения.

ВАЖНАЯ ИНФОРМАЦИЯ: sidux, как Linux LiveCD, базируется на высоко архивирующей технологии, которая была первоначально разработана для Knoppix, которая теперь используется в более усовершенствованной форме, и по этой причине нужно уделять особое внимание методу для запискаи ISO-образа. Мы рекомендуем высококачественные средства CD (или DVD+R), записку в режиме DAO (disk at once) и не быстрее чем восьмикратно (8x).

LiveCD прожигают в Windows

Само собой разумеется, CD может прожигаться также в Windows. Загруженный файл должен прожигаться как ISO файл. Если Winrar (или другая программа архивного хранения) был связан с ISO файлом, то эта программа могла бы просматривать ISO файл как архивный файл. Из этого ISO Файла должен прожигаться CD. ISO Файл не должен записывать как CD для данных!! И он должен записываться в режиме Disk-At-Once.

Имеются различные хорошие опции, чтобы прожигать ISO файлы в Windows.

Программное обеспечение с открытым исходным кодом для записки в Windows

cdrtfe: совместима с Windows 9x/ME/ в 2000/XP (протестирована с Win95, Win98SE, Win2000 и WinXP). Только для Win9x/ME: функционирующие ASPI-Layer (например, Adaptec ASPI 4.60)

  • Nero (не nero express!)
  • CDR-WIN тестовая версия может загружаться из интернета, однако, запизывают на маленькой скорости, что, тем не менее, для этой задачи должно было быть достаточным. В англоязычном menu нужно выбрать следующий пункт: "Burn a ISO 9660 Image File to CD".

Нажигание ISO-образа под Linux

Тот, кто уже имеет Linux на компьютере, может создать LiveCD с каждой установленной программой для записки дисков. В sidux k3b - это стандартная программа для записки дисков. Там нужно выбрать в пункте меню "дополнения" -> "Имидж файл записать ... ", записываемый ISO-образ (например, sidux-Имя.iso) выбирают DAO (Disk At Once) как метод для записки дисков.

K3b рассчитывает сначала сумму MD5 ISO-образа (длится некоторое время). Соответствует показанная проверочная сумма с указанной последовательностью символов, которые находятся в той же самой папке MD5 файл (например, sidux-Имя.iso.md5) значит загрузка прошла успешно и файл может быть записан со щелчком на "Start".

Если нажимают на рассчитанную проверочную сумму, появляется наряду с этим символ. Нажав снова на этот символ вводят в поле проверочную сумму из MD5 файла, таким образом, сравниваются обе проверочных суммы.

Создание актуального LiveCD с помощью xdelta-патч

xdelta является программой, с помощью которой в том числе из имиджа ISO и файла разницы создают новый имидж ISO. Это очень удобно, предлагать более маленькие исправленные ошибки и корректуры без больших издержек на загрузку.

Если, например, предположим, файлы разницы являются RC1 после RC3 (PATCH-RC1-RC2. XDELTA и PATCH-RC2-RC3.XDELTA) только 16,5 МБ и соответственно 5,4 МБ, в то время как сам польный ISO-образ будет 553 МБ. (Это - фиктивный пример.)

Предпосылка

  • ISO-Файл лежит на жестком диске, например, sidux.iso
  • Файл разницы, например, PATCH-RC2-RC3. XDELTA

Оба файла должны быть в той же самой папке.

Применение патча для ISO-образа как пользователь в консоли:

$ cd /pfad/zum/iso
$ xdelta patch PATCH-RC2-RC3.XDELTA

Программа распаковывает необходимые файлы из патча, проводя при этом проверку md5sum. Осторожно! При этом не проверяется достаточно ли есть места на жестком диске.

По следующей прямой ссылке находится программа xdelta для Windows, посредством которой можно пропачить ISO-образ. Здесь прямая ссылка к программе: xdelta.exe. На той же самой странице имеются информации, но только на английском языке.
