Cheat коды

Синтаксис использования данных из поля "значение" таблицы: cheat_код=значение . Если, например, 1280x1024 желаемое разрешение экрана, то необходимо ввести screen=1280x1024 в командную строку grub, или для португалького языка, lang=pt, или для acpi, acpi=off

Особые для sidux Live-CD Cheat коды

Cheat коды Значение Описание
desktop kde выбор менеджера окон: kde
fluxbox выбор менеджера окон: fluxbox
dpi auto or XX устанавливает желаемыe пискелы за пошлину для монитора (н.п. 15"-монитор c 1600x1200 экраном иммет 1600/(15*(4/5)) = 133, т.е dpi=133)
fromiso Смотрите в главе "fromiso"
hostname myhostname hostname, изменяет имя хоста системы на Live-CD
hsync 80 (z.B.) устанавливает горизонтальную X-частоту развертки монитора в килогерцах
lang au, be, bg, br, ch, cn, cz, da, de, es, fi, fr, gr, gb, il, hr, ie, ga, it, jp, nl, pl, pt, ru, si, sk, tr, tw, us устанавливает язык по умолчанию, основую локализацию (locale), расположение симболов на клавиатуре (в консоле и в X), пояс времени и зеркальный сервер Debian.
Если соответствующие пакеты i18n установлены, sidux принимаются желаемые установки.
Можно использовать также синтаксис "lang=ll-cc". Для "ll" и "cc" можно использовать каждому существующему из 2 букв поддержанный код языка. Таким образом, "ll" выбирает язык, "cc" клавиатуру и часовой пояс (например, "lang=ru-us"). Вследствие этого установки разъединяются для языка и клавиатуры.
md5sum проверяет, в порядке ли загруженный CD
nodhcp не использует DHCP (DHCP пробует автоматически создать Ethernet соединения)
noeject не удаляет CD из дисковода
nointro пропускает вывод index.html при запуске Live-CD
noml вынуждает конфигурацию X.org при применении новых расширений RandR 1.2 драйверов X.org.
nocpufreq не активирует Speedstep/Powernow
nonetwork не активирует автоматическую конфигурацию сетевых адаптеров при загрузке
noxorgconfig не активирует автоматическое создание файла конфигурации X.org (/etc/X11/xorg.conf)
noxprobe не активирует определение параметров монитора для legacy драйверы X
noxrandr не активирует применение расширений RandR 1.2 новыми драйверами X.org и использует старые техники для опроса свойств монитора
persist Смотрите в главе "fromiso и persist"
screen 1280x1024 устанавливает определенное пользователем разрешение для X (1260x1024 или другие разрешения дисплея)
smouse ищет посредством hwinfo серийные устройства ввода мыши
toram копирует CD в RAM и стартует с копии в RAM
vsync 60 (например) устанавливает вертикальную частоту монитора в килогерцах
xdepth values: 8 15 16 24 устанавливает цветную глубину, которую используется X.org (не все драйверы 1 и 4 поддерживают)
xkblayout (eg, us, de, gb) устанавливает расположение клавиш на клавиатуре, которое используется X.org
xkbmodel (eg, 104, 105) устанавливает тип клавиатуры, который используется X.org (число обозначает количество клавиш)
xkbvariant eg, nodeadkeys устанавливает вариант клавиатуры, который используется X.org
xmode (eg 800x600) устанавливает размер экрана по указанным данным, 1024x768, 1600x1200 etc
xmodule or xdriver ati, fbdev, i810, mga, nv, radeon, savage, vesa использует выбранный модуль X
xrandr вынуждает конфигурацию X.org при применении новых расширений RandR 1.2 драйверов
xrate XX вынуждает предпочитаемую частоту повторения при драйверах, которые поддержаны RandR 1.2. Эта опция должна использоваться в сочетании с загруочной опцией xmode. Подробная документация находится здесь.
xhrefresh 75 (например) устанавливает горизонтальную частоту в X (монитора) в килогерцах
xvrefresh 60 (например) устанавливает вертикальную частоту  в X (монитора) в герцах

irqpoll использует опрос IRQ

noagp не использует поддержку AGP (Accelerated Graphics Port)
noapic не использует опрос APIC (Advanced Programmable Interrupt Controller)
nodma не использует поддержку для DMA (Direct Memory Access)
noisapnpbios не проводит  ISA-" Plug and Play " опрос при запуске
nomce

quiet не производит вывод на дисплей
