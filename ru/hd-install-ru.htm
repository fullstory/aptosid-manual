Подготовка установки sidux на жесткий диск

Для нормального использования мы рекомендуем файловую систему ext3. ext3 wird im Upstream sehr gut gewartet.

Перед установкой нeобходимо удалить все карты памяти USB ("флэшки"), камеры USB и т.д. Инсталляция на USB-устройство требует другого подхода и описана в другом разреле. С помощью редактора можно изменить файл инсталлятора ~/.sidconf, чтобы выбрать другую файловую систему или произвести инсталляцию на несколько разделов.

В дальнейшем рекомендовано создание собственного отдельного раздела данных. Преимущества в случае спасения данных, а также для стабильности данных беспредельны.

Директория $HOME станет местом, в котором будут, сохранятся конфигурации программ.

На компьютерах с менее чем 512 МБ оперативной памяти должен быть раздел swap. Величина swap'а не должна была быть менее 128 МБ. Swap размером более 1ГБ обычно не нужен и в основном используются при Suspend-to-Disk и серверах.

Инсталляция на жестком диске несравненно комфортабельнее и быстрее, чем система, которая работает только с СD-привода.

Сначала надо в BIOS'е переставить последовательность загрузки на СD-привод. У большинства компьютеров в BIOS SETUP можно попасть нажатием ' Entf клавиши' во время загрузочного процесса (в нескольких версиях BIOS можно выбрать загрузочный дисковод так-же во время загузки, например с F11 или F8).

sidux должен загрузить теперь в большинстве случаев без проблем. Если это не произойдет, то тут помогут загрузочные опции (так называемые cheat коды), которые передаются загрузничному менеджеру. Используя параметры загрузчика (например для решения экрана или языкового выбора) можно сэкономить много времени на конфигурации системы после инсталляции. С клавишей F4 можно выбирать, к примеру, язык при загрузке LiveCD. Смотри также и разрешение VGA

Инсталляционная программа sidux'a

1. Инсталляционнoе приложение запускают удобно с меню KDE: Kменю > sidux > sidux installer gui или в консоли с командой

su-me sidux-installer

2. После чтения (и понимания) текста указания начинается выбор разделов.

Теперь выбирается, куда инсталлируется, и устанавливаются также точки монтирования. Разделение, для которого не было выбрано никакой пункт монтирования, назначаются инсталляционной программой. Для раздела root ("/") пункт оправы должен устанавливаться теперь. Также может теперь создаться раздел данных.

Пожалуйста, еще раз удостоверьтесь, сделано ли резерв-копирование для всех данных.

Если жесткий диск еще не является разделеным, должно нажиматься кнопку запуска менеджера разделения, или читают главы Разделение жесткого диска и соответственно разделение с gparted в руководствe.

Разделение можно также запустить их с консолью терминалом::

su
gparted

или

su
cfdisk

3. Как системный загрузчик мы используем grub в MBR! Другой выбор является только опытным пользователям рекомендован, которые знают, какие последствия это решение с собой влечет. Grub автоматически распознает, установлены ли другие операционные системы на компьютере (например, Windows), и он добавляет их к меню загрузки.

4. Затем устанавливают имя пользователя, пароль пользователя и пароль root  (пожалуйста, хорошо его запомните!). Пароли должны по соображениям  безопасности не слишком просто выбираться. Тот, кто хотел бы организовывать дальнейших пользователей, может это сделать после установки с управлением пользователя KDE kuser.

5. Вслед за этим выбирается имя установки. Имя может быть любым, но компьютерное имя (имя хоста) может состоять исключительно из букв и чисел (никаких специальных символов или умлаутов) и не может начинаться с цифрами.

6. Этот опрос - это последняя возможность, проверить избранные установки. Пожалуйста, снова основательно читайте и обдумывайте! Процесс установки продолжается в зависимости от системы между 5 и 60 минутами, со специальными опциями загрузки установка может проводиться даже еще быстрее.

Следующие пункты в меню упомянуты для опытных пользователей Linux, которые хотели бы установить особенные установки в файлах конфигурации (~/.sidconf) или жесткий магнитный диск был особым способом размечен, который автоматический контроль программы инсталляции отверг бы.

Если балка прогресса в течение некоторого времени не перемещается, то установку не прекращают, а просто нужно дать процессу установки время.

После завершения установки CD вынимается из дисковода и новая установка на жестком диске перезагружается!

Первоначальная загруска с жесткого диска

После первого запуска sidux сначала забывает установки сетей. Итак, сеть (Wlan, модем, интегрированная служба цифровой сети...) должны быть еще раз настроены.

Кто раньше, например, через DSL-Router посредством (DHCP) получал адрес сети, должен теперь однажды вызвать следующую команду:

netcardconfig

Соответствующие инструменты находят снова в Kmenu > sidux > Network/Internet. Смотри также: Интернет и сеть

Совет для консоли при установке: для получения прав root для различных заданий, просто открывают терминал и печатают следующее:

sux
