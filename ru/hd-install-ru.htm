Подготовка установки sidux на жесткий диск

Мы рекомендуем использовать файловую систему ext3, как проверенную временем и хорошо поддерживаемую

Перед установкой нeобходимо отключить все накопители USB ("флэшки"), камеры USB и т.д. Инсталляция на USB-устройство требует другого подхода и описана в другом разреле. С помощью редактора можно изменить файл инсталлятора ~/.sidconf, чтобы выбрать другую файловую систему или произвести инсталляцию на несколько разделов.

В дальнейшем рекомендуем создать отдельный раздел для размещения ваших данных. Это может потребоваться как для резервирования данных так и в целом полезно для достижения большей стабильности системы.

Директория $HOME станет местом, в котором будут, сохранятся пользовательские настройки.

На компьютерах с менее чем 512 МБ оперативной памяти должен быть раздел swap. Величина swap'а не должна была быть менее 128 МБ. Swap размером более 1ГБ обычно не нужен и в основном используются при Suspend-to-Disk и на серверах.

Инсталляция на жестком диске несравненно комфортабельнее и быстрее, чем система, которая работает только с СD-привода.

Сначала необходимо изменить в BIOS'е последовательность загрузки, указав первым СD-привод. Большинство компьютеров позволяет попасть в BIOS SETUP нажатием клавиши ' Enter' в самом начале загрузочного процесса (в некоторых версиях BIOS выбрать загрузочный дисковод возможно используя клавиши F11 или F8).

В большинстве случаев загрузка проходит без проблем. При возникновении трудностей помогут загрузочные опции (так называемые cheat коды), которые передаются загрузочному менеджеру. Используя параметры загрузчика (например для разрешения экрана или выбора языка) можно сэкономить время на конфигурации системы после инсталляции. С клавишей F4 можно выбрать, к примеру, язык при загрузке Live-CD. Смотри также и разрешение VGA

Выбор языка инсталляции
  1. Для выбора языка Live-CD, отличного от английского (например fr, pl, es), в меню grub используйте клавишу F4
  2. Инсталляция: используйте meta-installer для инсталляции языковых пакетов, согласно вашего выбора в grub (или поддержки любого другого языка)
  3. Осуществите установку системы на жесткий диск и перегрузитесь
  4. Ненужные Вам языки (пакеты предоставляющие их поддержку) можно удалить из системы. С помощью apt-get, если мета-пакет языковой поддержки, требующийся вам, пока не доступен, установите соответствующие пакеты вручную.

Теперь Live-CD позволяет проверить, были ли обнаружены все устройства Вашего компьютера. Локальная сеть (интернет-связь) комфортабельно настраивается через меню sidux. Для настройки беспроводной сети под Wifi-WLan смотрите

Инсталляционная программа sidux'a

1. Инсталляционнoе приложение удобно запускается из меню KDE: Kменю -> sidux -> sidux installer gui, или в консоли командой:

sux install-gui

2. После внимательного ознакомления с текстом указаний начинается выбор разделов.

Переходим к выбору точек монтирования созданных разделов. Если таковую не определить, точка монтирования будет создана автоматически инсталляционной программой. Также теперь можно создать дополнительные разделы.

Пожалуйста, еще раз удостоверьтесь, сделано ли резервное копирование ваших данных!

Если необходимо заново создать разделы, то кнопкой запускается графический менеджер разбиения дисков; подробно описаный в главах руководства Разметка диска: cfdisk и соответственно Разметка диска: gparted

Из консоли запуск осуществляется следующими командами:

su
gparted

или

su
cfdisk

Вам также будет предложено выбрать свой часовой пояс

3. Как системный загрузчик мы используем grub в MBR! Выбор других вариантов рекомендован только опытным пользователям, которые понимают, последствия принятия этого решения. Grub автоматически распознает, установлены ли другие операционные системы на компьютере (например, Windows), и добавляет их к меню загрузки.

4. На следующем этапе устанавливают имя пользователя, пароль пользователя и пароль root  (пожалуйста, хорошо его запомните!). По соображениям  безопасности пароли не должны быть слишком простыми. Утилита управления пользователями для KDE позволяет создавать дополнительных пользователей уже после установки и перезагрузки. kuser.

5. Следующим шагом Вам будет предложено выбрать сетевое имя для своей системы. Имя может быть любым, но компьютерное имя (имя хоста) может состоять исключительно из букв и чисел (никаких специальных символов или умлаутов) и не может начинаться с цифры.

Также Вам будет предложено установить автоматический старт призагрузке сервисов cups (печать) и ssh.

6. Этот запрос - последний шанс проверить корректировки, которые Вы сделали. Прочитайте это снова тщательно, затем нажмите 'Конфигурация сохранена и дальше'.

В этом пункте возможно измениться/редактировать файл конфигурации и затем запустить процедуру установки с измененной конфигурации. Инсталлятор не делает никаких проверок, и Вы не должны нажать 'Назад' на инсталлятор иначе, изменения, в которые вводят вручную будут потеряны.

Редактирование (~/.sidconf) - для опытных linux-пользователей, которые хотят записать, их специальное переходит на файл конфигурации или иметь специальную схему разделения на их жестком диске, который был бы отклонен автоматической проверкой инсталлятора.

To commence the installation click on Begin Installation The whole process takes, depending on your system, between 5 - 15 minutes, on older PCs it may take as long as 60 minutes..

If the progress bar hangs in one place for a while, don't abort, just give it some time.

Finished! Take the CD out of the tray. Now reboot to your new HD Install.

Первоначальная загруска с жесткого диска

После первого запуска sidux   "забывает" сетевые настройки. Поэтому сетевые соединения (Wlan, модем, интегрированная служба цифровой сети...) должны быть настроены заново.

Если во время инсталляции, например, сетевой адрес получался через DSL-Router посредством (DHCP), то теперь небходимо выполнить следующую команду:

/usr/sbin/netcardconfig

Соответствующие инструменты находятся в Kmenu > sidux > Network/Internet. Смотри также: Интернет и сеть

About sux

Numerous commands need to be run with root privileges. To achieve this you type:

sux

While the common command for becoming root is 'su' using sux instead will allow you run GUI / X11 applications from the command line and allow root to start graphical applications, as sux is a wrapper around the standard su command which will transfer your X credentials to the target user.

An example of running an X11 app via sux is to use a text editor to edit a root file like kwrite or kate, to do partitioning with gparted or an Xapp file manager like konqueror. You can also alter root files by right clicking the file and choosing 'edit-as-root', and entering your root password, this will call kdesu in the background.

Unlike 'sudo', it means that someone can't just come along and type 'sudo' and make potentially damaging changes to your system.

WARNING: While you are logged in as root, the system will not stop you from doing potentially dangerous things like deleting important files etc., you have to be absolutely sure about what you are doing, because it's very possible to seriously harm your system.

Under no circumstances should you be as root in the console/terminal to run applications that a standard user uses to go about being productive on a day to day basis, like sending emails, creating spreadsheets or surfing the internet and so forth.
