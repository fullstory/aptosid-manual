Резервное копирование раздела / home

Перед перемещением каталога /home Вы должны создать резервную копию все ниже каталога /home (как root):

tar cvzpf  где-то/home.tar.gz /home

для распаковки:

tar xvpf где-то/home.tar.gz

Альтернативным методом для резервного копирования является применение rdiff.

Это может быть полезно, если Вы хотите обновить ваш каталог /home c предыдушей ха новую установку.

Перемещение /home

Перемещение /home можно сделать двумя способами, первый способ с LiveCD, а второй с командной строки (обе способа являются несложными).

Для того система нуждается UUID информация о новых разделов.

Самый простой метод заключается в том, чтобы перед перемещением каталог /home добавить эту новую информацию с комментарием # в /etc/fstab.

Компьютер перезагрузится и в командную строку Grub введите цифр '1' и нажимаете клавишу ENTER. PC перезагрузаете в режим загрузи 1. В том случае Ваш каталог /home не используется, работа с ним является безопасным.

Входите в tty косоль и смонтируете Ваш новый каталог /home как root, например:

mount /dev/sdxX /media/new-home
(или любой другой раздел)
cp -pr /home /media/new-home

Затем измените /etc/fstab, чтобы активировать Ваш новый каталог /home: 

 
mcedit /etc/fstab

Теперь Вы раскомментируете (удаляете) с # в строке для нового каталога /home и , закомментируете с # в начале сроки старого каталога /home, сохраняете с F2, затем выйдете с F10 и перезагрузите.
