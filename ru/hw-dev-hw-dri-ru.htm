Несвободные бинарные драйвера для: Nvidia/ATI

Имеются 3 метода инсталлировать 3D-драйвер nVidia/ATI

  1. sgfxi / smxi

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxi поддерживает в натоящий момент ATI fglrx и nVidia драйверы. Переход от проприетарных к свободным драйверам и обратно поддерживается этим скриптом для графических карт ati, intel или nv. Скрипт smxi использует sgfxi как метод инсталляции для графического драйвера и добавляет несколько опций, упрощая процедуру установки.

    Драйвер по умолчанию инсталлируется, если не указываются никакие дополнительные опции инсталляции. Также скрипт автоматически определяет более старые графические карты и сразу инсталлирует драйвер Legacy.

    После старта скрипта и опознования графической карты выдается сообщение о том, какой драйвер должен инсталлироваться. При этом имеется возможность подтвердить этот выбор и инсталлировать драйвер или отменить инсталляционный процесс и перезапустить скрипт с выбранной опцией.

    ИЛИ: используют h2 smxi для установки графических драйверов: smxi. Для инсталляции только 3D драйвера, возможно использовать команду:

    smxi -kiwdt

    sgfxi-h перечисляет все опции. Опции со временем дополняются. Также отображаются поддерживаемые на текущий момент драйверы и спецификации используемой системы такие,как ядро и xorg версия:

    -c использовать composite (только nvidia).
-d удалить незадействованные дисплеи, разрешить TV, например (только nvidia).
-h это меню помощи.
-n Автоматическая установка правильных встроенных xorg-драйверов nv, intel, или ati для вашей системы.
   Старые бинарные драйвера удаляются, файл xorg.conf перезапмсывается с включением новой информации и удалением прежних, относящихся к 
   nvidia и fgrlx.
-N Форсировать установку выбранных xorg-драйверов. 
Синтаксис: sgfxi -N <один из списка драйверов, например -N radeon>
   Поддерживаемые драйвера: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Дополнительная настройка xorg.conf является необходимой, кроме radeon xorg 3d, конфигурируемого автоматически.
-o Override default: install a specific driver, this is an absolute override - syntax:
   -o 9746 (для nVidia, не используйте часть названия - 1.0) или: -o 8.32.5 (для fglrx)
-r Remove mode lines from xorg.conf
-t Включить dynamic twinview (только nVidia)

  2. install-binary-gfx

    Новый скрипт для установки драйверов графических карты называется: install-binary-gfx. Чтобы инсталлировать скрипт, необходимо выполнить в консоли, как пользователь root: get-sidux-binary-gfx.

    get-sidux-binary-gfx
install-binary-gfx -a (Применение исходного кода из репозитория sid)

    Во время выполнения команды выбирается правильный драйвер для графической карты, собирается и инсталлируется соответствующий пакет .deb. Все необходимые измения в файл конфигурации /etc/X11/xorg.conf вносятся автоматически. Для активации установленного драйвера необходим перезапуск сервера X (с помощью комбинации Сtrl+Alt+клавиша возврата).

    Опции

    Допустимы следующие опции:

     Usage: ./install-binary-gfx [options]
-a | --auto            автоматический способ
-b | --build           только строительство MODs модуля (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          только определение карт
-u | --update          обновление самого инсталяционного скрипта
-v | --verbose         подробный вывод

    Примеры

    install-binary-gfx -a (это определяет графическую карту и устанавливает автоматически необходимые драйверы из репозитория sid)
install-binary-gfx -b nvidia (собирается необходимый модуль для nVidia, пакет deb помещается в /usr/src. Это же самое верно для fglrx (ATI) и nvidia-legacy)
install-binary-gfx -d (это устанавливает драйвер графической  карты и выдает сведения о ней)
install-binary-gfx -u (происходит обновление инсталляционного скрипта.)

    Помощь при проблемах

    Если инсталляция драйвера прерывается с ошибкой, может использоваться метод получения отладочной информации debug. Он создает файл журнала /var/log/sidux-CARD-installer.log. Для получения помощи, содержание этого фай­ла может быть опубликовано в форуме sidux, или сохранено на http://paste.debian.net для указания созданной ссылки на канале IRC #sidux.

  3. Инсталляция бинарного несвободного драйвера для Live-CD:

    sudo sgfxi

    Выполнение этой команды обновит инсталляционный скрипт, после чего:

    ctrl+alt+f1

    и выполнить

    sgfxi

    sgfxi -h,
выводит список доступных опций

    По окончании инсталляции драйвера будет перезапущен X-сервер.

  4. Использование нативного инсталлятора Nvidia/ATI
    Инсталлятор непосредственно загружается с сайта Nvidia или ATI.

Свободные (предлагаемые xorg) драйвера графических карт nVidia, ATI, Radeon, Intel

  1. sgfxi

    вы можете использовать sgfxi для переключения между свободными и несвободными версиями драйверов:

    -n Инсталлятор автоматически выбирает и корректно устанавливает нативный xorg-драйвер nv, intel, или ati для Вашей системы.
   При этом информация о старых драйверах стирается, а о новых вносится в xorg.conf,
   полностью необходимым образом перезаписывая данный файл.
-N Форсированнная инсталяция указанного драйвера 
Синтаксис: sgfxi -N <один из драйверов в списке, например -N radeon>
   Драйвера поддерживаемые xorg: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Все необходимые изменения вносятся в файл xorg.conf автоматически. Настройка radeon xorg 3d также осуществляется автоматом.

    сперва

    apt-get update
apt-get install sidux-scripts
sgfxi

    Новая инсталляция или откат к использованию драйвера xorg

    sgfxi -n
или
sgfxi -N <xorg driver>

  2. sgfxi / du-fixes-h2.sh

    Использование sgfxi и h2 smxi. Запустите скрипт с указанными ниже опциями и вы сразу переместитесь в раздел установки графического драйвера. При использовании du-fixes необходимости устанавливать sgfxi не возникает, так как это автоматически выполняет du-fixes.

    Besuchen Sie diesmxi-домашняя страница для дополнительной информации. Выполняемая команда:

    smxi -kiwdt

  3. Более сложный способ

    Драйвера X-Сервера,предоставляемые X.Org (смотри xserver-xorg для более подробной информации) обеспечивают поддержку 2D для следующих карт: NVIDIA Riva-, TNT-, GeForce- и Quadro-карты, Rage-, Radeon- а также карты серии FireGL ATI, включая 'r128' и 'radeon'. Для Intel включена поддержка 3D.

    Для установки:

    выйти из KDE одновременным нажатием Ctrl+Alt+F1 в текстовую консоль и зарегестрироваться в системе суперпользователем root
выполнить команду "init 3"
(приводящую к остановке X-сервера)
apt-get update
apt-get install xserver-xorg-video-nv
или
apt-get install xserver-xorg-video-ati
init 5 && exit

    Примечание: Это не является бинарным драйвером NVidia, включающим поддержку 3D. Проприетарный драйвер NVidia нужно получать исключительно бинарным и в принципе он не поддержан в Linux.

    Теперь необходимо отредактировать /etc/X11/xorg.conf. Предварительно создайте резервную копию, иначе сохранение ошибочно отредактированного /etc/X11/xorg.conf принесет много проблем и потребует много времени на исправление.

    Откройте для редактирования /etc/X11/xorg.conf  с правами root, посредством щелчка мыши. В разделе SECTION DEVICE изменить драйвер дисплея на nv или ati

    Осуществление данных действий связано сопределенным риском!

Указания к аппаратным средствам, которые нуждаются в проприетарных модулях

В соответствиии с нормами права применительно лицензионного програмного обеспечения на sidux live-cd включает только ПО совместимое с dfsg-free.

Поэтому необходимо раскомментировать строку содержащую contrib/non-free в /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in seperate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • Графические карты ATI-Radeon: трехмерное ускорение для более старых карт до r35x должно функционировать. Более новые карты Radeon X1xxx нуждаются в независимых модулях, чтобы функционировать с 3D-ускорением. Эти модули еще не совместимы до конца с X.org 7.2.
  • Atheros/ "madwifi" wlan: в консоли: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: In der Konsole: 
    apt-get install atmel-firmware
  • Broadcom, bcm43xx/ b43/ b43 legacy wlan: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • Eagle USB ADSL модем: Микропрограммное обеспечение наидется здесь http://eagle-usb.org/ueagle-atm/non-free/ и сохраняется в: /lib/firmware/
  • Микропрограммное обеспечение DVB для различных карт телевидения DVB (большинство более простых карт не нуждаются в этом). Необходимое микропрограммное обеспечение (просмотреть dmesg) на http://www.linuxtv.org/downloads/firmware/ загружают и в /lib/firmware/ сохраняют.
  • Intel ipw2100, 11 MBit/s wlan: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan Please remove iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 and firmware-iwl4965, if already installed:.: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54-MBit/s-WLAN: микропрограммное обеспечение загружают с http://prism54.org/firmware/ и сохраняют в /lib/firmware/.
  • Графические карты nVidia: До сих пор трехмерное ускорение является невозможным со свободными драйверами. Однако при применении get-sidux-binary-gfx or sgfxi можно инсталлировать проприетарные драйверы.
  • RaLink rt61 and rt73 54 MBit/s wlan,: 
    apt-get install firmware-ralink

    Please remove rt61-fwcutter, rt73-fwcutter, firmware-rt61 and firmware-rt73, if already installed

  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) WLAN: микропрограммное обеспечение загружают с http://www.kazer.org/acx-firmware-20060207.tar.bz2 и сохраняют в /lib/firmware/
  • ZyDAS zd1201 11 MBit/s wlan: 
    apt-get install zd1201-firmware
  • ZyDAS zd1211 54 MBit/s wlan: 
    apt-get install zd1211-firmware

Мы исследуем также возможность совместно поставлять драйвера некоторых устройств. Законность использования этих драйверов часто, к сожалению, остается под вопросом.

Запись на NTFS раздел с помощью ntfs-3g

Внимание: Хотя драйвер ntfs-3g объявлен стабильным, мы рекомендуем его использование только с осуществлением предварительного копирования данных, и прежде всего не на "prodaction" платформах! Использование связано с риском!, потеря данных не исключена!

В консоль вводят следующее: Смотрите разбиение жесткого диска, для выяснения наименования разделов жесткого диска в Linux.

sidux@9[sidux]$ su
root@0[sidux]# apt-get update && apt-get install ntfs-3g
root@0[sidux]# umount /media/xdxx
root@0[sidux]# mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Zum Verlassen der Konsole gibt man nur ein: exit

Теперь присоединяем NTFS-разделс правом на запись.Теперь раздел NTFS доступен для сохранения данных. И еще раз наше предупреждение:Зтот образ действия упомянут для ситуаций, в которых нет никаких других возможных решений. Для ежедневного рутинного использования этот образ действий не рекомендован!
