Несвободные бинарные драйвера для: Nvidia/ATI

Имеются 3 метода инсталлировать 3D-драйвер NVIDIA/ATI

  1. sgfxi / smxi

    Use the sidux community contributed multi-function administration tool, smxi/sgfxi, which is short for sidux-maintenance. To read more refer to smxi and sgfxi Syntax Command Sets.

  2. Использование нативного инсталлятора NVIDIA/ATI 
    Инсталлятор непосредственно загружается с сайта NVIDIA или ATI.

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel

Open Source Xorg drivers for nVidia, ATI, Radeon, Intel are pre-installed with sidux. If you have overwitten them with closed source drivers you can revert or roll back to them with sgfxi

Драйвера X-Сервера,предоставляемые X.Org (смотри xserver-xorg для более подробной информации) обеспечивают поддержку 2D для следующих карт: NVIDIA Riva-, TNT-, GeForce- и Quadro-карты, Rage-, Radeon- а также карты серии FireGL ATI, включая 'r128' и 'radeon'. Для Intel включена поддержка 3D.

Для установки:

выйти из KDE одновременным нажатием Ctrl+Alt+F1 в текстовую консоль и зарегестрироваться в системе суперпользователем root
выполнить команду "init 3"
(приводящую к остановке X-сервера)
apt-get update
apt-get install xserver-xorg-video-nv
или
apt-get install xserver-xorg-video-ati
или
xserver-xorg-video-intel
init 5 && exit

For some cards it may add 3D support, however it is usually 2D.

Теперь необходимо отредактировать /etc/X11/xorg.conf. Предварительно создайте резервную копию, иначе сохранение ошибочно отредактированного /etc/X11/xorg.conf принесет много проблем и потребует много времени на исправление.

Откройте для редактирования /etc/X11/xorg.conf  с правами root, посредством щелчка мыши. В разделе SECTION DEVICE изменить драйвер дисплея на nv или ati.

Осуществление данных действий связано сопределенным риском!

Указания к аппаратным средствам, которые нуждаются в проприетарных модулях

В соответствиии с нормами права применительно лицензионного програмного обеспечения на sidux live-cd включает только ПО совместимое с dfsg-free.

Sources List

Поэтому необходимо раскомментировать строку содержащую contrib/non-free в /etc/apt/sources.list.d/debian.list or /etc/apt/sources.list.

# See sources.list(5) for more information, especially
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.
# The /etc/apt/sources.list.d/ directory provides a way to add sources.list
# entries in separate files that end with sources.list file.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
Video Cards
  • Графические карты ATI-Radeon: трехмерное ускорение для более старых карт до r35x должно функционировать. Более новые карты Radeon X1xxx нуждаются в независимых модулях, чтобы функционировать с 3D-ускорением. Эти модули еще не совместимы до конца с X.org 7.2.
Firmware
  • To help detect what firmware you need to install, run, as user:
$ fw-detect

This will give you a list of recommendations and what commands to run. It will return 'null' if you do not need anything, or firmware is already installed.

    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: в консоли: 
    apt-get install atmel-firmware
  • Broadcom, bcm43xx/ b43/ b43 legacy wlan: в консоли: 
    apt-get install bcm43xx-fwcutter b43-fwcutter
  • Broadcom Blutonium Bluetooth chipset (BCM2033): 
    apt-get install bluez-firmware
  • Eagle USB ADSL модем: Микропрограммное обеспечение наидется здесь http://eagle-usb.org/ueagle-atm/non-free/ и сохраняется в: /lib/firmware/
  • Микропрограммное обеспечение DVB для различных карт телевидения DVB (большинство более простых карт не нуждаются в этом). Необходимое микропрограммное обеспечение (просмотреть dmesg) на http://www.linuxtv.org/downloads/firmware/ загружают и в /lib/firmware/ сохраняют.
  • IVTV firmware:fetch the firmware from http://dl.ivtvdriver.org/ivtv/firmware/ and place it under/lib/firmware/.
  • Intel ipw2100, 11 MBit/s wlan: в консоли: 
    apt-get install ipw2100-firmware
  • Intel ipw2200, 54 MBit/s wlan: в консоли: 
    apt-get install ipw2200-firmware
  • Intel ipw3945 (54 Mbit/s wlan)/ ipw 4965 (draft-n) iwlwifi, 54 Mbit/s wlan, Пожалуйста удалите iwlwifi-ucode, iwlwifi-3945, firmware-iwl3945 and firmware-iwl4965, если уже установлены: 
    apt-get install firmware-iwlwifi
  • Intel ipw4965/ iwlwifi, draft-n wlan: 
    apt-get install firmware-iwlwifi
  • Intersil prism54, 54-MBit/s-WLAN: микропрограммное обеспечение загружают с http://prism54.org/firmware/ и сохраняют в /lib/firmware/.
  • Графические карты NVIDIA: До сих пор трехмерное ускорение является невозможным со свободными драйверами. Однако при применении get-sidux-binary-gfx or sgfxi можно инсталлировать проприетарные драйверы.
  • RaLink rt61 and rt73 54 MBit/s wlan,в консоли: 
    apt-get install firmware-ralink

    Пожалуйста удалите rt61-fwcutter, rt73-fwcutter, firmware-rt61 and firmware-rt73, если уже установлены.

  • Ricoh R5U870 OEM webcams:fetch the firmware from http://mediati.org/r5u870/r5u870-0.11.0.tar.gz and copy the r5u870_*.fw files to /lib/firmware/.
  • Texas Instruments ACX100 (22 Mbit/s)/ ACX111 (54 MBit/s) wlan:в консоли: 
    m-a a-i acx100
    микропрограммное обеспечение загружают с http://www.kazer.org/acx-firmware-20060207.tar.bz2 и сохраняют в /lib/firmware/
  • ZyDAS zd1201 11 MBit/s wlan: Fetch the firmware from http://surfnet.dl.sourceforge.net/sourceforge/linux-lc100020/zd1201-0.14-fw.tar.gz and place it under /lib/firmware/.
  • ZyDAS zd1211 54 MBit/s wlan:в консоли: 
    apt-get install zd1211-firmware

Запись на NTFS раздел с помощью ntfs-3g

Внимание: Хотя драйвер ntfs-3g объявлен стабильным, мы рекомендуем его использование только с осуществлением предварительного копирования данных, и прежде всего не на "prodaction" платформах! Использование связано с риском!, потеря данных не исключена!

В консоль вводят следующее: Смотрите разбиение жесткого диска, для выяснения наименования разделов жесткого диска в Linux.

sidux@9[sidux]$ su
root@0[sidux]# apt-get update && apt-get install ntfs-3g
root@0[sidux]# umount /media/xdxx
root@0[sidux]# mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Zum Verlassen der Konsole gibt man nur ein: exit

Теперь присоединяем NTFS-разделс правом на запись.Теперь раздел NTFS доступен для сохранения данных. И еще раз наше предупреждение:Зтот образ действия упомянут для ситуаций, в которых нет никаких других возможных решений. Для ежедневного рутинного использования этот образ действий не рекомендован!
