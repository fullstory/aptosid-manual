Трехмерный драйвер для графических карт nVidia

Имеются 3 метода инсталлировать трехмерный драйвер nVidia

  1. install-binary-gfx

    Новый скрипт для инсталляции графических драйверов карты называется: install-binary-gfx. Чтобы инсталлировать скрипт, команда get-sidux-binary-gfx должна запускаться как root в консоли.

    get-sidux-binary-gfx
install-binary-gfx -a (Применение исходного кода из репозитива sid)

    Вместе с тем выбирается правильный драйвер для графической карты, строится и инсталлируется инсталляционный файл Debian .deb. Также автоматически приспосабливается файл конфигурации /etc/X11/xorg.conf . Чтобы новый драйвер смог активироваться, графический сервер X должен перезапускаться (Strg+Alt+клавиша возврата).

    Опции

    Следующие опции могут ставиться:

     Usage: ./install-binary-gfx [options]
-a | --auto            автоматический способ
-b | --build           только строительство MODs модуля (nvidia|nvidia-legacy|fglrx)
-d | --detect          только графическое узнавание карт
-u | --update           скрипт модернизированной версии
-v | --verbose         с ответом

    Примеры

    install-binary-gfx -a (это устанавливает графическую карту и инсталлирует автоматически необходимые драйверы из репозитива sid)
install-binary-gfx -b nvidia (вместе с тем строится только модуль для nvidia, файл deb в /usr/src. Это же самое относится для fglrx (ATI) и nvidia-legacy)
install-binary-gfx -d (это устанавливает графическую карту и выдает сведения о ней)
install-binary-gfx -u (вместе с этим скрипт обновляется. )

    Помощь при проблемах

    Если скрипт не инсталлирует графические драйверы, может использоваться способ debug. Он производит файл регистрации /var/log/sidux-CARD-installer.log. Для получения помощи, может быть содержание этого фай­ла опубликован в форуме sidux, или его сохраняют на http://paste.debian.net и обращаются к IRC #sidux, где произведенная ссылка может указываться.

  2. sgfxi / du-fixes-h2.sh

    cd /usr/local/bin ; wget -Nc http://techpatterns.com/sgfxi
chmod +x sgfxi

    sgfxi поддерживает в на­тоящий момент ATI fglrx и Nvidia драйвер. Также конверсия с или на свободные драйверы xorg поддерживается как ati, intel или nv. du-fixes-h2.sh использует sgfxi как installationsbackend для графического драйвера и добавляет несколько опций к более простому использованию.

    sgfxi-h перечисляет все опции. Опции в течение времени дополняются. Также показываются с sgfxi-h в данный момент поддержанные драйверы также спецификации используемой системы как ядро и xorg версия. Драйвер по умолчанию инсталлируется, если никакие дальнейшие опции не указываются для инсталляции. Также скрипт автоматически определяет более старые графические карты и инсталлирует сразу драйвер Legacy.

    После старта скрипта и узнавания графической карты выдается, какой драйвер должен инсталлироваться. При этом имеют возможность подтверждать этот выбор и инсталлировать драйвер или отменять инсталляционный процесс и перезапускать скрипт с выбранной опцией.

    Или: используют h2-fixes-du.sh для инсталляции графических драйверов: du-fixes-h2.sh. Чтобы инсталлировать только 3D драйвер, есть команда:

    du-fixes-h2.sh -kiwdt

Nativ драйвер "nv" от X.Org для графических карт nVidia

Этот драйвер X сервера от X.Org (смотри xserver-xorg для более подробного описания) предлагает поддержку для карт nVIDIA Riva, TNT, GeForce и Quadro.

Он инсталлируется с:

Из KDE выйти с Ctrl+Alt+F1.
Зарегистрироваться как root.
Затем запускать: 
# init 3
(Сообщение должно было показывать, что сервер X остановился)
apt-get update
apt-get install xserver-xorg-video-nv
init 5 && exit

Примечание: Это не является бинарным драйвером NVidia, который предлагает трехмерную поддержку. Однако, бинарный драйвер NVidia нужно получать исключительно бинарным и в принципе он не поддержан в Linux.

Теперь должен редактироваться /etc/X11/xorg.conf. Вы должны былы создавать раньше резервную копию, иначе ошибочное редактирование /etc/X11/xorg.conf принесет Вам много слез и потребуется много времени на отладку.

Чтобы редактировать /etc/X11/xorg.conf , Вы можете его открыть,  для этого нужны права root, посредством щелчка мышкой. В разделе SECTION DEVICE изменяют драйвер дисплея в nv.

Редактирование xorg.conf происходит на собственный риск.

Двойные мониторы

Так как сейчас файлы конфигурации от xorg изменяются, мы просим делать резервную копию файл xorg.conf, прежде чем следующий шаг осуществляется!

Все это может измениться, когда xorg 7.3 появляется в конце 2007.

Native драйвер ATI - метод 1

В форуме с тематикой http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=19794#19794 имеется некоторые примеры работы xorg.confs с свободтным драйвером radeon.

ОТМЕТЬТЕ: Чтобы получать сведения конфигурации второго монитора, первый должен сниматься и перезагружаться с Live-CD, чтобы позволять производить xorg.conf. Копия должна вкладываться им xorg.conf. Чтобы получать сведения конфигурации первого монитора, тот же самый метод применяется.Von dieser xorg.conf muss eine Kopie angelegt werden.

Native драйвер ATI - метод 2

Сначала должно разрешение первого монитора вкладываться. Второй монитор использует установки разрешения первого (первоначального) монитора. Замечено должен быть, что второй монитор технически может осуществлять установки разрешения первого.

в разделе:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

следующее должно вставляться:

Option "MergedFB"
Option "CRT2Position" "RightOf"

затем раздел выглядит как указано ниже:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Здесь имеются обширные сведения об установке конфигурации: http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html

ПРИМЕЧАНИЕ: Упомянутая ссылка использует драйвер Radeon, но установки были протестированы также на драйверах ATI, и они функционируют.

зависимый драйвер ATI

Сначала инсталлируется зависимый драйвер (fglrx).

get-sidux-gfx
или
sgfxi / du-fixes-h2.sh

Обратит внимание, пожалуйста, на разделах в руководстве sidux.

В связи с этим покидаться сервер X.

ctrl+alt+F1
root
[Ввод данных пароля] # не выводится
init 3

Затем запускать:

aticonfig --dtop=horizontal
init 5 && exit

Если расположение монитора обменяно:

aticonfig --dtop=horizontal,reverse

Для информации может читаться руководство, которое очень подробно и обширные опции и примеры предоставляет:

aticonfig --help

Обратите вниманию: оба экрана должны владеть тем же разрешением. Как правило, устанавливают наивысшее разрешение слабого монитора. Читайте, пожалуйста, спецификации мониторов в руководствах, если максимальное разрешение неизвестно.

В форуме с тематикой http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=20231#20231 можно найти более сообщении.

Чтобы возвращаться от конфигурации двойственного монитора к конфигурации с отдельным монитором, следующая команда должна вводиться:

aticonfig --initial --input=/etc/X11/xorg.conf
nvidia

Для завицимого драйвера nvidia может программа конфигурации nvidia использоваться для xorg ( http://www.sorgonet.com/linux/nv-online/) и установки могут производиться в нем. Auch hier ist zu beachten: die maximale Auflösung des schwächeren Monitors soll gewählt werden bzw. beide Monitore müssen auf die gleiche Auflösung eingestellt sein.

Указания к аппаратным средствам, которые нуждаются в зависимых модулях

На основании трудного правового статуса только свободное для DFSG программное обеспечение доставляет sidux на Live-CD.

Поэтому это может быть желаемо для пользователя добавить contrib/non-free к /etc/apt/sources.list:

# See sources.list(5) for more information, especialy
# Remember that you can only use http, ftp or file URIs
# CDROMs are managed through the apt-cdrom tool.

# Unstable
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

# sidux
deb http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
#deb-src http://sidux.com/debian/ sid main contrib non-free firmware fix.main fix.contrib fix.non-free
  • Графические карты ATI-Radeon: трехмерное ускорение для более старых карт до r35x должно функционировать, более новые Radeon карты X1xxx нуждаются в независимых модулях, чтобы функционировать с 3D-ускорением. Зависимый модуль может инсталлироваться с sidux скриптом get-sidux-binary-gfx.
  • Atheros/ "madwifi" wlan: в консоли: 
    m-a a-i madwifi
  • Atmel AT76c50x 11 MBit/s wlan: в консоли: 
    apt-get install atmel-firmware
  •  Карты AVM ISDN/ADSL PCI/USB: AVM драйверы закрытого исходного программного обеспечения являются с ядром 2.6.19.x сейчас не совместимы. Загружайте, пожалуйста, ядро 2.6.18,x (http://sidux.net/files/kernel) и в консоли инсталлируйте драйвер: 
    m-a a-i avm
  • Broadcom/bcm43xx wlan: в консоли:

Разделы NTFS с ntfs-3g делают фиксирующимися

Внимание: ntfs-3g является ранний бетой модуль. Перед тем как использовать нужно, сохранить все данные, прежде всего, это предусмотрено не для рабочих платформ!! Использование на собственный риск, потеря данных не исключена!

В консоль вводят следующее: Смотрите разбиение жесткого диска, для выяснения наименования разделов жесткого диска в Linux.

umount /dev/XdXX
mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX
exit
exit

Теперь NTFS-Volume должен  быть присоединен с письменным доступом, и данные могут быть там сохранены.

Последний раз указание: это - экспериментальный метод, при применении нужно взвешивать риск!
