Binary, non-free drivers for: Nvidia / ATI

Имеются 3 метода инсталлировать трехмерный драйвер nVidia/ATI

  1. install-binary-gfx

    Новый скрипт для установки драйверов графических карты называется: install-binary-gfx. Чтобы инсталлировать скрипт, команда get-sidux-binary-gfx должна запускаться как root в консоли.

    get-sidux-binary-gfx
install-binary-gfx -a (Применение исходного кода из репозитива sid)

    Вместе с тем выбирается правильный драйвер для графической карты, строится и инсталлируется инсталляционный файл Debian .deb. Также автоматически приспосабливается файл конфигурации /etc/X11/xorg.conf. Чтобы новый драйвер смог активироваться, графический сервер X должен перезапускаться (с помощью комбинации Strg+Alt+клавиша возврата).

    Опции

    Следующие опции могут ставиться:

     Usage: ./install-binary-gfx [options]
-a | --auto            автоматический способ
-b | --build           только строительство MODs модуля (nvidia|nvidia-legacy|ati|ati-legacy|fglrx)
-d | --detect          олько графическое узнавание карт
-u | --update          скрипт модернизированной версии
-v | --verbose         с ответом

    Примеры

    install-binary-gfx -a (это определяет графическую карту и устанавливает автоматически необходимые драйверы из репозитива sid)
install-binary-gfx -b nvidia (вместе с тем строится только модуль для nVidia, файл deb находится в /usr/src. Это же самое относится для fglrx (ATI) и nvidia-legacy)
install-binary-gfx -d (это устанавливает графическую карту и выдает сведения о ней)
install-binary-gfx -u (вместе с этим скрипт обновляется. )

    Помощь при проблемах

    Если скрипт не инсталлирует графические драйверы, может использоваться способ debug. Он производит файл регистрации /var/log/sidux-CARD-installer.log. Для получения помощи, может быть содержание этого фай­ла опубликован в форуме sidux, или его сохраняют на http://paste.debian.net и обращаются к IRC #sidux, где произведенная ссылка может указываться.

  2. sgfxi / du-fixes-h2.sh

    apt-get update
apt-get install sidux-scripts
sgfxi

    sgfxi поддерживает в натоящий момент ATI fglrx и nVidia драйверы. Также конверсия с или на свободные драйверы xorg поддерживается как ati, intel или nv. du-fixes-h2.sh использует sgfxi как installationsbackend для графического драйвера и добавляет несколько опций к более простому использованию.

    Драйвер по умолчанию инсталлируется, если никакие дальнейшие опции не указываются для инсталляции. Также скрипт автоматически определяет более старые графические карты и инсталлирует сразу драйвер Legacy.

    После старта скрипта и узнавания графической карты выдается, какой драйвер должен инсталлироваться. При этом имеют возможность подтверждать этот выбор и инсталлировать драйвер или отменять инсталляционный процесс и перезапускать скрипт с выбранной опцией.

    ИЛИ: используют h2-fixes-du.sh для установки графических драйверов: du-fixes-h2.sh. Чтобы инсталлировать только 3D драйвер, есть команда:

    du-fixes-h2.sh -kiwdt

    sgfxi-h перечисляет все опции. Опции в течение времени дополняются. Также показываются с sgfxi-h в данный момент поддержанные драйверы также спецификации используемой системы как ядро и xorg версия. Here are some useful sgfxi options:

    -c Use composite mode (nVidia cards only).
-d Remove IgnoreDisplayDevices, allow TV for example (nVidia cards only).
-h View this help menu.
-n Automatically installs correct native xorg nv, intel, or ati driver for your system.
   Cleans up old binary driver stuff, updates xorg.conf with new driver information,
   and cleans nvidia and fgrlx stuff out of xorg.conf
-N Force install of specific xorg driver. 
Syntax: sgfxi -N <one of the drivers from the list, like -N radeon>
   Supported Xorg drivers: ati i128 i740 i810 i810-modesetting intel mga neomagic nv radeon sis tdfx vesa vga voodoo
   Also configures xorg.conf as required. Full radeon xorg 3d configuration automatically.
-o Override default: install a specific driver, this is an absolute override - syntax:
   -o 9746 (for nVidia, do not use the 1.0- part) or: -o 8.32.5 (for fglrx)
-r Remove mode lines from xorg.conf
-t Enables dynamic twinview (nVidia only)

Native драйвер ATI - метод 1

В форуме с тематикой http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=19794#19794 имеется некоторые примеры работы xorg.confs с свободтным драйвером radeon.

ОТМЕТЬТЕ: Чтобы получать сведения конфигурации второго монитора, первый монитор должен сниматься и перезагружаться с Live-CD, чтобы позволять производить xorg.conf. Копия должна вкладываться им xorg.conf. Чтобы получать сведения конфигурации первого монитора, тот же самый метод применяется.

Native драйвер ATI - метод 2

Сначала должно разрешение первого монитора вкладываться. Второй монитор использует установки разрешения первого (первоначального) монитора. Замечено должен быть, что второй монитор технически может осуществлять установки разрешения первого.

в разделе:

Section "Device"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

следующее должно вставляться:

Option "MergedFB"
Option "CRT2Position" "RightOf"

Затем раздел выглядит как указано ниже:

Section "Device"
	Option "MergedFB"
	Option "CRT2Position" "RightOf"
	Identifier "devname"
	Driver "ati"
   ...
EndSection

Здесь имеются обширные сведения об установке конфигурации: http://ftp.x.org/pub/X11R6.9.0/doc/html/radeon.4.html

ПРИМЕЧАНИЕ: Упомянутая ссылка использует драйвер Radeon, но установки были протестированы также на драйверах ATI, и они функционируют.

Зависимый драйвер ATI

Сначала инсталлируется зависимый драйвер (fglrx).

get-sidux-gfx
или
sgfxi / du-fixes-h2.sh

Обратите внимание, пожалуйста, на разделах в руководстве sidux.

В связи с этим покидаться сервер X.

ctrl+alt+F1
root
[Ввод данных пароля] # не выводится
init 3

Затем запускать:

aticonfig --dtop=horizontal
init 5 && exit

Если расположение монитора обменяно:

aticonfig --dtop=horizontal,reverse

Для информации можно читать руководство, которое очень подробно и обширные опции и примеры предоставляет:

aticonfig --help

Обратите вниманию: оба экрана должны владеть тем же разрешением. Как правило, устанавливают наивысшее разрешение слабого монитора. Читайте, пожалуйста, спецификации мониторов в руководствах, если максимальное разрешение неизвестно.

В форуме с тематикой http://sidux.com/index.php?name=PNphpBB2&file=viewtopic&p=20231#20231 можно найти более сообщении.

Чтобы возвращаться от конфигурации двойственного монитора к конфигурации с отдельным монитором, следующая команда должна вводиться:

aticonfig --initial --input=/etc/X11/xorg.conf

Разделы NTFS с ntfs-3g делают фиксирующимися

Внимание: Хотя драйвер ntfs-3g объяснен теперь стабильным, мы рекомендуем никакое использование без ресерв-копирования данных, прежде всего, не на рабочих платформах! Использование на собственный риск, потеря данных не исключена!

В консоль вводят следующее: Смотрите разбиение жесткого диска, для выяснения наименования разделов жесткого диска в Linux.

sidux@9[sidux]$ su
root@0[sidux]# apt-get update && apt-get install ntfs-3g
root@0[sidux]# umount /media/xdxx
root@0[sidux]# mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
Zum Verlassen der Konsole gibt man nur ein: exit

Теперь NTFS-раздел должен  быть присоединен с письменным доступом, и данные могут быть там сохранены. И еще раз наше предупреждение:Зтот образ действия упомянут для ситуаций, для которых не имеется никаких других возможностей решения. Для ежедневного рутинного использования этот образ действий не рекомендован!
