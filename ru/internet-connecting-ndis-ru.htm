Активизация ndiswrapper

sidux имеет уже все необходимые пакеты (модули ядра и ndiswrapper-utils) на жестком диске.

apt-get update
apt-get install ndiswrapper-modules-2.6-sidux-i686 (or ndiswrapper-modules-2.6-sidux-amd64)
Do it yourself

Find a suitable driver for your card and install it using the command:

ndiswrapper -i /pfad/zu/driver.inf

для проверки успешности инсталляции используйте опцию -l:

root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Отсутствие строк (или одной из) "driver installed" и "device present" в приведенном выводе говорит о произошедшей ошибке и невозможности установить соединение wlan.

Загрузка модуля Ndiswrapper:

modprobe ndiswrapper

Проверка правильности инсталляции:

iwconfig

Проверьте, был ли найден Access-Point:

iwlist wlan0 scan

Затем Вы можно приступить к конфигурированию устройства (Device):

ceni

При обновлении ядра системы, потребуется реинсталлировать и ndiswrapper. А также изменить описание ядра(номер) в соответствии с текущим:

dpkg -i /usr/src/kernel-downloads/2.6.21.5-slh-smp-2/ndiswrapper-modules-2.6.21.5-slh-smp-2_1.46-0+c0.sidux.1+1_i386.deb

И снова загрузить модуль с modprobe:

modprobe ndiswrapper

Переустанавливать windows-драйвер нет необходимости, т.к. он не является модулем ядра.

Для автоматической загрузки модуля ndiswrapper при старте системы необходимо внести в /etc/modules следующие изменения:

grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules

You can also do this with the GUI control centre siduxcc in KDE-Start-Menu

Ndiswrapper Advanced

Ndiswrapper - это модуль, который позволяет Linux использовать Windows WLAN-драйвера.The ndiswrapper is a module that enables Linux to use Windows WLAN drivers. Это необходимо, поскольку некоторые производители не предоставляют исходный код или сам драйвер для Linux, что крайне осложняет использование соответствующей карты. Ndiswrapper предоставляет обертку ("wrapping") к Windows драйверу и транслирует ввод-вывод между картой и Linux.

Определение необходимых драйверов

Прежде всего, вам потребуется надлежащее windows драйвера для вашей карты WLAN. WLAN карточки имеют множество различных наборов микросхем для одной модели и единственный способ определить вашу карту точно является id-идентификатор устройства (USBID/PCIID).

Откройте терминал:

Для адаптора USB WLAN
bla@blubb:~$ lsusb
Bus 004 Device 002: ID 2001:3700 D-Link Corp. [hex] DWL-122 802.11b
Bus 001 Device 001: ID 0000:0000
Bus 002 Device 003: ID 0db0:6982 Micro Star International Medion Flash XL V2.7A Card Reader
bla@blubb:~$

Как Вы можете видеть здесь, USBID для D-Link Corp. [hex] DWL-122 адаптора имеет номер 2001:3700.

Для (mini) PCI / PCMCIA WLAN адаптора
bla@blubb:~$ lspci -nn
00:1f.3 SMBus [0c05]: Intel Corporation 82801G (ICH7 Family) SMBus Controller [8086:27da] (rev 02)
01:00.0 Ethernet controller [0200]: Marvell Technology Group Ltd. 88E8053 PCI-E Gigabit Ethernet Controller [11ab:4362] (rev 22)
02:00.0 Network controller [0280]: Atheros Communications, Inc. Unknown device [168c:0024] (rev 01)
03:03.0 FireWire (IEEE 1394) [0c00]: Agere Systems FW323 [11c1:5811] (rev 61)

Этот 'id' имеет обозначение, соответственно схеме производитель:устройство, конкретно в примере 168c:0024 для карты Atheros.

Займемся поиском: ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,list записи для соответствующего 'id', если повезет, Вы найдете соответствующую запись и рекомендации для драйвера.

От данного драйвера Вам потребуются INF и SYS файлы. В идеальном случае Вы получили ZIP файл, который можно извлечь с unzip. Если файл будет с расширением EXE - самораспаковывающийся архив - используйте wine, в случае с файлом CAB распаковывайте с cabextract. Самый простой способ заключается в том, чтобы сохранить все файлы в новую папку в Вашем home.

Активизация Ndiswrapper и инсталляция драйвера

Теперь перейдем в дирректорию, содержащую файлы windows драйвера:

bla@blubb:~$cd /folder/with/driver
bla@blubb:~$su
root@blubb:/folder/with/driver# ndiswrapper -i <driver.inf>
installing net5416 ...
root@blubb:/folder/with/driver# ndiswrapper -l
installed drivers:
net5416 : driver installed
 device (168C:0024) present

Отсутствие строк (или одной из) "driver installed" и "device present" в приведенном выводе говорит о произошедшей ошибке и невозможности установить соединение wlan.

Загружаем модуль ядра: 

root@blubb:/some/folder# modprobe ndiswrapper

Если вы не получите здесь вывода,все прошло хорошо, если же будут выведены какието сообщения, используйте dmesg для получения дополнительной информации.

"iwconfig" теперь выведет список устройств wireless:

root@blubb:~# iwconfig
lo        no wireless extensions.

eth0      no wireless extensions.

lan0      no wireless extensions.

tun0      no wireless extensions.

wlan0     IEEE 802.11b  ESSID:"ConnectionPoint"
          Mode:Managed  Frequency:2.457 GHz  Access Point: 00:01:E3:0A:93:1B
          Bit Rate=11 Mb/s
          Encryption key:off
          Power Management:off
          Link Quality:0  Signal level:0  Noise level:0
          Rx invalid nwid:0  Rx invalid crypt:0  Rx invalid frag:0
          Tx excessive retries:0  Invalid misc:0   Missed beacon:0

Если необходимо стартовать ndiswrapper при загрузке, просто впишите ndiswrapper в /etc/modules, руками или с помощью: 

root@blubb:~# grep -q ^ndiswrapper /etc/modules|| echo ndiswrapper >> /etc/modules
переустановка Ndiswrapper после обновления ядра

После обновления ядра необходимо переустановить и модуль ndiswrapper. 

apt-get update
apt-get install ndiswrapper-modules-2.6-sidux-i686 (or ndiswrapper-modules-2.6-sidux-amd64)

После инсталляции загрузите модуль:

root@blubb:/some/folder# modprobe ndiswrapper

Драйвер windows не нуждается в переустановке, т.к. не является модулем ядра.

Общая информация

To configure your new device use Ceni

Подробное how-to можно найти здесь here .

Если вы найдете другой / новый драйвер пожалуйста протестируйте его тщательным образом. Это может вызвать проблемы, kernel panic в худшем случае, но этим вы можете помочь проекту ndiswrapper и другим пользователям воспользоваться лучшим драйверами для различных карт. Пожалуйста не забывайте обновить/дополнить ndiswrapper list after a successful test.
