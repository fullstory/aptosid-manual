Подключение к интернету и конфигурация сетевой карты

Если в локальной сети работает сервер DHCP,  при загрузке с LiveCD сетевые настройки определяются автоматически. Если этого не произошло, нужно запустить sidux-конфигуратор сетевой карты: KMenu > sidux> сеть/интернет > конфигурация сетевой карты.

Осуществить необходимые настройки возможно и из консоли:

netcardconfig

После чего ввести данные, соответствующие настройкам для вашей сети.

При инсталляции на жестком диске также возможны два варианта, с помощью графического конфигуратора:'Kmenu>sidux>Network/Internet>Networkcard configuration', или в командной строке после выполнения (как root):

/usr/sbin/netcardconfig

Далее следуйте появляющимся приглашениям.

Компьютер с 2 сетевыми картами и совместное использование интернета

Простое решение (пример) для настройки интернет-шлюза на компьютере с двумя сетевыми картами:

  • Откройте консоль, выполните "sux" регистрации пользователем root и выполнения следующих шагов:

  • Конфигурирование первой карты с:

    /usr/sbin/netcardconfig

  • Для 2 карты мы выбираем статический IP-адрес из "частного диапазона", например, 192.168.1.1, и конфигурируем его:

    /usr/sbin/netcardconfig

    Внимание: Ваш IP должен быть в той же самой субсети как IP-адреса, которые выделяются посредством DHCP. Для устранения ошибочной конфигурации межсетевого шлюза необходимо выполнить скрипт:

    remove-gateway

  • Установка сервера DHCP/DNS:

    apt-get install dnsmasq

    Редактируйте файл/etc/dnsmasq.conf добавлением такой строки как

    dhcp-range=192.168.1.50,192.168.1.150

    Остаток файла остается без изменений.

  • Теперь сервер DHCP настольного компьютера можно запустить. Если клиенты настроены на получение сетевого адреса по DHCP, он будет автоматически настроен из указанного выше диапазона.

    /etc/init.d/dnsmasq start

  • Для организации правильной маршрутизации в сети, использующей "маскарадинг", Вы должны запустить следующий скрипт:

    /usr/sbin/masquerade

    После указанных действий клиенты должны получить долступ к внешней сети (интернет).

  • Чтобы при загрузке настольного компьютера эти настройки восстанавливались, активируют dnsmasq в процессе загрузки с:

    update-rc.d -f dnsmasq defaults

  • И таже активируем запуск мasquerade в процессе загрузки:

    echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Модемное (56K) подключение

KDE имеет Frontend (графический интерфейс) для коммутируемого удалённого доступа по модему: KPPP (Internet Dial-up Tool). Это находится в K-Menu > Internet.

Приложение имеет встроенную справку и предлагает вполне понятное руководство к созданию соединения.

Firewalls

Фаервол обычно не требуется , если подключение происходит через правильно настроенный роутер, однако он играет очень важную роль в безопасности, если подключение к интернет будет осуществляться через adsl-модем или dial-up:

apt-get install guarddog
ODER
apt-get install firestarter

Guarddog дружественная пользователю утилита настройки фаервола для KDE. Лучшим способом ознакомиться станет прочтение короткого встроенного руководства.

Firestarter - графический интерфейс фаервола для Linux. [После того, как Вы выйдите из программы, настроенный фаервол будет продолжать работать в фоне.]
