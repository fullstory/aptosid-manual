Подключение к интернету и конфигурация сетевой карты

Если в LAN, локальной сети, работает сервер DHCP, должен бы LiveCD при загрузке автоматически с ним связаться. Если этого не произошло, нужно запустить sidux конфигурации сетевой карты, его находят в : KMenü > sidux> сеть/интернет > конфигурация сетевой карты.

Если этот путь в графической оболочке не подходит, можно свободно использовать консоль:

sudo
netcardconfig

Теперь нужно следовать только лишь за командами.

Активизация ndiswrapper

sidux имеет уже все необходимые пакеты (модули ядра и ndiswrapper-utils) на жестком диске.

Вы должены найти только лишь файл .inf для Windows (драйвер) и его загрузить:

ndiswrapper -i /pfad/zu/driver.inf

Загружайте модуль Ndiswrapper:

modprobe ndiswrapper

и проверьте, инсталлирован ли он правильно:

iwconfig
iwconfig wlan0 essid ESSIDNAME

Проверьте, был ли найден Access-Point:

iwlist wlan0 scan

Затем ты можешь конфигурировать устройство (Device) с:

netcardconfig

Компьютер с 2 сетевыми картами и совместное использование интернета

Как я конфигурирую персональный компьютер, что бы с ноутбуком (или другим компьютером) выйти в интернет Мы показываем здесь простое минимальное решение:

  • Откройте консоль и будь с "sux" root, для запуска следующих шагов:

  • Конфигурирование первой карты с:

    netcardconfig

  • Для 2 карты мы выбираем статический IP-Адрес в частной домене, например, 192.168.1.1, и конфигурируем также его:

    netcardconfig

    Внимание: Ваш IP должен быть в той же самой субсети как IP-адреса, которые выделяются посредством DHCP. Если не правильно установился межсетевой шлюз во время конфигурации, должен запуститься следующий скрипт:

    remove-gateway.sh

  • Установки DHCP/DNS:

    apt-get install dnsmasq

    Редактируйте файл/etc/dnsmasq.conf добавлением такой строки как

    dhcp-range=192.168.1.50,192.168.1.150

    Остаток файла остается таким , как он есть.

  • Теперь сервер DHCP настольного компьютера может стартовать. Если клиенты теперь установлены на DHCP, автоматически выделяются IP-адреса.

    /etc/init.d/dnsmasq start

  • Для запроса клиента к маршрутизации в сети, Вы должены запустить следующий скрипт:

    /usr/sbin/masquerade.sh

    Затем должен бы каждый клиент войти в интернет.

  • Чтобы загружать эту конфигурацию при загрузке настольного компьютера, активируют dnsmasq в процессе загрузки с:

    update-rc.d -f dnsmasq defaults

  • Затем Вы запускаете скрипт мasquerade:

    echo "/usr/sbin/masquerade" >> /etc/init.d/bootmisc.sh

Интернетсвязь с 56k Dial-up модемом

KDE имеет Frontend (графическое приложение) для Dial-up модемов: KPPP (Internet Dial-up Tool). Это находится в K-Menü > Internet.

Приложение имеет внутреннюю помощь и предлагает вполне понятное руководство к созданию связи.

Маленький брандмауэер

Если Вам нужно сприпт для брандмауэра используте этот скрипт:

wget http://rebelhomicide.demon.nl/scripts/mini-miyagi.sh
chmod 755 mini-miyagi.sh 
apt-get remove --purge {firestarter,guarddog}

Запускайте с

./mini-miyagi.sh

Um mini-miyagi zu stoppen

/etc/init.d/mini_miyagi_fw.sh stop
