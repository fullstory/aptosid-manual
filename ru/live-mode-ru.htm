Пароль суперпользователи root на Live-CD

Внимание: Когда всегда работают с правами root, нужно знать, что делать. Для навигации в интернете никакие права root не нужны.

На LiveCD никакой пароль root не установлен. Имеются несколько возможностей, запускать программу с правами root:

(Временный) пароль root устанавливают так:

Открывают консоль:

sidux@0[sidux]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

Этот пароль может использоваться до конца Live сеанса.

Программа как root запускается так

sudo
oder
su-me
Нажимают Alt+F2 и Enter и вводят"
sudo

перед желаемым именем программы, например:

sudo gparted
Программу из терминала root запускают так:

Открывают терминал / консоль и вводят, например:

sidux@0[sidux]$ su
root@0[sidux]# gparted ( (или желаемую программу)

Чтобы покинуть терминал, вводится:

exit

после этого еще раз:

exit
(вместе с этим терминал должен закрыться)

Installing software whilst on a Live-CD

apt-get update
apt-get install your-preferred-package

Note: when you power off the Live-CD, no changes will be kept

Разделы NTFS с ntfs-3g делают фиксирующимися

Внимание: модуль ntfs-3g является в ранней фазе беты. Перед тем как использовать нужно, сохранить все данные, прежде всего, это предусмотрено не для рабочих платформ! Использование на собственный риск, потеря данных не исключена!

В консоль вводят следующее: Смотрите разбиение жесткого диска, для выяснения наименования разделов жесткого диска в Linux.

sidux@9[sidux]$ su
root@0[sidux]# umount /dev/XdXX
root@0[sidux]# mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX

Теперь NTFS-Volume должен  быть присоединен с письменным доступом, и данные могут быть там сохранены. И еще раз наше предупреждение: этот образ действия классифицирован как экспериментальный!!!
