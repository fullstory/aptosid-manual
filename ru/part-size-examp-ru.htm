Размеры разделa: примеры

Для нормального употребления мы рекомендуем файловою систему ext3. Ext3 является стандартной файловой системой для sidux и при этом хорошо поддерживается (maintained).

Менеджер разделов gparted позволяет осуществить разбиение и форматирование жестких дисков. Приложение имеет графический и интуитивно понятный интерфейс

Gparted может уменьшать и соответственно перемещать, также разделы NTFS. Gparted еще не разработан полностью и текущая версия рассматривается как экспериментальная, однако исходя из опыта, может быть использован без проблем. Здесь обширная документация для GParted.. Изменения в разделах NTFS  также могут, проводится с патентованными приложениями (например, Partition Magic (TM), Acronis (TM)).

ВСЕГДА ДЕЛАЙТЕ РЕЗЕРВНОЕ КОПИРОВАНИЕ ВАЖНЫХ ДАННЫХ!

В случае, когда раздел оказывается смонтированным, необходимо выйти из gparted и отмонтировать соответствующий раздел. (Правый щелчок на символ устройства на рабочем столе -> откроет соответствующее меню), если раздел swap существует, операции с ним возможно осуществить в консоли:

sudo swapoff -a

Затем можно снова запускать gparted. В принципе, для инсталляции достаточно менеее 3 ГБ, но на практике для комфортной работы необходимо 5-8GB. Для новичка в Linux рекомендуют использовать в первый раз только 2 раздела (root и swap), так как это значительно упрощает процесс инсталляции. Но будет лучше организовать еще дополнительный раздел для /home.

Продвинутые пользователи могут выделить отдельные разделы так­же для /var, /tmp, ...и т.д. В этом руководстве мы не станем рассматривать подробно такую возможность, ограничившись вышесказанным. Также должен быть создан раздел swap (Соответствует в функциональности, например, Файлу подкачки в Windows, однако, гораздо эффективнее). В общем swap должен быть вдвое больше чем оперативная память, но не больше 500 МБ. Как стандартную файловою систему для Linux разделов использовать лучше всего ext3. ReiserFS имеет несколько преимуществ, но ее дальнейшее развитие остается неясным.

Для обмена данными с Windows выбор vfat (Fat32) является самым быстрым и несложным. Желающие использовать XFS для раздела/ (root), должны выделить в отдельный раздел /boot с файловой системой ext2 (так как поддержка xfs в Grub не достаточно надежна) или использовать загрузчик lilo. Программа sidux-installer в настоящее время позволяет использовать xfs на / только при редактировании файла ~/.siduxconf..

Создавая несколько разделов, запоминайте их обозначения (или запишите)! В конце изменений подтвердите свой выбор, дождитесь окончания всех операций и выйдите из gparted (символ дискет и соответственно Edit > Apply All Operations).

Имея в своем распоряжении больше места на жестком диске, этот раздел возможно увеличивать или разделить программы Windows и файлы на 2 раздела.

Общее примечание

Для sidux, без каких-нибудь особенных мультимедийных приложений или игр, 5 ГБ вполне достаточно, для данных создается раздел fat32. Кроме того нужно не забыть оставить 500 МБ для swap. Для разделения дисков существует большое количество удобных методов, но для первого раза должно быть достаточно приведенных примеров. Использование второго жесткого диска является рациональным для повышения эксплуатационных показателей, комфортности и надёжности или безопасности. При этом Windows всегда принадлежит первому жесткому диску!
