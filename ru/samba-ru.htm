Samba конфигурация, чтобы можно с sidux по сети работать с данными Windows

  • Все команды в терминале или консоли запускают как root.

  • URL в Konqueror выполняют как нормальный пользователь.

    server = имя сервера или IP компьютера Windows
    share = имя разрешения

В менеджере файлов KDE Konqueror вводят URL: smb://server или весь путь: smb://server/share

В консоли Вы можете видеть разрешения на сервере с:

smbclient -L server

Чтобы видеть разрешения в директории (с доступом для всех пользователей), не забывают : должен существовать пункт монтирования. Если нет, Вы должны создать директорию (имя любое):

mkdir -p /media/server_share

Затем монтируйте разрешение (файловая система-VFAT):

mount -t smbfs -o username=Administrator,fmask=777,dmask=777 //server/share /media/server_share

или при NTFS:

mount -t smbfs -o username=Administrator,fmask=777,dmask=777,lfs //server/share /media/server_share

Чтобы окончить связь:

umount /media/server_share

Если бы Вы хотели запись в /etc/fstab, чтобы упрощать монтирование, тогда дополняйте fstab строкой например:

//server/share /media/server_share smbfs defaults,username=dein_username,password=**********,fmask=777,dmask=777  0   0

sidux используют как сервер Samba

Samba не находится на LiveCD, поэтому необходимы следующие шаги, чтобы получать доступ к сети через Samba:

su
apt-get update
apt-get install samba
Инсталляция на жестком диске:
sux
apt-get update
apt-get install samba

На инсталляции жестких дисков конфигурация Samba должна приспосабливаться. В дальнейшем мы даем простой пример. Больше к использованию сервера Samba под Linux находят на страницах документации Samba..

Конфигурация Samba приспосабливается как указано ниже:

Как root файл /etc/samba/smb.conf открывается в редакторе таких как kedit или kwrite:

# Globale Changes - Proposal everything simple as possible - no passwords, perform like Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Разрешение без письменных прав - важно, если разрешаются NTFS разделы!

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/XXXX # <-- приспособьте это к Вашему разделу
public = yes

# Разрешение раздела с письменными правами -  раздел должен быть подмонтирован
# пишущийся  делает смысл, например, с Fat32

[DATA]
comment = Data Partition (first extended Partition)
browseable = yes
writable = yes
path = /media/hda5
public = yes

Не забывайте сохранить изменений. Samba запускается и останавливается с:

/etc/init.d/samba start

или

/etc/init.d/samba stop

Samba может запускаться также автоматически при загрузке:

update.rc.d samba defaults

Теперь Samba стартует при загрузке.
