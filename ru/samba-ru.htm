Использование Samba для работы с ресурсами Windows

  • Все команды в терминале или консоли выполняют как root.

  • URL в Konqueror открывают от имени обычного пользователя.

    server = имя сервера или IP компьютера Windows
    share = имя расшаренного файла

В менеджере файлов KDE Konqueror вводят URL: smb://server или весь путь: smb://server/share

В консоли для просмотра ресурсов, предоставляемых Windows, выполните:

smbclient -L server

Чтобы получить возможность работать в директории (с доступом для всех пользователей)используем существующую точку монтирования. или создаем заново, если таковая отсутствует (имя любое):

mkdir -p /media/server_share

Затем монтируем ресурс (файловая система-VFAT):

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777 //server/share /mnt/server_share

или NTFS:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777,lfs //server/share /mnt/server_share

Чтобы отмонтировать windows-ресурс:

umount /media/server_share

Для упрощения процедуры монтирования дополните fstab такой строкой:

//server/share  /mnt/server_share       cifs  defaults,username=your_username,password=**********,file_mode=0777,dir_mode=0777   0   0

sidux как сервер Samba

Пакет samba не содержится на LiveCD, поэтому необходимы следующие шаги, чтобы получать доступ к сети через Samba:

su
fix-unionfs
apt-get install samba
Инсталляция на жестком диске:
apt-get install samba

При существующей инсталляции (sidux) на жестком диске необходимо изменить конфигурацию Samba. Здесь будет рассмотрен простой пример. Больше об использовании сервера Samba под Linux можно найти на страницах документации Samba..

Конфигурация Samba должна быть изменена следующим образом:

Как root открываем файл /etc/samba/smb.conf в одном из редакторов, таких как kedit или kwrite:

# Globale Changes - Proposal everything simple as possible - no passwords, perform like Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Создание ресурса без права на запись - важно, если используются разделы NTFS !

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/XXXX # <-- приспособьте это к Вашему разделу
public = yes

# Создание ресурса с правом записи -  раздел должен быть смонтирован
# с разрешением на запись  , что имеет смысл в случае, например, с Fat32

[DATA]
comment = Data Partition (first extended Partition)
browseable = yes
writable = yes
path = /media/sda5
public = yes
Checking the shares in samba

To set the shares without regard to security in samba do the folowing commands (i.e. for a LAN setup):

Set folder and its content at least -rwxr-xr-x:

ls -la pathTo/dirShareName/*

If not, do:

chmod -R 755 pathTo/dirShareName

If you want it to be writable:

chmod -R 777 dirShareName

A way to make sure your share is working: ( do not forget to start the server):

smbclient -L localhost

Вы должны увидеть что-то вроде:

 smbclient -L localhost
Password:
Domain=[HOME] OS=[Unix] Server=[Samba 3.0.26a]

        Sharename       Type      Comment
        ---------       ----      -------
        IPC$            IPC       IPC Service (3.0.26a)
        MaShare         Disk      comment
        print$          Disk      Printer Drivers
Domain=[MSHOME] OS=[Unix] Server=[Samba 3.0.26a]

Если Вам не удалось установить пароль, просто нажмите ENTER.

Не забывайте сохранить изменения. (Samba должна быть запущена и остановлена):

/etc/init.d/samba start

или

/etc/init.d/samba stop

Samba может запускаться автоматически при загрузке:

update-rc.d samba defaults

Теперь Samba стартует при загрузке.
