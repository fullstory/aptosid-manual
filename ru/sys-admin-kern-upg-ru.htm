Обновление ядра

Ниже приведен список адресов, где расположены ядра, используемые в sidux. Пожалуйста, выберите ближайшее к вам зеркало:

http://debian.tu-bs.de/project/sidux//kernel Germany
http://sidux.com/files//kernel Germany
http://sidux.net/files//kernel Germany
http://www.mirrorservice.org/sites/sidux.com/sidux/kernel United Kingdom
http://ftp.heanet.ie/pub/sidux//kernel Ireland
http://sunsite.utk.edu/ftp/pub/linux/Sidux/kernel  TN, USA
ftp://ftp.spline.de/mirrors/sidux/kernel/ Germany
http://lug01.eecs.wsu.edu/sidux/kernel Pullman WA, USA
http://sidux.2wl.ru/downloads/kernel Russia
http://ftp.tw.debian.org/pub/Sidux//kernel Asia
http://mirror.aarnet.edu.au/pub/sidux/kernel Australia
http://ftp.leg.uct.ac.za/pub/linux/sidux/kernel South Africa
ftp://debian.co.nz/Sidux/kernel New Zealand

Ядра доступны в следующих формах:

  • smp - Ядро для процессоров архитектуры i686 моно/двух и более ядерных
  • 64-smp - ядро для процессоров AMD64, если используются sidux 64 bit

В дальнейшем наш архив ядра будет называться просто "kernel.zip", однако, действительное имя выглядит, например, таким образом:  kernel-2.6.22.3-rc1-slh-smp-2.zip). Ядро может быть инсталлированно в консольном режиме обычным пользователем,также как root-омp>

Пример установки обычным пользователем
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.22.3-rc1-slh-smp-2.zip
unzip kernel-2.6.22.3-rc1-slh-smp-2.zip -d kernel-2.6.22.3-rc1-slh-smp-2
cd kernel-2.6.22.3-rc1-slh-smp-2
./install-kernel-sidux.sh       # спрашивает о пароле root

Теперь необходимо перезапустить компьютер, и новое ядро будет выбрано автоматически после загрузки. Если использование нового ядра создает проблемы, снова перзапустите компьютер и в меню загрузки grub выбирете старое функционирующее ядро.

To check the modules loaded into the kernel:

ls /sys/module/
or
cat /proc/modules

ВАЖНО: Eсли необходимы определенные модули, например, для WLAN, то они должны инсталлироваться через пакеты deb, находящиеся в папке с распакованным kernel.zip. Модуль madwifi будет инсталлирован как root следующим образом:

dpkg -i madwifi-modules*.deb

Дополнительная информация getting on line and ndiswrapper и for WIFI

Обновление ядра завершено. Инсталляция 3D графический модуль для nVidia или ATI в случае необходимости, должна происходить перед запуском сервера X Наиболее просто загрузиться на runlevel 3 ("init 3"), при этом в командную строку загрузочного менеджера просто ввести3. Больше информации можно найти здесь.

Полуавтоматический метод установки нового ядра: скрипт du-fixes-h2.sh.now called smxi, являющийся сокращением от sidux-maintenance:
apt-get update
apt-get install sidux-scripts
smxi

Удаление старого ядра

После успешной установки нового ядра можно удалить kernel.zip и архив ядра, хотя рекомендуется его и более старое ядро сохранять ещё несколько дней, на случай непредвиденных проблем и необходимости перезагрузится с более старым ядром. Более старые ядра перечислены в меню менеджера загрузок Grub.

Старые ядра могут удаляться скриптом kernel-remover, который находится в пакете sidux-kernelhacking. Этот скрипт перечисляет те ядра, которые могут быть удалены.
