Обновление ядра

Самые свежие ядра можно найти здесь, которое скомпилировано в следующих форматах:
  • up - ядро для процессоров i686 с одним CPU
  • smp - ядро для процессоров i686 с 2 и более CPU
  • 64-smp - ядро для процессоров AMD64, если используются sidux 64 bit

В дальнейшем мы называем наш архив ядра просто "kernel.zip", однако, действительное имя называется, например, таким образом:  kernel-2.6.21.3-slh-up-2.zip). Ядро может инсталлироваться с помощью консоли как нормальный пользователь также как root:

Пример установки как пользователь в консоли
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.21.3-slh-up-2.zip
unzip kernel-2.6.21.3-slh-up-2.zip -d kernel-2.6.21.3-slh-up-2
cd kernel-2.6.21.3-slh-up-2
./install-kernel-sidux.sh       # спрашивает о пароле root

Теперь компьютер должен перезапускаться, и новое ядро будет автоматически выбрано. Если новое ядро делает проблемы, должен компьютеры перезапуститься, и в меню загрузки grub выбирается старое функционирующие ядро.

ВАЖНО: Eсли нуждаются в определенных модулях, например, для WLAN, то они должны инсталлироваться через пакеты deb, которые находятся в папке с распакованным kernel.zip. Модуль madwifi будет инсталлирован как root следующим образом:

dpkg -i madwifi-modules*.deb

Обновление ядра теперь завершено. Если должны инсталлироваться 3D графический модуль для nVidia или ATI, это должно происходить перед запуска сервера X Наиболее просто загружаются на runlevel 3 ("init 3"), при этом в командную строку загрузочного менеджера печатают просто 3. Большее количество сведений об этом здесь находятся.

Полуавтоматизированный метод установки нового ядра это скрыпт du-fixes-h2.sh.

Удаление старого ядра

После успешной установки нового ядра можно kernel.zip и архив ядра удалить, хотя рекомендуется, его и более старое ядро несколько дней ещё сохранять, если возникнут непредвиденные проблемы и с более старым ядром надо будет перезагрузится. Более старые ядра перечислены в меню менеджера загрузок Grub.

Старые ядра могут удаляться со скриптом kernel-remover, который находится в пакете sidux-kernelhacking. Этот скрипт перечисляет те ядра, которые могут удаляться.
