Инструкции для обновления установленной системы

sidux преследует простой путь обновления Debian пакетов и также sidux пакетов. Иногда является необходимым, ручно стабилизировать систему.

Эта страница будет обновлена с каждой новой версиею sidux. Pontos является текущей версиею.

Защитные рекомендации sidux / Закрытие каталога пользователя $HOME

При переработке инсталлятора sidux оказывалась проблема с потенциальным приобретением прав (локальное повышение привилегий). Копия конфигурационных файлов инсталлятора осталась на целевой системе. Этот файл конфигурации содержит MD5-хеш пароли для корневой и первой учетной записи пользователя (UID 1000). Пароли остаются разборчивыми для для местных пользователей. Несмотря на то, что копия непосредственно не разглашает пароли, информация можно использовать при атаке перебором.

Пострадавшие релизы:

  • 2007-01 Χάος (Chaos)
  • 2007-02 Τάρταρς (Tartaros)
  • 2007-03/ 2007-03.1 Γάια (Gaia)
  • 2007-04/ 2007-04.5 Έρως (Eros)
  • 2008-01 Νυξ (Nyx)
  • 2008-02 Έρεβος (Erebos)
  • 2008-03 Ουρέα (Ourea)

Исправление:

find /root /home -maxdepth 2 -type f -name '.sidconf*' -delete

В частности, для многопользовательных систем рекомендуется изменять пароли для root и первого пользователя (UID 1000).Причина для этой проблемы была выявлена и sidux 2008-04 "Πόντος" больше не затрагивается.

Debian создает каталоги с разрешениями доступа 0751, sidux отличается от этого по умолчанию и обеспечивает разрешение доступа с 0700 для новых установок.

sed -i 's/^DIR_MODE=.*/DIR_MODE=0700/' /etc/adduser.conf
dpkg-reconfigure adduser
("отклонить oбщесистемное чтение домашних каталогов")

Слой совместимости ALSA OSS

Начиная с alsa-driver 1.0.17.dfsg-3, модули совместимости OSS не автоматически загружаются. Хотя это не должно вызывать никаких проблем на современных систем Linux, некоторые коммерческе или устаревшие приложения нуждаются еще в поддержке OSS при определенных обстоятельствах. ALSA совместимые приложения должны бы предпочитаться, так как эмуляция ALSA для OSS ограниченa в ядре (никакой поддержки базирующихся на программное обеспечение микшеров, так как только одно приложение может одновременно действовать на /dev/dsp. Если совместимость OSS требуется, имеются 2 альтернативы:

  • aoss обертка
    aaoss является частью пакета alsa-oss и могут быть использованы в качестве обертки для унаследованных приложений. Это может вмешиваться в их вызовы OSS, чтобы осуществлять двоичный интерфейс приложений OSS прозрачно на ALSA. Это сохраняет обычно все преимущества ALSA.
  • Эмуляция ALSA, встроя в ядре
    Это позволяет лучшеe бинарную совместимость (эмулированные устройства /dev/dsp, /dev/audio и /dev/mixer имеются ), однако без программного микшера, так что только одно звуковое приложение может одновременно быть активно.

При наибольших затронутых программ является рациональным запускать приложения путом "aoss <приложение>". Таким образом преимущества программного микшера и вместе с тем одновременного доступа к звуковым аппаратным средствам сохраняются. Имеются, тем не менее, редкие случаи, в которых требуется обращение на слой совместимости OSS в ядре. В случае необходимости это можно активировать с помощью загрузки модулей snd-pcm-oss, snd-mixer-oss и snd-seq-midi.

for i in snd-pcm-oss snd-mixer-oss snd-seq-midi; do
	grep -q "^${i}" /etc/modules  || echo "${i}" >> /etc/modules
	grep -q "^${i}" /proc/modules || modprobe "${i}"
done

Безпроводные Broadcom b43 карты

Применение ядер актуальных чем >2.6.25 требуют микропрограммые версии ≥4.150.10.5 (внутренняя ревизия "410.2160 (2007-05-26 15:32:10)") с фирмы Broadcom. Старые ядар (до версии ядра 2.6.24) требуют версию 4.80.53.0 (внутренняя ревизия "351.126 (2006-07-29 05:54:02)"), которая былa установленa с помощью B43-fwcutter. Не возможно автоматически обновлять существующиe микропрограммz этим приложением. Обновление B43-fwcutter требует ручное вмешательство.

dpkg --purge b43-fwcutter
rm -rf /lib/firmware/b43 /lib/firmware/b43legacy
apt-get update
apt-get install b43-fwcutter

gspca2

В ядре 2.6.27 встроен новый gspca2 модуль на базе v4l2, который заменяет старый внешный gspca модуль на базе v4l1. Модуль используется в огромном количестве различных веб-камер. К сожалению, большинство пользователей используют космические программы, еще не новый API для v4l2 ранее обусловлен gspca веб-камеры, так что это означает вмешательство "LD_PRELOAD = / usr/lib/libv4l/v4l1compat.so <приложение>" должна быть запущена.

apt-get update
apt-get install libv4l-0

как пользователь
$ LD_PRELOAD=/usr/lib/libv4l/v4l1compat.so <приложение>

insserv

Применение insserv на существующую систему перед Pontos 2008-04

Предупреждение: Из-за наличия пользовательного скрипта Init или Symlinks инсталлируется insserv не автоматически на старые инсталляции sidux. НЕ рекомендуется установить insserv на инсталляции, выпушенные перед Pontos. Этот пакет должен быть использован с большой осторожностью. Неправильные или недостающие LSB зависимости имеющегося в наличии скрипта Init могут приводить к несамозагружаемой системе. Инсталляция insserv на существующую систему могут потребоваться некоторые вручные изменения.

Прочитайте Debian документацию и замечаниe о выпуске перед установкой и активацией insserv.

Устаревшие пакеты

В рамках дальнейшего развития sidux некоторые пакеты были заменены более альтернативee или становялись излишними.

  • capi-isdnconf, пакет был удален.
  • configure-xawtv, пакет был удален.
  • gprsconnect, пакет был заменен umtsmon.
  • ipw2100-firmware, пакет был заменен firmware-ipw2x00.
  • ipw2200-firmware, пакет был заменен firmware-ipw2x00.
  • modemlink, пакет был заменен umtsmon и wvdial.
  • ppp-scripts, пакет был заменен umtsmon.

Следующая команда сможет найти пакеты, которые не имеются в активном apt репозиторие. Будьте осторожным с вручнуюу установленными пакетами. apt-show-versions - это самостоятельный пакет и необязательно предустановлен.

$ apt-show-versions | awk '/No available version in archive/{print $1}'
