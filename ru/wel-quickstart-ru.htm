Краткое руководство sidux

Важное сообщение: sidux на ISO-образе очень сжат. По этой причине нужно обратить особое внимание на запись ISO. Мы рекомендуем для записи высококачественные CD (или DVD+RW), запись в модусе DAO-mode (disk-at-once) и не быстрее чем 8x.

sidux команда стремится держать sidux 100% совместимо с Debian. Тем не менее, в случае необходимости sidux издавает пакеты, которые временно заменить неисправные Debian пакеты. В sidux репозиторий находятся специализированные пакеты, такие, как актуальное sidux ядро, скрипты, утилиты и документация.

К стабильности Debian sid

Sid - это название нестабильных и тестовых архивов Debian. Debian sid имеет постоянно обновляемые репозитории, что позволяет производить быструю синхронизацию с последними и наиболее актуальными версиями программного обеспечения. Из-за частого обновления уменьшается общее время тестирование отдельных версий пакетов и сроки между релизом новой версии и распространением готовых пакетов Debian.

Ядро sidux

sidux ядро является оптимальным для большинства вариантов использования и конфигураций оборудования, что избавляет от многих проблем в работе, добавляет самые последние функциональные возможности, а также позволяет гибко настраивать все необходимые элементы для повышения стабильности и производительности самых последних ядер с http://www.kernel.org/.

Адреса зеркал официальных сборок ядер sidux можно найти здесь: Обновление ядра

Управление пакетами

sidux базируется на Debian правилах управления пакетов, касающиеся структуры и использует утилиты apt и dpkg. sidux и Debian репозиторий указанны в файлах /etc/sources.list.d/* .

Debian sid на данный момент имеет базу более 20000 пакетов. Вы можете без труда найти тот пакет, который Вы хотели бы использовать, с помощью простых в использовании утилит apt-cache или пойскового приложения Debian пакетов.

Чтобы установить новый пакет используйте команду apt-get install <название пакета>.  Подробнее об этом можно узнать на страничке руководства "Установка новых пакетов".

Debian sid's repositories can be updated as much as four times a day so running apt-get update is essential before installing any new packages or before running apt-get dist-upgrade to keep up-to-date with the server's list of packages.

Using other Debian based distributions' repositories, Source and RPMs

Инсталляции из первоначального кода не поддержаны. Если Вам действительно нужно скомпилировать Ваше приложение, сделайте это как пользователя, и поставьте его под свой домашний каталог, не устанавливая в систему. Применение программы checkinstall и конвертора для RPM пакетов как alien не поддержаны.

Debian based distributions' often use different file locations for various applications which could cause system instability and some packages won't install due to unresolvable dependencies from different package naming schemes or odd version numbers. For instance a different version of glibc could cause the application to not even run.

Обновление системы - dist-upgrade

apt-get dist-upgrade - это поддержанный метод, чтобы приводить инсталляцию sidux на самое новое состояние. Применение графических инсталляционных менеджеров не рекомендуется. Подробнее здесь: Обновление системы - dist-upgrade

Также dist-upgrade рекомендуется только при законченном графическом сервере X. Чтобы оканчивать графический сервер, вводят команду init 3 в консоль как root, или останавливают его с ctrl+alt+f1 (или ctrl+alt+f2 и т. д. при применении следующих виртуальных терминалов). После этого обновления системы конечно выполнимы.

Конфигурация сетей

Утилита ceni предназанчен для быстрой конфигурации сетевых карты (Ethernet и WLAN). Скриптом сканируются беспроволочные сети. Кроме того, можно выбирать методы кодирования WEP и WPA и использовать бекэнд wireless-tools или wpasupplicant для конфигурации беспроволочных сетей. Ethernet is straight forward if using dhcp (automatic ip address assigning) or you can manually set it from netmasks to nameservers.

Утилита ceni запускается командой Ceni или ceni. Если скрипта неt в наличии, инсталлируют это с помощью команды "apt-get install ceni".

Среды рабочего стола

KDE, XFCE и fluxbox являются оконными менеджерами по умолчанию в sidux. sidux не поддерживает Gnome. Как известно gnome в Debian Sid является довольно нестабильным и может приводить к проблемам. Некоторые пользователи в sidux форумов, в вики или в IRC чате могут иметь опыт и готовы помочь Вам, иначе Вы должны решаться проблемы с gnome на свой страх и риск.
