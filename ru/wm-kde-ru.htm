Рабочий стол KDE зависает

Ни в коем случае не используйте  непосредственно кнопку Reset! Это бы повредило файловую систему и вызвало бы потерю данных. В любом случае, файловая система является после такой жесткой перезагрузки (filesystem not clean) не чистой. Если даже выход в терминал, с STRG+ALT+BACKSPACE, или рестарт сервера X с STRG+ALT+BACKSPACE, не возможен, всегда имеется еще один вариант:

Клавиша SysRq (находится на немецких клавиатурах большей частью справа и надписана "печать/S-Abf") помогает чисто перезагрузить зависшую систему.

Для этого необходимы следующие комбинации клавиш:
* ALT+SYSRQ+R (вновь обретается контроль над клавvиатурой)
* ALT+SYSRQ+S (запускает sync)
* ALT+SYSRQ+E (посылает term на все процессы кроме init)
* ALT+SYSRQ+I (посылает kill на все процессы кроме init)
* ALT+SYSRQ+U (файловые системы будут присоединены read-only , избегает fsck при новой загрузке)
* ALT+SYSRQ+B (перезапускает заново без прежних шагов (Hardreset)).

Лучше всего подождать несколько секунд, так как, например, окончание процессов после каждого шага может немного продолжаться . Можно запомнить буквы, например, в предложении "Reboot System Even If Utterly Broken"

Не возможно начало сеанса в KDE

Содержание директории /tmp удаляется обычно при каждой перезагрузке, так что также удаляются необходимые для сервера X директории и Sockets.

В нормальном процессе загрузки выполняется скрипт x11-common, который заново создаёт эти директории для XOrg.

Возможно, этот скрипт при загрузке не был вызван. Необходимую ссылку восстанавливают так:

# dpkg-reconfigure x11-common

KDE needs a 5% allocation of the partition where the directory /tmp resides for temporary files to be created upon login. If you are running with a 95% full partition you will not be able to login with KDE and will be dropped into tty.

The same goes for kdm looping but not allowing you to log in. A solution is to log into a tty so you can delete and/or clear out some no longer needed applications or files

Alternatively you can use an X window manager that does not require so much space from the system ( for instance fluxbox is already present in a sidux install), or chroot using a sidux live-CD/DVD to clean out the partition to allow you to boot to KDE.

85% is the recommended absolute maximum for a partition that KDE accesses for its /tmp files.(15% free).

Изменение темы KDE

Все изменения нужно проводить по обыкновению как root.

Активация темы

В Debian в KDM темы обычно не активированы. Темы просто активируют вставкой следующей строки в файл /etc/kde3/kdm/kdmrc :

#в разделе [X-*-Greeter] 
UseTheme=true
#вставить
Смена тем

Если темы уже активированы в KDM, то будет использоваться тема по умолчанию "circles". Это можно изменить, если строку

Theme=/usr/share/apps/kdm/themes/circles

изменить на, например,

Theme=/usr/share/apps/kdm/themes/
Деактивирование тем

Если бы хотелось деактивировать тему, то  достаточно следующее изменение в файле /etc/kde3/kdm/kdmrc :

UseTheme=false

Замечание: KDM необходимо после изменений в файле kdmrc перезапустить.

Простой путь к использованию KDM-темы, инсталлировать её как root:

apt-get install kdmtheme

kdmtheme является менеджером тем KDM. Он имеет в себе контрольный KDE модуль (KCM), который делает возможным простую инсталляцию, удаление и изменение тем.
